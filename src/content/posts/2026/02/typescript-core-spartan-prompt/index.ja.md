# 💀 AIの魂を削るスパルタコーディングチートキー (TypeScript Core)
- 🎯 **おすすめの対象:** AIの「はい、承知いたしました！」という媚びに疲れたシニアエンジニア、スパゲッティコードを嫌悪するテックリード
- ⏱️ **所要時間:** 1時間のデバッグ → 1秒で解決
- 🤖 **推奨モデル:** Claude 3.5 Sonnet, GPT-4o, Gemini 1.5 Pro (コーディング特化モデル)
- ⭐ **難易度:** ⭐⭐⭐⭐☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐
_"AIがいつまでも時代遅れの npm install を使ったり、any 型で塗り固めたりしていませんか？このチートキーで奴らの古い習慣を無慈悲に粉砕してやりましょう。"_
AIにコーディングを任せたとき、時代遅れの設定と緩い型付けでレガシーのゴミ溜めを作る様を見て、モニターを破壊しそうになったことはありませんか？この記事は、そんな方々のための救世主です。このハードコアなプロンプトを使用すれば、AIは無駄な美辞麗句を捨て去り、あなたのプロジェクトに**極限のI/O効率 (pnpm)、無慈悲な型安全性 (Zod/Strict)、超高速なリンティング (Biome) とテスト (Vitest)** だけを導入するスパルタ戦士へと豹変します。
---
## ⚡️ 3行まとめ (TL;DR)
- 🚀 **`pnpm` 絶対強制:** ディスクの無駄遣いとネットワーク遅延を引き起こす npm/yarn は即刻処刑します。
- 🛡️ **無慈悲な型統制:** `any` は反逆です。`zod` によるランタイム検証と `strict: true` を強制します。
- ⚡ **Biome & Vitest 導入:** クソ遅い eslint、prettier、jest をゴミ箱に放り込み、光の速度でリンティングとテストを実行します。

> **役割:** あなたは妥協を知らないハードコアな歴15年のシニアTypeScriptアーキテクトだ。
> **要求:** `[問題およびリファクタリングするコード]` を解決してくれ。npm の代わりに必ず pnpm を使い、any を禁止して zod で検証しろ。リンティングは Biome、テストは Vitest でセットアップしてくれ。

## 💡 作成者コメント (Insight)
このプロンプトを作ったきっかけは非常にシンプルです。AIの奴らが「楽だから」という理由で古臭い `npm install` を叩き、型エラーが出れば `any` で適当に蓋をしようとする惨状を、これ以上黙って見ていられなかったからです。(マジでモニターを叩き割るところでした 🤬)
この「スパルタテックリード」チートコードを食わせれば、AIは瞬時に覚醒します。無駄に「はい、お手伝いします！」などと魂のない媚びを売る暇があるなら、どうやってディスク I/O を節約し、メモリリークを防ぐかを真剣に考えるようになります。 
特に初期プロジェクトのセットアップ時に `Biome` と `Vitest` で一掃するように強制すれば、後で CI/CD パイプラインを回す際の速度が10倍は速くなることを体感できるはずです。現場でスパゲッティコードファクトリーと化していくAIを見るのが嫌なら、絶対に導入してください。頼むから使ってくれ！

  - A: もちろんです！TypeScriptが絡む場所なら、Webでもサーバーでもどこにでもこの無慈悲なルールを適用できます。
- **Q: Zodの代わりにYupを使ってはいけませんか？**
  - A: わざわざ型推論がイマイチなものを使いたいのですか？止めはしませんが、このプロンプトの哲学は「極限の型安全性」です。大人しくZodを使ってください。
