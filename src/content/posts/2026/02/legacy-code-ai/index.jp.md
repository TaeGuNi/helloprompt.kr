## 📝 Modernizing Legacy Code (レガシーコードのモダナイゼーション)

- **🎯 おすすめの対象者:** シニアエンジニア、テックリード、技術的負債に悩むすべての開発者
- **⏱️ 所要時間:** 数日〜数週間 → AI活用で数時間に短縮
- **🤖 推奨モデル:** Claude 3.5 Sonnet, GPT-4o（コーディングやリファクタリングに特化したモデル）

- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果性:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「ドキュメントすら存在せず、誰もが触れるのを嫌がるスパゲッティコード。AIの力を借りて、安全かつ確実にモダンなアーキテクチャへと生まれ変わらせてみませんか？」_

レガシーコードの保守は、多くの開発チームを疲弊させ、新たなイノベーションの足かせとなる「技術的負債」の最たるものです。ドキュメントすら存在せず、誰も全貌を把握していないスパゲッティコードに手を入れる恐怖……。しかし、昨今のAI技術の飛躍的な進化により、この重い負債を返済するアプローチは劇的に変わりました。本記事では、AIの力をフル活用し、古びたコードを安全かつスピーディにモダンなアーキテクチャへと刷新するための、超実践的なプロンプトと戦略を余すところなく解説します。

---

## ⚡️ 3行まとめ (TL;DR)

1. AIの圧倒的な解析力を活かし、ブラックボックス化したコードの構造と依存関係を瞬時に紐解く。
2. 一気通貫ではなく「段階的なリファクタリング」を指示し、デグレ（退行バグ）のリスクを最小限に抑える。
3. リファクタリングと同時に自動テストを生成させ、モダナイズ後の品質と安全性を強固に担保する。

---

## 🚀 解決策: 「レガシー・レスキュー・プロンプト」

### 🥉 ベーシック版 (Basic Version)

まずは対象コードの全貌をざっくりと掴みたい、という状況に最適なプロンプトです。

> **役割:** あなたはシニアソフトウェアエンジニアです。
> **要求:** 以下の `[古いコード]` を分析し、どのような処理を行っているのかを箇条書きで分かりやすく説明した上で、最新の言語仕様に沿ったクリーンなコードに書き直してください。

### 🥇 プロ版 (Pro Version)

実稼働に耐えうるプロダクションレベルのリファクタリングと、堅牢なテストコードの生成を同時に完遂させたい場合は、こちらのプロンプトを使用してください。

> **役割 (Role):** あなたはレガシーシステムのモダナイゼーションを専門とする、10年以上の経験を持つエリートソフトウェアアーキテクトです。
>
> **状況 (Context):**
>
> - 背景: 現在のコードベースは `[古いバージョンの言語/フレームワーク]` で書かれており、保守性が著しく低下しています。また、ドキュメントも一切存在しません。
> - 目標: このコードの既存の振る舞いを完全に維持したまま、`[最新バージョンの言語/フレームワーク]` のベストプラクティスに従ってリファクタリングを行い、同時にユニットテストを追加することです。
>
> **要求 (Task):**
>
> 1. まず、提供されたコードの論理的なフローと隠れたエッジケースを詳細に分析してください。
> 2. SOLID原則に基づき、コードを最新のアーキテクチャ仕様へとリファクタリングしてください。
> 3. リファクタリング後のコードに対し、`[使用するテストフレームワーク]` を用いた網羅的なユニットテストを作成してください。
> 4. `[特定のビジネスロジックや制約]` が存在する場合は、必ずそれに従ってください。
>
> **制約事項 (Constraints):**
>
> - 出力は「分析結果」、「リファクタリング後のコード」、「ユニットテスト」の3つのセクションに明確に分け、コードは必ずMarkdownのコードブロックで記述してください。
> - 元のコードに存在する「副作用（Side Effects）」を独自の判断で排除せず、従来の振る舞いを厳格に維持してください。
>
> **注意事項 (Warning):**
>
> - 使用されているライブラリの非推奨（Deprecated）メソッドを推測で補わないでください。常に最新の公式ドキュメントに基づいた、確実で安全なコードのみを生成してください。

## 💡 筆者のコメント (Insight)

レガシーコードに立ち向かう際、誰もが陥りがちな**最悪のアンチパターン**があります。それは**「AIにコード全体を一気に書き換えさせてしまうこと」**です。AIはシステム全体の深い文脈を完璧に理解しているわけではなく、ただ表面的に「美しい」コードを出力しようとする傾向があります。その結果、「一見バグに見えるが、実は過去の歴史的経緯から絶対に外せない仕様」まで容赦なく消し去ってしまう危険性があるのです。

この「プロ版」プロンプトでは、そうした大事故を防ぐため、**「既存の振る舞いの厳格な維持」**と**「テストコードの同時生成」**という極めて強い制約をAIに課しています。現場の実務で適用する際の最大のコツは、いきなり数千行の巨大なクラスを丸投げしないことです。まずは単一の関数や、影響範囲の小さいモジュール単位に細かく分割し、**少しずつ確実にモダナイズを進めていく**ことが成功の絶対条件となります。

## 🙋 よくある質問 (FAQ)

- **Q: 独自の社内フレームワークで書かれたコードでも機能しますか？**
  - A: はい、問題なく機能します。ただし、AIが文脈を正確に捉えられるよう、プロンプトの「状況 (Context)」セクションに、その社内フレームワーク特有のコーディング規約や制約事項を事前知識としてしっかり追記しておくことが、精度の高い出力を引き出す鍵となります。

- **Q: セキュリティの観点から、機密性の高いコードをAIに渡しても問題ありませんか？**
  - A: 大前提として、必ず所属企業のセキュリティポリシーを遵守してください。実務で利用する際は、入力データがAIの学習モデルに利用されない「オプトアウト設定済み」のエンタープライズ版（ChatGPT Enterprise、Claude for Workなど）の使用を強く推奨します。また、APIキーやハードコードされた個人情報などは、入力前に必ずダミーデータへマスキングする運用を徹底してください。

## 🧬 プロンプトの解剖 (Why it works?)

1. **エッジケースへの着目指示:** 単に「コードを綺麗にして」と丸投げするのではなく、意図的に「隠れたエッジケースの分析」を要求しています。これにより、AIに対してコードの深層ロジックを読み解く思考プロセスを強制的に踏ませており、結果としてChain-of-Thought（思考の連鎖）と同等の高い精度を引き出しています。
2. **副作用の厳格な維持制約:** リファクタリングにおいて最も恐ろしいのは、気づかぬうちに既存の挙動を変えてしまうことです。そこで「既存の副作用を勝手に排除しない」という強力な制約（Constraints）を設けることで、後方互換性が意図せず破壊されるという悲劇をシステムレベルで未然に防いでいます。

## 📊 実証: Before & After

### ❌ Before (入力)

```javascript
// 10年前に書かれたコールバック地獄のレガシーコード
function getUserData(userId, callback) {
  db.query("SELECT * FROM users WHERE id = " + userId, function (err, result) {
    if (err) {
      callback(err, null);
    } else {
      cache.set("user_" + userId, result[0], function (err) {
        if (err) console.log(err);
        callback(null, result[0]);
      });
    }
  });
}
```

### ✅ After (結果)

```typescript
// 最新のTypeScriptとAsync/Awaitを用いて堅牢にリファクタリングされたコード
import { db } from "./database";
import { cache } from "./cache";
import { User } from "./types";

/**
 * ユーザーデータをデータベースから取得し、キャッシュに保存します。
 * SQLインジェクションを防ぐためパラメータ化クエリを使用しています。
 */
export async function getUserData(userId: number): Promise<User | null> {
  try {
    // パラメータ化クエリを使用して安全にデータを取得
    const [user] = await db.query<User[]>("SELECT * FROM users WHERE id = ?", [
      userId,
    ]);

    if (!user) {
      return null;
    }

    // キャッシュ保存（エラーはログに記録するが、処理はブロックしない）
    cache.set(`user_${userId}`, user).catch((err) => {
      console.error(`Failed to cache user ${userId}:`, err);
    });

    return user;
  } catch (err) {
    console.error(`Error fetching user ${userId}:`, err);
    throw new Error("Failed to retrieve user data");
  }
}
```

## 🎯 結論

現代のAIは、もはや単なる「便利なコード生成ツール」の枠に収まりません。長年にわたって蓄積され、誰も解き明かせなかった技術的負債という名の「暗号」を鮮やかに解読し、理路整然としたアーキテクチャへと整理してくれる、極めて頼もしいペアプログラミングの相棒です。

「テストの生成」と「副作用の維持」という適切な制約を組み込んだこのプロンプトを駆使すれば、これまで誰もが敬遠してきたレガシーコードとの孤独な戦いは、劇的に創造的で安全なプロセスへと生まれ変わるはずです。

さあ、恐れることはありません。AIの圧倒的な力を借りて、安全で快適なコード・モダナイゼーションへの第一歩を力強く踏み出しましょう！ 🍷
