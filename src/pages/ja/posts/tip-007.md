_"メールアドレスの有効性チェック一つしようとして `^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$` これを解読していますか？"_

正規表現(Regular Expression)は開発者に必須ですが、最も扱いたくない領域の一つです。書くたびに文法を調べなければならないし、他人が書いたものは宇宙語のようです。もう苦しまずにAIに「パターン」だけ伝えてください。

---

## ⚡️ 3行要約 (TL;DR)

1. 複雑な正規表現の文法を覚えたり検索するのに苦労しないでください。
2. 希望するパターン（電話番号、メールなど）だけ言えば、AIが正規表現を書いてくれて解釈までしてくれます。
3. テストコードまで一緒にリクエストして、正規表現の正確性をすぐに検証できます。

## 🧬 プロンプト解剖 (Why it works?)

---

## 📊 証明: Before & After

### ❌ Before (入力)

```text
韓国の携帯電話番号の正規表現ちょっと書いて。ハイフンあってもなくても。010以外に011もOK。
```

<br>

### ✅ After (結果)

```javascript
// JavaScript 例

const regex = /^01([0|1|6|7|8|9])-?([0-9]{3,4})-?([0-9]{4})$/;

// テスト
const phone1 = "010-1234-5678"; // マッチ O
const phone2 = "01012345678"; // マッチ O
const phone3 = "02-123-4567"; // マッチ X

console.log(regex.test(phone1)); // true
```

### 🔍 解説

- `^`: 文字列の始まり
- `01([0|1|6|7|8|9])`: 01で始まり、その後に0,1,6,7,8,9のいずれかが来る (識別番号)
- `-?`: ハイフン(-)が0個または1個ある (選択的)
- `([0-9]{3,4})`: 数字0-9が3個または4個来る (真ん中の桁)
- `([0-9]{4})`: 数字0-9が4個来る (最後の桁)
- `$`: 文字列の終わり

## 🎯 結論

正規表現の文法書を開かないでください。あなたが望む「パターン」が何なのかだけ明確に知っていれば良いのです。
