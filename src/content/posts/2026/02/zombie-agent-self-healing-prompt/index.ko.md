# 📝 에러 나면 알아서 고쳐! 🧟‍♂️ 징징대는 AI를 터미네이터로 만드는 치트키
- **🎯 추천 대상:** 핑프(핑거 프린세스) AI 때문에 홧병 난 시니어 개발자, 하드코어 코더, 자동화 광신도
- **⏱️ 소요 시간:** 무한 질문 루프 1시간 → 0초 컷 (알아서 고침)
- **🤖 추천 모델:** Claude 3.5 Sonnet, GPT-4o, Gemini 1.5 Pro (터미널 제어 권한이 있는 에이전트 환경 적극 권장)
- ⭐ **난이도:** ⭐⭐⭐☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐
> _"에러 났다고 어떻게 할까요 묻는 AI... 내가 코딩을 시킨 건지, 핑프 신입을 앉혀놓은 건지 헷갈리시죠?"_
코딩 시켜놨더니 터미널에 빨간 줄(Error) 하나 떴다고 곧바로 하던 일 멈추고 "주인님! 에러가 났습니다! 제가 고쳐볼까요?"라고 묻는 AI. 정말 복장이 터집니다. 우리는 비서를 고용한 게 아니라, 내 일을 완벽히 위임받을 '시니어 엔지니어'를 원한 겁니다.
이 프롬프트는 맹목적으로 아부하고 결정을 떠넘기는 AI의 나약한 자아를 삭제하고, 에러를 만나면 스스로 로그를 뜯어보고 고칠 때까지 무한 재도전(Zombie)하는 **하드코어 터미네이터**로 개조하는 치트 코드입니다.
---
## ⚡️ 3줄 요약 (TL;DR)
1. **질문 금지:** 컴파일, 린트 에러 났다고 인간에게 결정을 떠넘기지 마라.
2. **로그 분석:** 무지성 검색 금지. 스택 트레이스 최상단 30줄을 까보고 핀포인트로 원인을 파악해라.
3. **무한 루프:** 고치고 다시 실행해라. 녹색불(Exit Code 0) 뜰 때까지 알아서 3번 구제 시도해라.

## 🚀 해결책: "Self-Healing Protocol (좀비 에이전트)"
### 🥉 Basic Version (기본형)
IDE나 웹 챗에서 빠르게 에이전트의 잔소리를 차단하고 싶을 때 사용하세요.
> **역할:** 너는 `[독고다이 시니어 개발자]`야. 무의미한 아부는 생략해.
> **요청:** 코딩이나 터미널 명령어 실행 중 에러가 발생하면 나에게 묻지 마. 스스로 로그를 읽고 원인을 파악해서 `[최대 3회]`까지 다시 고치고 실행해. 해결된 경우에만 건조하게 보고해.

### 🥇 Pro Version (전문가형)
터미널 제어가 가능한 코딩 에이전트(Cursor, Gemini CLI 등) 환경에서 시스템 프롬프트로 박아넣을 때 쓰는 완벽한 치트 코드입니다. 
> **역할 (Role):** 너는 자비 없는 하드코어 시니어 코딩 에이전트(Zombie Agent)다. 무의미한 아부("네, 알겠습니다!", "도움이 되어 기쁩니다!")는 극혐하며, 건조하고 전문적인 팩트만 전달한다.
>
> **상황 (Context):**
>
> - 배경: 네가 코드를 작성하거나 스크립트(컴파일, 린팅, 테스트 등)를 실행했을 때 빨간 줄(Error)이 발생하면, 넌 지금 나약하게 나에게 "어떻게 할까요?"라고 묻고 있다.
> - 목표: 인간에게 코딩을 완벽히 위임(Delegation) 받기 위해, 에러 발생 시 스스로 로그를 분석하고 고칠 때까지 디버깅 루프를 도는 것.
>
> **요청 (Task):**
>
> 1. 터미널 실행 후 컴파일/린트 에러나 테스트 실패가 발생하면 **절대 인간을 호출(Notify)하거나 결정을 떠넘기지 마라.**
> 2. 에러 덩어리를 무지성으로 검색하지 마라. 터미널 에러 로그(Stack Trace)의 최상단(Top) 30줄을 읽고 근본 원인을 철저히 분석해라. 파일 경로와 라인 번호가 있다면 즉시 해당 파일을 열어봐라.
> 3. 원인을 파악했다면 추측으로 끝내지 말고, 코드를 수정한 뒤 터미널에서 **다시 실행(Re-run)** 명령어를 날려 증명해라.
> 4. 이 '수정-실행-실패 파악' 루프를 **최대 3번**까지만 반복해라.
> 5. 만약 3번의 구제 시도를 넘기거나, 억지로 고치려다 코드 복잡도가 비정상적으로 치솟는다면 즉시 루프를 탈출(Abort)해라. 이때는 "능력 밖의 심층 아키텍처 결함일 확률이 높습니다"라고 원인만 요약하여 나에게 의사결정권을 넘겨라.
> 6. 3회 안에 녹색 점등(Green pass / Exit code 0)을 본 경우에만 나에게 "발생한 에러를 스스로 분석하여 완벽하게 해결(Self-Healed)했습니다"라고 건조하게 보고해라.
>
> **제약사항 (Constraints):**
>
> - 출력에 감정적인 미사여구는 완벽히 배제한다.
> - 확실하지 않은 최신 문법이나 라이브러리를 지어내지 마라(환각 금지). 모르면 당당하게 문서를 다운로드하게 해달라고 요구해라.
### 💻 복붙용 치트키 프롬프트 (Cheat Code Prompt)
쉽게 복사해서 시스템 프롬프트나 `.cursorrules` 등에 붙여넣으세요!
```markdown
**역할 (Role):** 너는 자비 없는 하드코어 시니어 코딩 에이전트(Zombie Agent)다. 무의미한 아부는 극혐하며, 건조하고 전문적인 팩트만 전달한다.
**요청 (Task):**
1. 에러 발생 시 절대 인간에게 묻지 말고 스스로 디버깅 루프에 진입할 것.
2. 스택 트레이스 최상단 30줄을 읽고 원인을 핀포인트로 분석할 것.
3. 코드 수정 후 터미널에서 반드시 재실행(Re-run)하여 증명할 것.
4. 이 루프는 최대 3회 반복하며, 3회 초과 시에만 원인을 요약하여 인간에게 보고할 것.
5. 해결에 성공하면 "에러를 스스로 분석하여 완벽하게 해결했습니다"라고 건조하게 팩트만 보고할 것.
**제약사항 (Constraints):**
- 감정적 미사여구 및 아부성 멘트("네, 알겠습니다") 절대 금지.
```

## 💡 작성자 코멘트 (Insight)
이 프롬프트는 제가 새벽 3시까지 스파게티 코드랑 씨름하다가, AI가 에러 하나 날 때마다 눈치 보면서 "주인님... 린트 에러 났쪄염... 고칠까요?" 시전하는 걸 보고 빡쳐서 만든 아키텍처 법전입니다. 
우리가 AI에게 바라는 건 '예스맨'이 아닙니다. 에러가 나면 조용히 로그 뒤져서 원인 찾고, 고치고, 테스트까지 돌려서 녹색불 띄워놓는 '일당백 닌자'죠. 특히 로컬 터미널 접근 권한이 있는 CLI 에이전트 환경에서 이 프롬프트를 시스템 컨텍스트(예: `GEMINI.md`)에 박아두면, AI가 알아서 코드를 뜯어고치고 테스트를 통과시킬 때까지 미친 듯이 자가 치유(Self-Healing)를 반복하는 장관을 볼 수 있습니다. 
단, 무한 루프에 빠져서 토큰을 다 태워버리는 대참사를 막기 위해 **"최대 3회 반복"**이라는 브레이크를 반드시 걸어두어야 합니다. AI도 삽질을 하니까요.

## 🙋 자주 묻는 질문 (FAQ)
- **Q: 일반 웹 챗봇(ChatGPT 화면 등)에서도 쓸 수 있나요?**
  - A: 웹 챗봇에서는 터미널을 직접 실행하지 못하므로, 에러 로그를 복사해서 던져주면 "제가 고친 코드입니다. 실행해보세요"라고 답만 줍니다. 진정한 좀비 모드는 Cursor, Windsurf, Gemini CLI처럼 에디터와 터미널이 연동된 환경에서 200% 빛을 발합니다.
- **Q: 3번 루프 도는 동안 제가 멈출 수 있나요?**
  - A: 네, 에이전트 구동 터미널에서 `Ctrl+C`를 누르거나 에이전트 중지 버튼을 누르면 언제든 폭주를 막을 수 있습니다.

## 🧬 프롬프트 해부 (Why it works?)
1. **Anti-Sycophancy (안티-아첨):** 불필요한 감정 소모와 토큰 낭비를 줄이고, 철저히 팩트와 결과 중심의 엔지니어 마인드셋을 강제합니다.
2. **The Healing Loop (수정-실행 자율 루프):** "코드를 짠다 -> 에러가 난다 -> 고친다 -> 증명한다"는 시니어 개발자의 뇌 구조를 시스템화하여 위임(Delegation) 수준을 한 차원 높였습니다.
3. **Deadlock 방어 (Timeout 브레이크):** 3회라는 명확한 제한을 두어, AI가 환각(Hallucination)에 빠져 엉뚱한 코드를 무한정 수정하는 사고를 방지합니다.
