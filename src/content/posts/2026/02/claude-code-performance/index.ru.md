- **🤖 Рекомендуемые модели:** Claude 3.5 Sonnet, Claude 3 Opus

В последнее время сообщества на Reddit и X (Twitter) переполнены жалобами: некогда блестящий Claude Code начал выдавать обрывочные ответы и откровенно халтурить на сложных задачах. Действительно ли модель «отупела», или мы наблюдаем агрессивную оптимизацию ресурсов со стороны Anthropic? Независимо от причин, инженеры не могут зависеть от капризов нейросетей. В этой статье мы разберем суть проблемы и поделимся ультимативным «анти-лень» промптом, который заставит Claude вновь выкладываться на все 100%.

## ⚡️ 3 тезиса коротко (TL;DR)

1. **Жалобы обоснованы:** Разработчики массово сталкиваются с «ленивым кодом» и заглушками из-за скрытых внутренних оптимизаций провайдера.
2. **Контекст решает всё:** Отсутствие жестких рамок в запросе дает модели легальное право экономить токены на вашем коде.
3. **Строгие ограничения (Constraints) — ваш лучший друг:** Добавление ультимативных запретов в промпт полностью устраняет проблему усеченного кода.

## 🚀 Решение: Промпт «Анти-Лень» для Claude

### 🥉 Базовая версия (Basic Version)

Используйте для небольших правок и быстрых скриптов, когда нужен немедленный результат без сокращений.

> **Роль:** Ты `[Senior-разработчик]`.
> **Запрос:** Напиши полный код для `[описание задачи]`, не пропуская ни одной строки. Тебе категорически запрещено использовать комментарии-заглушки вида "// остальной код здесь".

### 🥇 Продвинутая версия (Pro Version)

Идеально подходит для сложных архитектурных изменений, создания новых компонентов или масштабного рефакторинга файлов.

> **Роль (Role):** Ты `[Ведущий Software Engineer]`, специализирующийся на `[ваша технология, например: React и Node.js]`.
>
> **Контекст (Context):**
>
> - Ситуация: Мне нужно получить полностью рабочий и детализированный код без каких-либо сокращений.
> - Цель: Написать `[абсолютно полный, готовый к продакшену файл/модуль]`.
>
> **Запрос (Task):**
>
> 1. Полностью реализуй `[название модуля или функционала]`.
> 2. Включи всю необходимую бизнес-логику, обработку краевых случаев (edge cases) и безопасную типизацию.
> 3. Детально пропиши функцию `[ключевая часть логики, которую нельзя упустить]`.
>
> **Ограничения (Constraints):**
>
> - Ответ должен содержать исключительно код в формате Markdown.
> - КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО использовать плейсхолдеры, такие как `// TODO`, `// ...existing code...` или `// реализация здесь`.
> - Код должен быть полностью готов к копированию и вставке (Copy-Paste Ready).
>
> **Предупреждение (Warning):**
>
> - Если код физически не помещается в лимит токенов одного ответа, остановись на логически завершенном месте и спроси "Продолжить генерацию?". Ни в коем случае не сокращай саму логику в угоду лимитам.

## 💡 Авторский комментарий (Insight)

Феномен деградации моделей (AI Slop или «лень ИИ») — это почти всегда результат скрытого снижения лимитов на генерацию (max_tokens) или агрессивного системного промптинга со стороны разработчиков ИИ с целью экономии серверных мощностей. Как инженеры, мы обязаны понимать: нейросеть всегда выберет путь наименьшего сопротивления, если мы оставим ей такую лазейку.

В моей практике блок **Ограничения (Constraints)** работает как своеобразный джейлбрейк для продуктивности. Claude в разы лучше реагирует на жесткие запретительные формулировки («КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО»), чем на вежливые просьбы («пожалуйста, напиши код полностью»). Этот неочевидный подход экономит часы ручного сшивания разрозненных кусков кода.

## 🧬 Анатомия промпта (Why it works?)

1. **Управление поведением через запреты (Constraints):** Прямой запрет на использование плейсхолдеров навсегда закрывает лазейку для экономии токенов.
2. **Задание высоких стандартов (Role):** Роль «Ведущего инженера» заставляет модель подтягивать качество архитектуры и не пренебрегать критически важными деталями.
3. **Легальный выход из ситуации (Warning):** Разрешая модели спросить "Продолжить?", мы избегаем молчаливого обрыва кода на полуслове. Это позволяет получить весь огромный скрипт частями, не теряя ни капли логики.

### ✅ После (С использованием нашего промпта)

````text
Пользователь: [Продвинутая версия промпта]

Claude:
```javascript
import express from 'express';
import bcrypt from 'bcrypt';
import { body, validationResult } from 'express-validator';
import { User } from '../models/User.js'; // Предполагаемая модель БД

const router = express.Router();

router.post('/register',
  [
    body('email').isEmail().withMessage('Неверный формат email'),
    body('password').isLength({ min: 8 }).withMessage('Пароль должен содержать минимум 8 символов'),
    body('username').notEmpty().withMessage('Имя пользователя обязательно')
  ],
  async (req, res) => {
    try {
      const errors = validationResult(req);
      if (!errors.isEmpty()) {
        return res.status(400).json({ errors: errors.array() });
      }

      const { email, password, username } = req.body;

      const existingUser = await User.findOne({ email });
      if (existingUser) {
        return res.status(409).json({ message: 'Пользователь с таким email уже существует' });
      }

      const saltRounds = 12;
      const hashedPassword = await bcrypt.hash(password, saltRounds);

      const newUser = new User({
        email,
        username,
        password: hashedPassword
      });

      await newUser.save();

      // Исключаем хэш пароля из ответа
      const userResponse = { id: newUser._id, email: newUser.email, username: newUser.username };

      return res.status(201).json({
        message: 'Регистрация прошла успешно',
        user: userResponse
      });

    } catch (error) {
      console.error('Ошибка при регистрации:', error);
      return res.status(500).json({ message: 'Внутренняя ошибка сервера при регистрации пользователя' });
    }
  }
);

export default router;
```
````

## 🎯 Заключение

Инструменты на базе ИИ могут временно терять «фокус» из-за высоких серверных нагрузок или внутренних обновлений, но профессиональный промпт-инжиниринг всегда спасает ситуацию. Не позволяйте нейросетям халтурить — устанавливайте жесткие правила игры, и качество вашего кода останется на недосягаемой высоте.
