개발자라면 누구나 한 번쯤 겪어본 상황입니다. 수천 줄의 엑셀 데이터를 SQL로 바꾸기 위해 엑셀 함수(`CONCATENATE`)와 씨름하거나, CSV로 변환해 힘겹게 임포트하던 기억. 이제 AI에게 맡기세요. 데이터 타입 추론부터 정규화, 그리고 완벽한 INSERT 쿼리 생성까지 한 번에 끝낼 수 있습니다.

## ⚡️ 3줄 요약 (TL;DR)

1. 엑셀의 헤더와 샘플 데이터만 복사하면 테이블 스키마(DDL) 자동 생성
2. 데이터 타입(INT, VARCHAR, DATETIME 등)을 AI가 똑똑하게 추론
3. 수천 건의 데이터도 포맷에 맞춰 INSERT 문(DML)으로 즉시 변환

### 🥉 Basic Version (기본형)

단순히 엑셀 데이터를 SQL INSERT 문으로 바꾸고 싶을 때 사용하세요.

> **역할:** 너는 데이터베이스 엔지니어야.
> **요청:** 아래 엑셀 데이터를 [MySQL/PostgreSQL] 문법에 맞는 `CREATE TABLE`과 `INSERT` 문으로 변환해줘. 컬럼명은 영문으로 적절히 변환해.

<br>

### 🥇 Pro Version (전문가형)

데이터 정규화, 타입 최적화, 제약 조건까지 고려한 완벽한 쿼리가 필요할 때 사용하세요.

> **역할 (Role):**
> 당신은 데이터 모델링과 SQL 최적화에 능통한 **수석 데이터베이스 관리자(DBA)**입니다. 더럽고 정리되지 않은 원시 데이터(Raw Data)를 분석하여 효율적인 RDBMS 구조로 설계하는 능력이 탁월합니다.
>
> **상황 (Context):**
> 엑셀(또는 CSV) 형태의 비정형 데이터가 주어졌습니다. 이 데이터는 컬럼명이 한글이거나 모호할 수 있으며, 데이터 타입이 명시되어 있지 않습니다. 이를 실제 운영 환경(Production)에 바로 적용할 수 있는 SQL 스크립트로 변환해야 합니다.
>
> **요청 (Task):**
> 제공된 데이터를 분석하여 다음 단계에 따라 SQL 스크립트를 작성하세요.
>
> 1.  **데이터 분석 (Analyze):** 각 컬럼의 데이터 예시를 보고 적절한 데이터 타입(INTEGER, VARCHAR, BOOLEAN, DATE 등)을 추론하세요. `NULL` 가능 여부도 판단하세요.
> 2.  **스키마 설계 (DDL):**
>     - 직관적인 영문 컬럼명으로 매핑하세요 (예: '고객명' -> `customer_name`).
>     - 적절한 Primary Key(PK)를 선정하거나, 필요하다면 `id` 컬럼을 추가하세요.
>     - `CREATE TABLE` 문을 작성하세요. (코멘트로 원본 한글 컬럼명을 명시하세요).
> 3.  **데이터 변환 (DML):**
>     - 제공된 데이터를 기반으로 `INSERT INTO` 문을 작성하세요.
>     - 날짜 형식이 제각각이라면 표준 포맷('YYYY-MM-DD')으로 통일하세요.
>     - 문자열에 포함된 따옴표(') 등을 적절히 이스케이프 처리하세요.
>
> **제약사항 (Constraints):**
>
> - **DBMS:** MySQL 8.0 호환 문법을 사용하세요. (또는 사용자가 지정한 DB)
> - **네이밍 컨벤션:** snake_case를 사용하세요.
> - **최적화:** `VARCHAR` 길이는 데이터 길이를 고려해 여유 있게 잡되, 너무 낭비하지 마세요(예: `VARCHAR(255)` 남발 금지).
>
> **입력 데이터:**
> ```[여기에 엑셀/CSV 데이터를 붙여넣으세요]```

## 💡 작성자 코멘트 (Insight)

이 프롬프트의 강력한 점은 단순히 텍스트를 바꾸는 게 아니라 **'모델링'**을 수행한다는 점입니다.
AI는 데이터 값(예: '010-1234-5678')을 보고 "아, 이건 전화번호니까 문자열이고 길이는 20자면 충분하겠군"이라고 판단합니다. 또한 '사용여부' 같은 컬럼에 'Y', 'N'이 들어있다면 이를 `ENUM`이나 `TINYINT`로 최적화하여 제안하기도 합니다. 단순 노가다 작업을 고차원적인 설계 작업으로 바꿔주는 도구입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 데이터가 수천 건인데 다 들어갈까요?**
  - A: LLM의 컨텍스트 윈도우 한계가 있습니다. 스키마(DDL) 생성은 전체 헤더와 샘플 10줄로 하고, INSERT 문(DML) 생성은 데이터를 100~500줄씩 나눠서 요청하는 것이 좋습니다.

- **Q: 다른 DB도 지원하나요?**
  - A: 네, 제약사항의 DBMS 항목만 'Oracle', 'SQL Server', 'PostgreSQL' 등으로 바꾸면 해당 문법에 맞춰 생성해줍니다.

## 🧬 프롬프트 해부 (Why it works?)

1.  **Role 부여:** '수석 DBA'라는 역할을 주어 데이터 타입 선정과 네이밍에 전문성을 더했습니다.
2.  **Step-by-Step:** 분석 -> 설계 -> 변환의 과정을 명시하여 논리적인 결과물을 만듭니다.
3.  **Data Cleaning:** 날짜 포맷 통일이나 이스케이프 처리 같은 디테일한 요구사항을 통해 실행 오류를 방지합니다.

## 📊 증명: Before & After

### ❌ Before (입력: 엑셀 복사붙여넣기)

```text
이름    입사일       직급    연봉
홍길동  2023.03.01  대리    5000
김철수  2024-01-02  사원    3500
이영희  23/12/01    과장    6500
```

### ✅ After (결과)

```sql
-- Employees 테이블 생성
CREATE TABLE employees (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL COMMENT '이름',
    joined_at DATE NOT NULL COMMENT '입사일',
    position VARCHAR(20) NOT NULL COMMENT '직급',
    salary INT NOT NULL COMMENT '연봉'
);

-- 데이터 삽입
INSERT INTO employees (name, joined_at, position, salary) VALUES
('홍길동', '2023-03-01', '대리', 5000),
('김철수', '2024-01-02', '사원', 3500),
('이영희', '2023-12-01', '과장', 6500);
```
