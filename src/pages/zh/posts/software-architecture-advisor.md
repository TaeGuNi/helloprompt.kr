## 📊 证明: Before & After

### ❌ Before (输入项目)

“要做个简单的待办事项 (To-Do) App。一个人开发，功能没多少。”

### ✅ After (结果)

**推荐架构:** **MVVM (Model-View-ViewModel)** 或 **Feature-based Structure**

**文件夹结构 (Feature-based):**

```text
src/
├── features/           # 按功能模块
│   ├── todos/
│   │   ├── components/ # 待办相关 UI
│   │   ├── hooks/      # 待办相关逻辑
│   │   └── api/        # 待办 API 调用
│   └── auth/           # 认证相关
├── shared/             # 公共模块
│   ├── components/     # 公共按钮，输入框
│   └── utils/          # 工具函数
├── App.js
└── index.js
```

_理由: 因为项目小，比起复杂的分层，按功能 (Feature) 捆绑更直观，开发速度也更快。_
