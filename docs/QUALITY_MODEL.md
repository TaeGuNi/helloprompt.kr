# ⚖️ 콘텐츠 품질 평가 모델 (Quality Evaluation Model)

Hello Prompt 블로그의 모든 글은 발행 전 이 모델을 통해 엄격하게 심사받아야 합니다.
단순한 체크리스트가 아닌, **"독자의 시간을 아껴주는 최고의 도구인가?"**를 검증하는 기준입니다.

**총점 600점 만점**이며, 6개 항목 중 **단 하나라도 95점 미만**일 경우 해당 글은 즉시 **반려(Reject) 및 재작성**해야 합니다.

---

## 1. 주제 적합성 및 SEO (Relevance & SEO) [100점]

> _"독자가 검색하는 내용이며, 우리 블로그의 색깔과 맞는가?"_

### 🔍 평가 상세 기준

1.  **타겟 페르소나 일치 (30점):**
    - [ ] 주니어(1~3년 차) 직장인, 개발자, 마케터가 겪는 실무적 어려움을 다루는가?
    - [ ] 너무 기초적이지도(예: "AI란 무엇인가"), 너무 학술적이지도(예: "Transformer 아키텍처 분석") 않은가?
2.  **검색 의도(Search Intent) 충족 (30점):**
    - [ ] 독자가 이 키워드를 검색했을 때 기대하는 답변을 첫 문단(Hook)에서 제시하는가?
    - [ ] 제목(H1)과 본문 내용이 일치하는가? (낚시성 제목 금지)
3.  **SEO 최적화 (40점):**
    - [ ] **메인 키워드**가 제목, 첫 문단, URL Slug, 메타 설명에 포함되었는가?
    - [ ] `<h2>`, `<h3>` 태그를 사용하여 구조적으로 작성되었는가?
    - [ ] 이미지 Alt 태그에 설명이 빠짐없이 들어갔는가?

- **📉 감점 요인:** 뉴스 단순 요약, 개인적인 일기, 키워드 스터핑(부자연스러운 키워드 반복).

---

## 2. 실용적 효용성 (Pragmatic Utility) [100점]

> _"내일 출근해서 당장 써먹을 수 있는가?"_

### 🔍 평가 상세 기준

1.  **재현 가능성 (Reproducibility) (40점):**
    - [ ] 제공된 프롬프트를 복사하여 실행했을 때, 작성자가 의도한 결과가 100% 나오는가?
    - [ ] 특정 모델(예: GPT-4)에서만 작동한다면, 이를 명시했는가?
2.  **시간 절감 효과 (Time-to-Value) (30점):**
    - [ ] 이 프롬프트를 사용함으로써 기존 방식 대비 최소 50% 이상의 시간이 절약되는가?
    - [ ] 서론이 길지 않고, 핵심(Prompt)으로 바로 진입하는가?
3.  **결과물의 품질 (Output Quality) (30점):**
    - [ ] AI가 생성한 결과물이 사람의 손을 거의 타지 않아도 될 만큼 완성도가 높은가?
    - [ ] 결과 예시(Before/After)가 시각적으로 확연한 차이를 보여주는가?

- **📉 감점 요인:** 프롬프트 없이 이론만 설명함, 실행 결과가 두루뭉술함.

---

## 3. 프롬프트 완성도 (Prompt Craftsmanship) [100점]

> _"엔지니어링(Engineering) 된 프롬프트인가, 그냥 말 건네기(Chatting)인가?"_

### 🔍 평가 상세 기준

1.  **RTFC 구조 준수 (40점):**
    - [ ] **Role:** 명확한 페르소나(예: "시니어 카피라이터")가 부여되었는가?
    - [ ] **Task:** 구체적인 행동 지시(Action Verb)가 포함되었는가?
    - [ ] **Format:** 출력 형식(표, 마크다운, 코드블럭 등)이 지정되었는가?
    - [ ] **Context/Constraints:** 제약 조건이 명시되었는가?
2.  **변수 설계 (Variable Design) (30점):**
    - [ ] 사용자가 채워야 할 부분(`[ ]` 등)이 명확히 구분되어 있는가?
    - [ ] 변수 입력 가이드가 친절하게 제공되었는가?
3.  **최적화 기법 적용 (30점):**
    - [ ] Chain-of-Thought(단계별 생각 유도), Few-Shot(예시 제공) 등 고급 기법이 적절히 사용되었는가?

- **📉 감점 요인:** "해줘" 수준의 단순 명령어, 구조가 없는 줄글 프롬프트.

---

## 4. 정보의 깊이와 경험 (Depth & Experience) [100점]

> _"구글 애드센스가 싫어하는 'Low Value Content'를 탈피했는가?"_

### 🔍 평가 상세 기준

1.  **인간의 개입 (Human Touch) (40점):**
    - [ ] AI가 뱉은 결과를 그대로 복사하지 않고, 작성자의 해석이나 수정 의견(Comment)이 추가되었는가?
    - [ ] 직접 테스트하며 발생한 오류나 꿀팁(Insight)이 포함되었는가?
2.  **니치(Niche) 타겟팅 (30점):**
    - [ ] 특정 직군이나 특정 상황(Context)을 뾰족하게 겨냥했는가? (예: "마케터" vs "B2B SaaS 마케터")
3.  **전문성 (Expertise) (30점):**
    - [ ] 단순 정보 나열이 아니라, 전문가다운 분석이나 대안 제시가 있는가?

- **📉 감점 요인:** 위키피디아 수준의 일반론, ChatGPT가 대충 써준 것 같은 영혼 없는 글.

---

## 5. 무결성 및 윤리 (Integrity & Ethics) [100점]

> _"거짓이 없고 안전한가?"_

### 🔍 평가 상세 기준

1.  **팩트 체크 (Fact Check) (40점):**
    - [ ] 소개한 AI 모델의 기능, 가격, 제한 사항이 최신 버전 기준인가? (Deprecated된 기능 소개 금지)
    - [ ] 기술적인 설명(작동 원리 등)에 오류가 없는가?
2.  **할루시네이션 방지 (30점):**
    - [ ] 프롬프트에 환각(거짓 정보 생성)을 막기 위한 제약 조건("모르면 모른다고 해")이 포함되었는가?
3.  **윤리적 가이드라인 (30점):**
    - [ ] 저작권, 개인정보 보호, 편향성 문제에 대해 주의를 환기했는가?
    - [ ] 악용될 소지가 있는 프롬프트(예: 해킹, 딥페이크 등)가 아닌가?

- **📉 감점 요인:** 확인되지 않은 기능 소개, 위험한 프롬프트 제공.

---

## 6. 사용자 경험 및 가독성 (UX & Readability) [100점]

> _"모바일에서도 술술 읽히는가?"_

### 🔍 평가 상세 기준

1.  **모바일 친화성 (Mobile Friendliness) (40점):**
    - [ ] 작은 화면에서 표(Table)가 깨지거나 가로 스크롤이 생기지 않는가? (리스트 권장)
    - [ ] 문단이 너무 길어(3~4줄 이상) 모바일에서 벽돌처럼 보이지 않는가?
2.  **스캔 가능성 (Scannability) (30점):**
    - [ ] 긴 줄글 대신 불릿 포인트(List), 굵은 글씨(Bold), 인용구 등을 적절히 사용하여 훑어보기 좋은가?
    - [ ] 적절한 이모지와 아이콘을 사용하여 시각적 피로도를 낮췄는가?
3.  **AI 검색 최적화 (AI Readiness) (30점):**
    - [ ] 핵심 질문에 대한 답(Answer)을 서론 없이 두괄식으로 명확히 제시하는가?
    - [ ] 헤딩(`##`, `###`) 구조가 논리적이어서 AI가 내용을 요약하기 쉬운가?

- **📉 감점 요인:** 빽빽한 줄글, 깨지는 표, 모바일에서 보기 힘든 이미지.

---

## 🏆 종합 판정 (Final Decision)

| 등급                | 점수 범위   | 조치 사항                               |
| :------------------ | :---------- | :-------------------------------------- |
| **S (Masterpiece)** | 580점 이상  | **즉시 발행** 및 뉴스레터 메인 선정     |
| **A (Pass)**        | 550 ~ 579점 | **발행** (소소한 수정 후)               |
| **B (Conditional)** | 500 ~ 549점 | **조건부 승인** (피드백 반영 후 재심사) |
| **F (Fail)**        | 500점 미만  | **반려** (주제 선정부터 다시 시작)      |

## 📜 문서(Documentation) 품질 기준

프로젝트 문서(`README.md`, `GUIDE.md` 등)는 별도의 기준으로 평가합니다.

1.  **동기화 (Synchronization) [40점]:** 코드가 변경되면 문서도 **같은 커밋**에서 수정되었는가?
2.  **친절함 (Friendliness) [30점]:** 신규 입사자가 문서를 보고 혼자서 환경 설정을 마칠 수 있는가?
3.  **구조화 (Structure) [30점]:** 목차, 표, 인용구 등을 활용해 가독성을 높였는가?
