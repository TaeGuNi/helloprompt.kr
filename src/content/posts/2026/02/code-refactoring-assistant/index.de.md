## 📝 Ist mein Code Spaghetti? Überlass es einem Refactoring-Experten!

- **🎯 Empfohlen für:** Junior-Entwickler, Programmier-Anfänger, Code-Reviewer
- **⏱️ Zeitaufwand:** 60 Minuten → 2 Minuten
- **🤖 Empfohlene Modelle:** ChatGPT (GPT-4), Claude 3.5 Sonnet, Gemini Advanced

- ⭐ **Schwierigkeitsgrad:** ⭐⭐☆☆☆
- ⚡️ **Effektivität:** ⭐⭐⭐⭐⭐
- 🚀 **Nützlichkeit:** ⭐⭐⭐⭐⭐

> _"Funktionierender Müll (Working Garbage) ist auf Dauer teurer als ein Serverausfall. Mach Schluss mit Spaghetti-Code und lass KI dein technisches Gewissen sein."_

Jeder kennt es: Die Deadline rückt unaufhaltsam näher, der Code funktioniert endlich, und man tröstet sich mit dem Gedanken: „Das refactore ich später“. Spoiler-Alarm: Dieses „später“ kommt so gut wie nie. Wenn dein Code schwer lesbar ist, vor Redundanzen strotzt oder schlichtweg unwartbar geworden ist, hilft dir dieser KI-Prompt, ihn in eine saubere, idiomatische und performante Architektur zu verwandeln – inklusive detaillierter Erklärungen für jede Änderung!

---

## ⚡️ 3-Sätze-Zusammenfassung (TL;DR)

1. Verwandelt unleserlichen „Spaghetti-Code“ in Sekundenschnelle in saubere, wartbare Strukturen.
2. Wendet konsequent Best Practices (SOLID, DRY) und etablierte Design Patterns der jeweiligen Programmiersprache an.
3. Liefert nicht nur den optimierten Code, sondern erklärt auch fundiert das _Warum_ hinter jeder architektonischen Entscheidung.

---

## 🚀 Die Lösung: "Der Clean-Code-Architekt"

### 🥉 Basic Version (Für schnelle Fixes)

Nutze diese Version, wenn du zügig eine unübersichtliche Funktion aufräumen möchtest.

> **Rolle:** Du bist ein Senior Software Engineer.
>
> **Aufgabe:** Refactore den folgenden Code, mache ihn besser lesbar und erkläre kurz, was du genau geändert hast:
>
> `[Dein Code hier]`

### 🥇 Pro Version (Für tiefgreifendes Refactoring)

Verwende diesen Prompt für komplexe Module, bei denen Code-Qualität, Performance und die strikte Einhaltung von Architektur-Standards höchste Priorität haben.

> **Rolle (Role):** Du bist ein Principal Software Engineer und ein kompromissloser Verfechter von Clean Code. Dein absolutes Fachgebiet ist `[Programmiersprache, z.B. TypeScript/Python]`.
>
> **Kontext (Context):**
>
> - Hintergrund: Ich habe Code geschrieben, der zwar funktioniert, aber schwer zu warten ist, schlecht skaliert und nicht den gängigen Best Practices entspricht.
> - Ziel: Den Code so zu refactoren, dass er elegant, exzellent lesbar, hochgradig effizient und leicht testbar wird.
>
> **Aufgabe (Task):**
>
> 1. Analysiere den folgenden Code und identifiziere Code Smells (z.B. tiefe Verschachtelungen, redundante Logik, irreführende Namensgebung).
> 2. Schreibe den Code komplett neu und wende dabei Prinzipien wie SOLID, DRY und KISS konsequent an.
> 3. Füge sinnvolle, professionelle Kommentare (Docstrings/JSDoc) hinzu.
> 4. Erstelle eine prägnante Liste der genauen Änderungen und erkläre den technischen Grund für jede einzelne Optimierung.
>
> **Code-Input:**
>
> `[Füge hier deinen Spaghetti-Code ein]`
>
> **Einschränkungen (Constraints):**
>
> - Verändere unter keinen Umständen die Kernfunktionalität oder die Ausgabewerte des Codes (**keine Breaking Changes**).
> - Verwende ausschließlich Standardbibliotheken oder Frameworks, die bereits im ursprünglichen Code sichtbar sind.
> - Formatiere deine Antwort klar strukturiert: Zuerst die tiefgehende Analyse, dann der optimierte Code im Markdown-Block, und zuletzt die fundierten Erklärungen.
>
> **Warnung (Warning):**
>
> - Erfinde keine Variablen oder Abhängigkeiten, die im gegebenen Kontext nicht existieren. Wenn dir entscheidende Informationen fehlen, frage proaktiv nach!

## 💡 Kommentar des Autors (Insight)

Als Entwickler ertappt man sich im stressigen Arbeitsalltag oft dabei, Quick-and-Dirty-Lösungen zu pushen. Das eigentliche Problem ist dabei nicht der erste Entwurf, sondern das konsequent ausbleibende Refactoring danach. Dieser Prompt ist deshalb so extrem mächtig, weil er KI nicht dazu nutzt, _neuen_ Code zu generieren, sondern deinen _bestehenden_ Code meisterhaft zu veredeln.

Besonders bei hartnäckigem Legacy-Code oder am Morgen nach einem langen Hackathon bewirkt diese Pro-Version wahre Wunder. Die KI deckt gnadenlos architektonische blinde Flecken auf, präzisiert die Benennung von Variablen und halbiert nicht selten die zyklomatische Komplexität im Handumdrehen. Mein persönlicher Tipp: Lass dir von der KI immer im Detail erklären, _warum_ sie eine bestimmte Änderung vorgenommen hat. Auf diese Weise verbesserst du ganz nebenbei dein eigenes Software-Design-Verständnis und lernst im Vorübergehen modernste Design Patterns kennen!

## 🙋 Häufig gestellte Fragen (FAQ)

- **F: Kann die KI durch das Refactoring unbeabsichtigt Bugs einbauen?**
  - A: Ja, das ist leider möglich (sogenannte Halluzinationen). Genau deshalb ist die strikte Anweisung „Verändere nicht die Kernfunktionalität“ im Prompt so essenziell. Teste den refactoreten Code immer ausgiebig mit deinen Unit-Tests, bevor du ihn jemals in die Produktion übernimmst!

- **F: Ist dieser Prompt wirklich für alle Programmiersprachen geeignet?**
  - A: Absolut. Du musst lediglich den Platzhalter `[Programmiersprache]` an deinen Tech-Stack anpassen. Führende KI-Modelle wie Claude 3.5 Sonnet oder GPT-4 beherrschen die Best Practices und Konventionen für nahezu alle modernen Sprachen exzellent.

- **F: Mein Code ist streng firmenintern. Darf ich ihn einfach in den Prompt einfügen?**
  - A: Vorsicht! Füge niemals sensible Unternehmensdaten, kritische API-Keys oder streng geheime Algorithmen in öffentliche KI-Tools ein. Anonymisiere Variablen im Vorfeld oder nutze sichere Enterprise-Versionen der KIs, die deine Eingaben nicht für das Training ihrer eigenen Modelle verwenden.

## 🧬 Anatomie des Prompts (Warum funktioniert das?)

1. **Spezifische Persona:** Die Zuweisung der Rolle als „Principal Software Engineer“ zwingt die KI dazu, höchste architektonische Qualitätsstandards (Clean Code, SOLID) anzulegen, anstatt sich mit funktionalen, aber unsauberen Anfänger-Lösungen zufriedenzugeben.
2. **Klarer Output-Flow:** Die KI wird durch die Strukturierung gezwungen, den Code zuerst zu analysieren, ihn dann optimiert zu liefern und die Änderungen abschließend zu begründen. Das verhindert zuverlässig, dass sie einfach nur stumm Code ausspuckt, ohne Kontext zu liefern.
3. **Sicheres Netz (Constraints):** Die unmissverständliche Anweisung „keine Breaking Changes“ minimiert das Risiko drastisch, dass der überarbeitete Code nach dem Refactoring nicht mehr nahtlos mit dem Rest deiner Applikation interagiert.

## 📊 Beweis: Vorher & Nachher

### ❌ Vorher (Der Spaghetti-Code)

```javascript
function calc(a, b, c) {
  let res = 0;
  if (c === "add") {
    res = a + b;
  } else {
    if (c === "sub") {
      res = a - b;
    } else {
      if (c === "mul") {
        res = a * b;
      } else {
        res = a / b;
      }
    }
  }
  return res;
}
```

### ✅ Nachher (Der Clean Code)

```javascript
/**
 * Führt grundlegende arithmetische Operationen aus.
 *
 * @param {number} a - Der erste Operand.
 * @param {number} b - Der zweite Operand.
 * @param {'add'|'sub'|'mul'|'div'} operation - Die auszuführende Operation.
 * @returns {number} Das Ergebnis der Operation.
 * @throws {Error} Wenn eine ungültige Operation übergeben wird oder durch Null geteilt wird.
 */
function calculateScore(a, b, operation) {
  const operations = {
    add: (x, y) => x + y,
    sub: (x, y) => x - y,
    mul: (x, y) => x * y,
    div: (x, y) => {
      if (y === 0) throw new Error("Division durch Null ist nicht erlaubt.");
      return x / y;
    },
  };

  const selectedOperation = operations[operation];
  if (!selectedOperation) {
    throw new Error(`Ungültige Operation: ${operation}`);
  }

  return selectedOperation(a, b);
}
```
