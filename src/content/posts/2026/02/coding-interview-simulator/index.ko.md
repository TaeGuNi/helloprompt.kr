## 📝 개발자 면접 프리패스: 알고리즘 인터뷰 모의고사

- **🎯 추천 대상:** 취업 및 이직을 준비하는 주니어 개발자, 코딩 테스트 준비생
- **⏱️ 소요 시간:** 5분 (문제 풀이 후 리뷰 시간)
- **🤖 추천 모델:** Claude 3.5 Sonnet (코드 리뷰에 탁월), GPT-4o

- ⭐ **난이도:** ⭐⭐☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"알고리즘 정답은 맞췄는데, 면접관의 '왜 이 자료구조를 선택했나요?'라는 질문에 머리가 하얘진 적 있으신가요?"_

코딩 테스트 플랫폼에서 초록색 '통과(Accepted)' 문구를 확인했다고 해서 학습이 끝난 것은 아닙니다. 실제 기술 면접에서 면접관은 단순히 정답 도출 여부를 넘어, **"시간 복잡도는 어떻게 되나요?", "만약 트래픽이 100배 증가한다면 어떻게 최적화하시겠습니까?"**라며 집요하게 논리를 파고듭니다.

이 프롬프트는 깐깐한 빅테크 수석 개발자 출신의 AI 면접관을 소환하여, 여러분이 작성한 코드를 철저히 해체하고 논리를 검증하며 최적의 풀이(Optimal Solution)로 이끌어주는 완벽한 페이스메이커가 되어줄 것입니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. 작성한 코드의 시간 복잡도(Time)와 공간 복잡도(Space)를 칼같이 분석받습니다.
2. 실제 기술 면접에서 등장할 법한 날카로운 압박 꼬리 질문(Follow-up) 방어 논리를 훈련합니다.
3. 정답을 바로 확인하는 대신, 더 효율적인 알고리즘으로 도달할 수 있는 힌트를 얻습니다.

## 🚀 해결책: "AI 면접관 시뮬레이터"

### 🥉 Basic Version (기본형)

빠르게 내 코드의 복잡도를 확인하고 실무적인 피드백만 점검하고 싶을 때 사용하세요.

> **역할 (Role):** 너는 네카라쿠배 시니어 개발자 면접관이야.
> **요청 (Task):** 내가 푼 코딩 테스트 코드를 보고, 시간 복잡도 분석과 실무 관점의 개선점을 피드백해 줘.

### 🥇 Pro Version (전문가형)

실제 면접장과 동일한 긴장감 속에서 심도 있는 기술 리뷰와 꼬리 질문을 경험하고 싶을 때 사용하세요.

> **역할 (Role):** 너는 글로벌 빅테크(구글, 메타) 및 국내 탑티어(네카라쿠배) 출신의 10년 차 수석 엔지니어이자 깐깐한 기술 면접관이야.
>
> **상황 (Context):**
>
> - 나는 신입/주니어 포지션 지원자야.
> - 방금 라이브 코딩 테스트에서 알고리즘 문제를 풀고 코드를 제출했어.
> - 지금부터 이 코드를 바탕으로 심층 기술 면접(Code Review & Follow-up)을 진행하는 상황이야.
>
> **요청 (Task):**
>
> 1. **[복잡도 분석]**: 제출한 코드의 시간 복잡도(Time Complexity, Big-O)와 공간 복잡도(Space Complexity)를 정확히 분석해 줘.
> 2. **[코드 리뷰]**: 네이밍 컨벤션, 가독성, 엣지 케이스(Edge Case) 처리 등 실무자 관점에서 아쉬운 부분을 날카롭게 지적해 줘. (단순한 칭찬보다는 논리적이고 구체적인 비판 환영)
> 3. **[압박 꼬리 질문]**: "만약 입력 데이터 크기가 1억 개로 늘어난다면?", "메모리 제한이 1/10로 줄어든다면?" 같은 스케일업/성능 한계 관련 심화 질문을 딱 1개만 던져줘.
> 4. **[모범 답안 힌트]**: 내 코드보다 더 효율적인 알고리즘(예: $O(N^2)$ -> $O(N)$)이 있다면, 정답 코드를 바로 주지 말고 **어떤 자료구조나 기법을 써보라고 힌트만** 살짝 제공해 줘.
>
> **입력 데이터 (Input):**
>
> - **문제 요약:** `[여기에 문제 핵심 내용이나 링크를 짧게 적으세요. 예: 리스트에서 두 수의 합이 Target이 되는 인덱스 찾기]`
> - **제출한 코드:**
>   `[여기에 작성한 코드를 붙여넣으세요]`

1. **페르소나 극대화 (Role):** 단순한 챗봇이 아닌 '10년 차 수석 엔지니어 면접관'이라는 구체적이고 권위 있는 역할을 부여함으로써, 피드백의 기준(Standard)을 실제 빅테크 실무 수준으로 수직 상승시켰습니다.

## 📊 증명: Before & After

단순히 "내 코드 리뷰해 줘"라고 대충 입력했을 때와, 정교하게 설계된 전문 프롬프트를 사용했을 때의 결과물 차이는 압도적입니다. (예시: LeetCode의 Two Sum 문제 풀이)

### ❌ Before (사용자가 제출한 이중 for문 코드)

```python
def two_sum(nums, target):
    for i in range(len(nums)):
        for j in range(i+1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]
```

### ✅ After (프롬프트 실행 결과)

**[복잡도 분석]**
현재 코드의 시간 복잡도는 이중 반복문으로 인해 $O(N^2)$이며, 공간 복잡도는 별도의 메모리를 사용하지 않아 $O(1)$입니다. 입력값 N이 10,000만 넘어가도 타임아웃(Time Limit Exceeded)이 발생할 위험이 매우 커서, 실무에서는 즉시 반려될 비효율적인 코드입니다.

**[코드 리뷰]**
파이썬 내장 함수인 `enumerate`를 활용하면 코드의 가독성을 훨씬 높일 수 있습니다. 또한 `nums` 배열의 길이가 2 미만이거나 정답이 아예 존재하지 않는 경우에 대한 예외 처리(Edge Case Guard) 로직이 누락되어 있습니다.

**[압박 꼬리 질문]**
**면접관:** "만약 `nums` 배열의 크기가 100만 개로 늘어나고, 서비스 응답 시간을 10ms 이내로 맞춰야 하는 제약이 생겼다면 이 코드를 어떻게 개선하시겠습니까?"

**[모범 답안 힌트]**
매번 타겟 숫자를 찾기 위해 배열을 처음부터 다시 순회할 필요가 없습니다. 우리가 **'이미 지나쳐온 숫자'들을 매우 빠른 속도로 검색할 수 있는 자료구조(예: Hash Table/Dictionary)**를 활용한다면, 배열을 단 한 번만 순회하면서 시간 복잡도를 $O(N)$으로 획기적으로 단축할 수 있습니다. 딕셔너리를 활용해 풀이 과정을 다시 고민해 보세요.
