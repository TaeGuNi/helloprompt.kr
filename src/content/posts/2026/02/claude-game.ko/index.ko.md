게임 개발의 패러다임이 근본적으로 변화하고 있습니다. 과거에는 2D 물리 엔진을 설계하고 복잡한 이벤트 리스너를 관리하는 데 수일이 소요되었지만, 이제는 **Claude Code**와 같은 자율형 AI 코딩 에이전트를 통해 그 진입 장벽이 완벽하게 무너졌습니다. 단순한 코드 스니펫 자동 완성을 넘어, 프로젝트의 전체 아키텍처를 이해하고 주도적으로 코드를 작성하는 Claude Code를 활용해 보십시오. 단 1시간 만에 브라우저 기반 2D 골프 게임을 완성하는 실전 과정과 단계별 프롬프트를 상세히 공개합니다.

## ⚡️ 3줄 요약 (TL;DR)

1. **AI 에이전트의 진화:** Claude Code는 단순한 코드 생성을 넘어, 물리 연산(속도, 마찰)부터 렌더링 로직에 이르는 게임 아키텍처 전체를 주도적으로 설계합니다.
2. **압도적인 생산성 향상:** 번거로운 마우스 이벤트 처리나 복잡한 충돌 수학 공식을 직접 고민할 필요 없이, 프롬프트 지시만으로 '드래그 앤 슛(Drag-to-shoot)' 메커니즘을 즉시 구현할 수 있습니다.
3. **확장성 높은 모듈화 구조:** AI가 초기 단계부터 `Physics`, `Input`, `Level` 클래스를 분리하여 모듈형 코드를 작성하므로, 향후 기능 확장(멀티플레이, 신규 레벨 추가 등)이 매우 용이합니다.

## 🚀 해결책: "Claude Code 게임 개발 프롬프트"

### 🥉 Basic Version (기본형)

게임의 초기 뼈대(Boilerplate)를 신속하게 구축할 때 사용하는 기본 프롬프트입니다.

> **역할:** 너는 `[시니어 게임 개발자]`야.
> **요청:** `[HTML5 Canvas]`를 활용하여 브라우저 기반의 2D 골프 게임을 만들어줘. 공과 홀이 존재해야 하며, 마우스로 드래그해서 쏘는(drag-to-shoot) 방식이어야 해.

### 🥇 Pro Version (전문가형)

물리 엔진, 입력 이벤트 처리, 레벨 디자인이 완벽하게 분리된 모듈형 게임 아키텍처를 설계할 때 사용하는 심화 프롬프트입니다.

> **역할 (Role):** 너는 10년 차 `[시니어 프론트엔드 게임 클라이언트 개발자]`이자 나의 완벽한 페어 프로그래밍 파트너야.
>
> **상황 (Context):**
>
> - 배경: 순수 바닐라 JS와 HTML5 Canvas만을 활용하여 경량화된 2D 탑뷰 골프 게임을 개발하고자 해.
> - 목표: 물리 엔진, 입력 이벤트, 레벨 상태 관리가 완벽히 모듈화되어 유지보수가 용이한 코드를 작성하는 것.
>
> **요청 (Task):**
>
> 1. `index.html` (캔버스 컨테이너), `game.js` (메인 루프), `physics.js` (벡터 및 마찰/충돌 수학)로 파일을 철저히 분리하여 구조화해.
> 2. `physics.js`에는 잔디의 마찰력(Friction)과 벽면 충돌 시의 반발력(Bounce) 로직을 정교하게 구현해.
> 3. 마우스의 `mousedown`, `mousemove`, `mouseup` 이벤트를 활용해, 공을 당길 때 예상 궤적과 파워를 시각적으로 보여주는 인디케이터(Indicator Line) 기능을 추가해.
> 4. `Obstacle` 클래스를 도입하여 난이도가 점진적으로 상승하는 3개의 개별 레벨(Level) 배열을 구성해.
>
> **제약사항 (Constraints):**
>
> - 외부 게임 엔진 라이브러리(Phaser.js 등)는 절대 사용하지 마.
> - 모든 주요 로직에는 주석을 추가하여 작동 원리를 명확히 설명해.
> - 모듈 패턴이나 ES6 Class 문법을 사용하여 철저히 객체지향적으로 코드를 작성해.

## 💡 작성자 코멘트 (Insight)

이 프롬프트의 핵심 성공 요인은 AI를 단순한 "코드 생성기"가 아닌 시스템 "아키텍트"로 대우하는 데 있습니다. 초기부터 "파일을 분리하라", "클래스를 나누어라"와 같이 명확한 제약(Constraints)을 주지 않으면, AI는 모든 코드를 `index.html` 단일 파일에 스파게티 코드 형태로 밀어 넣는 경향이 있습니다. 특히 물리 연산과 상태 관리를 독립적인 모듈로 분리하도록 강제함으로써, 추후 바람(Wind) 저항 효과를 추가하거나 WebSocket을 연동할 때 수정 범위를 획기적으로 줄일 수 있습니다. Claude Code는 넓은 컨텍스트를 안정적으로 유지하는 능력이 탁월하므로, 전체적인 아키텍처 뼈대를 먼저 설계한 뒤 세부 로직을 덧붙여 나가는 전개가 가장 효율적입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 게임에 적용되는 물리 수학 지식이 전혀 없어도 개발이 가능한가요?**
  - A: 네, 완벽하게 가능합니다. "잔디의 마찰력을 구현해", "벽에 부딪히면 자연스럽게 튕겨 나오게 해"처럼 물리적 현상을 자연어로 묘사하기만 하면, 벡터 내적이나 반발 계수와 같은 복잡한 수학 공식은 Claude가 문맥을 파악하여 정확한 코드로 변환해 줍니다.

- **Q: 모바일 브라우저 환경에서 터치 인터페이스도 지원하려면 어떻게 해야 하나요?**
  - A: Pro 버전 프롬프트의 Task 항목에 `"마우스 이벤트뿐만 아니라 touchstart, touchmove, touchend 이벤트도 매핑하여 모바일 환경에서도 동일한 사용자 경험을 제공하도록 해"`라는 지시를 한 줄만 추가하시면 깔끔하게 해결됩니다.

## 🧬 프롬프트 해부 (Why it works?)

1.  **명확한 아키텍처 지시:** 파일을 `game.js`, `physics.js` 등으로 세분화하도록 명시하여 코드의 복잡도를 낮추고, AI가 각 도메인의 로직에 온전히 집중하도록 유도했습니다.
2.  **구체적인 메커니즘 묘사:** 단순히 "골프 게임"이라고 지시하는 것을 넘어, "마우스로 당길 때 궤적과 파워를 보여주는 시각적 인디케이터"와 같이 사용자 경험(UX)을 상세히 묘사하여 기획 의도를 AI에게 정확히 전달했습니다.

### ✅ After (Pro 프롬프트 적용 결과)

```javascript
// 완벽히 모듈화되고 객체지향적인 물리 엔진 구조 (physics.js)
export class PhysicsBody {
  constructor(x, y) {
    this.position = new Vector(x, y);
    this.velocity = new Vector(0, 0);
    this.friction = 0.98; // 잔디 마찰 계수
  }

  update() {
    // 속도에 마찰력 적용
    this.velocity.multiply(this.friction);

    // 위치 업데이트
    this.position.add(this.velocity);

    // 캔버스 경계 충돌 처리 (반발 계수 적용 로직 호출)
    this.handleWallCollisions();
  }
}
```
