## 1. 아직도 AI와 '채팅'만 하고 계신가요?

2025년까지 우리는 AI에게 **"이 함수 좀 짜줘(Write this function)"**라고 부탁했습니다.
하지만 2026년 현재, 개발의 패러다임은 **"이 이슈 좀 해결해줘(Fix this issue)"**로 완전히 넘어갔습니다.

이것이 바로 **Agentic Workflow(에이전트 기반 워크플로우)**입니다.
단순히 코드를 자동 완성해주는 수준을 넘어, 프로젝트 전체의 맥락(Context)을 이해하고,
여러 파일을 동시에 수정하며, 테스트까지 수행하는 '동료' 같은 존재가 된 것이죠.

오늘은 그중에서도 가장 핫한 **Cursor Composer**와 **Windsurf**를 활용해,
말 한마디로 복잡한 기능을 구현하는 실전 프롬프트를 공유합니다.

---

## 2. 왜 지금 'Agentic AI'인가?

### ⚡️ Context Window의 혁명

과거엔 AI가 파일 하나만 보고 코드를 짰다면, 지금은 **전체 리포지토리**를 통째로 읽습니다.
Gemini 1.5 Pro나 Claude 3.5 Sonnet 같은 모델들은 수십만 줄의 코드를 한 번에 이해합니다.
덕분에 "저번 주에 만든 그 로그인 로직이랑 비슷하게 해줘"라는 말이 통하게 된 것이죠.

### 🛠 Multi-file Edit (다중 파일 수정)

가장 큰 변화는 **여러 파일을 동시에 수정**할 수 있다는 점입니다.
API 엔드포인트를 만들면, 프론트엔드 호출 함수도 만들고, UI 컴포넌트까지 한 번에 업데이트합니다.
사람이 일일이 파일을 열고 닫을 필요가 없습니다.

## 3. 실전: 30분 걸릴 일을 3분 만에 끝내는 프롬프트

단순한 기능 구현이 아니라, **기획부터 구현, 검증까지** 한 번에 시키는 프롬프트입니다.
Cursor의 `Composer` 모드(Cmd+I)나 Windsurf의 `Cascade` 패널에서 사용하세요.

> **# Role**
> 당신은 10년 차 시니어 풀스택 개발자입니다.
> Next.js (App Router), TypeScript, Tailwind CSS, Supabase 조합에 능숙합니다.
>
> **# Task**
> 현재 프로젝트에 '사용자 피드백 수집 기능'을 추가해주세요.
> 다음 3단계로 진행하고, 각 단계마다 저에게 확인을 받으세요.
>
> **1. 설계 (Design)**
>
> - 데이터베이스 스키마 (Supabase SQL)
> - API 라우트 설계 (`/api/feedback`)
> - UI 컴포넌트 구조 (Floating Button 형태)
>
> **2. 구현 (Implementation)**
>
> - `components/FeedbackWidget.tsx` 생성
> - `app/api/feedback/route.ts` 생성
> - `lib/supabase.ts` 활용
>
> **3. 검증 (Verification)**
>
> - 구현된 코드가 타입 에러 없이 컴파일되는지 체크
> - 모바일 반응형 디자인 적용 여부 확인
>
> **# Constraints**
>
> - 디자인은 `lucide-react` 아이콘을 사용하고, 깔끔한 모던 스타일로 작성하세요.
> - 에러 처리는 `try-catch` 블록으로 확실하게 감싸주세요.
> - 기존 코드 스타일(변수 명명법 등)을 그대로 따르세요.

## 4. Writer's Insight: 이것만은 주의하세요

직접 써보고 느낀 **Agentic AI의 한계와 팁**입니다.

### 🚨 1. "삭제"는 신중하게

에이전트는 때때로 너무 과감합니다. "리팩토링 해줘"라고 했더니, 멀쩡한 유틸리티 함수를
"안 쓰는 것 같아서 지웠어요"라며 날려버리는 경우가 종종 있습니다.
반드시 `git diff`를 통해 삭제된 코드가 없는지 확인하세요.

### 💡 2. 단계별(Step-by-Step) 지시가 핵심

아무리 똑똑한 모델이라도 한 번에 너무 많은 걸 시키면 탈이 납니다.
위 프롬프트처럼 **설계 -> 구현 -> 검증**으로 단계를 나눠서 시키면,
중간에 방향이 틀어졌을 때 바로잡을 수 있어 성공률이 비약적으로 올라갑니다.

### 🧪 3. 테스트 코드는 필수

에이전트가 짠 코드는 겉보기엔 완벽해 보이지만, 엣지 케이스(Edge Case)에서 터질 수 있습니다.
"이 기능에 대한 Jest 테스트 코드도 짜줘"라고 추가로 요청하는 습관을 들이세요.
