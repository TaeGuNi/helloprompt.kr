# 📝 Regex(정규표현식) 생성기: 암호문 같은 정규식, 말로 짜자

- **🎯 추천 대상:** `^([a-z0-9_.-]+)@([\da-z.-]+)\.([a-z.]{2,6})$` 이 코드만 봐도 머리가 아픈 분, 당장 입력 폼 유효성 검사를 구현해야 하는 프론트엔드 및 백엔드 개발자
- **⏱️ 소요 시간:** 10분 → 10초 단축
- **🤖 추천 모델:** ChatGPT-4o, Claude 3.5 Sonnet (복잡한 패턴 인식 및 코드 생성 최강)

- ⭐ **난이도:** ⭐☆☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"비밀번호는 영문, 숫자, 특수문자 포함 8자 이상... 이걸 정규식으로 어떻게 짜더라?"_

정규표현식(Regex)은 아무리 뛰어난 시니어 개발자라도 매번 구글링하는 영역입니다. 복잡한 기호의 나열을 억지로 외우는 것은 비효율적입니다. 이제 **자연어(Natural Language)**로 설명하세요. "이메일 형식 찾아줘"라는 간단한 문장 하나면, AI가 알아서 난해한 기호들을 조합해 무결점의 정규표현식을 만들어 줍니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. 찾고 싶은 문자열 패턴이나 추출할 데이터를 AI에게 자연어로 명확하게 설명합니다.
2. AI가 생성한 타겟 언어(JS, Python 등) 맞춤형 정규식 코드와 캡처 그룹(Capture Group) 설명을 확인합니다.
3. [Regex101](https://regex101.com) 사이트에서 실제 예시 데이터로 엣지 케이스를 테스트한 후 실무에 적용합니다.

## 🚀 해결책: "Regex Wizard"

### 🥉 Basic Version (기본형)

이메일, 휴대폰 번호 등 흔하게 쓰이는 간단한 패턴을 빠르게 찾을 때 사용하세요.

> **역할:** 너는 `[시니어 개발자]`야.
> **요청:** `[한국 휴대폰 번호(010-XXXX-XXXX) 형식을 검증하는 자바스크립트 정규표현식]`을 짜줘. 하이픈(-)은 있어도 되고 없어도 돼.

<br>

### 🥇 Pro Version (전문가형)

서버 로그 파일이나 비정형 데이터에서 특정 데이터만 정교하게 추출해야 할 때 사용하세요.

> **역할 (Role):** 너는 10년 차 시니어 백엔드 개발자이자 정규표현식(Regex) 마스터야.
>
> **상황 (Context):**
>
> - 배경: 시스템 로그 파일에서 특정 사용자 요청 데이터만 정제해서 데이터베이스에 저장해야 해.
> - 목표: 정제되지 않은 로그 텍스트에서 날짜, 유저 ID, IP 주소만 정확히 추출(Capture)하는 정규식 작성.
>
> **요청 (Task):**
>
> 1. 아래의 `[입력 데이터]`를 분석해서, `[추출 목표]`에 해당하는 정보만 캡처 그룹으로 묶는 파이썬(Python) 정규표현식을 작성해 줘.
> 2. 작성한 정규식의 각 부분과 캡처 그룹이 어떤 역할을 하는지 초보자도 이해할 수 있게 단계별로 설명해 줘.
> 3. 예외 케이스(Edge case)가 발생할 수 있는 부분이 있다면 어떻게 방어해야 하는지 조언해 줘.
>
> **입력 데이터 (Input):**
> `[2026-02-09 14:00:01] INFO [User:12345] Login Success (IP: 192.168.0.1)`
>
> **추출 목표 (Goal):**
>
> 1. 날짜 및 시간 (`2026-02-09 14:00:01`)
> 2. 유저 ID (`12345`)
> 3. IP 주소 (`192.168.0.1`)
>
> **제약사항 (Constraints):**
>
> - 출력 형식은 마크다운 코드블럭을 사용해 줘.
> - 파이썬의 `re` 모듈에서 즉시 사용할 수 있는 형태여야 해.
>
> **주의사항 (Warning):**
>
> - 불필요한 공백이나 문자가 캡처 그룹에 포함되지 않도록 정확하게 작성해 줘.

## 💡 작성자 코멘트 (Insight)

정규식의 가장 큰 적은 언제나 **'예외 케이스(Edge Case)'**입니다.
예를 들어 단순히 "숫자만 찾아줘"라고 지시하면 `123`은 정확히 찾지만, `-123`(음수)이나 `123.45`(소수점)는 놓치게 됩니다. 따라서 프롬프트를 작성할 때 처음부터 "음수나 소수점도 포함되나요?"라고 AI가 스스로 되묻게 하거나, 다양한 포맷의 예시 데이터를 함께 제공하여 테스트를 거치는 것이 실무에서 버그를 막는 핵심 팁입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 프로그래밍 언어마다 정규식 문법이 다른가요?**
  - A: 네, 미묘하지만 확실한 차이가 있습니다. JavaScript, Python, Java, Go 등 언어마다 지원하는 정규식 엔진(PCRE, RE2 등)과 문법이 다릅니다. 따라서 AI에게 지시할 때 반드시 "자바스크립트용으로 만들어 줘"처럼 타겟 언어를 명시해야 완벽히 호환되는 코드를 얻을 수 있습니다.

- **Q: AI가 만들어준 정규식을 바로 실무 코드에 적용해도 안전한가요?**
  - A: 절대 권장하지 않습니다. AI가 짠 코드는 논리적 허점이 있을 수 있습니다. 코드를 적용하기 전 반드시 [Regex101](https://regex101.com) 같은 정규식 검증 사이트에 코드를 붙여넣고, 실무에서 발생할 수 있는 여러 테스트 문자열을 입력해 직접 검증(Cross-check)하는 과정을 거쳐야 합니다.

## 🧬 프롬프트 해부 (Why it works?)

1.  **예시 데이터 제공 (Few-shot):** 정규식은 문맥에 따라 패턴이 천차만별로 달라집니다. 실제 로그나 텍스트 예시를 프롬프트에 직접 포함하면 AI의 패턴 인식률과 정확도가 100%에 가깝게 상승합니다.
2.  **캡처 그룹(Capture Group) 명시:** 단순히 "찾아줘(Match)"가 아니라 "이 부분과 저 부분만 뽑아줘(Extract)"라고 명확히 지시함으로써, AI가 괄호 `()`를 활용하여 후처리가 용이한 데이터를 발라내도록 유도했습니다.

## 📊 증명: Before & After

### ❌ Before (직접 작성한 정규식)

```text
\d+
```

_(단순히 숫자만 추출하려다 로그 내의 날짜, 시간 데이터까지 모두 매칭되어버려 데이터를 쓸 수 없는 엉망진창 상태 😱)_

### ✅ After (AI가 생성한 정규식 및 설명)

**패턴:**

```python
^\[(.*?)\] INFO \[User:(\d+)\] .* \(IP: ([\d\.]+)\)$
```

**설명:**

- `(.*?)`: 대괄호 안의 타임스탬프를 가장 짧게 매칭하여 캡처합니다. (날짜 및 시간)
- `(\d+)`: User: 뒤에 오는 숫자 ID를 캡처합니다. (유저 ID)
- `([\d\.]+)`: IP 주소 형식의 숫자와 점(.)을 캡처합니다. (IP 주소)

_(의도한 데이터만 완벽하게 캡처 그룹에 분리 성공 🎯)_
