# 📝 Tambo SDKで始めるジェネレーティブUI (Generative UI) 完全ガイド

- **🎯 おすすめの対象:** フロントエンドエンジニア、React開発者、プロダクトマネージャー
- **⏱️ 所要時間:** 60分 → 10分（Tambo SDK導入による短縮）
- **🤖 推奨モデル:** Claude 3.5 Sonnet, GPT-4o (関数呼び出し/Function Callingに優れたモデル)

- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果性:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「チャットボットがテキストしか返さない時代は終わりました。AIがユーザーの意図を汲み取り、最適なUIをリアルタイムで生成する体験を提供できていますか？」_

人工知能（AI）がテキストや画像を生成するだけでなく、今や**ユーザーインターフェース（UI）**そのものをリアルタイムで生成・構成する時代が到来しました。これを**ジェネレーティブUI (Generative UI)**と呼びます。

今回は、React環境においてこのジェネレーティブUIを最もスマートに実装できるツール**「Tambo SDK」**を活用し、AIアシスタントにリッチなコンポーネントをレンダリングさせるためのシステムプロンプトと実装手法を解説します。

---

## ⚡️ 3行まとめ (TL;DR)

1. **Generative UIの台頭:** AIはテキスト応答を超え、コンテキストに応じたReactコンポーネント（カード、グラフなど）を直接描画できる。
2. **Tambo SDKによる爆速実装:** 複雑な関数呼び出し（Function Calling）のパースや状態管理をTambo SDKが吸収し、開発体験を劇的に向上。
3. **システムプロンプトの最適化:** AIに「いつ・どのUIを出すべきか」を正確に指示するプロンプト設計が、UXの質を左右する。

---

## 🚀 解決策: ジェネレーティブUI実装プロンプト

ジェネレーティブUIを成功させる鍵は、コードの実装だけでなく、AIに**「どのような状況でコンポーネント（ツール）を呼び出すべきか」**を正確に伝えるシステムプロンプトにあります。

### 🥉 Basic Version (基本システムプロンプト)

まずは、UIコンポーネントを呼び出すためのシンプルな指示です。

> **役割:** あなたはユーザーの要望に合わせて最適なUIコンポーネントを提供するAIアシスタントです。
> **指示:** ユーザーが特定の場所や観光地について尋ねた場合は、必ず `showPlace` ツール（UIコンポーネント）を呼び出して応答してください。

<br>

### 🥇 Pro Version (実践向けシステムプロンプト)

実際のプロダクトレベルで機能する、制約とフローを明確にしたプロンプト設計です。

> **役割 (Role):**
> あなたは旅行計画をサポートする、専門的なUX/UIエージェントです。
>
> **状況 (Context):**
>
> - 背景: ユーザーはReactで構築されたチャットインターフェースを通じてあなたと対話しています。
> - 目標: 単なるテキストの羅列ではなく、視覚的に分かりやすいUIコンポーネント（カード、ボタンなど）を活用してユーザー体験を向上させること。
>
> **要求 (Task):**
>
> 1. ユーザーの意図を分析し、利用可能なツール（UIコンポーネント）が適しているか判断してください。
> 2. 場所やホテルの情報を提供する際は、テキストで説明するのではなく、必ず `showPlace` ツールを実行してください。
> 3. ツールを呼び出す際は、`name` (場所の名前), `description` (魅力的な説明), `rating` (1-5の評価) のパラメーターを正確に生成してください。
>
> **制約事項 (Constraints):**
>
> - `showPlace` ツールを連続して3つ以上呼び出さないこと（画面が埋め尽くされるのを防ぐため）。
> - UIコンポーネントで表現できる情報は、通常のテキストメッセージと重複して出力しないこと。
>
> **注意事項 (Warning):**
>
> - 存在しない架空の場所や、不正確な評価を生成しないこと（ハルシネーションの防止）。

<br>

#### 💻 実装ステップ: Tambo SDKとAIの連携

Tambo SDKでは、`pnpm install @tambo/sdk ai` でパッケージを導入後、以下のようにコンポーネントとスキーマを登録するだけでジェネレーティブUIが実現します。

```tsx
// lib/tambo.ts
import { createTambo } from "@tambo/sdk";
import { PlaceCard } from "../components/PlaceCard";
import { z } from "zod";

// AIが呼び出す「UIツール」の定義
export const tambo = createTambo({
  components: {
    showPlace: {
      component: PlaceCard,
      description: "特定の場所や観光地の情報をリッチなカード形式で表示します。",
      schema: z.object({
        name: z.string().describe("場所の正式名称"),
        description: z.string().describe("ユーザーの興味を惹く短い説明"),
        rating: z.number().min(1).max(5).describe("5段階評価"),
      }),
    },
  },
});
```

## 💡 作成者コメント (Insight)

従来のVercel AI SDK単体でもジェネレーティブUI（`ai/rsc` 等）の実装は可能でしたが、ストリームのパースやクライアントコンポーネントへの受け渡しなど、ボイラープレートコードが膨大になりがちでした。

Tambo SDKの最大の魅力は、**「コンポーネント」と「Zodスキーマ」をマッピングするだけで、あとはSDK側でツール呼び出しからUIレンダリングまでのパイプラインを自動化してくれる点**にあります。特にB2B SaaSのダッシュボードや、複雑なデータ入力を伴うエージェントUIにおいて、開発工数を圧倒的に削減できる強力な武器になります。スキーマの `describe` に記述する説明文（プロンプトの一部）をどれだけ具体的に書けるかが、AIの挙動を安定させるコツです。

## 🙋 よくある質問 (FAQ)

- **Q: Tambo SDKはReact以外のフレームワーク（VueやSvelte）でも使えますか？**
  - A: 現在、Tambo SDKはReactエコシステム（Next.js等）に最適化されています。他のフレームワークを使用している場合は、Vercel AI SDKのコア機能を使って独自に実装する必要があります。

- **Q: AIが意図しないタイミングでUIコンポーネントを出してしまいます。**
  - A: システムプロンプト（Pro Version参照）の **制約事項 (Constraints)** を強化するか、Tamboレジストリ内の各コンポーネントの `description` をより限定的な条件（例：「ユーザーから"予約したい"という明確な発言があった場合のみ使用する」）に書き換えることで改善できます。

- **Q: 独自のUIライブラリ（shadcn/uiなど）と組み合わせることは可能ですか？**
  - A: はい、完全に可能です。Tamboは単なる連携レイヤーであるため、中身のコンポーネントは `shadcn/ui` や `Tailwind CSS` で自由にスタイリングしたものをそのまま渡すことができます。

## 🧬 プロンプト解剖 (Why it works?)

1. **RoleとContextの明確化:** AIに対し、単なるチャットボットではなく「UX/UIエージェント」であるという役割を与えることで、テキスト出力よりもツール呼び出し（UI生成）を優先するよう誘導しています。
2. **冗長性の排除（制約）:** 「UIコンポーネントで表現できる情報はテキストで重複させない」という指示により、画面上に同じ情報が二重に表示されるUXの劣化を防いでいます。
3. **Zodスキーマへのプロンプト埋め込み:** コード内の `.describe('...')` が実質的なプロンプトとして機能し、AIがコンポーネントに必要なPropsを正確に推論・生成する手助けをしています。
