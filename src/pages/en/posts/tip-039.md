### âŒ Before (Situation)

"I need to store 1 million member IDs, no duplicates allowed, and I want to quickly check if a specific ID exists."

### âœ… After (Prescription)

**Recommended Data Structure:** **Hash Set**

**Reason:**

1.  **Duplicate Prevention:** By the nature of Set data structure, it automatically disallows duplicates.
2.  **Search Speed:** Checking for the existence of a specific ID (`contains`) is very fast at average **O(1)**. If you used an array or list, you would have had to compare 1 million times at O(N).

**Implementation Class (Java):** `java.util.HashSet`
(However, if order is important, use `LinkedHashSet`, if sorting is needed, use `TreeSet`.)
