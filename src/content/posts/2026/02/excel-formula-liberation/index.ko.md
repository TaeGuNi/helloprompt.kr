2. 원하는 데이터 정제 및 가공 방향을 일상적인 언어로 구체적이고 명확하게 지시하세요.
3. AI가 작성해 준 최적화된 수식이나 VBA 매크로 코드를 엑셀에 그대로 붙여넣어 업무를 자동화하세요.

## 🚀 해결책: "Excel Wizard Prompt"

### 🥉 Basic Version (기본형)

단순히 특정 목적의 함수가 기억나지 않거나, 간단한 연산 수식이 필요할 때 사용하세요.

> **역할:** 너는 10년 차 엑셀 마스터야.
> **상황:** A열에는 '상품명', B열에는 '단가', C열에는 '판매수량'이 있어.
> **요청:** D열에 '총 매출액(단가\*수량)'을 구하고, 그 값이 100만 원 이상이면 "VIP", 미만이면 "일반"이라고 표시하는 엑셀 수식을 작성해 줘.

<br>

### 🥇 Pro Version (전문가형)

여러 시트를 병합해야 하거나, 수식으로 해결하기 어려운 복잡한 조건의 가공, 또는 반복 업무를 위한 매크로(VBA)가 필요할 때 사용하세요.

> **역할 (Role):** 너는 10년 차 시니어 데이터 분석가이자 엑셀 VBA 및 자동화 전문가야.
>
> **상황 (Context):**
>
> - 배경: 현재 3개의 시트로 나뉜 방대한 로우 데이터(Raw Data)를 하나로 취합하고 분석해야 해. 각 시트의 데이터 형식이 조금씩 달라서 단순 복사 붙여넣기로는 해결이 불가능해.
> - 목표: 데이터를 완벽하게 정제하고, 내가 원하는 기준에 맞춰 자동으로 분류하는 시스템을 만드는 것.
>
> **데이터 샘플 (Sample):**
> (여기에 민감 정보를 제외한 데이터의 컬럼명 헤더와 상위 3행만 복사해서 붙여넣기)
>
> **요청 (Task):**
>
> 1. 제공된 샘플 데이터를 분석하여, 목적을 달성하기 위한 **가장 효율적인 엑셀 수식**을 먼저 제안해 줘.
> 2. 만약 수식만으로 처리하기 너무 복잡하거나 리소스 소모가 크다면, 버튼 한 번 클릭으로 모든 작업을 완료할 수 있는 **VBA 매크로 코드**를 작성해 줘.
> 3. 내가 작성해 준 수식이나 VBA 코드를 실제 엑셀 파일에 어떻게 적용하고 실행하는지 단계별(Step-by-step)로 상세히 가이드해 줘.
>
> **제약사항 (Constraints):**
>
> - 엑셀 초보자도 그대로 따라 할 수 있을 만큼 쉽고 명확하게 설명할 것.
> - 구형 함수(예: VLOOKUP) 대신 가급적 최신 Office 365 함수(예: `XLOOKUP`, `FILTER`, `UNIQUE` 등)를 우선적으로 활용하여 성능과 안정성을 높일 것.
>
> **주의사항 (Warning):**
>
> - 확실하지 않은 VBA 문법이나 존재하지 않는 함수를 지어내지 말고, 모르는 부분이나 데이터 부족으로 판단이 어려운 부분은 명확히 "추가 정보가 필요하다"고 답해.

## 💡 작성자 코멘트 (Insight)

이 프롬프트의 핵심은 **'보안'**과 **'VBA로의 확장성'**입니다. 많은 실무자들이 엑셀 데이터를 통째로 AI에게 넘겼다가 회사 보안 가이드라인을 위반하는 치명적인 실수를 범합니다. 전체 데이터가 아닌 **'헤더(컬럼명)와 샘플 3행'**만 주더라도 AI는 데이터의 구조와 패턴을 100% 이해하고 완벽한 수식을 도출합니다.

또한, "수식으로 안 되면 매크로(VBA)를 짜줘"라는 단 한 줄의 제약 조건이 압도적인 시간 단축을 가져옵니다. 엑셀 수식의 중첩 한계로 인해 1시간 동안 끙끙댈 문제를, AI가 10초 만에 짜주는 VBA 코드 복사+붙여넣기로 단숨에 해결할 수 있습니다. 특히 매달 반복되는 월간/주간 보고서 취합 작업에 이 프롬프트를 적용해 보세요. 단순 반복 작업 시간이 극적으로 줄어드는 것을 경험하실 수 있습니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 회사 엑셀 데이터는 대외비인데, AI에게 올려도 안전한가요?**
  - A: 절대 원본 데이터를 통째로 올리지 마세요! 이 프롬프트의 핵심은 구조만 알려주는 것입니다. 데이터의 최상단 '제목줄(헤더)'과 '가상의 더미 데이터 2~3줄'만 임의로 만들어서 프롬프트에 입력하시면 보안 리스크 없이 원하는 수식을 얻을 수 있습니다.

- **Q: AI가 짜준 VBA 코드를 넣었더니 엑셀에서 오류가 납니다.**
  - A: 엑셀 버전(Office 2016 vs Office 365) 차이로 인해 현재 환경에서 지원하지 않는 함수가 포함되었을 가능성이 큽니다. 오류가 발생한 줄이나 에러 팝업 메시지를 그대로 복사해서 AI에게 "이런 에러가 나는데 엑셀 2016 버전에 맞춰서 코드를 다시 수정해 줘"라고 피드백(Re-prompting)을 주시면 즉시 해결됩니다.

- **Q: Mac용 엑셀에서도 이 프롬프트로 얻은 수식이나 매크로가 작동하나요?**
  - A: 일반적인 수식(`XLOOKUP` 등)은 100% 완벽하게 호환됩니다. 다만, VBA 매크로의 경우 Mac 환경에서는 일부 파일 경로 시스템(Windows의 `C:\` 등)이나 특정 ActiveX 컨트롤 등에서 호환성 문제가 발생할 수 있습니다. 프롬프트 요청 시 **"Mac용 엑셀 환경에 맞춰서 코드를 작성해 줘"**라는 제약조건을 추가하시면 이러한 오류를 미연에 방지할 수 있습니다.

1. **상황과 목표의 명확한 정의:** 단순한 질문이 아닌 "데이터 취합과 자동 분류"라는 목표를 제시하여 AI가 전체 그림을 이해하고 최적의 방법론을 스스로 선택하도록 유도했습니다.
2. **보안 지향적 샘플 데이터 가이드:** 프롬프트 자체에 "민감 정보를 제외한 헤더와 상위 3행"이라는 룰을 명시하여 사용자의 부주의로 인한 보안 사고를 원천 차단했습니다.
3. **Escalation 전략 (수식 → VBA):** 엑셀 자체 기능의 한계에 부딪혔을 때, 사용자가 추가로 묻기 전에 AI가 스스로 상위 기술(VBA)로 자동 전환(Escalation)하도록 프로세스를 영리하게 설계했습니다.
4. **최신 함수 강제:** `VLOOKUP`의 치명적인 단점(열 추가 시 참조 오류 발생)을 피하고자 `XLOOKUP` 등의 최신 함수 사용을 제약 조건으로 걸어 결과물의 내구성과 안정성을 확보했습니다.

## 📊 증명: Before & After

### ❌ Before (인간의 뇌)

```text
=IF(ISERROR(VLOOKUP(A2, Sheet2!A:B, 2, 0)), "없음", VLOOKUP(A2, Sheet2!A:B, 2, 0))
```

- **결과:** 수식이 불필요하게 길고 복잡하며, Sheet2의 A열과 B열 사이에 새로운 열이 삽입되는 순간 참조가 어긋나 수식이 즉시 망가집니다. 괄호 하나를 빼먹어서 30분 동안 에러 원인을 찾다 결국 야근을 하게 됩니다.

### ✅ After (AI의 답변)

```text
=XLOOKUP(A2, Sheet2!A:A, Sheet2!B:B, "없음")
```

- **결과:** AI의 제안으로 최신 `XLOOKUP` 함수를 적용했습니다. 수식이 압도적으로 간결해졌으며, 중간에 새로운 열이 추가되어도 절대 참조 범위가 깨지지 않아 에러가 나지 않습니다. 단 10초 만에 완벽한 수식을 얻어내 칼퇴가 가능해집니다.
