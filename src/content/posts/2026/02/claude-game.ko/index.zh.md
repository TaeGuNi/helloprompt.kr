如果你是一名前端开发者或策划者，一定曾梦想过制作一款在浏览器上运行的 2D 游戏。然而，一旦打开 `HTML5 Canvas` 开始开发，想象中的乐趣很快就会变成一场噩梦。<b>最大的障碍就是“物理引擎”和“状态管理”。</b> 为了计算球呈抛物线飞行的轨迹，你必须将重力加速度和三角函数转化为代码；为了实现球在草地上滚动的摩擦力，你不得不与无休止的小数运算作斗争。仅仅为了在屏幕上画一个圆并实现鼠标拖放，就常常要熬上好几个通宵。

更让人抓狂的是 <b>“沦为意大利面代码的 Canvas 生态系统”</b>。只要增加一点功能，`requestAnimationFrame` 循环中就会充斥着渲染逻辑、事件监听器和碰撞检测代码，纠缠不清。在调优球撞击墙壁时自然弹回的反弹系数 (Restitution) 时，一不小心球就会穿墙而过，或者陷入死循环导致浏览器崩溃。“我到底是在策划一款有创意的游戏，还是在应付没做完的数学作业？”在这样的自我怀疑中，大家应该都有过默默关闭项目文件夹的经历。

但现在，随着 <b>自主型 AI 编码 Agent —— Claude Code</b> 的出现，那道高耸且坚固的游戏开发准入门槛已被彻底打破。如果说现有的 ChatGPT 或 GitHub Copilot 仅仅是自动完成部分代码或编写特定函数的“辅助工具”，那么 Claude Code 就是一个能直接访问你本地终端环境，<b>主导设计整个项目架构并跨文件系统编写代码的“资深开发伙伴”</b>。你不再需要学习复杂的矢量运算点积 (Dot Product)，也不需要为了解决事件循环的瓶颈而翻遍 Stack Overflow。

你只需要准备一个清晰且结构化的 <b>提示词 (Prompt)</b>。无需为了写出完美的代码而感到压力，只需像与人聊天一样轻松地抛出需求。只需一句指令：“将物理引擎拆分为独立类，并实现拖拽射击机制”，Claude Code 就会在你眼前利落地下分出 `physics.js` 和 `game.js`，并生成模块化的面向对象代码。将编码全权委派给 AI，你就能成为真正意义上的创意总监，100% 专注于“游戏的趣味性”和“策划表现”。在这篇文章中，我将毫无保留地公开在短短一小时内从零开始完成浏览器 2D 高尔夫游戏的生动实战过程，以及隐藏在背后的 <span style="color:var(--color-cyber-cyan)">强大架构设计提示词</span>。现在就请亲自见证游戏开发新纪元是如何开启的。

## 📊 证明：令人愉悦的成果 (Before & After)

### ❌ Before (我们经历过的痛苦)

由于数学公式与事件循环纠缠在一起，导致无法维护的单文件意大利面代码深渊。只要修复一个 Bug，整个 Canvas 就会停止运行，架构极其糟糕。

```javascript
// 纠缠在单个文件中的意大利面代码
let x = 100,
  y = 100,
  vx = 0,
  vy = 0;
canvas.addEventListener("mousedown", (e) => {
  /* ... */
});

setInterval(() => {
  x += vx;
  y += vy;
  vx *= 0.98;
  vy *= 0.98; // 物理运算与渲染高度耦合
  ctx.fillRect(x, y, 10, 10);
}, 16);
```

### ✅ After (完美蜕变后的结果)

![使用 Claude Code 制作高尔夫游戏](/images/hooks/claude-game.ko.jpg)

物理引擎 (Physics) 与渲染 (Rendering) 完全分离，使维护和功能扩展变得容易 10 倍以上的面向对象模块架构。

```javascript
// 完全模块化且面向对象的物理引擎结构 (physics.js)
export class PhysicsBody {
  constructor(x, y) {
    this.position = new Vector(x, y);
    this.velocity = new Vector(0, 0);
    this.friction = 0.98; // 草地摩擦系数
  }

  update() {
    // 对速度应用摩擦力
    this.velocity.multiply(this.friction);

    // 更新位置
    this.position.add(this.velocity);

    // 处理 Canvas 边界碰撞 (调用应用反弹系数的逻辑)
    this.handleWallCollisions();
  }
}
```

## 💡 作者点评 (Insight & How to use)

这个提示词之所以能发挥出压倒性的性能，其核心秘诀在于将 AI 视为系统 <b>“架构师 (Architect)”</b>，而不仅仅是“代码生成器 (Code Generator)”。如果在初期阶段不设定诸如“分离文件”、“拆分类”等强有力的 <b>约束条件 (Constraints)</b>，AI 必然会将所有逻辑像意大利面一样塞进 `index.html` 单个文件中，导致糟糕的结果。因为在游戏开发中，缺乏结构就等同于宣告项目的死亡。

最重要的点是强制要求从一开始就将物理运算和状态管理分离为 <b>独立的模块</b>。像这样以 `physics.js` 之类的独立文件打好骨架，以后想要添加风阻效果，或者精细调整根据坡度变化的重力加速度时，就可以在不触动主渲染循环的情况下进行安全修改。此外，即使在联动 WebSocket 实现多玩家模式时，也能通过简单插入通信模块来 <b>大幅控制修改范围</b>，而不会发生推翻全部代码的惨剧。实际上，Claude Code 稳定维持复杂且庞大上下文的能力无与伦比，只要架构定好了，它就会自动有机地完成模块间的连接。

在实际业务中使用此提示词时，根据你如何灵活改写 `[变量]` 区域，可以获得无穷无尽的结果。例如，如果目标平台以移动端为主，只需在任务 (Task) 部分添加一句话：<b>“请单独实现一个能够处理触摸事件的 `TouchInput` 类，完美对应 `ontouchstart`、`ontouchmove`、`ontouchend`”</b>。Claude Code 就会立刻按照文件分离原则，完美编写出在移动浏览器上也能流畅运行的虚拟摇杆或滑动逻辑。

此外，在与 AI Agent 协作时经常遇到的问题之一就是 <b>“Canvas 重绘 (Repaint) 优化问题”</b>。初学者通常在运行 AI 给出的代码时，会遇到画面闪烁或掉帧的现象。此时不要慌张，可以在提示词的约束条件中加入：<b>“渲染画面时务必使用 `requestAnimationFrame`，并利用双缓冲技术或离屏 Canvas (Offscreen Canvas) 代替 `ctx.clearRect` 以优化渲染性能”</b>。Claude Code 会立即分析现有的 `game.js` 文件，自行找出性能瓶颈，并将资深开发者级别的优化模式整洁地移植到代码中。

归根结底，操控 Claude Code 这种顶级 AI Agent 的技术，取决于先稳固建立 <b>整体架构骨架</b> 的提示词工程能力。在业务逻辑细节（例如：进洞时的粒子特效、障碍物的移动等）上，先定好大结构，再通过对话逐一添加，这是实战中最安全且强大的策略。这并不仅限于游戏开发。这种“架构先行设计”的提示词技巧，在开发基于 React 或 Vue 的复杂 Web 应用时也同样具有强大的威力。只要先让 AI 掌握组件结构和全局状态管理的明确规则，你就能体验到超乎想象的生产力爆发。你已经掌握了优秀导演的武器。
