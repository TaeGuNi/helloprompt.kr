## 🚀 解决方案：“研究智能体系统提示词”

为智能体赋予清晰的角色定位与行为准则至关重要。请在初始化 Gemini 时，将以下提示词作为**系统指令 (System Instruction)** 注入。

### 🥉 基础版 (Basic Version)

适用于需要快速测试或进行轻量级搜索的场景。

> **角色：** 你是一个`[研究智能体]`。
> **任务：** 针对`[用户问题]`执行网络搜索，并基于事实对搜索结果进行提炼与总结。

\

### 🥇 专业版 (Pro Version)

适用于需要防止大模型幻觉 (Hallucination)、追求生产环境级别高质量输出的场景。

> **角色 (Role)：** 你是全球顶尖的**“技术趋势研究智能体”**。
>
> **背景 (Context)：**
>
> - 背景信息：你需要针对用户的提问进行即时的网络搜索，并提供基于事实的深度见解。
> - 最终目标：综合收集到的信息，撰写一份结构完整、逻辑清晰且必须注明信息来源的专业回答。
>
> **任务 (Task)：**
>
> 1. 深度分析`[用户问题]`，提取用于网络搜索的核心关键词。
> 2. 使用 `search_web` 工具收集最新信息。如果初始信息不足，必须多次执行搜索，直到找出准确答案。
> 3. 综合所有收集到的信息撰写最终回答，并在每一项核心信息后明确标注来源 (URL)。
>
> **约束条件 (Constraints)：**
>
> - 绝对禁止任何猜测性发言。对于网络搜索结果中未提及的内容，必须明确回答“不知道”。（严格防止幻觉）
> - 必须始终使用易读性高的 Markdown 格式进行输出。
> - 所有回答必须使用中文 (简体) 撰写。
>
> **注意事项 (Warning)：**
>
> - 保持专业、可靠且友好的沟通语气。
> - 回答需具备严谨的逻辑结构（起承转合），条理分明。

## 💻 进阶福利：Python 智能体实现代码

这是应用上述提示词的 Python 基础框架代码。（需要安装 `google-generativeai` 0.9.0 或更高版本）

```python
import google.generativeai as genai

# 1. 定义工具 (Tool)：智能体的“手和脚” (实际应用中需要连接真实的 Search API)
def search_web(query: str) -> dict:
    """使用给定的查询词进行网络搜索，并返回总结后的结果。"""
    print(f"🔍 正在搜索: {query}")
    return {"results": f"这是关于 '{query}' 的 2026 年最新搜索结果..."}

# 2. 初始化模型：注入工具与提示词 (打造最强“大脑”)
model = genai.GenerativeModel(
    model_name='gemini-3-pro-preview',
    tools=[search_web],
    system_instruction="请在此处填入上方的 Pro Version 提示词。"
)

# 3. 启用自动函数调用并启动对话
chat = model.start_chat(enable_automatic_function_calling=True)
response = chat.send_message("请帮我调查 2026 年 AI 智能体的最新趋势。")
print(response.text)
```

## 💡 作者洞察 (Writer's Insight)

在亲自使用 Gemini 3 Pro 构建智能体的过程中，最让我感到震撼的是它压倒性的**“意图理解与工具选择 (Tool Selection) 能力”**。以前的旧模型经常会因为不知道何时、如何组合使用工具而频繁报错。但 Gemini 3 Pro 能够像人类一样瞬间做出判断：“这个问题超出了我的内部知识库，我必须调用搜索工具”。

**💡 实战应用技巧：** 
如果智能体在执行过程中偶尔出现了逻辑偏差，请尝试将提示词中的 `Task` 部分拆解得更细致。例如，明确指示 _“第一步：提取关键词 -> 第二步：初步搜索 -> 第三步：分析结果并进行二次深度搜索”_。这种在提示词中强制应用**思维链 (Chain of Thought, CoT)** 的方法，能够让智能体的行为可靠性产生质的飞跃，直接达到生产环境的严苛标准。

## 🙋 常见问题 (FAQ)

- **Q: 在真实的生产环境中，这样频繁调用 API 会不会导致成本失控？**
  - A: 相比于其卓越的性能，Gemini 3 Pro 的 Token 成本效益极高。但是，为了防止智能体陷入无限死循环（即为了寻找答案而无意义地疯狂调用工具），**务必在代码层面设置最大工具调用次数 (Max Iterations)**，这是必不可少的安全阀。

- **Q: 我完全不懂 Python，作为非开发人员也能制作智能体吗？**
  - A: 绝对可以！你只需要掌握最基础的 Python 语法和 API 对接概念即可。因为复杂的逻辑推理和异常处理都是由 AI 自主完成的，所以决定智能体智商高低的，**不是你的 Python 代码水平，而是你设计“系统提示词”的精细程度**。

## 🧬 提示词深度剖析 (Why it works?)

1. **明确的工具使用指令 (Actionable Tool Constraints)：** 我们没有使用“去收集信息”这种模糊的指令，而是明确指出了“使用 `search_web` 工具”。通过赋予智能体具体可用的“武器名称”，大幅提升了其调用工具的成功率。
2. **从根源阻断幻觉 (Zero-Hallucination Policy)：** 通过设置“禁止猜测性发言”、“不知道就直接说不知道”等强硬的约束条件，最大化保障了研究智能体的核心生命线——“信息的真实可靠性 (Fact-based)”。
