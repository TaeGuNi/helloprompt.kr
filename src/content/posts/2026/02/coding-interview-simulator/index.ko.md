# 📝 개발자 면접 프리패스: 알고리즘 인터뷰 모의고사

- **🎯 추천 대상:** 취업/이직을 준비하는 주니어 개발자, 코딩 테스트 준비생
- **⏱️ 소요 시간:** 5분 (문제 풀이 후 리뷰 시간)
- **🤖 추천 모델:** Claude 3.5 Sonnet (코드 리뷰에 탁월), GPT-4o

- ⭐ **난이도:** ⭐⭐☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"알고리즘 정답은 맞췄는데, 면접관의 '왜 이 자료구조를 썼나요?' 한마디에 머리가 하얘진 적 있나요?"_

코딩 테스트 플랫폼에서 초록색 '통과(Accepted)'를 봤다고 공부가 끝난 것이 아닙니다. 실무 면접에서 면접관은 단순히 답을 구했는지가 아니라, **"시간 복잡도는 어떻게 되나요?", "만약 트래픽이 100배 늘어난다면 어떻게 최적화할 건가요?"**를 집요하게 파고듭니다.

이 프롬프트는 깐깐한 빅테크 수석 개발자 출신의 면접관이 되어 여러분의 코드를 해체하고, 논리를 검증하며, 더 나은 풀이(Optimal Solution)로 이끌어주는 최고의 페이스메이커가 되어줄 것입니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. 내 코드의 시간 복잡도(Time)와 공간 복잡도(Space)를 칼같이 분석받기
2. 실제 기술 면접에서 던질 법한 날카로운 압박 꼬리 질문(Follow-up) 방어 연습
3. 정답을 바로 주지 않고, 더 효율적인 알고리즘으로 유도하는 힌트 얻기

---

## 🚀 해결책: "AI 면접관 시뮬레이터"

### 🥉 Basic Version (기본형)

빠르게 내 코드의 복잡도와 피드백만 확인하고 싶을 때 사용하세요.

> **역할:** 너는 네카라쿠배 시니어 개발자 면접관이야.
> **요청:** 내가 푼 코딩 테스트 코드를 보고, 시간 복잡도 분석과 실무 관점의 개선점을 피드백해 줘.

<br>

### 🥇 Pro Version (전문가형)

실제 면접장과 같은 긴장감 속에서 심도 있는 기술 리뷰와 꼬리 질문을 받고 싶을 때 사용하세요.

> **역할 (Role):** 너는 글로벌 빅테크(구글, 메타) 및 국내 탑티어(네카라쿠배) 출신의 10년 차 수석 엔지니어이자 깐깐한 기술 면접관이야.
>
> **상황 (Context):**
>
> - 나는 신입/주니어 포지션 지원자야.
> - 방금 라이브 코딩 테스트에서 알고리즘 문제를 풀고 코드를 제출했어.
> - 지금부터 이 코드를 바탕으로 심층 기술 면접(Code Review & Follow-up)을 진행하는 상황이야.
>
> **요청 (Task):**
>
> 1. **[복잡도 분석]**: 제출한 코드의 시간 복잡도(Time Complexity, Big-O)와 공간 복잡도(Space Complexity)를 정확히 분석해 줘.
> 2. **[코드 리뷰]**: 네이밍 컨벤션, 가독성, 엣지 케이스(Edge Case) 처리 등 실무자 관점에서 아쉬운 부분을 날카롭게 지적해 줘. (단순한 칭찬보다는 논리적이고 구체적인 비판 환영)
> 3. **[압박 꼬리 질문]**: "만약 입력 데이터 크기가 1억 개로 늘어난다면?", "메모리 제한이 1/10로 줄어든다면?" 같은 스케일업/성능 한계 관련 심화 질문을 딱 1개만 던져줘.
> 4. **[모범 답안 힌트]**: 내 코드보다 더 효율적인 알고리즘(예: $O(N^2)$ -> $O(N)$)이 있다면, 정답 코드를 바로 주지 말고 **어떤 자료구조나 기법을 써보라고 힌트만** 살짝 제공해 줘.
>
> **입력 데이터 (Input):**
>
> - **문제 요약:** `[여기에 문제 핵심 내용이나 링크를 짧게 적으세요. 예: 리스트에서 두 수의 합이 Target이 되는 인덱스 찾기]`
> - **제출한 코드:**
>   `[여기에 작성한 코드를 붙여넣으세요]`

## 💡 작성자 코멘트 (Insight)

코딩 테스트 공부를 할 때 '돌아가는 쓰레기 코드'를 짜고 넘어가는 습관이 가장 위험합니다. 저는 이 프롬프트를 활용해 릿코드(LeetCode)나 백준 문제들을 풀 때마다 반드시 코드 리뷰를 받았습니다.

특히 Claude 3.5 Sonnet을 사용했을 때, 제가 놓친 미세한 엣지 케이스(예: 배열이 비어있을 때, 음수가 포함될 때, Integer Overflow 발생 가능성)를 기가 막히게 짚어내어 실제 면접에서 엄청난 방어력을 갖출 수 있었습니다. 면접 전날, 본인이 과거에 풀었던 코드들을 이 프롬프트에 넣고 예상 꼬리 질문에 입 밖으로 대답하는 시뮬레이션을 꼭 해보시기 바랍니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 어떤 AI 모델이 코드 리뷰와 면접관 역할에 가장 적합한가요?**
  - A: 2026년 기준, 알고리즘 로직 분석과 예리한 코드 리뷰에는 **Claude 3.5 Sonnet** 또는 **GPT-4o**를 강력히 추천합니다. 모델의 코딩 추론 능력이 뛰어날수록 면접관의 질문 퀄리티가 높아집니다.

- **Q: 면접관(AI)이 너무 매섭게 비판해서 멘탈이 흔들려요. 톤을 조절할 수 있나요?**
  - A: 네, 프롬프트의 Task 2번 항목에서 `(단순한 칭찬보다는 논리적이고 구체적인 비판 환영)` 부분을 `(주니어 개발자가 상처받지 않게 부드럽고 친절한 멘토 톤으로 피드백해 줘)`로 수정하시면 편안하게 학습하실 수 있습니다.

- **Q: 코드가 너무 길어서 AI가 제 의도를 잘 파악하지 못하는 것 같아요.**
  - A: 코드 중간중간 주석으로 `// 여기서 투 포인터를 사용해 O(N)으로 순회함` 처럼 본인의 '의도'를 적어서 함께 프롬프트에 넣어주세요. AI가 여러분의 생각의 흐름까지 평가하여 훨씬 더 정확한 피드백을 줍니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **페르소나 극대화 (Role):** 단순한 챗봇이 아닌 '10년 차 수석 엔지니어 면접관'이라는 구체적이고 권위 있는 역할을 부여하여, 피드백의 기준(Standard)을 실무 빅테크 수준으로 끌어올렸습니다.
2. **사고 유도 제약 (Constraints):** "정답 코드를 바로 주지 말고 힌트만 제공해 줘"라는 제약을 통해, 사용자가 스스로 생각하고 알고리즘을 개선할 수 있는 기회를 강제합니다. 이는 정답을 떠먹여 주는 방식보다 훨씬 높은 학습 효율을 자랑합니다.

## 📊 증명: Before & After

단순히 "내 코드 리뷰해 줘"라고 했을 때와, 전문 프롬프트를 사용했을 때의 차이는 압도적입니다. (예시: Two Sum 문제 풀이)

### ❌ Before (사용자가 제출한 이중 for문 코드)

```python
def two_sum(nums, target):
    for i in range(len(nums)):
        for j in range(i+1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]
```

### ✅ After (프롬프트 실행 결과)

**[복잡도 분석]**
현재 코드의 시간 복잡도는 이중 반복문으로 인해 $O(N^2)$이며, 공간 복잡도는 별도 메모리를 쓰지 않아 $O(1)$입니다. 입력값 N이 10,000만 넘어가도 타임아웃(Time Limit Exceeded)이 발생할 위험이 큰, 실무에서는 반려될 비효율적인 코드입니다.

**[코드 리뷰]**
파이썬의 `enumerate`를 활용하면 가독성을 더 높일 수 있습니다. 또한 `nums` 배열의 길이가 2 미만일 때나 정답이 없는 경우에 대한 예외 처리(Edge Case Guard)가 누락되어 있습니다.

**[압박 꼬리 질문]**
**면접관:** "만약 `nums` 배열의 크기가 100만 개이고, 서비스 응답 시간을 10ms 이내로 맞춰야 한다면 이 코드를 어떻게 개선하시겠습니까?"

**[모범 답안 힌트]**
매번 타겟 숫자를 찾기 위해 배열을 다시 순회할 필요가 없습니다. 우리가 **'이미 지나쳐온 숫자'들을 매우 빠르게 검색할 수 있는 자료구조(예: Hash Table/Dictionary)**를 사용한다면, 배열을 단 한 번만 순회하며 시간 복잡도를 $O(N)$으로 획기적으로 줄일 수 있습니다. 딕셔너리를 활용해 풀이를 다시 고민해 보세요.
