## 📝 내 코드가 쓰레기라니! 클린 코드 리팩토링 코치

- **🎯 추천 대상:** 주니어 개발자, 사수 없이 고군분투하는 1~3년 차 프론트/백엔드 엔지니어
- **⏱️ 소요 시간:** 30분 → 1분 단축
- **🤖 최상위 성능:** Claude 3.5 Sonnet, GPT-4o, Gemini 2.5 Pro (코딩 특화 모델 권장)

- ⭐ **난이도:** ⭐⭐☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"분명 제가 짠 코드인데 3개월 만에 다시 보니 외계어가 따로 없습니다. 섣불리 건드렸다가 시스템 전체가 터질까 두려워, 소심하게 주석만 하나 덧붙이고 도망칩니다."_

스파게티처럼 복잡하게 뒤엉킨 **레거시 코드(Legacy Code)**는 모든 개발자가 겪는 가장 끔찍한 악몽이자, 프로젝트의 숨통을 서서히 조이는 **기술 부채(Technical Debt)**의 근원입니다. 분명 내가 불과 3개월 전에 심혈을 기울여 작성한 코드인데도, 오랜만에 다시 열어보면 마치 고대 외계인의 상형문자를 해독하는 것 같은 끔찍한 기분이 듭니다. `data1`, `temp`, `flag`, `is_ok_final` 등 도무지 원래의 목적을 알 수 없는 정체불명의 변수명들이 화면을 가득 채우고, 모니터 우측 끝을 뚫고 나갈 듯이 무한정 깊어진 `if-else` 중첩 지옥(Arrow Anti-Pattern)을 마주하는 순간 깊은 한숨부터 나옵니다. "어쨌든 지금 당장은 돌아가니까 절대 건드리지 말자"는 비겁한 변명과 함께, 차마 기존 로직을 과감하게 수정할 엄두를 내지 못합니다. 결국 그 코드 덩어리 밑에 소심하게 `// TODO: 시간 날 때 반드시 리팩토링 할 것`이라는, 영원히 지켜지지 않을 부질없는 주석만 하나 덧붙이고 황급히 파일을 닫고 도망치게 됩니다.

하지만 이러한 무책임한 회피는 결국 혹독한 대가를 치르게 합니다. 가독성이 처참하게 무너진 코드는 언제 터질지 모르는 시한폭탄이자, 미세한 **버그가 끝없이 자라나는 최악의 온상**이 됩니다. 사소한 텍스트 하나를 수정하거나, 아주 간단한 버튼 기능 하나를 추가해 달라는 기획자의 가벼운 요청에도 혹시나 엮여있는 시스템 전체가 연쇄적으로 터질까 봐 손에 땀을 쥐며 전전긍긍해야 합니다. 남이 싼 똥(혹은 과거의 내가 싼 똥)을 치우고 디버깅하는 데 하루 업무 시간의 80% 이상을 덧없이 낭비하다 보면, 내가 진정 가치를 창출하는 소프트웨어 엔지니어인지 아니면 그저 화면의 에러 로그만 쫓아다니는 단순한 코드 청소부인지 깊은 자괴감이 밀려옵니다. 특히 사수 없이 홀로 고군분투하는 1~3년 차 주니어 개발자라면 상황은 더욱 참담합니다. 코드 리뷰 시간에 쏟아질 시니어 개발자들의 매서운 지적과 깊은 한숨 소리가 너무나 두려워, 완성된 코드를 두고도 PR(Pull Request) 버튼을 누르는 것조차 극도로 망설여집니다. 결국 퇴근 시간은 매일같이 늦어지고, 주말에도 불안감에 휩싸여 노트북을 켜야 하는 **끝없는 야근의 굴레**에 빠지게 되는 것입니다.

이 지긋지긋한 악순환의 고리를 영원히 끊어낼 혁신적인 해결책이 절실합니다. 단순히 챗봇에게 "내 코드를 고쳐줘"라고 대충 부탁하는 아마추어적인 수준을 넘어, 전 세계 개발자들의 바이블을 집필한 로버트 C. 마틴(Robert C. Martin) 수준의 깐깐하고 타협을 모르는 **수석 아키텍트(Chief Architect)** 페르소나를 AI에게 완벽하게 부여하는 방법입니다. 오늘 여러분께 소개할 프롬프트는 여러분의 악취 나는 스파게티 코드를 강력한 세탁기에 넣은 듯, 구조적으로 완벽하고 우아한 **클린 코드(Clean Code)**로 단숨에 탈바꿈시켜 주는 마법 같은 도구입니다. 이 프롬프트는 복잡한 비즈니스 로직을 스스로 정밀하게 분석하여 거대한 함수를 목적에 맞게 작게 쪼개고, 직관적이고 아름다운 이름을 붙여주며, 각 언어의 가장 트렌디한 최신 모던 문법을 한 치의 오차도 없이 적용해 줍니다. 

이 강력한 리팩토링 코치를 실무 워크플로우에 도입하는 순간, 여러분의 개발 라이프스타일은 이전과는 비교할 수 없을 정도로 완전히 뒤바뀝니다. 수백 줄에 달하던 끔찍한 몬스터 함수가 단 몇 개의 깔끔한 단일 책임(SRP) 함수로 눈앞에서 분해되는 것을 보며, 이전에는 느껴보지 못한 **짜릿한 지적 카타르시스**를 느끼게 될 것입니다. 두렵기만 하던 코드 리뷰 시간은 "요즘 코드 퀄리티가 엄청나게 좋아졌네요. 정말 깔끔하게 잘 짜셨습니다!"라는 동료들의 극찬을 듣는 즐거운 시간으로 변모할 것입니다. 더 이상 남이 남긴 레거시 코드의 늪에서 길을 잃고 허우적대지 않아도 됩니다. 코드의 퀄리티를 최상급으로 끌어올리는 데 걸리던 수십 분, 아니 수 시간의 고통이 단 **1분**으로 기적처럼 단축됩니다. 그렇게 절약된 금쪽같은 시간은 오롯이 시스템의 새로운 비즈니스 가치를 창출하는 데 투자하거나, 여러분의 사랑하는 사람들과 함께하는 '저녁이 있는 삶'을 위해 온전히 사용하세요. 자, 이제 쳐다보기도 싫었던 쓰레기 같던 내 코드를, 누구나 감탄하는 예술 작품으로 바꿀 준비가 되셨나요?

## 📊 증명: 통쾌한 결과물 (Before & After)

단순하게 코드를 고쳐달라는 명령어와 완벽하게 설계된 프롬프트의 결과물 차이는 극명합니다. 이 프롬프트를 사용하면 훨씬 더 구체적이고 논리적으로 구조화된 리뷰를 받을 수 있으며, 끔찍한 들여쓰기 지옥에서 완벽하게 탈출할 수 있습니다.

### ❌ Before (우리가 겪던 고통)

```python
> def calc(x, y):
>     if x > 19:
>         print("성인")
>         p = y * 0.9
>         return p
>     else:
>         print("학생")
>         return y
```
> _(문제점: 함수명 `calc`로는 도대체 무엇을 계산하는지 알 수 없고, 매직 넘버 `19`와 `0.9`가 의미하는 바가 전혀 불명확하며, 불필요한 else 구문이 코드를 복잡하게 만듭니다.)_

### ✅ After (완벽하게 변신한 결과)

```python
def calculate_discounted_price(age: int, original_price: float) -> float:
    ADULT_AGE_THRESHOLD = 20
    ADULT_DISCOUNT_RATE = 0.9

    if age >= ADULT_AGE_THRESHOLD:
        print("성인입니다.")
        return original_price * ADULT_DISCOUNT_RATE

    print("학생입니다.")
    return original_price
```

**[수석 아키텍트의 리뷰 포인트]**
1. **네이밍 개선**: `calc`라는 모호한 이름을 `calculate_discounted_price`로 변경하여 함수의 목적을 명확히 드러냈습니다. 매개변수 또한 `x, y`에서 `age, original_price`로 구체화하여 가독성을 극대화했습니다.
2. **매직 넘버 제거**: `19`, `0.9` 같은 정체불명의 숫자를 의미를 담은 명시적 상수(`ADULT_AGE_THRESHOLD`, `ADULT_DISCOUNT_RATE`)로 선언하여 향후 유지보수를 훨씬 용이하게 만들었습니다.
3. **Early Return 적용**: 불필요한 `else` 블록을 과감히 제거하여 코드의 들여쓰기(Depth)를 줄이고 전체적인 논리 흐름을 직선적이고 단순하게 개선했습니다.
4. **타입 힌팅(Type Hinting) 추가**: Python 3.5+의 모던 문법을 적극 적용하여 인자와 반환값의 타입을 명시함으로써, 잠재적인 휴먼 에러와 버그 발생 확률을 완벽히 낮췄습니다.

## ⚡️ 3줄 요약 (TL;DR)

1. **네이밍 컨벤션 정규화:** 목적을 알 수 없는 `a`, `temp` 등의 모호한 변수명을 직관적이고 명확한 영단어로 전면 교체합니다.
2. **함수 단일 책임 원칙(SRP) 적용:** 수백 줄에 달하는 거대한 몬스터 함수를 목적에 맞게 작고 논리적인 단위로 분해합니다.
3. **모던 신택스 및 최적화:** Early Return, 구조 분해 할당 등 언어별 최신 관용 문법을 적용하여 코드의 들여쓰기 깊이(Depth)를 획기적으로 줄여줍니다.

## 🚀 진짜 전문가들은 이렇게 작성합니다

### 🥉 Basic Version (기본형)

빠르고 간결한 코드 개선이 필요할 때 활용하세요.

> **역할:** 너는 시니어 소프트웨어 엔지니어이자 클린 코드 전문가야.
>
> **요청:** 아래 코드를 기능 변경 없이 가독성과 유지보수성이 극대화되도록 리팩토링해 줘. 그리고 변경된 핵심 이유를 주석으로 간단히 달아줘.

### 🥇 Pro Version (전문가형)

디테일한 코드 품질 향상과 심도 있는 아키텍처 리뷰가 필요할 때 활용하세요.

> **역할 (Role):** 너는 구글, 메타 등 빅테크 출신의 `[수석 아키텍트]`이자 '클린 코드'의 광신도야.
>
> **상황 (Context):**
>
> - 배경: 현재 레거시 코드는 동작은 하지만, 네이밍이 엉망이고 들여쓰기(Depth)가 깊어 유지보수가 사실상 불가능한 수준이야.
> - 목표: 코드의 비즈니스 로직(기능)은 100% 동일하게 유지하면서, 가독성과 확장성을 극대화한 코드로 완벽하게 리팩토링하는 것.
>
> **요청 (Task):**
>
> 1. **[직관적 네이밍]**: 모든 변수, 함수, 클래스명을 누구나 즉시 이해할 수 있는 명확한 영단어로 변경해. (매직 넘버 제거 포함)
> 2. **[구조적 분해]**: 단일 책임 원칙(SRP)에 따라 하나의 함수가 오직 하나의 역할만 수행하도록 잘게 쪼개고 분리해.
> 3. **[Depth 최소화]**: Early Return(가드 패스) 패턴을 적극적으로 도입하여 불필요한 `else` 구문이나 중첩 `if`문을 완벽히 제거해.
> 4. **[모던 문법]**: 현재 사용 중인 언어(`[Python/JavaScript/Java 등]`)의 가장 최신이자 관용적인(Idiomatic) 문법을 적용해.
> 5. **[리뷰 제공]**: 변경된 코드 하단에 `AS-IS`와 `TO-BE`의 차이점을 분석하고, **왜 이렇게 변경했는지(Why)** 수석 아키텍트의 엄격한 관점에서 상세히 설명해 줘.
>
> **제약사항 (Constraints):**
>
> - 원본 코드의 최종 실행 결과(Output)는 어떠한 경우에도 절대 달라져선 안 돼.
> - "코드가 곧 문서다"라는 철학을 바탕으로, 코드로 충분히 읽히는 부분의 불필요한 주석은 모두 삭제해. 단, 비즈니스 로직상 반드시 필요한 핵심 설명만 남겨둬.
>
> **대상 코드:**
> `[여기에 리팩토링할 코드를 붙여넣으세요]`

## 💡 작성자 코멘트 (Insight & How to use)

이 프롬프트를 실무 현장에서 가장 스마트하고 극적으로 활용하는 방법은, 바로 나만의 **'24시간 대기 중인 개인 코드 리뷰(Code Review) 봇'**으로 사용하는 것입니다. 메인 브랜치에 코드를 병합하기 위해 PR(Pull Request)을 올리기 직전, 본인이 작성한 날것의 코드를 이 프롬프트에 먼저 통과시키는 습관을 들여보세요. 

특히, 이제 막 실무에 뛰어들어 사수에게 질문하기조차 눈치 보이는 주니어 개발자라면 이 프롬프트는 그야말로 **'성장의 치트키'**가 됩니다. 뼈아픈 지적을 받기 전에 네이밍 컨벤션, 변수 스코프, Early Return(가드 패스) 패턴의 적용 여부를 완벽하게 사전 점검받을 수 있기 때문입니다. 수석 아키텍트의 관점에서 내려지는 디테일한 코드 리뷰(TO-BE 분석)를 꾸준히 읽는 것만으로도, 여러분의 뇌 구조 자체가 '클린 코드'를 지향하는 아키텍트의 사고방식으로 자연스럽게 동기화되는 엄청난 속도의 성장을 경험하게 될 것입니다. 

프롬프트 내부의 `[변수]` 통제는 결과물의 품질을 결정짓는 핵심 키입니다. 기본적으로 제공된 프롬프트는 언어에 구애받지 않는 범용성을 지니고 있지만, 여러분이 처한 특정 환경에 맞춰 Task 항목의 제약사항을 정밀하게 조율(Fine-tuning)하면 더욱 폭발적인 성능을 낼 수 있습니다. 예를 들어, 프론트엔드 환경에서 React를 사용 중이라면 Task 항목에 **"React Hooks의 모범 사례(Best Practices)를 엄격하게 적용하고, 불필요한 리렌더링을 방지하기 위해 useMemo와 useCallback을 적재적소에 배치해 줘"**라는 단 한 줄의 강력한 제약을 추가해 보세요. 백엔드에서 Spring Boot를 다룬다면 **"비즈니스 로직과 영속성(Persistence) 계층을 철저히 분리하고, Controller에서는 절대 엔티티(Entity)를 직접 반환하지 말고 DTO를 거치도록 구조를 재편해 줘"**라고 지시할 수 있습니다. 이렇게 구체적인 생태계의 컨텍스트를 주입하는 순간, AI는 단순한 코드 정리기를 넘어 해당 프레임워크에 완벽히 들어맞는 세련되고 안정적인 아키텍처를 즉각적으로 설계해 줍니다.

더 나아가, 이 프롬프트를 팀 차원의 **'코드 컨벤션 강제화 도구'**로 응용해 볼 수도 있습니다. 여러 명의 개발자가 협업하는 프로젝트에서는 각자의 코딩 스타일이 달라 전체적인 통일성이 쉽게 무너지곤 합니다. 이때 프롬프트의 상황(Context) 섹션에 여러분 팀의 고유한 코딩 가이드라인 문서(예: "우리 팀은 인터페이스명 앞에 'I'를 붙이지 않으며, 모든 비동기 함수는 'Async' 접미사를 붙인다")를 요약하여 주입해 보십시오. AI는 여러분이 정의한 팀의 로컬 규칙을 마치 종교처럼 엄격하게 준수하며 코드를 교정해 냅니다. 결과적으로 누가 코드를 작성하든 마치 한 명의 뛰어난 장인이 처음부터 끝까지 정성스럽게 짠 것 같은 일관되고 아름다운 코드베이스를 유지할 수 있게 됩니다.

마지막으로 주의할 점은, 코딩에 특화된 최상위 추론형 AI 모델(Claude 3.5 Sonnet, GPT-4o 등)과 결합했을 때 이 프롬프트의 진가가 100% 발휘된다는 것입니다. 비용이 조금 들더라도 반드시 가장 똑똑한 모델을 선택하십시오. 저렴한 하위 모델은 구조를 제대로 파악하지 못하고 코드의 겉면만 포장하거나, 심지어 핵심 비즈니스 로직을 은근슬쩍 누락시켜 버리는 치명적인 환각(Hallucination) 현상을 일으킬 수 있습니다. "코드가 곧 문서다"라는 철학을 가슴에 새기고, 이 프롬프트를 통해 여러분의 기술적 역량과 코드의 품격을 한 차원 높이 끌어올려 보시길 강력히 권장합니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: AI가 비즈니스 로직을 마음대로 수정해버리면 어떡하나요?**
  - A: 프롬프트 내에 '기능 변경 금지'라는 강력한 제약을 걸어두었지만, AI의 산출물을 100% 맹신하는 것은 위험합니다. 리팩토링된 코드를 실제 프로젝트에 적용한 후에는 반드시 기존의 유닛 테스트(Unit Test)를 실행하여 모든 로직이 정상적으로 통과하는지 교차 검증해야 합니다.

- **Q: 회사 내부의 실제 코드를 프롬프트에 그대로 올려도 안전한가요?**
  - A: **절대 안 됩니다.** 사내 보안 규정에 심각하게 위배될 수 있으므로 API 키, 데이터베이스 스키마, 회사 고유의 민감한 핵심 비즈니스 로직은 반드시 마스킹(Masking) 처리하거나 가상의 변수명으로 치환한 뒤 입력하세요. (단, 회사 차원에서 도입한 철저한 보안 통제 환경의 Enterprise AI를 사용 중이라면 예외입니다.)

- **Q: AI가 코드를 쪼개다 보니 오히려 함수가 지나치게 파편화되었습니다.**
  - A: AI가 단일 책임 원칙(SRP)에 너무 극단적으로 집착했을 때 발생하는 전형적인 부작용입니다. 이럴 때는 제약사항에 `함수 분리는 최대 2단계 뎁스까지만 진행해 줘`와 같은 명확한 가이드라인을 추가로 제시하면 과도한 분리를 막고 적정선을 유지할 수 있습니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **명확한 페르소나(수석 아키텍트):** AI에게 단순한 '코드 수정 도구'가 아닌 '엄격하고 깐깐한 시니어 개발자'라는 페르소나를 부여함으로써, 결과물의 구조적 품질과 코드 리뷰의 깊이를 대폭 끌어올렸습니다.
2. **구체적인 리팩토링 지침(Task):** 직관적 네이밍, 구조 분해, 깊이(Depth) 축소 등 클린 코드의 절대 원칙을 강제합니다. 이를 통해 AI가 가독성을 오히려 해치는 과도한 숏코딩(Short Coding) 등 엉뚱한 방향으로 최적화하는 것을 원천 차단합니다.
3. **제약 조건(Constraints)의 전략적 활용:** '기능 변경 불가'와 '불필요한 주석 제거'를 명확히 지시하여, 개발자가 실무에서 즉시 복사해 붙여넣을 수 있는 극도로 깔끔하고 완성도 높은 산출물을 유도해 냅니다.
