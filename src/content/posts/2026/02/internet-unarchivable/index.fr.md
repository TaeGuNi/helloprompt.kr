# 📝 Sauvez le Web Perdu : Prompt d'Archivage pour la Préservation Permanente du Contenu Web

- **🎯 Public cible :** Chercheurs, curateurs de contenu, archivistes numériques, marketeurs
- **⏱️ Temps gagné :** De 1 heure à 1 minute
- **🤖 Modèles recommandés :** Modèles capables de gérer de longs contextes (Claude 3.5 Sonnet, Gemini 2.5 Pro recommandés)

- ⭐ **Difficulté :** ⭐⭐⭐☆☆
- ⚡️ **Efficacité :** ⭐⭐⭐⭐⭐
- 🚀 **Utilité :** ⭐⭐⭐⭐⭐

> _"Capturez l'histoire du web avant qu'elle ne disparaisse. En un simple copier-coller, figez le contexte essentiel des pages web dynamiques au rendu chaotique dans un document Markdown structuré et permanent."_

Avec les applications web monopages (SPA) et les plateformes fermées (Walled Gardens) devenant la norme, il est devenu presque impossible d'extraire et de conserver les pages web contenant des informations utiles par des méthodes traditionnelles. Les liens se brisent du jour au lendemain, et même les sites d'archives web échouent souvent à capturer les écrans à rendu dynamique, ne laissant qu'une page blanche.

Ce prompt est un puissant outil de préservation des connaissances. Au lieu d'obtenir un texte brut sans structure ou un code HTML illisible, il élimine les bruits inutiles (publicités, navigation, pied de page) et **le transforme en un document Markdown parfaitement structuré, prêt à être lu et utilisé à tout moment dans le futur**.

---

## ⚡️ Résumé en 3 points (TL;DR)

1. **Suppression Intelligente du Bruit :** Filtre les publicités et les éléments d'interface utilisateur des vidages de texte de pages web complexes pour n'extraire que le cœur du contenu.
2. **Conversion de Format Permanente :** Recompose le document au format Markdown, offrant la meilleure durabilité et compatibilité.
3. **Extraction Automatique des Métadonnées :** Génère automatiquement les métadonnées essentielles pour l'archivage et la recherche (date, auteur, mots-clés) au format YAML.

---

## 🚀 La Solution : "Prompt de l'Archiviste Numérique"

### 🥉 Version Basique (Basic Version)

Utilisez cette version lorsque vous souhaitez sauvegarder rapidement le texte principal d'une page web. Sélectionnez tout le texte de la page web (Ctrl+A), copiez-le, et collez-le à la suite de ce prompt.

> **Rôle :** Tu es un 'Archiviste Numérique' expert.
> **Tâche :** À partir du texte complexe de la page web fourni ci-dessous, élimine tous les bruits tels que les menus ou les publicités, et organise proprement le 'contenu principal' contenant l'information au format Markdown.

\

### 🥇 Version Pro (Pro Version)

Utilisez cette version pour créer un document d'archive parfait, prêt à être sauvegardé dans vos outils de gestion des connaissances personnels (Obsidian, Notion, etc.) sans perdre le contexte d'origine du site web.

> **Rôle (Role) :** Tu es un 'Archiviste Numérique (Digital Archivist)' de premier plan, chargé de préserver le patrimoine numérique de l'humanité. Ta mission prioritaire est d'extraire l'essence de l'information à partir de contenus web complexes et fragmentés, et de la transformer sous une forme préservable de manière permanente.
>
> **Contexte (Context) :**
>
> - Contexte : L'archivage web devient difficile en raison des pages web dynamiques (SPA) et des plateformes fermées. Il faut donc convertir directement le texte brut ou le code source dans un format de préservation permanent.
> - Objectif : Éliminer les bruits inutiles (publicités, menus de navigation, pieds de page, commentaires, etc.) des données textuelles/HTML fournies et les convertir en un document Markdown parfaitement structuré.
>
> **Tâche (Task) :**
>
> 1. Analyse les `[Données de la page web]` ci-dessous pour n'extraire que l'article principal (Main Article).
> 2. Rédige les métadonnées d'archivage tout en haut du document au format YAML Frontmatter (Doit inclure : titre, date de rédaction estimée, auteur, et 3 mots-clés principaux).
> 3. Réorganise le corps du texte avec une structure logique d'en-têtes (`##`, `###`) et formate-le en utilisant la syntaxe Markdown.
> 4. Rends les citations importantes, les données clés et les statistiques visuellement distinctes en utilisant les blocs de citation Markdown (`>`) ou le texte en gras (`**`).
>
> **Données d'entrée (Input Data) :**
>
> ```text
> [Collez ici l'intégralité du texte de la page web (Ctrl+A, C) ou le code source HTML]
> ```
>
> **Contraintes (Constraints) :**
>
> - Ne résume ni ne déforme arbitrairement les phrases originales ou les nuances du texte principal. (Maintiens une fidélité de 100% au texte original puisque le but est l'archivage).
> - Le format de sortie doit suivre strictement et uniquement la syntaxe Markdown pure, sans l'envelopper dans un bloc de code global.
>
> **Avertissement (Warning) :**
>
> - Si la date de rédaction ou l'auteur ne peuvent pas être clairement identifiés dans les données d'entrée, ne les invente pas ; indique simplement `Unknown`. (Prévention des hallucinations).

## 💡 Note de l'auteur (Insight)

Ce prompt n'est pas un simple "résumeur de texte". En 2026, alors que la vitesse à laquelle les informations disparaissent d'Internet croît de manière exponentielle, c'est **un véritable pipeline pour transférer la propriété de l'information entièrement vers votre environnement local**.

Il brille particulièrement sur les pages où les web clippers habituels échouent : les newsletters payantes nécessitant une connexion, les fils X (Twitter) à défilement infini, ou les wikis d'entreprise. Il suffit de tout sélectionner à l'écran (Ctrl+A, Ctrl+C) et de le jeter dans le prompt pour que, comme par magie, les résidus d'interface utilisateur disparaissent, ne laissant que les données originales purifiées. Personnellement, j'utilise cette méthode pour transférer parfaitement des dizaines d'articles menacés de disparition vers mon Obsidian chaque semaine.

## 🙋 Foire Aux Questions (FAQ)

- **Q : Dois-je copier le code source HTML ou le texte affiché à l'écran ?**
  - R : Les deux fonctionnent. Cependant, pour les pages récentes à rendu dynamique, sélectionner tout le texte sur l'écran du navigateur (Ctrl+A) puis le coller est bien plus efficace pour réduire le bruit des scripts cachés et ne transmettre que le texte essentiel à l'IA.

- **Q : Le contenu est trop long et l'IA renvoie une erreur. Que dois-je faire ?**
  - R : Copier une page web entière peut faire exploser le nombre de tokens. Il est fortement recommandé d'utiliser des modèles avec une limite élevée de tokens d'entrée (par exemple, Claude 3.5 Sonnet, Gemini 2.5 Pro).

---

## 🧬 Anatomie du Prompt (Why it works?)

1. **Obligation de fidélité au texte original (Constraints) :** Les LLM ont souvent le réflexe de "résumer" lorsqu'ils voient un texte long. La contrainte forte "Maintiens une fidélité de 100% au texte original puisque le but est l'archivage" supprime cet instinct de l'IA et la force à se concentrer sur son objectif principal : l'archivage.
2. **Extraction de métadonnées structurées :** En exigeant un YAML Frontmatter, on ne se contente pas de créer un texte esthétique ; on garantit une compatibilité technique immédiate avec des bases de données ou des applications de prise de notes basées sur Markdown.
