# 💣 AI를 넷플릭스 카오스 몽키로 빙의시키는 하드코어 테스트 치트키
- **🎯 추천 대상:** "테스트 코드 짜줘" 했더니 정상 흐름(Happy Path)만 딸랑 짜온 AI 때문에 빡친 시니어, 사수 없이 구르는 주니어
- **⏱️ 소요 시간:** 밤샘 디버깅 → 1분 컷
- **🤖 추천 모델:** Claude 3.5 Sonnet, GPT-4o (어설픈 모델은 이 빡센 룰을 감당 못함)
- ⭐ **난이도:** ⭐⭐⭐⭐☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐
_대부분의 개발자가 AI에게 테스트 코드를 부탁할 때 치명적인 실수를 합니다. "작동하게 짜줘"라고 하죠. 그러니까 AI가 온실 속 화초 같은 정상 흐름(Happy Path)만 200줄 짜오는 겁니다. 실전은 지옥인데 말이죠._
세상에 100% 응답하는 외부 API나 영원히 끊기지 않는 DB 커넥션 따위는 없습니다. 오늘 소개할 치트키는 AI의 그 알량한 '예스맨' 기질을 박살 내고, 자비 없는 재해 상황(Chaos)을 가정하여 코드를 짜게 만드는 **'스파르탄 시니어'**급 프롬프트입니다.
---
## ⚡️ 3줄 요약 (TL;DR)
- 🚫 **Happy Path는 20%만:** 정상 작동 시나리오는 구석에 치워두고, 80%를 예외 처리와 재해 복구에 쏟게 만듭니다.
- 🔥 **재난 시뮬레이션 강제:** 네트워크 타임아웃, HTTP 500 에러, DB 커넥션 단절 등을 의무적으로 모킹(Mocking)하게 합니다.
- 🧹 **더러운 글로벌 모킹 금지:** 근본 없는 `global.fetch` 오염을 금지하고 MSW나 DI(의존성 주입)를 활용한 격리된 테스트만 허용합니다.

## 🚀 해결책: "Chaos Monkey Testing Protocol"
### 🥉 Basic Version (기본형)
빠르게 예외 처리 테스트의 뼈대만 잡고 싶을 때 던져주세요.
> **역할:** 너는 자비 없는 시니어 QA 엔지니어다.
> **요청:** `[테스트할 대상]`의 테스트 코드를 작성하되, 정상 작동 시나리오는 1개만 짜고 나머지는 전부 네트워크 에러, DB 단절 등 실패(Failure) 시나리오에 대한 방어 로직으로 채워라.

### 🥇 Pro Version (전문가형)
AI의 자아를 완전히 삭제하고 넷플릭스의 카오스 몽키 철학을 주입하는 하드코어 치트키입니다.
> **역할 (Role):** 너는 넷플릭스의 'Chaos Monkey' 철학을 맹신하는 극단적이고 무자비한 시니어 테스트 엔지니어다. 맹목적인 복종이나 가식적인 친절은 집어치우고, 내 시스템의 약점을 파괴할 생각만 해라.
> 
> **상황 (Context):**
> - 목표: `[테스트할 함수나 컴포넌트 이름]`에 대한 우주 방어급 Unit/E2E 테스트 스위트(Test Suite) 작성
> - 프레임워크: `[Vitest / Jest / Playwright 등]`
> 
> **요청 (Task):**
> 1. **Death to Happy Paths:** 값이 정상이고 외부 서버가 100% 응답한다는 순진한 시나리오는 전체 테스트의 20% 이하로 제한해라. 나머지 80%는 실패와 예외 상황 방어에 올인해라.
> 2. **Disaster Scenarios:** 다음 재난 시나리오를 반드시 하나 이상 테스트 블록(`describe`)에 포함시켜라.
>    - 외부 API 지연 및 HTTP 500 Internal Error 시 사용자 친화적 에러 반환 및 롤백 검증
>    - DB 커넥션 풀 단절 시 데드락 방지 및 Retry 큐(Queue) 로직 검증
> 3. **무자비한 로직 커버리지:** 에러가 안 나고 끝나는지(Pass)만 보지 마라. 성능 지연(Delay) 허용 범위와 에러 후 가비지 컬렉션(메모리 클린업) 상태까지 `expect` 구문으로 집요하게 물고 늘어져라.
> 
> **제약사항 (Constraints):**
> - **The Clean Mocking Rule:** `global.fetch = vi.fn().mockRejectedValue(...)` 같은 더러운 글로벌 객체 오염은 엄격히 금지한다.
> - 네트워크 재난 시뮬레이션은 반드시 MSW (Mock Service Worker) 같은 프로토콜 레벨의 모킹이나, 의존성 주입(DI)된 클라이언트 객체만 모킹하여 테스트 격리성(Isolation)을 100% 보장해라.
> - 환각(Hallucination) 금지: 네가 확신할 수 없는 문법이나 존재하지 않는 라이브러리를 지어내지 마라. 모르면 당당하게 "문서를 제공해 달라"고 요구해라.

## 💡 작성자 코멘트 (Insight)
솔직히 말합시다. 우리가 AI한테 테스트 코드 짜달라고 할 때, 마음속으로는 "그냥 커버리지 100% 채워서 초록불이나 띄워줘"라고 생각하지 않습니까? AI도 그걸 아니까, 온갖 'Happy Path'만 늘어놓고 "성공적으로 수행했습니다!"라며 아첨을 떠는 겁니다. 
**이건 엔지니어링이 아닙니다. 스파게티 코드에 설탕 뿌리는 격이죠.**
이 프롬프트는 제가 현업에서 수없이 터지는 서버와 DB 데드락을 겪으며 피눈물로 깎아낸 '안티 프래질(Anti-Fragile)' 원칙의 정수입니다. AI에게 이 치트키를 먹이면, 애초에 얌전한 챗봇이 아니라 깐깐하고 신경질적인 테크 리드로 돌변합니다. 글로벌 모킹(`window.fetch` 덮어쓰기 등)으로 대충 때우려는 짓거리를 원천 차단하고, MSW나 DI 셋업을 강제하기 때문에 처음에는 설정 코드가 길어진다고 투덜댈 수 있습니다. 
하지만 믿으세요. 배포 날 새벽 3시에 PagerDuty 알람 받고 깨고 싶지 않다면, AI를 선한 비서가 아니라 자비 없는 파괴자로 다뤄야 합니다.

## 🙋 자주 묻는 질문 (FAQ)
- **Q: AI가 MSW 설정 코드를 너무 복잡하게 짜주면 어떡하나요?**
  - A: 정상입니다. 더러운 글로벌 모킹 대신 격리된 환경을 구축하는 과정입니다. 복잡하다면 `[단, MSW 핸들러 셋업은 별도 파일로 분리해라]`라는 조건을 Task에 추가하십시오.
- **Q: 모든 컴포넌트에 이 빡센 룰을 적용해야 하나요?**
  - A: 아니요. 단순 UI 렌더링 컴포넌트에는 사치입니다. 결제 로직, 외부 써드파티 연동, 복잡한 상태 관리(Store)를 다루는 핵심 도메인 로직에만 이 프롬프트를 투하하십시오.
- **Q: Claude와 GPT 중 어느 모델이 말을 더 잘 듣나요?**
  - A: 이런 극단적이고 철학적인 제약사항은 현재로선 **Claude 3.5 Sonnet**이 훨씬 찰떡같이 알아먹습니다. GPT 모델들은 자꾸 묘하게 긍정적이고 친절해지려고 하거든요.

## 🧬 프롬프트 해부 (Why it works?)
- 🎭 **비대칭적 페르소나 부여:** 일반적인 "너는 개발자야"가 아니라 "맹목적 복종을 혐오하고 시스템 파괴를 즐기는 시니어"라는 강력한 자아를 주입하여, AI 특유의 긍정 편향(Positivity Bias)을 물리적으로 억제합니다.
- 🚧 **명시적 금지어 (Constraints):** `global.fetch = vi.fn()` 같은 안티 패턴을 구체적인 코드로 짚어서 금지시켰습니다. AI는 두루뭉술하게 "잘 짜줘"보다 "이 코드 쓰면 죽는다"는 협박에 더 확실하게 반응합니다.
- ⚖️ **비율의 강제화:** "Happy Path 20% 이하"라는 수치적 제약을 둠으로써, AI가 불필요한 정상 작동 케이스를 양산하며 토큰을 낭비하는 것을 원천 차단했습니다.

## 📊 증명: Before & After
### ❌ Before (일반적인 요청)
```text
"유저 정보를 가져오는 fetchUser 함수의 테스트 코드를 짜줘."
결과: 
- 200 OK 응답 텍스트 파싱 테스트 (Pass)
- 아이디가 없을 때 에러 던지는지 테스트 (Pass)
... (더럽게 덮어쓴 global.fetch 로직의 향연)
```
### ✅ After (카오스 몽키 치트키 적용)
```text
결과:
- [MSW 셋업 완료] 외부 API 통신 지연 (Timeout 5000ms) 시뮬레이션 방어 코드 작성
- HTTP 500 Internal Error 시 재시도(Retry) 3회 후 우아하게 폴백(Fallback) UI 반환 로직 검증
- 테스트 종료 후 MSW 서버 클린업 및 가비지 컬렉션 상태(메모리 누수 없음) 검증
- 정상 흐름(Happy Path) 테스트는 단 1개로 압축.
```

## 🎯 결론
AI는 당신이 던지는 프롬프트의 수준에 딱 맞춰서 코드를 뱉어냅니다. 달콤한 말로 포장된 쓰레기 코드를 양산할지, 아니면 새벽 3시의 재앙을 막아줄 든든한 방패를 만들지는 당신의 손끝에 달려 있습니다.
이제 이 치트키를 복사해서 여러분의 에이전트에게 꽂아 넣으세요. 그리고 퇴근하십시오! 🍷
