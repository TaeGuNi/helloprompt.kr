# 🎨 Потрясающие анимации за секунды! Генератор CSS Keyframe

- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐

> _«Сделай так, чтобы при нажатии кнопка плавно увеличивалась, пружинила как желе и светилась всеми цветами радуги». Снова готовитесь к рутинному прописыванию ключевых кадров после такой абстрактной просьбы дизайнера?_

CSS-анимации выглядят топорно и дешево, стоит лишь немного ошибиться в значениях. Чтобы перенести естественные «законы физики» и плавность движений в код, требуются бесконечные перезагрузки страницы и метод проб и ошибок. Хватит мучить себя, вытачивая графики кубических кривых Безье (Cubic-bezier) в инструментах разработчика. Просто опишите желаемый эффект человеческим языком, и ИИ мгновенно выдаст оптимизированный CSS-код, обеспечивающий невероятно плавную и упругую анимацию при стабильных 60 кадрах в секунду.

3. Гарантирует высокую производительность веб-приложений за счет аппаратного ускорения (GPU) и предотвращения Reflow.

Используйте, когда нужно быстро получить код для простых и стандартных анимаций.

> **Роль:** Ты `[Frontend UI-разработчик]`.
> **Задача:** Напиши CSS Keyframe-анимацию для `[всплывающего уведомления, которое плавно выезжает из правого верхнего угла экрана]`.

<br>

### 🥇 Продвинутая версия (Pro Version)

Используйте для создания высококачественного кода (production-ready) со сложной физикой движений и строгой оптимизацией производительности под GPU.

> **Роль (Role):** Ты `[Senior UI/UX-разработчик и Interaction-дизайнер с 10-летним опытом]`.
>
> **Контекст (Context):**
>
> - Ситуация: `[Мне нужно реализовать естественные и стильные микро-взаимодействия, чтобы максимизировать качество пользовательского опыта (UX) в веб-приложении]`.
> - Цель: `[Сгенерировать высококачественный CSS-код для анимации, обеспечивающий плавность при 60fps без единого подтормаживания (Jank)]`.
>
> **Задача (Task):**
>
> 1. На основе предоставленного ниже **[Описания анимации]** напиши детализированный и выверенный код `@keyframes`.
> 2. Предоставь CSS-код в виде класса `.class`, чтобы анимацию можно было сразу применить к элементу.
> 3. Вместо банального `ease-in-out` подбери и примени изысканное пользовательское значение `cubic-bezier`, которое идеально передаст "настроение", описанное в моих требованиях.
>
> **[Описание анимации]**
>
> - Элемент: `[Всплывающее окно уведомления (Toast Message)]`
> - Поведение: `[Плавно выезжает из правого верхнего угла (Slide In), ненадолго задерживается на экране, затем исчезает, растворяясь и уходя вверх (Fade Out)]`
> - Настроение: `[Очень плавно и изящно, но не статично — с легким, упругим эффектом подпрыгивания (Bounce), как у желе]`
> - Продолжительность: `[Общее время анимации — 3 секунды]`
>
> **Ограничения (Constraints):**
>
> - Для максимальной производительности СТРОГО ЗАПРЕЩАЕТСЯ использовать свойства, вызывающие Repaint и Reflow (такие как `width`, `top`, `margin` и т.д.).
> - Используй ИСКЛЮЧИТЕЛЬНО свойства `transform` и `opacity` для полного задействования аппаратного ускорения (GPU Acceleration).
> - Выведи готовый код в формате блока Markdown CSS, чтобы его можно было просто скопировать и вставить.
>
> **Предупреждение (Warning):**
>
> - Этот код пойдет напрямую в production. Абсолютно недопустимы синтаксические ошибки или решения, нарушающие кроссбраузерную совместимость.

## 💡 Комментарий автора (Инсайт)

Душа любой анимации — это тайминги и кривые ускорения. Даже малейшее изменение значений `animation-duration` или `cubic-bezier` в сгенерированном ИИ коде (буквально на десятые доли) способно кардинально изменить атмосферу взаимодействия.

Делюсь профессиональным секретом: попробуйте добавить в промпт фразу **"Сделай эффект премиального замедления (Deceleration) в стиле веб-сайтов Apple"**. Вы увидите настоящую магию — скучное, стандартное движение моментально превратится в роскошную high-end анимацию, словно вы наняли дорогое дизайн-агентство. 🍎 Больше никаких бессонных ночей в попытках настроить кривые Безье наугад!

## 🙋 Часто задаваемые вопросы (FAQ)

- **В: Можно ли использовать этот промпт для Tailwind CSS или Styled-Components?**
  - О: Конечно! Просто добавьте в секцию "Контекст (Context)" или "Задача (Task)" всего одну строчку: "Напиши это с использованием utility-классов `animate-` и конфигурации `tailwind.config.js` для Tailwind CSS" или "Используй синтаксис хелпера `keyframes` из Styled-Components". ИИ адаптирует код под вашу экосистему.

- **В: А что насчет перевода в синтаксис Framer Motion (React) или Transition (Vue)?**
  - О: Да, это тоже отлично работает. Укажите предпочитаемую библиотеку (например, "Создай компонент с использованием свойств `initial`, `animate` и `transition` из Framer Motion для React"), и ИИ сгенерирует оптимизированный компонент, используя все преимущества нужного синтаксиса.

- **В: Я применил код от ИИ, но анимация работает рывками. В чем дело?**
  - О: В 9 из 10 случаев падение производительности связано с тем, что вы или ИИ пытаетесь анимировать свойства, вызывающие перерасчет геометрии страницы (Reflow) — например, `width`, `height`, `top` или `left`. Как указано в разделе "Ограничения" продвинутой версии промпта, жестко требуйте от ИИ использовать только `transform: translate()` и `opacity`.

## 🧬 Анатомия промпта (Почему это работает?)

1. **Экспертная роль (Role):** Мы назначаем ИИ не просто генератором кода, а "Senior Interaction-дизайнером", что заставляет его выдавать эстетически выверенные и детальные результаты.
2. **Оцифровка абстракций (Task):** Промпт переводит человеческие эмоции ("как желе", "упруго", "плавно") в точный математический язык графиков `cubic-bezier`, избавляя вас от необходимости подбирать цифры.
3. **Ограничения ради производительности (Constraints):** Жесткий запрет на свойства, снижающие FPS (Jank), гарантирует создание безопасного для production-окружения кода с использованием только GPU-ускоряемых свойств.

## 📊 Доказательство: До и После

### ❌ До (Ручной подбор наугад)

```css
/* Бесконечные обновления страницы в попытках настроить margin/top... */
/* Анимация дергается из-за постоянного Reflow браузера */
.toast {
  animation: slide-in 3s ease;
}

@keyframes slide-in {
  0% {
    top: -50px;
    opacity: 0;
  }
  100% {
    top: 20px;
    opacity: 1;
  }
}
```

### ✅ После (Результат ИИ-промпта)

```css
/* Идеальное натяжение пружины (custom Cubic-bezier) и полное GPU-ускорение! */
.toast-jelly {
  animation: jelly-slide-fade 3s cubic-bezier(0.34, 1.56, 0.64, 1) forwards;
  will-change: transform, opacity;
}

@keyframes jelly-slide-fade {
  0% {
    transform: translateY(-100px) scale(0.8);
    opacity: 0;
  }
  40% {
    transform: translateY(10px) scale(1.05);
    opacity: 1;
  }
  60% {
    transform: translateY(0) scale(1);
    opacity: 1;
  }
  80% {
    transform: translateY(0) scale(1);
    opacity: 1;
  }
  100% {
    transform: translateY(-50px) scale(0.9);
    opacity: 0;
  }
}
```
