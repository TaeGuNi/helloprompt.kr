# 📝 사수 없이도 완벽한 코드 리뷰 & 리팩토링 가이드

- **🎯 추천 대상:** 주니어 개발자, 1~3년 차 백엔드/프론트엔드 엔지니어, 독학러
- **⏱️ 소요 시간:** 2시간 삽질 → 1분 컷
- **🤖 추천 모델:** Claude 3.5 Sonnet (코드 리뷰 최강자), GPT-4o
- ⭐ **난이도:** ⭐⭐☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐
_퇴근 시간은 다가오는데 에러 로그는 산더미 같고, 내 코드가 쓰레기 같지만 어디서부터 고쳐야 할지 막막할 때가 있지 않나요?_
사수에게 물어보기 눈치 보이는 자잘한 에러부터, "동작은 하는데 영 찝찝한" 스파게티 코드까지. 이제 AI를 당신의 개인 시니어 개발자로 활용해 보세요. 단순히 에러만 잡아주는 것을 넘어, 클린 코드 원칙에 입각한 실전 리팩토링 제안까지 받아낼 수 있습니다.
---
## ⚡️ 3줄 요약 (TL;DR)
1. AI에게 단순 '수정해 줘'가 아닌 '시니어 코드 리뷰어' 역할을 부여하세요.
2. 에러 로그와 현재 코드의 의도를 명확히 컨텍스트로 제공하세요.
3. 무작정 복붙하지 말고, AI가 제시한 '왜 고쳐야 하는지'에 대한 이유를 학습하세요.
---
## 🚀 해결책: "무자비한 시니어 개발자 프롬프트"
### 🥉 Basic Version (기본형)
빠르게 에러 원인만 파악하고 싶을 때 사용하세요.
> **역할:** 너는 10년 차 `[프론트엔드/백엔드]` 시니어 개발자야.
> **요청:** 아래 코드에서 발생하는 `[에러 내용]`의 원인을 찾고 해결 코드를 줘.
> **코드:** `[여기에 코드 붙여넣기]`
### 🥇 Pro Version (전문가형)
단순 버그 수정을 넘어, 아키텍처 개선과 클린 코드 리팩토링이 필요할 때 사용하세요.
> **역할 (Role):**
> 너는 FAANG 출신의 무자비하지만 실력 있는 15년 차 수석 소프트웨어 엔지니어(Staff Engineer)야. 가독성, 성능, 유지보수성을 극도로 타협하지 않는 성격이야.
> 
> **상황 (Context):**
> - 환경: `[React / Node.js / Python 등 기술 스택 입력]`
> - 목적: `[이 코드가 수행해야 하는 비즈니스 로직 설명]`
> - 현재 문제: `[발생 중인 에러 로그 전체 또는 성능 저하 등 문제 상황]`
> 
> **요청 (Task):**
> 1. 제공된 코드를 분석하고 근본적인 문제점(버그 원인, 안티 패턴 등)을 지적해.
> 2. 클린 코드 원칙(SOLID, DRY 등)에 입각하여 코드를 리팩토링해.
> 3. 수정된 전체 코드를 제공해.
> 4. 코드 리뷰 코멘트 형식으로 **왜** 이렇게 수정했는지 3가지 포인트로 설명해.
> 
> **제약사항 (Constraints):**
> - 불필요한 서론이나 인사말은 생략하고 즉시 리뷰를 시작해.
> - 최신 문법(ES6+ 등)과 해당 언어의 관용적(Idiomatic)인 패턴을 사용해.
> - 에러 핸들링과 엣지 케이스를 반드시 고려해.
> - 출력은 리스트 형태로 작성해 줘.
> 
> **주의사항 (Warning):**
> - 존재하지 않는 라이브러리나 메서드를 지어내지 마. (환각 금지)
> 
> **코드 (Code):**
> `[여기에 리뷰받을 코드를 붙여넣으세요]`

## 💡 작성자 코멘트 (Insight)
이 프롬프트의 핵심은 '무자비한 수석 엔지니어'라는 페르소나와 '왜 수정했는지 이유를 요구하는 것'입니다. 
현업에서 주니어들이 가장 많이 하는 실수는 AI가 짜준 코드를 원인도 모른 채 복붙(Ctrl+C, Ctrl+V)하고 넘어가는 것입니다. 당장 버그는 고쳐지겠지만 당신의 실력은 1mm도 성장하지 않습니다. 
이 프롬프트는 AI에게 "코드를 고쳐라"가 아니라 "나를 가르쳐라"라고 지시합니다. 안티 패턴을 지적받고, 관용적인 패턴(Idiomatic pattern)을 눈으로 익히세요. 특히 `Context` 영역에 여러분이 이 코드를 짤 때 무슨 의도였는지(비즈니스 로직)를 적어주면, AI가 단순히 문법 오류를 잡는 것을 넘어 구조적인 결함을 짚어냅니다. 변명하지 말고, AI 사수에게 영혼까지 털리며 배우십시오. 그것이 가장 빨리 성장하는 길입니다.

## 🙋 자주 묻는 질문 (FAQ)
- **Q: 회사 내부 코드를 그대로 올려도 안전한가요?**
  - A: 절대 안 됩니다! API 키, 사내 IP, 민감한 비즈니스 로직은 반드시 더미(Dummy) 데이터로 마스킹 처리하거나 변수명을 일반화한 뒤에 프롬프트에 넣으세요. 보안은 타협의 대상이 아닙니다.
- **Q: AI가 준 코드가 실행이 안 될 때도 있던데요?**
  - A: 환각(Hallucination) 현상입니다. 그럴 때는 당황하지 말고 발생하는 에러 로그를 그대로 긁어서 다시 질문하세요. "네가 준 코드에서 이런 에러가 나는데?"라고 추궁하면 알아서 수정본을 뱉어냅니다.
- **Q: 어떤 AI 모델이 코드 리뷰에 가장 좋나요?**
  - A: 2026년 기준, 코딩 분야에서는 Claude 3.5 Sonnet 이상의 모델이 압도적인 맥락 이해력과 리팩토링 능력을 보여줍니다. ChatGPT Plus(GPT-4o)도 훌륭한 선택지입니다.

## 🧬 프롬프트 해부 (Why it works?)
1. **무자비한 페르소나 부여:** 일반적인 AI는 사용자의 기분을 상하지 않게 하려고 치명적인 구조적 결함은 대충 넘어가고 작동만 되게 고쳐주는 경향이 있습니다. 깐깐한 수석 엔지니어 역할을 부여하여 타협 없는 리뷰를 강제합니다.
2. **Context 명시:** 컴퓨터는 코드를 읽지만, 시니어는 '의도'를 읽습니다. 코드가 무엇을 하려는지 설명하면 AI가 더 나은 접근 방식(다른 자료구조, 더 적절한 디자인 패턴)을 제안할 수 있습니다.
3. **이유(Why) 요구:** 성장을 위한 핵심 장치입니다. 결과 코드만 던져주는 것을 방지하고 지식 전달을 강제합니다.

## 📊 증명: Before & After
### ❌ Before (입력)
```javascript
// 유저 데이터를 가져와서 가공하는 코드 (스파게티 예시)
async function getU(id) {
  let res = await fetch('/api/user/' + id);
  let d = await res.json();
  if(d) {
    if(d.status === 'active') {
      let r = await fetch('/api/roles/' + d.roleId);
      let rd = await r.json();
      return { n: d.name, r: rd.name };
    } else {
      return null;
    }
  }
}
```
### ✅ After (결과)
```javascript
// AI 시니어 엔지니어의 리팩토링 결과
async function getActiveUserRoleSummary(userId) {
  try {
    const userResponse = await fetch(`/api/user/${userId}`);
    if (!userResponse.ok) throw new Error('Failed to fetch user');
    
    const user = await userResponse.json();
    
    // 조기 반환(Early Return) 패턴 적용으로 중첩 if문 제거
    if (!user || user.status !== 'active') {
      return null; 
    }
    const roleResponse = await fetch(`/api/roles/${user.roleId}`);
    if (!roleResponse.ok) throw new Error('Failed to fetch role');
    
    const role = await roleResponse.json();
    // 명확한 변수명 사용 (n, r -> name, roleName)
    return { 
      name: user.name, 
      roleName: role.name 
    };
  } catch (error) {
    console.error(`[getActiveUserRoleSummary] Error for userId ${userId}:`, error);
    throw error; // 에러 은닉 방지
  }
}
```
*(AI 코멘트: 중첩된 조건문을 Early Return으로 풀어 가독성을 높였고, 변수명을 직관적으로 수정했으며, 누락된 에러 핸들링을 추가했습니다.)*
