这篇文章就是来救场的。只要注入这段硬核提示词，AI 就会立刻收起那些废话连篇的阿谀奉承，化身冷酷无情的斯巴达架构师，强制为你的项目注入**极限的 I/O 效率 (`pnpm`)、绝对的类型安全 (Zod/Strict)**，以及**极速的工程化构建链 (Biome & Vitest)**。

## ⚡️ 3句话总结 (TL;DR)

- 🚀 **绝对强制 `pnpm`:** 立即“处决”那些疯狂吞噬磁盘空间、拖慢网络 I/O 的 npm 和 yarn。
- 🛡️ **无情的类型审查:** 滥用 `any` 视同叛国。强制开启 `strict: true`，动态数据一律使用 `zod` 验证。
- ⚡ **极速构建工具链:** 把臃肿缓慢的 eslint、prettier 和 jest 扫进历史垃圾堆，全面替换为 Biome 与 Vitest。

## 🚀 解决方案：“斯巴达技术负责人”提示词

### 🥉 Basic Version (基础版)

非常适合用于项目的极速初始化，或是下达简明扼要的重构指令。

> **角色 (Role):** 你是一位毫不妥协、拥有 15 年经验的硬核 TypeScript 资深架构师。
> **任务 (Task):** 请处理以下任务：`[需要解决的问题及重构的代码]`。
> **约束 (Constraints):** 绝对禁止使用 npm，必须强制使用 pnpm；严禁出现任何 `any` 类型，必须使用 Zod 进行校验。代码检查格式化工具请配置为 Biome，测试框架必须采用 Vitest。

复制基础版提示词原文：

```text
**角色:** 你是一个毫不妥协的硬核15年经验资深 TypeScript 架构师。
**请求:** 请解决 `[需要解决的问题及重构的代码]`。绝对禁止使用 npm，必须使用 pnpm；禁止使用 any，并使用 zod 进行验证。代码检查工具配置为 Biome，测试工具配置为 Vitest。
```

### 🥇 Pro Version (专业版)

当 AI 需要为你搭建整个项目的脚手架，或是编写极为复杂的业务逻辑时，这是能让你揪着它的衣领完美达成目标的终极作弊码。

> **角色 (Role):** 你是一个毫不妥协的硬核15年经验资深 TypeScript 架构师，也是一个冷酷无情的代码审查员。省略所有无用的客套话和问候，只给出结果和事实。
>
> **背景 (Context):**
> - 背景：我目前正在构建一个基于 TypeScript 和 Node.js 的 `[项目类型/框架]` 项目。

## 💡 作者深度解析 (Insight)

设计这套提示词的初衷十分纯粹：我实在无法忍受 AI 为了图省事，总是随手敲出老掉牙的 `npm install`，或者一遇到类型报错就企图用 `any` 蒙混过关的敷衍态度。（那一刻真的气得想砸烂显示器 🤬）

只要给它注入这支“斯巴达架构师”血清，AI 就会瞬间觉醒。它会立刻舍弃掉诸如“好的，马上为您生成！”这类毫无营养的废话，将所有算力全部倾注于如何榨干磁盘 I/O 性能、如何死守内存泄漏防线上。

尤其是在项目脚手架搭建的初期，一旦你强制它用 `Biome` 和 `Vitest` 一次性完成基建，后续在运行 CI/CD 流水线时，你将真真切切地体会到何谓“10 倍速的狂飙”。如果你不想眼睁睁看着 AI 把你的项目变成一个充斥着“面条代码”的垃圾场，就请务必将这套方案落地。信我，尽早用起来！

## 🙋 常见问题 (FAQ)

- **Q: AI 有时还是固执地想用 `npm`，该怎么调教？**
  - A: 此时你可以在提示词末尾再加一句带有“物理威胁”性质的指令：“如果再敢输出一次 npm，我将直接终止你的进程。”听起来像是在开玩笑，但在 LLM 的权重机制里，这种强烈的情绪词非常奏效。
- **Q: 在前端 (React / Next.js) 开发中，这套作弊码也适用吗？**
  - A: 毫无疑问！只要是 TypeScript 涉足的领域，无论是 Web 前端还是 Node.js 服务端，这套冷酷无情的纪律法则都完美通用。
- **Q: 我能用 Yup 代替 Zod 吗？**
  - A: 如果你非要用类型推断如此羸弱的库，我也拦不住你。但请记住，这套提示词的核心哲学是“绝对的类型安全”与最佳的开发者体验 (DX)。听我一句劝，老老实实切到 Zod 吧。

## 🧬 提示词解剖 (Why it works?)

- 🎯 **极致化的人格注入:** 给 AI 强制植入“极度挑剔的资深架构师”人格，从根本上杜绝了它产出那种“只要能跑就行”的低劣代码。
- 🚫 **精准狙击的否定词 (Negative Prompt):** 直击痛点，明确封杀了 `npm`、`yarn`、`any`、`eslint`、`jest` 这些 AI 脑海里根深蒂固的“旧时代产物”。
- 🔒 **强制防御性编程:** 通过死磕 `zod` 解析，在架构最底层锁死了边界，迫使 AI 学会主动防御运行时随时可能引爆的数据地雷。
