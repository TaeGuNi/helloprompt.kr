# 🎨 Создание дизайн-системы: Как дизайнерам и разработчикам перестать воевать

- **🎯 Для кого:** Для разработчиков, которым говорят: «Цвет кнопки не как на макете», и дизайнеров, вздыхающих: «Разработчик снова сломал мой дизайн».
- **⏱️ Сэкономленное время:** От 10 минут до 1 минуты (на базовое проектирование токенов и автоматическую генерацию кода).
- **🤖 Рекомендуемые модели:** Любые ИИ-ассистенты (ChatGPT-4o, Claude 3.5 Sonnet для архитектуры и кода).

- ⭐ **Сложность:** ⭐⭐⭐☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Применимость:** ⭐⭐⭐⭐⭐

> _«Разве наш Primary Color не `#0055FF`? Почему здесь `#0050FF`?»_

Захардкоженные цвета и отступы, разбросанные по всей кодовой базе — главные виновники того, что поддержка проекта превращается в ад. Внедрите **Дизайн-токены (Design Tokens)**. Это самый надежный способ согласовать все визуальные элементы: цвета, шрифты, отступы и тени, превратив их в осмысленные переменные. Как только система на базе токенов будет выстроена, вы ощутите настоящую магию: дизайнер меняет значение в Figma, и код разработчика синхронизируется автоматически.

---

## ⚡️ Главное в 3 пунктах (TL;DR)

1. **Токенизация дизайна:** Вместо абстрактного Hex-кода вроде `#0055FF` мы используем общий язык, например, `primary-500`, который понятен и людям, и машинам.
2. **Внедрение Figma Variables:** Используйте переменные (Variables) и стили (Styles) в Figma для структурирования визуальных свойств в виде системных токенов.
3. **Синхронизация кода:** На основе извлеченных токенов автоматически генерируйте конфигурацию `tailwind.config.ts` или глобальные CSS-переменные для мгновенного применения во фронтенде.

## 🚀 Решение: "Design Token Generator Prompt"

### 🥉 Базовая версия (Basic Version)

Используйте этот промпт, когда вам нужно быстро собрать начальную цветовую палитру и вы не знаете, как правильно и семантично назвать переменные.

> **Задача (Task):**
> Я разрабатываю цветовую палитру для дизайн-системы нового продукта.
> Мне нужно разбить цвета `[Primary (Синий)]`, `[Secondary (Серый)]` и `[Error (Красный)]` на 9 шагов — от 100 до 900.
> Чтобы эти цвета можно было сразу внедрить в код, предложи для каждого шага Hex-код и понятное семантическое имя (например, `text-primary`, `bg-surface-default`). Оформи результат в виде таблицы.

<br>

### 🥇 Продвинутая версия (Pro Version)

Идеально подходит для преобразования JSON-данных токенов, выгруженных из Figma, в готовый, безупречный код для фронтенда.

> **Роль (Role):** Ты — Senior Design Ops инженер и Frontend Architect с 10-летним опытом.
>
> **Контекст (Context):**
>
> - Ситуация: Дизайнер определил дизайн-токены в Figma и экспортировал их в формате JSON.
> - Цель: Безупречно преобразовать эти JSON-данные в готовый код для немедленного внедрения во фронтенд-проект.
>
> **Входные данные:**
>
> `[Вставьте сюда JSON-данные токенов, выгруженные из Figma]`
>
> **Задача (Task):**
>
> 1. Проанализируй входные данные и преобразуй их в формат современного конфигурационного файла **Tailwind CSS (`tailwind.config.ts`)**.
> 2. Чтобы не зависеть исключительно от CSS-фреймворков, сгенерируй также глобальные **CSS Variables (`:root { --color-blue-500: ... }`)**.
> 3. Добавь стратегию и примеры кода для поддержки темной темы (используя `@media (prefers-color-scheme: dark)` или класс `.dark`), чтобы система адаптировалась под настройки пользователя.
>
> **Ограничения (Constraints):**
>
> - Выведи результат исключительно в виде блоков кода Markdown (`ts`, `css`).
> - Строго соблюдай формат kebab-case для всех имен переменных.
>
> **Внимание (Warning):**
>
> - Не выдумывай и не добавляй произвольные цвета или шаги, которых нет в предоставленной JSON-структуре. (Предотвращение галлюцинаций)

## 💡 Заметки автора (Insight)

Создание дизайн-системы — это не то, что делается за один раз по грандиозному плану. Если вы попытаетесь сразу токенизировать абсолютно все компоненты и типографику, то, скорее всего, быстро выгорите и забросите эту идею.
Я настоятельно рекомендую стратегию **постепенного внедрения (Incremental Adoption)**. Начните с **самых часто используемых элементов** — например, цвета фона главной кнопки или цвета основного текста, — и постепенно заменяйте их на переменные.
Один маленький токен станет тем самым первым шагом, который радикально сократит издержки на коммуникацию между дизайнерами и разработчиками.

## 🙋 Часто задаваемые вопросы (FAQ)

- **В: Какой плагин лучше использовать для экспорта токенов из Figma?**
  - О: На практике стандартом де-факто (De facto) является плагин **'Tokens Studio for Figma'**. Он позволяет выгружать JSON и напрямую синхронизировать его с репозиторием GitHub. В последнее время нативные функции Figma Variables также стали очень мощными, поэтому интеграция через REST API набирает огромную популярность.

- **В: Нашему стартапу из 2-3 человек действительно нужна дизайн-система?**
  - О: Если в команде больше одного человека или проект планируется поддерживать дольше трех месяцев — внедрять нужно обязательно. Технический долг с отговоркой «потом как-нибудь наведем порядок» в итоге станет бомбой замедленного действия, которая критически замедлит разработку. Ваше «Я» в будущем — это тоже другой человек, которому придется в этом разбираться.

## 🧬 Разбор промпта (Why it works?)

1. **Принудительный семантический нейминг:** Вместо абстрактного «светло-синий» мы заставляем ИИ использовать осмысленные имена, вроде «цвет фона основной кнопки (`bg-primary-default`)». Если позже брендовый цвет изменится с синего на фиолетовый, вам потребуется изменить только один Hex-код, и это мгновенно отразится на всем проекте.
2. **Делегирование рутинного форматирования:** Вручную переводить формат JSON в объекты настроек Tailwind или синтаксис CSS-переменных — сущая пытка для человека. ИИ справляется с этой структурной трансформацией кода с поразительной точностью и скоростью, полностью исключая человеческий фактор (Human Error).

## 📊 Доказательство: До и После (Before & After)

### ❌ До (Ввод)

При каждом изменении дизайна приходится искать и заменять Hex-коды вручную в сотнях файлов. 😱

```css
.button-primary {
  background-color: #3b82f6;
}

.text-highlight {
  color: #3b82f6;
}
```

### ✅ После (Результат)

Достаточно изменить значение всего одной переменной, чтобы UI всего продукта идеально обновился. 🚀

```css
:root {
  --primary-500: #3b82f6;
  --bg-primary-default: var(--primary-500);
}

.button-primary {
  background-color: var(--bg-primary-default);
}
```

Перестаньте спорить из-за пикселей и оттенков. Начните общаться на общем языке токенов.
