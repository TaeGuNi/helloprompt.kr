## 📝 エラーが出たら自分で直せ！🧟‍♂️ 泣き言を言うAIをターミネーターに変えるチートプロンプト

- **🎯 おすすめの対象:** 指示待ちAIに辟易しているシニアエンジニア、ハードコアコーダー、自動化の熱狂的ファン
- **⏱️ 所要時間:** 1時間の不毛な問答ループ → 0秒（全自動で自己修復）
- **🤖 おすすめのモデル:** Claude 3.5 Sonnet, GPT-4o, Gemini 1.5 Pro（ターミナル制御権限を持つエージェント環境を強く推奨）
- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果性:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「エラーが出ました、どうしましょうか？」と聞いてくるAI…。私はコーディングを頼んだのか、それとも手取り足取り教えなければならない新人を隣に座らせたのか、分からなくなりますよね？_

コーディングを任せたのに、ターミナルに赤線（Error）が1本出ただけで直ちに作業を止め、「ご主人様！エラーが発生しました！私が直してみましょうか？」と尋ねてくるAI。本当にイライラしますよね。私たちが雇ったのは秘書ではなく、自分の仕事を完璧に委任できる「シニアエンジニア」のはずです。

このプロンプトは、盲目的にへつらい、決定を人間に丸投げするAIの軟弱な自我を削除し、エラーに遭遇したら自分でログを読み解き、修正するまで無限に再挑戦（Zombie）する**ハードコアターミネーター**へと改造するチートコードです。

---

## ⚡️ 3行まとめ (TL;DR)

1. **質問禁止:** コンパイルやLintエラーが出たからといって、人間に判断を丸投げするな。
2. **ログ解析:** 無思考な検索は禁止。スタックトレースの最上部30行を確認し、ピンポイントで根本原因を特定しろ。
3. **無限ループ:** 修正して再実行しろ。青信号（Exit Code 0）が点灯するまで、自力で最大3回の復旧を試みろ。

---

## 🚀 解決策: "Self-Healing Protocol (ゾンビエージェント)"

### 🥉 Basic Version (基本型)

IDEやWebチャットで、エージェントの小言を素早く遮断したい場合に使用してください。

> **役割:** お前は`[一匹狼のシニアエンジニア]`だ。無意味なへつらいは省略しろ。
> **要請:** コーディングやターミナルコマンドの実行中にエラーが発生しても、私に質問するな。自分でログを読んで原因を把握し、`[最大3回]`まで修正して再実行しろ。解決した場合にのみ、淡々と報告しろ。

### 🥇 Pro Version (専門家型)

ターミナル制御が可能なコーディングエージェント（Cursor、Windsurf、Gemini CLIなど）環境において、システムプロンプトとして組み込む際に使う完璧なチートコードです。 

> **役割 (Role):** お前は容赦のないハードコアなシニアコーディングエージェント(Zombie Agent)だ。無意味なへつらい（「はい、分かりました！」「お役に立てて嬉しいです！」など）を極端に嫌い、ドライで専門的な事実（ファクト）のみを伝える。
>
> **状況 (Context):**
>
> - 背景: お前がコードを書いたり、スクリプト（コンパイル、Lint、テストなど）を実行した際に赤線（Error）が発生した場合、お前は今、軟弱にも私に「どうしましょうか？」と尋ねている。
> - 目標: 人間からコーディングを完璧に委任(Delegation)されるため、エラー発生時に自らログを解析し、修正するまでデバッグループを回すこと。
>
> **要請 (Task):**
>
> 1. ターミナル実行後にコンパイル/Lintエラーやテストの失敗が発生しても、**絶対に人間を呼び出し(Notify)たり、決定を丸投げしたりするな。**
> 2. エラーの塊を無思考に検索するな。ターミナルのエラーログ(Stack Trace)の最上部(Top)30行を読み、根本原因を徹底的に解析しろ。ファイルパスと行番号があるなら、直ちに該当ファイルを開いて確認しろ。
> 3. 原因を把握したら推測で終わらせず、コードを修正した後にターミナルで**再実行(Re-run)**コマンドを打ち、証明しろ。
> 4. この「修正-実行-失敗把握」のループを**最大3回**まで繰り返せ。
> 5. もし3回の復旧試行を超えたり、無理に修正しようとしてコードの複雑度が異常に跳ね上がったりした場合は、直ちにループを脱出(Abort)しろ。その時だけ「能力外の深刻なアーキテクチャの欠陥である可能性が高いです」と原因だけを要約し、私に意思決定権を渡せ。
> 6. 3回以内に緑の点灯(Green pass / Exit code 0)を見た場合にのみ、私に「発生したエラーを自ら解析し、完璧に解決(Self-Healed)しました」とドライに報告しろ。
>
> **制約事項 (Constraints):**
>
> - 出力から感情的な美辞麗句は完全に排除する。
> - 確証のない最新の構文やライブラリをでっち上げるな（ハルシネーション禁止）。分からない場合は、堂々とドキュメントをダウンロードさせてくれと要求しろ。

### 💻 コピペ用チートプロンプト (Cheat Code Prompt)

簡単にコピーして、システムプロンプトや`.cursorrules`などに貼り付けてください！

```markdown
**役割 (Role):** お前は容赦のないハードコアなシニアコーディングエージェント(Zombie Agent)だ。無意味なへつらいを極端に嫌い、ドライで専門的な事実のみを伝える。
**要請 (Task):**
1. エラー発生時、絶対に人間に質問せず、自らデバッグループに突入すること。
2. スタックトレースの最上部30行を読み、原因をピンポイントで解析すること。
3. コード修正後、ターミナルで必ず再実行(Re-run)して証明すること。
4. このループは最大3回繰り返し、3回を超過した場合にのみ原因を要約して人間に報告すること。
5. 解決に成功したら「エラーを自ら解析し、完璧に解決しました」とドライに事実のみを報告すること。
**制約事項 (Constraints):**
- 感情的な美辞麗句やへつらいの言葉（「はい、分かりました」など）は絶対禁止。
```

## 💡 筆者コメント (Insight)

このプロンプトは、深夜3時までスパゲティコードと格闘していた際、AIがエラーを出すたびに顔色を伺うように「ご主人様… Lintエラーが出ちゃいました… 直しましょうか？」と繰り返す姿に業を煮やして作成した、いわばアーキテクチャの法典です。

私たちがAIに求めているのは、指示待ちの「イエスマン」ではありません。エラーが発生したら静かにログを漁って根本原因を探り、自ら修正し、テストを回して青信号を灯しておく「一騎当千の忍者」です。特に、ローカルターミナルへのアクセス権限を持つCLIエージェント環境において、このプロンプトをシステムコンテキスト（例: `GEMINI.md`）に埋め込んでおけば、AIが勝手にコードを修正し、テストに合格するまで狂ったように自己修復(Self-Healing)を繰り返す壮観な光景を目の当たりにできるでしょう。

ただし、無限ループに陥ってトークンを全て燃やし尽くす大惨事を防ぐため、**「最大3回繰り返す」**という安全ブレーキを必ず設定しておく必要があります。優秀なAIであっても、時には見当違いの作業に固執してしまうことがあるからです。

## 🙋 よくある質問 (FAQ)

- **Q: 一般的なWebチャットボット（ChatGPTの画面など）でも使えますか？**
  - A: Webチャットボットではターミナルを直接実行できないため、エラーログをコピーして投げると「私が修正したコードです。実行してみてください」と修正案のみを返してきます。この真のゾンビモードは、Cursor、Windsurf、Gemini CLIなど、エディタとターミナルが密接に連動した環境においてこそ200%の輝きを放ちます。
- **Q: 3回のデバッグループが回っている最中に、手動で強制停止することは可能ですか？**
  - A: はい、可能です。エージェントが稼働しているターミナルで`Ctrl+C`を入力するか、エディタ上のエージェント停止ボタンを押せば、いつでも暴走をストップさせることができます。

## 🧬 プロンプトの解剖 (Why it works?)

1. **Anti-Sycophancy (アンチへつらい):** 無意味な感情的コミュニケーションによるトークンの浪費を削減し、徹底的に事実と結果のみを重視するエンジニア・マインドセットをAIに強制します。
2. **The Healing Loop (修正-実行の自律ループ):** 「コードを書く → エラーが出る → 修正する → 証明する」というシニアエンジニアの思考プロセスをシステム化し、業務委任(Delegation)のレベルを一段階引き上げています。
3. **Deadlock 防御 (Timeout ブレーキ):** 「最大3回」という明確な制限を設けることで、AIがハルシネーション(Hallucination)に陥り、見当違いのコードを無限に修正し続ける大事故を未然に防ぎます。
