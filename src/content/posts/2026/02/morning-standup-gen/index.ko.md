# 📝 1분 만에 끝내는 어제 업무 요약 & 스탠드업 준비

- **🎯 추천 대상:** 매일 아침 스탠드업 회의(데일리 스크럼)가 부담스러운 주니어 개발자, PM, 기획자
- **⏱️ 소요 시간:** 30초 (기존 대비 최소 10분 단축)
- **🤖 추천 모델:** 모든 대화형 AI (ChatGPT, Claude, Gemini 등)

- ⭐ **난이도:** ⭐☆☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐☆
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"어... 어제 저는... 그... 뭐냐... 자잘한 버그 좀 고쳤고요..." (이제 이런 자존감 갉아먹는 보고는 그만합시다!)_

아침 회의 시간, 더 이상 모니터 구석에 띄워둔 메모장이나 깃허브(GitHub) 커밋 로그를 더듬거리며 읽지 마세요. AI가 당신의 파편화된 작업 내역을 **'비즈니스 가치가 돋보이는 성과 중심의 브리핑'**으로 완벽하게 번역해 드립니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. `git log`, 지라(Jira) 티켓 제목, 혹은 두서없이 적어둔 메모를 그대로 복사합니다.
2. 아래 제공되는 '스탠드업 자동화 프롬프트'에 붙여넣고 실행합니다.
3. "어제 한 일 / 오늘 할 일 / 공유 사항(Blocker)" 포맷으로 깔끔하게 정리된 스크립트를 당당하게 읽기만 하면 끝!

---

## 🚀 해결책: "스탠드업 브리핑 생성기"

### 🥉 Basic Version (기본 요약형)

복잡한 설정 없이, 흩어진 메모를 빠르게 정돈하고 싶을 때 사용하세요.

> **역할:** 너는 내 업무를 깔끔하게 요약해주는 `[비서]`야.
> **요청:** 내가 제공하는 작업 내용을 바탕으로 '어제 한 일', '오늘 할 일', '공유 사항(Blocker)' 세 가지 항목으로 요약해줘.
>
> **작업 내용:**
> `[여기에 커밋 로그나 메모, 티켓 제목 등을 자유롭게 붙여넣기]`

<br>

### 🥇 Pro Version (성과 강조 및 가치 번역형)

단순한 작업 나열을 넘어, 내가 조직에 기여한 '가치(Value)'를 팀원들과 리더에게 확실하게 어필하고 싶을 때 사용하세요.

> **역할 (Role):** 너는 커뮤니케이션 스킬이 뛰어난 `[유능한 시니어 개발자 및 테크 리드]`야.
>
> **상황 (Context):**
>
> - 배경: 매일 아침 팀원들과 진행하는 스탠드업 회의(데일리 스크럼)를 준비하고 있어.
> - 목표: 파편화된 작업 기록을 바탕으로, 내가 달성한 성과를 명확하고 프로페셔널하게 전달하는 스크립트를 작성해야 해.
>
> **요청 (Task):**
>
> 1. 내가 입력한 데이터의 기술적 작업 내역을 팀원(비개발자 포함)들이 이해하기 쉬운 **'비즈니스 가치'로 번역**해 줘. (예: "로그인 API 리팩토링" -> "로그인 속도 개선 및 보안 강화")
> 2. 다음 템플릿 구조에 맞춰서 스크립트를 작성해.
>
> **[어제 진행 상황]**
>
> - (단순 나열이 아닌 핵심 성과 위주로 3개 이내의 불릿 포인트로 작성)
>
> **[오늘 계획]**
>
> - (구체적이고 달성 가능한 목표 1~2개)
>
> **[도움 필요한 점 (Blocker)]**
>
> - (입력 데이터에 문제가 있거나 지연 요소가 있다면 명시, 없으면 "현재 공유할 Blocker는 없습니다"로 처리)
>
> **입력 데이터 (Input):**
> `[어제 작업한 커밋 로그, 지라(Jira) 티켓 목록 또는 작업 메모 입력]`
>
> **제약사항 (Constraints):**
>
> - 말투는 "했습니다", "예정입니다"로 끝나는 정중하고 자신감 있는 구어체를 사용해.
> - 내용이 너무 길어지지 않게, 항목당 발표 시간이 20초를 넘지 않도록 간결하게 유지해.
> - 입력 데이터에 없는 내용을 지어내지 마. (환각 방지)

## 💡 작성자 코멘트 (Insight)

이 프롬프트는 특히 비동기 커뮤니케이션이 잦은 리모트 워크(재택근무) 환경에서 슬랙(Slack)이나 팀즈(Teams)로 데일리 리포트를 남길 때 강력한 위력을 발휘합니다.

단순히 "로그인 에러 수정함"이라고 무미건조하게 보고하는 것과, "사용자 로그인 시 발생하던 간헐적 인증 오류를 해결하여 이탈률을 방어했습니다"라고 보고하는 것은 리더가 당신의 퍼포먼스를 평가하는 데 있어 **하늘과 땅 차이**를 만듭니다. 이 프롬프트는 당신의 기술적 수고로움을 비즈니스 언어로 포장해 주는 훌륭한 '연봉 협상 대비용' 무기이기도 합니다. 결과물이 조금 오버스럽다면, 당신의 평소 말투에 맞게 톤앤매너(Tone & Manner)만 살짝 다듬어 사용하세요.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 개발자 말고 마케터나 디자이너도 쓸 수 있나요?**
  - A: 물론입니다! `Pro Version` 프롬프트의 **역할 (Role)** 부분을 `[퍼포먼스 마케터]`나 `[UX/UI 디자이너]`로 변경하고, 작업 내용에 어제 분석한 지표나 작업한 디자인 시안 내역을 넣으시면 해당 직군에 완벽하게 맞는 브리핑이 생성됩니다.

- **Q: 지라(Jira)나 노션(Notion) 티켓 번호만 넣어도 알아서 요약해주나요?**
  - A: 보안 및 권한 문제로 AI가 외부 프라이빗 링크에 직접 접속할 수는 없습니다. 귀찮더라도 티켓의 제목과 핵심 내용(Description)을 긁어서 텍스트 형태로 붙여넣어 주셔야 합니다.

- **Q: 글로벌 팀이라 영어로 발표해야 하는데 어떻게 하죠?**
  - A: 프롬프트 하단 **제약사항 (Constraints)**에 `- 출력 언어는 비즈니스 영어(Business English)로 작성해 줘.` 한 줄만 추가하시면 네이티브 수준의 영문 스크립트를 얻을 수 있습니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **가치 번역(Value Translation) 지시:** 단순 요약이 아니라 "기술 용어를 비즈니스 가치로 번역해 줘"라는 명확한 행동 지침(Action Task)을 주어 결과물의 질을 극적으로 끌어올렸습니다.
2. **명확한 출력 구조(Template):** 스탠드업 회의의 표준 포맷인 '어제/오늘/블로커'를 템플릿화하여 강제함으로써, AI가 엉뚱한 형태로 답변하는 것을 방지했습니다.

## 📊 증명: Before & After

### ❌ Before (입력 데이터)

```text
- 결제 모듈 API 500 에러 나는거 원인 파악함. (DB 커넥션 문제인듯?)
- 어드민 페이지 버튼 CSS 깨지는거 수정 완.
- 지라 티켓 #1024 (회원가입 약관 동의 버그) 작업 중인데 QA팀 확인 필요함.
```

### ✅ After (AI 결과물)

```text
**[어제 진행 상황]**
- 결제 모듈에서 발생하던 서버 에러(500)의 근본 원인이 DB 커넥션 병목임을 파악하고 대응 준비를 마쳤습니다. 이를 통해 결제 실패율을 낮출 수 있을 것으로 기대합니다.
- 어드민 페이지의 UI 레이아웃 깨짐 현상을 수정하여 운영팀의 업무 불편을 해소했습니다.

**[오늘 계획]**
- 어제 파악한 결제 모듈 DB 커넥션 문제를 수정하여 배포할 예정입니다.
- 회원가입 약관 동의 관련 버그(티켓 #1024) 수정을 마무리하고 QA를 요청하겠습니다.

**[도움 필요한 점 (Blocker)]**
- 회원가입 약관 동의 버그 수정 건과 관련하여, QA팀에서 오늘 중으로 테스트 환경을 확인해 주시면 일정 내 배포가 가능할 것 같습니다. 이 부분 도움 부탁드립니다.
```
