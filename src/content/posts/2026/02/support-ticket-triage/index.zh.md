# 📝 别让愤怒的客户等待

- **🎯 推荐对象：** 客服经理、电商运营、客户体验 (CX) 负责人
- **⏱️ 所需时间：** 1小时 → 1分钟
- **🤖 推荐模型：** GPT-3.5 Turbo (快速), GPT-4o (精准)

- ⭐ **难度：** ⭐⭐☆☆☆
- ⚡️ **效果：** ⭐⭐⭐⭐☆
- 🚀 **实用度：** ⭐⭐⭐☆☆

> _"速度就是最好的服务。请守住愤怒客户的黄金救援时间。"_

面对留言板上堆积如山的数百条客服询单，您是否感到过无从下手？按照时间顺序逐一回复，却错过了最紧急的“物流事故”或“重复扣款”问题，最终导致客户怒火中烧——您一定也有过这样的经历。现在，借助AI的力量，您可以提前筛选出客户的情绪状态和问题的紧急程度。就像急诊室的“分诊系统（Triage）”一样，彻底重塑您的客服优先级吧。

---

## ⚡️ 3行总结 (TL;DR)

1. **数据收集：** 汇总客户的咨询记录，并将其提交给AI。
2. **AI 智能分诊：** AI 将分析并提取每条咨询的“情绪分数”与“问题分类”。
3. **优先响应：** 优先处理情绪分数高（愤怒/不满）且分类紧急（付款/物流）的客诉，迅速化解危机。

---

## 🚀 解决方案："客服急诊室分诊提示词"

### 🥉 Basic Version (基础版)

当您只需要快速对问题进行分类时使用。

> **角色 (Role)：** 你是一位响应迅速且判断精准的 `[客服主管]`。
> **任务 (Task)：** 请阅读以下咨询内容，判断其 `[分类]` 和 `[紧急程度（高/中/低）]` 并给出结论。
> **咨询内容 (Input)：** `[在此处粘贴客户咨询内容]`

<br>

### 🥇 Pro Version (专业版)

当您需要结合客户情绪分析来管理流失风险，并明确处理优先级时使用。此结构非常适合与自动化流水线（如 Zapier、Make 等）无缝对接。

> **角色 (Role)：** 你是一位兼具冷静判断力与深度共情能力的 `[资深客户体验 (CX) 专家]`。
>
> **背景 (Context)：**
>
> - **现状：** 在客服人力有限的情况下，我们需要处理海量激增的客户咨询，因此迫切需要设定明确的响应优先级。
> - **目标：** 核心目标是优先识别并安抚高不满情绪（高流失风险）的客户，防止用户流失。
>
> **任务 (Task)：**
>
> 请分析提供的 `[咨询列表]`，并完成以下三项任务：
>
> 1. **情绪评分 (Sentiment Score)：** 评估每条咨询的情绪状态，打分范围为 1~5 分。（1分：非常满意/愉悦 ～ 5分：极其愤怒/强烈不满）
> 2. **问题分类：** 将咨询归入以下分类之一：物流 / 退款 / 产品缺陷 / 普通咨询 / 其他。
> 3. **优先级及理由：** 综合情绪评分和分类，指定处理优先级（High/Medium/Low），并用一句话清晰说明判断理由。
>
> **约束条件 (Constraints)：**
>
> - **输出格式：** 必须仅以 JSON 数组（Array）格式输出。（此为系统对接专用，绝不可添加任何解释性文字）
> - **JSON 结构示例：** `[{"id": 1, "sentiment": 5, "category": "退款", "priority": "High", "reason": "强烈抗议退款延迟，并提及将采取法律手段"}]`
>
> **输入数据 (Input)：**
> `[在此处粘贴需要分析的完整咨询列表]`

## 💡 作者点评 (Insight)

这个提示词的真正价值，不在于单纯地获取一段文本回复，而在于为您构建**“自动化工作流”**时大放异彩。由于我们严格将输出格式限制为 JSON，它可以与 Zapier 或 Make 等自动化工具完美兼容。例如，您可以这样设置：**“当 AI 识别出情绪评分为 5 分（极其愤怒）的咨询时，立即向客服团队的 Slack 频道发送 [紧急警报] 消息。”** 这样就能在客户的怒火蔓延到社交媒体或社区之前，抢占黄金时间进行干预。自从我引入这套方法后，成功将可能演变为恶性客诉的事件数量大幅降低。

## 🙋 常见问题 (FAQ)

- **Q: 一次最多可以分析多少条咨询记录？**
  - A: 以 GPT-4o 为例，一次处理 50~100 条咨询毫无压力。但考虑到 Token 限制，建议不要一次性输入过多（如数千条），而是按每小时或半天为单位分批进行处理。

- **Q: 如果客户使用反讽或委婉的说法，AI 也能准确捕捉吗？**
  - A: 像 GPT-4o 这样的最新模型能够根据上下文，准确识别出“你们这发货速度真是‘快’得惊人啊^^”这类反讽（Sarcasm）背后的不满情绪。不过，如果您所在的行业有特殊的黑话或行话，建议在 `背景 (Context)` 部分补充说明：“在我们的行业中，‘某某’一词意味着出现了严重错误”，这样准确率会大幅提升。

---

## 🧬 提示词解剖 (Why it works?)

1.  **情绪量化 (Sentiment Scoring)：** 将客户模糊的情绪感受转化为 1~5 分的定量数据，从而实现了系统化的机器分类与过滤。
2.  **明确约束 (Constraints)：** 强制 AI 仅输出 JSON 格式，杜绝任何废话，确保在与 API 或无代码工具对接时不会出现解析错误。
3.  **要求决策依据 (Reasoning)：** 要求 AI 用一句话说明将其评为 High（高优先级）的理由，使客服人员在点开具体咨询前就能预判事态，赋予了工作以“可预测性”。
