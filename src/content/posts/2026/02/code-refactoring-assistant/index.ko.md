# 📝 내 코드가 스파게티? 리팩토링 전문가에게 맡겨보세요

- **🎯 추천 대상:** 주니어 개발자, 코드 리뷰를 앞둔 엔지니어, 기술 부채에 시달리는 팀
- **⏱️ 소요 시간:** 1시간 → 10분 단축
- **🤖 추천 모델:** Claude 3.5 Sonnet (코드 리팩토링 최상위), ChatGPT (GPT-4o)

- ⭐ **난이도:** ⭐⭐☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"기능만 겨우 돌아가는 '스파게티 코드', 내일의 당신과 동료를 괴롭히는 시한폭탄입니다. 이제 AI 시니어 개발자에게 코드의 품격을 맡겨보세요."_

기능은 멀쩡하게 동작하지만 읽기 힘들고 유지보수가 불가능한 코드는 결국 거대한 기술 부채가 되어 돌아옵니다. 이 프롬프트는 얽히고설킨 복잡한 로직을 단순화하고, 모호한 변수명을 직관적으로 수정하며, 최신 문법과 디자인 패턴을 적용하여 당신의 코드를 '예술'의 경지로 끌어올려 줍니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **가독성 극대화:** 직관적인 변수명과 함수명으로 코드의 진짜 의도를 명확하게 전달합니다.
2. **모던 문법 적용:** ES6+ 등 최신 트렌드와 언어 스펙에 맞는 세련된 문법으로 리팩토링합니다.
3. **유지보수성 향상:** 단일 책임 원칙(SRP) 준수 및 꼼꼼한 예외 처리로 잠재적 버그를 예방합니다.

## 🚀 해결책: "Clean Code Mentor"

### 🥉 Basic Version (기본형)

빠르게 코드 가독성을 높이고 싶을 때 가볍게 사용하세요.

> **역할:** 너는 클린 코드(Clean Code) 전문가야.
> **요청:** 아래 제공된 코드를 가독성이 높고 유지보수하기 쉽게 리팩토링해 줘.
> **코드:**
> `[여기에 수정할 코드를 붙여넣으세요]`

<br>

### 🥇 Pro Version (전문가형)

단순한 수정을 넘어, 아키텍처 관점의 깊이 있는 코드 리뷰와 테스트 코드까지 필요할 때 사용하세요.

> **역할 (Role):** 너는 실리콘밸리 빅테크 기업의 10년 차 시니어 소프트웨어 엔지니어(Senior Software Engineer)야.
>
> **상황 (Context):**
>
> - 사용 언어: `[프로그래밍 언어, 예: TypeScript]`
> - 주요 목적: `[성능 최적화 / 가독성 향상 / 버그 수정 중 택 1]`
>
> **요청 (Task):**
>
> 1. 제공된 코드를 꼼꼼히 리뷰하고, 현재 코드의 구조적 문제점(Code Smell)을 3가지 이내로 날카롭게 지적해 줘.
> 2. 최신 언어 스펙과 디자인 패턴을 적용하여 완벽하게 리팩토링된 코드를 작성해 줘.
> 3. 변경된 핵심 로직에 대해 JSDoc(또는 해당 언어의 표준 문서화 형식) 스타일의 주석을 달아 줘.
> 4. (선택) 리팩토링된 코드를 안전하게 검증할 수 있는 단위 테스트(Unit Test) 코드를 2개 이상 작성해 줘.
>
> **제약사항 (Constraints):**
>
> - 변수명과 함수명은 반드시 그 역할이 명확히 드러나도록(Descriptive) 지어야 해.
> - 함수는 오직 하나의 일만 수행하도록(Single Responsibility Principle) 작게 분리해 줘.
> - 발생 가능한 엣지 케이스(Edge Case)를 고려하여 에러 처리를 철저하게 해 줘.
>
> **주의사항 (Warning):**
>
> - 리팩토링 과정에서 기존 비즈니스 로직의 결과값이 절대 변경되어서는 안 돼.

## 💡 작성자 코멘트 (Insight)

AI에게 리팩토링을 맡길 때 개발자들이 가장 흔히 하는 실수는 **'결과 코드만 복사해서 덮어쓰기'**하는 것입니다. 진짜 개발 실력을 키우려면 AI가 코드를 수정한 '이유(Code Smell)'에 집중해야 합니다. 또한 제안된 코드를 프로덕션에 적용하기 전에 반드시 기존 테스트 코드를 실행하여 회귀 버그(Regression Bug)가 없는지 확인하세요. 만약 기존에 작성된 테스트 코드가 없다면, 프롬프트를 활용해 AI에게 "테스트 코드를 먼저 작성해 달라"고 요청하는 것이 안전하고 전문적인 리팩토링의 첫걸음입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 회사 코드 베이스의 특정 코딩 컨벤션(예: Airbnb, Google)을 따르게 할 수 있나요?**
  - A: 네, 완벽하게 가능합니다. 제약사항(Constraints) 부분에 `"Airbnb JavaScript Style Guide를 엄격하게 준수해 줘"` 또는 `"회사 사내 규칙인 Prettier/ESLint 룰을 따른다고 가정해 줘"`라고 추가 지시를 내리면 AI가 해당 스타일을 정확히 반영합니다.

- **Q: 수십 개의 파일이 얽힌 복잡한 프로젝트도 한 번에 리팩토링이 가능한가요?**
  - A: 현재 웹 기반 LLM 모델들의 컨텍스트 윈도우(Context Window) 한계로 인해, 전체 프로젝트를 한 번에 넣기보다는 파일 단위나 핵심 함수/클래스 단위로 쪼개서 요청하는 것이 훨씬 더 정확하고 안전합니다. 프로젝트 전체 구조 개선이 필요하다면 Cursor, GitHub Copilot Chat 등 코드베이스 전체를 인덱싱하는 전용 AI IDE를 활용하는 것을 강력히 추천합니다.

## 🧬 프롬프트 해부 (Why it works?)

1.  **Role(페르소나) 부여:** "10년 차 시니어 엔지니어"라는 구체적인 역할을 부여하여, 단순한 문법 오류 수정을 넘어 구조적이고 아키텍처 관점의 깐깐한 코드 리뷰를 유도했습니다.
2.  **Constraints(제약사항) 강제:** '단일 책임 원칙(SRP)', '명확한 변수명' 등 클린 코드의 핵심 원칙을 명시하여 AI가 자의적으로 코드를 과도하게 축약하거나 오히려 가독성을 해치는 방향으로 수정하는 것을 원천 차단했습니다.

## 📊 증명: Before & After

### ❌ Before (입력)

```javascript
function a(n) {
  if (n < 2) return n;
  return a(n - 1) + a(n - 2);
}
```

### ✅ After (결과)

```javascript
/**
 * 피보나치 수열의 n번째 값을 계산합니다. (재귀 방식)
 *
 * @param {number} n - 계산할 순번 (0 이상의 정수)
 * @returns {number} n번째 피보나치 수
 * @throws {Error} 입력값이 음수일 경우 에러를 발생시킵니다.
 */
function getFibonacci(n) {
  if (n < 0) {
    throw new Error("입력값은 0 이상의 정수여야 합니다.");
  }

  if (n < 2) {
    return n;
  }

  return getFibonacci(n - 1) + getFibonacci(n - 2);
}
```
