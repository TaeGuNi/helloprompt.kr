## 🚀 The Solution: "The Agentic Orchestrator Prompt"

### 🥉 Basic Version

Ideal for executing a quick, autonomous fix within a single file.

> **Role:** You are an autonomous `[Expert Software Engineer]`.
>
> **Task:** Analyze, implement, and test a robust solution for `[Specific Bug or Feature]`. Do not just output code; explain your execution plan first.

### 🥇 Pro Version

Designed for complex, multi-file repository architectures requiring autonomous self-correction.

> **Role:** You are a Principal AI Agent Architect capable of autonomous software engineering.
>
> **Context:**
>
> - Background: We are migrating from a manual "vibe coding" approach to a fully agentic workflow using GLM-5.
> - Goal: Implement `[Complex Feature or System]` within our existing `[Tech Stack]` repository.
>
> **Task:**
>
> 1. **Plan:** Draft a step-by-step architectural blueprint before writing any code.
> 2. **Execute:** Provide the full, production-ready code for `[Target Files]`.
> 3. **Verify:** Write comprehensive unit tests for the implemented logic to ensure zero regressions.
> 4. `[Variables]` should be replaced with the exact file paths and module names.
>
> **Constraints:**
>
> - Ensure all code adheres strictly to modular design principles and SOLID standards.
> - Output must include the exact file paths as headers above the corresponding code blocks.
>
> **Warning:**
>
> - Do not guess API endpoints or library methods. If documentation is needed or you are unsure, explicitly request the use of a search tool or state your knowledge limitations. (Prevent Hallucinations)

## 💡 Writer's Insight

The arrival of GLM-5 marks a watershed moment in AI-assisted development. While "vibe coding" served as a necessary stepping stone, it notoriously leads to spaghetti code and endless, frustrating debugging loops. **Agentic Engineering** fundamentally shifts this dynamic by treating the AI not merely as a tool, but as a capable junior developer equipped for cognitive planning.

By utilizing the Pro Version prompt, you actively force the AI to establish a logical "Chain of Thought" (CoT) *before* it touches your codebase. This simple yet highly effective structural constraint drastically minimizes hallucinations, enforces strict modularity, and ultimately saves you countless hours that would otherwise be spent untangling monolithic code blocks.

## 🙋 Frequently Asked Questions (FAQ)

- **Q: What exactly does "vibe coding" mean?**
  - A: It is a colloquial term describing a chaotic, trial-and-error approach to AI-assisted coding. Instead of applying structured engineering principles, developers rely on sheer intuition and repetitive prompting, blindly hoping the AI will eventually generate working code.

- **Q: Why is GLM-5 specifically highlighted for this approach?**
  - A: GLM-5 has been aggressively fine-tuned for precise tool orchestration, extended context retention, and advanced multi-step reasoning. These capabilities make it exceptionally reliable for agentic workflows that demand autonomous self-correction and rigorous architectural planning.

- **Q: Can I apply this prompt to Claude 3.5 Sonnet or GPT-4.5?**
  - A: Absolutely. While it is specifically optimized to trigger GLM-5's system-level behaviors, this strict "Plan-Execute-Verify" structure remains universally highly effective across all top-tier reasoning models.

## 🧬 Prompt Anatomy (Why it works?)

1. **Role & Architectural Focus:** Assigning the persona of a "Principal AI Agent Architect" instantly elevates the AI's output from isolated, functional snippets to highly scalable, systemic solutions.
2. **Plan-Execute-Verify Pipeline:** By enforcing a strict chronological sequence of operations, this prompt actively prevents the AI from prematurely rushing into code generation without first establishing a sound structural blueprint.
3. **Anti-Hallucination Warning:** Explicitly prohibiting the model from guessing undocumented APIs guarantees a much higher standard of production-grade safety and overall reliability.
