## 🚀 해결책: "엑셀/VBA 마스터 프롬프트"
### 🥉 Basic Version (기본형)
빠르게 간단한 수식이나 단축키가 필요할 때 사용하세요.
> **역할:** 너는 20년 차 `[엑셀/VBA 최고 전문가]`야.
> **요청:** 내가 가진 데이터는 `[A열: 이름, B열: 부서, C열: 매출]`이야. `[영업부의 매출 합계]`를 구하는 가장 효율적이고 에러 없는 수식을 작성해줘.
### 🥇 Pro Version (전문가형)
복잡한 다중 조건, 데이터 전처리 자동화, 또는 VBA 매크로 코드가 필요할 때 사용하세요. 이 프롬프트는 에러를 최소화하고 정확한 셀 참조를 만들어냅니다.
> **역할 (Role):** 너는 포춘 500대 기업의 데이터 분석가이자 `[엑셀 및 VBA 최고 전문가]`야.
> 
> **상황 (Context):**
> - 배경: 나는 매주 `[수백 줄의 로우 데이터를 정제해서 보고서 포맷으로 변환]`하는 작업을 하고 있어.
> - 목표: `[수동으로 복사/붙여넣기 하는 과정을 없애고, 버튼 하나로 자동화하거나 완벽한 수식을 적용]`하는 것이 목표야.
> 
> **데이터 구조 (Data Structure):**
> - 시트 이름: `[RawData]`
> - A열: `[날짜 (YYYY-MM-DD)]`
> - B열: `[제품 코드 (예: PRD-1023)]`
> - C열: `[판매 수량 (숫자)]`
> - D열: `[담당자명]`
> 
> **요청 (Task):**
> 1. 위 데이터를 바탕으로 `[특정 날짜 범위 내, 제품 코드별 판매 수량 합계를 구하는 VBA 매크로 코드]`를 작성해줘.
> 2. `[작동 방식과 코드를 엑셀 모듈에 적용하는 방법]`을 아주 짧고 명확하게 설명해.
> 3. 만약 함수(수식)로 해결하는 것이 VBA보다 더 효율적이고 빠르다면, 함수 방식을 우선적으로 제안해줘.
> 
> **제약사항 (Constraints):**
> - 출력 형식은 불릿 포인트(List)와 마크다운 코드블럭으로만 해줘.
> - 불필요한 서론이나 인사말은 절대 생략하고 바로 본론(코드/수식)부터 출력해.
> - VBA 코드를 작성할 때는 각 줄마다 한글로 상세한 주석을 반드시 달아줘.
> 
> **주의사항 (Warning):**
> - 엑셀 버전 충돌이 발생할 수 있는 최신 함수(예: XLOOKUP)를 사용할 경우, 구버전(VLOOKUP+MATCH) 대안도 짧게 언급해. 확실하지 않은 객체나 메서드는 지어내지 말고 엑셀 기본 함수만 사용해.

## 💡 작성자 코멘트 (Insight)
이 프롬프트의 핵심은 **'데이터 구조(Data Structure)'를 AI에게 명확히 맵핑(Mapping)해주는 것**입니다.
대부분의 실무자가 AI에게 "A팀 매출 합계 구하는 함수 알려줘"라고 대충 묻습니다. AI는 A팀이 어느 열(Column)에 있는지, 매출이 숫자인지 텍스트 섞인 값인지 알 길이 없어 일반적인 템플릿만 던져주고, 결국 사용자가 셀 주소(A2:A100 등)를 일일이 수정해야 하는 귀찮음이 발생합니다.
하지만 위 Pro 프롬프트처럼 A열, B열에 어떤 데이터가 있는지 선언해주면, AI가 `SUMIFS(RawData!C:C, RawData!A:A, ...)` 처럼 당장 복붙해서 쓸 수 있는 실전용 코드를 뱉어냅니다. 
실무는 실전입니다. 함수 구조를 뜯어보고 매크로 문법을 공부하느라 귀중한 1시간을 버리지 마십시오. 여러분의 진짜 가치는 엑셀 문법을 암기하는 데 있는 것이 아니라, 산출된 데이터를 해석하고 비즈니스 결정을 내리는 데 있습니다. 타이핑은 AI에게 외주를 주고 본질에 집중하십시오.

## 🙋 자주 묻는 질문 (FAQ)
- **Q: 구글 스프레드시트에서도 동일하게 작동하나요?**
  - A: 네, 가능합니다. 다만 요청(Task) 부분에 "구글 스프레드시트용 수식/앱스 스크립트(Apps Script)로 작성해줘"라고 명시해야 함수 호환성 문제가 생기지 않습니다.
- **Q: AI가 준 수식을 넣었는데 에러 값(#N/A, #VALUE!)이 나오면 어떻게 하나요?**
  - A: 에러가 뜬 해당 셀을 복사해서 AI에게 "이 수식을 썼더니 `#N/A` 에러가 났어. IFERROR로 빈 칸 처리하는 코드도 추가해줘"라고 피드백하면 10초 만에 완벽한 수정본을 줍니다.
- **Q: 무료 버전 ChatGPT에서도 되나요?**
  - A: 네, 간단한 엑셀 수식 작성은 무료 버전으로도 충분합니다. 다만, 수십 줄이 넘어가는 복잡한 VBA 코드를 작성할 때는 맥락을 잘 유지하는 Pro 버전이 훨씬 정교합니다.

## 🧬 프롬프트 해부 (Why it works?)
- **데이터 구조 맵핑 (Data Structure):** 데이터의 생김새를 구체적으로 알려주어, 사용자가 수동으로 셀 참조 범위($A$1 등)를 수정해야 하는 번거로움을 원천 차단했습니다.
- **제약 조건 (Constraints):** 현업에서 복붙하기 바쁜데 AI가 "안녕하세요! 엑셀 작업을 도와드릴게요~" 하며 떠드는 서론 시간을 없앴습니다.
- **최적화 기법 (Task 3번):** 가벼운 작업에 무거운 VBA를 쓰지 않도록, 수식이 더 빠르면 수식을 제안하라는 논리를 넣어 가장 효율적인 결과물을 유도했습니다.

## 📊 증명: Before & After
### ❌ Before (입력)
```text
A열에 날짜, B열에 제품코드, C열에 수량 있는데 PRD-1023 수량 합계 구하는 매크로 좀.
```
(결과: 변수명이 엉망이거나, 셀 범위가 고정되어 있어 데이터가 늘어나면 에러가 나는 코드를 뱉거나 불필요한 서론이 긺)
### ✅ After (결과)
```text
1. 엑셀 수식 (VBA보다 더 효율적이고 빠름)
=SUMIFS(RawData!C:C, RawData!B:B, "PRD-1023")
2. VBA 매크로 코드 적용이 반드시 필요한 경우
Sub CalculateProductSales()
    Dim ws As Worksheet
    Dim lastRow As Long
    Dim totalSales As Double
    Dim i As Long
    
    ' 시트 설정
    Set ws = ThisWorkbook.Sheets("RawData")
    
    ' 데이터의 마지막 행 찾기 (데이터 추가 시 자동 대응)
    lastRow = ws.Cells(ws.Rows.Count, "A").End(xlUp).Row
    totalSales = 0
    
    ' 조건 검색 및 합산
    For i = 2 To lastRow ' 1행이 헤더이므로 2행부터 시작
        If ws.Cells(i, 2).Value = "PRD-1023" Then
            totalSales = totalSales + ws.Cells(i, 3).Value
        End If
    Next i
    
    ' 결과 출력
    MsgBox "PRD-1023의 총 판매 수량은 " & totalSales & "개 입니다.", vbInformation, "결과"
End Sub
```
