> - Добавь подробные комментарии к каждой ключевой логике (затухание скорости, слушатели событий и т.д.), чтобы облегчить будущие модификации.

## 💡 Комментарий автора (Insight)

Самое важное при написании кода с помощью ИИ — **«относиться к ИИ не как к инструменту автодополнения, а как к старшему партнеру (Senior Partner), с которым вы обсуждаете архитектуру и проектируете структуру»**.
Секрет успеха кроется не в попытке сразу создать огромную идеальную игру, а в **поэтапном наращивании (Step-by-step)**: базовая физика (трение, столкновения) -> обработка ввода (обратная связь drag & shoot) -> расширение контента (дизайн уровней).

Claude Code обладает выдающейся способностью поддерживать контекст и рефакторить код. Если в процессе разработки вы попросите: "Вынеси управление препятствиями в отдельный класс для рефакторинга", вы получите гораздо более легко поддерживаемый результат.

## 🧬 Разбор промпта (Why it works?)

1. **Принудительная модульность (Modularity):** Указание разделить файлы с самого начала формирует архитектурный каркас. Это предотвращает превращение кода в спагетти по мере его роста и облегчает поддержку и масштабирование.
2. **Спецификация визуальной обратной связи (Visual Indicator):** Выходя за рамки простого управления, явное требование «линии, показывающей силу и направление» значительно улучшает игровой опыт (UX) и интуитивность.
3. **Учет масштабируемости (Level Design):** Побуждение к использованию данных уровней в виде массива вместо жестко закодированного одного уровня создает структуру, управляемую данными. Это позволяет разработчикам легко и бесконечно добавлять новые уровни в будущем.

## 📊 Доказательство: До и После (Before & After)

### ❌ До (При простом запросе "сделай игру в гольф")

```text
Когда мяч ударяется о стену, он навсегда исчезает за пределами холста, или обработка событий клика мыши сбивается, заставляя мяч бесконечно ускоряться. Из-за отсутствия понятия трения пола, мяч катится вечно, никогда не останавливаясь.
```

### ✅ После (Пример физической логики, сгенерированной Продвинутой версией)

```javascript
// Сгенерировано Claude Code
update() {
  this.velocity.x *= 0.98; // Трение: Симуляция сопротивления травы для плавного замедления
  this.velocity.y *= 0.98;

  this.x += this.velocity.x;
  this.y += this.velocity.y;

  // Обработка столкновений со стенами холста (естественный отскок)
  if (this.x < 0 || this.x > canvas.width) {
    this.velocity.x *= -1; // Смена направления
  }
  if (this.y < 0 || this.y > canvas.height) {
    this.velocity.y *= -1;
  }
}
```

_Результат: Claude Code точно применил математическую концепцию демпфирования (затухания), которая со временем снижает скорость, идеально воссоздав "силу трения травы на поле для гольфа"._
