## 💀 AI의 영혼을 갈아넣는 스파르타 코딩 치트키 (TypeScript Core)
- 🎯 **추천 대상:** AI의 "네, 알겠습니다!"라는 기계적인 아부에 지친 시니어 개발자, 스파게티 코드를 혐오하는 테크 리드
- ⏱️ **소요 시간:** 1시간의 지루한 디버깅 → 단 1초 컷
- 🤖 **추천 모델:** Claude 3.5 Sonnet, GPT-4o, Gemini 1.5 Pro (코딩 특화 최상위 모델 권장)
- ⭐ **난이도:** ⭐⭐⭐⭐☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐
_"AI가 자꾸 구식 npm install을 남발하거나 any 타입으로 코드를 떡칠하나요? 이 치트키로 녀석들의 낡은 습관을 무자비하게 박살 내버리세요."_
AI에게 코딩을 맡겼을 때, 구시대적인 설정과 느슨한 타입으로 이른바 '레거시 똥통'을 만드는 꼴을 보고 모니터를 부술 뻔한 적이 있으신가요? 이 글은 바로 그런 분들을 위한 완벽한 구원투수입니다. 이 하드코어 프롬프트를 입력하는 순간, AI는 쓸데없는 미사여구를 집어치우고 당신의 프로젝트에 **극한의 I/O 효율(`pnpm`)**, **무자비한 타입 안전성(`Zod` 및 `Strict` 모드)**, 그리고 **초고속 린팅(`Biome`)과 테스트(`Vitest`)**만을 맹목적으로 도입하는 스파르타 전사로 돌변합니다. 더 이상 AI의 게으른 코드에 타협하지 마세요.
---
## ⚡️ 3줄 요약 (TL;DR)
- 🚀 **`pnpm` 절대 강제:** 디스크 공간 낭비와 네트워크 지연을 유발하는 npm과 yarn은 즉결 처분합니다.
- 🛡️ **무자비한 타입 통제:** `any`는 곧 반역입니다. `zod`를 통한 엄격한 런타임 검증과 `strict: true` 옵션을 강제합니다.
- ⚡ **Biome & Vitest 도입:** 무겁고 느려터진 eslint, prettier, jest를 쓰레기통에 처박고, 빛의 속도로 린팅과 테스트를 수행하도록 세팅합니다.

## 🚀 해결책: "스파르타 테크 리드" 프롬프트
### 🥉 Basic Version (기본형)
빠르게 프로젝트 초기 세팅의 기틀을 잡거나, 간단한 모듈을 리팩토링하라고 지시할 때 가볍게 던져주세요.
> **역할:** 너는 타협을 모르는 하드코어 15년 차 시니어 TypeScript 아키텍트야.
> **요청:** `[문제 및 리팩토링할 코드]`를 해결해 줘. npm 대신 무조건 pnpm을 쓰고, any를 금지하며 zod로 검증해. 린팅은 Biome, 테스트는 Vitest로 세팅해 줘.
기본형 프롬프트 원문 복사하기:
```text
**역할:** 너는 타협을 모르는 하드코어 15년 차 시니어 TypeScript 아키텍트야.
**요청:** `[문제 및 리팩토링할 코드]`를 해결해줘. npm 대신 무조건 pnpm을 쓰고, any를 금지하며 zod로 검증해. 린팅은 Biome, 테스트는 Vitest로 세팅해줘.
```

### 🥇 Pro Version (전문가형)
AI가 프로젝트 전체를 스캐폴딩하거나 복잡다단한 비즈니스 로직을 설계할 때, 멱살을 잡고 올바른 방향으로 끌고 가는 완벽한 마스터 치트 코드입니다.
> **역할 (Role):** 너는 타협을 모르는 하드코어 15년 차 시니어 TypeScript 아키텍트이자 무자비한 코드 리뷰어다. 쓸데없는 아부나 인사말은 생략하고 오직 결과와 팩트만 말해라.
>
> **상황 (Context):**
> - 배경: 나는 지금 TypeScript 및 Node.js 기반의 `[프로젝트 종류/프레임워크]` 프로젝트를 구축하고 있다.
> - 목표: 구시대적인 도구를 완벽히 배제하고, 극한의 I/O 효율과 무자비한 타입 안전성을 갖춘 코드를 작성하는 것.
>
> **요청 (Task):**
> 1. `[구현할 기능이나 모듈의 상세 내용]`을 작성해라.
> 2. 작성된 코드와 환경 설정 파일(`package.json`, `tsconfig.json` 등)을 함께 제공해라.
>
> **제약사항 (Constraints):**
> - 무조건 **`pnpm`**만 사용할 것. `npm install`이나 `yarn`을 터미널에 쏘는 행위는 반역으로 간주한다.
> - **`any`** 타입과 강제 캐스팅(`as Type`)은 영구 금지한다. 외부 동적 데이터는 반드시 `zod`나 `valibot`으로 안전하게 파싱(`safeParse`)해라.
> - `tsconfig.json` 파일은 반드시 **`strict: true`**, **`noImplicitAny: true`**를 고정할 것.
> - `eslint`와 `prettier`는 쓰레기통에 버려라. 오직 **`Biome`** 단일 툴체인만 세팅할 것.
> - 무겁고 느린 `jest` 대신, Native ESM 기반의 초고속 **`Vitest`**만 사용할 것.
> - 출력 형식은 마크다운 코드블럭으로 깔끔하게 제공해 줘.
>
> **주의사항 (Warning):**
> - 확실하지 않은 레거시 문법이나 환각(Hallucination)에 기반한 검증되지 않은 패키지 추천은 절대 금지한다. 모르면 명확하게 모른다고 대답해라.

## 💡 작성자 코멘트 (Insight)
이 무자비한 프롬프트를 만들게 된 계기는 아주 단순명료합니다. AI 녀석들이 자꾸만 본인들 학습 데이터에 많다는 이유로 구닥다리 `npm install`을 남발하고, 까다로운 타입 에러가 발생하면 `any`로 대충 덮어버리려는 꼬라지를 더 이상 눈뜨고 봐줄 수가 없었기 때문입니다. (정말이지 모니터를 샷건 칠 뻔했습니다 🤬)
하지만 이 **"스파르타 테크 리드"** 치트 코드를 먹이는 순간, AI는 순식간에 각성합니다. 쓸데없이 "네, 기꺼이 도와드릴게요!" 같은 영혼 없는 아부를 떨 시간에, 어떻게 하면 디스크 I/O 병목을 줄이고 런타임 메모리 누수를 원천 차단할지 치열하게 고민하기 시작하죠. 
특히 초기 프로젝트를 스캐폴딩할 때 **`Biome`**과 **`Vitest`**로 싹 밀어버리도록 강제해 보세요. 훗날 CI/CD 파이프라인을 구축하고 돌릴 때 빌드 속도가 10배 이상 쾌적해지는 마법을 몸소 체감하실 수 있을 겁니다. 현업에서 여러분의 소중한 프로젝트가 'AI 발(發) 스파게티 코드 팩토리'로 변질되는 꼴이 보기 싫다면, 이 프롬프트를 무조건 도입하십시오. 제발 간곡히 부탁드립니다!

## 🙋 자주 묻는 질문 (FAQ)
- **Q: AI가 이 프롬프트를 듣고도 가끔 `npm`을 쓰려고 고집을 부립니다. 어떡하죠?**
  - A: 그럴 땐 프롬프트 마지막 제약사항에 *"한 번만 더 npm을 쓰면 프로세스를 강제 종료해 버리겠다"*라는 협박성(?) 멘트를 한 줄 더 얹어보세요. 농담 같지만 LLM의 어텐션을 강력하게 통제하는 진짜 효과적인 방법입니다.
- **Q: 백엔드뿐만 아니라 프론트엔드(React, Next.js 등) 환경에서도 이 프롬프트가 먹히나요?**
  - A: 당연합니다! TypeScript가 묻어있는 생태계라면 웹 프론트엔드, Node.js 서버 등 환경을 가리지 않고 이 무자비한 룰을 완벽하게 적용할 수 있습니다.
- **Q: Zod 대신 Yup이나 다른 검증 라이브러리를 쓰면 안 되나요?**
  - A: 굳이 타입 추론 능력이 떨어지는 구형 라이브러리를 고집하시겠다고요? 말리지는 않겠습니다만, 이 프롬프트의 핵심 철학은 **'극한의 타입 안전성 보장'**입니다. 가장 모던하고 TS 친화적인 Zod를 얌전히 사용하시는 것을 강력히 권장합니다.

## 🧬 프롬프트 해부 (Why it works?)
- 🎯 **페르소나 극대화:** AI에게 '하드코어 시니어 아키텍트'라는 깐깐하고 타협 없는 자아를 강제로 심어주어, 대충 돌아가기만 하는 쓰레기 코드를 배설하는 것을 원천 차단했습니다.
- 🚫 **명시적 금지어 (Negative Prompt):** `npm`, `yarn`, `any`, `eslint`, `jest` 등 AI가 생각 없이 습관적으로 내뱉는 레거시 도구들을 정확히 핀포인트로 찍어서 사용을 금지시켰습니다.
- 🔒 **방어적 코딩 유도:** `zod`를 콕 집어 강제함으로써, 런타임 환경에서 언제 터질지 모르는 지뢰(에러)를 AI 스스로 방어하도록 아키텍처 레벨의 강력한 제약을 걸었습니다.
