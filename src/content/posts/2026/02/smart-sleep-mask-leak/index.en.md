## 🚀 The Solution: "Crisis Management & Incident Report Prompt"

When a critical IoT security vulnerability goes public, you need to rapidly assess the technical fallout and draft a transparent, ironclad response. Use this prompt to transform raw network findings into a polished, publication-ready report.

### 🥉 Basic Version

Deploy this for a rapid, high-level summary of any emerging security threat.

> **Role:** You are an expert `[Cybersecurity Analyst]`.
> **Task:** Summarize the `[Somnium X data leak]` incident and suggest three immediate remediation steps.

### 🥇 Pro Version

Deploy this when you need a meticulously detailed, publication-ready incident report paired with a strategic PR response.

> **Role:** You are a seasoned `[Cybersecurity Journalist and PR Crisis Manager]`.
>
> **Context:**
>
> - Background: `[The Somnium X smart mask, designed for snoring detection, was caught uploading unencrypted .wav audio files of users' sleep and ambient noise to a public AWS S3 bucket.]`
> - Goal: `[Write a comprehensive incident report and draft an official PR apology for the company.]`
>
> **Task:**
>
> 1. Detail the technical problem (e.g., unauthorized data transmission, exposed metadata like location and device ID).
> 2. Explain the discovery process (e.g., network traffic analysis by anonymous researcher 'SleepWalker').
> 3. Provide a draft for the company's official public response (citing a "beta testing debugging feature error") along with a concrete mitigation strategy.
> 4. Keep variables enclosed in `[brackets]` so they can be seamlessly adapted for other IoT devices.
>
> **Constraints:**
>
> - Format the output as a professional Markdown document with clear, hierarchical headings.
> - Maintain an objective, highly authoritative journalistic tone throughout.
>
> **Warning:**
>
> - Do not invent or hallucinate technical details. If the exact scale of the data leak is unknown, explicitly state that it is "yet to be determined."

## 🙋 Frequently Asked Questions (FAQ)

- **Q: Can this prompt analyze actual network logs?**
  - A: While AI can effectively summarize plain-text log snippets, you should **never** upload sensitive or raw user data to public AI models. Stick strictly to analyzing the _findings_ and _summaries_ of those logs.

- **Q: How can I adapt this prompt for a different IoT device?**
  - A: Simply swap out the variables in the `[Context]` section. Replace "Somnium X" with your target device, and update the vulnerability specifics (e.g., "hardcoded admin passwords" instead of "unencrypted audio").

## 🧬 Prompt Anatomy (Why it works?)

1. **Dual Persona:** Blending rigorous technical analysis with strategic crisis communication yields a well-rounded, highly actionable output.
2. **Anti-Hallucination Constraints:** Explicitly instructing the AI not to guess the scale of the damage is absolutely crucial for credible cybersecurity reporting.
3. **Structured Context:** Segmenting the prompt into Background, Goal, and Task ensures the AI captures critical elements—like the discoverer ('SleepWalker') and the specific file format (`.wav`)—without losing its narrative focus.
