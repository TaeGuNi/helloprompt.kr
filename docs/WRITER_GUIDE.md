# ✍️ 작가 가이드 (Writer's Guide)

Hello Prompt의 콘텐츠 품질을 유지하기 위한 집필 매뉴얼입니다.
모든 작가(AI 포함)는 이 가이드를 준수해야 합니다.

## 1. 집필 워크플로우 (Workflow)

1.  **템플릿 복사:** `docs/POST_TEMPLATE.md` 파일을 복사하여 시작합니다.
2.  **파일명 규칙:** `kebab-case` 영문으로 작성합니다. (예: `excel-pivot-table.md`)
3.  **작성:** 템플릿의 빈칸을 채웁니다.
4.  **검수:** `docs/QUALITY_MODEL.md` 기준으로 자가 점검을 수행합니다.

## 2. 톤앤매너 (Tone & Manner)

- **친절한 전문가:** 너무 딱딱하지 않게, 하지만 신뢰감 있게 말합니다.
  - ❌ "이것은 A입니다." (너무 딱딱함)
  - ❌ "이거 완전 대박!" (너무 가벼움)
  - ✅ "이 방법을 쓰면 10분 걸릴 일을 1분에 끝낼 수 있습니다." (적절함)
- **독자의 언어:** 전문 용어는 풀어서 설명합니다. (예: "LLM" -> "AI 모델")

## 3. SEO 키워드 배치 (SEO Keyword Placement)

검색 엔진 최적화를 위해 키워드를 전략적으로 배치합니다.

- **제목(H1):** 메인 키워드를 반드시 포함합니다. (예: **엑셀 피벗 테이블** 쉽게 만드는 법)
- **첫 문단:** 글의 첫 100자 이내에 핵심 키워드를 자연스럽게 언급합니다.
- **소제목(H2, H3):** 연관 키워드를 사용하여 검색 엔진이 구조를 이해하기 쉽게 합니다.
- **이미지 Alt 태그:** 이미지 설명에 키워드를 포함하여 이미지 검색에도 노출되도록 합니다.

## 4. 포맷팅 규칙 (Formatting)

### 프롬프트 박스 (Prompt Box)

독자가 복사해서 쓸 명령어는 반드시 `>` (인용구)로 감쌉니다.

> 예시:
>
> > **역할:** 너는 엑셀 전문가야.

### 결과 박스 (Result Box)

AI의 답변 예시는 반드시 ` ```text ` (코드 블럭)으로 감쌉니다.

```text
예시:
# 결과물
...
```

### 프론트매터 날짜 (Frontmatter Date)

- 글의 상단 메타데이터(`date`, `updatedDate`)는 **ISO 8601 포맷**(`YYYY-MM-DDTHH:mm:ss.sssZ`)을 사용합니다.
- 예: `date: "2026-02-08T12:00:00.000Z"`
- **자동화:** 빌드 스크립트가 파일 생성 시간을 기준으로 자동 업데이트하므로, 수동으로 수정할 필요는 거의 없습니다.

### 강조 (Highlight)

- 중요한 단어는 **볼드체**로 강조합니다.
- 핵심 문구는 `인라인 코드`로 감싸서 눈에 띄게 합니다.

## 5. 금지 사항 (Do Not)

- **서론 길게 쓰기:** 독자는 바쁩니다. 바로 본론(해결책)으로 들어가세요.
- **거짓 정보:** 확인되지 않은 AI 기능(예: "이미지 생성 됩니다" 등)을 쓰지 마세요.
- **모호한 제목:** "AI 꿀팁" (X) -> "회의록 3초 만에 요약하는 프롬프트" (O)

## 6. 다국어 번역 가이드 (Localization)

Hello Prompt는 전 세계 사용자를 위해 9개 국어로 서비스를 제공합니다.

- **지원 언어:** `en` (영어), `de` (독일어), `es` (스페인어), `fr` (프랑스어), `it` (이탈리아어), `ja` (일본어), `pt` (포르투갈어), `ru` (러시아어), `zh` (중국어)
- **파일 위치:** `src/pages/{언어코드}/posts/`
- **파일명 규칙:** 원본(한국어) 파일의 영문 파일명과 **동일하게** 유지해야 합니다. (자동 동기화 스크립트가 이를 기준으로 동작합니다)
