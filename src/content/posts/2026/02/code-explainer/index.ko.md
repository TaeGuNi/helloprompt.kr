# 📝 이 코드, 무슨 뜻이죠? 친절한 코드 해설사

- **🎯 추천 대상:** 주니어 개발자, 비전공자, 코드 분석이 막막한 실무자
- **⏱️ 소요 시간:** 10분 → 1분 단축
- **🤖 추천 모델:** 모든 대화형 AI (ChatGPT, Claude, Gemini 등)

- ⭐ **난이도:** ⭐☆☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"돌아는 가는데... 도대체 왜 돌아가는 거지?"_

개발자의 영원한 미스터리입니다. 스택오버플로우나 깃허브에서 긁어온 코드가 작동은 하는데, 어딜 고쳐야 내 입맛대로 바꿀지 모르겠다면? 혹은 퇴사한 전임자가 남기고 간 '암호문' 같은 레거시 코드를 당장 내일까지 수정해야 한다면 어떨까요?

단순히 "이 코드 설명해 줘"라고 AI에게 묻는다면, 교과서적인 뻔한 대답만 돌아올 확률이 높습니다. 이 프롬프트는 AI를 **친절한 시니어 사수**로 변신시켜, 복잡한 로직을 한 줄 한 줄 뜯어보고 실무 수준의 주석을 달아주도록 설계되었습니다. 이제 모르는 코드 앞에서 식은땀 흘릴 필요 없습니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **전체 맥락 파악:** 복잡한 코드의 핵심 로직과 목적을 3문장으로 깔끔하게 요약합니다.
2. **한 줄 한 줄 완벽한 주석:** 코드의 모든 라인에 직관적인 한글 주석을 달아 작동 원리를 시각화합니다.
3. **핵심 문법 과외:** 초보자가 헷갈리기 쉬운 특수 문법이나 낯선 라이브러리 함수를 따로 빼서 명쾌하게 설명합니다.

## 🚀 해결책: "친절한 시니어의 코드 리뷰"

### 🥉 Basic Version (기본형)

빠르게 대략적인 흐름만 파악하고 싶을 때 사용하세요.

> **역할:** 너는 친절하고 경험 많은 시니어 개발자야.
> **요청:** 내가 제공한 코드가 어떤 기능을 하는지 요약하고, 핵심 로직에 한글 주석을 달아 알기 쉽게 설명해 줘.

<br>

### 🥇 Pro Version (전문가형)

코드의 원리를 완벽하게 이해하고 내 것으로 만들고 싶을 때 사용하세요.

> **역할 (Role):** 너는 경험이 풍부하고 후배 육성에 진심인 `[시니어 개발자 멘토]`야.
>
> **상황 (Context):**
>
> - 배경: 나는 이 코드가 정확히 무슨 일을 하는지, 어떤 원리로 작동하는지 100% 이해하고 싶어.
> - 목표: 내 현재 실력은 `[초급/중급]` 수준이야. 내 눈높이에 맞춰서 이 코드를 완벽하게 해설해 줘.
>
> **요청 (Task):**
>
> 1. **[핵심 요약]**: 이 코드의 존재 이유와 전체 흐름을 누구나 이해할 수 있게 3문장으로 요약해.
> 2. **[라인 바이 라인 분석]**: 코드의 모든 줄(혹은 중요한 논리적 블록) 위에 직관적인 **한글 주석**을 달아서 코드를 다시 작성해 줘.
> 3. **[문법 및 패턴 포인트]**: 이 코드에서 사용된 특수한 문법(예: 리스트 컴프리헨션, 화살표 함수 등)이나 특정 라이브러리의 핵심 함수가 있다면 따로 빼서 예시와 함께 설명해 줘.
>
> **제약사항 (Constraints):**
>
> - 불필요하게 어려운 전문 용어 남발은 금지하며, 비유를 들어 설명하면 더 좋아.
> - 변수나 함수의 이름이 가진 의미와 역할을 명확히 짚어줘.
> - **할루시네이션 방지:** 모르는 라이브러리나 문법이 있다면 억지로 지어내지 말고, "해당 부분은 정확한 확인이 필요하다"라고 답변해.
>
> **입력 코드 (Input):**
>
> ```
> [여기에 분석할 코드를 붙여넣으세요]
> ```

## 💡 작성자 코멘트 (Insight)

실무에서 남의 코드를 읽는 시간은 코드를 직접 짜는 시간보다 훨씬 깁니다. 특히 문서화가 전혀 되어 있지 않은 레거시 프로젝트를 인수인계받았을 때, 이 프롬프트는 엄청난 위력을 발휘합니다.

단순히 코드를 번역하듯 해석하는 데 그치지 않고, `[초급/중급]`이라는 변수를 통해 AI의 설명 난이도를 직접 컨트롤할 수 있다는 점이 핵심입니다. 만약 주니어 프론트엔드 개발자라면, `[상황]` 영역에 *"나는 React 기반의 프론트엔드 개발자야. 이 백엔드(Python) 코드가 프론트엔드와 어떻게 데이터를 주고받는지 중심으로 설명해 줘"*라고 구체적인 맥락을 추가해 보세요. 훨씬 입체적이고 실무에 직결되는 인사이트를 얻을 수 있습니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 코드 길이가 수천 줄이 넘어가도 분석이 가능한가요?**
  - A: 한 번에 수천 줄을 통째로 넣으면 AI가 중요한 문맥을 놓치거나 출력이 중간에 끊길 수 있습니다. 핵심이 되는 함수나 클래스 단위(50~100줄 내외)로 잘게 쪼개서 질문하는 것을 강력히 권장합니다.

- **Q: 에러가 발생하는 코드의 원인도 찾아줄 수 있나요?**
  - A: 네, 가능합니다. 프롬프트 하단에 `[발생한 에러 로그]`를 추가하고, `요청 (Task)` 부분에 "4. 에러 발생 원인과 해결 방법 제시" 항목을 추가하면 디버깅 모드로도 훌륭하게 작동합니다.

- **Q: 최신 프레임워크(Next.js 14 등) 문법도 정확하게 설명하나요?**
  - A: AI 모델의 학습 데이터 기준일에 따라 최신 문법(예: React Server Components)에 대한 답변이 부정확할 수 있습니다. 최신 기술 스택을 다룰 때는 GPT-4o나 Claude 3.5 Sonnet처럼 웹 검색 기능이 있거나 최신 지식을 갖춘 모델을 사용하는 것이 안전합니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **페르소나와 타겟의 매칭:** AI에게 `[시니어 개발자]`라는 역할을 부여함과 동시에, 질문자인 나의 수준을 `[초급/중급]`으로 명시하여 답변의 난이도를 내게 딱 맞게 최적화했습니다.
2. **구조화된 출력 (Task 쪼개기):** 단순 해설이 아닌 `[요약] -> [라인별 분석] -> [문법 포인트]`의 3단계로 출력을 강제하여, 독자가 숲(전체 로직)과 나무(세부 문법)를 모두 조망할 수 있도록 설계했습니다.
3. **안전장치 (Constraints):** 어려운 용어 금지 및 환각(Hallucination) 방지 지시를 통해, AI의 그럴듯한 거짓말 때문에 개발자가 삽질하는 시간을 원천 차단했습니다.

## 📊 증명: Before & After

### ❌ Before (입력 - Python)

```python
result = [x * 2 for x in range(10) if x % 2 == 0]
```

### ✅ After (결과)

**[핵심 요약]**
이 코드는 0부터 9까지의 숫자 중에서 '짝수'만 골라내는 로직입니다. 골라낸 짝수들에 각각 2를 곱하여, 그 결과값들로 이루어진 새로운 리스트를 생성합니다. 데이터를 조건에 맞게 필터링하고 가공할 때 자주 쓰이는 패턴입니다.

**[라인 바이 라인 분석]**

```python
# 1. range(10): 0부터 9까지의 숫자를 하나씩 차례대로 생성합니다.
# 2. if x % 2 == 0: 생성된 숫자(x)를 2로 나누었을 때 나머지가 0인 경우(즉, 짝수)만 통과시킵니다.
# 3. x * 2: 조건을 통과한 짝수(x)에 2를 곱해줍니다.
# 4. **[...]**: 이 모든 과정을 거친 최종 값들을 모아 새로운 리스트(result)로 완성합니다.
result = [x * 2 for x in range(10) if x % 2 == 0]
```

**[문법 및 패턴 포인트]**

- **리스트 컴프리헨션 (List Comprehension):** 파이썬의 꽃이라 불리는 문법입니다. `for` 반복문과 `if` 조건문을 여러 줄로 길게 쓰지 않고, 단 한 줄의 대괄호 `[]` 안에서 처리하여 새로운 리스트를 우아하고 빠르게 만들어냅니다.
  - 구조: `[적용할_수식 for 변수 in 반복가능한_데이터 if 조건문]`
