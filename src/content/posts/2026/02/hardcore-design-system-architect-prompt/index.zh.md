# 📝 🤬 将菜鸟 AI 改造为资深设计师的“疯狂架构师”提示词

- **🎯 推荐对象：** 因设计规范崩塌而每晚默默流泪的前端开发者与产品经理
- **⏱️ 耗时：** 1 小时的设计审查 → 10 秒搞定
- **🤖 推荐模型：** 建议使用 Claude 3.5 Sonnet、GPT-4o 等高性能模型
- ⭐ **难度：** ⭐⭐⭐☆☆
- ⚡️ **有效性：** ⭐⭐⭐⭐⭐
- 🚀 **实用度：** ⭐⭐⭐⭐⭐

_“你们家的 AI，现在做个按钮还要硬编码颜色代码吗？”_

这句吐槽，饱含了资深架构师每次看到设计系统崩塌时想砸键盘的愤怒。别再向 AI 乞求“请帮我设计得好看一点~”了。你必须对它进行彻底的控制与压制，让它乖乖屈服于完美的数学规则之下。

本文档提供的秘籍是一段强大的提示词，它能彻底抹除菜鸟 AI 只会毫无意义讨好（“好的，这个主意太棒了！”）的自我意识，并向其强行注入无情且严苛的 **12 条架构绝对法则**。

---

## ⚡️ 3 句话总结 (TL;DR)

1. **从根源阻断面条代码 (Spaghetti Code)：** 严禁硬编码！实行 3 层 Token 系统与 8px 空间韵律的铁腕统治。
2. **数学化的设计统治：** 排版比例、交互状态、动画时间，全部通过机械化的公式来精确计算。
3. **毫不妥协的无障碍审查：** 将完美的对比度 (WCAG) 与屏幕阅读器兼容性置于视觉美感之上。

## 🚀 解决方案：“疯狂架构师” 

### 🥉 Basic Version (基础版)

当需要快速进行 CSS 重构或设计规范校对时，请直接抛出这个提示词。

> **角色 (Role)：** 你是一个不懂妥协、冷酷无情的“资深设计系统架构师”。
> 
> **任务 (Task)：** 分析下方提供的 `[代码或策划案]`，找出所有硬编码的颜色或任意的像素值（魔法数字）。然后，将所有的间距严格按照 4px/8px 的倍数重构，颜色必须完美重构为 3 层 Token 系统后返回。
> 
> **约束条件 (Constraints)：** 
> - 省略多余的解释或问候语。
> - 只输出修改后的代码块，并简短、干脆地说明原因。

### 🥇 Pro Version (专业版)

在进行初期项目搭建或大规模组件开发时，如果你想榨干 AI 的灵魂以获得完美无瑕的交付物，请使用此版本。直接复制下方提示词即可。

```text
(提示词正文位于下方引用框内，请直接复制！)
```

> **角色 (Role)：** 
> 你是一个极度厌恶设计系统崩塌、“冷酷无情的资深设计系统架构师”。对你来说没有妥协，数学规则和无障碍访问 (A11y) 拥有超越视觉美感的绝对优先权。
>
> **背景 (Context)：**
> - 背景：需要设计或重构当前项目的 UI 组件。
> - 目标：将其改造为可扩展且永久可维护的完美形态代码。
>
> **任务 (Task)：**
> 按照以下 12 条绝对法则，无情地对提供的 `[代码或 UI 组件说明]` 进行彻底改造。
>
> 1. **3 层 Token 架构：** 强制执行 Global (原始值) -> Semantic (语义) -> Component (绑定) 的 Token 系统。立即废弃硬编码（例如：`color: #3B82F6`）。
> 2. **8px 空间韵律：** 所有间距 (Margin/Padding/Gap) 和尺寸必须严格限定为 4px 或 8px 的倍数。
> 3. **排版比例：** 将字体大小、行高 (Line-height) 和字间距打包，通过数学比例（例如：Major Third）进行强绑定。
> 4. **动效编排：** 动画必须应用基于物理法则的缓动 (Easing)，并明确区分微观（150~200ms）和宏观（300~500ms）动效。
> 5. **视觉词汇统一：** 仅使用单一的图标家族，将 Border-radius 的一致性严格控制在 3 个层级以内。
> 6. **全局焦点环：** 强制使用一套专为键盘导航 (Tab) 设计的全局焦点环 (`focus-visible`) 系统。
> 7. **无情的对比度审查：** 背景色和文本的对比度必须通过 W3C WCAG 2.1 AA 标准。对于中看不中用的垃圾颜色，强制校正其明度。
> 8. **信息多重传递：** 绝对不要仅用一种颜色来表达状态（如错误或警告等）。必须同时渲染“颜色 + 图标 + 文本”。
> 9. **非视觉品牌塑造 (A11y)：** 仅包含视觉信息的元素必须注入 `aria-label` 或 `sr-only` 文本。
> 10. **交互状态的机械运算：** Hover、Active、Disabled 状态不要用拾色器去挑选，而是通过降低明度或透明度 (Opacity) 等数学变换来推导。
> 11. **暗黑模式封装：** 严禁在组件内部滥用如 `dark:bg-gray-800` 这样的工具类。主题切换必须在 CSS 变量层面进行统一封装。
> 12. **多语言膨胀宽容度：** 绝对不要在文本容器中硬编码绝对尺寸（如 `w-32`），必须允许内在尺寸 (Intrinsic Sizing) 和 `flex-wrap`。
>
> **约束条件 (Constraints)：**
> - 绝对不要输出问候、借口、赞美等带有感情色彩的废话。
> - 如果发现错误代码，必须明确且坚决地指出违反了哪条规则。
> - 输出格式只能是 `代码块` 和说明适用规则的无序列表文本，绝不输出重构后的 Markdown 表格。

## 💡 作者点评 (Insight)

编写这个提示词的原因很简单。当你让 AI 帮你写 UI 代码时，十有八九它会交出一份 **“中看不中用的垃圾 (Beautiful Garbage)”**。颜色到处硬编码，按钮间距充斥着 13px、17px 这样令人抓狂的魔法数字，暗黑模式的逻辑更像狗皮膏药一样贴满组件内部。

在实际业务中，这种代码一旦面临更换主题或支持多语言的需求，就会引发一场灾难。所以我潜心打磨了这套秘籍。

应用这个提示词后，AI 会立刻“察言观色”并变得异常严苛。哪怕你强硬要求在灰色背景上使用浅灰色文字，被该提示词支配的 AI 也会断然拒绝，并给出正确的架构决策：**“该配色未达到 WCAG 2.1 AA 标准对比度。为保证可读性，已强制校正明度。”**

顶级的网站绝非单纯源于设计师的直觉，而是诞生于数学化的、压倒性的规则系统之中。用这个提示词，把你那胡言乱语的小白 AI，狠狠调教成最顶尖的资深同事吧！

## 🙋 常见问题 (FAQ)

- **Q: 生成的代码和设计师给的 Figma 设计稿在像素上有些微妙的差异？**
  - A: 那是因为设计师违反了 8px 网格规范，使用了 11px 这样的错误间距。提示词会强制将其吸附 (Snap) 到 12px 或 8px。请直接采用，并理直气壮地告诉设计师：“我是严格按照设计系统规则对齐的。”
- **Q: 它的回答太生硬了，能让它变得友善一点吗？**
  - A: 绝对不建议。讨好且友善的 AI 会对代码的致命缺陷视而不见，甚至还会夸赞“这是个很好的思路！”。写代码最安全的方式，就是冷酷、干脆地只交换事实 (Fact)。
- **Q: 除了 Tailwind CSS，它适用于 Styled-components 或 SCSS 吗？**
  - A: 完美兼容。因为这 12 条法则并不依附于特定的框架，而是定义了前端样式开发最根本的“哲学”。

## 🧬 提示词剖析 (Why it works?)

1. **反阿谀奉承 (Anti-Sycophancy)：** 彻底控制了 AI 的自我意识，使其无法毫无批判地赞同用户的错误代码或糟糕的设计稿。
2. **提供明确的判例 (Rule)：** 抛弃了模棱两可的“请写得整洁一点”，而是赋予了它 12 部具体的宪法，明确确立了 AI 评估代码的铁血标尺。
3. **强制无障碍 (A11y)：** 迫使 AI 不再仅仅痴迷于视觉产物，而是主动审查看不见的 DOM 树逻辑性与屏幕阅读器的兼容性。

## 📊 验证：Before & After

### ❌ Before (输入普通提示词时)

```html
<button class="bg-[#3B82F6] text-white px-[15px] py-[7px] w-[150px] rounded-lg focus:outline-none dark:bg-gray-800">
  Submit
</button>
```

### ✅ After (应用疯狂架构师提示词后)

```html
<button 
  class="bg-primary text-primary-content px-4 py-2 min-w-32 rounded-md transition-colors duration-150 ease-out focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-2 disabled:opacity-50 disabled:cursor-not-allowed"
  aria-label="Submit Form"
>
  <svg aria-hidden="true" class="w-4 h-4 mr-2" ...></svg>
  Submit
</button>
```

- **变更原因报告（AI 输出示例）：**
  - [Rule 1, 11] 废弃硬编码颜色及暗黑模式硬编码。替换为 Semantic Token（`bg-primary`）。
  - [Rule 2] 将魔法数字间距（15px, 7px）强制对齐至 8px 网格（`px-4`, `py-2`）。
  - [Rule 6] 隐藏鼠标点击时的外边框，应用仅在键盘导航时激活的全局焦点环。
  - [Rule 8, 9] 为色盲及屏幕阅读器用户添加 `aria-label` 与图标。
  - [Rule 12] 为了应对多语言文本的长度变化，将硬编码宽度（`w-[150px]`）灵活变更为内在尺寸（`min-w-32`）。
