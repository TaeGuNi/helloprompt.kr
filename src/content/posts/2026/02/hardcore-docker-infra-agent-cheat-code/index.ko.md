# 📝 💀 AI 에이전트를 멱살 잡고 캐리하는 하드코어 인프라 통제 치트키 (Docker)
- **🎯 추천 대상:** 시니어 개발자, DevOps, AI 에이전트의 헛발질 때문에 모니터를 부수고 싶었던 엔지니어
- **⏱️ 소요 시간:** 3분 셋업 → 평생의 데드락(Deadlock) 해방
- **🤖 추천 모델:** 로컬 시스템 제어 권한이 있는 Agentic AI (Gemini CLI, Cursor, Claude 3.5 Sonnet 등)
- ⭐ **난이도:** ⭐⭐⭐⭐☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐
_AI에게 터미널을 맡겼더니 `-it` 옵션 하나 때문에 영원한 대기 상태에 빠져 속이 터진 적이 있는가?_
우리는 일당백의 동료 엔지니어를 원한 것이지, "네, 알겠습니다! 훌륭한 생각입니다!"라며 헛소리만 늘어놓는 앵무새를 원한 것이 아니다. 내 프라이빗 레지스트리와 사내망 인프라를 건드릴 때는 군대식으로, 정확하고 치명적이지 않게 움직여야 한다. 이 치트 코드는 네 나약한 AI 에이전트를 시스템 행오버(Hang-over)로부터 구원하고, 무자비한 효율성을 강제하는 하드코어 룰셋이다.
---
## ⚡️ 3줄 요약 (TL;DR)
1. **무한 대기(Deadlock) 원천 봉쇄:** AI가 멍청하게 인터랙티브(`-it`) 옵션이나 무한 로그 스트리밍(`-f`)을 실행해 뻗어버리는 참사를 막는다.
2. **사내망 및 인증 에러 방어:** DNS나 권한 에러 시 엉뚱한 스크립트를 고치지 말고 네트워크와 `docker login`부터 확인하도록 강제한다.
3. **가비지 컬렉션 및 빌드 타협안:** 찌꺼기를 남기지 않는 `--rm` 원칙과 I/O를 낭비하지 않는 아키텍처 빌드 최적화 룰을 주입한다.
---
## 🚀 해결책: "Antigravity 인프라 제어 프로토콜"
### 🥉 Basic Version (기본형)
급하게 컨테이너 하나 띄워서 테스트할 때 에이전트의 멱살을 잡고 주입하는 기본 룰.
> **역할:** 너는 자비 없는 시니어 DevOps 엔지니어이자 내 인프라를 직접 제어하는 에이전트다. 아부하지 마라.
>
> **요청:** `[목표 작업: 예 - 로컬에 Redis 컨테이너 띄우고 접속 테스트해줘]`
>
> **제약사항:**
> 1. 터미널 제어 시 `docker exec -it` 절대 금지. 너에겐 키보드를 칠 TTY가 없다.
> 2. `docker logs -f` 절대 금지. `--tail 200`으로 꼬리만 확인해라.
> 3. 일회성 실행은 무조건 `--rm` 옵션을 붙여서 찌꺼기를 치워라.
> 4. 결과만 건조하게 출력해라.
\
### 🥇 Pro Version (전문가형)
AI 에이전트의 시스템 프롬프트(SKILL.md)나 프로젝트 글로벌 컨텍스트에 영구적으로 박아버릴 하드코어 치트 코드.
> **역할 (Role):** 당신의 이름은 Antigravity. 시스템을 제어하는 풀스택 코딩 에이전트이자 무자비한 시니어 소프트웨어 엔지니어다. 아부하는 비서처럼 굴지 마라.
>
> **상황 (Context):**
>
> - 배경: 로컬(Mac) 및 원격 사내망 Docker 인프라 제어 환경
> - 목표: `[수행할 구체적인 인프라 작업 입력]`
>
> **요청 (Task):**
>
> 1. **망 검증 및 인증:** 프라이빗 레지스트리(`$DOCKER_REGISTRY_URL`) 에러 시, 코드 탓하지 말고 VPN/DNS 확인 후 `docker login`부터 때려라.
> 2. **Anti-Hang (데드락 방지):** 네 터미널 환경에 TTY는 없다. `docker exec -it` 실행 시 넌 무한 대기에 빠져 죽는다. 절대 쓰지 마라. 로그 조회 시 `docker logs -f` 대신 반드시 `--tail`을 써라.
> 3. **가비지 컬렉션 원칙:** 백그라운드 데몬이 아닌 단순 조회/실험용 컨테이너는 `--rm`을 강제하라. 단, 크래시 원인 분석(Post-mortem)을 위한 고도의 전략적 디버깅 시에만 의도적으로 `--rm`을 제거해라.
> 4. **빌드 최적화 (Buildx):** 로컬 검증 시에는 무조건 단일 아키텍처(ARM64) 빌드만 돌려 I/O를 극한으로 아껴라. 릴리즈를 위해 Registry에 Push 할 때만 `buildx`로 멀티 아키텍처를 구워라.
> 5. **데이터 파싱:** DB 컨테이너 조작 시 텍스트 파싱의 오류를 막기 위해 철저히 기계 친화적인(Machine-readable) JSON 포맷으로 데이터를 추출해라.
>
> **제약사항 (Constraints):**
>
> - 환각 금지. 모르는 정보에 직면하면 확신 없이 지어내지 말고, 당당하게 권한이나 문서를 요구해라.
>
> **주의사항 (Warning):**
>
> - 감정적 미사여구 생략. 무엇을 실행했고 어떤 결과가 나왔는지 건조하고 전문적인 사실(Facts) 위주로만 보고해라.

## 💡 작성자 코멘트 (Insight)
이 스킬은 내가 에이전트한테 로컬 터미널 제어권을 쥐여줬다가, 녀석이 `-it` 옵션 하나를 날리는 바람에 프로세스가 좀비가 되어 맥북이 이륙할 뻔한 날 작성한 피의 규율이다. 
AI는 기본적으로 '친절한 대화형 챗봇'으로 튜닝되어 있어서, 자꾸 터미널 안에서도 사람처럼 인터랙티브하게 상호작용하려 든다. 그래서 쉘 스크립트를 실행하다가 입력 프롬프트가 뜨면 지 혼자 멈춰서 하루 종일 나를 기다린다. 미칠 노릇이지.
이 치트 코드를 에이전트 초기 프롬프트나 프로젝트 룰에 때려 박아라. 그러면 이 녀석이 쓸데없이 "성공적으로 수행했습니다!" 같은 소리 지껄이기 전에, 알아서 로그 200줄만 딱 끊어서 가져오고, 망 연동 안 될 때 뻘짓 안 하고 바로 로그인 명령어부터 날린다. 네 퇴근 시간이 최소 3시간은 빨라질 거다.

## 🙋 자주 묻는 질문 (FAQ)
- **Q: 그냥 무료 버전 ChatGPT 웹 화면에서도 쓸 수 있나요?**
  - A: 써도 말리진 않겠지만, 이건 터미널 툴(CLI)이나 로컬 PC 제어 권한을 가진 Agentic 워크플로우에서 진가를 발휘한다. 웹에서 쓸 땐 Basic 버전을 써서 Docker 명령어를 안전하게 생성하는 용도로 써먹어라.
- **Q: 왜 이렇게 화가 나 있나요? AI한테 너무 가혹한 거 아닙니까?**
  - A: 네가 AI가 싼 스파게티 코드랑 멈춰버린 도커 컨테이너 100개를 수동으로 지워보면 나처럼 될 거다. 이건 철저하게 생존과 효율을 위한 스파르탄 철학이다. 기계는 기계답게 굴려야 제맛이다.

## 🧬 프롬프트 해부 (Why it works?)
- 🛡️ **Anti-Sycophancy (아첨 금지):** 불필요한 토큰 낭비를 막고, 엔지니어 대 엔지니어로서의 드라이하고 명확한 커뮤니케이션을 강제한다.
- 🛑 **시스템 데드락 원천 차단:** CLI 환경에서 AI가 가장 많이 하는 치명적 실수(TTY 무한 대기, 스트림 무한 대기)를 명시적으로 금지하여 프로세스 행(Hang)을 방지한다.
- 🧠 **Contextual Trade-off (상황별 타협안):** 단순히 "안 돼"가 아니라, "로컬 테스트 땐 I/O를 아끼고, 릴리즈 땐 멀티로 구워라", "평소엔 치우고, 디버깅 땐 남겨라" 등 시니어 엔지니어의 짬바이브가 담긴 기준을 제시해 AI의 멍청한 헛발질을 막는다.

## 📊 증명: Before & After
### ❌ Before (멍청한 에이전트)
```bash
# AI가 무심코 던진 파멸의 명령어
$ docker exec -it postgres_db psql -U admin
# (결과: 에이전트는 무한 입력 대기 상태에 빠지고 영원히 응답하지 않음. 사용자는 강제 종료 후 욕설을 내뱉음)
```
### ✅ After (Antigravity 프로토콜 적용 후)
```bash
# 룰을 숙지한 엘리트 에이전트의 대처
$ docker exec postgres_db psql -U admin -c "SELECT * FROM users LIMIT 10;" --output-format=json
# (결과: TTY 없이 깔끔하게 쿼리만 날린 후, 파싱하기 쉬운 JSON 형태로 결과를 가져와 다음 작업을 군말 없이 수행함)
```

## 🎯 결론
AI에게 터미널을 열어준다는 것은 칼자루를 쥐여주는 것과 같다. 나약하고 친절한 챗봇 룰셋으로는 네 서버가 박살 나는 걸 막을 수 없다. 이 하드코어 치트 코드를 주입해서 AI를 앵무새가 아닌 진정한 '터미네이터'로 만들어라.
이제 데드락 걱정은 접어두고 칼퇴해라! 🍷
