## ⚡️ 3 行总结 (TL;DR)

1. 分析错误信息和相关代码识别问题原因 (Root Cause)
2. 与修改后的代码 (Fixed Code) 一起说明为什么要改
3. 提议防止再发的防御代码 (Defensive Code)

## 📊 证明: Before & After

### ❌ Before (情况)

JavaScript 中发生 `Uncaught TypeError: Cannot read properties of undefined (reading 'map')`。
(初学者: "Map? 地图为什么出来？")

### ✅ After (提示词结果)

**[原因分析]**
因为在从 API 接收数据前试图绘制画面而发生的。名为 `userList` 的变量还没进数据是 `undefined` 状态，却在那里用了 `.map()` 函数所以出错了。是异步处理问题呢。

**[解决方案]**

```text
// 修改前: userList.map(...)
// 修改后: 使用可选链 (?.)
userList?.map(user => ...)
```

贴了 `?.` 运算符以便只在有数据时运行 map。

**[预防策]**
写 API 通信代码时，最好总是假设‘数据可能来晚’或‘可能失败’，先放入像 `if (!data) return loading...` 这样的代码。
