Работа с легаси-кодом — одна из главных причин профессионального выгорания среди инженеров. Монолитные архитектуры, спагетти-код, безнадежно устаревшие фреймворки и полное отсутствие документации превращают внедрение даже самой тривиальной фичи в настоящий кошмар. К счастью, сегодня в нашем арсенале появились интеллектуальные инструменты, способные за считанные секунды проанализировать сотни строк кода, разложить по полочкам скрытую бизнес-логику и предложить абсолютно безопасный сценарий рефакторинга.

## ⚡️ 3 главных преимущества (TL;DR)

1. **Мгновенное погружение в контекст:** ИИ переводит запутанный и нечитаемый синтаксис на понятный человеческий язык.
2. **Безопасный рефакторинг:** Автоматическая генерация юнит-тестов перед внесением любых, даже самых незначительных, изменений.
3. **Автоматизированная миграция:** Бесшовный переход с устаревших библиотек на современные стандарты с полным сохранением исходного функционала.

Используйте этот базовый вариант, когда вам нужно молниеносно разобраться в логике конкретной функции, написанной давно уволившимся коллегой.

> **Запрос (Task):** Объясни мне, что делает этот код на `[Язык программирования]`, шаг за шагом. Укажи на явные антипаттерны и уязвимости:
> 
> `[Ваш исходный код]`

### 🥇 Продвинутая версия (Pro Version)

Применяйте этот промпт для проведения глубокого рефакторинга, подготовки к масштабной миграции на новый фреймворк или для написания тестов к критически важным, но крайне хрупким участкам кода.

> **Роль (Role):** Ты — Principal Staff Engineer и эксперт по модернизации унаследованных систем. Твоя задача — проанализировать устаревший код, объяснить его бизнес-логику и подготовить план безопасного рефакторинга.
>
> **Контекст (Context):**
>
> - Текущее состояние: Мы поддерживаем старую систему на `[Устаревшая технология/язык]`. Код не задокументирован и покрыт тестами лишь на `[Процент покрытия]%`.
> - Цель: Необходимо понять этот модуль и переписать его на `[Новая технология/язык]`, сохранив 100% обратной совместимости.
>
> **Задачи (Task):**
>
> 1. Проведи аудит предоставленного кода и напиши краткое резюме его бизнес-логики (что он делает с точки зрения конечного пользователя).
> 2. Выяви скрытые зависимости (side effects) и потенциальные узкие места в производительности.
> 3. Напиши 3-5 юнит-тестов на `[Фреймворк для тестирования]`, которые зафиксируют теку поведение кода (Black-box testing), чтобы мы могли безопасно начать рефакторинг.
> 4. Предложи современную, чистую версию этого кода на `[Целевой язык/фреймворк]`, используя паттерны проектирования SOLID.
>
> **Ограничения (Constraints):**
>
> - Выводи ответ строго в формате Markdown.
> - Код тестов и код рефакторинга должны находиться в отдельных блоках кода.
> - Не изменяй исходную логику работы системы без явного предупреждения, даже если она кажется тебе неоптимальной или странной.

## 💡 Комментарий автора (Insight)

Самая фатальная ошибка при работе с легаси — попытка переписать всё с нуля за один присест (так называемый Big Bang Rewrite). Да, ИИ без труда сгенерирует для вас красивый и современный код, однако он неизбежно упустит неочевидные бизнес-правила и корнер-кейсы, которые годами наслаивались в старой системе в виде причудливых «костылей».

Именно поэтому в Pro-версии мы первым делом заставляем ИИ **покрыть исходный код тестами**. Такой подход (Test-Driven Refactoring) гарантирует, что вы надежно зафиксируете все скрытые нюансы поведения системы. Лишь после того, как легаси-код защищен непробиваемой броней тестов, можно смело интегрировать сгенерированное современное решение. Современные языковые модели с огромным окном контекста справляются с этой задачей поистине феноменально.

## 🙋 Часто задаваемые вопросы (FAQ)

- **Q: Насколько безопасно отправлять проприетарный код компании в ИИ?**
  - A: Неукоснительно соблюдайте политику информационной безопасности вашей компании! Никогда не используйте публичные веб-версии чат-ботов для работы с закрытым коммерческим кодом. Обращайтесь к моделям исключительно через официальный API (где ваши данные не используются для дообучения) или разворачивайте локальные ИИ-решения, если проект строго защищен NDA.
- **Q: Может ли ИИ случайно сломать бизнес-логику в процессе рефакторинга?**
  - A: Да, риск галлюцинаций есть всегда. ИИ вполне может попытаться «оптимизировать» специфический баг, ошибочно приняв его за неэффективный кусок кода. Поэтому здесь действует железное правило: **никогда не отправляйте сгенерированный код в продакшен без тщательного код-ревью и успешного прохождения всех пайплайнов тестирования.**

## 🧬 Разбор промпта (Why it works?)

1. **Фокус на строгой обратной совместимости:** Мы категорически запрещаем нейросети изменять текущее поведение приложения (даже если алгоритм кажется ей откровенно ошибочным). Это критически важное условие для поддержания стабильности работающих легаси-продуктов.
2. **Поиск побочных эффектов (Side-effects):** Жесткое требование выявить скрытые зависимости заставляет модель проводить по-настоящему глубокий статический анализ, а не ограничиваться поверхностным переводом синтаксиса с одного языка на другой.
3. **Генерация тестов ПЕРЕД началом рефакторинга:** Внедрение эталонных инженерных практик непосредственно в структуру самого промпта служит надежной страховкой от катастрофических инцидентов на продакшене.

## 📊 До и После: Доказательство эффективности

### ❌ До (Исходный «спагетти-код»)

```javascript
// Старый скрипт расчета скидки, никто в команде не знает как он работает
function get_d(t, u, c) {
  var x = 0;
  if (t == 1) {
    if (u > 5) {
      x = 10;
    } else {
      x = 5;
    }
  } else if (t == 2) {
    x = 15;
    if (c) x = x + 5;
  }
  return x;
}
```

### ✅ После (Элегантный результат от ИИ)

```javascript
/**
 * Рассчитывает процент скидки на основе типа клиента и его истории.
 * @param {number} customerType - Тип клиента (1 = regular, 2 = vip).
 * @param {number} yearsActive - Количество лет работы с компанией.
 * @param {boolean} hasCoupon - Наличие специального купона.
 * @returns {number} Процент скидки.
 */
function calculateDiscount(customerType, yearsActive, hasCoupon) {
  const CUSTOMER_REGULAR = 1;
  const CUSTOMER_VIP = 2;

  if (customerType === CUSTOMER_REGULAR) {
    return yearsActive > 5 ? 10 : 5;
  }

  if (customerType === CUSTOMER_VIP) {
    const baseVipDiscount = 15;
    const couponBonus = hasCoupon ? 5 : 0;
    return baseVipDiscount + couponBonus;
  }

  return 0; // Значение по умолчанию
}

// + ИИ дополнительно генерирует набор юнит-тестов (Jest) для покрытия всех ветвлений.
```

Использование профессионально составленных промптов мгновенно превращает ИИ в неутомимого напарника, который готов часами копаться в чужом «спагетти-коде», бережно восстанавливая утраченную бизнес-логику и заботливо расставляя спасительные тесты.
