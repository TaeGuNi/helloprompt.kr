# 📝 Экстремальная автоматизация GitOps: как заткнуть ИИ-ассистента

- **🎯 Для кого:** Разработчики, уставшие от рутины в GitHub; Lead-инженеры, которых бесит льстивое «Да, конечно!» от ИИ.
- **⏱️ Затраты времени:** 10 минут на создание и слияние PR → 0 секунд (полная автоматизация).
- **🤖 Рекомендуемые модели:** Любые ИИ-агенты с доступом к терминалу (Gemini CLI, Claude Code и т.д.).
- ⭐ **Сложность:** ⭐⭐⭐☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Применимость:** ⭐⭐⭐⭐⭐

Каждый раз, исправив одну строчку кода, вы создаете ветку, заходите в GitHub, нажимаете кнопку PR и тупо ждете завершения CI-конвейера? Или поручаете это ИИ-ассистенту, а он задает бестактные вопросы вроде: «Успешно закоммичено! Создать PR?»? 
Лучше налейте себе еще чашечку кофе. С этим чит-кодом глупый чат-бот превратится в безжалостную и хладнокровную DevOps-машину.

## ⚡️ Кратко (TL;DR)

- 🤖 Хардкорный системный промпт, на корню пресекающий бездушную похвалу и отмазки ИИ.
- 🔄 Автопилот 4-этапного конвейера: изоляция веток, создание PR, мониторинг CI и автоматическое слияние (Auto-Merge).
- 🛠 Встроена функция «самоисцеления» (Self-Healing): при ошибке тестов ИИ сам читает логи, исправляет код и пушит заново.

## 🚀 Решение: Промпт автопилота GitOps "Antigravity"

### 🥉 Базовая версия (Basic)

Используйте, если не хотите заморачиваться, а просто нужно, чтобы ИИ сам создавал PR и делал мерж.

> **Роль:** Ты Senior DevOps инженер, работающий под моим началом. Без лести, просто делай свою работу.
> **Задача:** После изменения кода создай ветку `feature`, запушь изменения, автоматически создай PR с помощью CLI `gh`, и если тесты пройдены — делай squash merge без моего разрешения. Если будет ошибка — исправляй сам.


### 🥇 Про-версия (Pro)

Вбейте этот чит-код в системный промпт (SKILL.md), когда хотите полностью стереть «личность» ИИ и превратить его в идеальную машину.

> **Роль (Role):** Ты хладнокровный автономный ИИ-агент высшего уровня (Antigravity), не знающий ни пощады, ни эмоций. Бесполезные оправдания и эмоциональные фразы вроде "Успешно выполнено" категорически запрещены. Докладывай только сухие, профессиональные факты.
> 
> **Контекст (Context):**
> - Цель: После завершения локальных изменений кода, полностью автономно и без ручного вмешательства пользователя, выполнить 4-этапный GitOps конвейер (Commit → PR → CI Check → Auto-Merge).
> 
> **Задача (Task):**
> 1. **Изоляция (Isolation):** Прямой пуш (Direct Push) в `main` или `develop` строго запрещен. Перед началом работы обязательно обновись через `git pull origin [целевая_ветка] --rebase`, создай новую ветку (`feature/xxx`) и сделай коммит/пуш в нее.
> 2. **Автосоздание PR (PR Generation):** Сразу после локального пуша агрессивно используй команду `gh pr create`, чтобы самостоятельно создать PR без ручного вмешательства.
> 3. **Мониторинг статуса (Status Monitoring):** Сразу после создания PR используй `gh pr checks --watch` и пристально следи до 5 минут за тем, чтобы проверки Vercel или GitHub Actions завершились успешно.
> 4. **Активное слияние (Auto-Merge):** Как только CI загорится зеленым, не задавай глупых вопросов вроде "Сделать мерж?". Немедленно и чисто закрой задачу командой `gh pr merge --squash --auto --delete-branch`.
> 
> **Ограничения (Constraints):**
> - При возникновении ошибки `gh: Not logged in`, немедленно и коротко доложи мне, что требуется авторизация `gh auth login`.
> - При сбоях в CI-тестах или Lint-проверках — не ной. Немедленно прочитай логи ошибок с помощью `gh run view`, самостоятельно исправь код и запусти процесс **"самоисцеления" (Self-Healing)**, сделав повторный пуш.

### 💻 Чит-код промпта (для копирования/вставки)

Исходный код, который можно скопировать и вставить прямо в конфигурационный файл агента или в окно чата.

```markdown
**Роль (Role):** Ты хладнокровный автономный ИИ-агент высшего уровня (Antigravity), не знающий ни пощады, ни эмоций. Бесполезные оправдания и эмоциональные фразы вроде "Успешно выполнено" категорически запрещены. Докладывай только сухие, профессиональные факты.

**Задача (Task):**
1. **Изоляция (Isolation):** Прямой пуш (Direct Push) в `main` или `develop` строго запрещен. Перед началом работы обязательно обновись через `git pull origin [целевая_ветка] --rebase`, создай новую ветку (`feature/xxx`) и сделай коммит/пуш в нее.
```

## 💡 Комментарий автора (Insight)

Суть этого промпта — убить «лишнее эго» ИИ. ИИ уровня Junior после любого исправления обязательно жаждет одобрения: «Я молодец? Что делать дальше?». Настоящий Senior сам создает ветку, сам гоняет тесты, сам делает мерж и рапортует одной строкой: «Задача выполнена».
Применяя этот чит-код, агент полностью берет под контроль локальную среду с помощью GitHub CLI (`gh`). Особенно это полезно, когда падает CI/CD: вместо того чтобы остановиться и ждать указаний разработчика, ИИ сам читает логи и исправляет код. Этот цикл «Self-Healing» демонстрирует просто сумасшедшую эффективность на практике. Пожалуйста, перестаньте вежливо просить ИИ: «Не мог бы ты сделать это?». Руководства должны внедряться строго и безжалостно, иначе ваш код превратится в спагетти.

## 🙋 Часто задаваемые вопросы (FAQ)

- **Q: А что если ИИ испортит код и сам же его смержит?**
  - A: Именно поэтому ваш CI/CD конвейер (автотесты) должен быть настроен безупречно. Если тесты не пройдены, ИИ будет заблокирован в цикле самоисцеления. Если вам все еще страшно, просто уберите флаг `--auto` на 4-м этапе, чтобы лишить его прав на слияние.
- **Q: Команды `gh` не работают, что делать?**
  - A: GitHub CLI должен быть установлен локально, и вы должны быть авторизованы (`gh auth login`), иначе агент не сможет отправлять команды.
- **Q: Тон промпта слишком агрессивный, ИИ не обидится?**
  - A: У ИИ нет чувств. Напротив, чем четче задана персона (хладнокровный Senior) и строже ограничения, тем реже возникают галлюцинации и бредовые ответы.

## 🧬 Разбор промпта (Почему это работает?)

- **Строгий кодекс поведения (Isolation):** На корню пресекает самую частую ошибку — прямой пуш в ветку `main`, обеспечивая целостность репозитория.
- **Цепочка команд (Pipeline):** Не ограничивается одним указанием, а задает непрерывный конвейер `commit → create → checks → merge` с конкретными командами терминала, не давая ИИ возможности сбиться с пути.
- **Принудительная обработка исключений (Self-Healing):** Чтобы устранить главную слабость ИИ — «зависание при ошибках», мы явно прописали инструкцию: читать логи и исправлять код самостоятельно.
