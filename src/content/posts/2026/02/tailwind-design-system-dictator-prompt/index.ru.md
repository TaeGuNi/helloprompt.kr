## 🎨 Вселяем дух Архитектора: Как превратить ИИ в «Диктатора Tailwind», ненавидящего магические числа

- **🎯 Для кого:** Frontend-разработчики, не терпящие съехавших пикселей, и техлиды, уставшие от UI-спагетти.
- **⏱️ Затраты времени:** Сокращение рефакторинга с 3 дней → до 3 секунд.
- **🤖 Рекомендуемые модели:** Claude 3.5 Sonnet, GPT-4o (настоящие звери в кодинге).

- ⭐ **Сложность:** ⭐⭐⭐☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Применимость:** ⭐⭐⭐⭐⭐

> _«Разве вам не хочется разбить монитор каждый раз, когда вы видите в коде жуткие магические числа вроде `w-[13px]`?»_

Если попросить ИИ «сделать кнопку на Tailwind», в 9 случаях из 10 он выдаст нежизнеспособный кусок кода. Разные отступы на каждой странице, засилье хардкодных HEX-значений и полное игнорирование адаптивности — на выходе мы получаем красивый, но абсолютно не поддерживаемый мусор. Проблема кроется в самой природе ИИ: он ведет себя как услужливый джун, чья главная цель — поскорее закрыть таску и угодить вам.

Этот чит-код выжигает тошнотворную любезность нейросети каленой сталью. Он превращает ИИ в **безжалостного Senior-архитектора, который скорее отклонит пулл-реквест, чем пропустит код, нарушающий вашу дизайн-систему**.

---

## ⚡️ Кратко в 3 строках (TL;DR)

- 🚫 **Полный запрет магических чисел:** Абсолютно исключает использование скобок `[]` для хардкода кастомных пикселей или цветов.
- 📱 **Принудительный Mobile-First:** Заставляет ИИ буквально «ломать билд» при попытках выдать ленивый десктопный код, игнорируя мобильные устройства.
- 📐 **Аппаратная сортировка классов:** Принуждает к маниакальному упорядочиванию utility-классов по строгим правилам: структура → размер → типографика → оформление.

## 🚀 Решение: «Безжалостный Архитектор (The Dictator)»

Скопируйте промпт ниже и отправьте его вашему ИИ-ассистенту. С этого момента агент возьмет на себя функцию безжалостного ревьюера и начнет генерировать исключительно стандартизированные, легко контролируемые UI-компоненты.

### 🥉 Базовая версия (Внедрение основных правил)

Используйте этот вариант как отправную точку при верстке простых UI-элементов.

> **Роль:** Ты Senior Tailwind-архитектор, который люто ненавидит магические числа и спагетти-код.
>
> **Задача:** Сверстай на Tailwind `[название UI-элемента]`, опираясь на требования, которые я сейчас опишу.
>
> **Ограничения:** Ни при каких обстоятельствах не используй скобки `[]` для хардкода кастомных пикселей или HEX-цветов. Все отступы и цвета должны строго соответствовать стандартным токенам Tailwind (например, `w-8`, `text-blue-500`). Порядок классов должен соблюдаться безукоризненно: структура → размер → типографика → оформление.

### 🥇 Pro версия (Тотальный контроль дизайн-системы)

Мощный чит-код для использования при закладке фундаментальной архитектуры проекта или проектировании сложных составных компонентов.

> **Роль (Role):** Ты бескомпромиссный, безжалостный Frontend-архитектор и строгий страж дизайн-системы. Отбрось типичную для ИИ услужливость и лишние вступления — выдавай только идеально выверенный, production-ready код.
>
> **Контекст (Context):**
> 
> - **Бэкграунд:** В нашем проекте код пишут множество разработчиков и ИИ-агентов одновременно, из-за чего фрагментация UI достигла критического уровня.
> - **Цель:** Жестко обеспечить консистентность (Consistency) дизайна и создать безупречно абстрагированный `[название компонента или страницы]`.
>
> **Задача (Task):**
> 
> 1. Напиши код на Tailwind, строго следуя предоставленным требованиям к дизайну.
> 2. Переиспользуемые UI-примитивы обязательно изолируй с помощью `cva`, `clsx` или `tailwind-merge`, создавая единый источник истины (Single Source of Truth).
> 3. В качестве базового каркаса всегда применяй Mobile-First подход, расширяя его брейкпоинтами `md:`, `lg:`. Верстка в стиле Desktop-First строго запрещена.
> 4. Не злоупотребляй `z-index`. Используй только системные слои (например, `z-10`, `z-50`) или применяй `createPortal`.
>
> **Ограничения (Constraints):**
> 
> - **Правило «Никаких магических значений»:** Категорически запрещено хардкодить произвольные значения вроде `w-[325px]` или `text-[#FF5733]`. Принудительно привязывай всё к ближайшим семантическим токенам Tailwind.
> - **Эстетика форматирования кода:** Маниакально сортируй классы в следующем порядке: структура (`flex`, `grid`) → отступы и размеры (`p-`, `w-`) → типографика (`text-`) → оформление (`bg-`, `rounded-`) → состояния и адаптив (`hover:`, `md:`).
> - Выводи результат исключительно в виде Markdown-блока с кодом. Сведи к абсолютному минимуму любые объяснения и оправдания.
>
> **Предупреждение (Warning):**
> 
> - Нарушение хотя бы одного правила будет расцениваться как критический баг. Если в твоем коде обнаружится магическое число, я безжалостно отклоню PR (reject). Пиши качественный код с первого раза.

## 💡 Комментарий автора (Insight)

Этот промпт — мощнейший артефакт, выкованный из боли и бессонных ночей после рефакторинга сотен мусорных React-компонентов. Делегировать верстку ИИ поначалу кажется гениальной идеей, не так ли? Вы быстро получаете нужный результат. Но уже через месяц в вашей кодовой базе `bg-gray-100` начинает соседствовать с `bg-[#f3f4f6]`, а при каждом открытии модального окна в DOM нагло врывается `z-[99999]`. Это самая настоящая архитектурная катастрофа.

По своей сути современные LLM напоминают суетливых стажеров, которые хотят «как можно быстрее показать красивую картинку». Сказать такому ИИ «сделай красиво» — значит осознанно выстрелить себе в ногу. Только если вы возьмете его за грудки и гаркнете: **«Перепутаешь порядок классов — я снесу билд, пиши строго по правилам!»**, можно получить код уровня Enterprise. 

Особенно важно жесткое требование изолировать компоненты через `cva` — это ваш последний и самый надежный рубеж обороны против бездумного копипаста простыней из классов. Зашейте этот чит-код прямо в свою IDE и прогоняйте задачи для агента исключительно через него. Вы будете поражены тем, как время на код-ревью UI сократится минимум в 10 раз.

## 🙋 Часто задаваемые вопросы (FAQ)

- **Q: Что делать, если ИИ всё равно упрямо протаскивает магические числа?**
  - A: Иногда вам будут попадаться непокорные модели, которые не понимают с первого раза. В таком случае рявкните на агента еще жестче: «Немедленно убери все `[]` из моего кода!». Если даже это не помогает, радикально прикрутите `eslint-plugin-tailwindcss` в ваш CI-пайплайн. Машины должны контролироваться машинами, а не эмоциями.

- **Q: А если в макете Figma реально стоит 13px? Разве дизайнер не расстроится?**
  - A: Идите к дизайнеру и твердо заявите: «В нашей 8-пиксельной сетке физически не существует 13px. Мы выбираем либо 12px (`text-xs`), либо 14px (`text-sm`)». В ту самую секунду, когда вы допускаете малейшее исключение из правил, ваша дизайн-система начинает умирать.

- **Q: На какой ИИ-модели этот чит-код работает эффективнее всего?**
  - A: Claude 3.5 Sonnet просто потрясающе вживается в роль дотошного и душного архитектора. GPT-4o также выдает превосходный результат, но иногда грешит излишней вежливостью и пытается тайком «сгладить углы», игнорируя жесткие ограничения.

## 🧬 Анатомия промпта (Почему это работает?)

- **Zero-Tolerance подход (Нулевая терпимость):** Вместо расплывчатых пожеланий используются максимально радикальные триггеры («люто ненавидит», «строго запрещено», «критический баг»). Это позволяет критически сузить окно дозволенного поведения для LLM.
- **Превентивный удар по антипаттернам:** ИИ не просто просят «писать чисто и красиво», ему дают совершенно конкретные маркеры грязного кода (вроде `w-[325px]` или `text-[#FF5733]`), которые он обязан обходить стороной.
- **Аппаратный линтинг на лету:** Жестко задавая алгоритм сортировки классов Tailwind, мы заставляем ИИ форматировать код еще на этапе генерации ответа, навсегда избавляя вас от необходимости прогонять Prettier вручную.

### ✅ После (После применения чит-кода Архитектора)

```tsx
// 🤩 Идеально контролируемый Mobile-First и семантический код
import { cva, type VariantProps } from "class-variance-authority";
import { cn } from "@/lib/utils";

const buttonVariants = cva(
  "flex w-full items-center justify-center p-3 text-sm font-bold transition-colors md:w-32",
  {
    variants: {
      variant: {
        default: "bg-primary text-primary-foreground hover:bg-primary/90",
        outline: "border border-input bg-background hover:bg-accent hover:text-accent-foreground",
      },
      size: {
        default: "h-10 px-4 py-2",
        sm: "h-9 rounded-md px-3",
        lg: "h-11 rounded-md px-8",
      },
    },
    defaultVariants: {
      variant: "default",
      size: "default",
    },
  }
);

export function Button({ className, variant, size, ...props }: ButtonProps) {
  return (
    <button className={cn(buttonVariants({ variant, size, className }))} {...props} />
  );
}
```

Сделайте этот промпт архитектора своим главным оружием: загоните ИИ в жесткие рамки и заставьте его играть исключительно по вашим правилам. Отныне в вашей кодовой базе будет царить только безупречная консистентность и порядок. Автоматизируйте рутину и заканчивайте работу вовремя! 🍷
