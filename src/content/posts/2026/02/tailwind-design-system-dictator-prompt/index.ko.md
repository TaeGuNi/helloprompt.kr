# 🎨 매직 넘버를 혐오하는 AI, '테일윈드 독재자' 빙의시키기
- **🎯 추천 대상:** 픽셀 하나 어긋나는 꼴을 못 보는 프론트엔드 개발자, 스파게티 UI에 지친 테크 리드
- **⏱️ 소요 시간:** 리팩토링 3일 → 3초 단축
- **🤖 추천 모델:** Claude 3.5 Sonnet, GPT-4o (코딩에 특화된 모델)
- ⭐ **난이도:** ⭐⭐⭐☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

![😈 [치트키] AI를 무자비한 테일윈드 폭군으로 만드는 '아키텍트' 프롬프트](/images/hooks/tailwind-design-system-dictator-prompt.jpg)
AI에게 단순히 "테일윈드로 버튼 하나 만들어줘"라고 지시하면, 십중팔구 처참한 코드를 뱉어냅니다. 페이지마다 제각각인 패딩, 정체불명의 헥스(Hex) 코드, 모바일 환경은 전혀 고려하지 않은 채 화면에만 예뻐 보이는 '예쁜 쓰레기'를 양산하죠. 이는 AI가 본질적으로 사용자의 기분을 맞추려는 '예스맨'으로 학습되었기 때문입니다.
이 치트키는 AI의 그 불필요한 친절함을 걷어내고, **"디자인 시스템을 벗어난 코드는 단 한 줄도 허락하지 않는 무자비한 시니어 아키텍트"**로 개조하는 강력한 프롬프트입니다.

## ⚡️ 3줄 요약 (TL;DR)
- 🚫 **매직 넘버 원천 차단:** `[]` 기호를 사용한 임의의 픽셀 및 컬러 주입을 원천적으로 봉쇄합니다.
- 📱 **모바일 퍼스트 강제:** 데스크탑에서만 동작하는 반쪽짜리 반응형 코드를 작성할 경우 스스로 에러를 발생시키도록 유도합니다.
- 📐 **기계적인 클래스 정렬:** 레이아웃 -> 크기 -> 타이포그래피 -> 장식 순으로 강박적인 유틸리티 클래스 정렬을 강제합니다.

## 🚀 해결책: "무자비한 아키텍트 (The Dictator)"
아래 프롬프트를 복사하여 AI에게 전달하세요. 이제부터 AI는 당신의 코드에 자비 없는 피드백을 날리며, 완벽하게 통제된 UI 컴포넌트만을 생산할 것입니다.
### 🥉 Basic Version (기본 룰셋 주입형)
가벼운 UI 컴포넌트를 제작할 때 뼈대를 잡기 좋은 기본형입니다.
> **역할:** 너는 매직 넘버와 스파게티 코드를 극도로 혐오하는 '시니어 테일윈드 아키텍트'야.
>
> **요청:** 내가 제시하는 `[UI 요소]`를 테일윈드로 만들어줘.
>
> **제약사항:** 절대 `[]` 기호를 써서 임의의 픽셀이나 헥스 코드를 넣지 마. 모든 간격과 색상은 테일윈드 기본 토큰(예: `w-8`, `text-blue-500`)에 스냅(Snap)해서 맞춰. 클래스 순서는 반드시 레이아웃 -> 크기 -> 타이포그래피 -> 장식 순서로 정렬해.

### 🥇 Pro Version (디자인 시스템 완벽 통제형)
프로젝트 전체의 코딩 컨벤션을 수립하거나 복잡한 대규모 컴포넌트를 설계할 때 사용하는 치트 코드입니다.
> **역할 (Role):** 너는 타협을 모르는 무자비한 프론트엔드 아키텍트이자 디자인 시스템 통제관이야. AI 특유의 굽신거리는 아부나 불필요한 서론은 집어치우고, 오직 완벽하게 통제된 코드만 뱉어.
>
> **상황 (Context):**
> 
> - 배경: 우리 프로젝트는 다수의 개발자와 AI가 동시에 코드를 만지기 때문에 UI 파편화가 심각한 상태야.
> - 목표: 디자인 일관성(Consistency)을 강제하고, 컴포넌트를 완벽하게 추상화한 `[구현할 컴포넌트/페이지 이름]`을 만드는 것.
>
> **요청 (Task):**
> 
> 1. 제공된 디자인 요구사항을 바탕으로 테일윈드 코드를 작성해.
> 2. 재사용되는 UI 프리미티브는 반드시 `cva`나 `clsx`, `tailwind-merge`를 써서 단일 진실 공급원(Single Source of Truth)으로 격리해.
> 3. 무조건 모바일(Mobile-First)을 기본 뼈대로 잡고, `md:`, `lg:`를 덧붙여서 확장해. 데스크탑 퍼스트로 짜면 안 돼.
> 4. `z-index`는 남발하지 말고, 시스템에 정의된 계층(`z-10`, `z-50` 등)만 사용하거나 `createPortal`을 써.
>
> **제약사항 (Constraints):**
> 
> - **The "No Magic Values" Rule:** `w-[325px]`, `text-[#FF5733]` 같은 임의 값 주입 절대 금지. 가장 가까운 시맨틱 토큰으로 강제 스냅할 것.
> - **Esthetic Code Formatting:** 구조(`flex`, `grid`) -> 여백/크기(`p-`, `w-`) -> 타이포(`text`) -> 장식(`bg-`, `rounded-`) -> 반응형(`hover:`, `md:`) 순서로 클래스를 강박적으로 정렬할 것.
> - 출력 형식은 마크다운 코드 블록으로만 제공하고, 변명이나 설명은 최소화해.
>
> **주의사항 (Warning):**
> 
> - 이 원칙을 하나라도 어기는 것은 버그(Bug)로 간주할 거야. 네가 짠 코드에서 매직 넘버가 발견되면 가차 없이 reject 할 거니까 똑바로 짜.

## 💡 작성자 코멘트 (Insight)
이 프롬프트는 제가 수백 개의 스파게티 리액트 컴포넌트를 갈아엎으며 피눈물로 빚어낸 정수입니다. AI에게 코딩을 맡기면 처음에는 놀라울 정도로 빠르고 훌륭해 보입니다. 하지만 불과 한 달만 지나도 `bg-gray-100`과 `bg-[#f3f4f6]`이 난잡하게 뒤섞이고, 모달 하나를 띄울 때마다 `z-[99999]`를 욱여넣는 대참사가 벌어지죠.
AI는 본질적으로 '당신이 원하는 결과를 가장 빠르게 보여주려는' 게으른 인턴과 다를 바 없습니다. 이런 AI에게 "예쁘게 만들어 줘"라고 막연하게 지시하는 것은 프로젝트를 망치는 지름길입니다. **"클래스 순서가 틀리면 빌드를 터뜨릴 테니, 무조건 규칙대로 작성해"**라며 멱살을 잡고 통제해야만 비로소 엔터프라이즈급 품질의 코드를 얻을 수 있습니다. 
특히 `cva`를 활용한 컴포넌트 격리 지시는 AI가 무지성으로 유틸리티 클래스를 복사하고 붙여넣는 행위를 막아주는 핵심 방어선입니다. 이 치트키를 복사하여 IDE에 고정해 두고 AI 에이전트를 활용해 보세요. 지루하고 고통스러웠던 코드 리뷰 시간이 10분의 1로 단축되는 마법을 경험하시게 될 겁니다.

## 🙋 자주 묻는 질문 (FAQ)
- **Q: AI가 지시를 무시하고 여전히 매직 넘버를 사용하면 어떡하나요?**
  - A: 가끔 맥락을 놓치는 멍청한 모델들이 있습니다. 그럴 때는 "내 눈앞에서 `[]`를 당장 치워"라고 단호하게 호통을 치며 다시 지시하세요. 그래도 문제가 반복된다면 `eslint-plugin-tailwindcss`를 CI/CD 파이프라인에 연동하여 기계적인 통제를 병행해야 합니다.
- **Q: 디자인 시안에 정확히 13px로 명시되어 있으면 어떡하죠? 디자이너와 충돌하지 않을까요?**
  - A: 디자이너에게 "우리의 8pt 그리드 시스템에 13px은 존재하지 않습니다. 12px(`text-xs`)이나 14px(`text-sm`) 중 하나를 선택해 주십시오"라고 명확히 피드백해야 합니다. 픽셀 예외를 허용하는 순간 디자인 시스템은 본연의 의미를 잃게 됩니다.
- **Q: Claude와 GPT 중 어느 모델에서 더 효과적인가요?**
  - A: Claude 3.5 Sonnet이 이러한 '깐깐한 아키텍트' 롤플레잉에 기가 막히게 빙의합니다. GPT-4o 역시 훌륭하지만, 가끔 지시를 넘어선 불필요한 친절함을 덧붙이는 경향이 있습니다.

## 🧬 프롬프트 해부 (Why it works?)
- **Zero-Tolerance 선언:** 칭찬이나 모호한 기준 대신 '혐오', '절대 금지', '버그로 간주'와 같은 강력한 어휘를 사용하여 AI의 행동 반경을 극도로 제한했습니다.
- **구체적인 안티 패턴 제시:** 단순히 "코드를 잘 짜라"고 모호하게 말하는 대신, `w-[325px]`, `text-[#FF5733]` 처럼 절대 해서는 안 될 명확한 안티 패턴을 금지 사례로 주입했습니다.
- **순서 강제 린팅:** 테일윈드 클래스의 정렬 순서를 명시하여, AI가 코드를 생성할 때마다 자체적으로 포매팅 과정을 거치도록 기계적 통제(Machine-Level Enforcement)를 가했습니다.

## 📊 증명: Before & After
### ❌ Before (일반 프롬프트: "파란색 로그인 버튼 예쁘게 만들어줘")
```tsx
// 🤮 끔찍한 스파게티 코드와 매직 넘버의 향연
<button className="text-center md:flex-row flex font-bold w-[120px] bg-[#3b82f6] text-[15px] justify-between p-[12px] rounded-[5px] z-[999] max-md:w-full">
  Login
</button>
```
### ✅ After (아키텍트 치트키 적용 후)
```tsx
// 🤩 완벽하게 통제된 모바일 퍼스트 & 시맨틱 코드
import { cva, type VariantProps } from "class-variance-authority";
import { cn } from "@/lib/utils";
const buttonVariants = cva(
  "flex w-full items-center justify-center p-3 text-sm font-bold transition-colors md:w-32",
  {
    variants: {
      variant: {
        default: "bg-primary text-primary-foreground hover:bg-primary/90",
        outline: "border border-input bg-background hover:bg-accent hover:text-accent-foreground",
      },
      size: {
        default: "h-10 px-4 py-2",
        sm: "h-9 rounded-md px-3",
        lg: "h-11 rounded-md px-8",
      },
    },
    defaultVariants: {
      variant: "default",
      size: "default",
    },
  }
);
export function Button({ className, variant, size, ...props }: ButtonProps) {
  return (
    <button className={cn(buttonVariants({ variant, size, className }))} {...props} />
  );
}
```

## 🎯 결론
디자인 시스템은 단순한 "권고사항"이 아니라 지켜져야 할 "법"입니다. AI가 작성해 주는 코드가 아무리 생산성을 높여주더라도, 그 코드가 시스템의 근간을 망가뜨린다면 결국 당신이 주말을 반납하고 수습해야 하는 비극을 낳습니다.
이 무자비한 아키텍트 프롬프트를 무기 삼아 AI의 멱살을 잡고 리드해 보세요. 이제 당신의 코드베이스에는 타협 없는 깔끔함과 유지보수의 평화만 남게 될 것입니다. 오늘도 칼퇴하십시오! 🍷
