# 構建複雜提示詞：CREATE 架構

隨著 AI 模型的不斷發展，我們向它們指派的任務也越來越複雜。雖然簡單的提示詞足以應對簡單的問題，但在指導 AI 處理複雜的業務邏輯或創造性任務時，需要一種系統化的方法。

本文將介紹 **CREATE 架構**，這是一種邏輯化構建提示詞以最大化 AI 效能的方法。

## 什麼是 CREATE 架構？

CREATE 代表了構建有效提示詞的六個關鍵要素。

1.  **C**ontext (背景): 為 AI 分配角色和情境。
2.  **R**equest (請求): 明確說明需要執行的具體任務。
3.  **E**xamples (範例): 提供期望輸出的範例 (Few-shot)。
4.  **A**djustments (調整): 設定語氣、風格和限制條件。
5.  **T**ype of Output (輸出類型): 指定結果的格式（表格、程式碼、Markdown 等）。
6.  **E**xtras (額外事項): 包含邊緣情況處理或特殊說明。

---

## 分步應用指南

### 1. Context (背景設定)

賦予 AI 一個角色會顯著改變回答的品質。

> 「你是一名擁有 10 年經驗的資深軟體工程師。你擅長將遺留程式碼重構為現代的整潔程式碼。」

### 2. Request (具體請求)

避免模稜兩可，使用清晰的動詞。

> 「分析下面提供的 Python 函式，並對其進行重構，以提高可讀性並最佳化執行速度。」

### 3. Examples (提供範例)

AI 透過範例學習模式。展示輸入和輸出對。

> **輸入:**
> `def calc(x,y): return x+y`
>
> **輸出:**
>
> ```python
> def add_numbers(a: int, b: int) -> int:
>     """返回兩個數字的和。"""
>     return a + b
> ```

### 4. Adjustments (細節調整)

決定回答的語調和風格。

> 「解釋應簡潔並使用專業術語，但要保持親切的語調，以便初級開發人員也能理解。」

### 5. Type of Output (輸出格式)

指定你希望如何接收結果。

> 「請首先在程式碼區塊中展示修改後的完整程式碼，然後在下方以列點摘要說明修改原因。」

### 6. Extras (額外說明)

防止意外情況。

> 「如果在程式碼中發現安全漏洞，請在重構之外單獨輸出警告訊息。」

---

## 結論

使用 CREATE 架構可以縮短編寫提示詞的時間，同時保持輸出品質的一致性。當面臨複雜問題時，不要模糊地提問，而是按照 CREATE 的結構整理思緒。你設計的提示詞越精細，AI 的回答就越智能。
