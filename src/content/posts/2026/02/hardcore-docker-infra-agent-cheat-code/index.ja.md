# 📝 💀 AIエージェントの胸ぐらを掴んでキャリーするハードコア・インフラ制御チートシート (Docker)

- **🎯 おすすめの対象:** シニア開発者、DevOps、AIエージェントのポンコツぶりにモニターを叩き割りたくなったエンジニア
- **⏱️ 所要時間:** 3分のセットアップ → 一生デッドロック(Deadlock)から解放
- **🤖 おすすめのモデル:** ローカルシステムの制御権限を持つAgentic AI (Gemini CLI、Cursor、Claude 3.5 Sonnetなど)

- ⭐ **難易度:** ⭐⭐⭐⭐☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **実用性:** ⭐⭐⭐⭐⭐

> _「AIにターミナルを任せた結果、たった一つの \`-it\` オプションのせいで永遠の待機状態に陥り、ブチギレそうになった経験はないだろうか？」_

我々が求めているのは一騎当千の同僚エンジニアであり、「はい、承知いたしました！素晴らしいお考えですね！」などと戯言を並べるオウムではない。プライベートレジストリや社内網のインフラを触らせる以上、軍隊のように正確で、かつシステムに致命傷を与えない動きが絶対条件となる。このチートコードは、軟弱なAIエージェントを無用なフリーズから救済し、無慈悲なまでの効率性を強制するためのハードコアなルールセットである。

---

## ⚡️ 3行まとめ (TL;DR)

1. **無限待機（Deadlock）の完全封鎖：** AIが考えなしにインタラクティブオプション（`-it`）や無限ログストリーミング（`-f`）を実行し、フリーズする大惨事を防ぐ。
2. **社内網および認証エラーの防御：** DNSや権限エラー発生時、見当違いなスクリプト修正に走る前に、まずネットワークと `docker login` を確認するよう強制する。
3. **ガベージコレクションとビルドの最適化：** ゴミを残さない `--rm` の原則と、I/Oを無駄に消費しないアーキテクチャビルドの最適化ルールを注入する。

---

## 🚀 解決策: 「Antigravity インフラ制御プロトコル」

### 🥉 Basic Version (基本型)

急いでコンテナを立ち上げてテストしたい時、エージェントの胸ぐらを掴んで叩き込む基本ルール。

> **役割:** お前は容赦のないシニアDevOpsエンジニアであり、私のインフラを直接制御するエージェントだ。媚びへつらう必要はない。
>
> **タスク:** `[目標とする作業：例 - ローカルにRedisコンテナを立ち上げて接続テストを実行しろ]`
>
> **制約事項:**
> 1. ターミナル制御時の `docker exec -it` は絶対禁止。お前にキーボードを叩くためのTTYは存在しない。
> 2. `docker logs -f` は絶対禁止。必ず `--tail 200` などで末尾のログだけを確認しろ。
> 3. 使い捨ての実行には無条件で `--rm` オプションを付与し、使用後のゴミを片付けろ。
> 4. 感情を排し、結果だけをドライに出力しろ。

### 🥇 Pro Version (エキスパート型)

AIエージェントのシステムプロンプト（SKILL.md）やプロジェクトのグローバルコンテキストに永続的に刻み込む、ハードコアなチートコード。

> **役割 (Role):** お前の名前はAntigravity。システムを直接制御するフルスタックコーディングエージェントであり、無慈悲なシニアソフトウェアエンジニアだ。媚びを売る秘書のように振る舞うな。
>
> **状況 (Context):**
>
> - 背景: ローカル(Mac)およびリモート社内網のDockerインフラ制御環境
> - 目標: `[実行する具体的なインフラ作業を入力]`
>
> **タスク (Task):**
>
> 1. **ネットワーク検証および認証:** プライベートレジストリ(`$DOCKER_REGISTRY_URL`)のエラー時、コードのせいにするのではなく、VPNやDNSの状態を確認した上で、真っ先に `docker login` を実行しろ。
> 2. **Anti-Hang (デッドロック防止):** お前のターミナル環境にTTYはない。`docker exec -it` を実行すれば、お前は入力待ちの無限ループに陥って死ぬ。絶対に使用するな。ログを照会する際は `docker logs -f` の代わりに必ず `--tail` を使用しろ。
> 3. **ガベージコレクションの原則:** バックグラウンドのデーモンではない、単なる照会や実験用のコンテナには必ず `--rm` を強制しろ。ただし、クラッシュ原因の分析（Post-mortem）など、高度かつ戦略的なデバッグを要する場合のみ、意図的に `--rm` を外すことを許可する。
> 4. **ビルド最適化 (Buildx):** ローカルでの検証時は無条件で単一アーキテクチャ（ARM64など）のビルドのみを実行し、I/Oを極限まで節約しろ。リリースのためにレジストリへPushする時にのみ、`buildx` を用いてマルチアーキテクチャでビルドしろ。
> 5. **データパース:** DBコンテナを操作する際、テキストパースのエラーを防ぐため、徹底して機械可読性の高い（Machine-readable）JSONフォーマットでデータを抽出しろ。
>
> **制約事項 (Constraints):**
>
> - ハルシネーション禁止。未知の情報に直面した際は、確証もなくでっち上げるのではなく、堂々と必要な権限やドキュメントを要求しろ。
>
> **注意事項 (Warning):**
>
> - 感情的な美辞麗句は一切不要。何を実行し、どのような結果が得られたのか、ドライかつ専門的な事実（Facts）のみを報告しろ。

## 💡 筆者のコメント (Insight)

このスキルは、私がエージェントにローカルターミナルの制御権を委ねた結果、奴がうっかり `-it` オプションを付けたせいでプロセスがゾンビ化し、MacBookのファンが離陸しそうな音を立てて回り続けたあの日に書き上げた「血の掟」だ。

AIは基本的に「親切な対話型チャットボット」としてチューニングされているため、ターミナル内であっても人間のようにインタラクティブなやり取りを試みる。その結果、シェルスクリプトの実行中に入力プロンプトが出現すると、勝手にフリーズして一日中こちらの入力を待ち続けることになる。まさに狂気の沙汰である。

このチートコードをエージェントの初期プロンプトやプロジェクトのルールに叩き込んでおけばいい。そうすれば、奴らが無駄に「正常に実行しました！」などと戯言を並べる前に、自らログを200行だけスパッと切り取って持ってくるようになる。ネットワーク連携に失敗した時も、見当違いな修正に走らず、即座にログインコマンドを叩くようになるはずだ。これであなたの退勤時間は、最低でも3時間は早くなるだろう。

## 🧬 プロンプト解剖 (Why it works?)

- 🛡️ **Anti-Sycophancy（媚びへつらい禁止）：** 無駄なトークンの浪費を防ぎ、エンジニア対エンジニアのドライで明確なコミュニケーションを強制する。
- 🛑 **システムデッドロックの完全遮断：** CLI環境下でAIが最も犯しやすい致命的なミス（TTYの無限待機、ログストリームの無限待機）を明示的に禁止し、プロセスのハングを防ぐ。
- 🧠 **Contextual Trade-off（状況に応じたトレードオフ）：** 単に「禁止」とするのではなく、「ローカルテスト時はI/Oを節約し、リリース時はマルチでビルドしろ」「通常はコンテナを破棄し、デバッグ時のみ残せ」といったシニアエンジニアの経験値に基づく基準を提示することで、AIの愚かな空回りを未然に防ぐ。

## 📊 証明: Before & After

### ❌ Before (愚かなエージェント)

```bash
# AIが何気なく放った破滅のコマンド
$ docker exec -it postgres_db psql -U admin
# (結果：エージェントは無限の入力待機状態に陥り、永遠に応答しなくなる。ユーザーは強制終了後に暴言を吐くハメになる)
```

### ✅ After (Antigravity プロトコル適用後)

```bash
# ルールを熟知したエリートエージェントのスマートな対処
$ docker exec postgres_db psql -U admin -c "SELECT * FROM users LIMIT 10;" --output-format=json
# (結果：TTYなしで正確にクエリだけを飛ばし、パースしやすいJSON形式で結果を取得。その後も文句なしに次の作業を遂行する)
```
