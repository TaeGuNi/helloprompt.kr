# 📝 Gemini 3 Pro：実務コーディングにおける「Deep Think」モードの正しい使い方

- **🎯 おすすめの対象：** ジュニアエンジニア、レガシーコードを保守するバックエンドエンジニア
- **⏱️ 所要時間：** 1時間 → 5分に短縮
- **🤖 推奨モデル：** Gemini 3 Pro（Deep Thinkモード必須）

- ⭐ **難易度：** ⭐⭐⭐☆☆
- ⚡️ **効果性：** ⭐⭐⭐⭐⭐
- 🚀 **活用度：** ⭐⭐⭐⭐⭐

> _「Gemini 3 Proはコードが書けない？それはAIに『考える時間』を与えていないからです。」_

2026年2月にGoogleが公開した**Gemini 3 Pro**の最大の目玉は、間違いなく**「Deep Think」**モードです。OpenAIのo2モデルが示した「推論（Reasoning）」能力を、はるかに高速に活用できるようになりました。しかし、単に「このコードをリファクタリングして」と指示するだけでは、見当違いのコードを出力し、計算リソースを無駄にするケースが多発します。本記事では、ジュニアエンジニアでもシニアレベルのアーキテクチャ設計を引き出せる、実践的なプロンプトをご紹介します。

---

## ⚡️ 3行まとめ (TL;DR)

1. Gemini 3 Proの「Deep Think」モードは、明確な方向性を示さないと真価を発揮できません。
2. コードを書かせる前に、「コード監査（Audit）」と「リファクタリング戦略の策定」のステップを踏ませるよう強制する必要があります。
3. 複雑な依存関係が絡み合うレガシーコードを、安全かつ体系的にマイグレーションする際、このアプローチは圧倒的な効率を誇ります。

## 🚀 解決策："The 'Architect' Protocol"

### 🥉 Basic Version（基本型）

コードのセキュリティリスクやパフォーマンスのボトルネックを素早く把握したい場合に使用してください。

> **役割：** あなたは`[経験20年のシニアバックエンドエンジニア]`です。
> **タスク：** 以下の`[レガシーコード]`に潜むセキュリティおよびパフォーマンスの問題点を3つ分析して教えてください。

<br>

### 🥇 Pro Version（専門家型）

単なるコードの修正にとどまらず、アーキテクチャレベルのリファクタリングからエッジケースのテストコードまで、完璧に抽出したい場合に使用してください。

> **役割 (Role)：** あなたは経験20年のチーフソフトウェアアーキテクトであり、クリーンコードの伝道師です。単なる機能実装にとどまらず、保守性、拡張性、パフォーマンスを最優先に考える専門家です。
>
> **状況 (Context)：**
>
> - 背景：現在、スパゲッティのように絡み合った`[Pythonのレガシーコード]`を、保守可能な構造へとマイグレーションする必要があります。
> - 目標：単一責任の原則（SRP）を遵守し、依存関係を疎結合にしたコードへと生まれ変わらせることが目標です。
>
> **タスク (Task)：**
> ユーザーが提供するレガシーコードを分析し、以下の4つのステップに従ってリファクタリングを実行してください。
>
> 1. **コード監査 (Code Audit)：** 現在のコードの問題点（アンチパターン、セキュリティリスク、パフォーマンスのボトルネック）を3つ以上分析すること。
> 2. **深い思考戦略 (Deep Thinking Strategy)：** リファクタリングのための具体的な戦略を策定すること。（例：どのデザインパターンを適用するか、なぜそのパターンが最適なのか）
> 3. **リファクタリング (Refactoring)：** 改善されたコードを作成し、コアロジックにはコメントで変更理由を明記すること。
> 4. **テストケース生成 (Test Case Generation)：** リファクタリングされたコードが安全であることを証明できる、エッジケースのテストコードを3つ作成すること。
>
> **制約事項 (Constraints)：**
>
> - 言語およびバージョン：`[Python 3.12+]`（Type Hintingの適用必須）
> - スタイルガイド：PEP 8準拠
> - 外部ライブラリ：標準ライブラリのみ使用すること（外部依存関係の最小化）
> - 出力形式：ジュニアエンジニアでも理解できるように、Markdown形式で見やすく整理すること。
>
> **注意事項 (Warning)：**
>
> - 分析結果が不確実または曖昧な場合は、無理にコードをでっち上げず、「判定不可」と明記すること。（ハルシネーションの防止）
>
> **入力データ (Input Data)：**
> `[ここにリファクタリングするコードを貼り付けてください]`

## 💡 筆者のコメント (Insight)

正直に告白すると、私も最初は単純なスクリプト作成において高速かつ優秀なGPT-5の方を信頼していました。しかし、実務で数千行に及ぶレガシーAPIをマイグレーションする際、Gemini 3 Proの広大なコンテキストウィンドウと「Deep Think」の組み合わせは、まさに圧倒的でした。

このプロンプトの核心は、AIの創造性を一定の枠内で「制御」することにあります。いきなりコードを書かせるのではなく、**「どのようなパターンを使うか、まず計画を立てよ」**と強制することで、AIがコーディング中に迷走したりハルシネーション（幻覚）を起こしたりする頻度を劇的に減らすことができます。メール送信ロジックを分離しながら、トランザクションのロールバック処理までAI自らが考慮している姿を見ると、残業時間が半分に減りそうなほどの心強さを感じます。

## 🙋 よくある質問 (FAQ)

- **Q: GPT-5でこのプロンプトを使っても問題ありませんか？**
  - A: GPT-5でも見事に機能します。ただし、コードベースが大きく、ファイル間の依存関係を深く読み解く必要がある「複雑なリファクタリング」の場面では、Gemini 3 ProのDeep Thinkモードの方がエッジケースをはるかに正確に発見します。一方、シンプルなユーティリティ関数の修正であれば、GPT-5の方が高速な場合があります。

- **Q: 他のプログラミング言語（Java、TypeScriptなど）に適用するにはどうすればよいですか？**
  - A: 制約事項（Constraints）セクションの`言語およびバージョン`と`スタイルガイド`の部分を、`[TypeScript 5.0+, ESLint Airbnbルール準拠]`などに修正して使用すれば、完璧に動作します。

- **Q: AIが分析結果を頻繁に「判定不可」と出力してしまいます。**
  - A: 入力データとして提供されたコードのコンテキストが不足しているためです。関連するデータベースのスキーマや、他のクラスのインターフェース情報を一緒に提供することで、AIはより正確な設計図を描き出すことができます。

## 🧬 プロンプトの解剖学 (Why it works?)

1. **Chain-of-Thought（思考の連鎖）の誘導：** `コード監査 -> 戦略 -> リファクタリング -> テスト`という4つのステップを明確に分離し、AIに段階的かつ論理的な思考を展開するよう強制しました。
2. **ハルシネーションの防止：** `Warning`ブロックを通じて、不確実な状況でもっともらしい嘘のコードを生成しないよう、強力な制約条件を設けました。
3. **役割（Role）付与のディテール：** 単なる「エンジニア」ではなく、「クリーンコードの伝道師」かつ「チーフソフトウェアアーキテクト」という鋭角なペルソナを与えることで、成果物の品質基準（Quality Standard）を大幅に引き上げました。

```python
# UserCreateService.py
from dataclasses import dataclass
from typing import Protocol

class UserRepository(Protocol):
    def save(self, user: 'User') -> bool: ...

class EmailSender(Protocol):
    def send_welcome_email(self, email: str) -> bool: ...

@dataclass
class UserDTO:
    email: str
    name: str

class UserCreateService:
    """
    [Refactoring Insight]

    また、依存性逆転の原則（DIP）を適用し、単体テストを容易にしています。
    """
    def __init__(self, user_repo: UserRepository, email_sender: EmailSender):
        self.user_repo = user_repo
        self.email_sender = email_sender

    def execute(self, user_dto: UserDTO) -> None:
        # トランザクションおよび失敗時のロールバックロジックがここに含まれます。
```
