## 🚀 解決策：「Claude Code 2Dゲームジェネレーター」

### 🥉 Basic Version（基本型）

プロジェクトの初期の骨組みを作り、コアとなるメカニクスが正常に動作するかを素早くテストしたい場合に使用してください。

> **役割:** あなたはシニアゲームクライアントエンジニアです。
> 
> **タスク:** HTML5 Canvasを使用して、ブラウザベースの2Dゴルフゲームを作成してください。ボール、ホール（Hole）、そしてマウスをドラッグして打つ（drag-to-shoot）メカニクスを含める必要があります。

### 🥇 Pro Version（専門家型）

物理エンジンからレベルデザイン、コードのモジュール化まで、プロダクションレベルに近いゲームを企画する際に使用してください。

> **役割 (Role):** あなたは10年以上の経験を持つシニアウェブゲームエンジニアであり、私のAIコーディングパートナーです。
>
> **状況 (Context):**
>
> - 背景: 重いゲームエンジンや外部ライブラリを一切使わず、純粋なHTML5 CanvasとVanilla JavaScriptだけで、軽快で高速な2Dミニゲームを開発しようとしています。
> - 目標: プレイヤーがマウスのドラッグ＆ドロップでボールの軌道とパワーを調整し、ホールインワンを目指す2Dゴルフゲームの構築。
>
> **指示 (Task):**
>
> 1. プロジェクトを `index.html`（Canvasコンテナ）、`game.js`（ゲームループと状態管理）、`physics.js`（ベクトル数学と物理演算処理）に完全に分離し、モジュール化してください。
> 2. `[摩擦力（芝生の抵抗シミュレーション）]`やキャンバス端の`[壁との衝突バウンド]`など、基本的な物理エンジンを精巧に実装してください。
> 3. プレイヤーがドラッグする際、発射されるパワーと方向を事前に示す視覚的なインジケーター（照準線のガイド）を追加してください。
> 4. 障害物の要素を導入し、難易度が徐々に上がる3つの`[レベル設定配列]`を構成してください。
>
> **制約事項 (Constraints):**
>
> - コードは即座に実行可能でなければならず、省略することなく完全な形で提供してください。
> - 今後の修正が容易になるよう、各コアロジック（速度の減衰、イベントリスナーなど）には日本語で詳細なコメントを追加してください。
>
> **注意事項 (Warning):**
>
> - 過度に複雑な現実の物理法則よりも、アーケードゲーム特有の軽快で直感的な操作感を最優先に設計してください。

## 💡 筆者のインサイト（Insight）

AIを活用したコーディングで最も重要なのは、**「AIを単なる自動補完ツールではなく、構造を議論しアーキテクチャを設計するシニアパートナーとして扱うこと」**です。

最初から完璧で巨大なゲームを作ろうとするのではなく、コアとなる物理メカニクス（摩擦、衝突）→ 入力処理（ドラッグ＆シュートのフィードバック）→ コンテンツの拡張（レベルデザインの追加）の順に、**段階的なビルドアップ（Step-by-step）**を経ることが成功確率を飛躍的に高める秘訣です。

特にClaude Codeは、コンテキストを維持しながらコードをリファクタリングする能力が圧倒的に優れています。実装の途中で「障害物を管理する別のクラスを切り出してリファクタリングして」といった構造的な改善を要求すれば、より保守性の高い、洗練された成果物を得ることができます。

## 🙋 よくある質問（FAQ）

- **Q: ゲーム開発や数学（ベクトル）を全く知らない初心者でも、このプロンプトでゲームを作れますか？**
  - A: はい、可能です！プロンプトが全体的なアーキテクチャと厄介な数学的計算をすべてカバーしてくれるため、提供されたコードを構造に合わせてコピー＆ペーストするだけで、ブラウザ上ですぐに実行して遊ぶことができます。

- **Q: 障害物や新しいマップをさらに追加したい場合、自分でコードを修正する必要がありますか？**
  - A: 直接コードを解析して修正する必要はありません。Claudeに「レベル3の次に、中央に回転する風車の障害物があるレベル4を追加して」と自然言語でリクエストすれば、既存のレベル配列の構造を把握し、適切なコードを即座に生成してくれます。

- **Q: Claude Codeの代わりにChatGPT環境で使用しても同じ結果が得られますか？**
  - A: はい、最新モデル（GPT-4oなど）でも素晴らしい結果が得られます。ただし、ターミナルベースのClaude CodeやCursorエディタのように複数ファイルを自動で生成・上書きする機能がWeb版のChatGPTにはないため、AIが出力したコードを指示に従って手動で`game.js`や`physics.js`などに分けて保存する必要があります。

## 🧬 プロンプトの解剖学（Why it works?）

1. **モジュール化の強制 (Modularity):** 最初からファイルを分離するよう強制的に指示することで、コードが長くなってもスパゲッティ化を防ぎ、保守と拡張が容易なアーキテクチャの骨組みを構築しました。
2. **視覚的フィードバックの明示 (Visual Indicator):** 単純な操作の実装にとどまらず、「パワーと方向を示す線」を具体的に明示することで、ゲームの操作感（UX）と直感性を大幅に向上させました。
3. **拡張性の考慮 (Level Design):** ハードコーディングされた単一のステージではなく、「配列形式のレベルデータ構成」を誘導することで、今後開発者がレベルを無限に、そして簡単に追加できるデータ駆動型の構造を作成しました。

## 📊 検証：Before & After

### ❌ Before（単に「ゴルフゲームを作って」と入力した場合）

```text
ボールが壁にぶつかるとキャンバスの外に永遠に消えてしまったり、マウスクリックイベントの処理が絡まってボールが無限に加速するバグが発生する。床の摩擦力の概念がないため、ボールが永遠に止まらずに転がり続ける。
```

### ✅ After（Pro Version適用後に生成された物理ロジックの例）

```javascript
// Generated by Claude Code
update() {
  this.velocity.x *= 0.98; // 摩擦力：芝生の抵抗をシミュレーションして徐々に減速
  this.velocity.y *= 0.98;

  this.x += this.velocity.x;
  this.y += this.velocity.y;

  // キャンバス壁面との衝突処理（自然なバウンド）
  if (this.x < 0 || this.x > canvas.width) {
    this.velocity.x *= -1; // 方向の反転
  }
  if (this.y < 0 || this.y > canvas.height) {
    this.velocity.y *= -1;
  }
}
```

_結果：Claude Codeは、時間の経過とともに速度を落とすダンピング（Damping）の数学的概念を正確に適用し、「ゴルフ場の芝生の摩擦力」を完璧に実装しきりました。_
