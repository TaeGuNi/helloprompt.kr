## 🐍 Solucionador de Errores en Python: Prompt de Depuración en 10 Segundos

- **🎯 Recomendado para:** Desarrolladores Junior, Científicos de Datos, Estudiantes de Python
- **⏱️ Tiempo estimado:** De 1 hora → a 10 segundos
- **🤖 Modelos recomendados:** GPT-4, Claude 3.5 Sonnet, Gemini 2.5 Pro

- ⭐ **Dificultad:** ⭐☆☆☆☆
- ⚡️ **Efectividad:** ⭐⭐⭐⭐⭐
- 🚀 **Utilidad:** ⭐⭐⭐⭐⭐

> _"¿Estás perdiendo horas mirando un 'SyntaxError' o 'IndentationError' sin sentido? Deja de adivinar y deja que la IA encuentre el error por ti."_

Cualquier desarrollador de Python conoce la frustración de perder horas rastreando un error tipográfico invisible o un fallo lógico indescifrable. En lugar de copiar y pegar tu código en Google o Stack Overflow cruzando los dedos para hallar una respuesta útil, este prompt transforma a la IA en tu ingeniero de software senior personal. Analizará el error a fondo, te explicará exactamente por qué ocurrió y te entregará el código corregido de forma instantánea.

---

## ⚡️ Resumen en 3 líneas (TL;DR)

1. Introduce tu código defectuoso junto con el mensaje de error exacto de la consola.
2. La IA asume el rol de un Ingeniero Senior en Python para ejecutar un Análisis de Causa Raíz (RCA).
3. Recibes una explicación cristalina y el código refactorizado, listo para implementar.

## 🚀 La Solución: "Detector de Errores Python"

### 🥉 Versión Básica (Basic Version)

Úsala cuando necesites una solución rápida sin demasiada explicación técnica.

> **Rol:** Eres un experto desarrollador de Python.
> **Tarea:** Encuentra y corrige el error en el siguiente código.
> **Código:** `[Inserta tu código aquí]`
> **Error:** `[Inserta el mensaje de error aquí]`

### 🥇 Versión Profesional (Pro Version)

Úsala para errores lógicos complejos o cuando desees aprender a fondo _por qué_ falló el código.

> **Rol (Role):** Eres un Ingeniero de Software Senior especializado en Python con más de 10 años de experiencia en depuración avanzada y optimización de código.
>
> **Contexto (Context):**
>
> - Escenario: Estoy desarrollando un script/aplicación en Python y me he topado con un error bloqueante que no logro resolver.
> - Objetivo: Comprender la causa raíz del fallo, obtener el código corregido y asimilar cómo prevenir este problema en el futuro.
>
> **Tarea (Task):**
>
> 1. Analiza exhaustivamente el `[Código Problemático]` y el `[Mensaje de Error]` proporcionados.
> 2. Explica la **Causa Raíz** del error de forma clara, directa y concisa (máximo 3 líneas).
> 3. Proporciona el **Código Corregido** íntegro, incluyendo comentarios que destaquen exactamente qué líneas se modificaron y su justificación.
> 4. Sugiere una **Mejor Práctica** para blindar el código contra errores similares.
>
> **Datos (Data):**
>
> - `[Código Problemático]`: `[Pega tu código aquí]`
> - `[Mensaje de Error]`: `[Pega el traceback o log del error aquí]`
>
> **Restricciones (Constraints):**
>
> - No reescribas la totalidad del código a menos que sea estrictamente necesario; respeta mi estilo de codificación original al máximo.
> - La explicación técnica debe ser digerible para un desarrollador de nivel junior.
> - Presenta el código corregido dentro de un bloque de código Markdown limpio y estructurado.

## 💡 Comentarios del Autor (Insight)

Este prompt es un auténtico salvavidas, especialmente si trabajas con librerías robustas como Pandas y NumPy, o con frameworks de desarrollo web como Django y FastAPI. Con frecuencia, un _traceback_ en Python puede resultar abrumador, apuntando a archivos internos de la propia librería en lugar de señalar el fallo en tu código.

Al instruir a la IA para que asuma el rol de un **Ingeniero Senior** y exija explícitamente la **Causa Raíz**, evitas que te devuelva un bloque de código genérico y descontextualizado. Esta estrategia no solo soluciona la urgencia inmediata, sino que acelera tu crecimiento como programador al desgranar tus propios fallos lógicos o de sintaxis. En mi experiencia práctica, aplicar este enfoque ha comprimido horas de depuración frustrante a literalmente un par de segundos.

## 🙋 Preguntas Frecuentes (FAQ)

- **P: ¿Este método funciona con otros lenguajes como JavaScript o C++?**
  - R: ¡Por supuesto! Solo necesitas sustituir la palabra "Python" por "JavaScript" o el lenguaje específico de tu stack. La arquitectura de este análisis de errores es totalmente universal.

- **P: ¿Qué hago si mi código es demasiado extenso para pegarlo por completo?**
  - R: Aísla y pega únicamente la función o clase donde se detonó el error, acompañándola del _traceback_ íntegro. Por regla general, la IA solo requiere el contexto de ejecución inmediato para cazar el bug.

- **P: ¿Es seguro compartir código propietario de mi empresa con la IA?**
  - R: Debes extremar las precauciones. **Jamás** incluyas claves API, contraseñas, ni lógica de negocio confidencial. Lo ideal es ofuscar variables o simplificar el bloque problemático hasta lograr un "ejemplo mínimo reproducible" antes de procesarlo.

## 🧬 Anatomía del Prompt (¿Por qué funciona?)

1. **Rol Altamente Especializado:** Al imponer el rol de un "Ingeniero Senior con más de 10 años de experiencia", forzamos al modelo de IA a calibrar su tono y profundidad técnica, bloqueando de raíz las respuestas superficiales.
2. **Estructura de Datos Delimitada:** Aislar el `[Código Problemático]` del `[Mensaje de Error]` garantiza que la IA procese el contexto de ejecución sin contaminarlo con la lógica defectuosa.
3. **Restricciones de Estilo Estrictas:** La directiva de "mantener el estilo de codificación" impide que la IA sobreescriba tu función utilizando _list comprehensions_ o funciones _lambda_ crípticas que, a largo plazo, arruinarían la mantenibilidad de tu código.

## 📊 Demostración: Antes y Después

### ❌ Antes (Tu problema)

```python
def calcular_promedio(numeros):
    total = sum(numeros)
    cantidad = len(numeros)
    promedio = total / cantidad
    return promedio

print(calcular_promedio([]))
# ZeroDivisionError: division by zero
```

### ✅ Después (Respuesta de la IA)

**Causa Raíz:**
El error ocurre porque estás intentando calcular el promedio de una lista vacía `[]`. La longitud de la lista es 0, lo que provoca un intento de división por cero (`total / 0`).

**Código Corregido:**

```python
def calcular_promedio(numeros):
    if not numeros:  # Verificamos si la lista está vacía
        return 0     # O puedes retornar None o lanzar una excepción personalizada

    total = sum(numeros)
    cantidad = len(numeros)
    promedio = total / cantidad
    return promedio

print(calcular_promedio([])) # Retorna 0
```

**Mejor Práctica:**
Siempre valida las entradas de tus funciones, especialmente cuando realizas operaciones matemáticas como divisiones. Considera qué valor tiene sentido devolver (como `0`, `None`, o levantar un `ValueError` descriptivo) cuando los datos de entrada están vacíos.
