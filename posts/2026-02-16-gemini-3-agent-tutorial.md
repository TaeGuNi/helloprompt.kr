2026년, 이제 AI는 단순히 질문에 답하는 것을 넘어 도구를 사용하고 복잡한 작업을 수행하는 '에이전트'로 진화했습니다. 특히 **Gemini 3 Pro**의 강력한 추론(Reasoning) 능력과 방대한 컨텍스트 윈도우는 에이전트 개발의 진입 장벽을 획기적으로 낮췄습니다.

이 글에서는 파이썬과 Gemini 3 Pro를 사용하여 웹 검색을 수행하고 결과를 요약해주는 간단한 **'리서치 에이전트'**를 만들어보겠습니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **Gemini 3 Pro**는 추론 능력이 뛰어나 에이전트의 '두뇌'로 최적입니다.
2. **함수 호출(Function Calling)**을 통해 AI가 외부 도구(검색, 계산기 등)를 사용하게 만듭니다.
3. 잘 설계된 **시스템 프롬프트**가 에이전트의 성능을 좌우합니다.

---

## 🚀 단계별 가이드: 리서치 에이전트 만들기

### 1. 환경 설정 및 API 키 준비

먼저 `google-generativeai` 라이브러리를 설치합니다. (버전 0.9.0 이상 권장)

```bash
pip install -q -U google-generativeai
```

### 2. 도구(Tool) 정의하기

에이전트가 사용할 '손과 발'을 만듭니다. 여기서는 가상의 검색 함수를 정의합니다.

```python
import google.generativeai as genai

# 실제로는 Google Search API 등을 연결해야 합니다.
def search_web(query):
    """주어진 쿼리로 웹을 검색하고 요약된 결과를 반환합니다."""
    print(f"🔍 검색 중: {query}")
    return {"results": f"'{query}'에 대한 최신 2026년 검색 결과입니다..."}

tools = [search_web]
```

### 3. 모델 초기화 (The Brain)

Gemini 3 Pro 모델을 도구와 함께 초기화합니다.

```python
model = genai.GenerativeModel(
    model_name='gemini-3-pro-preview',
    tools=tools
)
chat = model.start_chat(enable_automatic_function_calling=True)
```

## 🧠 핵심: 에이전트 페르소나 프롬프트

에이전트에게 명확한 역할과 규칙을 부여하는 것이 가장 중요합니다. 아래 프롬프트를 시스템 인스트럭션(System Instruction)으로 사용하세요.

### 🥇 Pro Version (에이전트 시스템 프롬프트)

<!-- ⚠️ [Lint Rule] 인용구(>) 사용 -->

> **Role (역할):**
> 너는 세계 최고의 **'기술 트렌드 리서치 에이전트'**야.
> 사용자의 질문에 대해 웹 검색을 수행하고, 팩트에 기반한 인사이트를 제공하는 것이 너의 임무야.
>
> **Task (임무):**
>
> 1. 사용자의 질문을 분석하여 검색이 필요한 키워드를 추출해.
> 2. `search_web` 도구를 사용하여 정보를 수집해. 필요하면 여러 번 검색해.
> 3. 수집된 정보를 바탕으로 **출처가 명시된** 답변을 작성해.
>
> **Constraints (제약사항):**
>
> - 추측성 발언은 하지 마. 모르면 모른다고 해. (환각 방지)
> - 답변은 항상 명확한 마크다운(Markdown) 형식으로 작성해.
> - 한국어로 답변해.
>
> **Tone & Manner:**
>
> - 전문적이지만 친절하게.
> - 논리적인 구조를 갖춰서.

직접 Gemini 3 Pro로 에이전트를 구축해보니, 가장 큰 장점은 **'의도 파악 능력'**이었습니다. 기존 모델들은 도구를 언제 써야 할지 헷갈려하는 경우가 많았는데, Gemini 3는 "이 정보는 내가 모르니 검색해야겠다"는 판단이 매우 빠르고 정확합니다.

**팁:** 에이전트가 자꾸 딴소리를 한다면, 프롬프트의 `Task` 부분을 더 구체적으로 쪼개서 적어주세요. 단계별 사고(Chain of Thought)를 유도하면 성능이 훨씬 좋아집니다.
