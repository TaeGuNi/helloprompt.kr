# 📝 Resgate a Web Perdida: O Prompt Definitivo para Arquivamento Permanente

- **🎯 Público-Alvo:** Pesquisadores, Curadores de Conteúdo, Arquivistas Digitais, Profissionais de Marketing
- **⏱️ Tempo Estimado:** 1 hora → Reduzido para 1 minuto
- **🤖 Modelos Recomendados:** Modelos com ampla janela de contexto (Recomendado: Claude 3.5 Sonnet, Gemini 1.5 Pro)

- ⭐ **Dificuldade:** ⭐⭐⭐☆☆
- ⚡️ **Eficácia:** ⭐⭐⭐⭐⭐
- 🚀 **Versatilidade:** ⭐⭐⭐⭐⭐

> _"Capture a história da web antes que ela desapareça. Com um simples copiar e colar, transforme o caos de uma página dinâmica e desconfigurada em um documento Markdown imaculado e preservado para sempre."_

Com as Single Page Applications (SPA) e as plataformas fechadas (Walled Gardens) se tornando o padrão da web moderna, raspar e preservar páginas valiosas pelos métodos tradicionais tornou-se quase impossível. Links quebram da noite para o dia, e até mesmo sites de Web Archive muitas vezes falham em capturar telas renderizadas dinamicamente, deixando para trás apenas um espaço em branco.

Este prompt é uma ferramenta poderosa de preservação de conhecimento. Ao inserir um texto copiado brutalmente da tela ou um código HTML confuso, ele elimina todo o ruído desnecessário (anúncios, navegação, rodapés) e o **transforma em um documento Markdown perfeitamente estruturado, pronto para ser lido e utilizado em qualquer momento no futuro.**

---

## ⚡️ Resumo em 3 Tópicos (TL;DR)

1. **Remoção Inteligente de Ruído:** Filtra anúncios e elementos de UI do despejo de texto complexo de uma página web, extraindo com precisão apenas a essência do artigo principal.
2. **Conversão de Formato Permanente:** Reestrutura o documento para Markdown, o formato mais superior em termos de preservação e compatibilidade.
3. **Extração Automática de Metadados:** Gera automaticamente metadados cruciais para arquivamento e busca — como data de publicação, autor e palavras-chave — em formato YAML.

## 🚀 A Solução: "Prompt do Arquivista Digital"

### 🥉 Versão Básica (Basic Version)

Use esta versão quando precisar salvar rapidamente apenas o texto principal da página. Selecione toda a página (Ctrl+A), copie e cole junto com o prompt abaixo.

> **Role:** Você é um 'Arquivista Digital' especialista.
> **Task:** A partir do texto confuso da página web fornecido abaixo, remova todo o ruído (como menus e anúncios) e organize apenas o 'conteúdo principal' informativo de forma limpa usando Markdown.

<br>

### 🥇 Versão Pro (Professional Version)

Ideal para criar um documento de arquivo perfeito que pode ser salvo instantaneamente em suas ferramentas de gestão de conhecimento pessoal (Obsidian, Notion, etc.), sem perder o contexto original do site.

> **Role:** Você é um 'Arquivista Digital' de elite, dedicado a preservar a herança digital da humanidade. Sua missão principal é extrair a pura essência da informação de conteúdos web fragmentados e complexos, processando-os em um formato arquivável e permanente.
>
> **Context:**
>
> - Cenário: O arquivamento da web está se tornando cada vez mais difícil devido a páginas dinâmicas (SPA) e plataformas fechadas. Precisamos converter o texto original ou dumps de código-fonte diretamente para um formato de preservação permanente.
> - Objetivo: Remover ruídos desnecessários (anúncios, GNB, rodapés, comentários) dos dados de texto/HTML fornecidos e transformá-los em um documento Markdown perfeitamente estruturado.
>
> **Task:**
>
> 1. Analise os `[Dados da Página Web]` abaixo e extraia exclusivamente o artigo principal (Main Article).
> 2. Crie metadados de arquivamento no topo do documento usando o formato YAML Frontmatter (Obrigatório incluir: Título, Data de publicação estimada, Autor e 3 Palavras-chave principais).
> 3. Reorganize o corpo do texto usando uma estrutura lógica de cabeçalhos (`##`, `###`) e formate-o estritamente com a sintaxe Markdown.
> 4. Destaque visualmente citações importantes, dados cruciais e estatísticas usando blockquotes do Markdown (`>`) ou negrito (`**`).
>
> **Input Data:**
>
> ```text
> [Cole todo o texto da página web (Ctrl+A, Ctrl+C) ou o código-fonte HTML aqui]
> ```
>
> **Constraints:**
>
> - Não resuma nem distorça arbitrariamente as frases ou nuances originais do texto. (Mantenha 100% de fidelidade ao original, pois o objetivo é a preservação em arquivo).
> - O formato de saída deve seguir rigorosamente apenas a sintaxe Markdown pura, sem ser envolvido em blocos de código (code blocks) em sua resposta final.
>
> **Warning:**
>
> - Se não conseguir identificar claramente a data de publicação ou o autor nos dados fornecidos, não invente. Marque como `Unknown` para evitar alucinações.

## 💡 O Insight do Autor

Este prompt não é um mero 'resumidor de textos'. Em 2026, onde a velocidade de evaporação das informações na internet cresce exponencialmente, este é **um pipeline definitivo para trazer a propriedade da informação inteiramente para o seu ambiente local.**

Ele brilha especialmente em cenários onde os web clippers convencionais falham miseravelmente: newsletters pagas que exigem login, threads no X (Twitter) construídas com rolagem infinita, ou wikis corporativas internas. Basta selecionar a tela inteira (Ctrl+A, Ctrl+C) e jogar no prompt; como num passe de mágica, o lixo da interface desaparece, deixando apenas o dado original refinado. No meu fluxo de trabalho, utilizo esse método semanalmente para migrar dezenas de artigos, que de outra forma estariam em risco de sumir, diretamente e com perfeição para o meu Obsidian.

## 🙋 Perguntas Frequentes (FAQ)

- **P: Devo copiar o código-fonte HTML ou simplesmente o texto visível na tela?**
  - R: Ambos funcionam. No entanto, para as páginas modernas de renderização dinâmica, selecionar tudo (Ctrl+A) e copiar diretamente da tela do navegador é muito mais vantajoso. Isso reduz significativamente o ruído de scripts ocultos e entrega apenas o texto essencial para a IA.

- **P: O texto é muito longo e o modelo de IA está retornando um erro. O que eu faço?**
  - R: Copiar uma página web inteira pode causar um pico massivo na contagem de tokens. É altamente recomendável usar modelos que suportam um limite elevado de tokens de entrada (por exemplo, Claude 3.5 Sonnet ou Gemini 1.5 Pro).

## 🧬 Anatomia do Prompt (Por que funciona?)

1. **Imposição de Fidelidade ao Original (Constraints):** LLMs têm o reflexo condicionado de "resumir" ao se depararem com textos longos. A restrição estrita de "manter 100% de fidelidade ao original, pois o objetivo é a preservação" suprime esse instinto de resumo da IA e a força a focar em seu propósito real: arquivar.
2. **Extração Estruturada de Metadados:** Ao exigir um YAML Frontmatter, garantimos não apenas um texto agradável de ler, mas também alcançamos compatibilidade técnica imediata para integração com bancos de dados ou aplicativos de anotações baseados em Markdown.
