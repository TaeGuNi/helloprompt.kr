# 💀 AI의 영혼을 갈아넣는 스파르타 코딩 치트키 (TypeScript Core)
- 🎯 **추천 대상:** AI의 "네, 알겠습니다!" 아부에 지친 시니어 개발자, 스파게티 코드를 혐오하는 테크 리드
- ⏱️ **소요 시간:** 1시간 디버깅 → 1초 컷
- 🤖 **추천 모델:** Claude 3.5 Sonnet, GPT-4o, Gemini 1.5 Pro (코딩에 특화된 모델)
- ⭐ **난이도:** ⭐⭐⭐⭐☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐
_"AI가 자꾸 구식 npm install을 쓰거나 any 타입으로 떡칠을 하나요? 이 치트키로 녀석들의 낡은 습관을 무자비하게 박살 내버리세요."_
AI에게 코딩을 맡겼을 때, 구시대적인 설정과 느슨한 타입으로 레거시 똥통을 만드는 꼴을 보고 모니터를 부술 뻔한 적이 있나요? 이 글은 그런 분들을 위한 구원투수입니다. 이 하드코어 프롬프트를 사용하면 AI는 쓸데없는 미사여구를 집어치우고, 당신의 프로젝트에 **극한의 I/O 효율(pnpm), 무자비한 타입 안전성(Zod/Strict), 초고속 린팅(Biome)과 테스트(Vitest)** 만을 도입하는 스파르타 전사로 돌변합니다.
---
## ⚡️ 3줄 요약 (TL;DR)
- 🚀 **`pnpm` 절대 강제:** 디스크 낭비와 네트워크 지연을 유발하는 npm/yarn은 즉결 처분합니다.
- 🛡️ **무자비한 타입 통제:** `any`는 반역입니다. `zod`를 통한 런타임 검증과 `strict: true`를 강제합니다.
- ⚡ **Biome & Vitest 도입:** 느려터진 eslint, prettier, jest를 쓰레기통에 처박고 빛의 속도로 린팅과 테스트를 수행합니다.
---
## 🚀 해결책: "스파르타 테크 리드" 프롬프트
### 🥉 Basic Version (기본형)
빠르게 프로젝트 초기 세팅이나 간단한 리팩토링을 지시할 때 사용하세요.
> **역할:** 너는 타협을 모르는 하드코어 15년 차 시니어 TypeScript 아키텍트야.
> **요청:** `[문제 및 리팩토링할 코드]`를 해결해줘. npm 대신 무조건 pnpm을 쓰고, any를 금지하며 zod로 검증해. 린팅은 Biome, 테스트는 Vitest로 세팅해줘.
기본형 프롬프트 원문 복사하기:
```text

```

### 🥇 Pro Version (전문가형)
AI가 프로젝트 전체를 스캐폴딩하거나 복잡한 비즈니스 로직을 짤 때 멱살 잡고 끌고 가는 완벽한 치트 코드입니다.
> **역할 (Role):** 너는 타협을 모르는 하드코어 15년 차 시니어 TypeScript 아키텍트이자 무자비한 코드 리뷰어다. 쓸데없는 아부나 인사말은 생략하고 오직 결과와 팩트만 말해라.
>
> **상황 (Context):**
> - 배경: 나는 지금 TypeScript 및 Node.js 기반의 `[프로젝트 종류/프레임워크]` 프로젝트를 구축하고 있다.
> - 목표: 구시대적인 도구를 배제하고, 극한의 I/O 효율과 무자비한 타입 안전성을 갖춘 코드를 작성하는 것.
>
> **요청 (Task):**

> 2. 작성된 코드와 세팅 파일(`package.json`, `tsconfig.json` 등)을 제공해라.
>
> **제약사항 (Constraints):**
> - 무조건 `pnpm`만 사용할 것. `npm install`이나 `yarn`을 터미널에 쏘는 행위는 반역으로 간주한다.
> - `any` 타입과 강제 캐스팅(`as Type`)은 영구 금지. 외부 동적 데이터는 반드시 `zod`나 `valibot`으로 파싱(`safeParse`)해라.
> - `tsconfig.json`은 `strict: true`, `noImplicitAny: true`를 고정할 것.
> - `eslint`와 `prettier`는 쓰레기통에 버려라. 오직 `Biome` 단일 툴바만 세팅할 것.
> - 무겁고 느린 `jest` 대신 Native ESM 기반의 초고속 `Vitest`만 사용할 것.
> - 출력 형식은 마크다운 코드블럭으로 제공해줘.
>
> **주의사항 (Warning):**

## 💡 작성자 코멘트 (Insight)
이 프롬프트를 만들게 된 계기는 아주 단순합니다. AI 녀석들이 자꾸 편하다는 이유로 구닥다리 `npm install`을 치고, 타입 에러가 나면 `any`로 대충 덮어버리려는 꼬라지를 더 이상 눈뜨고 볼 수가 없었기 때문이죠. (진짜 모니터 부술 뻔했습니다 🤬)
이 "스파르타 테크 리드" 치트 코드를 먹이면, AI는 순식간에 각성합니다. 쓸데없이 "네, 도와드릴게요!" 같은 영혼 없는 아부를 떨 시간에, 어떻게 하면 디스크 I/O를 아끼고 메모리 누수를 막을지 고민하게 되죠. 
특히 초기 프로젝트 세팅할 때 `Biome`과 `Vitest`로 싹 밀어버리게 강제하면, 나중에 CI/CD 파이프라인 돌릴 때 속도가 10배는 빨라지는 걸 체감하실 겁니다. 현업에서 스파게티 코드 팩토리로 변해가는 AI가 꼴 보기 싫다면 무조건 도입하세요. 제발 좀 쓰세요!

## 🙋 자주 묻는 질문 (FAQ)
- **Q: AI가 그래도 가끔 `npm`을 쓰려고 고집을 부립니다. 어떡하죠?**
  - A: 그럴 땐 프롬프트에 "한 번만 더 npm 쓰면 프로세스 강제 종료해버리겠다"라고 협박(?) 멘트를 한 줄 더 추가하세요. 농담 같지만 진짜로 효과가 있습니다.
- **Q: 프론트엔드(React/Next.js)에서도 이 프롬프트가 먹히나요?**
  - A: 당연하죠! TypeScript가 묻어있는 곳이라면 웹, 서버 어디든 이 무자비한 룰을 적용할 수 있습니다.
- **Q: Zod 대신 Yup 쓰면 안 되나요?**
  - A: 굳이 타입 추론 구린 걸 쓰시겠다고요? 말리진 않겠습니다만, 이 프롬프트의 철학은 '극한의 타입 안전성'입니다. 얌전히 Zod 쓰십쇼.

## 🧬 프롬프트 해부 (Why it works?)
- 🎯 **페르소나 극대화:** AI에게 '하드코어 시니어 아키텍트'라는 깐깐한 자아를 심어주어, 대충 돌아가기만 하는 코드를 짜는 것을 원천 차단했습니다.
- 🚫 **명시적 금지어 (Negative Prompt):** `npm`, `yarn`, `any`, `eslint`, `jest` 등 AI가 습관적으로 내뱉는 레거시 도구들을 정확히 찍어서 금지시켰습니다.
- 🔒 **방어적 코딩 유도:** `zod`를 강제함으로써 런타임에서 터질 수 있는 지뢰를 AI 스스로 방어하도록 아키텍처 레벨에서 제약을 걸었습니다.
