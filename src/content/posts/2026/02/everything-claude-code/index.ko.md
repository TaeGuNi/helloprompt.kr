![\"Claude Code 200% 활용법: 4만 스타 `everything-claude-code` 설정 가이드\"](/images/hooks/everything-claude-code.jpg)

하루가 다르게 쏟아지는 AI 코딩 에이전트, 여러분은 100% 활용하고 계신가요? 혹시 아직도 챗봇에게 대화하듯 코드를 짜달라고 부탁하고 있다면, 이는 페라리를 타고 동네 마트에 가는 것과 다를 바 없습니다. 진정한 AI의 위력은 디테일한 **설정(Configuration)**과 체계적인 **워크플로우(Workflow)**에서 발휘됩니다.

오늘은 출시 한 달 만에 GitHub 스타 4만 개를 돌파하며 전 세계 개발자 커뮤니티를 강타한 **[`everything-claude-code`](https://github.com/affaan-m/everything-claude-code)** 리포지토리 활용법을 소개합니다. 이 설정 하나만 적용해도 여러분의 단순한 코딩 비서가 든든한 '시니어 엔지니어링 팀'으로 진화할 것입니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. `everything-claude-code`는 단순한 프롬프트 모음이 아닌, AI 에이전트를 위한 일종의 '운영 체제'입니다.
2. 기획, TDD, 코드 리뷰 등 13명의 전문 에이전트 페르소나를 강제 부여하여 압도적인 코드 품질을 보장합니다.
3. `/evolve` 명령어 하나면 AI가 프로젝트의 코딩 스타일을 스스로 학습하고 끊임없이 진화합니다.

## 🚀 해결책: "Everything Claude Code" 프롬프팅

### 🥉 Basic Version (빠른 설치 및 기본 기획)

터미널에서 단 2분 만에 시니어 에이전트를 영입해 보세요.

> **1. 플러그인 설치 (Claude Code 기준):**
> `/plugin marketplace add affaan-m/everything-claude-code`
> `/plugin install everything-claude-code@everything-claude-code`
>
> **2. 기획 프롬프트 실행:**
> `/plan "Next.js 14 App Router를 사용해서 반응형 블로그 포스트 목록 페이지를 만들어줘. Tailwind CSS와 shadcn/ui를 사용할 거야."`

### 🥇 Pro Version (TDD 및 커스텀 룰 강제)

우리 프로젝트에 완벽히 특화된 시니어 개발자의 워크플로우를 강제하고 싶을 때 활용하세요.

> **역할 (Role):** 너는 엄격한 TDD(테스트 주도 개발) 원칙을 준수하는 `[시니어 타입스크립트 개발자]`야.
>
> **상황 (Context):**
>
> - 우리는 `[everything-claude-code]`의 규칙을 따르고 있어.
> - 목표: `[사용자 인증 기능]`을 구현해야 해.
>
> **요청 (Task):**
>
> 1. 터미널에서 `/tdd` 명령어를 활성화해.
> 2. 실제 코드를 작성하기 전에 반드시 예외 케이스를 포함한 테스트 코드를 먼저 작성해.
> 3. 테스트 커버리지가 80%를 넘지 않으면 절대 다음 단계로 넘어가지 마.
> 4. `[추가 요구사항]`이 있다면 이를 완벽히 반영해서 코드를 작성해.
>
> **제약사항 (Constraints):**
>
> - 모든 변수와 함수에는 명시적인 Type Hint를 추가해.
> - `any` 타입 사용은 절대 금지야.
>
> **주의사항 (Warning):**
>
> - 기존 아키텍처나 코드 컨벤션을 위반하는 코드는 작성하지 마. 확신이 서지 않는다면 코드를 짜기 전에 내게 먼저 질문해. (환각 방지)

## 💡 작성자 코멘트 (Insight)

이 설정을 실무에 직접 적용해 보고 가장 소름이 돋았던 부분은 바로 **"AI가 개발자의 게으른 프롬프팅을 단호하게 거부한다"**는 점이었습니다.

예전처럼 "로그인 페이지 하나 붙여줘"라고 대충 지시하면, 에러 처리조차 안 된 엉성한 코드를 뱉어내곤 했죠. 하지만 이 설정을 적용하면 완전히 다릅니다. `/plan` 명령어를 입력하는 순간, AI가 먼저 _"어떤 인증 방식(OAuth, JWT)을 적용할까요? 에러 상태의 UI는 어떻게 처리하는 게 좋을까요?"_ 라며 역으로 날카로운 질문들을 쏟아냅니다. 마치 **기획서가 완벽하지 않으면 키보드에 손도 올리지 않는 깐깐한 수석 개발자**를 사수로 둔 기분마저 듭니다.

가장 감탄했던 기능은 단연 **`/evolve`** 명령어입니다. 프로젝트가 진행될수록 AI가 우리 팀 고유의 코딩 컨벤션(예: "우리 팀은 인터페이스 이름 앞에 'I'를 붙이지 않는다")을 스스로 학습하고, 이를 확고한 규칙으로 정립해 나갑니다. 스파게티 코드를 수습하느라 지친 분들이나, 날카로운 시니어의 코드 리뷰가 간절했던 1인 개발자에게 이 워크플로우는 더 이상 선택이 아닌 필수입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: Claude Code CLI 환경에서만 사용할 수 있나요?**
  - A: 아닙니다! Cursor 에디터의 `.cursorrules` 파일에 적용하거나, OpenCode 환경에서도 완벽하게 호환되어 작동합니다.

- **Q: 회사 내부의 보안이 중요한 프로젝트에 사용해도 괜찮을까요?**
  - A: 룰(Rule) 파일 자체는 오픈소스이며 사용자의 로컬 환경에 저장되므로 안전합니다. 다만, 작성한 코드를 LLM 모델로 전송하는 행위 자체는 사내 보안 규정(Enterprise 플랜 도입 여부 등)을 사전에 반드시 확인하셔야 합니다.

- **Q: 초보 개발자가 쓰기에는 설정이 너무 엄격하지 않나요?**
  - A: 처음에는 AI가 끊임없이 질문을 던지고 테스트 코드를 강제해서 다소 번거롭게 느껴질 수 있습니다. 하지만 이 깐깐한 과정을 거쳐야만 훗날 감당할 수 없는 기술 부채(Technical Debt)를 갚느라 밤을 새우는 대참사를 막을 수 있습니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **강제된 워크플로우 (Forced Workflow):** 단순한 일방적 지시를 넘어, `/plan`, `/tdd` 등의 슬래시 명령어를 통해 AI가 체계적인 프로세스(요구사항 분석 → 테스트 코드 작성 → 본 코드 구현)를 밟도록 강력하게 유도합니다.
2. **페르소나 분리 (Persona Separation):** 코딩하는 자아(Developer), 리뷰하는 자아(Reviewer), 기획하는 자아(Planner)를 철저히 분리하여, AI가 스스로 작성한 코드의 객관성을 교차 검증하도록 만듭니다.
3. **지속적 학습 (Instincts):** 대화 맥락이 금방 휘발되어 버리는 기존 챗봇과 달리, AI가 프로젝트 전반을 관통하며 사용자의 코드 수정 패턴을 파악하고 이를 `.rules` 파일에 내재화하여 스스로 진화합니다.

## 📊 증명: Before & After

### ❌ Before (일반적인 AI 코딩)

```text
User: "게시판 목록 만들어줘"
AI: (묻지도 따지지도 않고 바로 React 코드를 길게 출력함. 페이지네이션 없음, 에러 처리 누락, 프로젝트 디자인 시스템 무시)
```

### ✅ After (everything-claude-code 적용 후)

```text
User: "> /plan 게시판 목록 만들어줘"
AI: Planner 에이전트가 가동되었습니다. 완벽한 구현을 위해 다음 3가지를 먼저 확정해야 합니다.
1. 페이지당 몇 개의 게시물을 보여줄까요? (무한 스크롤 vs 페이지네이션)
2. 데이터 패칭은 서버 사이드(SSR)로 할까요, 클라이언트 사이드(CSR)로 할까요?
3. 데이터 로딩 시 보여줄 Skeleton UI 디자인이 필요합니까?

답변해 주시면 TDD 에이전트에게 테스트 코드 작성을 지시하겠습니다.
```
