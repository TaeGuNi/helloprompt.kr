## 📝 宇宙語のような正規表現(Regex)、日本語で注文してください

- **🎯 おすすめの対象:** ジュニア開発者、データアナリスト、マーケター
- **⏱️ 所要時間:** 30分 → 1分に短縮
- **🤖 おすすめのモデル:** すべての対話型AI (ChatGPT, Claude, Geminiなど)

- ⭐ **難易度:** ⭐⭐☆☆☆
- ⚡️ **効果性:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「正規表現を書くたびにStack Overflowをさまよい、暗号解読のような記号の羅列に頭を抱えていませんか？」_

正規表現（Regex）は、文字列の検索やバリデーションにおいて避けては通れない強力なツールです。しかし、その構文はまるで宇宙人が残した暗号のようであり、自分が数日前に苦労して書いたコードでさえ、後から見直すと全く解読できなくなることが日常茶飯事です。「このエスケープ文字はここで合っているのか？」「括弧のネストがどこで終わるのか分からない…」と、チートシートと睨めっこしながら何時間も画面の前でフリーズした経験は誰にでもあるでしょう。

もうStack Overflowの海を彷徨い、意味不明な記号の羅列をコピペして祈るようなテストを繰り返す必要はありません。これからは、AIに向かって「どんなパターンの文字列を、どうやって見つけたいのか」を自然な人間の言葉で説明するだけで十分です。プロンプト一つで、AIがあなたの専属アシ তৎপরとして、完璧な正規表現コードとその詳細な動作原理を一瞬で生成し、実務のフラストレーションを完全に消し去ってくれます。

---

## ⚡️ 3行まとめ (TL;DR)

1. 難解な正規表現のコーディングはAIに任せ、人間は要件となる「パターンのルール定義」にのみ集中する。
2. 完成した正規表現だけでなく、各記号がどのように機能するのか「ステップ別の解説」も同時に要求する。
3. 想定外のエッジケース（例外状況）に対するテストケースも出力させ、プロダクション環境での確実性を検証する。

## 🚀 解決策: "Regexジェネレーター・プロンプト"

### 🥉 Basic Version (基本型)

素早く正規表現のコードだけが必要な状況で使用してください。

> **役割:** あなたはシニア開発者です。
>
> **要求:** `[メールアドレス]`を検証するための正規表現を作成してください。

### 🥇 Pro Version (専門家型)

特定のプログラミング言語に合わせた最適な正規表現に加え、詳細なコード解説やテストケースまで包括的に取得したい場合に使用してください。

> **役割 (Role):** あなたは正規表現(Regex)のマスターであり、10年の経験を持つシニアソフトウェアエンジニアです。
>
> **状況 (Context):**
>
> - 背景: ユーザーの入力データから特定のパターンを抽出、または検証する機能を作成しています。
> - 目標: バグのない安全で効率的な正規表現を作成し、その動作原理を正確に理解すること。
>
> **要求 (Task):**
>
> 1. 次の条件を満たす正規表現を作成してください: `[条件: 日本の携帯電話番号 (例: 090-1234-5678, 08012345678 の両方に対応)]`
> 2. `[使用言語: JavaScript]` でそのまま使用できるコードスニペットを提供してください。
> 3. 提供した正規表現の各パーツ（記号）が何を意味するのか、ステップバイステップで解説してください。
> 4. この正規表現がマッチする成功例と、マッチしない失敗例（エッジケース）をそれぞれ3つ以上提示してください。
>
> **制約事項 (Constraints):**
>
> - 出力形式は読みやすいマークダウンを使用してください。
> - コードには簡潔なコメントを含めてください。
> - ReDoS（正規表現サービス拒否攻撃）の脆弱性がない安全なパターンを提示してください。
>
> **注意事項 (Warning):**
>
> - 与えられた条件で処理しきれない極端なエッジケースがある場合は、その限界点も合わせて説明してください。

## 💡 作成者コメント (Insight)

正規表現において最も厄介なのは、「書く」こと以上に**「読む」こと**、そして**「検証する」こと**です。単にAIへ正規表現の生成を丸投げするのではなく、その**動作原理の解説**と**テストケース**を同時に要求することが、このプロンプトの真の価値です。

実際の業務では、ユーザーの予期せぬ入力（エッジケース）によって致命的なバグが引き起こされるケースが後を絶ちません。AIに自ら失敗例（マッチしないパターン）を挙げさせることで、エンジニアは事前にコードの脆弱性を防ぐことができます。さらに、システムをダウンさせかねない「ReDoS（正規表現を使ったDoS攻撃）」への対策を**制約事項（Constraints）**に組み込むことで、プロダクション環境にも安心してデプロイできる、堅牢でエンタープライズ品質のコードを獲得できるのです。

## 🙋 よくある質問 (FAQ)

- **Q: 特定のプログラミング言語（例えばPythonやGo）でも使えますか？**
  - A: はい、完全に互換性があります。プロンプト内の`[使用言語]`の変数を、現在開発で使用している言語（例：Python、Go、Javaなど）に変更するだけで、その言語固有の正規表現モジュール（Pythonの`re`パッケージなど）に最適化されたコードが生成されます。

- **Q: 既存の複雑な正規表現の意味を解読してもらうこともできますか？**
  - A: もちろんです。「状況（Context）」を「この正規表現の意図を解読したい」に変更し、「要求（Task）」の項目に解読したい正規表現の文字列をそのまま貼り付けてみてください。AIが各記号の役割を人間が理解できる言葉で丁寧に分解して解説してくれます。

## 🧬 プロンプト解剖 (Why it works?)

1. **Role（役割の付与）:** AIに「正規表現マスター」というペルソナを与えることで、単なる文字列検索の域を超え、処理パフォーマンスとシステム安全性を考慮したエキスパートレベルの回答を引き出しています。
2. **Task（テストと解説の要求）:** コードを出力させて終わりにせず、「解説」と「成功/失敗例」を明示的に要求しています。これにより、AI特有のハルシネーション（もっともらしい嘘）をセルフチェックさせ、同時に開発者自身のコードに対する理解度を大幅に深めることができます。
3. **Constraints（厳格な制約）:** 「ReDoS脆弱性の排除」をプロンプトに明記することで、重大なセキュリティインシデントのリスクを未然に防ぎ、実務レベルでそのままコピペ可能な高いコード品質を担保しています。

## 📊 証明: Before & After

### ❌ Before (これまでのやり方)

```javascript
// Stack Overflowからコピーしてきた、何をしているか分からないコード
const phoneRegex =
  /^(0[5-9]0[-(]?[0-9]{4}[-)]?[0-9]{4}|0120[-]?\d{1,3}[-]?\d{4})$/;
// 適用後、一部の正しい番号が弾かれたりするが、修正方法が分からない...
```

### ✅ After (プロンプト適用後の結果)

```javascript
/**
 * 日本の携帯電話番号を検証する正規表現
 * - 090, 080, 070から始まる11桁の番号
 * - ハイフンの有無（090-1234-5678 または 09012345678）の両方に対応
 */
const mobilePhoneRegex = /^0[789]0-?\d{4}-?\d{4}$/;

function isValidMobilePhone(number) {
  return mobilePhoneRegex.test(number);
}

// テストケース
console.log(isValidMobilePhone("090-1234-5678")); // true (成功)
console.log(isValidMobilePhone("08012345678")); // true (成功)
console.log(isValidMobilePhone("06-1234-5678")); // false (固定電話なので失敗)
console.log(isValidMobilePhone("090-123-5678")); // false (桁数不足により失敗)
```

_(AIはこれに加えて、`^`は行頭、`\d{4}`は数字4桁、などの詳細な解説も提供してくれます)_
