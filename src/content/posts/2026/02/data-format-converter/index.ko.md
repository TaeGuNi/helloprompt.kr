## 📝 엑셀, JSON, CSV... 데이터 포맷 변환 노가다 탈출

- **🎯 추천 대상:** 개발자, 데이터 분석가, 퍼포먼스 마케터
- **⏱️ 소요 시간:** 30분 → 1분 단축
- **🤖 추천 모델:** Claude 3.5 Sonnet (복잡한 구조 변환에 탁월함), ChatGPT (GPT-4o)

- ⭐ **난이도:** ⭐☆☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"대리님, 이 엑셀 데이터 웹에 올려야 하는데 JSON으로 좀 바꿔주실 수 있나요? 아, 그리고 키(Key) 값은 전부 영문 소문자 스네이크 케이스로 통일 부탁드려요."_

![엑셀, JSON, CSV... 데이터 포맷 변환 노가다 탈출](/images/hooks/data-format-converter.jpg)

데이터를 다루는 실무자라면 누구나 '데이터 포맷 변환'이라는 지독한 단순 반복 노동과 필연적으로 마주합니다. 
온라인에 널려 있는 무료 변환 툴을 쓰자니 사내 보안 규정 위반이 찜찜하고, 파이썬(Python) 스크립트나 복잡한 정규표현식을 직접 짜자니 '고작 이 작업 하나 때문에 코딩까지 해야 하나' 싶어 깊은 무력감이 밀려오죠. 
하지만 이제 더 이상 엑셀 셀을 붙잡고 씨름할 필요가 없습니다. 원본 데이터를 있는 그대로 복사해서 AI에게 던져주기만 하세요. 단순한 포맷 변경을 넘어, 까다로운 데이터 구조 매핑, 필드명 일괄 정규화, 불필요한 더미 데이터 제거까지 단 한 번의 프롬프트로 완벽하게 클렌징(Cleansing)해 드립니다.

## ⚡️ 3줄 요약 (TL;DR)

1. **보안 리스크 원천 차단:** 엔터프라이즈 AI나 사내망 모델을 활용하여, 외부 툴 데이터 유출 걱정 없이 안전하게 변환할 수 있습니다.
2. **복합 정제 파이프라인:** 단순한 포맷 변경(CSV → JSON 등)을 넘어, 복잡한 키(Key) 값 매핑과 데이터 타입(Type) 캐스팅까지 한 번에 처리합니다.
3. **즉시 실행 가능한 코드 출력:** 직접적인 데이터 변환뿐만 아니라 DB 마이그레이션을 위한 SQL Insert 쿼리문이나 파이썬 데이터프레임 스크립트로도 즉각 생성할 수 있습니다.

## 🚀 해결책: "만능 데이터 컨버터"

### 🥉 Basic Version (기본형)

복잡한 조건 없이, 빠르고 직관적으로 데이터 포맷만 변경하고 싶을 때 사용하세요.

> **역할:** 너는 시니어 데이터 엔지니어 역할을 수행해.
> **요청:** 아래 데이터를 `[변환할 타겟 포맷 (예: JSON)]` 포맷으로 변환해 줘. 다른 부연 설명은 일절 생략하고 코드블럭으로 최종 결과물만 깔끔하게 출력해.
>
> `[여기에 변환할 원본 데이터를 붙여넣으세요]`

### 🥇 Pro Version (전문가형)

데이터 클렌징, 키(Key) 값 정규화, 특정 필드 마스킹 등 디테일하고 정교한 가공 파이프라인이 필요할 때 활용하세요.

> **역할 (Role):** 너는 10년 차 데이터 엔지니어이자 ETL 파이프라인 설계 전문가야.
>
> **상황 (Context):**
>
> - 배경: 제공된 소스 데이터를 타겟 시스템(DB, API 등)의 규격에 맞게 변환하고 완벽하게 정제해야 해.
> - 목표: 주어진 조건에 한 치의 오차도 없이 부합하는 무결점 데이터를 산출하는 것.
>
> **요청 (Task):**
>
> 1. 제공된 **[소스 데이터]**를 심층 분석하여 **[타겟 포맷]**으로 변환해.
> 2. 변환 과정에서 반드시 아래의 **[변환 규칙]**을 엄격하게 적용해.
> 3. 코드블럭 안에 변환이 완료된 최종 데이터만 깔끔하게 출력해.
>
> **입력 변수 (Variables):**
>
> - [소스 포맷]: `[변환 전 원본 데이터 포맷 (예: CSV)]`
> - [타겟 포맷]: `[변환 후 타겟 데이터 포맷 (예: JSON Array)]`
> - [변환 규칙]:
>   1. `[적용할 규칙 1 (예: 한글 헤더명을 영문 스네이크 케이스(snake_case)로 변경)]`
>   2. `[적용할 규칙 2 (예: '나이' 필드는 숫자형(Integer)으로 타입 캐스팅)]`
>   3. `[적용할 규칙 3 (예: '개인정보_주민번호' 필드는 보안을 위해 결과물에서 완전히 제외할 것)]`
>
> **제약사항 (Constraints):**
>
> - 데이터를 임의로 누락하거나 존재하지 않는 값을 절대 지어내지 마.
> - 변환 프로세스에 대한 설명이나 너의 인사이트는 일절 출력하지 말고, 오직 최종 데이터만 코드블럭으로 제시해.
>
> **소스 데이터:**
>
> `[여기에 변환할 원본 데이터를 붙여넣으세요]`

## 💡 작성자 코멘트 (Insight)

이 프롬프트의 진정한 가치는 **'데이터 정제(Data Cleansing)'와 '포맷 변환'을 단일 파이프라인으로 완벽하게 통합**했다는 점에 있습니다.
실무 현장에서는 단순히 CSV를 JSON으로 바꾸는 작업 자체보다, 프론트엔드 API 명세서 규격에 맞춰 키(Key) 값을 카멜 케이스(camelCase)로 일괄 수정하거나 중구난방인 날짜 포맷(`YYYY-MM-DD`)을 일관되게 통일하는 전처리 과정이 훨씬 고통스럽고 긴 시간을 요구합니다.
Pro 버전의 `[변환 규칙]` 항목에 정규표현식으로도 구현하기 까다로운 엣지 케이스(예: *"빈 값은 null로 처리하고, 직업란이 비어있는 사람은 '무직'으로 통일해"*)를 자연어로 가볍게 던져보세요. 소름 돋을 정도의 정확도로 데이터를 입맛에 맞게 가공해 줍니다. 특히 Claude 3.5 Sonnet 모델은 뎁스가 깊고 복잡한 JSON 중첩 구조(Nested Object)를 파싱하고 재구성하는 데 있어 압도적인 퍼포먼스를 보여줍니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 엑셀 파일(.xlsx)을 직접 첨부해서 변환해도 되나요?**
  - A: Advanced Data Analysis 기능이 탑재된 ChatGPT(GPT-4o)나 Claude에서는 엑셀 파일을 업로드해도 완벽하게 분석하고 변환해 냅니다. 다만, 사내 보안에 민감한 데이터라면 파일 전체를 첨부하기보다는 필요한 내용 일부만 복사하여 프롬프트에 붙여넣고, 핵심 개인정보는 사전에 마스킹(Masking) 처리하는 방식을 강력히 권장합니다.

- **Q: 변환해야 할 데이터가 수만 줄이 넘어가면 어떻게 처리하나요?**
  - A: 데이터 볼륨이 너무 크면 대형 언어 모델(LLM)의 컨텍스트 윈도우(토큰 한도)를 초과하여 출력이 중간에 끊길 수밖에 없습니다. 이럴 때는 프롬프트의 목적을 살짝 비틀어 *"이 변환 규칙을 완벽하게 수행하는 파이썬(Python) 코드를 작성해 줘"*라고 우회하여 요청하세요. AI가 짜준 스크립트를 로컬 PC에서 직접 실행하는 것이 대용량 데이터 처리의 가장 안전하고 확실한 정석입니다.

- **Q: 결과물이 다 출력되지 않고 중간에 뚝 끊겨서 나옵니다.**
  - A: 대화창에 *"계속해 줘"*라고 입력하면 끊긴 지점부터 이어서 출력해 줍니다. 애초에 프롬프트 하단 제약사항(Constraints) 부분에 *"결과물이 길더라도 임의로 생략하지 말고, 나누어서라도 끝까지 출력해 줘"*라는 조건을 명시해 두는 것도 실무에서 유용하게 쓰이는 팁입니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **ETL 전문가 페르소나 부여:** AI에게 단순한 '포맷 변환기'가 아닌 '시니어 데이터 엔지니어'의 자아를 심어줌으로써, 스스로 데이터의 무결성(Integrity)과 데이터 타입(Type)을 꼼꼼하게 교차 검증하도록 유도했습니다.
2. **명확한 파라미터 분리 구조:** 입력(소스 데이터), 출력(타겟 포맷), 그리고 제어 규칙(변환 룰)을 마크다운 문법으로 철저하게 블록화했습니다. 덕분에 AI가 복잡다단한 지시사항 앞에서도 혼동 없이 순차적이고 논리적으로 파이프라인을 실행합니다.
3. **엄격한 출력 통제 (Constraints):** *"부연 설명은 생략하고 코드블럭만 출력해"*라는 강력한 제약을 걸었습니다. 사용자가 변환 완료된 데이터를 즉시 '복사-붙여넣기'하여 실무에 곧바로 적용할 수 있도록 압도적인 UX(사용자 경험)를 제공합니다.

## 📊 증명: Before & After

단순히 "이거 바꿔줘"라고 말하는 것과 정교하게 엔지니어링 된 프롬프트를 사용하는 것의 결과물 차이는 말 그대로 하늘과 땅 차이입니다. 이 프롬프트를 활용하면 데이터 파이프라인 수준의 정형화된 가공이 가능해져, 소모적인 반복 노가다에 버려지던 귀중한 시간을 극적으로 구원할 수 있습니다.

### ❌ Before (입력 데이터 - CSV 형태의 지저분한 텍스트)

```csv
고객 이름,나이,가입 일자,직업,개인정보_주민번호
홍길동,삼십,2023/01/05,백엔드 개발자,900101-1234567
김철수,25,23-11-12,,990203-2345678
```

### ✅ After (결과 - 정제된 타겟 JSON Array)

**요청한 규칙:** 한글 헤더명을 영문 스네이크 케이스로 변경, 나이는 숫자형(Integer)으로 치환, 가입일자는 `YYYY-MM-DD` 표준 포맷으로 일괄 통일, 직업이 비어있을 경우 '미상'으로 예외 처리, 민감한 주민번호 필드 완전 삭제.

```json
[
  {
    "customer_name": "홍길동",
    "age": 30,
    "join_date": "2023-01-05",
    "job": "백엔드 개발자"
  },
  {
    "customer_name": "김철수",
    "age": 25,
    "join_date": "2023-11-12",
    "job": "미상"
  }
]
```
