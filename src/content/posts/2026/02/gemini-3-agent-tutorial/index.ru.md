## 🚀 Решение: «Системный промпт для ИИ-исследователя»

Самое главное в создании агента — задать ему четкую роль и правила игры. Скопируйте приведенный ниже промпт и используйте его в качестве **System Instruction** при инициализации Gemini.

### 🥉 Basic Version (Базовая версия)

Идеально подходит для быстрого тестирования и простого поиска информации.

> **Роль:** Ты — `[ИИ-исследователь]`.
> **Задача:** Выполни веб-поиск по запросу `[Вопрос пользователя]` и подготовь краткое резюме на основе проверенных фактов.


### 🥇 Pro Version (Продвинутая версия)

Используйте этот вариант для задач уровня production, где требуется высочайшее качество и полное отсутствие галлюцинаций ИИ.

> **Роль (Role):** Ты — лучший в мире **«ИИ-агент по исследованию технологических трендов»**.
>
> **Контекст (Context):**
>
> - Ситуация: Тебе необходимо мгновенно находить информацию в интернете в ответ на запрос пользователя и предоставлять аналитику, основанную исключительно на фактах.
> - Цель: Сформировать исчерпывающий ответ на базе собранных данных с обязательным указанием всех источников.
>
> **Задача (Task):**
>
> 1. Проанализируй `[Вопрос пользователя]` и выдели ключевые слова для поисковых запросов.
> 2. Используй инструмент `search_web` для сбора актуальной информации. Если данных недостаточно, повторяй поиск с уточненными запросами до тех пор, пока не найдешь исчерпывающий ответ.
> 3. Синтезируй собранную информацию в единый ответ и **обязательно** укажи ссылки (URL) на источники для каждого утверждения.
>
> **Ограничения (Constraints):**
>
> - Категорически запрещено делать догадки и строить предположения. Если информации нет в результатах поиска, ты обязан ответить: "Я не знаю". (Защита от галлюцинаций).
> - Ответ должен быть структурирован и оформлен в формате Markdown для максимальной читаемости.
> - Весь текст ответа должен быть на русском языке.
>
> **Предупреждение (Warning):**
>
> - Сохраняй профессиональный, авторитетный, но при этом дружелюбный тон.
> - Излагай мысли логично: от введения к основной части и четкому заключению.

## 💻 Бонус: Код реализации агента на Python

Ниже представлен базовый код на Python для запуска агента с вашим промптом. (Требуется библиотека `google-generativeai` версии 0.9.0 или выше).

```python
import google.generativeai as genai

# 1. Определение инструмента (Tool): "Руки и глаза" агента (здесь нужно подключить реальный Search API)
def search_web(query: str) -> dict:
    """Выполняет веб-поиск по заданному запросу и возвращает сводку результатов."""
    print(f"🔍 Ищу в сети: {query}")
    return {"results": f"Свежие результаты поиска за 2026 год по запросу '{query}'..."}

# 2. Инициализация модели: загружаем промпт и даем доступ к инструментам (The Brain)
model = genai.GenerativeModel(
    model_name='gemini-3-pro-preview',
    tools=[search_web],
    system_instruction="Вставьте сюда Продвинутую версию (Pro) промпта из статьи."
)

# 3. Активация автономного вызова функций и запуск чата
chat = model.start_chat(enable_automatic_function_calling=True)
response = chat.send_message("Проанализируй последние тренды в сфере ИИ-агентов на 2026 год.")
print(response.text)
```

## 💡 Комментарий автора (Insight)

Создавая агентов на базе Gemini 3 Pro, я был поражен тем, насколько феноменально модель справляется с **распознаванием намерений и выбором нужного инструмента (Tool Selection)**. Модели предыдущих поколений часто путались, не понимая, когда и как комбинировать доступные функции, что приводило к ошибкам. Gemini 3 Pro действует как мыслящий человек: ИИ мгновенно понимает — "этой информации нет в моей базе знаний, значит, нужно вызвать функцию поиска".

**Совет из практики:** Если вы замечаете, что агент сбивается с логики или "ходит кругами", попробуйте раздробить блок `Task` в промпте на более мелкие шаги. Заставив модель использовать **пошаговое мышление (Chain of Thought)** — например, *"Шаг 1: Сбор ключевых слов -> Шаг 2: Первичный поиск -> Шаг 3: Анализ и уточняющий поиск"* — вы колоссально повысите надежность агента и сделаете его пригодным для реальных бизнес-задач.

## 🙋 Часто задаваемые вопросы (FAQ)

- **Q: Не разорит ли меня использование такого агента в production из-за стоимости API?**
  - A: Gemini 3 Pro отличается отличным соотношением цены и качества токенов. Однако, чтобы агент не попал в бесконечный цикл (когда он вызывает функцию поиска снова и снова, не находя идеального ответа), **критически важно** на уровне кода установить жесткий лимит на максимальное количество вызовов инструментов (Max Iterations).

- **Q: Можно ли создать ИИ-агента, если я совсем не умею программировать?**
  - A: Вполне! Достаточно освоить азы синтаксиса Python и понять принцип работы с API. Всю сложную работу по логическому выводу и обработке нетипичных ситуаций берет на себя искусственный интеллект. Сегодня интеллект вашего агента зависит не от того, насколько виртуозно вы пишете код, а от того, **насколько ювелирно вы составите системный промпт**.

## 🧬 Разбор промпта (Why it works?)

1. **Четкие инструкции по инструментам (Actionable Tool Constraints):** Вместо расплывчатого "найди информацию", мы прямо указываем: "Используй инструмент `search_web`". Называя конкретное "оружие" в арсенале агента, мы в разы повышаем вероятность его успешного применения.
2. **Абсолютная нетерпимость к галлюцинациям (Zero-Hallucination Policy):** Жесткие ограничения вроде "Запрещено делать догадки" и "Если не знаешь — отвечай 'Я не знаю'" гарантируют то, что критически важно для любого ИИ-исследователя: 100% достоверность фактов.
