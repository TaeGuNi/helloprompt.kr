# 💀 Cheat Code Espartano para Exprimir el Alma de la IA (TypeScript Core)

- 🎯 **Público objetivo:** Desarrolladores Senior hartos del "¡Sí, por supuesto!" complaciente de la IA, Tech Leads que odian el código espagueti.
- ⏱️ **Tiempo estimado:** 1 hora de debugging → Resuelto en 1 segundo.
- 🤖 **Modelos recomendados:** Claude 3.5 Sonnet, GPT-4o, Gemini 1.5 Pro (Modelos especializados en código).
- ⭐ **Dificultad:** ⭐⭐⭐⭐☆
- ⚡️ **Efectividad:** ⭐⭐⭐⭐⭐
- 🚀 **Utilidad:** ⭐⭐⭐⭐⭐

_"¿La IA sigue usando el obsoleto npm install o llenando todo con el tipo any? Destruye sus viejos hábitos sin piedad con este cheat code."_

¿Alguna vez has estado a punto de romper la pantalla al ver cómo la IA, cuando le encargas programar, crea un pozo de código legacy con configuraciones anticuadas y tipado débil? Este artículo es el salvavidas para ustedes. Al usar este prompt hardcore, la IA dejará a un lado la retórica inútil y se transformará en un guerrero espartano que solo introducirá en tu proyecto **una eficiencia de I/O extrema (pnpm), una seguridad de tipos despiadada (Zod/Strict), y un linting y testing ultrarrápidos (Biome y Vitest)**.

---

## ⚡️ Resumen en 3 líneas (TL;DR)
- 🚀 **Uso obligatorio de `pnpm`:** Se ejecutará sumariamente a npm/yarn, causantes de desperdicio de disco y latencia de red.
- 🛡️ **Control de tipos despiadado:** Usar `any` es traición. Se fuerza la validación en tiempo de ejecución con `zod` y el uso de `strict: true`.
- ⚡ **Adopción de Biome y Vitest:** Tira a la basura los lentos eslint, prettier y jest, y ejecuta el linting y las pruebas a la velocidad de la luz.

---

## 🚀 Solución: Prompt "Tech Lead Espartano"

### 🥉 Versión Básica (Basic Version)
Úsala para ordenar configuraciones iniciales rápidas o refactorizaciones sencillas de un proyecto.

> **Rol:** Eres un Arquitecto Senior de TypeScript hardcore con 15 años de experiencia que no conoce concesiones.
> **Tarea:** Resuelve `[Problema y código a refactorizar]`. Usa obligatoriamente pnpm en lugar de npm, prohíbe el uso de any y valida con zod. Configura Biome para el linting y Vitest para las pruebas.

### 🥇 Versión Profesional (Pro Version)
Este es el cheat code perfecto para arrastrar y forzar a la IA a hacer las cosas bien cuando hace el scaffolding de un proyecto completo o diseña lógica de negocio compleja.

> **Rol (Role):** Eres un Arquitecto Senior de TypeScript hardcore con 15 años de experiencia y un revisor de código despiadado. Omite los saludos o adulaciones inútiles y dime solo los resultados y los hechos.
>
> **Contexto (Context):**
> - Fondo: Estoy construyendo un proyecto basado en TypeScript y Node.js con `[Tipo de proyecto/Framework]`.
> - Objetivo: Descartar herramientas obsoletas y escribir código con una eficiencia de I/O extrema y una seguridad de tipos implacable.
>
> **Tarea (Task):**
> 1. Escribe los detalles de `[Función o módulo a implementar]`.
> 2. Proporciona el código escrito y los archivos de configuración (`package.json`, `tsconfig.json`, etc.).
>
> **Restricciones (Constraints):**
> - Usa obligatoriamente solo `pnpm`. Lanzar `npm install` o `yarn` en la terminal se considerará traición.
> - El tipo `any` y el casting forzado (`as Type`) están prohibidos permanentemente. Los datos dinámicos externos deben ser parseados (`safeParse`) obligatoriamente con `zod` o `valibot`.
> - En `tsconfig.json`, debes fijar `strict: true` y `noImplicitAny: true`.
> - Tira `eslint` y `prettier` a la basura. Configura únicamente `Biome` como herramienta unificada.
> - En lugar del pesado y lento `jest`, usa exclusivamente `Vitest`, que es ultrarrápido y está basado en Native ESM.
> - Proporciona el formato de salida en bloques de código Markdown.
>
> **Advertencia (Warning):**
> - Está absolutamente prohibido recomendar paquetes basados en sintaxis legacy incierta o alucinaciones (Hallucination). Si no lo sabes, di que no lo sabes.

## 💡 Comentarios del Autor (Insight)

Al darle este cheat code de "Tech Lead Espartano", la IA despierta instantáneamente. En lugar de perder el tiempo con adulaciones vacías como "¡Sí, te ayudaré!", se centrará en pensar cómo ahorrar I/O de disco y prevenir fugas de memoria.
Especialmente al configurar un proyecto inicial, si la fuerzas a arrasar con todo usando `Biome` y `Vitest`, notarás que la velocidad se multiplica por 10 cuando ejecutes tu pipeline CI/CD más adelante. Si estás harto de que la IA se convierta en una fábrica de código espagueti en tu trabajo diario, impleméntalo sin dudar. ¡Por favor, úsalo!

## 🙋 Preguntas Frecuentes (FAQ)
- **P: Aun así, a veces la IA se empeña en usar `npm`. ¿Qué hago?**
  - R: En ese caso, añade una línea amenazadora en el prompt: "Si usas npm una vez más, forzaré el cierre del proceso". Parece una broma, pero realmente funciona.
- **P: ¿Este prompt también funciona en el Frontend (React/Next.js)?**
  - R: ¡Por supuesto! Dondequiera que haya TypeScript involucrado, ya sea en web o servidor, puedes aplicar esta regla despiadada.
- **P: ¿No puedo usar Yup en lugar de Zod?**
  - R: ¿En serio quieres usar algo con peor inferencia de tipos? No te detendré, pero la filosofía de este prompt es la "seguridad de tipos extrema". Sé buen chico y usa Zod.

## 🧬 Anatomía del Prompt (Why it works?)
- 🎯 **Maximización del Persona:** Al infundir a la IA el ego exigente de un "Arquitecto Senior Hardcore", bloqueamos de raíz la creación de código que solo funciona a medias.
- 🚫 **Palabras prohibidas explícitas (Negative Prompt):** Señalamos y prohibimos con precisión herramientas legacy que la IA escupe por costumbre, como `npm`, `yarn`, `any`, `eslint` y `jest`.
- 🔒 **Inducción a la programación defensiva:** Al forzar el uso de `zod`, establecemos restricciones a nivel de arquitectura para que la propia IA se defienda de las minas que pueden estallar en tiempo de ejecución.
