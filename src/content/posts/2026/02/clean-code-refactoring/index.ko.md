# 📝 내 코드가 쓰레기라니! 클린 코드 리팩토링 코치

- **🎯 추천 대상:** 주니어 개발자, 사수 없이 일하는 1~3년 차 프론트/백엔드 엔지니어
- **⏱️ 소요 시간:** 30분 → 1분 단축
- **🤖 추천 모델:** Claude 3.5 Sonnet, GPT-4o, Gemini 2.5 Pro (코딩 특화 모델 권장)

- ⭐ **난이도:** ⭐⭐☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"분명 내가 짰는데 3개월 뒤에 보니 외계어가 되어있습니다. 건드리면 터질까 봐 무서워서 주석만 남겨둡니다."_

스파게티처럼 뒤엉킨 코드는 개발자의 가장 큰 적이자 기술 부채(Technical Debt)의 핵심입니다. 가독성이 떨어지는 코드는 버그의 온상이 되며, 새로운 기능을 추가할 때마다 야근을 부릅니다. 이 프롬프트는 로버트 C. 마틴(클린 코드 저자) 수준의 수석 아키텍트 페르소나를 AI에게 부여하여, 여러분의 냄새나는 코드를 구조적이고 우아하게 세탁해 줍니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **네이밍 컨벤션 정규화:** 의미를 알 수 없는 `a`, `temp` 등의 변수명을 직관적으로 변경합니다.
2. **함수 단일 책임 원칙(SRP) 적용:** 수백 줄짜리 몬스터 함수를 목적에 맞게 여러 개로 잘게 쪼갭니다.
3. **모던 신택스 및 최적화:** Early Return, 구조 분해 할당 등 최신 문법을 적용하여 코드의 Depth(들여쓰기)를 획기적으로 줄입니다.

## 🚀 해결책: "클린 코드 세탁소"

### 🥉 Basic Version (기본형)

빠르게 결과만 필요할 때 사용하세요.

> **역할:** 너는 시니어 소프트웨어 엔지니어이자 클린 코드 전문가야.
> **요청:** 아래 코드를 기능 변경 없이 가독성과 유지보수성이 극대화되도록 리팩토링해 줘. 그리고 변경된 핵심 이유를 주석으로 간단히 달아줘.

<br>

### 🥇 Pro Version (전문가형)

디테일한 퀄리티와 코드 리뷰가 필요할 때 사용하세요.

> **역할 (Role):** 너는 구글, 메타 등 빅테크 출신의 `[수석 아키텍트]`이자 '클린 코드'의 광신도야.
>
> **상황 (Context):**
>
> - 배경: 현재 레거시 코드는 동작은 하지만, 네이밍이 엉망이고 들여쓰기(Depth)가 깊어 유지보수가 불가능한 수준이야.
> - 목표: 코드의 비즈니스 로직(기능)은 100% 동일하게 유지하면서, 가독성과 확장성을 극대화한 코드로 리팩토링하는 것.
>
> **요청 (Task):**
>
> 1. **[직관적 네이밍]**: 모든 변수, 함수, 클래스명을 누구나 이해할 수 있는 명확한 영단어로 변경해. (매직 넘버 제거 포함)
> 2. **[구조적 분해]**: 단일 책임 원칙(SRP)에 따라 하나의 함수가 하나의 일만 하도록 쪼개고 분리해.
> 3. **[Depth 최소화]**: Early Return(가드 패스) 패턴을 적극 도입하여 불필요한 `else`나 중첩 `if`문을 제거해.
> 4. **[모던 문법]**: 사용 중인 언어(`[Python/JavaScript/Java 등]`)의 가장 최신, 그리고 관용적인(Idiomatic) 문법을 적용해.
> 5. **[리뷰 제공]**: 변경된 코드 아래에 `AS-IS`와 `TO-BE`의 차이점, 그리고 **왜 이렇게 바꿨는지(Why)** 아키텍트의 관점에서 설명해 줘.
>
> **제약사항 (Constraints):**
>
> - 절대 원본 코드의 실행 결과(Output)가 달라지면 안 돼.
> - "코드가 곧 문서다"라는 철학 하에, 코드로 이해 가능한 부분의 불필요한 주석은 모두 삭제해. 단, 비즈니스 로직상 꼭 필요한 설명만 남겨.
>
> **대상 코드:**
> `[여기에 리팩토링할 코드를 붙여넣으세요]`

## 💡 작성자 코멘트 (Insight)

이 프롬프트를 실무에서 가장 잘 활용하는 방법은 **'개인 코드 리뷰(Code Review) 봇'**으로 쓰는 것입니다. PR(Pull Request)을 올리기 전에 본인의 코드를 이 프롬프트에 먼저 돌려보세요.

특히 주니어 개발자라면 사수에게 지적받기 전에 미리 네이밍 컨벤션과 Early Return 패턴을 점검받을 수 있어 엄청난 성장을 이룰 수 있습니다. 코딩 전용 모델(Claude 3.5 Sonnet 등)과 결합하면 거의 완벽에 가까운 결과를 보여주며, 특정 프레임워크(예: React, Spring Boot)를 사용 중이라면 Task 항목에 "React Hooks의 모범 사례를 적용해 줘"라고 한 줄만 추가해도 해당 생태계에 완벽히 맞는 코드를 얻을 수 있습니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: AI가 비즈니스 로직을 마음대로 바꾸면 어떡하나요?**
  - A: 프롬프트 내에 '기능 변경 금지' 제약을 강하게 걸어두었지만, 100% 맹신은 금물입니다. 반드시 리팩토링된 코드를 적용한 후 기존의 유닛 테스트(Unit Test)를 돌려 통과하는지 확인해야 합니다.

- **Q: 회사 내부 코드를 올려도 안전한가요?**
  - A: **절대 안 됩니다.** 사내 보안 규정에 위배될 수 있으므로, API 키, 데이터베이스 스키마, 회사 고유의 민감한 비즈니스 로직은 마스킹(Masking) 처리하거나 가상의 변수명으로 치환한 후 프롬프트에 입력하세요. (단, 회사에서 계약된 Enterprise AI를 사용 중이라면 예외입니다.)

- **Q: 결과물이 너무 파편화되어 함수가 지나치게 많아졌어요.**
  - A: AI가 단일 책임 원칙(SRP)에 너무 집착한 결과입니다. 이럴 때는 제약사항에 `함수 분리는 최대 2단계 뎁스까지만 진행해 줘`와 같이 기준을 추가로 제시하면 적정선을 유지할 수 있습니다.

## 🧬 프롬프트 해부 (Why it works?)

1.  **명확한 페르소나(수석 아키텍트):** AI에게 단순한 '코드 수정자'가 아닌 '엄격한 시니어 개발자'라는 역할을 부여하여 결과물의 품질과 코드 리뷰의 수준을 대폭 높였습니다.
2.  **구체적인 리팩토링 지침(Task):** 네이밍, 구조 분해, Depth 축소 등 클린 코드의 핵심 원칙을 강제하여 AI가 엉뚱한 방향(예: 가독성을 해치는 과도한 숏코딩)으로 최적화하는 것을 막습니다.
3.  **제약 조건(Constraints)의 활용:** '기능 변경 불가'와 '불필요한 주석 제거'를 명시하여 실무에서 바로 복사해 붙여넣을 수 있는 깔끔한 산출물을 유도합니다.

## 📊 증명: Before & After

단순한 명령어와 최적화된 프롬프트의 차이는 명확합니다. 제공된 프롬프트를 사용했을 때 더 구체적이고 구조화된 답변을 얻을 수 있으며, 들여쓰기 지옥에서 탈출할 수 있습니다.

### ❌ Before (입력)

```python
def calc(x, y):
    if x > 19:
        print("성인")
        p = y * 0.9
        return p
    else:
        print("학생")
        return y
```

_(문제점: 함수명 `calc`는 뭘 계산하는지 모르겠고, 매직 넘버 `19`와 `0.9`의 의미가 불명확하며, 불필요한 else문이 존재함.)_

### ✅ After (결과)

```python
def calculate_discounted_price(age: int, original_price: float) -> float:
    ADULT_AGE_THRESHOLD = 20
    ADULT_DISCOUNT_RATE = 0.9

    if age >= ADULT_AGE_THRESHOLD:
        print("성인입니다.")
        return original_price * ADULT_DISCOUNT_RATE

    print("학생입니다.")
    return original_price
```

**[수석 아키텍트의 리뷰 포인트]**

1. **네이밍 개선**: `calc` -> `calculate_discounted_price`로 변경하여 함수의 목적을 명확히 했습니다. 매개변수 또한 `x, y`에서 `age, original_price`로 변경하여 가독성을 높였습니다.
2. **매직 넘버 제거**: `19`, `0.9` 같은 숫자를 의미 있는 상수(`ADULT_AGE_THRESHOLD`, `ADULT_DISCOUNT_RATE`)로 선언하여 유지보수를 용이하게 했습니다.
3. **Early Return 적용**: 불필요한 `else` 블록을 제거하여 코드의 들여쓰기(Depth)를 줄이고 논리 흐름을 단순화했습니다.
4. **타입 힌팅(Type Hinting) 추가**: Python 3.5+의 모던 문법을 적용하여 인자와 반환값의 타입을 명시, 버그 발생 확률을 낮췄습니다.
