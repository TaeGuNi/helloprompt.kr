쏟아지는 AI 코딩 에이전트, 과연 100% 활용하고 계신가요? 아직도 챗봇에게 말 걸듯 코드를 짜달라고 부탁하고 계신다면, 이는 페라리를 타고 동네 마트에 가는 격입니다. 진정한 AI의 위력은 디테일한 **설정(Configuration)**과 촘촘하게 설계된 **워크플로우(Workflow)**에서 비로소 완성됩니다.

오늘은 출시 한 달 만에 GitHub 스타 4만 개를 돌파하며 전 세계 개발자 커뮤니티를 뒤흔든 **[`everything-claude-code`](https://github.com/affaan-m/everything-claude-code)** 리포지토리 활용법을 파헤쳐 봅니다. 이 설정 하나만 적용해도 평범했던 코딩 비서가 내 프로젝트를 속속들이 꿰뚫고 있는 '시니어 엔지니어링 팀'으로 진화합니다.

## ⚡️ 3줄 요약 (TL;DR)

1. **단순한 프롬프트가 아닌 에이전트 OS:** `everything-claude-code`는 AI 코딩 비서의 두뇌를 통째로 업그레이드하는 강력한 운영 체제입니다.
2. **13인의 시니어 페르소나 탑재:** 기획, TDD, 코드 리뷰 등 철저히 분업화된 전문 에이전트들이 개입하여 압도적인 코드 품질을 강제합니다.
3. **스스로 진화하는 `/evolve` 명령어:** 우리 팀만의 고유한 코딩 컨벤션과 아키텍처를 AI가 스스로 학습하고 전역 규칙으로 내재화합니다.

## 🚀 해결책: "Everything Claude Code" 프롬프팅

### 🥉 Basic Version (빠른 설치 및 기본 기획)

터미널에서 단 2분만 투자해 최정상급 시니어 에이전트를 프로젝트에 영입해 보세요.

> **1. 플러그인 설치 (Claude Code 기준):**
> `/plugin marketplace add affaan-m/everything-claude-code`
> `/plugin install everything-claude-code@everything-claude-code`
>
> **2. 기획 프롬프트 실행:**
> `/plan "Next.js 14 App Router를 사용해서 반응형 블로그 포스트 목록 페이지를 만들어줘. Tailwind CSS와 shadcn/ui를 사용할 거야."`

### 🥇 Pro Version (TDD 및 커스텀 룰 강제)

우리 프로젝트에 완벽하게 동기화된, 빈틈없는 시니어 개발자의 워크플로우를 강제하고 싶을 때 이 프롬프트를 활용하세요.

> **역할 (Role):** 너는 엄격한 TDD(테스트 주도 개발) 원칙을 준수하는 `[Senior TypeScript Developer]`야.
>
> **상황 (Context):**
>
> - 우리는 `[everything-claude-code]`의 규칙을 따르고 있어.
> - 목표: `[User Authentication Feature]` 구현을 완료해야 해.
>
> **요청 (Task):**
>
> 1. 터미널에서 `/tdd` 명령어를 활성화해.
> 2. 실제 코드를 작성하기 전에 반드시 예외 케이스를 포함한 테스트 코드를 먼저 작성해.
> 3. 테스트 커버리지가 80%를 넘지 않으면 절대 다음 단계로 넘어가지 마.
> 4. `[Additional Requirements]`이(가) 있다면 이를 완벽히 반영해서 코드를 작성해.
>
> **제약사항 (Constraints):**
>
> - 모든 변수와 함수에는 명시적인 Type Hint를 추가해.
> - `any` 타입 사용은 절대 금지야.
>
> **주의사항 (Warning):**
>
> - 기존 아키텍처나 코드 컨벤션을 위반하는 코드는 작성하지 마. 확신이 서지 않는다면 코드를 짜기 전에 내게 먼저 질문해. (환각 방지)

## 💡 작성자 코멘트 (Insight)

이 설정을 실무에 직접 적용하며 가장 소름 돋았던 순간은, **"AI가 저의 게으른 프롬프팅을 단호하게 거부할 때"**였습니다.

과거에는 "로그인 페이지 하나 붙여줘"라고 대충 던지면, 엣지 케이스나 에러 처리는 온데간데없는 엉성한 코드가 돌아오곤 했습니다. 하지만 `everything-claude-code`를 적용하면 상황이 180도 달라집니다. `/plan` 명령어를 입력하는 순간, AI가 오히려 _"OAuth와 JWT 중 어떤 인증 방식을 채택할까요? 에러 발생 시 사용자에게 보여줄 UI 상태는 어떻게 처리할까요?"_ 라며 날카로운 역질문을 쏟아냅니다. 마치 **요구사항이 완벽하게 정의되기 전에는 절대 키보드에 손을 올리지 않는 깐깐한 수석 개발자**를 곁에 둔 기분입니다.

그중에서도 압권은 단연 **`/evolve`** 명령어의 존재입니다. 프로젝트가 진행됨에 따라 AI가 우리 팀만의 고유한 코딩 컨벤션(예: "우리 팀은 인터페이스 이름 앞에 'I'를 붙이지 않는다")을 스스로 파악하고, 이를 확고한 전역 규칙으로 굳혀 나갑니다. 얽히고설킨 스파게티 코드를 수습하느라 지쳐있거나, 냉철한 시니어의 코드 리뷰가 간절했던 1인 개발자에게 이 워크플로우는 선택이 아닌 **생존을 위한 필수 무기**입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 반드시 Claude Code CLI 환경에서만 동작하나요?**
  - A: 그렇지 않습니다! Cursor 에디터의 `.cursorrules` 파일에 규칙을 직접 이식하거나, OpenCode 같은 타 환경에서도 훌륭하게 호환됩니다.

- **Q: 사내 보안이 매우 중요한 엔터프라이즈 프로젝트에 도입해도 안전할까요?**
  - A: 설정에 사용되는 룰(Rule) 파일 자체는 오픈소스이며 100% 로컬 환경에만 저장되므로 구조적으로 안전합니다. 단, 코드를 LLM API로 전송하는 행위 자체에 대해서는 사내 보안 가이드라인(Enterprise 플랜 도입 여부 등)을 반드시 선행 검토하시길 권장합니다.

- **Q: 주니어 개발자가 사용하기에는 진입 장벽이나 설정이 너무 엄격하지 않을까요?**
  - A: 도입 초기에는 AI가 쉴 새 없이 엣지 케이스를 묻고 테스트 코드를 강제하기 때문에 다소 번거로울 수 있습니다. 하지만 이 엄격한 통과 의례를 거쳐야만, 훗날 감당할 수 없는 **기술 부채(Technical Debt)**에 짓눌려 밤을 새우는 끔찍한 대참사를 미연에 방지할 수 있습니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **강제된 워크플로우 (Forced Workflow):** 일방적인 "해줘"식 지시를 원천 차단합니다. `/plan`, `/tdd` 등 명시적인 슬래시 명령어를 통해 AI가 '요구사항 분석 → 테스트 코드 작성 → 본 코드 구현'이라는 정석적인 엔지니어링 프로세스를 밟도록 강력하게 견인합니다.
2. **다중 페르소나 교차 검증 (Persona Separation):** 기획하는 자아(Planner), 코딩하는 자아(Developer), 리뷰하는 자아(Reviewer)를 철저하게 분리합니다. 이를 통해 AI가 자신이 짠 코드를 스스로 날카롭게 비판하고 검증하는 완벽한 자체 피드백 루프를 형성합니다.
3. **영구적인 컨텍스트 학습 (Instincts):** 세션이 끝나면 맥락이 휘발되는 일반 챗봇과 차원이 다릅니다. 사용자의 코드 수정 패턴과 아키텍처 의도를 프로젝트 전반에 걸쳐 파악하고, 이를 `.rules` 파일에 지속적으로 각인시켜 에이전트 스스로 끊임없이 진화하게 만듭니다.

## 📊 증명: Before & After

### ❌ Before (일반적인 AI 코딩)

```text
User: "게시판 목록 만들어줘"
AI: (묻지도 따지지도 않고 바로 React 코드를 길게 출력함. 페이지네이션 없음, 에러 처리 누락, 프로젝트 디자인 시스템 무시)
```

### ✅ After (everything-claude-code 적용 후)

```text
User: "> /plan 게시판 목록 만들어줘"
AI: Planner 에이전트가 가동되었습니다. 완벽한 구현을 위해 다음 3가지를 먼저 확정해야 합니다.
1. 페이지당 몇 개의 게시물을 보여줄까요? (무한 스크롤 vs 페이지네이션)
2. 데이터 패칭은 서버 사이드(SSR)로 할까요, 클라이언트 사이드(CSR)로 할까요?
3. 데이터 로딩 시 보여줄 Skeleton UI 디자인이 필요합니까?

답변해 주시면 TDD 에이전트에게 테스트 코드 작성을 지시하겠습니다.
```
