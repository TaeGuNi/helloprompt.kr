## 📊 証明: Before & After

### ❌ Before (入力プロジェクト)

「簡単なToDo管理アプリ作るよ。一人で開発するし機能あまりない。」

### ✅ After (結果)

**推薦アーキテクチャ:** **MVVM (Model-View-ViewModel)** または **Feature-based Structure**

**フォルダ構造 (Feature-based):**

```text
src/
├── features/           # 機能別モジュール
│   ├── todos/
│   │   ├── components/ # Todo 関連 UI
│   │   ├── hooks/      # Todo 関連ロジック
│   │   └── api/        # Todo API 呼び出し
│   └── auth/           # 認証関連
├── shared/             # 共通モジュール
│   ├── components/     # 共通ボタン、インプット
│   └── utils/          # ユーティリティ関数
├── App.js
└── index.js
```

_理由: プロジェクトが小さいので複雑なレイヤー分離よりは機能(Feature)別にまとめるのが直感的で開発速度が速いです。_
