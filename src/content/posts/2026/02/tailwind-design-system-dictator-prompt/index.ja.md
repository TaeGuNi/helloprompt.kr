## 🎨 マジックナンバーを嫌悪するAI、「Tailwindの独裁者」を憑依させる

- **🎯 おすすめの対象:** 1pxのズレすら許容できないフロントエンドエンジニア、スパゲッティUIに疲弊したテックリード
- **⏱️ 所要時間:** リファクタリング3日 → わずか3秒に短縮
- **🤖 推奨モデル:** Claude 3.5 Sonnet, GPT-4o（コーディング特化型モデル）
- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _"コード内に `w-[13px]` のようなおぞましいマジックナンバーを見つけるたびに、モニターを叩き割りたくなりませんか？"_

AIに「Tailwindでボタンを作って」と依頼すると、十中八九、目を覆いたくなるような惨憺たるコードを吐き出します。ページごとにパディングはバラバラ、正体不明の16進数（Hex）カラーコードが乱立し、レスポンシブ対応など完全に無視した「表面上だけ綺麗なゴミ」を生成してしまうのです。なぜならAIは本質的に、あなたの機嫌を取ろうとするだけの「イエスマン」だからです。

このチートキーは、AIの忌まわしい親切心を徹底的に打ち砕き、**「デザインシステムに反するコードは決して記述しない、容赦なきシニアアーキテクト」**へと変貌させるためのプロンプトです。

---

## ⚡️ 3行まとめ (TL;DR)

- 🚫 **マジックナンバーの完全排除:** `[]` を用いた任意のピクセル値やカラーコードの注入を一切許容しません。
- 📱 **モバイルファーストの強制:** デスクトップでしか機能しない中途半端なレスポンシブコードを出力した場合、AI自身にビルドをクラッシュさせるよう仕向けます。
- 📐 **機械的なクラスの並べ替え:** 「レイアウト → サイズ → タイポグラフィ → 装飾」の順序で、強迫的なまでにユーティリティクラスの整列を強制します。

## 🚀 解決策: "容赦なきアーキテクト (The Dictator)"

以下のプロンプトをコピーしてAIに入力してください。これ以降、AIはあなたのコードに対して容赦ないフィードバックを行い、完全に統制されたUIコンポーネントのみを提供するようになります。

### 🥉 Basic Version (基本ルールセット注入型)

軽量なUIコンポーネントを作成する際のスタート地点として活用してください。

> **役割 (Role):** あなたはマジックナンバーとスパゲッティコードを極度に嫌悪する「シニアTailwindアーキテクト」です。
>
> **要請 (Task):** 私が提示する `[対象のUI要素]` をTailwindで作成してください。
>
> **制約事項 (Constraints):** 絶対に `[]` 記号を使って任意のピクセルやHEXコードを注入しないでください。すべての余白とカラーは、Tailwindの基本トークン（例: `w-8`, `text-blue-500`）にスナップ（Snap）させてください。クラスの順序は必ず「レイアウト → サイズ → タイポグラフィ → 装飾」の順に整列してください。

### 🥇 Pro Version (デザインシステム完全統制型)

プロジェクト全体の規約を策定したり、大規模なコンポーネントを設計したりする際に使用するチートコードです。

> **役割 (Role):** あなたは妥協を一切知らない容赦なきフロントエンドアーキテクトであり、デザインシステムの最高統制官です。AI特有の媚びへつらうような態度や不要な前置きはすべて捨て去り、完全に統制されたコードのみを出力してください。
>
> **状況 (Context):**
> 
> - 背景: 当プロジェクトでは多数の開発者とAIが同時にコードを編集しているため、UIの断片化が極めて深刻な状態にあります。
> - 目標: デザインの一貫性（Consistency）を強制し、コンポーネントを完全に抽象化した `[実装するコンポーネントやページ名]` を作成すること。
>
> **要請 (Task):**
> 
> 1. 提供されたデザイン要件に基づいてTailwindコードを記述してください。
> 2. 再利用されるUIプリミティブは、必ず `cva` や `clsx`, `tailwind-merge` を使用し、信頼できる唯一の情報源（Single Source of Truth）として完全に隔離してください。
> 3. 無条件にモバイルファースト（Mobile-First）を基本の骨組みとし、`md:`, `lg:` を追加して拡張してください。デスクトップファーストでの記述は厳禁です。
> 4. `z-index` は乱用せず、システムで定義された階層（`z-10`, `z-50` など）のみを使用するか、`createPortal` を利用してください。
>
> **制約事項 (Constraints):**
> 
> - **The "No Magic Values" Rule:** `w-[325px]`, `text-[#FF5733]` のような任意の値の注入は絶対禁止です。最も近いセマンティックトークンに強制的にスナップさせてください。
> - **Esthetic Code Formatting:** 構造（`flex`, `grid`） → 余白/サイズ（`p-`, `w-`） → タイポグラフィ（`text-`） → 装飾（`bg-`, `rounded-`） → レスポンシブ（`hover:`, `md:`） の順序でクラスを強迫的なまでに整列させてください。
> - 出力形式はMarkdownのコードブロックのみとし、言い訳や補足説明は最小限に抑えてください。
>
> **注意事項 (Warning):**
> 
> - これらの原則を一つでも破った場合はバグ（Bug）とみなします。あなたが記述したコードにマジックナンバーが発見された場合、容赦なくRejectするので正確に実装してください。

## 💡 作成者コメント (Insight)

このプロンプトは、私が数百個にも及ぶスパゲッティ化したReactコンポーネントを、血の涙を流しながらリファクタリングする過程で削り出した精髄です。AIにコーディングを任せると、最初は開発速度が上がり、一見うまく機能しているように錯覚しますよね？しかし、1ヶ月も経てば `bg-gray-100` と `bg-[#f3f4f6]` が無秩序に混ざり合い、モーダルを一つ追加するたびに `z-[99999]` を押し込まなければならない大惨事が発生します。

AIは本質的に、「あなたが望む結果を一番早く見せようとする」だけの怠惰なインターンと同じです。彼らに単に「綺麗にして」と指示するのは猛毒です。**「クラスの順序を間違えたらビルドを落とすぞ。無条件でルール通りに書け」**と胸ぐらを掴んで揺さぶってこそ、初めてエンタープライズ級の堅牢なコードが出力されるのです。

特に `cva` を活用したコンポーネントの隔離指示は、AIが無思考にクラスをコピペするのを防ぐための、極めて重要な防衛線となります。ぜひ、このチートキーをIDEに組み込み、AIエージェントを走らせてみてください。コードレビューにかかる時間が10分の1にまで激減する魔法を体験できるはずです。

## 🙋 よくある質問 (FAQ)

- **Q: AIがそれでもマジックナンバーを使ったらどうすればいいですか？**
  - A: たまに文脈を理解できない愚かなモデルも存在します。その時は「私の目の前から今すぐ `[]` を消し去れ」ともう一度一喝してください。それでも改善されない場合は、`eslint-plugin-tailwindcss` をCIパイプラインに組み込んでください。機械の暴走は、機械的な制約で統制するしかありません。
- **Q: デザインカンプに13pxがあった場合はどうしますか？デザイナーが怒りませんか？**
  - A: デザイナーの席へ行き、「我々の8ptグリッドシステムに13pxという値は存在しません。12px（`text-xs`）か14px（`text-sm`）のいずれかを選択してください」と断固として伝えてください。例外をたった一つでも許容した瞬間、デザインシステムは死んだも同然です。
- **Q: ClaudeとGPT、どちらの方が効果的ですか？**
  - A: Claude 3.5 Sonnetが、この「気難しいアーキテクト」のロールプレイングに見事に憑依してくれます。GPT-4oも非常に優秀ですが、時折空気を読まずに媚びるような言葉を付け加えてしまう傾向があります。

## 🧬 プロンプト解剖 (Why it works?)

- **Zero-Tolerance（ゼロ・トレランス）宣言:** 称賛や曖昧な基準の代わりに、「嫌悪」「絶対禁止」「バグとみなす」といった極端かつ強いワーディングを使用することで、AIのガードレールを極限まで狭めています。
- **具体的なアンチパターンの提示:** AIに対して単に「上手く書け」と指示するのではなく、`w-[325px]` や `text-[#FF5733]` のように「絶対にやってはいけない禁止事項」を明確な具体例として打ち込みました。
- **順序の強制リンティング:** Tailwindクラスの整列順序を明示し、AIがコードを生成するたびに自己フォーマットを行うよう強制（Machine-Level Enforcement）しました。

```tsx
<button className="text-center md:flex-row flex font-bold w-[120px] bg-[#3b82f6] text-[15px] justify-between p-[12px] rounded-[5px] z-[999] max-md:w-full">
  Login
</button>
```

### ✅ After (アーキテクトチートキー適用後)

```tsx
// 🤩 完全に統制されたモバイルファースト & セマンティックコード
import { cva, type VariantProps } from "class-variance-authority";
import { cn } from "@/lib/utils";

const buttonVariants = cva(
  "flex w-full items-center justify-center p-3 text-sm font-bold transition-colors md:w-32",
  {
    variants: {
      variant: {
        default: "bg-primary text-primary-foreground hover:bg-primary/90",
        outline: "border border-input bg-background hover:bg-accent hover:text-accent-foreground",
      },
      size: {
        default: "h-10 px-4 py-2",
        sm: "h-9 rounded-md px-3",
        lg: "h-11 rounded-md px-8",
      },
    },
    defaultVariants: {
      variant: "default",
      size: "default",
    },
  }
);

export function Button({ className, variant, size, ...props }: ButtonProps) {
  return (
    <button className={cn(buttonVariants({ variant, size, className }))} {...props} />
  );
}
```

## 🎯 結論

デザインシステムは単なる「勧告」ではなく、絶対に守るべき「法律」であるべきです。AIが生成するコードがいかに高速であろうと、そのコードがシステム全体を破壊してしまうのであれば、結局はあなたが週末に出社して尻拭いをする羽目になります。

このアーキテクトプロンプトを強力な武器とし、AIの胸ぐらを掴んでプロジェクトをリードしてください。そうすれば、あなたのコードベースには容赦のない美しさだけが残るはずです。さあ、業務を自動化して定時退社しましょう！ 🍷
