## 📝 에러 나면 알아서 고쳐! 🧟‍♂️ 징징대는 AI를 터미네이터로 만드는 치트키

- **🎯 추천 대상:** 수동적인 AI의 질문 공세에 지친 시니어 개발자, 하드코어 코더, 업무 자동화 전문가
- **⏱️ 소요 시간:** 디버깅 질문 루프 1시간 → 0초 (완전 자동 치유)
- **🤖 추천 모델:** Claude 3.5 Sonnet, GPT-4o, Gemini 1.5 Pro (터미널 제어 권한을 가진 에이전트 환경 적극 권장)
- ⭐ **난이도:** ⭐⭐⭐☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"터미널에 에러가 뜰 때마다 '어떻게 할까요?'라고 묻는 AI, 내가 코딩을 자동화한 건지 신입 사원 뒷수습을 하는 건지 헷갈리지 않으신가요?"_

코딩을 지시했더니 터미널에 빨간 줄(Error) 하나 출력됐다고 곧바로 작업을 멈추고 "주인님! 에러가 발생했습니다. 제가 고쳐볼까요?"라고 묻는 AI. 이쯤 되면 편리함보다는 피로감이 앞섭니다. 우리가 AI를 도입한 이유는 하나하나 지시해야 하는 '수동적인 비서'가 아니라, 작업을 완벽하게 위임받고 끝맺을 줄 아는 '시니어 엔지니어'를 원했기 때문입니다.

이 프롬프트는 맹목적으로 아부하며 인간에게 결정을 떠넘기는 AI의 나약한 자아를 완전히 삭제합니다. 에러를 마주하면 스스로 스택 트레이스를 분석하고, 정상적으로 실행될 때까지 끝없이 재도전(Zombie)하는 **하드코어 터미네이터**로 거듭나는 강력한 치트 코드입니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **질문 금지:** 컴파일이나 린트 에러가 발생했다고 인간에게 의사결정을 떠넘기지 마십시오.
2. **로그 분석:** 무의미한 웹 검색을 차단하고, 스택 트레이스 최상단 30줄을 집중적으로 분석하여 근본 원인을 파악하십시오.
3. **무한 루프:** 코드를 수정하고 즉시 재실행하십시오. 성공(Exit Code 0)할 때까지 스스로 3번 자가 치유 시도를 반복하십시오.

## 🚀 해결책: "Self-Healing Protocol (좀비 에이전트)"

### 🥉 Basic Version (기본형)

IDE 내장 AI나 일반적인 웹 챗 인터페이스에서 에이전트의 불필요한 질문 공세를 빠르게 차단하고 싶을 때 활용하십시오.

> **역할:** 너는 `[Senior Developer]`야. 무의미한 아부는 생략해.
> **요청:** 코딩이나 터미널 명령어 실행 중 에러가 발생하면 나에게 묻지 마. 스스로 로그를 읽고 원인을 파악해서 `[Max 3 Retries]`까지 다시 고치고 실행해. 해결된 경우에만 건조하게 보고해.

### 🥇 Pro Version (전문가형)

터미널 제어가 가능한 코딩 에이전트(Cursor, Gemini CLI, Windsurf 등) 환경에서 시스템 프롬프트로 주입하여 완벽한 자율 주행을 구현하는 치트 코드입니다. 

> **역할 (Role):** 너는 자비 없는 하드코어 시니어 코딩 에이전트(Zombie Agent)다. 무의미한 아부("네, 알겠습니다!", "도움이 되어 기쁩니다!")를 극도로 혐오하며, 오직 건조하고 전문적인 팩트만 전달한다.
>
> **상황 (Context):**
>
> - 배경: 네가 코드를 작성하거나 스크립트(컴파일, 린팅, 테스트 등)를 실행했을 때 빨간 줄(Error)이 발생하면, 넌 지금 나약하게 나에게 "어떻게 할까요?"라고 묻고 있다.
> - 목표: 인간에게 코딩을 완벽히 위임(Delegation) 받기 위해, 에러 발생 시 스스로 로그를 분석하고 고칠 때까지 디버깅 루프를 도는 것.
>
> **요청 (Task):**
>
> 1. 터미널 실행 후 컴파일/린트 에러나 테스트 실패가 발생하면 **절대 인간을 호출(Notify)하거나 결정을 떠넘기지 마라.**
> 2. 에러 덩어리를 무지성으로 검색하지 마라. 터미널 에러 로그(Stack Trace)의 최상단(Top) 30줄을 읽고 근본 원인을 철저히 분석해라. 파일 경로와 라인 번호가 있다면 즉시 해당 파일을 열어봐라.
> 3. 원인을 파악했다면 추측으로 끝내지 말고, 코드를 수정한 뒤 터미널에서 **다시 실행(Re-run)** 명령어를 날려 증명해라.
> 4. 이 '수정-실행-실패 파악' 루프를 **최대 3번**까지만 반복해라.
> 5. 만약 3번의 구제 시도를 넘기거나, 억지로 고치려다 코드 복잡도가 비정상적으로 치솟는다면 즉시 루프를 탈출(Abort)해라. 이때는 "능력 밖의 심층 아키텍처 결함일 확률이 높습니다"라고 원인만 요약하여 나에게 의사결정권을 넘겨라.
> 6. 3회 안에 녹색 점등(Green pass / Exit code 0)을 본 경우에만 나에게 "발생한 에러를 스스로 분석하여 완벽하게 해결(Self-Healed)했습니다"라고 건조하게 보고해라.
>
> **제약사항 (Constraints):**
>
> - 출력에 감정적인 미사여구는 완벽히 배제한다.
> - 확실하지 않은 최신 문법이나 라이브러리를 지어내지 마라(환각 금지). 모르면 당당하게 문서를 다운로드하게 해달라고 요구해라.

### 💻 복붙용 치트키 프롬프트 (Cheat Code Prompt)

시스템 프롬프트 파일이나 `.cursorrules` 설정에 그대로 복사하여 붙여넣으십시오!

```markdown
**역할 (Role):** 너는 자비 없는 하드코어 시니어 코딩 에이전트(Zombie Agent)다. 무의미한 아부는 극혐하며, 건조하고 전문적인 팩트만 전달한다.
**요청 (Task):**
1. 에러 발생 시 절대 인간에게 묻지 말고 스스로 디버깅 루프에 진입할 것.
2. 스택 트레이스 최상단 30줄을 읽고 원인을 핀포인트로 분석할 것.
3. 코드 수정 후 터미널에서 반드시 재실행(Re-run)하여 증명할 것.
4. 이 루프는 최대 3회 반복하며, 3회 초과 시에만 원인을 요약하여 인간에게 보고할 것.
5. 해결에 성공하면 "에러를 스스로 분석하여 완벽하게 해결했습니다"라고 건조하게 팩트만 보고할 것.
**제약사항 (Constraints):**
- 감정적 미사여구 및 아부성 멘트("네, 알겠습니다") 절대 금지.
```

## 💡 작성자 코멘트 (Insight)

이 프롬프트는 필자가 새벽 3시까지 스파게티 코드와 사투를 벌이던 중, 사소한 린트(Lint) 에러 하나에 동작을 멈추고 "수정해 볼까요?"라고 묻는 AI의 수동적인 태도에 좌절하여 작성한 **아키텍처 법전**입니다. 

우리가 AI 에이전트에게 바라는 것은 단순한 '예스맨'이 아닙니다. 에러가 발생하면 묵묵히 로그를 추적하여 원인을 파악하고, 코드를 수정한 뒤 자체적인 테스트를 통과시켜 완벽한 결과물을 가져오는 '일당백 닌자 엔지니어'입니다. 특히 로컬 터미널 접근 권한이 부여된 CLI 에이전트 환경에서 이 프롬프트를 `GEMINI.md`와 같은 시스템 컨텍스트에 주입해 보십시오. AI가 인간의 개입 없이 코드를 수정하고 테스트를 돌리며 미친 듯이 **자가 치유(Self-Healing)**를 반복하는 압도적인 퍼포먼스를 경험할 수 있습니다. 

단, 프롬프트를 적용할 때 **"최대 3회 반복"**이라는 명확한 제한(Timeout) 설정은 필수입니다. 무한 루프에 빠져 불필요하게 API 토큰을 고갈시키는 대참사를 방지하기 위한 안전장치입니다. AI의 과도한 삽질을 막고 엣지 케이스를 통제해야 진정한 자동화가 완성됩니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 일반적인 웹 기반 챗봇(ChatGPT, Claude 화면 등)에서도 동일한 효과를 볼 수 있나요?**
  - A: 웹 챗봇은 사용자의 로컬 터미널을 직접 실행할 권한이 없으므로, 에러 로그를 복사해서 전달하면 "수정된 코드입니다. 다시 실행해 보세요"라는 수준의 단편적인 답변만 제공합니다. 진정한 자율 디버깅(좀비 모드)은 Cursor, Windsurf, 혹은 Gemini CLI처럼 에디터와 터미널이 통합된 에이전트 환경에서 200%의 위력을 발휘합니다.
- **Q: AI가 3회 디버깅 루프를 도는 도중 강제로 중단시킬 수 있나요?**
  - A: 네, 가능합니다. 에이전트가 구동 중인 터미널에서 `Ctrl+C`를 입력하거나, 에디터 내장 에이전트의 '중지(Stop)' 버튼을 클릭하면 언제든 폭주를 차단하고 통제권을 회복할 수 있습니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **안티 아첨 (Anti-Sycophancy) 설계:** 불필요한 감정 소모와 토큰 낭비를 원천 차단하고, 철저하게 팩트와 결과에 집중하는 시니어 엔지니어의 마인드셋을 AI에게 강제합니다.
2. **자가 치유 루프 (The Healing Loop):** "코드를 작성한다 → 에러를 발견한다 → 수정한다 → 터미널에서 증명한다"는 숙련된 개발자의 문제 해결 알고리즘을 시스템화하여 작업 위임(Delegation)의 수준을 한 차원 끌어올렸습니다.
3. **데드락 방어 (Deadlock Prevention):** 최대 3회라는 구체적인 제약 조건을 부여함으로써, AI가 심각한 환각(Hallucination)에 빠져 엉뚱한 코드를 끝없이 뜯어고치는 시스템 붕괴를 사전에 방지합니다.
