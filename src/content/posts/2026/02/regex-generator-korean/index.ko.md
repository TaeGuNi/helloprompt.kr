# 📝 외계어 같은 정규표현식(Regex), 한국어로 주문하세요

- **🎯 추천 대상:** 주니어 개발자, 데이터 분석가, 퍼포먼스 마케터
- **⏱️ 소요 시간:** 30분 → 1분 단축
- **🤖 추천 모델:** Claude 3.5 Sonnet (코드 생성에 강점), ChatGPT-4o

- ⭐ **난이도:** ⭐☆☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"스택오버플로우에서 긁어온 정규식, 예외 케이스 하나 터지면 어떻게 수정하실 건가요?"_

정규표현식(Regular Expression)은 강력하지만, 쓸 때마다 새롭고 남이 짠 코드를 보면 외계어처럼 느껴집니다.
`^([a-z0-9_\.-]+)@([\da-z\.-]+)\.([a-z\.]{2,6})$`... 이메일 하나 검증하자고 이런 암호를 직접 해독하고 타이핑하는 건 귀중한 시간을 낭비하는 일입니다.
이제 복잡한 정규식 문법을 끙끙대며 외울 필요 없이, 한국어로 "이메일 형식 찾아줘"라고 자연스럽게 주문하세요. AI가 완벽한 정규식과 함께 친절한 주석까지 달아줍니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. 구글링하며 헤매던 복잡한 **정규표현식을 한국어 자연어로 쉽게 생성**합니다.
2. 생성된 암호 같은 **문자열의 각 부분이 어떤 의미인지 상세한 설명**을 받아 유지보수성을 높입니다.
3. 엣지 케이스를 포함한 **테스트 코드를 함께 제공받아 운영 환경 배포 전 안전하게 검증**합니다.

## 🚀 해결책: "정규표현식 통역사"

### 🥉 Basic Version (기본형)

빠르게 특정 패턴의 정규식 코드만 필요할 때 사용하세요.

> **역할:** 너는 시니어 개발자이자 정규표현식(Regex) 마스터야.
> **요청:** '010-1234-5678' 형식의 한국 휴대폰 번호를 검증하는 JavaScript 정규표현식을 하나만 작성해줘. 설명은 생략하고 코드블럭만 출력해.

<br>

### 🥇 Pro Version (전문가형)

정확한 엣지 케이스 커버와 코드 리뷰용 상세 주석이 필요할 때 사용하세요.

> **역할 (Role):** 너는 10년 차 시니어 백엔드 엔지니어이자 정규표현식(Regex) 마스터야.
>
> **상황 (Context):**
>
> - 배경: 서비스 사용자의 입력 폼(Form)에서 데이터를 받아 유효성을 검사해야 해.
> - 목표: 엣지 케이스를 모두 방어할 수 있는 완벽한 정규표현식 작성 및 검증.
>
> **요청 (Task):**
>
> 1. 내가 설명하는 `[찾고 싶은 패턴]`에 완벽히 일치하는 정규표현식을 `[사용 언어]` 문법에 맞게 작성해줘.
> 2. 작성된 정규식이 어떻게 동작하는지 쪼개서 상세히 설명해줘. (동료 개발자 코드 리뷰용)
> 3. 이 정규식이 매칭되는 예시(Match) 5개와 매칭되지 않는 예시(Non-match, 엣지 케이스 포함) 5개를 보여줘.
>
> **찾고 싶은 패턴:**
> `[여기에 원하는 패턴을 자연어로 설명하세요. 예: "비밀번호는 8자~16자 사이, 영문 대문자, 소문자, 숫자, 특수문자(!@#$%^&*)를 각각 최소 1개 이상 포함"]`
>
> **사용 언어/환경:**
> `[사용 언어를 입력하세요. 예: JavaScript / Python / Java]`
>
> **제약사항 (Constraints):**
>
> - 정규식은 성능(Catastrophic Backtracking 등)을 고려하여 최대한 최적화해 줘.
> - 특수문자 이스케이프(Escape) 처리를 해당 언어의 문법에 맞게 정확히 해줘.
> - 출력은 마크다운 형식으로 깔끔하게 정리해 줘.
>
> **주의사항 (Warning):**
>
> - 불필요한 서론은 생략하고, 바로 정규표현식 코드와 설명부터 제공해.

## 💡 작성자 코멘트 (Insight)

이 프롬프트의 진짜 가치는 '정규식 생성' 자체가 아니라, **'엣지 케이스 검증'과 '상세한 설명'**에 있습니다. 실무에서는 정규식을 짜는 것보다, 그 정규식이 예상치 못한 문자열에서 버그(예: 백트래킹 지연으로 인한 서버 다운)를 일으키지 않는지 확인하는 것이 훨씬 중요합니다.

특히 **Claude 3.5 Sonnet** 모델을 사용했을 때 정규식 작성 및 코드 해석 능력이 매우 뛰어납니다. 만약 회사에서 기존 레거시 코드에 적힌 알 수 없는 정규식을 발견했다면, 이 프롬프트를 반대로 활용해 *"이 정규식이 대체 무슨 뜻인지 해석해 줘"*라고 요청해 보세요. 코드 분석 시간이 획기적으로 줄어들 것입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: AI가 만들어준 정규식을 바로 실서버에 적용해도 안전한가요?**
  - A: 프롬프트 결과물로 제공되는 '테스트 케이스(Match/Non-match)'를 반드시 먼저 확인하세요. AI도 가끔 특정 예외 상황을 놓칠 수 있으므로, 제공된 정규식을 `regex101.com` 같은 사이트에서 한 번 더 크로스 체크하는 것을 강력히 권장합니다.

- **Q: 프로그래밍 언어마다 정규식 문법이 다르지 않나요?**
  - A: 맞습니다. 그래서 Pro 버전에 `[사용 언어/환경]` 변수를 두었습니다. Java의 `String.matches()`와 JavaScript의 `RegExp`는 이스케이프(`\`) 처리 방식 등 미묘한 차이가 있으므로, 본인이 사용하는 정확한 환경을 명시하는 것이 중요합니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **상황(Context) 명확화:** 폼 유효성 검사라는 구체적인 목적을 제시하여, AI가 단순히 '작동하는' 코드를 넘어 '안전하고 실무적인' 코드를 작성하도록 유도했습니다.
2. **엣지 케이스 강제:** 매칭/비매칭 예시를 각각 5개씩 요구하여, 사용자가 미처 생각하지 못한 기상천외한 입력값(엣지 케이스)에 대해 AI가 스스로 검증 과정을 거치게 만들었습니다.
3. **성능 제약 조건(Constraints):** 'Catastrophic Backtracking(파멸적 백트래킹)'을 언급하여 성능 저하를 유발하는 비효율적인 정규식을 피하도록 안전장치를 마련했습니다.

## 📊 증명: Before & After

### ❌ Before (일반적인 질문 방식)

사용자가 엉성하게 입력했을 때의 결과입니다.

```text
사용자: "비밀번호 검사하는 정규식 만들어줘."

AI 답변: ^(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d]{8,}$
```

_(설명 없이 코드만 덜렁 주거나, 대문자/소문자/특수문자 조건이 누락되는 등 의도를 파악하지 못한 부정확한 결과를 초래합니다.)_

### ✅ After (Pro Version 적용 결과)

Pro Version 프롬프트를 사용하여 JavaScript 환경에 맞는 정규식과 리뷰용 주석을 얻은 결과입니다.

```javascript
// 생성된 정규표현식
/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[!@#$%^&*])[A-Za-z\d!@#$%^&*]{8,16}$/;
```

**상세 설명 (코드 리뷰용):**

- `^` : 문자열의 시작을 의미합니다.
- `(?=.*[a-z])` : 전방 탐색(Lookahead)을 사용하여, 최소 1개의 소문자가 포함되어 있는지 검사합니다.
- `(?=.*[A-Z])` : 최소 1개의 대문자가 포함되어 있는지 검사합니다.
- `(?=.*\d)` : 최소 1개의 숫자(Digit)가 포함되어 있는지 검사합니다.
- `(?=.*[!@#$%^&*])` : 지정된 특수문자 중 최소 1개가 포함되어 있는지 검사합니다.
- `[A-Za-z\d!@#$%^&*]{8,16}` : 허용된 문자들로만 구성되며, 길이는 8자 이상 16자 이하이어야 합니다.
- `$` : 문자열의 끝을 의미합니다.

**테스트 케이스 검증:**

- ✅ **Match (성공):** `Pass1234!` (조건 완벽 충족), `aB3#cdEF` (8자, 모든 조건 충족)
- ❌ **Non-match (실패):** `pass1234!` (대문자 누락), `PASS1234!` (소문자 누락), `Pas1!` (8자 미만)
