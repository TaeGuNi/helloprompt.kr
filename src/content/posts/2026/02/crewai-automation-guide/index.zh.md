# 🤖 CrewAI：如何向你的 AI 团队下达指令（流程自动化）

- **🎯 推荐受众：** 拥有大量“先做A，再做B，最后做C”类流程性任务的产品经理（PM），或是希望实现内容发布完全自动化的营销人员与开发者。
- **⏱️ 预计耗时：** 15 分钟（用于编写与测试 Python 脚本）
- **🤖 推荐模型：** GPT-4o（或本地 Ollama 模型）

- ⭐ **难度等级：** ⭐⭐⭐☆☆
- ⚡️ **预期效果：** ⭐⭐⭐⭐⭐
- 🚀 **实用程度：** ⭐⭐⭐⭐⭐

> _"觉得 AutoGen 太过发散、难以控制？如果你需要一支指令明确、如军队般纪律严明的 AI 团队，那么 CrewAI 就是你的终极答案。"_

随着智能体（Agent）技术的飞速发展，多 AI 协同工作的时代已经到来。然而，许多主打自由讨论的框架在实际业务中往往会跑偏，甚至陷入死循环。在真实的职场环境中，我们更需要一条**“按既定顺序、精准执行任务”**且高度可控的流水线。CrewAI 正是为此而生——它为每个智能体赋予明确的角色（Role）与目标（Goal），让它们像传送带一样顺畅、高效地流水化作业，是目前最实用、最稳定的 AI 协作框架。

---

## ⚡️ 核心三要素 (TL;DR)

1. **招聘员工 (Agent)：** 定义具有不同专业背景和人设故事的 AI 员工（如研究员、撰稿人等）。
2. **派发任务 (Task)：** 为每位员工分配具体的工作内容，并明确预期的最终交付物格式。
3. **执行流程 (Process)：** 将他们组建成一支团队（Crew），以流水线（Sequential）模式运行，确保前一个人的产出完美交接给下一个人。

## 🚀 核心解决方案："CrewAI 流水线提示词"

### 🥉 Basic Version（基础版）

适用于快速搭建简单的数据收集与摘要提取流水线。你可以通过这段提示词快速生成代码并进行测试。

> **角色：** 你是一位精通 Python 和 CrewAI 框架的高级工程师。
>
> **指令：** 请使用 CrewAI 编写一段基础的 Python 脚本，构建一个由两名智能体（Agent）组成的团队：
>
> 1.  **Researcher（研究员）：** 负责在网络上检索“2026年最新 AI 发展趋势”，并提炼出核心要点。
> 2.  **Writer（撰稿人）：** 基于 Researcher 提供的摘要，撰写一篇通俗易懂、具有吸引力的博客文章。
>
> 请应用 `Sequential`（顺序执行）流程，确保两名智能体按顺序协同工作，并输出完整的可运行代码。

### 🥇 Pro Version（专业版）

包含外部工具（Tools）集成与详尽背景故事（Backstory）的实战级高级流水线。这是一套真正为智能体配备“武器”的自动化生产力脚本。

> **角色 (Role)：** 你是一位实战经验丰富的资深 CrewAI 架构师。
>
> **背景 (Context)：**
>
> - 现状：每天早晨手动收集全球股市动态和重大经济新闻并撰写摘要，耗费了大量的人力与时间。
> - 目标：构建一个能自动调用外部 API 和搜索工具的 AI 团队（Crew），实现“每日晨间股市分析报告”的完全自动化生成。
>
> **指令 (Task)：**
>
> 请编写一段包含以下三名智能体的 CrewAI 完整 Python 代码。你必须为每个智能体设定极为具体的 `Role`（角色）、`Goal`（目标）和 `Backstory`（背景故事）：
>
> 1.  **Market Analyst（市场分析师）：** 使用 `yfinance` 工具收集并分析昨日美股（S&P 500, Nasdaq）的核心指标数据。
> 2.  **News Scraper（新闻抓取专家）：** 调用 `SerperDevTool`（谷歌搜索工具）爬取 5 条最重磅的宏观经济新闻头条。
> 3.  **Chief Editor（主编）：** 综合上述两名智能体的数据，撰写一份直观的“3分钟晨间简报”，并将其保存为 Markdown 格式的文件（命名为 `[今日日期]_report.md`）。
>
> **约束条件 (Constraints)：**
>
> - 执行流程必须严格设置为 `sequential`（顺序执行）。
> - 必须在代码中用注释清晰标明如何导入、初始化并分配智能体所需的工具（Tools）。
> - 必须包含从 `.env` 文件加载环境变量及 API 密钥（OPENAI_API_KEY, SERPER_API_KEY）的基础设置代码。

## 💡 作者深度洞察 (Insight)

CrewAI 真正的杀伤力，在于你为智能体**“配备工具（Tool）”**的那一刻。如果仅依赖大语言模型（LLM）本身的内置知识，很容易产生幻觉（Hallucination）。但是，当你将 `FileReadTool`、`WebsiteSearchTool`，或是你自己编写的 Python 自定义函数作为工具赋予它们时，智能体就拥有了触达和操作真实世界数据的能力。

**🔥 实战排坑指南（Troubleshooting）：**
如果智能体出现不可预测的行为，或者迟迟无法结束任务，十有八九是因为 **`Task` 中的 `expected_output`（预期交付物）** 定义得过于模糊。不要只写“请帮我总结一下”，而是要给出极度具体的指令，例如：“一段不超过 500 字、包含 3 个核心要点（Bullet Points）的 Markdown 格式文本”。只有交付标准精准，整个流水线才能畅通无阻地运转。

## 🙋 常见问题解答 (FAQ)

- **Q: 这种多智能体协作，会不会导致 Token 消耗成本过高？**
  - A: 建议采用模型委派（Model Delegation）策略。对于前置的资料搜集或简单摘要任务，可以分配成本较低的模型（如 GPT-4o-mini 或 Claude-3.5-Haiku）；而对于负责最终润色排版的 Editor（主编）智能体，再调用强大的 GPT-4o。这样可以将整体 API 成本压缩到原来的十分之一。

- **Q: 公司对数据安全要求极高，无法使用外部 API 怎么办？**
  - A: CrewAI 完美支持本地 LLM 的接入。你可以通过 Ollama 调用部署在企业内网的 Llama 3 或 Qwen 模型（只需设置 `llm=Ollama(model="llama3")`）。这样即可在确保数据零泄露的前提下，打造专属的企业内部 AI 团队。

- **Q: 流水线运行中途，人类可以介入审核把关吗？**
  - A: 完全可以！在配置智能体时加上 `human_input=True` 参数即可。当该智能体完成阶段性工作后，程序会在终端控制台暂停并等待用户的反馈。此时你可以直接输入“重写这一段”或是“没问题，继续执行”的指令。

## 🧬 提示词解剖学 (Why it works?)

1.  **Backstory（背景故事）的魔力：** 在提示词中，我们强烈要求赋予智能体详尽的背景（例如：“你在华尔街摸爬滚打了 10 年”）。LLM 会将这种人设发挥到极致，其产出内容的专业度、语气和分析深度将远远超越机械式的简单总结。
2.  **无缝 Handoff（任务交接）：** 通过显式声明 `Sequential` 流程，我们将第一位智能体（分析师）的输出，自然地转化为了第二位（抓取专家）和第三位（主编）的上下文背景。这本质上就是用代码完美复刻了人类团队在工作群里传递文件的协作过程。
