## 🚀 The Solution: "Excel Merger Bot"

### 🥉 Basic Version

This is the fastest and most effective prompt when you simply need to merge multiple Excel files that have the exact same format and headers (first row).

> **Role:** You are a `[Python Expert]`.
>
> **Task:** In my `C:/reports/` folder, I have 12 Excel files with the same format, such as `Jan.xlsx`, `Feb.xlsx`, etc. Write a Python code that sequentially reads all the Excel files in this folder, merges them into a single dataframe, and saves it to my desktop as `total_merged.xlsx`. The header (column name) structure is identical across all files. Please include friendly comments in the code and explain step-by-step how I should run it.


### 🥇 Pro Version

This is a professional-grade prompt used when you need more than just simple merging—such as extracting data under specific conditions or converting formats (e.g., converting text-based numbers into actual integers) simultaneously during the consolidation process.

> **Role:** You are a senior Python data analyst highly skilled in data preprocessing and workflow automation.
>
> **Context:**
>
> - Target Folder: `./data/`
> - File Name Pattern: `store_*.xlsx` (e.g., `store_newyork.xlsx`, `store_london.xlsx`)
> - Data Structure: 'Branch Name' in Column B, 'Revenue' data in Column E.
>
> **Task:**
>
> 1. Iterate through all Excel files matching the specified file name pattern within the target folder and read the data.
> 2. **Preprocessing:** Remove all commas (,) and currency symbols ($) from the 'Revenue' data in Column E, and convert them into calculable integer types. If there are any null values, replace them with 0.
> 3. **Data Aggregation:** Group by 'Branch Name' in Column B (Groupby), and calculate the total sum of the 'Revenue' in Column E for each branch.
> 4. **Save:** Save only the final aggregated result as a new file named `summary_report.xlsx`.
> 5. **Libraries Used:** Use `pandas` for data manipulation and `openpyxl` for Excel file input/output.
>
> **Constraints:**
>
> - Write the completed Python code inside a Markdown `python` code block.
> - Include a `try-except` block for error handling, so that if a specific file is corrupted or has a different format, the entire script does not stop but simply skips that file.

## 🙋 Frequently Asked Questions (FAQ)

- **Q: What if the Excel files each have a different password?**
  - A: You can use Python's `msoffcrypto-tool` library to unlock password-protected files and read the data. Try asking the AI: *"Given a list of passwords like ['1234', '5678'], add code to sequentially try these to decrypt and merge the files."*

- **Q: Won't there be an error if the column positions or formats of the Excel files to be merged are slightly different?**
  - A: Yes, if the structures differ, a simple merge can mess up your data. In this case, you need to add specific exception handling and data normalization conditions to the prompt, like: *"In the Branch A file, data starts from the 3rd row, and in Branch B, the column names are slightly different. Find only the columns containing the word 'Revenue' and map them to a single standard column."*

- **Q: Is it possible to handle massive Excel files with hundreds of thousands of rows?**
  - A: While Excel itself becomes sluggish or fails to open when exceeding 1 million rows, Python's `pandas` can process millions of rows of data in a flash, provided you have enough memory. If you encounter an Out of Memory (OOM) error, instruct the AI: *"Optimize the code to read and save the data in chunks instead of loading it all into memory at once."*

## 🧬 Prompt Anatomy (Why it works?)

1. **Clear File Name Pattern Designation (`store_*.xlsx`):** By specifying a regular expression pattern in the prompt, it guides the AI to use the `glob` module. This prevents the catastrophe of accidentally merging unrelated Excel files or images mixed in the folder.
2. **Specific Data Type Conversion (Preprocessing Request):** It is very common for numbers in Excel to be recognized as strings by Python, causing errors during sum calculations. Giving instructions upfront to remove commas and symbols and convert them to integers (`Integer`) can drastically reduce unnecessary debugging time.
3. **Forced Exception Handling (`try-except`):** When running dozens of files, there's always at least one that is corrupted or structurally broken. By specifying exception handling in the constraints, we prevent the script from crashing midway and complete a robust automation bot.
