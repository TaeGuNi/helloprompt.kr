# 💾 Git 커밋 컨벤션 (Commit Convention)

우리 프로젝트는 **[Conventional Commits](https://www.conventionalcommits.org/)** 규칙을 따릅니다.
일관된 커밋 메시지는 히스토리 파악과 자동화(ChangeLog 생성)에 필수적입니다.

## 1. 메시지 구조 (Structure)

```text
<type>(<scope>): <subject>

<body>

<footer>
```

- **Subject:** 50자 이내, 명령문 사용 (마침표 금지)
- **Body:** (선택) 무엇을, 왜 변경했는지 설명
- **Footer:** (선택) 이슈 트래커 ID (예: #123)

## 2. 타입 (Type)

| 타입         | 설명                                           | 예시                                    |
| :----------- | :--------------------------------------------- | :-------------------------------------- |
| **feat**     | 새로운 기능 추가                               | `feat: 사용자 검색 기능 추가`           |
| **fix**      | 버그 수정                                      | `fix: 검색 결과 클릭 안 되는 버그 수정` |
| **docs**     | 문서 수정                                      | `docs: README.md 배지 추가`             |
| **style**    | 코드 포맷팅, 세미콜론 누락 등 (로직 변경 없음) | `style: 메인 페이지 CSS 정렬`           |
| **refactor** | 코드 리팩토링 (기능 변경 없음)                 | `refactor: 검색 로직 함수 분리`         |
| **test**     | 테스트 코드 추가/수정                          | `test: 검색 기능 E2E 테스트 추가`       |
| **chore**    | 빌드 설정, 패키지 매니저 설정 등               | `chore: pnpm 버전 업데이트`             |

## 3. 작성 규칙 (Rules)

1.  **명령문 사용:** "수정함" 대신 **"수정"** 또는 **"Fix"** (예: `Fixed bug` X -> `Fix bug` O)
2.  **제목 끝 마침표 금지:** `feat: 기능 추가.` (X) -> `feat: 기능 추가` (O)
3.  **한글/영어:** 기본적으로 **한글**을 사용하되, 글로벌 협업 시 영어로 전환합니다.

## 4. 예시 (Examples)

**기능 추가:**

```text
feat(search): 실시간 검색 기능 구현

Fuse.js 라이브러리를 사용하여 클라이언트 사이드 검색 구현함.
제목과 태그 기준으로 필터링 가능.
```

**버그 수정:**

```text
fix(ui): 모바일에서 메뉴가 깨지는 현상 수정

화면 너비 600px 이하일 때 flex-direction을 column으로 변경.
```
