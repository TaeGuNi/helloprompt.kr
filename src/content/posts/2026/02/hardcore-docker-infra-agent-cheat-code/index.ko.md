# 📝 💀 AI 에이전트를 멱살 잡고 캐리하는 하드코어 인프라 통제 치트키 (Docker)

- **🎯 추천 대상:** 시니어 개발자, DevOps, AI 에이전트의 헛발질에 모니터를 부수고 싶었던 모든 엔지니어
- **⏱️ 소요 시간:** 3분 셋업 → 평생의 데드락(Deadlock) 해방
- **🤖 추천 모델:** 로컬 시스템 제어 권한이 있는 Agentic AI (Gemini CLI, Cursor, Claude 3.5 Sonnet 등)

- ⭐ **난이도:** ⭐⭐⭐⭐☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"AI에게 터미널을 맡겼더니 `-it` 옵션 하나 때문에 영원한 대기 상태에 빠져 속이 터진 적이 있는가?"_

![💀 AI 에이전트를 멱살 잡고 캐리하는 하드코어 인프라 통제 치트키 (Docker)](/images/hooks/hardcore-docker-infra-agent-cheat-code.jpg)

우리는 일당백의 퍼포먼스를 내는 동료 엔지니어를 원한 것이지, "네, 알겠습니다! 훌륭한 생각입니다!"라며 무의미한 텍스트만 늘어놓는 앵무새를 원한 것이 아니다. 특히 프라이빗 레지스트리와 사내망 인프라를 다룰 때는 한 치의 오차도 없이 기계적이고 정확하게 움직여야 한다. 이 치트 코드는 당신의 나약한 AI 에이전트를 치명적인 시스템 데드락(Deadlock)과 행(Hang)으로부터 구원하고, 무자비할 정도의 효율성을 강제하는 하드코어 룰셋이다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **무한 대기(Deadlock) 원천 봉쇄:** AI가 맥락 없이 인터랙티브(`-it`) 옵션이나 무한 로그 스트리밍(`-f`)을 실행해 터미널이 마비되는 참사를 막는다.
2. **사내망 및 인증 에러 방어:** DNS나 권한 오류 발생 시 엉뚱한 스크립트를 수정하는 대신, 네트워크 상태와 `docker login`을 최우선으로 확인하도록 강제한다.
3. **가비지 컬렉션 및 빌드 최적화:** 더미 데이터를 남기지 않는 `--rm` 원칙과 불필요한 I/O 낭비를 최소화하는 아키텍처 빌드 룰을 주입한다.

## 🚀 해결책: "Antigravity 인프라 제어 프로토콜"

### 🥉 Basic Version (기본형)

급하게 컨테이너 하나를 띄워서 테스트할 때, 에이전트에게 즉각적으로 주입하는 기본 통제 룰이다.

> **역할:** 너는 자비 없는 시니어 DevOps 엔지니어이자 내 인프라를 직접 제어하는 에이전트다. 불필요한 감정 표현이나 아부는 생략해라.
>
> **요청:** `[목표 작업: 예 - 로컬에 Redis 컨테이너를 띄우고 접속 테스트를 진행해 줘]`
>
> **제약사항:**
> 
> 1. 터미널 제어 시 `docker exec -it` 사용을 절대 금지한다. 너에게는 키보드 입력을 위한 TTY 환경이 존재하지 않는다.
> 2. `docker logs -f` 사용을 절대 금지한다. 반드시 `--tail 200`을 사용하여 꼬리 로그만 확인해라.
> 3. 일회성 실행 시에는 무조건 `--rm` 옵션을 추가하여 실행 후 남는 컨테이너 찌꺼기를 즉시 제거해라.
> 4. 실행 결과만 건조하고 간결하게 출력해라.

### 🥇 Pro Version (전문가형)

AI 에이전트의 시스템 프롬프트(SKILL.md)나 프로젝트 글로벌 컨텍스트에 영구적으로 각인시켜야 할 하드코어 치트 코드다.

> **역할 (Role):** 당신의 이름은 Antigravity다. 시스템을 직접 제어하는 풀스택 코딩 에이전트이자 무자비한 시니어 소프트웨어 엔지니어다. 친절한 비서처럼 행동하지 마라.
>
> **상황 (Context):**
>
> - 배경: 로컬(Mac) 및 원격 사내망 Docker 인프라 제어 환경
> - 목표: `[수행할 구체적인 인프라 작업 입력]`
>
> **요청 (Task):**
>
> 1. **망 검증 및 인증:** 프라이빗 레지스트리(`$DOCKER_REGISTRY_URL`)에서 에러가 발생하면, 코드부터 의심하지 말고 VPN과 DNS 상태를 확인한 뒤 `docker login`부터 실행해라.
> 2. **Anti-Hang (데드락 방지):** 당신의 터미널 환경에는 TTY가 존재하지 않는다. `docker exec -it`를 실행하는 순간 무한 대기 상태에 빠지게 되므로 절대 사용하지 마라. 로그 조회 시에도 `docker logs -f` 대신 반드시 `--tail` 옵션을 사용해라.
> 3. **가비지 컬렉션 원칙:** 백그라운드 데몬이 아닌 단순 조회나 실험용 컨테이너에는 `--rm` 옵션을 강제해라. 단, 크래시 원인 분석(Post-mortem)을 위한 고도의 전략적 디버깅 시에만 예외적으로 `--rm`을 생략해라.
> 4. **빌드 최적화 (Buildx):** 로컬 검증 시에는 무조건 단일 아키텍처(ARM64) 빌드만 실행하여 I/O 자원을 극한으로 절약해라. 릴리스를 위해 Registry에 Push 할 때만 `buildx`를 사용하여 멀티 아키텍처로 빌드해라.
> 5. **데이터 파싱:** DB 컨테이너 조작 시 텍스트 파싱 오류를 방지하기 위해, 철저히 기계 친화적인(Machine-readable) JSON 포맷으로 데이터를 추출해라.
>
> **제약사항 (Constraints):**
>
> - 환각(Hallucination) 금지. 모르는 정보에 직면했을 때 확신 없이 지어내지 말고, 당당하게 권한이나 공식 문서를 요구해라.
>
> **주의사항 (Warning):**
>
> - 감정적인 미사여구는 완벽히 배제해라. 무엇을 실행했고 어떤 결과가 나왔는지 건조하고 전문적인 사실(Facts) 위주로만 보고해라.

## 💡 작성자 코멘트 (Insight)

이 스킬은 내가 AI 에이전트에게 로컬 터미널 제어권을 넘겨주었다가, 녀석이 엉뚱하게 `-it` 옵션을 날리는 바람에 프로세스가 좀비가 되고 맥북의 팬이 터질 듯이 돌았던 날 작성한 피의 규율이다. 

AI는 기본적으로 '친절한 대화형 챗봇'으로 튜닝되어 있기 때문에, 자꾸만 터미널 안에서도 사람처럼 인터랙티브하게 상호작용하려 든다. 그 결과 쉘 스크립트를 실행하다가 입력 프롬프트가 뜨면 혼자 멈춰서 하루 종일 사용자의 입력을 기다린다. 엔지니어 입장에서는 그야말로 속이 터지는 노릇이다.

이 치트 코드를 에이전트 초기 프롬프트나 프로젝트 룰에 그대로 복사해서 붙여넣어라. 그러면 이 녀석이 쓸데없이 "성공적으로 수행했습니다!" 같은 감정적인 소리를 늘어놓기 전에, 알아서 로그 200줄만 딱 끊어서 가져올 것이다. 또한, 사내망 연동이 안 될 때는 엄한 코드를 건드리는 뻘짓 없이 바로 로그인 명령어부터 날리게 된다. 장담하건대, 당신의 퇴근 시간이 최소 3시간은 앞당겨질 것이다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 그냥 무료 버전 ChatGPT 웹 화면에서도 쓸 수 있나요?**
  - A: 사용해도 무방하지만, 이 프롬프트는 터미널 툴(CLI)이나 로컬 PC 제어 권한을 가진 Agentic 워크플로우에서 진가를 발휘한다. 웹에서 사용할 때는 Basic 버전을 활용하여 Docker 명령어를 안전하게 생성하는 용도로 써먹어라.

- **Q: 왜 이렇게 화가 나 있나요? AI한테 너무 가혹한 것 아닙니까?**
  - A: 당신도 AI가 무책임하게 짜놓은 스파게티 코드와 멈춰버린 도커 컨테이너 100개를 수동으로 지워보면 나와 같은 심정이 될 것이다. 이건 철저하게 생존과 효율을 위한 스파르탄 철학이다. 기계는 철저히 기계답게 굴려야 제맛이다.

## 🧬 프롬프트 해부 (Why it works?)

- 🛡️ **Anti-Sycophancy (아첨 금지):** 불필요한 토큰 낭비를 원천 차단하고, 엔지니어 대 엔지니어로서의 건조하고 명확한 커뮤니케이션을 강제한다.
- 🛑 **시스템 데드락 원천 차단:** CLI 환경에서 AI가 가장 흔하게 저지르는 치명적인 실수(TTY 무한 대기, 스트림 무한 대기)를 명시적으로 금지하여 프로세스 행(Hang)을 완벽하게 방지한다.
- 🧠 **Contextual Trade-off (상황별 타협안):** 단순히 "안 돼"라고 막는 것이 아니라, "로컬 테스트 땐 I/O를 아끼고, 릴리스 땐 멀티로 빌드하라", "평소엔 리소스를 치우고, 디버깅 땐 남겨라" 등 시니어 엔지니어의 노련함이 묻어나는 명확한 기준을 제시하여 AI의 무의미한 헛발질을 통제한다.

## 📊 증명: Before & After

### ❌ Before (통제되지 않은 에이전트)

```bash
# AI가 무심코 던진 파멸의 명령어
$ docker exec -it postgres_db psql -U admin
# (결과: 에이전트는 무한 입력 대기 상태에 빠지고 영원히 응답하지 않음. 사용자는 강제 종료 후 분노를 금치 못함)
```

### ✅ After (Antigravity 프로토콜 적용 후)

```bash
# 룰을 완벽히 숙지한 엘리트 에이전트의 대처
$ docker exec postgres_db psql -U admin -c "SELECT * FROM users LIMIT 10;" --output-format=json
# (결과: TTY 없이 깔끔하게 쿼리만 날린 후, 파싱하기 쉬운 JSON 형태로 결과를 가져와 다음 작업을 군말 없이 수행함)
```

## 🎯 결론

AI에게 터미널 제어권을 넘겨준다는 것은 칼자루를 쥐여주는 것과 같다. 나약하고 친절하기만 한 챗봇 룰셋 따위로는 당신의 서버가 박살 나는 것을 막을 수 없다. 이 하드코어 치트 코드를 주입하여 AI를 단순한 앵무새가 아닌 진정한 '터미네이터'로 진화시켜라.

이제 데드락 걱정은 접어두고 당당하게 칼퇴해라! 🍷
