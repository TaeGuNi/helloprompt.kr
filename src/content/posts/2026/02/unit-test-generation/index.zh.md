## 📝 写测试代码很烦吧？让 AI 代劳

- **🎯 推荐对象:** 后端与前端开发工程师、QA 工程师、为测试覆盖率发愁的职场人
- **⏱️ 所需时间:** 30 分钟 → 1 分钟
- **🤖 推荐模型:** Claude 3.5 Sonnet（最适合编写代码）、ChatGPT (GPT-4o)

- ⭐ **难度:** ⭐⭐☆☆☆
- ⚡️ **有效性:** ⭐⭐⭐⭐⭐
- 🚀 **实用性:** ⭐⭐⭐⭐⭐

> _“业务逻辑都快写不完了，哪还有时间补测试？但不写的话，每次上线合并代码时，双手都在颤抖……”_

为了保障线上服务的绝对稳定性，单元测试（Unit Test）无疑是不可或缺的一环。然而，编写测试代码往往费时费力，且极其枯燥乏味。尤其是在项目排期紧迫的压力下，想要严谨地覆盖各种异常分支（Edge Cases）更是难上加难。

现在，把这个枯燥的体力活统统交给 AI 吧！让它化身为你的专属“测试覆盖率守护者”，你只需将宝贵的精力集中在核心业务逻辑的实现上。

## ⚡️ 3 行总结 (TL;DR)

1. **解放双手**：将繁琐枯燥的单元测试编写工作交由 AI 处理，大幅节省开发时间。
2. **滴水不漏**：不仅能快速生成常规的成功路径（Happy Path），还能自动补充出人意料的边缘情况（Edge Case）。
3. **角色赋能**：通过赋予 AI “资深 QA 工程师”的专业视角，有效提升代码的测试覆盖率与整体工程质量。

## 🙋 常见问题 (FAQ)

- **Q: AI 生成的测试代码可以直接提交到代码库吗？**
  - A: 强烈建议先在本地运行一次。尽管 AI 能够覆盖 90% 以上的场景，但它偶尔会产生幻觉，去“脑补”一些根本不存在的依赖库或内部私有方法。跑通验证之后再提交，才是最稳妥、最负责任的做法。

- **Q: 如果我的代码逻辑非常复杂，深度依赖诸多外部服务（比如数据库、第三方 API），该怎么办？**
  - A: 对于涉及外部依赖的复杂代码，你只需在提示词的 **背景 (Context)** 中额外补充一句指令：“_请使用 Mock 技术（如 Jest.mock 或 Mockito）来完美模拟所有外部依赖项。_” 这样，AI 就会自动为你生成包含完善 Mock 逻辑的高级测试代码。

- **Q: 这个提示词支持哪些编程语言？**
  - A: 它几乎完美支持所有主流编程语言（如 Java, Python, TypeScript, Go 等）。只要在提示词的变量中明确指定你当前使用的测试框架（如 JUnit, PyTest, Jest），AI 就会精准输出符合该语言和框架规范的测试代码。

## 🧬 提示解剖 (Why it works?)

1.  **设定资深 QA 角色**：如果没有赋予特定的角色定位，AI 只会像个实习生一样敷衍地写出最简单的 `1+1=2` 测试。一旦为其赋予“拥有 10 年经验的资深 QA 工程师”角色，它便会立刻切换视角，主动以极其专业且挑剔的眼光去审视代码，寻找漏洞。
2.  **强制规定输出结构 (Happy Path + Edge Case)**：明确要求它必须严格区分正常情况与边界情况，这不仅迫使 AI 跳出表面的代码逻辑，更引导它深入思考各种极端场景（例如：输入为空、负数、天文数字、并发冲突等）。
3.  **附加代码审查（Code Review）功能**：在 Task 4 中要求它指出潜在 Bug，这一巧妙的设计相当于在编写测试的同时，顺便让一位经验丰富的高级工程师免费为你做了一次深度的 Code Review。

## 📊 证明: Before & After

### ❌ Before (输入代码：简单计算函数)

```javascript
function divide(a, b) {
  if (b === 0) throw new Error("不能除以 0。");
  return a / b;
}
```

### ✅ After (生成的测试代码)

```javascript
// Jest 框架使用示例

describe("divide 函数单元测试", () => {
  // 1. Happy Path: 正常除法逻辑验证
  test("Given 10 和 2, When 调用 divide, Then 返回 5", () => {
    expect(divide(10, 2)).toBe(5);
  });

  // 2. Edge Case: 负数除法处理
  test("Given 负数参数, When 调用 divide, Then 符号处理正确", () => {
    expect(divide(-10, 2)).toBe(-5);
    expect(divide(-10, -2)).toBe(5);
  });

  // 3. Exception Case: 核心边界条件 - 除以 0
  test("Given 除数为 0, When 调用 divide, Then 抛出 Error", () => {
    expect(() => divide(10, 0)).toThrow("不能除以 0。");
  });

  // 4. Edge Case: 浮点数/小数结果处理
  test("Given 无法整除的数字, When 调用 divide, Then 返回精确小数", () => {
    expect(divide(5, 2)).toBe(2.5);
  });
});
```
