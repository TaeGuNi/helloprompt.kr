## 🙋 Häufig gestellte Fragen (FAQ)

- **F: Der generierte SVG-Code wird im Browser nicht gerendert, sondern nur als reiner Text angezeigt. Woran liegt das?**
  - A: Gelegentlich schmuggelt die KI unerwünschte Markdown-Blöcke oder eine veraltete `<?xml...>`-Deklaration mit ein. Die Lösung ist simpel: Fügen Sie den Output in Ihre IDE (z.B. VS Code) ein, isolieren Sie strikt den Block zwischen dem öffnenden `<svg>`- und dem schließenden `</svg>`-Tag und entfernen Sie alles andere. Danach können Sie den Code entweder als `.svg`-Datei speichern oder direkt als Inline-SVG in Ihr DOM injizieren.

- **F: Ist der Prompt ausschließlich auf kuriose Tiermotive wie den "fahrradfahrenden Pelikan" beschränkt?**
  - A: Absolut nicht! Dieser Workflow ist Ihr universelles Canvas. Tauschen Sie die Variable `[Ziel]` einfach gegen Ihre kühnsten Ideen aus – sei es eine „Katze im Cockpit eines Raumschiffs“ oder ein „meditierender Cyborg-Mönch“. Auch die stilistische Variable `[modernen, poppigen Neon-Cyberpunk-Stil]` lässt sich mit einem Wimpernschlag an Ihre Corporate Identity anpassen, etwa durch „minimalistischer, monochromer Bauhaus-Stil“.

- **F: Die KI hat die Code-Generierung plötzlich mittendrin abgebrochen. Was nun?**
  - A: Das passiert, wenn das strenge Token-Limit für Ausgaben bei extrem detailreichen Vektoren greift. Tippen Sie einfach "Bitte setze den Code exakt an der abgerissenen Stelle fort" in den Chat. Alternativ können Sie die Dateigröße präventiv reduzieren, indem Sie den Stil-Parameter um die Anweisung `[Verwende einen simplen, detailarmen Flat-Design-Stil]` ergänzen.
