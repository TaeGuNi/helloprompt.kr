来到 2026 年，**MCP (Model Context Protocol)** 已经成为连接 LLM 与现实数据的“事实标准 (De-facto standard)”。你可以把它想象成专为 AI 模型设计的“USB 接口”。与其每次手动把上下文“喂”给 AI，不如只需连接一次服务器，让模型在需要时自动、实时地“读取”你的本地文件、数据库或内部 API。

目前社区中已经有许多连接 GitHub 或本地文件系统 (Filesystem) 的优秀通用 MCP 服务器。但这仅仅是冰山一角。MCP 真正的强大之处在于，它能将**“你们公司独有的业务逻辑和私有数据”**无缝接入 LLM。

今天，我们不需要从零开始手写那些枯燥的样板代码 (Boilerplate)，而是介绍一段高效的提示词，直接指挥 AI 为你生成代码。只需 5 分钟，你就能搭建出一个完美的、基于 TypeScript 的定制化 MCP 服务器脚手架 (Scaffolding)。

## 💡 作者点评 (Insight)

这段提示词的核心魔力，在于它没有宽泛地要求 AI "写一段代码"，而是强制规定了 MCP 服务器必须具备的**协议核心规范（如处理 ListTools 和 CallTool 请求）**。

自己动手搭建并运行 MCP 服务器，最大的收益在于**“绝对的隐私安全 (Privacy)”**。当你使用通用的网页搜索插件时，你的查询和数据不可避免地会经过外部服务器。但如果使用直连本地 PostgreSQL 数据库或内部 API 的定制版 MCP 服务器，数据的调取完全掌握在你的控制之下。你再也不用冒险把包含机密的内部文档或客户数据复制到网页聊天框里了。

建议你先用这段提示词创建一个简单的“只读 (Read-only)”工具练手，比如读取本地的某个日志文件。当你熟悉了它的运作机制后，就可以逐步扩展出能够修改数据库、调用复杂业务接口的功能，最终打造出专属于你的超级 AI 助理。

## 🙋 常见问题 (FAQ)

- **Q: 如果我完全不懂编程，也可以使用这个提示词吗？**
  - A: 完全可以！你只需要在 `[核心功能]` 的括号里用自然语言描述你的需求，比如“一个能告诉我北京今天天气的功能”，Claude 就会自动生成规范的代码。你只要把生成的代码保存下来并配置运行就可以了。

- **Q: 生成的服务器代码该如何运行并连接到模型？**
  - A: 将生成的代码保存为 `index.ts` 并编译运行。然后，在你的 Claude Desktop 配置文件 (`claude_desktop_config.json`) 中的 `mcpServers` 字段下添加这个脚本的启动路径，重启客户端即可自动连接。

- **Q: 这样搭建的服务器只能给 Claude 用吗？**
  - A: 不是的。Model Context Protocol 是一个开放标准。任何支持 MCP 协议的客户端或平台（例如 Ollama, Cursor 甚至是未来的其他 AI 工具）都可以无缝接入并使用你搭建的服务器。

## 🧬 提示词解剖 (Why it works?)

1.  **强大的约束机制 (Constraints)：** 强制使用 `@modelcontextprotocol/sdk` 和 `zod`，并明确要求实现核心协议接口 (`ListToolsRequestSchema`, `CallToolRequestSchema`)。这从根本上杜绝了 AI 因为不了解 MCP 最新规范而产生的“幻觉 (Hallucination)”。
2.  **极简的单文件输出：** 明确要求只输出一个 `index.ts` 文件，去除了不必要的项目结构搭建和冗长的文字解释，极大缩短了用户的“获得价值时间 (Time-to-Value)”，复制即用。
3.  **强制防御性编程：** 预先规定了在核心逻辑中必须使用 `try/catch`，防止 AI 生成脆弱的代码，确保了工具在面对异常输入时，服务器进程依然能够稳定运行。
