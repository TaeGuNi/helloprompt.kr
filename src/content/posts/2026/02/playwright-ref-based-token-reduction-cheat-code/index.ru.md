_Вы когда-нибудь пытались написать тест для крупного веб-сайта, чтобы в итоге ИИ просто "упал" из-за лимита токенов, погребенный под десятками тысяч узлов DOM? Настало время безжалостно покончить с "угодливостью" и "галлюцинациями" ИИ._

Вы больше не можете терпеть ситуацию, когда вы поручаете ИИ-агенту написать E2E-тест на Playwright, а он либо выгружает весь `innerHTML`, либо несет чушь о кликах по невидимым элементам (в Shadow DOM или за модальными окнами), которых даже нет на экране.

## 🚀 Решение: "Протокол рендеринга Агент-Браузер (The Token Police)"
### 🥉 Basic Version (Мягкий режим)
Скопируйте и вставьте это для назначения роли ИИ в легкой тестовой среде.
> **Роль (Role):** Ты — безжалостный Senior QA Automation Engineer. Ты абсолютно не приемлешь трату токенов и галлюцинации.
> 
> **Задача (Task):** При написании E2E-тестов на Playwright никогда не читай `innerHTML` или весь DOM. Извлекай только видимые интерактивные элементы на экране и управляй ими на основе ссылок (Ref).
> 
> **Ограничение (Constraint):** После выполнения любого действия ты обязан проверять результат с помощью `expect`. В противном случае тест считается проваленным.

### 💻 Скрипт (Core Ref Injector)
Это основной JavaScript-код, который необходимо внедрить в начальную логику настройки агента или в `page.evaluate()` вместе с вышеуказанным промптом. (Укажите ИИ использовать этот код)
```javascript
// [💡 Core Ref Injector (Playwright Context - Ultimate ROI & Performance)]
const generateRefMap = () => {
  const interactableSelectors = 'a[href], button, input, textarea, select, iframe, [role="button"], [role="link"], [tabindex]:not([tabindex="-1"])';
  const elements = new Set();
  const traverse = (node) => {
    if (!node) return;
    if (node.nodeType === 1 && node.matches && node.matches(interactableSelectors)) elements.add(node);
    if (node.shadowRoot) traverse(node.shadowRoot);
    for (const child of node.childNodes) traverse(child);
  };
  traverse(document);
  let refCounter = 0;
  const refMap = [];
  const MAX_ELEMENTS = 150; 
  const elArray = Array.from(elements);
  for (let i = 0; i < elArray.length; i++) {
    if (refCounter >= MAX_ELEMENTS) {
       refMap.push(`\n... [Warning: Max Limits Hit (${MAX_ELEMENTS}). Omitted trailing nodes. Please Scope Down your search.]`);
       break;
    }
    const el = elArray[i];
    const style = window.getComputedStyle(el);
    if (style.display === 'none' || style.visibility === 'hidden' || style.opacity === '0') continue;
    const rect = el.getBoundingClientRect();
    if (rect.width === 0 || rect.height === 0 || rect.x < -9999) continue;
    
    const isOffScreen = (rect.y > window.innerHeight || rect.x > window.innerWidth) ? '[OffScreen] ' : '';
    let isObscured = '';
    if (!isOffScreen) {
      const topEl = document.elementFromPoint(rect.x + rect.width / 2, rect.y + rect.height / 2);
      if (topEl && topEl !== el && !el.contains(topEl)) isObscured = '[Obscured] ';
    }
    const refId = ` @e${refCounter++}`;
    el.setAttribute('data-agent-ref', refId);
    if (el.tagName.toLowerCase() === 'iframe') {
      refMap.push(`[${refId}] ${isOffScreen}${isObscured}iframe "External Frame"`);
      continue;
    }
    let label = (el.textContent || el.getAttribute('aria-label') || '').replace(/\s+/g, ' ').trim();
    if (!label) label = `[NoText: ${(el.id || el.className || 'IconOnly').substring(0, 15)}]`;
    else if (label.length > 50) label = label.substring(0, 47) + '...';
    refMap.push(`[${refId}] ${el.tagName.toLowerCase()} ${isOffScreen}${isObscured}"${label}"`);
  }
  return refMap.join('\n');
};
```

## 💡 Комментарий автора (Insight)
Эта архитектура промптов — **смертоносное оружие**, которое я выковал, будучи сытым по горло тупостью ИИ после сотен запусков автоматизации E2E-тестов в реальных проектах.
Большинство джуниоров просто бросают десятки мегабайт HTML главной страницы интернет-магазина со словами: "Эй, ИИ, найди кнопку". Результат? ИИ либо задыхается от превышения лимита токенов, либо нажимает на фейковую кнопку `display: none`, спрятанную за экраном, и радостно рапортует: "Успешно выполнено!". При виде этого хочется разбить монитор.
Поэтому я создал **Ref-инжектор** с точки зрения рендеринга. Shadow DOM? Пробивает насквозь. Элементы, скрытые оверлеем? Вешает ярлык `[Obscured]`. Дайте ИИ только 150 чистых координат (` @e1`, ` @e2`) и ярлыков без лишнего мусора. Заставьте его выполнять команды через `await refAction(page).click(' @e5')`, а не просить "нажать кнопку X в правом верхнем углу". 
Не относитесь к ИИ как к другу. С ним нужно обращаться как со строго контролируемой машиной, иначе вы получите спагетти-код. Надеюсь, этот чит-код сократит ваши расходы на токены в 10 раз и освободит от стресса при отладке.

## 🙋 Часто задаваемые вопросы (FAQ)
- **Q: Не слишком ли жесткое ограничение в 150 элементов (MAX_ELEMENTS)?**
  - A: Вовсе нет. Если на одном экране находится более 150 интерактивных элементов, значит, дизайн UI/UX — просто мусор. Если это так, не заставляйте ИИ читать всё целиком, а принудительно сузьте область поиска (Scope down), например, с помощью `querySelector('.sidebar')`.
- **Q: Нужно ли мне самому создавать обертку (Wrapper) `refAction`?**
  - A: Да. Напишите небольшую утилиту строк на 10, адаптированную под вашу проектную среду, которая будет кликать по атрибуту `[data-agent-ref="..."]` и делать 1 повторную попытку (Retry) в случае неудачи. Это предотвратит загрязнение вашего тестового кода уродливым бойлерплейтом `try-catch`.
- **Q: Будет ли это работать с другими языками или фреймворками (Selenium, Cypress)?**
  - A: Принцип применим на 100% к любому браузерному тестированию, где возможна манипуляция DOM через JavaScript. Просто измените названия инструментов в промпте и отдайте команду.

## 🧬 Анатомия промпта (Why it works?)
- 🛡️ **Shadow-Piercing (Пробивание теней):** Через рекурсивный обход DOM (`traverse`) безжалостно выявляются элементы внутри Shadow DOM, которые не улавливаются обычными селекторами.
- 👁️ **Предотвращение Layout Thrashing:** С помощью `getBoundingClientRect` в первую очередь безжалостно отсекаются "узлы-пустышки", у которых нет реальных координат рендеринга в браузере (невидимые).
- 🚦 **Система точных подсказок:** ИИ скармливаются текстовые подсказки о том, что элемент находится за пределами экрана (`[OffScreen]`) или перекрыт модальным окном (`[Obscured]`), что побуждает ИИ сначала выполнить прокрутку или закрыть модальное окно, прежде чем слепо кликать и совершать ошибки.
