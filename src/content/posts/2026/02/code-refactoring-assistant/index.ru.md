# 🍝 Ваш код похож на спагетти? Доверьте его эксперту по рефакторингу

- **🎯 Рекомендуется для:** Младших и средних разработчиков, инженеров-программистов, желающих улучшить качество кода
- **⏱️ Экономия времени:** От 1 часа → до 2 минут
- **🤖 Рекомендуемые модели:** Claude 3.5 Sonnet, GPT-4o, Gemini 2.5 Pro

- ⭐ **Сложность:** ⭐⭐☆☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Полезность:** ⭐⭐⭐⭐⭐

> _"Работает — не трогай? Нет, если ваш код превратился в неподдерживаемое спагетти, настало время для глубокого рефакторинга."_

Никто не любит копаться в чужом (или даже собственном) коде, написанном несколько месяцев назад, если он напоминает запутанный клубок. Технический долг накапливается, баги множатся, а добавление новой фичи превращается в кошмар. Но что, если бы рядом всегда находился опытный наставник, готовый мгновенно провести код-ревью и переписать проблемные участки? С помощью правильного ИИ-промпта это становится реальностью.

---

## ⚡️ 3 кратких вывода (TL;DR)

1. ИИ не просто меняет названия переменных, он способен применять паттерны проектирования (Design Patterns) и принципы SOLID.
2. С помощью правильного промпта можно превратить запутанный "спагетти-код" в модульный и легко тестируемый.
3. Обязательно требуйте от ИИ подробных комментариев и объяснений причин изменений, чтобы учиться в процессе.

## 🚀 Решение: "Мастер рефакторинга чистого кода"

### 🥉 Basic Version (Базовая версия)

Используйте, когда нужно быстро привести в порядок небольшой фрагмент кода.

> **Роль:** Ты — Senior Software Engineer, эксперт по рефакторингу.
> **Задача:** Улучши читаемость и структуру этого кода, сделав его более понятным и современным: `[Вставьте ваш код]`

\

### 🥇 Pro Version (Профессиональная версия)

Используйте для сложной бизнес-логики, где важны архитектура, принципы проектирования и масштабируемость.

> **Роль (Role):** Ты — Staff Software Engineer и строгий сторонник принципов чистого кода (Clean Code) и SOLID. Ты известен тем, что превращаешь запутанный спагетти-код в элегантные, модульные и легко тестируемые архитектуры.
>
> **Контекст (Context):**
>
> - Фон: Этот код написан на `[Язык программирования, например: TypeScript]`. Он работает, но его трудно поддерживать, масштабировать и тестировать. Логика сильно запутана.
> - Цель: Провести глубокий рефакторинг без изменения внешнего поведения системы (полное сохранение функциональности).
>
> **Задача (Task):**
>
> 1. Проанализируй предоставленный код и выяви основные проблемы (нарушения SOLID, запахи кода, дублирование).
> 2. Перепиши код, сделав его модульным, читаемым и самодокументируемым. Избавься от лишней вложенности (callback hell / глубокие if-else).
> 3. Добавь JSDoc/Docstring комментарии для сложных алгоритмов и публичных методов.
> 4. Код для рефакторинга:
>
> ```
> [Вставьте ваш спагетти-код сюда]
> ```
>
> **Ограничения (Constraints):**
>
> - Выведи исправленный код строго в одном блоке кода.
> - Не используй сторонние библиотеки, если они не были импортированы в исходном коде или явно не запрошены.
> - Сохрани все существующие внешние зависимости и переменные среды.
>
> **Формат (Format):**
>
> - Сначала кратко (в виде маркированного списка) перечисли, **что именно** было изменено и **почему** (архитектурные обоснования).
> - Затем предоставь финальный результат в виде Markdown-блока с кодом.

## 💡 Комментарий автора (Insight)

В реальной разработке рефакторинг часто откладывают "на потом" из-за страха что-то сломать, нехватки времени или дедлайнов. Этот промпт работает как ваш личный Tech Lead, который проводит въедливое Code Review. Ключевая фишка версии **Pro** — это требование к ИИ объяснить **причины** изменений. Это не только улучшает вашу кодовую базу здесь и сейчас, но и помогает вам расти как разработчику: вы изучаете лучшие практики на примере собственного кода. Рекомендую всегда прогонять Unit-тесты после применения рефакторинга от ИИ!

## 🙋 Часто задаваемые вопросы (FAQ)

- **В: Может ли ИИ случайно сломать работающую логику при рефакторинге?**
  - О: Да, иногда ИИ может упустить скрытые пограничные случаи (edge cases) или нюансы бизнес-логики. Всегда используйте системы контроля версий (Git) и обязательно запускайте тесты после любых изменений.

- **В: Какую модель ИИ лучше всего использовать для рефакторинга кода?**
  - О: На данный момент (2026 год) модели Claude 3.5 Sonnet и GPT-4o показывают наилучшие результаты в понимании сложных архитектурных паттернов и контекста объемной кодовой базы.

- **В: Понимает ли ИИ специфичные для фреймворка паттерны (например, React, Vue или NestJS)?**
  - О: Абсолютно! Но для идеального результата обязательно уточните в разделе **Контекст**, какой фреймворк и архитектурный паттерн (например, Feature-Sliced Design или Clean Architecture) вы используете.

## 🧬 Разбор промпта (Why it works?)

1. **Роль Staff Software Engineer:** Заставляет ИИ применять высокие стандарты индустрии, а не просто делать поверхностные косметические изменения.
2. **Фокус на SOLID и Clean Code:** Задает четкий вектор того, по каким именно критериям оценивать и менять код (модульность, отсутствие глубокой вложенности).
3. **Объяснение "Почему":** Требование объяснить архитектурные решения предотвращает слепое копирование кода и способствует вашему профессиональному обучению.

## 📊 Доказательство: До и После (Before & After)

### ❌ До (Исходный спагетти-код)

```javascript
function processData(users, req) {
  let res = [];
  for (let i = 0; i < users.length; i++) {
    if (users[i].age > 18 && users[i].isActive == true) {
      if (req.type == "admin" || users[i].role == "vip") {
        let u = users[i];
        u.discount = 20;
        res.push(u);
      } else {
        let u = users[i];
        u.discount = 5;
        res.push(u);
      }
    }
  }
  return res;
}
```

### ✅ После (Результат рефакторинга)

```javascript
const DISCOUNT_RATES = {
  VIP_OR_ADMIN: 20,
  DEFAULT: 5,
};

const isEligibleForDiscount = (user) => user.age > 18 && user.isActive;
const hasPremiumAccess = (user, requestType) =>
  requestType === "admin" || user.role === "vip";

/**
 * Обрабатывает список пользователей и применяет к ним скидки
 * в зависимости от их статуса и типа запроса.
 *
 * @param {Array} users - Список пользователей для обработки.
 * @param {Object} request - Объект текущего запроса.
 * @returns {Array} Список пользователей с примененными скидками.
 */
function applyDiscountsToEligibleUsers(users, request) {
  return users.filter(isEligibleForDiscount).map((user) => ({
    ...user,
    discount: hasPremiumAccess(user, request.type)
      ? DISCOUNT_RATES.VIP_OR_ADMIN
      : DISCOUNT_RATES.DEFAULT,
  }));
}
```
