# 📝 Начало работы с Генеративным UI с помощью Tambo SDK

- **🎯 Рекомендуется для:** Фронтенд-разработчиков, React-инженеров, UX/UI дизайнеров
- **⏱️ Экономия времени:** Недели разработки сложной логики → Пара часов настройки
- **🤖 Рекомендуемый стек:** React, Vercel AI SDK, LLM с поддержкой Function Calling

- ⭐ **Сложность:** ⭐⭐⭐☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Применимость:** ⭐⭐⭐⭐⭐

> _"Пользователи устали от скучных текстовых чат-ботов. Что если ваш ИИ сможет генерировать полноценные интерактивные интерфейсы на лету?"_

Помимо генерации текста и изображений, мы вступили в эпоху, когда ИИ создает **Пользовательские Интерфейсы (UI)** в реальном времени. Это направление получило название **Generative UI (Генеративный UI)**.

Сегодня мы рассмотрим **Tambo SDK** — инструмент, который помогает максимально просто реализовать Generative UI в среде React, и разберем, как заставить ИИ отдавать интерактивные компоненты вместо громоздкого полотна текста.

---

## ⚡️ 3 кратких вывода (TL;DR)

1. **Динамический рендеринг:** ИИ сам решает, какой UI-компонент показать, основываясь на контексте диалога.
2. **Безопасность типов:** Tambo SDK элегантно связывает ИИ и React через Zod-схемы, исключая ошибки типизации.
3. **Новый уровень UX:** Текстовый чат превращается в полноценное мини-приложение с картами, кнопками и формами.

---

## 🚀 Решение: Системный промпт для Generative UI

Чтобы языковая модель (например, GPT-4 или Claude) правильно вызывала ваши компоненты через Tambo SDK, ей нужен четкий системный промпт.

### 🥉 Basic Version (Базовая версия)

Простой промпт для вызова базовых информационных карточек. Идеально для старта.

> **Роль:** Ты — умный ассистент по планированию путешествий.
> **Задача:** Помогай пользователю находить интересные локации и отели.
> **Инструкция:** Если пользователь спрашивает о конкретном месте или отеле, ВСЕГДА используй инструмент `showPlace` для отображения карточки, вместо того чтобы просто описывать его текстом.

<br>

### 🥇 Pro Version (Продвинутая версия)

Подходит для сложных интерфейсов, где ИИ должен анализировать данные и решать, какой из нескольких компонентов отрендерить.

> **Роль (Role):** Ты — Senior UX-Ассистент туристического агентства.
>
> **Ситуация (Context):**
>
> - Цель: Предоставить пользователю максимально интерактивный и удобный опыт бронирования прямо в чате.
> - Доступные инструменты (UI-компоненты): `showPlace` (карточка места), `showFlightInfo` (авиабилеты), `showMap` (карта локации).
>
> **Задача (Task):**
>
> 1. Внимательно проанализируй поисковый запрос пользователя.
> 2. Если запрос связан с отелями или достопримечательностями, вызови `showPlace`.
> 3. Если пользователь ищет маршруты или перелеты, вызови `showFlightInfo`.
> 4. Не дублируй текстовым сообщением ту информацию, которую ты уже передал в UI-компонент.
>
> **Ограничения (Constraints):**
>
> - Убедись, что аргументы, передаваемые в функции, строго соответствуют требуемым типам (например, поле `rating` должно быть числом от 1 до 5).
> - Если данных для вызова компонента недостаточно, задай уточняющий вопрос коротким текстом.

## 🛠 Реализация: Интеграция Tambo SDK в код

Как это работает под капотом? Давайте настроим базовый React-проект.

### 1. Установка пакетов

Сначала установите необходимые зависимости. Мы используем Zod для безопасной валидации данных от ИИ.

```bash
npm install @tambo/sdk ai zod
```

### 2. Определение компонентов

Создайте обычный React-компонент, который ИИ будет возвращать пользователю.

```tsx
// components/PlaceCard.tsx
import { Card, Text, Button } from "@tambo/ui";

export function PlaceCard({ name, description, rating }) {
  return (
    <Card className="shadow-md hover:shadow-lg transition-shadow">
      <Text size="lg" weight="bold">
        {name}
      </Text>
      <Text className="text-yellow-500">⭐ {rating}</Text>
      <Text className="text-gray-600 mt-2">{description}</Text>
      <Button className="mt-4 w-full">Забронировать</Button>
    </Card>
  );
}
```

### 3. Настройка реестра (Registry)

Теперь нужно "объяснить" ИИ структуру данных вашего компонента через Zod. Это ключевой шаг: описания (`.describe()`) будут использоваться моделью как подсказки.

```tsx
// lib/tambo.ts
import { createTambo } from "@tambo/sdk";
import { PlaceCard } from "../components/PlaceCard";
import { z } from "zod";

export const tambo = createTambo({
  components: {
    showPlace: {
      component: PlaceCard,
      description:
        "Отображает интерактивную карточку с информацией о туристическом месте или отеле.",
      schema: z.object({
        name: z.string().describe("Название локации или отеля"),
        description: z
          .string()
          .describe("Краткое и привлекательное описание места"),
        rating: z.number().describe("Рейтинг заведения (число от 1 до 5)"),
      }),
    },
  },
});
```

### 4. Рендеринг UI в чате

Используйте провайдер Tambo в вашем клиентском коде для перехвата вызовов от ИИ и их автоматического рендеринга.

```tsx
// app/chat/page.tsx
import { useChat } from "ai/react";
import { TamboProvider, GenerativeUI } from "@tambo/sdk/react";
import { tambo } from "@/lib/tambo";

export default function ChatPage() {
  const { messages, input, handleInputChange, handleSubmit } = useChat();

  return (
    <TamboProvider registry={tambo}>
      <div className="flex flex-col max-w-2xl mx-auto p-4 space-y-4">
        {messages.map((m) => (
          <div
            key={m.id}
            className="p-4 rounded-xl bg-gray-50 border border-gray-100"
          >
            <strong className="text-blue-600">
              {m.role === "user" ? "Вы" : "ИИ-Ассистент"}:
            </strong>
            <div className="mt-3">
              {/* Именно здесь происходит магия: ИИ возвращает компонент, а Tambo его рендерит */}
              <GenerativeUI content={m.content} />
            </div>
          </div>
        ))}

        <form onSubmit={handleSubmit} className="flex gap-2 pt-4">
          <input
            value={input}
            onChange={handleInputChange}
            placeholder="Куда отправимся в этот раз?"
            className="flex-1 p-3 border rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
          />
          <button
            type="submit"
            className="px-6 py-3 text-white bg-blue-600 rounded-lg hover:bg-blue-700 font-medium"
          >
            Отправить
          </button>
        </form>
      </div>
    </TamboProvider>
  );
}
```

## 💡 Комментарий автора (Insight)

Generative UI — это настоящий сдвиг парадигмы во фронтенд-разработке. До появления специализированных инструментов вроде Tambo SDK разработчикам приходилось вручную парсить JSON-ответы, возвращаемые LLM, и прописывать бесконечные уязвимые `if/else` конструкции для рендеринга.

Tambo элегантно абстрагирует всю эту сложность. Комбинация строгой типизации Zod и нативных React-компонентов дает гарантию того, что ИИ не "сломает" фронтенд некорректными типами данных. Самое главное здесь — правильно настроить **системный промпт (Role & Task)** и сфокусироваться на описаниях полей в `z.object().describe()`. Чем детальнее вы опишете каждый `prop` в схеме Zod, тем увереннее и точнее ИИ будет генерировать ваш интерфейс. Это лучший способ быстро превратить линейный чат-бот в мощное микро-приложение.

## 🙋 Часто задаваемые вопросы (FAQ)

- **Q: Поддерживает ли Tambo SDK другие фреймворки, кроме React?**
  - A: На данный момент Tambo максимально оптимизирован под экосистему React (включая Next.js и Remix). Тем не менее, сам концепт Generative UI (генерация UI через Function Calling) активно развивается, и аналогичные паттерны уже появляются для Vue и Svelte.

- **Q: Что делать, если ИИ галлюцинирует и придумывает несуществующие пропсы?**
  - A: Именно для этого в связке работает валидатор Zod. Если языковая модель попытается передать пропс, которого нет в вашей схеме, парсер Tambo либо отбросит его, либо выбросит ошибку на уровне сервера, заставляя ИИ переписать JSON до того, как он обрушит клиентский интерфейс.

- **Q: Будет ли Generative UI стабильно работать с недорогими моделями вроде GPT-3.5?**
  - A: Для стабильного рендеринга UI критически важна безупречная поддержка `Function Calling` и следование формату JSON. Модели уровня GPT-4o, Claude 3.5 Sonnet или Gemini 1.5 Pro справляются с этим отлично. Младшие модели (включая локальные) могут путаться в сложных вложенных схемах, поэтому для них требуется более жесткий системный промпт с примерами (Few-Shot Prompting).
