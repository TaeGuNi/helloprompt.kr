# 복잡한 프롬프트 구조화하기: CREATE 프레임워크

AI 모델이 발전함에 따라 우리는 점점 더 복잡한 작업을 요청하게 됩니다. 단순한 질문에는 단순한 프롬프트로 충분하지만, 복잡한 비즈니스 로직이나 창의적인 작업을 지시할 때는 체계적인 접근이 필요합니다.

이 글에서는 프롬프트를 논리적으로 구조화하여 AI의 성능을 극대화하는 **CREATE 프레임워크**를 소개합니다.

## CREATE 프레임워크란?

CREATE는 효과적인 프롬프트 구성을 위한 6가지 핵심 요소의 약자입니다.

1.  **C**ontext (맥락): AI에게 역할과 상황을 부여합니다.
2.  **R**equest (요청): 수행해야 할 구체적인 작업을 명시합니다.
3.  **E**xamples (예시): 원하는 결과물의 예시(Few-shot)를 제공합니다.
4.  **A**djustments (조정): 톤, 스타일, 제약 조건을 설정합니다.
5.  **T**ype of Output (출력 형태): 결과물의 형식을 지정합니다 (표, 코드, 마크다운 등).
6.  **E**xtras (추가 사항): 엣지 케이스 처리나 특별한 지시사항을 포함합니다.

---

## 단계별 적용 가이드

### 1. Context (맥락 설정)

AI에게 페르소나를 부여하면 답변의 품질이 달라집니다.

> "당신은 10년 차 시니어 소프트웨어 엔지니어입니다. 레거시 코드를 현대적인 클린 코드로 리팩토링하는 전문 지식을 가지고 있습니다."

### 2. Request (구체적 요청)

모호함을 피하고 명확한 동사를 사용하세요.

> "아래 제공된 Python 함수를 분석하고, 가독성을 높이고 실행 속도를 최적화하는 방향으로 리팩토링해 주세요."

### 3. Examples (예시 제공)

AI는 예시를 통해 패턴을 학습합니다. 입력과 출력의 쌍을 보여주세요.

> **입력:**
> `def calc(x,y): return x+y`
>
> **출력:**
>
> ```python
> def add_numbers(a: int, b: int) -> int:
>     """두 숫자의 합을 반환합니다."""
>     return a + b
> ```

### 4. Adjustments (세부 조정)

답변의 톤앤매너를 결정합니다.

> "설명은 간결하고 전문적인 용어를 사용하되, 주니어 개발자도 이해할 수 있도록 친절한 어조를 유지하세요."

### 5. Type of Output (출력 형식)

결과물을 어떻게 받을지 지정합니다.

> "결과는 먼저 변경된 전체 코드를 코드 블록으로 제시하고, 그 아래에 변경 사유를 글머리 기호 목록으로 요약해 주세요."

### 6. Extras (추가 지침)

예상치 못한 상황을 방지합니다.

> "만약 코드에 보안 취약점이 발견되면, 리팩토링 외에도 별도의 경고 메시지를 출력해 주세요."

## 결론

CREATE 프레임워크를 사용하면 프롬프트 작성 시간을 줄이면서도 결과물의 품질을 일관되게 유지할 수 있습니다. 복잡한 문제를 해결해야 할 때, 막연하게 질문하기보다 CREATE의 구조에 맞춰 생각을 정리해 보세요. AI는 여러분이 설계한 만큼 똑똑하게 대답할 것입니다.
