## 📝 Stoat: LLM이 작성한 코드를 삭제하는 안티-코파일럿

- **🎯 추천 대상:** 레거시 코드에 고통받는 개발자, 코드 리뷰어, 시니어 엔지니어
- **⏱️ 소요 시간:** 1시간 → 5분 단축
- **🤖 추천 모델:** Claude 3.5 Sonnet, GPT-4o (코딩 특화 모델 권장)

- ⭐ **난이도:** ⭐⭐⭐☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"어제 짠 AI 코드, 오늘 보니 스파게티가 되어 있지 않나요? 이제 코드를 '생성'할 때가 아니라 '삭제'할 때입니다."_

2024년과 2025년이 'AI 코딩 어시스턴트'의 황금기였다면, 2026년은 단연코 'AI 클리너'의 해가 될 것입니다. 최근 글로벌 개발자 커뮤니티를 뜨겁게 달구고 있는 **Stoat**는 기존 AI 코딩 도구들의 궤도를 완전히 역행합니다. GitHub Copilot이나 Cursor가 "더 많은 코드"를 빠르게 찍어내는 데 혈안이 되어 있다면, Stoat의 철학은 단 하나, **"코드를 가차 없이 지우는 것"**입니다.

LLM 기반 코딩 도구의 대중화는 분명 폭발적인 생산성 향상을 가져왔습니다. 하지만 그 이면에는 끔찍한 부작용이 도사리고 있습니다. AI가 무심코 뱉어낸 코드는 종종 필요 이상으로 장황하며, 의미 없는 방어적 로직과 중복된 보일러플레이트로 가득 차 있습니다. 이 포스트에서는 Stoat의 파격적인 철학을 벤치마킹하여, 여러분이 매일 사용하는 LLM을 '피도 눈물도 없는 잔혹한 코드 청소부'로 완벽하게 탈바꿈시키는 마법의 프롬프트를 공개합니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **폭증하는 기술 부채:** AI 코딩 어시스턴트의 남용으로 인해 코드베이스가 걷잡을 수 없이 비대해지고 있습니다.
2. **잔혹한 코드 다이어트:** Stoat 프롬프트는 의미론적 중복 제거, 보일러플레이트 축소, 데드 코드 삭제를 통해 코드를 날렵하게 깎아냅니다.
3. **즉각적인 리팩토링:** 복사 및 붙여넣기만으로 여러분의 LLM을 가장 깐깐한 시니어 코드 리뷰어로 변신시킬 수 있습니다.

## 🚀 해결책: "Stoat 안티-코파일럿 다이어트 프롬프트"

### 🥉 Basic Version (기본형)

빠르게 코드 라인 수(LOC)를 줄이고 싶을 때 사용하세요.

> **역할:** 너는 세상에서 가장 깐깐한 '안티-코파일럿(Anti-Copilot)' 시니어 개발자야.
> **요청:** 아래 코드를 리뷰하고, 기능은 100% 동일하게 유지하면서 코드 라인 수를 최소한으로 줄여서 다시 작성해줘. 불필요한 추상화, 중복 로직, 데드 코드는 가차 없이 삭제해.
> **코드:** `[여기에 리팩토링할 코드 붙여넣기]`

### 🥇 Pro Version (전문가형)

프로덕션 레벨의 코드에서 안전하게 기술 부채를 청산하고 싶을 때 사용하세요.

> **역할 (Role):** 너는 코드 다이어트의 장인이자, 불필요한 코드를 혐오하는 'Stoat(안티-코파일럿)' 수석 엔지니어입니다. "최고의 코드는 코드가 없는 것(The best code is no code)"이라는 철학을 맹신합니다.
>
> **상황 (Context):**
>
> - 배경: 주니어 개발자와 AI 코딩 어시스턴트가 무분별하게 작성하여 비대해진 `[사용 언어 및 프레임워크 입력, 예: React, TypeScript]` 프로젝트입니다.
> - 목표: 코드의 가독성을 높이고, 유지보수성을 극대화하며, 전체 라인 수(LOC)를 극단적으로 줄이는 것입니다.
>
> **요청 (Task):**
>
> 1. **Semantic De-duplication (의미론적 중복 제거):** 겉모습은 달라도 같은 역할을 하는 로직을 찾아 철저하게 통합하세요.
> 2. **Boilerplate Reduction (보일러플레이트 축소):** 최신 언어 문법을 활용하여 장황한 패턴을 완벽히 제거하세요.
> 3. **Dead Code Hunting (죽은 코드 사냥):** 호출되지 않거나 불필요한 방어적 코딩, 과도한 타입 선언을 모두 삭제하세요.
> 4. 리팩토링된 코드와 함께 **삭제된 코드의 비율(%)** 및 **주요 최적화 포인트**를 리포트로 상세히 작성하세요.
>
> **제약사항 (Constraints):**
>
> - 기존 코드의 비즈니스 로직과 시간/공간 복잡도를 훼손하거나 저하시켜서는 절대 안 됩니다.
> - 출력 형식은 마크다운 코드블럭을 사용하고, 리팩토링 전후의 주요 차이점은 불릿 포인트로 깔끔하게 정리해 주세요.
> - 확실하지 않은 최적화는 억지로 진행하지 말고 반드시 코멘트를 남겨주세요.
>
> **입력 변수:**
>
> - 언어/프레임워크: `[프레임워크 이름 입력, 예: Next.js]`
> - 원본 코드:
>
>   `[여기에 최적화할 스파게티 코드를 붙여넣으세요]`

## 💡 작성자 코멘트 (Insight)

이 프롬프트는 최초 설계 당시, **"어떻게 하면 AI가 무책임하게 짜 놓은 장황한 코드를, AI 스스로 부끄러워하며 흔적도 없이 지우게 만들 수 있을까?"**라는 아주 본질적인 고민에서 출발했습니다. 우리는 코드를 생성하는 방법론에는 익숙하지만, 정작 그 코드를 어떻게 우아하게 덜어낼 것인가에 대해서는 간과하는 경향이 있습니다.

프롬프트의 핵심은 바로 **페르소나의 극적인 전환**에 있습니다. AI에게 단순히 "코드를 리팩토링해 줘"라고 모호하게 요청하면, 모델은 종종 불필요한 디자인 패턴을 덧붙이거나 새로운 유틸리티 함수를 생성하여 오히려 코드베이스를 더 복잡하게 만듭니다. 하지만 컨텍스트를 극도로 제한하고 **'안티-코파일럿(Anti-Copilot)'**이라는 명확하고도 파괴적인 페르소나를 부여하는 순간, AI의 쓸데없는 창의성은 완벽하게 통제됩니다. 오직 **'최적화'**와 **'삭제'**라는 두 가지 목표에만 맹렬하게 집중하게 되는 것입니다.

실제로 현업 프로젝트에서 유지보수가 불가능할 정도로 뚱뚱해진 수백 줄짜리 레거시 컴포넌트를 이 프롬프트에 던져 넣었을 때, 비즈니스 로직의 손실 없이 전체 코드 라인이 40% 이상 깔끔하게 증발해 버리는 놀라운 마법을 직접 경험했습니다. 변수 통제법 또한 매우 직관적입니다. `[사용 언어 및 프레임워크 입력, 예: React, TypeScript]` 부분에 팀 내 기술 스택을 명시하기만 하면, 해당 언어의 최신 문법(예: ES6+의 옵셔널 체이닝, 널 병합 연산자 등)을 극한으로 활용하여 보일러플레이트를 썰어버립니다. 이 프롬프트는 밤낮없이 쏟아지는 PR(Pull Request)의 늪에서 허우적대는 시니어 엔지니어들의 **코드 리뷰 피로도를 획기적으로 줄여주는 가장 날카로운 무기**가 될 것입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 코드를 너무 많이 지워서 숨겨진 버그가 터지면 어떡하나요?**
  - A: Pro 프롬프트의 **'제약사항(Constraints)'**에 비즈니스 로직 보존을 강력하게 명시해 두었지만, AI의 결과물을 100% 맹신해서는 안 됩니다. 리팩토링된 코드를 병합하기 전에 반드시 기존에 작성된 단위 테스트(Unit Test)나 E2E 테스트를 돌려, 가장자리(Edge) 케이스가 누락되지 않았는지 **2차 교차 검증**을 거치는 것을 강력히 권장합니다.

- **Q: 어떤 언어 모델에서 가장 성능이 좋나요?**
  - A: 코드의 복잡한 문맥 구조를 깊이 파악하고, 의존성을 해치지 않으면서 리팩토링하는 고난도의 작업입니다. 따라서 논리적 추론 능력이 현재 시장에서 가장 압도적인 **Claude 3.5 Sonnet**이나 프로그래밍에 특화된 **GPT-4o**의 사용을 적극 추천합니다. 일반적인 텍스트 생성형 모델은 코드 구조를 망가뜨릴 위험이 높습니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **역할 역전 (Role Reversal):** AI에게 '코드를 무한히 뱉어내는 순종적인 조수'가 아닌, '불필요한 코드를 혐오하는 깐깐한 수석 엔지니어' 역할을 강제로 부여하여 기존의 관성적인 생성 패턴을 완전히 뒤집어 버렸습니다.
2. **구체적인 타격 지점 설정 (Targeting):** 의미론적 중복 제거, 보일러플레이트 축소, 데드 코드 사냥이라는 **3가지 명확한 액션 아이템(Action Item)**을 지시함으로써, AI가 코드를 두루뭉술하게 건드려 오히려 의존성을 망가뜨리는 치명적인 사고를 사전에 차단했습니다.
3. **심리적 철학 주입:** **"The best code is no code (최고의 코드는 코드가 없는 것이다)"**라는 극단적이고 도발적인 슬로건을 페르소나에 주입하여, 무조건적인 코드 라인 수 감축을 절대적인 지상 과제로 삼도록 집요하게 유도했습니다.

## 📊 증명: Before & After

### ❌ Before (입력: AI가 무분별하게 짠 React 코드)

```typescript
import React, { useState, useEffect } from 'react';

const UserProfile = ({ user }) => {
  const [userName, setUserName] = useState('');
  const [userAge, setUserAge] = useState(0);

  useEffect(() => {
    if (user && user.name) {
      setUserName(user.name);
    }
  }, [user]);

  useEffect(() => {
    if (user && user.age) {
      setUserAge(user.age);
    }
  }, [user]);

  const handleNameClick = () => {
    console.log("Name clicked");
  }

  return (
    <div>
      {user ? (
        <div>
          <h1 onClick={handleNameClick}>{userName ? userName : 'Unknown'}</h1>
          <p>Age: {userAge ? userAge : 'N/A'}</p>
        </div>
      ) : (
        <div>Loading...</div>
      )}
    </div>
  );
};

export default UserProfile;
```

### ✅ After (결과: Stoat 프롬프트 적용 후)

```typescript
import React from 'react';

const UserProfile = ({ user }) => {
  // Early return으로 불필요한 조건부 렌더링 깊이 제거
  if (!user) return <div>Loading...</div>;

  return (
    <div>
      <h1 onClick={() => console.log("Name clicked")}>
        {user.name || 'Unknown'}
      </h1>
      <p>Age: {user.age || 'N/A'}</p>
    </div>
  );
};

export default UserProfile;
```

> **[Stoat 리포트]**
>
> - **코드 감소율:** 원본 대비 약 **55% 삭제 완료** (36줄 → 16줄)
> - **주요 최적화:** 불필요한 `useState` 및 `useEffect` 상태 동기화 완벽 제거, 얼리 리턴(Early Return) 패턴 적용을 통한 Depth 감소, 단축 평가(`||`)를 활용한 삼항 연산자 보일러플레이트 극단적 축소.

## 🎯 결론

코드를 끝없이 찍어내는 것은 이제 주니어 개발자든, 어떤 기초적인 AI든 누구나 할 수 있는 가장 쉬운 일이 되었습니다. 하지만 거미줄처럼 파편화된 로직의 맥락을 정확히 꿰뚫어 보고, 불필요한 코드를 날카롭게 걷어내어 시스템의 본질만 남기는 것은 여전히 **최상위 수준의 아키텍처 통찰력**을 요구합니다.

Stoat의 파괴적인 철학을 오롯이 담아낸 이 프롬프트를 통해, 무겁고 둔해진 여러분의 코드베이스에 강력한 극약 처방을 내려보세요. 군더더기 없이 얇고 가벼워진 코드가 유지보수의 악몽을 끝내고, 여러분의 쾌적한 퇴근 시간을 앞당겨 줄 것입니다. 주저하지 말고, 이제 과감하게 지워버리세요! 🍷
