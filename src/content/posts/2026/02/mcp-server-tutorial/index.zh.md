来到 2026 年，**MCP（Model Context Protocol，模型上下文协议）**已然确立了其作为 LLM 与现实业务数据连接的“事实标准（De-facto standard）”。你可以将其视作专为 AI 模型量身定制的“通用 USB 接口”。与其每次开启对话都手动把冗长的背景知识“喂”给 AI，不如花几分钟时间建立一次长效的服务器连接，让模型在需要时能够自动、实时地调取你的本地文件、数据库乃至内部核心 API。

目前，开源社区中已涌现出众多直接对接 GitHub 或本地文件系统（Filesystem）的优秀 MCP 服务器，但这仅仅是冰山一角。MCP 真正的杀手锏在于，它能够将**“团队独有的业务逻辑与私有核心数据”**与大语言模型进行无缝的深度对接。

今天，我们完全不需要从零开始手敲那些枯燥乏味的样板代码（Boilerplate）。本文为你准备了一段经过实战检验的高效提示词，能直接指挥 AI 为你生成核心代码。只需短短 5 分钟，你就能搭建出一个完美运行、基于 TypeScript 的定制化 MCP 服务器脚手架（Scaffolding）。

## 💡 作者点评 (Insight)

这段提示词的核心魔力在于，它没有宽泛地要求 AI “写一段代码”，而是强制约束了 MCP 服务器必须遵循的**核心协议规范（例如完整实现 ListTools 和 CallTool 请求）**。

亲自上手搭建并运行 MCP 服务器，最大的收益莫过于获得**“绝对的隐私安全（Privacy）”**。当你使用常规的网页版插件时，查询请求和敏感数据不可避免地会流经外部服务器；而通过直连本地 PostgreSQL 数据库或内部 API 的定制版 MCP 服务器，数据的调用权限将完全掌握在你自己的手中。你再也不必冒着泄露商业机密的风险，将内部核心文档或客户数据手动粘贴到公开的网页聊天框里了。

强烈建议你先用这段提示词创建一个简单的**“只读（Read-only）”**工具来练手，例如读取本地的某个日志文件。当你彻底摸透了它的底层运作机制后，就可以逐步向外扩展，加入修改数据库、调用复杂业务接口等高级功能，最终将其打造成一个专属于你个人的超级 AI 助理。

## 🙋 常见问题 (FAQ)

- **Q: 如果我完全不懂编程，也能使用这段提示词吗？**
  - A: 完全可以！你只需在 `[核心功能]` 的括号内用自然语言清晰描述你的需求（例如：“写一个能查询北京今天天气的功能”），Claude 就会自动为你生成符合规范的代码。你只需将生成的代码保存并配置运行即可。

- **Q: 生成的服务器代码该如何运行并连接到 AI 模型？**
  - A: 首先，将生成的代码保存为 `index.ts` 并编译运行。接着，在你的 Claude Desktop 配置文件（`claude_desktop_config.json`）的 `mcpServers` 字段下添加该脚本的启动路径。最后重启客户端，即可完成自动连接。

- **Q: 这样搭建出来的服务器只能专属 Claude 吗？**
  - A: 并非如此。Model Context Protocol 是一个完全开放的行业标准。任何支持 MCP 协议的客户端或平台（例如 Ollama、Cursor 甚至是未来的其他 AI 生产力工具），都可以无缝接入并调用你所搭建的服务器。

## 🧬 提示词解剖 (Why it works?)

1. **强大的约束机制（Constraints）：** 强制要求采用 `@modelcontextprotocol/sdk` 与 `zod`，并明确指定需实现的核心协议接口（`ListToolsRequestSchema`、`CallToolRequestSchema`）。这从根本上杜绝了 AI 因不熟悉 MCP 最新规范而产生的“幻觉（Hallucination）”。
2. **极简的单文件输出：** 明确规定仅输出一个 `index.ts` 文件，大刀阔斧地砍掉了不必要的项目结构搭建与冗长的文字解释，极大地缩短了用户的“获得价值时间（Time-to-Value）”，真正做到了复制即用。
3. **强制防御性编程：** 预先规定核心逻辑中必须包裹 `try/catch`，有效防止 AI 生成脆弱的代码，确保即使在面对未知的异常输入时，服务器进程依然能够坚如磐石地稳定运行。
