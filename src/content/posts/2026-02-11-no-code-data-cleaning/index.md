_"서울시 강남구, 강남구 삼성동, Seoul Gangnam... 제각각인 주소록 정리하다가 퇴근 시간 놓치셨나요?"_

데이터 분석의 8할은 전처리(Pre-processing)라고 합니다. 하지만 우린 데이터 과학자가 아니잖아요? 파이썬 판다스(Pandas) 코드를 짤 필요 없습니다. AI에게 "예시"만 몇 개 던져주면, 알아서 규칙을 찾아내고 정렬해줍니다.

---

## ⚡️ 3줄 요약 (TL;DR) {#tl-dr}

1.  **패턴 인식(Pattern Recognition):** AI는 정규표현식(Regex)의 신입니다. 규칙을 말로 설명하지 말고 예시를 보여주세요.
2.  **퓨샷 러닝(Few-Shot Learning):** "이렇게 바꿔줘"라고 3개 정도만 보여주면 나머지는 알아서 합니다.
3.  **포맷 통일:** CSV, 마크다운 표 등 원하는 출력 형식을 지정하면 그대로 뱉어냅니다.

---

## 🚀 해결책: "데이터 클리너(Data Cleaner) 프롬프트" {#data-cleaner}

### 🥉 Basic Version (기본형)

간단한 텍스트 변환에 사용합니다.

> **역할:** 너는 `[데이터 분석가]`야.
> **요청:** 아래 목록에서 휴대폰 번호만 추출해서 `010-xxxx-xxxx` 형식으로 통일해줘.
>
> `[데이터 목록]`

<br>

### 🥇 Pro Version (전문가형)

복잡한 규칙과 오탈자가 섞인 데이터를 완벽하게 구조화합니다.

> **역할 (Role):** 너는 데이터 전처리(Data Pre-processing) 전문가야.
>
> **상황 (Context):**
>
> - 배경: 고객들이 손으로 입력한 주소와 연락처 데이터가 있어. 오타도 있고, 형식도 제각각이야.
> - 목표: CRM에 업로드할 수 있도록 표준 포맷으로 변환(Normalization)하는 것.
>
> **요청 (Task):**
> 아래 `[Input Data]`를 읽고, `[Output Format]`에 맞춰 변환해줘.
>
> **변환 규칙 (Rules):**
>
> 1. **주소:** '시/도' '구/군' 단위까지 표준 행정구역 명칭으로 통일해. (예: 서울 -> 서울특별시, 강남 -> 강남구)
> 2. **전화번호:** 특수문자 제거 후 `010-0000-0000` 형식으로 통일. 없는 경우 'NULL'로 표기.
> 3. **이름:** 공백 제거.
>
> **예시 (Few-Shot Examples):**
>
> - 입력: "홍 길 동 / 010 1234 5678 / 서울 강남 삼성동"
> - 출력: | 홍길동 | 010-1234-5678 | 서울특별시 강남구 삼성동 |
> - 입력: "김철수 / 010-9876-4321 / 경기 성남 분당"
> - 출력: | 김철수 | 010-9876-4321 | 경기도 성남시 분당구 |
>
> **제약사항 (Constraints):**
>
> - 결과는 반드시 CSV 형식(콤마로 구분) 코드블록으로 출력해.
> - 해석 불가능한 데이터는 맨 뒤에 'Error' 태그를 붙여줘.
>
> **[Input Data]:**
> `[여기에 더러운(?) 데이터 붙여넣기]`

## 💡 작성자 코멘트 (Insight) {#insight}

이 프롬프트의 강력함은 **'Few-Shot Prompting(예시 제공)'**에서 나옵니다. 규칙을 말로 설명하는 것보다, "A는 B로 바꿔"라고 예시를 2~3개 보여주는 것이 훨씬 정확합니다. ChatGPT의 경우 'Advanced Data Analysis' (구 Code Interpreter) 기능을 켜고 엑셀 파일을 통째로 업로드하면, 내부적으로 파이썬 코드를 짜서 처리해주므로 수만 건의 데이터도 처리 가능합니다.

## 🙋 자주 묻는 질문 (FAQ) {#faq}

- **Q: 개인정보 올려도 되나요?**
  - A: 절대 안 됩니다! 실명이나 실제 전화번호는 마스킹(홍\*동) 처리하거나, 로컬(Local) LLM을 사용하세요. 회사 보안 규정을 꼭 확인하세요.

- **Q: 데이터가 너무 많아요.**
  - A: 50건씩 끊어서 하거나, ChatGPT Plus 유저라면 엑셀 파일 업로드를 이용하세요.

## 🧬 프롬프트 해부 (Why it works?) {#why-it-works}

1.  **Standardization (표준화):** '서울특별시', '경기도' 등 표준 명칭을 지정하여 데이터의 일관성을 확보했습니다.
2.  **Error Handling:** 해석 불가능한 데이터를 'Error'로 태깅하여, 나중에 사람이 검수할 수 있게 했습니다.
3.  **Format Specification:** CSV로 출력을 강제하여 엑셀로 바로 복사할 수 있게 했습니다.
