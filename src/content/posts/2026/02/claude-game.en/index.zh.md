## 🧬 提示词剖析 (Why it works?)

1. **强制模块化 (Modularity)：** 在提示词中从一开始就强制要求分离核心文件，提前搭建好了架构骨架。这样即使后续代码量激增，也能避免沦为难以维护的意大利面条式代码，极大地方便了后续的迭代与功能扩展。
2. **明确的视觉反馈 (Visual Indicator)：** 并没有仅仅局限于“实现鼠标拖拽”这个干瘪的动作，而是明确要求添加“显示力度和方向的辅助线”。这一细节极大地提升了游戏的操控手感 (UX) 和玩家的直观体验。
3. **兼顾扩展性 (Level Design)：** 刻意引导 AI 采用“数组形式的关卡数据配置”，而不是将单一关卡硬编码写死在逻辑里。这种做法建立了一个数据驱动的底层结构，让开发者在未来能够轻松、无上限地添加新关卡。

### ✅ After (应用专业版提示词后生成的物理逻辑示例)

```javascript
// Generated by Claude Code
update() {
  this.velocity.x *= 0.98; // 摩擦力：模拟草地阻力，使球速缓慢衰减
  this.velocity.y *= 0.98;

  this.x += this.velocity.x;
  this.y += this.velocity.y;

  // 画布墙壁碰撞处理（自然反弹）
  if (this.x < 0 || this.x > canvas.width) {
    this.velocity.x *= -1; // 翻转 X 轴方向
  }
  if (this.y < 0 || this.y > canvas.height) {
    this.velocity.y *= -1; // 翻转 Y 轴方向
  }
}
```

_结果：Claude Code 准确运用了阻尼 (Damping) 数学概念（使速度随时间推移按比例降低），完美还原了“高尔夫球场草地的真实摩擦力”。_
