# 📝 Паттерны генеративного UI (Generative UI)

- **🎯 Рекомендуется для:** Фронтенд-разработчиков, UX/UI дизайнеров, продакт-менеджеров
- **⏱️ Экономия времени:** Часы ручного проектирования → Мгновенная кодогенерация
- **🤖 Рекомендуемые модели:** Gemini 2.5 Pro, GPT-4o, Claude 3.5 Sonnet

- ⭐ **Сложность:** ⭐⭐⭐⭐☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Применимость:** ⭐⭐⭐⭐⭐

> _"Что, если бы интерфейс вашего приложения создавался сам, на лету угадывая желания пользователя, вместо того чтобы заставлять его блуждать по лабиринтам статичных меню?"_

Десятилетиями фронтенд-разработка подчинялась строгому контракту: разработчики рисуют экраны, а пользователи по ним кликают. Мы создаем навигационные панели, бургер-меню и статичные дашборды, надеясь, что наша информационная архитектура совпадет с ментальной моделью пользователя. Но генеративный UI (Generative UI) меняет правила игры. Это парадигмальный сдвиг и переход от "создания экранов" к "созданию интеллектуальных систем, которые сами собирают экраны под контекст".

---

## ⚡️ 3 тезиса (TL;DR)

1. **Смерть статичного меню:** Интерфейсы теперь генерируются под конкретный запрос пользователя в реальном времени, избавляя от лишних кликов.
2. **Оркестрация ИИ:** Большие языковые модели (LLM) возвращают не просто текст, а структурированные данные (например, JSON), которые клиент превращает в готовые UI-виджеты.
3. **Безопасность компонентов:** ИИ не "галлюцинирует" сырым HTML-кодом; он собирает интерфейс из вашей строго типизированной библиотеки (Registry) заранее написанных безопасных компонентов.

---

## 🚀 Решение: "Промпт для генерации UI-компонентов"

### 🥉 Basic Version (Базовая версия)

Используйте для быстрой генерации простой структуры интерфейса при прототипировании.

> **Роль:** Ты `[UX/UI Инженер]`.
> **Задача:** Сгенерируй JSON-схему для `[виджета графика расходов за февраль]`.

\

### 🥇 Pro Version (Профессиональная версия)

Используйте для глубокой интеграции в production-ready системы генеративного UI (например, при работе с Vercel AI SDK).

> **Роль (Role):** Ты `[Senior Frontend Architect]`, специализирующийся на Generative UI и React.
>
> **Контекст (Context):**
>
> - Фон: `[Пользователь запрашивает выписку по банковскому счету за прошлый месяц через чат-интерфейс]`
> - Цель: `[Вернуть структурированный JSON, который фронтенд мгновенно превратит во взаимодействующий виджет графика и таблицу транзакций]`
>
> **Задача (Task):**
>
> 1. Проанализируй интент пользователя и выбери подходящие компоненты из доступной библиотеки.
> 2. Сгенерируй валидный JSON, соответствующий требуемой схеме пропсов (props) для каждого выбранного компонента.
> 3. `[Дополнительные данные]` - используй реалистичные мок-данные для заполнения графика, если реальные вытянуть не удалось.
>
> **Ограничения (Constraints):**
>
> - Ты должен возвращать ТОЛЬКО валидный JSON. Никакого вступительного или пояснительного текста.
> - Доступные компоненты строго ограничены списком: `BarChart`, `TransactionTable`, `ExportButton`.
>
> **Предупреждение (Warning):**
>
> - Не придумывай компоненты, которых нет в списке (критично для предотвращения галлюцинаций). Если нужного виджета для ответа нет, используй стандартный `MarkdownMessage`.

## 💡 Комментарий автора (Insight)

Генеративный UI — это не просто красивый трюк для презентаций, это логичное эволюционное развитие декларативных фреймворков (React, Vue) в эпоху вероятностного искусственного интеллекта. В реальных production-проектах самая большая инженерная сложность заключается не в том, чтобы заставить ИИ выдать правильный JSON, а в том, чтобы **стримить** этот JSON (Streaming Hydration). Вы должны добиться того, чтобы интерфейс "проявлялся" на экране плавно, компонент за компонентом, вместе с генерируемым текстом, сохраняя у пользователя ощущение мгновенного отклика. Я настоятельно рекомендую начинать с внедрения небольших умных виджетов (например, карточка прогноза погоды или микро-дашборд) внутри существующего UI, прежде чем пытаться сделать всё ваше приложение полностью генеративным.

## 🙋 Часто задаваемые вопросы (FAQ)

- **Q: Безопасно ли позволять ИИ самостоятельно генерировать интерфейс для пользователей?**
  - A: Абсолютно безопасно, если вы используете архитектурный паттерн "Component Registry" (Реестр компонентов). ИИ не пишет исполняемый код, он лишь передает аргументы (props) в уже написанные, стилизованные и проверенные вами React-компоненты.

- **Q: Какие инструменты и библиотеки лучше всего подходят для реализации этого подхода в 2026 году?**
  - A: Обратите пристальное внимание на Vercel AI SDK (в частности, на функционал `streamUI`). На данный момент это индустриальный стандарт и самый мощный инструмент для создания генеративных интерфейсов на базе React и Next.js.

## 🧬 Анатомия промпта (Why it works?)

1.  **Строгие ограничения вывода (Constraints):** Требование выдавать исключительно JSON критически важно для машинной обработки ответа. Любой "вежливый" лишний текст от нейросети сломает парсер на клиенте и приведет к ошибке рендеринга.
2.  **Ограничение галлюцинаций через Registry:** Четкий список `Доступных компонентов` не дает нейросети проявлять излишнюю креативность и придумывать несуществующие элементы интерфейса, надежно защищая ваше приложение от падений (Runtime Errors).

## 📊 Доказательство: Before & After

### ❌ Before (Традиционный подход)

```text
Пользователь: "Покажи мои расходы за февраль."
Бот: "Чтобы посмотреть ваши расходы за февраль, пожалуйста, перейдите в главное Меню -> раздел Выписки -> Выберите месяц 'Февраль 2026' -> Нажмите кнопку 'Показать график'."
```

### ✅ After (Generative UI)

```json
{
  "type": "ui_component",
  "component": "ExpenseDashboard",
  "props": {
    "month": "February 2026",
    "chartData": [
      { "category": "Еда и рестораны", "amount": 450 },
      { "category": "Транспорт", "amount": 120 },
      { "category": "Подписки", "amount": 45 }
    ],
    "showExport": true
  }
}
// ⚡️ Результат: Фронтенд мгновенно рендерит красивый интерактивный график прямо в потоке чата, без необходимости куда-то кликать!
```
