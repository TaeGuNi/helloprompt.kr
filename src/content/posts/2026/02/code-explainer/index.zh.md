## 📝 这代码什么意思？亲切的代码解说员

- **🎯 推荐对象:** 所有人，尤其是初中级开发者
- **⏱️ 所需时间:** 5 分钟
- **🤖 推荐模型:** 所有大语言模型（ChatGPT, Claude, Gemini 等）

- ⭐ **难度:** ⭐⭐☆☆☆
- ⚡️ **有效性:** ⭐⭐⭐⭐⭐
- 🚀 **实用性:** ⭐⭐⭐⭐☆

> _"代码明明跑通了…… 但它到底是怎么跑通的？"_

这堪称每一位开发者职业生涯中的“终极未解之谜”。从 Stack Overflow 上复制粘贴来的代码虽然神奇地生效了，可一旦需求变更需要修改，却往往让人无从下手；又或者接手了前任同事留下的毫无注释、犹如“天书”般的祖传代码（Legacy Code），那种绝望感简直令人窒息。别担心，这套提示词将化身为你的**专属 AI 导师**，帮你把晦涩难懂的代码瞬间“翻译”成通俗易懂的人类语言，彻底粉碎阅读代码的认知障碍。

---

## ⚡️ 3 行总结 (TL;DR)

1. **宏观解析**：用极简的人类语言，精准概括代码的整体架构与核心业务逻辑。
2. **微观剖析**：为令人费解的核心代码块自动生成详尽的**逐行注释 (Comment)**。
3. **知识扩展**：额外拆解并科普初学者极易踩坑的进阶语法与冷门内置函数。

---

## 🚀 解决方案："代码降维打击器"

### 🥉 Basic Version (基础版)

当你时间紧迫，只需快速把握代码大意时，请使用此版本。

> **角色:** 你是一位`[极其耐心的高级研发工程师]`。
> **请求:** 请将下面这段复杂的代码逐行添加注释，并用初学者能听懂的语言解释它的运行原理。

### 🥇 Pro Version (专家版)

当你需要死磕代码细节，甚至为接下来的重构做准备时，请复制以下 **PROMPT**。

> **角色 (Role):** 你是一位`[拥有10年经验、极其耐心的高级研发工程师]`。
>
> **背景 (Context):**
>
> - 现状：我接手了一段完全没有注释的复杂代码，无法确切理解它的业务作用与底层逻辑。
> - 我的水平：我是`[初级/中级]`水平的开发者。
>
> **任务 (Task):**
>
> 1. **[逻辑摘要]**: 请用不超过 3 句话，通俗易懂地总结这段代码的核心业务逻辑。
> 2. **[逐行剖析]**: 请为代码的每一行（或每一个关键逻辑块）添加极其详尽的**中文注释**，并重新输出完整的代码。
> 3. **[语法避坑]**: 如果代码中使用了高阶语法、特定的设计模式或冷门库函数（例如：列表推导式、闭包、解构赋值等），请单独提取出来并进行通俗解释。
>
> **约束条件 (Constraints):**
>
> - 绝对不要使用生僻晦涩的学术名词，必须用日常生活中的生动例子来打比方。
> - 必须明确指出关键变量（如 `x`, `result` 等）所代表的具体业务含义。
>
> **待解析代码:**
> `[在此处粘贴你的代码]`

## 💡 作者笔记 (Insight)

在真实的商业开发环境中，我们绝大部分的时间其实不是在“写代码”，而是在“读代码”。特别是接手历史悠久的老项目时，去阅读那些既缺乏文档又没有注释的遗留代码（Legacy Code），简直是一场精神折磨。

这套提示词的真正杀手锏在于：它不仅让 AI 给出大段的文字解释，更是强制 AI **“把注释直接写进代码里并重新输出”**。这意味着你可以直接将带有详尽注释的代码原封不动地复制回 IDE（集成开发环境）中，极大地降低了上下文切换带来的认知负担。此外，强制要求 AI 解释“高阶语法”，能让你在梳理业务逻辑的同时，顺便恶补薄弱的编程基础，实现业务与技术的双丰收。

## 🙋 常见问题 (FAQ)

- **Q: 如果代码极其庞大，超出了 AI 的上下文长度限制怎么办？**
  - A: 强烈建议将代码按函数 (Function) 或类 (Class) 进行物理拆分，每次只“喂”给 AI 一个核心模块。或者，你可以先让 AI 解释入口函数（Main Function），在理清整体调用链后，再像剥洋葱一样逐个攻破底层的子函数。

- **Q: AI 自动补充的注释会改变代码原有的运行逻辑吗？**
  - A: 绝对不会。只要我们在提示词中明确约束 AI “仅添加注释，严禁修改原有变量名和执行流”，代码的运行结果就是 100% 一致的。不过，在将代码替换回生产环境前，跑一遍单元测试（Unit Test）永远是工程师应有的基本素养。

## 🧬 提示词解剖 (Why it works?)

1. **精准锚定受众认知水平**: 通过设定 `[初级/中级]` 开发者水平，成功迫使 AI 放下“资深专家”的高冷姿态，转而使用接地气的生活比喻，而非干巴巴、令人昏昏欲睡的官方文档术语。
2. **结构化输出约束 (Task)**: 将复杂的解析过程强行拆分为“摘要”、“逐行注释”和“语法解析”三大步骤，完美契合了人类大脑由浅入深、由宏观到微观的认知规律。

## 📊 证明: Before & After

### ❌ Before (输入 - Python)

```python
result = [x * 2 for x in range(10) if x % 2 == 0]
```

### ✅ After (结果)

**[逻辑摘要]**
这段代码的核心作用是：从 0 到 9 的数字堆里精准挑出所有的“偶数”，把它们各自放大两倍后，打包塞进一个新的列表里。

**[逐行剖析]**

```python
# 遍历 0 到 9 的数字 (range(10))，每次取出一个数字赋值给 x。
# 条件判断 (if)：如果 x 除以 2 的余数是 0（说明 x 是偶数）。
# 执行操作：将符合条件的 x 乘以 2，然后放入最终的列表中。
result = [x * 2 for x in range(10) if x % 2 == 0]
```

**[语法避坑: 列表推导式 List Comprehension]**
这是 Python 语言中极其经典的“语法糖”。它把原本需要写 4 行的 `for` 循环和 `if` 判断，优雅地压缩到了短短 1 行代码中。

- 核心公式结构: `[操作表达式 for 变量 in 集合 if 筛选条件]`
