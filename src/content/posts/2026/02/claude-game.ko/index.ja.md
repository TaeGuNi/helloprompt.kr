# ⛳️ Claude Codeでゴルフゲームを作成：AIゲーム開発の新時代

- **🎯 おすすめの対象:** ゲーム開発の初心者、フロントエンドエンジニア、AIコーディングアシスタントの実践的な活用法を模索している実務者
- **⏱️ 所要時間:** 1時間 → 10分に短縮（プロトタイプ構築時）
- **🤖 おすすめのモデル:** Claude 3.7 Sonnet（Claude Code環境）

- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「複雑な物理演算やCanvasのレンダリングを、もうゼロからコーディングする必要はありません。AIエージェントが、あなたにとって完璧なシニア開発パートナーとして機能します。」_

ゲーム開発のパラダイムは今、根本的な転換期を迎えています。かつては2D物理エンジンの実装やイベントリスナーの管理だけでも数日を要していましたが、現在では**Claude Code**のような自律型AIコーディングエージェントの登場により、その技術的なハードルは事実上消滅しました。単なるコードの自動補完にとどまらず、プロジェクト全体のアーキテクチャを理解し、主体的にコーディングを行うClaude Codeを活用することで、わずか1時間でブラウザベースの2Dゴルフゲームを完成させる実践的なプロセスを、段階的なプロンプトとともに大公開します。

---

## ⚡️ 3行まとめ (TL;DR)

1. **AIエージェントの進化:** Claude Codeは単なるコードスニペットの生成にとどまらず、物理演算（速度や摩擦など）からレンダリングロジックに至るまで、ゲームアーキテクチャ全体を主導的に設計します。
2. **圧倒的な生産性の向上:** 煩雑なマウスイベントの処理や衝突計算の数式に悩まされることなく、プロンプトで指示するだけで「ドラッグ＆シュート（引っ張って飛ばす）」のメカニズムを瞬時に実装できます。
3. **高拡張性のモジュール設計:** AIが初期段階から `Physics`、`Input`、`Level` の各クラスを独立させたモジュール構造でコードを生成するため、将来的な機能拡張（マルチプレイや新ステージの追加など）が極めて容易になります。

## 🚀 解決策：「Claude Code ゲーム開発プロンプト」

### 🥉 Basic Version（基本型）

プロトタイプの骨組み（ボイラープレート）を迅速に構築したい場合に最適な、初期設定用のプロンプトです。

> **役割:** あなたは `[シニアゲームエンジニア]` です。
> **タスク:** `[HTML5 Canvas]` を使用して、ブラウザベースの2Dゴルフゲームを作成してください。ボールとホールを配置し、マウスのドラッグ操作でボールを打つ（drag-to-shoot）仕組みを実装してください。

### 🥇 Pro Version（専門家型）

物理エンジン、入力処理、レベルデザインが完全に分離された、高度なモジュール型ゲームアーキテクチャを設計・構築する際に威力を発揮するプロンプトです。

> **役割 (Role):** あなたは10年の実務経験を持つ `[シニアフロントエンド兼ゲームクライアントエンジニア]` であり、私の完璧なペアプログラミングパートナーです。
>
> **状況 (Context):**
>
> - 背景: 純粋なVanilla JSとHTML5 Canvasのみを用いて、軽量なトップダウンビューの2Dゴルフゲームを開発しようとしています。
> - 目標: 物理演算、入力イベント処理、レベルの状態管理がそれぞれ完全にモジュール化された、保守性の高い堅牢なコードを記述することです。
>
> **タスク (Task):**
>
> 1. ファイル構成を `index.html`（Canvasコンテナ）、`game.js`（メインループ）、`physics.js`（ベクトル演算、摩擦、衝突処理などの数学的ロジック）に分割し、構造化してください。
> 2. `physics.js` 内に、芝生の摩擦（Friction）や壁面衝突時の跳ね返り（Bounce）に関する精緻なロジックを実装してください。
> 3. マウスの `mousedown`、`mousemove`、`mouseup` イベントを活用し、ボールを引っ張った際の予想軌道とパワーを視覚的に示すインジケーター（Indicator Line）機能を追加してください。
> 4. `Obstacle`（障害物）クラスを定義し、難易度が段階的に上昇する3つの異なるレベル（Level）配列を構築してください。
>
> **制約事項 (Constraints):**
>
> - Phaser.jsなどの外部ゲームエンジンやライブラリは一切使用しないでください。
> - すべての主要なロジックには詳細なコメントを添え、動作原理を明確に説明してください。
> - モジュールパターンまたはES6のClass構文を採用し、徹底したオブジェクト指向プログラミング（OOP）に基づいたコードを記述してください。

## 💡 筆者のコメント (Insight)

このプロンプトの最大の真髄は、AIを単なる「コード生成機」としてではなく「システムアーキテクト」として位置づけている点にあります。初期段階で「ファイルの分割」や「クラスの分離」といった明確な制約（Constraints）を与えない限り、AIはすべての処理を `index.html` という単一ファイルに詰め込み、解読困難なスパゲッティコードを生成しがちです。特に、物理演算と状態管理を別々のモジュールとして切り出すよう強制することで、後日「風（Wind）の抵抗」という新要素を追加したり、WebSocketを活用してオンライン対戦を実装したりする際の改修コストを劇的に削減できました。Claude Codeは文脈（コンテキスト）の維持能力に非常に長けているため、まずは強固な骨組み（アーキテクチャ）を構築させ、その上に少しずつ機能を肉付けしていくアプローチが最も効果的です。

## 🙋 よくある質問 (FAQ)

- **Q: 物理学や高度な数学の知識が全くなくても開発できますか？**
  - A: はい、完全に可能です。「芝生の摩擦力を再現して」「壁にぶつかったら自然に跳ね返るようにして」といった具合に、自然言語で物理現象を具体的に描写するだけで、内積や反発係数といった複雑な数学的処理はClaudeが自動的にコードへと変換してくれます。

- **Q: スマートフォンなどのモバイルブラウザ環境（タッチ操作）にも対応させるにはどうすればよいですか？**
  - A: Pro Versionプロンプトのタスク（Task）セクションに、「`マウスイベントだけでなく、touchstart, touchmove, touchend イベントも適切にマッピングし、モバイルのタッチ環境でも同様の操作性を実現してください`」という一文を付け加えるだけで、デバイスを問わずシームレスに動作するようになります。

## 🧬 プロンプトの解剖 (Why it works?)

1. **明確なアーキテクチャ設計の指示:** 処理を `game.js` や `physics.js` といった具合にファイル単位で分割するよう明示することで、コードの複雑度を効果的に抑制し、AIが各ドメインの処理に集中できる環境を整えました。
2. **具体的なメカニズムとUXの描写:** 単に「ゴルフゲームを作って」と丸投げするのではなく、「マウスで引っ張った際の軌道とパワーを視覚的に示すインジケーター」といった形でユーザーエクスペリエンス（UX）を極めて詳細に言語化し、開発の意図をAIへ正確に伝達しました。

```javascript
// 完全にモジュール化され、オブジェクト指向で設計された物理エンジン (physics.js)
export class PhysicsBody {
  constructor(x, y) {
    this.position = new Vector(x, y);
    this.velocity = new Vector(0, 0);
    this.friction = 0.98; // 芝生の摩擦係数
  }

  update() {
    // 速度に摩擦力を適用
    this.velocity.multiply(this.friction);

    // 位置の更新
    this.position.add(this.velocity);

    // Canvasの境界での衝突処理（反発係数を適用するロジックの呼び出し）
    this.handleWallCollisions();
  }
}
```
