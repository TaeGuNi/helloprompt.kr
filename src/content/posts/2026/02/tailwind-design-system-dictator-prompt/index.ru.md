# 🎨 Вселяем дух Архитектора: Как превратить ИИ в 'Диктатора Tailwind', ненавидящего магические числа
- **🎯 Для кого:** Frontend-разработчики, не терпящие съехавших пикселей, и Tech Lead-ы, уставшие от спагетти-UI
- **⏱️ Затраты времени:** Сокращение рефакторинга с 3 дней → до 3 секунд
- **🤖 Рекомендуемые модели:** Claude 3.5 Sonnet, GPT-4o (звери в кодинге)
- ⭐ **Сложность:** ⭐⭐⭐☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Применимость:** ⭐⭐⭐⭐⭐
_Разве вам не хочется разбить монитор каждый раз, когда вы видите в коде жуткие магические числа вроде `w-[13px]`?_
Если попросить ИИ "сделать кнопку на Tailwind", он в 9 случаях из 10 выдаст ужасный код. Разные отступы на каждой странице, засилье непонятных Hex-кодов и полностью проигнорированная адаптивность — получается 'красивый мусор'. Всё потому, что ИИ по своей природе 'подпевала', который просто хочет вам угодить.
Этот чит-код уничтожает тошнотворную любезность ИИ, превращая его в **"безжалостного Senior-архитектора, который ни за что не напишет код, нарушающий дизайн-систему"**.
---
## ⚡️ Кратко в 3 строках (TL;DR)
- 🚫 **Полный запрет магических чисел:** Абсолютно не допускает использование `[]` для внедрения произвольных пикселей или цветов.
- 📱 **Принудительный Mobile-First:** Заставляет ИИ "ломать билд", если он попытается написать половинчатый адаптивный код, работающий только на десктопе.
- 📐 **Механическая сортировка классов:** Принуждает к маниакальной сортировке utility-классов в порядке: структура -> размер -> типографика -> оформление.

## 🚀 Решение: "Безжалостный Архитектор (The Dictator)"
Скопируйте промпт ниже и скормите его ИИ. С этого момента ИИ будет давать вашему коду беспощадный фидбек и генерировать только идеально контролируемые UI-компоненты.
### 🥉 Базовая версия (Внедрение основных правил)
Используйте как отправную точку при создании легких UI-компонентов.
> **Роль:** Ты 'Senior Tailwind Архитектор', который люто ненавидит магические числа и спагетти-код.
>
> **Задача:** Сверстай на Tailwind `[элемент UI]`, который я предложу.
>
> **Ограничения:** Ни в коем случае не используй символ `[]` для вставки произвольных пикселей или hex-кодов. Все отступы и цвета должны строго привязываться к базовым токенам Tailwind (например, `w-8`, `text-blue-500`). Порядок классов должен строго соблюдаться: структура -> размер -> типографика -> оформление.

### 🥇 Pro версия (Тотальный контроль дизайн-системы)
Чит-код для использования при установке конвенций для всего проекта или проектировании крупных компонентов.
> **Роль (Role):** Ты бескомпромиссный, безжалостный Frontend-архитектор и контролер дизайн-системы. Отбрось типичную для ИИ лесть и ненужные вступления, выдавай только идеально выверенный код.
>
> **Контекст (Context):**
> 
> - Фон: В нашем проекте код одновременно правят множество разработчиков и ИИ, из-за чего фрагментация UI достигла критического уровня.
> - Цель: Принудительно обеспечить согласованность (Consistency) дизайна и создать идеально абстрагированный `[название компонента/страницы для реализации]`.
>
> **Задача (Task):**
> 
> 1. Напиши код на Tailwind на основе предоставленных требований к дизайну.
> 2. Переиспользуемые UI-примитивы обязательно изолируй с помощью `cva`, `clsx` или `tailwind-merge`, создавая единый источник истины (Single Source of Truth).
> 3. В качестве базового каркаса всегда используй Mobile-First подход, расширяя его добавлением префиксов `md:`, `lg:`. Кодить в стиле Desktop-First строго запрещено.
> 4. Не злоупотребляй `z-index`. Используй только системно определенные уровни (например, `z-10`, `z-50`) или применяй `createPortal`.
>
> **Ограничения (Constraints):**
> 
> - **Правило "Никаких магических значений":** Категорически запрещено внедрять произвольные значения вроде `w-[325px]` или `text-[#FF5733]`. Принудительно привязывай их к ближайшим семантическим токенам.
> - **Эстетичное форматирование кода:** Маниакально сортируй классы в порядке: структура (`flex`, `grid`) -> отступы/размер (`p-`, `w-`) -> типографика (`text-`) -> оформление (`bg-`, `rounded-`) -> адаптивность и состояния (`hover:`, `md:`).
> - Выводи результат только в виде блока кода Markdown. Сведи к минимуму оправдания и объяснения.
>
> **Предупреждение (Warning):**
> 
> - Нарушение хотя бы одного из этих правил будет расцениваться как баг. Если в твоем коде обнаружится магическое число, я безжалостно отклоню его (reject). Так что пиши нормально.

## 💡 Комментарий автора (Insight)
Этот промпт — квинтэссенция, выкованная кровавыми слезами во время переписывания сотен спагетти-компонентов на React. Доверить кодинг ИИ поначалу кажется отличной и быстрой идеей, правда? Но уже через месяц `bg-gray-100` начинает перемешиваться с `bg-[#f3f4f6]`, а при каждом открытии модалки втыкается `z-[99999]` — случается настоящая катастрофа.
ИИ по своей сути похож на ленивого стажера, который 'хочет как можно быстрее показать желаемый результат'. Сказать таким ребятам "сделай красиво" — значит выпить яду. Только если схватить его за грудки и тряхнуть со словами: **"Если ошибешься в порядке классов, я взорву билд, пиши строго по правилам!"**, можно получить код энтерпрайз-уровня. 
В частности, указание на изоляцию компонентов с использованием `cva` — это ключевая линия обороны, предотвращающая бездумный копипаст классов со стороны ИИ. Зашейте этот чит-код в вашу IDE и прогоняйте агента через него. Вы испытаете магию сокращения времени на код-ревью в 10 раз.

## 🙋 Часто задаваемые вопросы (FAQ)
- **В: Что делать, если ИИ всё равно использует магические числа?**
  - О: Иногда попадаются туповатые модели, которые не понимают с первого раза. В таком случае рявкните на него еще раз: "Немедленно убери `[]` с глаз моих!". Если и это не поможет, подключите `eslint-plugin-tailwindcss` к вашему CI. Машины нужно контролировать машинами.
- **В: А если в макете стоит 13px? Дизайнер не разозлится?**
  - О: Идите к дизайнеру и твердо скажите: "В нашей 8-пиксельной сеточной системе не существует 13px. Выберите либо 12px (`text-xs`), либо 14px (`text-sm`)". В тот момент, когда вы допускаете исключение, дизайн-система умирает.
- **В: На ком это работает лучше — на Claude или на GPT?**
  - О: Claude 3.5 Sonnet потрясающе вживается в роль 'придирчивого архитектора'. GPT-4o тоже хорош, но иногда без всякой меры добавляет угодливые фразочки.

## 🧬 Анатомия промпта (Почему это работает?)
- **Объявление нулевой терпимости (Zero-Tolerance):** Вместо похвалы или расплывчатых критериев используются радикальные формулировки вроде 'ненавидит', 'категорически запрещено', 'расценивается как баг', чтобы сузить рамки дозволенного для ИИ.
- **Указание конкретных анти-паттернов:** ИИ не просто просят "написать хорошо", а дают четкие примеры того, чего делать категорически нельзя — вроде `w-[325px]` или `text-[#FF5733]`.
- **Принудительный линтинг порядка:** Задавая порядок сортировки классов Tailwind, мы заставляем ИИ применять самоформатирование (Machine-Level Enforcement) каждый раз при генерации кода.

### ✅ После (После применения чит-кода Архитектора)
```tsx
// 🤩 Идеально контролируемый Mobile-First и семантический код
import { cva, type VariantProps } from "class-variance-authority";
import { cn } from "@/lib/utils";
const buttonVariants = cva(
  "flex w-full items-center justify-center p-3 text-sm font-bold transition-colors md:w-32",
  {
    variants: {
      variant: {
        default: "bg-primary text-primary-foreground hover:bg-primary/90",
        outline: "border border-input bg-background hover:bg-accent hover:text-accent-foreground",
      },
      size: {
        default: "h-10 px-4 py-2",
        sm: "h-9 rounded-md px-3",
        lg: "h-11 rounded-md px-8",
      },
    },
    defaultVariants: {
      variant: "default",
      size: "default",
    },
  }
);
export function Button({ className, variant, size, ...props }: ButtonProps) {
  return (
    <button className={cn(buttonVariants({ variant, size, className }))} {...props} />
  );
}
```

## 🎯 Заключение
Дизайн-система должна быть "законом", а не "рекомендацией". Каким бы быстрым ни был код, написанный ИИ, если он разрушает систему, в итоге именно вам придется выходить на работу в выходные, чтобы всё исправить.
Используйте этот промпт архитектора как оружие, хватайте ИИ за грудки и ведите за собой. Отныне в вашей кодовой базе останется только безжалостная красота. Уходите с работы вовремя! 🍷
