# 📝 開発者の生涯の難題「変数名付け」、AIが代わりにやります

- **🎯 推奨対象:** すべての開発者、エンジニア、プログラミング学習者
- **⏱️ 所要時間:** 5分 → 30秒に短縮
- **🤖 推奨モデル:** すべての対話型AI (ChatGPT, Claude, Geminiなど)

- ⭐ **難易度:** ⭐☆☆☆☆
- ⚡️ **効果性:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「ユーザーが退会した時に残りのポイントを払い戻す関数... 名前はどうしよう？refundUser？それともreturnPoint？」_

プログラミングにおいて最も難しい2つのこと。それは「キャッシュの無効化」と「名前付け」だと言われています。

不自然な直訳英語の変数名は同僚を困惑させ、曖昧な名前は未来の自分を激怒させます。これからはAIに「やりたいことの状況」だけを説明してください。ネイティブのシニアエンジニアが使うような、洗練された可読性の高い名前をAIが即座に提案します。

---

## ⚡️ 3行要約 (TL;DR)

1. 変数や関数の役割に合った、直感的で自然な英語の命名を提案
2. camelCaseやsnake_caseなど、言語別のコーディング規約を自動適用
3. 提案された名前の細かいニュアンスの違いまで解説し、英語学習にも貢献

---

## 🚀 解決策：「ネーミングマスター」

### 🥉 Basic Version (基本形)

手っ取り早く、パッと結果だけが欲しい時に使用してください。

> **役割:** あなたは英語ネイティブのシニアエンジニアです。
> **リクエスト:** `[ここに変数の役割や状況を説明]`に最適な変数名/関数名を3つ提案して。

<br>

### 🥇 Pro Version (専門家形)

より文脈に沿った、精巧なネーミングと解説が必要な場合に使用してください。

> **役割 (Role):** あなたは英語圏で長年活躍しているシニアソフトウェアエンジニアです。優れた言語感覚とクリーンコードの原則を深く理解しています。
>
> **状況 (Context):**
>
> - 背景: 新しい機能の実装中で、変数または関数の命名に悩んでいます。
> - 目標: 最も適切で、自然な英語表現であり、他の開発者が見ても意図がすぐに伝わる自己文書化（Self-documenting）された名前を見つけたいです。
>
> **リクエスト (Task):**
>
> 以下の条件に従って、最適な名前を **3〜5個** 提案してください。
>
> 1. **フォーマット:** `[camelCase / snake_case / PascalCase]` の形式で提案してください。
> 2. **解説:** なぜその単語を選んだのか、それぞれの名前にどのようなニュアンス（状態、アクション、真偽値など）が含まれているかを詳しく説明してください。
> 3. **ベストピック:** 提案した中から、あなたが最も推奨するものを1つ選び、その理由を述べてください。
>
> **状況説明:**
>
> - 役割・処理内容: `[変数や関数が何をするのかを具体的に記載。例: "ユーザーの新規登録が完了した後に、ウェルカムメールを送信する非同期関数"]`
>
> **使用言語:** `[Java / Python / JavaScript など]`
>
> **制約事項 (Constraints):**
>
> - 変数名は長すぎず、かつ省略しすぎない絶妙なバランスを保ってください。
> - ハンガリアン記法は避け、現代的なコーディング規約に従ってください。

## 💡 作成者コメント (Insight)

このプロンプトの真の価値は、単なる英単語の翻訳ではなく「プログラミングの文脈」を理解した提案を得られる点にあります。

特に、`is` `has` `should` のような真偽値（boolean）の接頭辞の使い分けや、`fetch` `get` `load` のようなデータ取得系動詞の微細なニュアンスの違いは、非ネイティブにとって非常に悩ましい部分です。このプロンプトを日常的に使えば、コードレビューで「この変数名、ちょっと分かりにくいね」と指摘されることが劇的に減り、同時にあなた自身の「開発用英語力」も自然と鍛えられます。

## 🙋 よくある質問 (FAQ)

- **Q: 独自の社内命名規則（プレフィックスなど）があるのですが、対応できますか？**
  - A: はい。「状況説明」のセクションに「社内ルールとして、グローバル変数には必ず `g_` を付ける」や「インターフェース名には `I` を付けない」などの条件を追記するだけで、そのルールに従った提案をしてくれます。

- **Q: データベースのカラム名やAPIのエンドポイント名にも使えますか？**
  - A: もちろんです。その場合は「使用言語」の部分を「SQL」や「REST API エンドポイント」に変更し、フォーマットを `snake_case` や `kebab-case` に指定して実行してください。

## 🧬 プロンプト解剖 (Why it works?)

1. **シニアエンジニアのペルソナ設定:** 単なる翻訳機ではなく、コーディングのベストプラクティスを理解した専門家として振る舞わせることで、実務に即した質の高い提案を引き出しています。
2. **フォーマットと命名規則の明示:** camelCaseやsnake_caseなどを明示することで、出力結果をそのままエディタにコピー＆ペーストできるように最適化しています。
3. **ニュアンスの解説要求:** なぜその単語が選ばれたのかという背景（Why）を知ることで、開発者が納得して名前を採用でき、応用力も身につきます。

## 📊 証明: Before & After

### ❌ Before (入力)

```text
状況: ユーザーがログインしているかどうかを確認する変数（boolean）
言語: JavaScript
```

### ✅ After (結果)

```text
推奨リスト (JavaScript - camelCase):

1. `isLoggedIn` (最も普遍的で、現在の状態をシンプルに表す表現)
2. `isAuthenticated` (セキュリティや認証プロセスの観点から、より専門的で厳密な表現)
3. `hasActiveSession` (セッションが現在有効かどうかという「状態の保持」を強調したい時)

🏆 Best Pick: `isLoggedIn`
理由: 最も直感的で、他の開発者が見た瞬間にboolean値であることが伝わり、JavaScriptの標準的な命名規則にも最も適しているため。
```

## 🎯 結論

名前さえ的確であれば、コードに長々とコメントを書く必要はなくなります。

「a」や「temp」「data」のような、その場しのぎの名前からはもう卒業しましょう。AIを優秀なペアプログラミングの相棒にすれば、あなたのコードはまるで美しい英文学のように、誰にとっても読みやすく保守しやすいものに生まれ変わるはずです。
