# 📝 100 строк тестовых данных JSON для фронтенда в один клик! (Генератор Mock-данных с ИИ)

- **🎯 Рекомендуется для:** Фронтенд-разработчиков, Full-stack разработчиков, QA-инженеров
- **⏱️ Затраты времени:** 30 минут → Сокращено до 1 минуты
- **🤖 Рекомендуемые модели:** Любые диалоговые ИИ (ChatGPT-4o, Claude 3.5 Sonnet, Gemini и др.)

- ⭐ **Сложность:** ⭐☆☆☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Полезность:** ⭐⭐⭐⭐⭐

> _"Вы все еще хардкодите 'test1' и 'user123', ожидая готовности бэкенд API?"_

При разработке фронтенда часто возникают узкие места из-за ожидания готовности бэкенд API. Временные, скудно захардкоженные данные скрывают дефекты UI и в конечном итоге приводят к множеству багов на этапе интеграции. Нам нужны реалистичные фиктивные данные (Mock Data), учитывающие крайние случаи (Edge Cases), такие как длинные имена, email-адреса со спецсимволами и различные форматы дат. Однако ручное создание таких данных — это колоссальная трата времени. Используя ИИ, вы можете сгенерировать идеальный массив JSON всего за 1 минуту.

---

## ⚡️ Краткая выжимка (TL;DR)

1. **Избавление от зависимости от бэкенда:** Возможна независимая разработка фронтенда и тестирование UI без ожидания API.
2. **Полная готовность к крайним случаям (Edge Cases):** Предотвращение поломки верстки за счет использования непредсказуемых данных, похожих на реальные.
3. **Максимальная продуктивность:** Мгновенное получение больших объемов качественного JSON просто путем описания схемы на естественном языке, без необходимости писать сложные скрипты на Faker.js.

## 🚀 Решение: Генератор Mock-данных с ИИ

### 🥉 Базовая версия (Basic Version)

Используйте, когда вам нужен просто быстрый массив JSON с базовой структурой.

> **Роль:** Ты `[Senior Фронтенд-разработчик]`.
> **Задача:** Создай 5 записей фиктивных данных (JSON) `[профилей пользователей]` для UI-тестирования фронтенда. Включи поля: id, name, email.

<br>

### 🥇 Продвинутая версия (Pro Version)

Используйте, когда вам нужны высококачественные данные для продакшена с различными переменными и крайними случаями.

> **Роль (Role):** Ты `[Senior Фронтенд-разработчик и QA-эксперт]`.
>
> **Контекст (Context):**
>
> - Ситуация: Я провожу тестирование рендеринга UI для `[страницы профиля пользователя на новой социальной платформе]`.
> - Цель: Необходимо сгенерировать массив из 10 реалистичных записей фиктивных данных (JSON), включающих различные исключительные ситуации (Edge Cases), которые могут возникнуть в реальной продакшен-среде.
>
> **Задача (Task):**
>
> 1. Сгенерируй массив JSON-объектов (`[]`), строго следующий указанной ниже структуре (Schema).
> 2. Используй естественное сочетание русского и английского языков и заполни реалистичными данными, которые можно встретить в настоящем сервисе.
>
> **Структура данных (Schema):**
>
> - `id`: UUID (v4)
> - `username`: Реалистичные никнеймы (могут включать пробелы и цифры)
> - `email`: Различные домены (@gmail.com, @yandex.ru, @mail.ru и т.д.)
> - `bio`: Короткое описание профиля из 1-3 предложений (иногда с эмодзи; включи 1-2 записи с пустой строкой `""` или `null`)
> - `role`: Одно из значений 'Admin', 'User', 'Guest' (80% должны быть 'User')
> - `createdAt`: Случайная дата в пределах последнего года (в формате ISO 8601)
>
> **Ограничения (Constraints):**
>
> - Выведи ТОЛЬКО чистый JSON код. Абсолютно никаких дополнительных пояснений или текста вне блока кода markdown (` ```json `).
>
> **Предупреждения (Warning):**
>
> - В целях тестирования верстки UI намеренно включи как минимум одну запись с очень длинным `username` или `bio` (для проверки переполнения текста - text overflow).

## 💡 Комментарий автора (Insight)

Истинная ценность этого промпта заключается не просто в быстром получении текста, а в **предварительном обнаружении архитектурных дефектов и недочетов дизайна**. Например, намеренно требуя значения `null` или аномально длинный текст в поле `bio`, фронтенд-разработчик может мгновенно найти места, где забыт условный рендеринг или отсутствует CSS-свойство `text-overflow: ellipsis`.

**💡 Совет из практики:** Попробуйте попросить ИИ написать код обработчика для `MSW (Mock Service Worker)` или `Mirage JS` на основе сгенерированных JSON-данных. Если вы добавите: _"Используя сгенерированный выше JSON, напиши код обработчика MSW, который возвращает эти данные при GET-запросе к эндпоинту (/api/users)"_, скорость вашей разработки возрастет многократно.

## 🙋 Часто задаваемые вопросы (FAQ)

- **В: Можно ли сгенерировать сразу более 1000 записей данных?**
  - О: В среде веб-интерфейса ИИ-чатботов (из-за ограничений на количество выходных токенов) наиболее стабильным будет создание 10-20 записей за один запрос. Если вам нужны тысячи строк, настоятельно рекомендуется немного изменить промпт: _"Напиши Node.js скрипт на базе Faker.js (или Chance.js), который может сгенерировать большой объем фиктивных данных на основе этой схемы"_, а затем выполнить сгенерированный скрипт локально в вашем проекте.

- **В: Можно ли добавить URL-адреса для аватаров пользователей?**
  - О: Да, безусловно. Достаточно добавить в схему промпта строку `- avatarUrl: URL случайного сервиса изображений, например https://picsum.photos или https://ui-avatars.com`, и вы получите готовые данные с идеальными ссылками на изображения для немедленной привязки в UI.

## 🧬 Разбор промпта (Why it works?)

1.  **Принудительные крайние случаи (Edge Case Enforcement):** Указав "намеренно включи очень длинный текст" в разделе предупреждений, мы заставляем ИИ тестировать не только идеальные сценарии (Happy Path), но и суровые условия, которые могут сломать верстку.
2.  **Настройка распределения данных:** Установив конкретное ограничение "80% должны быть User" для поля `role`, мы настроили генерацию реалистичных данных, точно отражающих распределение прав в реальном сервисе.
3.  **Строгий контроль формата:** Ограничение "Выведи ТОЛЬКО чистый JSON код" предотвращает появление ненужных вступлений от ИИ, позволяя разработчику мгновенно скопировать ответ в буфер обмена.
