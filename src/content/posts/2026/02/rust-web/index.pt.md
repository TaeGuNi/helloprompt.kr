Aplique esta versão para arquitetar um sistema impecável, com performance de nível empresarial, perfeitamente adaptado para operar em ambientes cloud-native de alta exigência.

> **Função (Role):** Você é um `[Arquiteto Rust Principal]` com vasta experiência no design de sistemas de grande escala que suportam tráfego global massivo.
>
> **Contexto (Context):**
>
> - Cenário: Estamos migrando nosso sistema legado escrito em TypeScript/Go para Rust. Nosso objetivo primário é reduzir drasticamente os custos computacionais na nuvem.
> - Objetivo: Projetar uma arquitetura de backend cloud-native de altíssima performance para gerenciar a `[Principal funcionalidade do serviço]` e implementar a lógica de negócios central.
>
> **Tarefa (Task):**
>
> 1. Proponha uma arquitetura de sistema robusta, fundamentada no framework mais estável de 2026 (ex: Axum) e no runtime assíncrono definitivo (Tokio).
> 2. Escreva o código do `main.rs` em nível de produção. Isso deve incluir: um pool de conexões com o banco de dados, tratamento de erros elegante (usando o padrão `Result`) e uma configuração abrangente de logs distribuídos (Tracing).
> 3. Otimize rigorosamente o código para suportar as seguintes `[Variáveis]`:
>    - Tráfego esperado: `[ex: 10.000 requisições por segundo]`
>    - Banco de dados principal: `[ex: PostgreSQL]`
>
> **Restrições (Constraints):**
>
> - Forneça a saída final inteiramente dentro de um único bloco de código Markdown. Explique detalhadamente a finalidade de cada módulo através de comentários diretos no código.
> - Escreva um código que cumpra religiosamente as regras de propriedade (Ownership) do Rust. Assegure a ausência total de vazamentos de memória (Memory Leaks) e condições de corrida (Data Races).

> **Avisos (Warning):**
>
> - É terminantemente proibido o uso de Crates (pacotes) de terceiros não verificados ou obscuros; adote exclusivamente bibliotecas consolidadas como padrão de fato (De facto standard) na comunidade.
> - Não aplique técnicas de otimização excessivas, prematuras ou incertas. Se qualquer otimização não convencional for aplicada, justifique seu uso detalhadamente. Em hipótese alguma invente informações (Zero Hallucinations).

### ✅ Depois (Usando o Prompt Pro)

```rust
// Resultado da geração de um código base em nível de produção usando Axum e Tokio
use axum::{routing::get, Router};
use std::net::SocketAddr;
use tracing::info;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // 1. Inicialização do sistema de Logs Distribuídos
    tracing_subscriber::fmt::init();
    info!("Iniciando o servidor web Rust de altíssima performance...");

    // 2. Configuração do Router e Estruturação de Tratamento de Erros
    let app = Router::new()
        .route("/health", get(health_check));

    // 3. Bind e Execução do Servidor Assíncrono
    let addr = SocketAddr::from(([0, 0, 0, 0], 8080));
    info!("Ouvindo requisições na porta {}", addr);
    axum::Server::bind(&addr)
        .serve(app.into_make_service())
        .await?;

    Ok(())
}

async fn health_check() -> &'static str {
    "OK"
}
```

_(Resultado: Uma fundação de servidor robusta, blindada e profissional. O código já implementa instrumentação de logs nativa (Tracing), um fluxo escalável de tratamento de erros baseado em `Result`, e extrai o máximo do motor de execução moderno do Axum/Tokio.)_
