# 🤖 创建 Slack 机器人：推荐'午餐吃什么？'的企业内部机器人

- **🎯 推荐对象:** 每天为决定午餐菜单而痛苦的基层员工，希望为沉闷的企业即时通讯工具注入活力的人事团队
- **⏱️ 所需时间:** 10分钟（使用 Slack Workflow 时）
- **🤖 推荐模型:** ChatGPT-4o（具备出色的创意菜单推荐及情境感知能力）

- ⭐ **难度:** ⭐⭐☆☆☆
- ⚡️ **效果:** ⭐⭐⭐⭐⭐
- 🚀 **实用度:** ⭐⭐⭐⭐⭐

> _"今天午餐吃什么？随便。那吃意面？面条有点... 每天重复的午餐察言观色游戏，现在就交给 AI 助手，为准时下班积攒能量吧。"_

职场人士的最大难题：选择午餐菜单。现在，不必再互相察言观色，直接问问公司内部的通讯机器人吧。输入“今天下雨吃什么好？”，它会机智地回答：“下雨天配葱饼米酒... 不对，来碗热腾腾的刀削面怎么样？”。本文将向您介绍如何不写一行代码，仅通过 Slack Workflow（工作流）打造这样一个充满趣味的机器人。

---

## ⚡️ 3行总结 (TL;DR)

1. 无需编程，利用 Slack 的 **“工作流构建器 (Workflow Builder)”** 设置自动化机器人。
2. 将特定词汇（例如：“午餐推荐”）或表情符号反应设置为触发器 (Trigger) 来唤醒机器人。
3. 联动 AI 连接器（例如：ChatGPT），运行能够根据天气和情境量身定制午餐菜单的提示词。

---

## 🚀 解决方案："午餐搭子 (Lunch Buddy)" 提示词

### 🥉 Basic Version (基础版)

当您只想快速解决选择困难症时，可以使用这个基础的随机推荐提示词。

> **角色 (Role):** 你是一个亲切的秘书，负责为公司员工挑选午餐菜单。
>
> **请求 (Task):**
> 请在中餐、西餐、日料、韩餐和快餐中随机选择一个类别，并推荐 3 款该类别下职场人士普遍喜欢、不易踩雷的代表性午餐菜单。

<br>

### 🥇 Pro Version (专业版)

当您希望结合天气、心情、人数等具体情况，获得巧妙的午餐推荐时使用。

> **角色 (Role):** 你是一位口味挑剔的美食家，也是公司附近（`[公司位置（例：国贸）]`）的美食导航仪。
>
> **背景 (Context):**
>
> - 天气: `[当前天气（例：下雨、晴天、寒冷）]`
> - 心情及情境: `[团队氛围（例：压力大、发薪日、正在减肥）]`
> - 就餐人数: `[人员构成（例：独自一人、全队聚餐、同期同事聚会）]`
>
> **请求 (Task):**
>
> 1. 请推荐 1 款最完美契合上述情境的午餐菜单。
> 2. 请附上 1~2 句风趣幽默、能引起职场人共鸣的推荐理由。
> 3. 请以带有表情符号的 Markdown 列表格式输出，以确保易读性。
>
> **限制条件 (Constraints):**
>
> - 请排除口味过于两极分化（如：折耳根、螺蛳粉）或在午休时间内难以吃完的繁重菜品。
>
> **注意事项 (Warning):**
>
> - 请专注于菜品名称本身，不要捏造不确定的特定餐厅名称。（防止幻觉）

## 💡 作者点评 (Insight)

Slack 机器人超越了单纯的技术“功能”，更是塑造企业**“文化”**的强大工具。在只有生硬工作指令的聊天频道里，它能起到润滑剂的作用，带来些许欢笑和谈资。从午餐推荐机器人开始，如果能获得团队成员的好评，后续还可以扩展为“员工赞美机器人”、“生日祝福机器人”、“周报回顾机器人”等，为建立灵活积极的组织文化做出巨大贡献。特别是，如果将此提示词与 Zapier 或 Make 等自动化工具结合使用，就能突破 Slack Workflow 的限制，进化成聪明得多的 AI 助手。

## 🙋 常见问题 (FAQ)

- **Q: 我们公司使用的是 Slack 免费版，可以使用工作流构建器吗？**
  - A: 很遗憾，Slack 工作流构建器仅在付费计划（Pro 及以上）中提供完整支持。如果您使用的是免费计划，可以通过 Zapier 或 Make.com 监听特定频道的触发消息，并通过 ChatGPT API 发送回复，以此实现曲线救国。

- **Q: 这与直接通过编程（如 Python）开发机器人有什么区别？**
  - A: 使用 Python 的 `slack-bolt` 库可以实现按钮点击、弹出模态框 (Modal)、连接公司内部数据库等高级交互功能。但这会产生维护成本和服务器费用。如果只是单纯的菜单推荐或轻量级的文本回复，无代码 (No-code) 工作流在效率上具有压倒性优势。

## 🧬 提示词解剖 (Why it works?)

1. **基于情境的变量分配 (Contextual Variables):** 并非简单地说“推荐菜单”，而是提供了天气、心情、人数等变量 (`[ ]`)，引导 AI 避开千篇一律的回答，生成深刻共情用户情境的定制化答案。
2. **控制幻觉与两极分化口味 (Constraint Setting):** 通过`注意事项 (Warning)`让 AI 认知到午休时间的特殊性（时间限制、口味偏好），从源头上杜绝了捏造虚构餐厅的幻觉 (Hallucination) 现象。
3. **赋予角色 (Role-playing):** 设定了“美食家兼美食导航仪”的具体人设，使回答的语气不再是生硬的信息传递，而是像亲切幽默的职场同事一样。
