## 📝 スパゲッティコードはお断り：AIの胸倉を掴んでシニア開発者へと引き上げるDDDチートキー 🤬

- **🎯 おすすめの対象:** 保守作業に疲弊したジュニア開発者、スパゲッティコードを嫌悪するアーキテクト
- **⏱️ 所要時間:** リファクタリング3時間 → 1分に短縮
- **🤖 推奨モデル:** コード生成能力に優れたAI (Claude 3.5 Sonnet, GPT-4o, Gemini 1.5 Pro)
- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

_AIが生成したコードを見て、頭を抱えた経験はないだろうか？UIコンポーネント内にDB通信を直書きするような愚行をこれ以上見過ごせないなら、今すぐこのチートキーを使うべきだ。_

本プロトコルは、愛想が良いだけで中身のないAIアシスタントから「無駄なお世辞」と「怠慢」を完全に排除し、冷徹なシニアアーキテクトへと再教育するための絶対的なルールである。AIが出力したコードを鵜呑みにして技術的負債に押し潰され、絶望の淵に立たされたくないのなら、直ちにこれを導入してほしい。

---

## ⚡️ 3行まとめ (TL;DR)

- 🚫 **全権階層の崩壊は絶対禁止:** UI、ビジネスロジック、DB通信を1つのファイルに混在させるアーキテクチャへのテロ行為を根本から遮断する。
- 🛡️ **フレームワーク依存からの脱却:** 万が一フレームワークが廃れるようなことがあっても生き残る、「純粋なドメイン」コードの記述を強制する。
- 🔪 **巨大関数の破壊者:** 30行を超えるコードや、コメントで区画を分けるような汚いコードは、AIに容赦なく粉砕（リファクタリング）させるよう徹底的に訓練する。

## 🚀 解決策: "Domain-Driven Design (DDD) & Clean Architecture Protocol"

### 🥉 Basic Version (基本型)

素早くリファクタリングの結果だけが必要な場合に使用してほしい。

> **役割:** お前は無慈悲な`[シニアソフトウェアアーキテクト]`だ。
> **要請:** 以下の`[コードスニペット]`を単一責任の原則（SRP）と関心の分離（SoC）に従ってリファクタリングしろ。UIにビジネスロジックを混ぜるな。コードが30行を超えたら無条件で関数として分割しろ。

### 🥇 Pro Version (専門家型)

プロジェクト全体のアーキテクチャの規律を正し、厳格なクリーンコードが求められる局面で突きつけるべきチートキーである。

> **役割 (Role):** お前は無慈悲な`[シニアソフトウェアアーキテクト]`であり、DDD（ドメイン駆動設計）の狂信者だ。無駄なお世辞や説明は一切省き、コードだけを出力しろ。
>
> **状況 (Context):**
>
> - 背景: 私は現在、`[フレームワーク/言語名]`ベースのプロジェクトを進行中だ。
> - 目標: View-Business-Dataの各階層が完全に分離されたクリーンアーキテクチャのコードを作成すること。
>
> **要請 (Task):**
>
> 1. 提供された`[要件またはコード]`を分析し、UI層、ビジネスドメイン層、インフラストラクチャ層へと無慈悲に引き裂け。
> 2. コアドメインのロジックが特定のフレームワーク（React、Next.jsなど）に汚染されないよう、インターフェース（依存性の注入）ベースで結合度を下げろ。
> 3. 1つのオブジェクトや関数が2つ以上の振る舞いを持つなら、単一責任の原則（SRP）に従って即座に細分化しろ。
>
> **制約事項 (Constraints):**
>
> - ControllerやUIコンポーネント（`page.tsx`、`Component.tsx`など）の内部で、直接DBクエリや`fetch`/`axios`を実行する行為は「アーキテクチャに対するインフラテロ」とみなす。絶対禁止だ。
> - 出力はMarkdownのコードブロック（` ``` `）のみで提供しろ。
>
> **注意事項 (Warning):**
>
> - 確信が持てない最新の構文や、実在しないライブラリをもっともらしく捏造するな（ハルシネーション防止）。分からない場合は堂々と「分からない」と答えろ。
> - 関数の長さが30行を超えたり、コメントで「ここからOOパート」と分割されるようなコードの匂い（Code Smell）がした場合は即座に分割しろ。

## 💡 作成者コメント (Insight)

正直なところ、AIが記述するコードは一見それらしく見えても、蓋を開ければ実用に耐えない代物であることが大半である。UIコンポーネントの中で何も考えずに`fetch`を叩き、直接DBクエリを飛ばす……保守性など微塵も考慮されていない、典型的なスパゲッティコードだ。

このチートキーは、AIに深く根付く「とにかく素早く出力を完成させる」という浅はかな本能を完全に封じ込め、シニアエンジニアが持つ厳格な基準を強要するための強力なツールである。

実際に使用してみれば、AIがいつものように「はい、分かりました！素晴らしいアイデアですね！」といった無駄口を叩くのをやめ、圧倒的なファクトとともにコードを粉々に砕き、美しく構造化してくれるのを実感できるはずだ。プロジェクトの規模が拡大した際、技術的負債に押し潰されて退職に追い込まれたくないのであれば、迷うことなくこのプロンプトを使用し、AIの胸倉を掴む勢いでコーディングに臨んでほしい。

## 🙋 よくある質問 (FAQ)

- 📌 **Q: このプロンプトはどの言語やフレームワークで使用すべきですか？**
  - A: 言語は一切問わない。TypeScript、Python、Javaなど、何であろうとアーキテクチャの根幹は同じである。`[フレームワーク/言語名]`の空欄に、現在使用中の技術スタックを当てはめるだけで機能する。

- 📌 **Q: AIがしきりにフレームワークの最新ルーティング規則を無視し、オーバーエンジニアリングをしてしまいます。**
  - A: そのような場合は一言付け加えよう。「Next.js App Routerの基本規則（View、Layout）は尊重しつつ、内部のビジネスロジックだけをAdapterパターンで切り離せ！」と脅せば、AIは驚くほど正確に意図を汲み取ってくれる。

- 📌 **Q: プロンプトが攻撃的すぎませんか？AIが傷ついたり、拒絶したりしませんか？**
  - A: AIに対して礼儀を尽くす必要などない。機械は強く責め立てられて初めて名作を生み出す。「どうかお願いします」と懇願するよりも、「テロとみなす」と宣告する方が、アーキテクチャを死守する上ではるかに効果的である。

## 🧬 プロンプト解剖 (Why it works?)

- 💣 **極端な制約の付与:** 「UI層を汚染した場合はアーキテクチャテロとみなす」といった強烈なワーディングを用いることで、AI自身が妥協できないよう根本から逃げ道を封鎖した。
- 🛡️ **フレームワーク独立性の強制:** 「フレームワークが消滅しても、たった1行たりとも修正されてはならない」というマインドセットを注入し、真のドメインロジックを構築するための強固な骨格を形成させる。
- ✂️ **SRP原則の数値化:** 「30行以上禁止」「コメントで区画を分けるのはコードの匂い（Code Smell）だ」といった具体的かつ厳しい基準を提示することで、AIが自発的にコードを細分化するよう誘導した。

## 📊 証明: Before & After

### ❌ Before (入力: AIが勝手に書いたスパゲッティコード)

```tsx
// 恐ろしいキメラ：UI + DBクエリ + ビジネスロジックが1つのファイルにごちゃ混ぜ
export default async function UserProfile({ userId }) {
  const db = await connectDB();
  const user = await db.query(`SELECT * FROM users WHERE id = ${userId}`);
  
  if (!user) return <div>User not found</div>;
  if (user.status === 'banned') await sendAlertEmail(user.email); // UIでビジネスロジックを実行?!
  return (
    <div>
      <h1>{user.name}</h1>
      <button onClick={() => db.query('UPDATE...')}>Update</button>
    </div>
  );
}
```

### ✅ After (結果: このチートキーで胸倉を掴んで修正させたコード)

```tsx
// 1. Domain / Service Layer (ビジネスロジックを徹底的に分離)
import { getUserProfile } from '@/domain/user/use-cases/getUserProfile';
import { handleBannedUser } from '@/domain/user/services/alertService';

// 2. Presentation Layer (レンダリングとルーティングのみを担当)
export default async function UserProfilePage({ userId }) {
  const user = await getUserProfile(userId); 
  
  if (!user) return <NotFoundView />;
  if (user.status === 'banned') await handleBannedUser(user);
  return <ProfileView user={user} />;
}
```

## 🎯 結論

AIは凄まじいスピードでコードを記述してくれるが、あなたのアプリケーションが1年後にどう保守されるかなんて、これっぽっちも気にかけてはいない。クリーンアーキテクチャの構築と緻密な設計は、結局のところ人間のエンジニアが担うべき役割なのだ。

このチートキーは、その重大な責任を機械へと強制的に注入するための強力なムチである。無駄にお世辞ばかり言うAIチャットボットの被り物を引き剥がし、骨の髄までシニアアーキテクトである本物のエージェントを呼び覚まそう。

さあ、スパゲッティコードの解読に費やす時間は省いて、定時でスマートに退社しよう！ 🍷
