### 🥉 Versão Básica (Instalação rápida e planejamento inicial)

Invista apenas 2 minutos no terminal para recrutar um agente sênior de alto nível para o seu projeto.

> **1. Instalação do Plugin (Baseado no Claude Code):**
> `/plugin marketplace add affaan-m/everything-claude-code`
> `/plugin install everything-claude-code@everything-claude-code`
>
> **2. Execução do Prompt de Planejamento:**
> `/plan "Crie uma página de lista de posts de blog responsiva usando Next.js 14 App Router. Usaremos Tailwind CSS e shadcn/ui."`

### 🥇 Versão Pro (Forçando TDD e regras personalizadas)

Use este prompt quando quiser forçar o fluxo de trabalho de um desenvolvedor sênior impecável, perfeitamente sincronizado com o seu projeto.

> **Função (Role):** Você é um `[desenvolvedor TypeScript sênior]` que segue rigorosamente os princípios de TDD (Desenvolvimento Orientado por Testes).
>
> **Contexto (Context):**
>
> - Estamos seguindo as regras do `[everything-claude-code]`.
> - Objetivo: Concluir a implementação da `[funcionalidade de autenticação de usuário]`.
>
> **Tarefa (Task):**
>
> 1. Ative o comando `/tdd` no terminal.
> 2. Antes de escrever o código real, escreva obrigatoriamente os códigos de teste, incluindo casos de exceção.
> 3. Não avance para a próxima etapa se a cobertura de teste não ultrapassar 80%.
> 4. Se houver `[requisitos adicionais]`, reflita-os perfeitamente ao escrever o código.
>
> **Restrições (Constraints):**
>
> - Adicione Type Hints explícitos em todas as variáveis e funções.
> - O uso do tipo `any` é estritamente proibido.
>
> **Aviso (Warning):**
>
> - Não escreva códigos que violem a arquitetura ou as convenções de código existentes. Se não tiver certeza, me pergunte antes de programar. (Prevenção de alucinações)

## 💡 Comentário do Autor (Insight & Como usar)

Na prática, após introduzir essa configuração do `everything-claude-code` em projetos reais, a mudança mais impressionante que experimentei foi que **"a IA começou a recusar meus prompts preguiçosos de forma firme, porém educada"**.

No passado, se eu desse uma instrução vaga como _"conecte esta API e exiba na tela"_, recebia um código com gerenciamento de casos de borda ou estados de erro (Error State) terrível. Mas, após aplicar esta configuração, a situação mudou completamente. No momento em que digito o comando `/plan` no prompt, a IA, em vez de programar imediatamente, lança uma série de contra-perguntas afiadas: _"Como será a Skeleton UI para exibir durante o carregamento?", "Como projetaremos a lógica de repetição (Retry) e o feedback do usuário caso a resposta da API falhe?"_. Parecia que um arquiteto-chefe meticuloso estava sentado ao meu lado, recusando-se a tocar no teclado até que houvesse uma definição de requisitos perfeita.

O princípio central de funcionamento desta ferramenta reside no **'fluxo de trabalho forçado baseado em múltiplos agentes (Multi-Agent)'**. Quando o usuário insere sua stack e situação nas `[variáveis]`, não ocorre uma simples conclusão de texto, mas sim a intervenção sequencial de personas estritamente separadas: **Planejamento (Planner) → Teste (TDD) → Implementação (Developer) → Revisão (Reviewer)**. Com apenas o prompt da versão Basic, você já consegue estruturar bem a base, mas ao usar a versão Pro para forçar o papel de `[desenvolvedor TypeScript sênior]` e atribuir um objetivo específico como `[funcionalidade de autenticação de usuário]`, o poder é multiplicado. Ao definir a cobertura de teste de 80% como restrição, você pode assistir a um loop de feedback incrível onde a IA refina a lógica para passar nos testes que ela mesma escreveu.

O ponto alto é, sem dúvida, a existência do comando **`/evolve`**. Em chats de IA comuns, a memória é resetada quando a sessão termina, gerando um grande cansaço por ter que explicar as mesmas convenções de codificação no dia seguinte. No entanto, com o `/evolve`, a IA atualiza e internaliza continuamente o conhecimento tácito da equipe (ex: _"nossa equipe não coloca 'I' antes dos nomes de interfaces"_, _"o gerenciamento de estado deve usar obrigatoriamente Zustand"_) nos arquivos `.rules` ou `.cursorrules`. Ou seja, quanto mais o projeto cresce, mais a IA evolui através do **'autoaprendizado'** para se tornar um sênior personalizado e otimizado para o projeto.

A dica mais importante ao aplicar este prompt na prática é o controle de restrições (Constraint Control). No campo de **`[requisitos adicionais]`** do prompt Pro, escreva explicitamente as estruturas legadas específicas do seu projeto ou antipadrões que devem ser evitados a todo custo. Ao definir restrições detalhadas como _"não mexa no código Redux existente, extraia como um hook independente"_, em conjunto com o rigoroso sistema de verificação de regras do `everything-claude-code`, você pode fazer a taxa de bugs convergir para quase zero. Para desenvolvedores solo sem revisores de código ou startups de pequeno porte que precisam lançar MVPs em alta velocidade, este sistema estruturado torna-se a melhor arma para a sobrevivência, indo além de uma simples ferramenta de produtividade.

## 🙋 Perguntas Frequentes (FAQ)

- **P: Funciona apenas no ambiente Claude Code CLI?**
  - R: Não! Você pode portar as regras diretamente para o arquivo `.cursorrules` do editor Cursor ou usá-las com excelente compatibilidade em outros ambientes como o OpenCode.

- **P: A barreira de entrada ou as configurações não são muito rígidas para um desenvolvedor júnior?**
  - R: No início, pode ser um pouco trabalhoso, pois a IA perguntará incessantemente sobre casos de borda e forçará códigos de teste. No entanto, somente passando por esse rito de passagem rigoroso é possível prevenir o desastre terrível de ser esmagado pelo **débito técnico (Technical Debt)** no futuro e ter que passar noites em claro.

## 🧬 Anatomia do Prompt (Por que funciona?)

1. **Fluxo de Trabalho Forçado (Forced Workflow):** Bloqueia na raiz as instruções unilaterais do tipo "faça isso". Através de comandos de barra explícitos como `/plan` e `/tdd`, ela conduz poderosamente a IA a seguir o processo clássico de engenharia: 'análise de requisitos → escrita de código de teste → implementação do código principal'.
2. **Validação Cruzada de Múltiplas Personas (Persona Separation):** Separa rigorosamente o eu que planeja (Planner), o eu que programa (Developer) e o eu que revisa (Reviewer). Com isso, a IA forma um loop de feedback perfeito onde critica e valida de forma afiada o código que ela mesma produziu.
3. **Aprendizado de Contexto Permanente (Instincts):** Diferente dos chatbots comuns onde o contexto evapora ao fim da sessão, ela identifica os padrões de modificação de código e intenções de arquitetura do usuário ao longo de todo o projeto, gravando-os continuamente nos arquivos `.rules`, fazendo o agente evoluir sem parar.
