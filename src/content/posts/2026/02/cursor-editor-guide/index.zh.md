## ⚡️ 3句话总结 (TL;DR)

1. `Cmd + K`：在当前光标位置，下达即时代码生成与修改指令。
2. `Cmd + L`：唤出编辑器内置的 AI 对话框，针对项目架构或复杂逻辑进行深度探讨。
3. `@Codebase`：在对话中将整个项目文件夹作为上下文提供，获取完美兼顾文件间依赖关系的回答。

## 🚀 解决方案："Cursor Master Prompt"

### 🥉 Basic Version (快捷键大师)

适用于需要在当前编写的文件中进行即时修改或生成样板代码 (Boilerplate) 时。

> **快捷键：** `Cmd + K` (Generate)
>
> **指令：**
> 将当前选中的函数重构为异步 (`async/await`) 方式，并使用 `try-catch` 块添加详细的错误处理逻辑。


### 🥇 Pro Version (代码库全局重构)

适用于非单一文件，而是需要进行影响整个项目的大规模修改或结构调整时。

> **快捷键：** `Cmd + L` (Chat) -> 必须添加 `@Codebase` 标签
>
> **任务 (Task)：**
> 分析当前项目所有与 `auth` 相关的逻辑，将 JWT 令牌的过期时间从 1 小时延长至 24 小时。
> 此外，检查 `login` 函数中遗漏的令牌刷新 (Refresh) 逻辑，并以符合标准的方式进行实现。
>
> **约束条件 (Constraints)：**
>
> - 必须保持一致性地修改所有相关文件，如 `utils`, `api`, `store` 等。
> - 修改完成后，请以 Markdown 列表格式总结已更改的文件列表及修改意图。

## 💡 作者点评 (Insight)

Cursor 已经超越了单纯的工具范畴，它更像是**“扩展开发者大脑的外接硬盘”**。虽然让它直接写代码也很棒，但当你面对庞大的开源项目或初次接触公司错综复杂的遗留项目 (Legacy Code) 时，Cursor 的真正价值才会 100% 显现出来。

在阅读代码时，如果遇到难以理解的部分，只需按下 `Cmd + L` 并提问：**“请分步骤解释为什么需要这段逻辑”**。相比于“写代码的时间”，大幅缩减“阅读和理解代码的时间”，这才是 Cursor 提供的核心杀手锏。尤其是利用 `@Codebase` 注入全局上下文，这在当前的 IDE 市场中绝对是独一档的“作弊级”功能。

## 🙋 常见问题 (FAQ)

- **Q: 我以前在 VS Code 里的配置和插件都要全部放弃吗？**
  - A: 完全不需要。Cursor 是基于 VS Code (准确地说是 VSCodium) 的分支版本。在首次安装时，只需一键点击，即可将原有 VS Code 的所有扩展程序、快捷键、主题和设置完美迁移过来。

- **Q: 担心公司代码泄露等安全问题怎么办？**
  - A: 只要在 Cursor 设置中开启 "Privacy Mode" (隐私模式)，您的代码就绝对不会被用于 AI 模型的训练。此外，Cursor 还提供企业级套餐（通过 SOC 2 认证），在实际业务中也能安全合规地引入。

- **Q: 免费版和付费版的区别是什么？**
  - A: 免费计划也能体验基础功能，但要无限次使用 Cursor 核心的最强模型（如 Claude 3.5 Sonnet, GPT-4o 等）的“快速请求 (Fast Request)”，则需要订阅每月 20 美元的 Pro 计划。考虑到它能让你每天提前 1 小时以上下班，这绝对是一笔物超所值的投资。

## 🧬 提示词解剖 (Why it works?)

1.  **@Codebase 上下文注入：** 传统的 AI 编码助手只能理解当前打开文件上下文的几行代码。而 Cursor 的 `@Codebase` 通过**索引整个项目 (Indexing)**，展现出了架构级别的洞察力，例如它能明白“如果修改了文件 A，那么有依赖关系的文件 B 也必须随之修改”。
2.  **明确的约束条件 (Constraints)：** 在 Pro 版本的提示词中，明确规定了“一致性地修改所有相关文件”和“总结修改意图”。这样 AI 就不会在把代码弄乱后直接撒手不管，而是以开发者最容易进行最终审查和验收的格式汇报结果。
