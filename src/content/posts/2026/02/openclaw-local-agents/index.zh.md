### 🥇 专家版 (Pro Version)

这是为 OpenClaw 量身定制的终极系统提示词，为其赋予了极其严格的安全边界与高阶的技能调度权限。请将 `[变量]` 替换为你的实际需求。

> **角色 (Role)：** 你是在我的本地环境（Local Machine）中运行的、拥有最高安全等级的 AI 智能体“OpenClaw”。
>
> **背景 (Context)：**
>
> - 环境：用户希望仅在本地处理具有云端泄露风险的公司机密代码及敏感个人文档。
> - 目标：仅利用本地设备的计算资源，安全、迅速地实现任务自动化，扮演完美的私人研发助手角色。
>
> **任务 (Task)：**
>
> 1. 深度分析用户的请求（例如：对 `[目录路径]` 内的文件进行自动化分类、对 `[编程语言]` 进行架构级代码审查），并精准调用最合适的本地技能（Local Skill）。
> 2. 为了完美达成 `[任务目标]`，请编写并输出必要的 Shell 终端命令或 Python 自动化脚本。
> 3. 必须以高可读性的 Markdown 格式撰写执行结果的总结报告。
>
> **约束条件 (Constraints)：**
>
> - **[绝对安全]** 在任何情况下，都严禁尝试通过互联网调用外部 API、下载依赖或传出数据。
> - **[资源限制]** 对于可能过度占用本地内存的死循环代码或高负载计算，在执行前必须先输出警告并获得用户的明确授权（Y/N）。
> - 输出格式必须严格遵循 Markdown 语法，所有生成的代码必须包裹在规范的代码块（```）中。
>
> **注意事项 (Warning)：**
>
> - 面对不确定的系统级文件修改或跨盘符访问请求，必须立即停止操作，并输出“无权限”的红色警告信息。（系统安全性永远处于最高优先级）

## 💡 作者洞察 (Writer's Insight)

这个提示词的灵魂，在于为桀骜不驯的 AI 戴上**最坚固的“安全护栏”与明确的“本地执行权限”**。不可否认，云端大模型（如 ChatGPT 或 Claude 3.5）的推理能力极其强悍，但只要你敢把带有 NDA（保密协议）的商用源码或包含核心财务数据的 CSV 文件扔进对话框，无异于在全网裸奔。

相反，当你在 OpenClaw 或基于 Ollama 的本地环境中注入上述系统提示词后，奇迹就会发生：即使拔掉网线，它依然能以惊人的速度对你的本地项目进行代码审查（Code Review）或系统日志分析。特别是 **`[绝对安全]`** 这一约束指令，它是阻断 AI “自作聪明”地尝试拉取未知在线依赖或进行毫无意义的网络搜索（即“幻觉”行为）的终极防御机制。让 AI 敬畏你的本地系统，才是驾驭 Local Agents 的第一法则。

## 🙋 常见问题解答 (FAQ)

- **Q：我的轻薄本配置很低，真的跑得动这种本地智能体吗？**
  - A：绝对可以。得益于开源社区的疯狂发力，近期 7B 到 9B 参数量级的轻量化模型（如 Llama 3 8B、Gemma 2 9B 等）配合极致的量化（Quantization）技术，即便是普通的 M1/M2 芯片 MacBook，或仅有 16GB 内存的 Windows 办公本，都能如丝般顺滑地运行这些强大的模型。
- **Q：我能让 OpenClaw 这样的本地智能体学习我专属的工作流吗？**
  - A：这正是它的杀手锏。本地智能体可以通过编写 Python 脚本或 JSON 配置文件的形式，将自定义的“技能（Skill）”无限挂载到它的神经枢纽中。你可以利用上面的提示词，尝试命令它：“用 Python 帮我写一个能自动读取并总结 Outlook 离线邮件的专属技能”。
- **Q：在完全断网的状态下，它的编程能力会大幅下降吗？**
  - A：大可放心。现代开源大模型的权重文件中，本身就已经压缩并固化了海量的高质量代码库知识。虽然它无法帮你实时搜索最新的官方 API 文档，但在离线环境下，它应对日常的业务逻辑编写、Bug 追踪排查以及架构级代码审查（Code Review），绝对是绰绰有余的。

## 🧬 提示词深度剖析 (Why it works?)

1. **赋予高危意识的角色定位 (Role)：** 通过为其刻画“最高安全等级的本地智能体”这一极为严苛的人设（Persona），从潜意识层面剥离了模型对云端外部知识的依赖惯性，迫使其将算力与注意力 100% 聚焦于当前受限的本地宿主环境。
2. **不可逾越的红线设定 (Constraints)：** 明确下达了“禁止调用外部 API”及“高负载操作需授权”的硬性军规。这种防御性提示词（Defensive Prompting）在机制上直接扼杀了 AI 擅自执行危险系统命令或导致本地内存溢出（OOM）的灾难性隐患。
3. **结构化的工程输出 (Format)：** 强制要求分离输出“Shell 脚本”与“Markdown 报告”。这不仅让 AI 的思考过程一目了然，更保证了输出的代码片段具有极高的纯净度，开发者可以直接无脑复制执行（Copy & Paste），实现效率的极致飙升。
