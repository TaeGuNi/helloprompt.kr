# 💀 AIの魂を削るスパルタコーディングチートキー (TypeScript Core)
- 🎯 **おすすめの対象:** AIの「はい、承知いたしました！」という無駄な愛想笑いにウンザリしているシニアエンジニア、スパゲッティコードを親の仇のように嫌うテックリード
- ⏱️ **所要時間:** 1時間のデバッグ → わずか1秒で解決
- 🤖 **推奨モデル:** Claude 3.5 Sonnet, GPT-4o, Gemini 1.5 Pro (コーディング特化モデル推奨)
- ⭐ **難易度:** ⭐⭐⭐⭐☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐
_"AIがいつまで経っても時代遅れの npm install を叩き、any 型でコードを汚染していく様に、もはや我慢の限界ではありませんか？このチートキーで、奴らの甘え腐った悪習を無慈悲に粉砕してやりましょう。"_
AIにコーディングを任せたとき、時代遅れの設定や緩すぎる型付けのせいで、プロジェクトがレガシーコードのゴミ捨て場と化していく様を見て、思わずモニターを叩き割りたくなったことはありませんか？この記事は、そんな絶望を味わってきたエンジニアのための救世主です。このハードコアなプロンプトを叩き込めば、AIは無駄な美辞麗句を一切捨て去り、あなたのプロジェクトに**極限のI/O効率 (pnpm)、一切の妥協を許さない無慈悲な型安全性 (Zod/Strict)、そして超高速なリンティング (Biome) とテスト (Vitest)** だけを完璧に導入する、冷徹なスパルタ戦士へと豹変します。
---
## ⚡️ 3行まとめ (TL;DR)
- 🚀 **`pnpm` の絶対強制:** ディスク容量を食いつぶし、ネットワーク遅延を引き起こす npm/yarn は即刻処刑します。
- 🛡️ **無慈悲な型統制:** `any` 型の使用はプロジェクトへの反逆です。`zod` による厳格なランタイム検証と `strict: true` を強制します。
- ⚡ **Biome & Vitest の導入:** 鈍重な eslint、prettier、jest をゴミ箱へ叩き込み、光の速度でリンティングとテストを実行させます。

> **要求:** `[問題およびリファクタリングするコード]` を解決してくれ。npm の代わりに必ず pnpm を使い、any を禁止して zod で検証しろ。リンティングは Biome、テストは Vitest でセットアップしてくれ。

## 🙋 よくある質問 (FAQ)
- **Q: プロンプトで縛っても、AIが頑なに `npm` を使おうとする場合はどうすればいいですか？**
  - A: そのような場合は、プロンプトの末尾に「二度と npm を提案するな。次やったらプロセスを強制終了する」という強烈な警告を1行追加してみてください。冗談のように聞こえるかもしれませんが、AIの行動を矯正する上で驚くほどの効果を発揮します。
- **Q: このプロンプトは、フロントエンド開発 (React/Next.js) でも通用しますか？**
  - A: もちろんです！TypeScriptが動く環境であれば、Webフロントエンドであれバックエンドであれ、いかなる領域でもこの無慈悲なルールを完璧に適用できます。
- **Q: Zod の代わりに Yup を採用しても問題ありませんか？**
  - A: どうしても型推論の精度が甘いツールを使いたいのであれば止めはしません。しかし、このプロンプトの根底にある哲学は「極限の型安全性」の追求です。特別な理由がない限り、素直に Zod を採用することを強く推奨します。

## 🧬 プロンプト解剖 (Why it works?)
- 🎯 **ペルソナの極大化:** AIに「一切の妥協を許さないシニアアーキテクト」という厳格な自我を憑依させることで、「とりあえず動けばいい」という次元の低いコードを生成する悪習を根絶しました。
- 🚫 **明示的な禁止語 (Negative Prompt) の設定:** `npm`、`yarn`、`any`、`eslint`、`jest` など、AIが息を吐くように提案してくるレガシーツール群を名指しで徹底的に封じ込めています。
- 🔒 **防御的コーディングの仕組み化:** `zod` の使用を強制することで、ランタイムで致命傷になり得るバグをAI自身が未然に防ぐよう、アーキテクチャレベルでの強力な制約を機能させています。
