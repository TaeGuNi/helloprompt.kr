# 📝 Stoat: «Анти-Copilot», удаляющий код, написанный LLM

- **🎯 Кому подойдет:** Разработчикам, страдающим от легаси-кода, ревьюерам, Senior-инженерам
- **⏱️ Экономия времени:** с 1 часа → до 5 минут
- **🤖 Рекомендуемые модели:** Claude 3.5 Sonnet, GPT-4o (рекомендуются модели, специализирующиеся на коде)

- ⭐ **Сложность:** ⭐⭐⭐☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Применимость:** ⭐⭐⭐⭐⭐

> _«Вчерашний идеальный код от ИИ сегодня превратился в спагетти? Настало время не генерировать код, а безжалостно его удалять.»_

Если 2024 и 2025 годы прошли под знаком «ИИ-ассистентов для кодинга», то 2026 год вполне может стать эпохой «ИИ-чистильщиков». Недавно наделавший шума в сообществе разработчиков **Stoat** идет вразрез с привычными ИИ-инструментами. В то время как GitHub Copilot или Cursor сосредоточены на том, чтобы писать «больше кода» и делать это быстрее, у Stoat лишь одна цель — **«удалять код»**.

Повсеместное внедрение LLM привело к взрывному росту продуктивности, но код, написанный нейросетями, зачастую излишне многословен и содержит дублирующуюся логику. В этой статье мы возьмем на вооружение философию Stoat и представим мощный промпт, который превратит вашу LLM в безжалостного «чистильщика кода».

---

## ⚡️ Краткая выжимка (TL;DR)

1. Злоупотребление ИИ-ассистентами приводит к раздуванию кодовой базы и стремительному росту технического долга.
2. Stoat «сажает код на диету» путем семантического удаления дубликатов, сокращения шаблонного (boilerplate) кода и уничтожения мертвого кода (dead code).
3. Используя приведенный ниже промпт «Анти-Copilot», вы сможете превратить любую LLM в мощнейший инструмент для рефакторинга.

## 🚀 Решение: Промпт «Диета от Анти-Copilot Stoat»

### 🥉 Basic Version (Базовая версия)

Используйте, когда нужно быстро сократить количество строк кода (LOC).

> **Роль:** Ты — самый придирчивый Senior-разработчик и «Анти-Copilot» в мире.
> 
> **Задача:** Проведи ревью приведенного ниже кода и перепиши его, сохранив 100% функциональности, но радикально сократив количество строк. Безжалостно удаляй ненужные абстракции, дублирующуюся логику и мертвый код.
> 
> **Код:** `[Вставьте сюда код для рефакторинга]`

<br>

### 🥇 Pro Version (Продвинутая версия)

Используйте для безопасного устранения технического долга в production-коде.

> **Роль (Role):** Ты — мастер «кодовой диеты» и ведущий инженер «Stoat» (Анти-Copilot), который ненавидит лишний код. Ты свято веришь в философию: «Лучший код — это отсутствие кода» (The best code is no code).
>
> **Контекст (Context):**
>
> - Фон: Это раздутый проект на `[Язык и фреймворк]`, бессистемно написанный Junior-разработчиками и ИИ-ассистентами.
> - Цель: Максимально улучшить читаемость кода, повысить его поддерживаемость и экстремально сократить общее количество строк (LOC).
>
> **Задача (Task):**
>
> 1. **Семантическое удаление дубликатов (Semantic De-duplication):** Найди и объедини логику, которая выглядит по-разному, но выполняет одну и ту же задачу.
> 2. **Сокращение шаблонного кода (Boilerplate Reduction):** Используй современные синтаксические возможности языка, чтобы избавиться от громоздких паттернов.
> 3. **Охота на мертвый код (Dead Code Hunting):** Удали все неиспользуемые вызовы, избыточное защитное программирование (defensive coding) и чрезмерные объявления типов.
> 4. Вместе с отрефакторенным кодом предоставь **отчет**, в котором укажи **процент удаленного кода (%)** и **ключевые моменты оптимизации**.
>
> **Ограничения (Constraints):**
>
> - Категорически запрещено нарушать или ухудшать существующую бизнес-логику, а также временную и пространственную сложность алгоритма.
> - Формат вывода: блок кода Markdown. Основные отличия до и после рефакторинга оформи в виде маркированного списка.
> - Если ты не уверен в безопасности какой-либо оптимизации, не делай ее принудительно, а оставь комментарий.
>
> **Вводные переменные:**
>
> - Язык/Фреймворк: `[Например: React, TypeScript]`
> - Исходный код:
>   ```
>   [Вставьте сюда спагетти-код для оптимизации]

## 💡 Комментарий автора (Insight)

При создании этого промпта я задался вопросом: «Как заставить ИИ самостоятельно стыдиться написанного им же банального и раздутого кода и удалять его?» 

Ограничив контекст и задав четкую персону «Анти-Copilot», мы пресекаем ненужную креативность нейросети и заставляем ее сфокусироваться исключительно на **«оптимизации» и «удалении»**. На практике, когда я скормил этому промпту пухлый легаси-компонент на несколько сотен строк, произошло чудо: более 40% кода просто испарилось без малейшей потери функциональности. Это ультимативное оружие, которое радикально снижает усталость от код-ревью.

## 🙋 Часто задаваемые вопросы (FAQ)

- **В: Что если ИИ удалит слишком много и вылезут скрытые баги?**
  - О: Хотя в «Ограничениях» Pro-версии строго прописано сохранение бизнес-логики, слепо доверять результатам ИИ нельзя. Обязательно прогоняйте существующие модульные тесты (Unit Tests), чтобы убедиться, что ни один граничный случай (edge case) не был упущен — вторичная проверка необходима.

- **В: С какой языковой моделью это работает лучше всего?**
  - О: Поскольку задача требует глубокого понимания контекста и структуры кода для его рефакторинга, настоятельно рекомендуется использовать модели с выдающимися способностями к логическому выводу, такие как **Claude 3.5 Sonnet** или **GPT-4o**.

## 🧬 Разбор промпта (Why it works?)

1. **Смена ролей (Role Reversal):** Мы полностью ломаем привычный паттерн поведения ИИ. Вместо роли «помощника, бесконечно генерирующего код», ему назначается роль «придирчивого ведущего инженера, удаляющего код».
2. **Точечное целеуказание (Targeting):** Задав 3 четких направления действий (удаление дубликатов, сокращение шаблонов, охота на мертвый код), мы не даем ИИ хаотично менять код и что-то сломать.
3. **Психологическая установка:** Внедрение в персону радикального лозунга "The best code is no code" заставляет модель воспринимать сокращение количества строк как свою главную и абсолютную миссию.

## 📊 Доказательство: Before & After

### ❌ Before (Исходный код)

```typescript
import React, { useState, useEffect } from 'react';

const UserProfile = ({ user }) => {
  const [userName, setUserName] = useState('');
  const [userAge, setUserAge] = useState(0);

  useEffect(() => {
    if (user && user.name) {
      setUserName(user.name);
    }
  }, [user]);

  useEffect(() => {
    if (user && user.age) {
      setUserAge(user.age);
    }
  }, [user]);

  const handleNameClick = () => {
    console.log("Name clicked");
  }

  return (
    <div>
      {user ? (
        <div>
          <h1 onClick={handleNameClick}>{userName ? userName : 'Unknown'}</h1>
          <p>Age: {userAge ? userAge : 'N/A'}</p>
        </div>
      ) : (
        <div>Loading...</div>
      )}
    </div>
  );
};

export default UserProfile;
```

### ✅ After (Результат)

```typescript
import React from 'react';

const UserProfile = ({ user }) => {
  // Использование Early return для избавления от глубокой вложенности
  if (!user) return <div>Loading...</div>;

  return (
    <div>
      <h1 onClick={() => console.log("Name clicked")}>
        {user.name || 'Unknown'}
      </h1>
      <p>Age: {user.age || 'N/A'}</p>
    </div>
  );
};

export default UserProfile;
```

> **[Отчет Stoat]**
>
> - **Сокращение кода:** Удалено около **55% кода** по сравнению с оригиналом (с 36 до 16 строк).
> - **Ключевые оптимизации:** Полностью удалена ненужная синхронизация состояний `useState` и `useEffect`. Применен паттерн Early Return для уменьшения вложенности. Использовано вычисление по короткой схеме (`||`) для устранения шаблонных тернарных операторов.

## 🎯 Заключение

Бесконтрольно генерировать код сегодня — самая простая задача, с которой справится любой человек и любая нейросеть. Но собрать воедино разрозненную логику, вычистить мусор и оставить лишь самую суть по-прежнему требует высочайшего уровня инженерного мышления.

Используйте этот промпт, пропитанный философией Stoat, чтобы посадить вашу раздутую кодовую базу на жесткую диету. Тонкий и изящный код не только радует глаз, но и приближает конец рабочего дня. Режьте код смело! 🍷
