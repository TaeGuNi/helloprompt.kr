## ⚡️ 3行摘要 (TL;DR)

1. 从 AWS Cost Explorer 导出“按月/按服务分类的成本” CSV 数据。
2. 将该数据与当前架构信息喂给 AI，令其排查出所有“低效的支出项”。
3. 立即在实际业务中落地 AI 建议的竞价实例 (Spot Instance)、自动扩展 (Auto Scaling) 及 Savings Plan 策略。

## 🚀 解决方案："Cloud Cost Cutter" 提示词

### 🥉 Basic Version (基础版：揪出僵尸资源)

当您对资金流失毫无头绪时，使用此提示词可快速揪出闲置的冗余资源。

> **角色：** 你是一位资深云架构师 (Senior Cloud Architect)。
> **请求：** 我查看了 AWS 账单，发现 `[特定服务名称，例如：EBS 卷]` 的费用异常偏高。请帮我找出所有未连接 (Unattached) 的卷、过期的快照以及已分配但未使用的 Elastic IP，并提供准确的 AWS CLI 删除命令以及相关注意事项。

### 🥇 Pro Version (专业版：竞价实例与架构优化)

当您需要系统性的降本与自动化扩展策略时，请直接套用这款高阶提示词。

> **角色 (Role)：** 你是一位负责企业级基础设施的首席 AWS FinOps 顾问。
>
> **背景 (Context)：**
>
> - **当前架构：** 5 台 EC2（On-Demand 按需实例，24 小时常驻），1 台 RDS (MySQL)。
> - **流量模式：** 流量仅集中在工作日时段（09:00~18:00），夜间和周末几乎零流量。
> - **目标：** 在保障现有业务稳定性的前提下，将基础设施成本至少压降 50%。
>
> **请求 (Task)：**
>
> 1. **竞价实例 (Spot Instance) 引入策略：** 针对当前工作负载，提出一套安全利用竞价实例使计算成本降低 70% 以上的架构方案。（必须包含实例中断时的容灾机制）
> 2. **自动扩展 (Auto Scaling) 与调度：** 提供具体的 `ASG (Auto Scaling Group)` 配置值和 `Instance Scheduler` 落地指南，以便在夜间将服务器缩容至最低限度（1 台），并在周末彻底关闭开发/测试环境。
> 3. **承诺折扣 (Savings Plan / RI)：** 请以 Markdown 表格形式，直观对比采用 1 年预付承诺（Compute Savings Plan）与当前按需付费模式的预计节省金额。
>
> **限制条件 (Constraints)：**
>
> - 必须基于 AWS 官方的“架构完善框架” (Well-Architected Framework) 提出建议。
> - RDS 数据库**绝对禁止**使用竞价实例。
>
> **注意事项 (Warning)：**
>
> - 省去冗长空洞的概念解释，直接输出工程师明早就能照做执行的、极具实操性的行动清单 (Action Item)。

## 💡 作者点评 (Insight)

在云端成本管理中，最常见也最致命的误区就是**“在周末和夜间依然让开发/测试环境（Dev/Stage）24 小时空转”**。仅仅是引入 `Instance Scheduler` 在非工作时间自动关停这些实例，就能立竿见影地砍掉开发服务器 60% 以上的开销。基于上述提示词获取优化思路后，您完全可以向 AI 下达进阶指令：**“请帮我编写能够落地该策略的 Terraform 代码（或 CloudFormation 模板）”**，从而一步到位实现基础设施的自动化降本。

## 🙋 常见问题 (FAQ)

- **Q: 听说竞价实例 (Spot Instance) 随时可能被系统强制回收，这难道不危险吗？**
  - A: 确实有风险。因此，竞价实例仅适用于无状态 (Stateless) 的 API 服务器、容器化工作负载或异步批处理作业 (Batch Processing)。**绝对不能**将其用于承载核心流量的主流服务器或数据库集群。

- **Q: RDS（数据库）的费用实在太高，这块该如何降本？**
  - A: 与其死守昂贵的按需实例，不如考虑引入可根据流量弹性伸缩的 Aurora Serverless v2，或是通过分离只读副本 (Read Replica) 来分散负载并优化架构。您可以直接让 AI 帮您评估这种架构演进的改造成本及预期收益。

## 🧬 提示词剖析 (Why it works?)

1. **精准界定流量模式：** 交代了“日间集中，夜间/周末闲置”的具体业务底色，引导 AI 产出基于**调度和动态伸缩**的高阶策略，而非一句干瘪的“建议降低实例规格”。
2. **锚定量化目标（降本 50%）：** 与其泛泛而谈“帮我省点钱”，不如直接拍出硬性数字指标，倒逼 AI 给出引入竞价实例或 Savings Plan 等突破性的大招。
3. **严守风险底线 (Constraints)：** 明确下达“数据库严禁使用竞价实例”的死命令，提前掐断了 AI 因盲目追求降本而给出导致服务宕机的致命幻觉 (Hallucination)。
