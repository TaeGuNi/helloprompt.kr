## 📝 摆脱 Excel、JSON、CSV... 数据格式转换的繁重体力劳动

- **🎯 推荐对象：** 开发人员、数据分析师、绩效营销人员
- **⏱️ 所需时间：** 30分钟 → 缩短至1分钟
- **🤖 顶尖性能：** Claude 3.5 Sonnet (擅长复杂结构转换), ChatGPT (GPT-4o)

> _“副经理，要把这个 Excel 数据上传到网页，能帮我改成 JSON 格式吗？啊，还有，键名（Key）请全部统一成英文小写蛇形命名法（snake_case）。”_

如果你是处理数据的实务人员（开发人员、数据分析师、绩效营销人员等），不可避免地会遇到被称为<b>“数据格式转换”</b>的枯燥重复劳动。为了将策划人员或客户提供的数千行 Excel 数据推送到前端或后端系统，你需要将其映射并转换为 JSON、CSV 或 XML 结构。遇到这种情况，你是否也曾感到头大？如果只是简单地更改扩展名，直接使用 Excel 的“另存为”就足够了。但现实中的工作绝非如此简单。

“这里的空白请全部明确处理为 `null`，各种各样的日期格式请全部统一为 ISO 标准格式 `YYYY-MM-DD`。另外，考虑到安全问题，身份证号等敏感列请直接删除后再提取。转换后的键名（Key）请全部符合前端规范，统一使用驼峰命名法（camelCase）。”

一旦加入这些细节要求，原本看似简单的任务瞬间就会变成<b>可怕的苦差事</b>。使用网上随处可见的免费 JSON 转换网站，又担心违反公司的信息安全规定，不敢轻易上传公司数据。如果特意为了这个一次性的数据转换而去编写 Python 的 `pandas` 脚本，或者搜索复杂难懂的正则表达式（Regex）来写代码，又会产生一种“为了这点小事值得投入我昂贵的时间和精力吗？”的深深挫败感。尝试利用 Excel 的 VLOOKUP 或替换功能耍点小聪明，结果漏掉了一个括号或一个逗号导致解析（Parsing）错误，就必须盯着数千行文本费力寻找原因。

最痛苦的是当数据本身具有深度复杂的<b>嵌套结构（Nested Structure）</b>时。当需要将平铺（Flat）的 Excel 表格编织成多层级的 JSON 树时，简单的替换或脚本往往难以应对。好不容易熬夜写好了代码，结果策划案一变，数据结构也跟着变了，之前写好的解析逻辑全部作废，只能从头再来。在这种令人厌烦的数据流水线循环中，实务人员的精神状态会逐渐崩溃。

但现在，你再也不必纠结于 Excel 单元格，也不必在 Google 上搜索正则表达式浪费时间了。你可以使用<b>将大型语言模型（LLM）作为完美的定制化数据转换器和清洗（Cleansing）工具</b>的终极提示词。无需复杂的编程或编写 Python 脚本，只需将杂乱的原始数据原样复制并扔给 AI，然后像对人说话一样用自然语言指示：“请按照这些规则加工并转换数据”，一切就结束了。这种方式超越了简单的文本解析，AI 会自行理解数据的<b>上下文（Context）</b>和结构并智能地重新编排，这与传统的笨拙转换器有着质的区别。

这个提示词不仅能完成简单的格式转换（CSV → JSON 等），还能通过<b>一次提示词完美执行</b>苛刻的数据结构映射、字段名批量标准化、数据类型转换（Type Casting）、删除不必要的垃圾数据及敏感信息。原本需要 30 分钟甚至 1 小时以上的乏味痛苦的数据清洗工作，只需 1 分钟，就能以毫无差池的完美 Markdown 代码块形式展现在你的显示器上。错综复杂的数据将变身为美观整齐的结构，随时可以插入前端或后端代码。现在剩下的只有点击“复制”并直接应用到你的代码库中了。请亲自体验这种能大幅提前下班时间的数据转换魔法吧。

## 💡 作者点评 (Insight & How to use)

这个提示词的真正价值和威力在于<b>将“数据清洗（Data Cleansing）”和“格式转换（Format Conversion）”这两个原本独立且繁琐的流程完美整合为一个单一的流水线</b>。在实务现场，最令数据工程师或开发人员苦恼的并不是简单的将 CSV 包装成 JSON Array 的外壳转换工作，而是为了符合前端 API 规范，将韩文或各异的英文键名（Key）批量修改为驼峰式（camelCase）或蛇形式（snake_case），以及将乱七八糟的日期格式（如 `23/11/12`、`2023年 1月 5일` 等）统一为 ISO 8601 标准或 `YYYY-MM-DD` 格式的<b>预处理（Pre-processing）过程</b>，因为这更加痛苦且需要大量的冗余时间。

Pro 版提示词的核心就在于 `[转换规则]` 项。这个空间是一个<b>魔法盒，你可以用自然语言轻松扔进无数难以通过 Python 脚本或复杂 SQL 查询实现的边缘案例（Edge Cases）</b>。例如，尝试下达非常详细的业务逻辑指令，如 *“空白值不是简单的空字符串，而是明确处理为 `null`，职业栏空缺的人员统一将文本替换为 ‘无业’”*，或者 *“只过滤并提取客户电子邮箱后缀为 `@gmail.com` 的数据”*。按照传统方式，你需要混合编写数十行条件语句（If-else）和正则表达式，而 AI 能以惊人的准确度把握语境，并完美地按照你的喜好加工数据。特别是像<b>条件映射（Conditional Mapping）</b>这样的复杂任务，甚至连“将年龄在 20 多岁且居住在首尔的用户的特定标志位改为 true”这样的查询，也可以通过一行自然语言规则来处理。当面对数据结构不一致、中间缺少字段或格式损坏的<b>脏数据（Dirty Data）</b>时，AI 的真实价值就会显现出来。如果是严格的正则引擎，可能会报错并停止运行，但 LLM 会通过推断语境来进行最合适的自动校正（Auto-correction）。

特别是在处理复杂的数据结构转换任务时，<b>强烈建议使用 Claude 3.5 Sonnet 模型</b>。虽然 ChatGPT (GPT-4o) 性能也很出色，但在解析深度极深且数据结构错综复杂的多层嵌套 JSON 对象（Nested Object）并将其重构为全新的层级结构方面，Claude 3.5 Sonnet 表现出了压倒性的稳定性和精细度。在需要颠覆结构层级、将现有一维数据分离为关系型（Relational）形式的高难度模式迁移（Schema Migration）场景中，Claude 拥有近乎完美的转换率。请亲自验证它如何能听懂你编写的甚至是有些简陋的转换规则，并将其映射到最优化的数据结构中。

此外，在实务中使用提示词的一个重要技巧是保持<b>“对输入输出的严格控制”</b>。在约束条件（Constraints）部分放置 *“严禁随意遗漏数据或编造不存在的值”* 这一防止幻觉（Hallucination）的提示词，是为了预先防止 LLM 随意推测模式并自行生成垃圾数据而导致的严重数据污染事故。务必交叉检查转换前的原始行（Row）数与转换后的结果对象数是否一致。通过 *“不要输出关于转换过程的说明或你的见解，只在代码块中展示最终数据”* 这一指示，完成了无需拖动鼠标即可直接“复制-粘贴（Copy-Paste）”转换后代码的<b>最佳实用用户体验（UX）</b>。

最后，从安全角度出发，处理公司机密信息或<b>个人信息（PII）</b>时需要注意。在通过 LLM 之前，请构建一个流水线，将关键信息（姓名、邮件原文等）在本地进行哈希（Hash）处理或掩码处理后再投入转换工作。这个提示词不仅仅是一个简单的一次性转换器，它将成为你在组织内建立安全且一致的数据标准化（Data Standardization）流程的基石。现在，再也不要为了寻找一个逗号、一个括号而瞪大眼睛清洗数据，浪费你宝贵的精力了。这个提示词将成为为你 24 小时待命的、最可靠、最干练的数据工程师。

## 🙋 常见问题 (FAQ)

- **Q: 可以直接上传 Excel 文件 (.xlsx) 进行转换吗？**
  - A: 搭载了高级数据分析（Advanced Data Analysis）功能的 ChatGPT (GPT-4o) 或 Claude 可以完美地分析并解析上传的 Excel 文件。但是，如果是对公司安全敏感的数据，比起上传整个文件，强烈建议只复制必要的文本内容并粘贴到提示词对话框中，并事先在本地对核心个人信息（身份证号、卡号等）进行掩码（Masking）处理。
- **Q: 如果需要转换的数据超过数万行，该如何处理？**
  - A: 如果数据量太大，由于超出了大型语言模型（LLM）的上下文窗口（Token 限制），输出可能会中途断开。在这种情况下，可以稍微改变提示词的目的，请求 *“请编写一段能完美执行此转换规则的 Python pandas 代码”*。在本地电脑上直接运行 AI 为你写好的完美脚本，是处理海量数据的最安全、最可靠的准则。
- **Q: 结果没有全部输出，代码块在中途断开了。**
  - A: 在对话框中输入 *“请继续输出”*，它就会从断开的地方开始准确地继续输出 Markdown。在提示词底部的约束条件部分预先写明 *“即使结果很长也不要随意省略，即使分段也要完整输出”* 也是实务中非常有用的技巧。

## 🧬 提示词解构 (Why it works?)

1. **赋予 ETL 专家人格：** 通过为 AI 植入“资深数据工程师”的专业自我，而非简单的“文本格式转换器”，诱导其主动怀疑数据的完整性（Integrity）并仔细交叉验证数据类型（Type）错误。

3. **严格的输出控制 (Constraints)：** 设定了 *“省略多余说明，只输出代码块”* 的强力约束。这为用户提供了极大的便利，使其无需手动选取转换后的数据即可直接复制并应用到实务代码库中。
