# 📝 Stoat: LLMが書いたコードを削除するアンチ・コパイロット

- **🎯 おすすめの対象:** レガシーコードに悩む開発者、コードレビュアー、シニアエンジニア
- **⏱️ 所要時間:** 1時間 → 5分に短縮
- **🤖 おすすめのモデル:** Claude 3.5 Sonnet, GPT-4o (コーディング特化モデルを推奨)

- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「昨日AIに書かせたコード、今日見たらスパゲッティになっていませんか？今はもうコードを『生成』する時代ではなく、『削除』する時代です。」_

2024年と2025年が「AIコーディングアシスタント」の年だったとすれば、2026年は「AIクリーナー」の年になるかもしれません。最近、開発者コミュニティで話題を集めている**Stoat**は、従来のAIツールとは全く逆の道を歩んでいます。GitHub CopilotやCursorが「より多くのコード」を素早く記述することに集中していたのに対し、Stoatの目標はただ一つ。それは**「コードを消し去ること」**です。

LLMベースのコーディングツールの普及により生産性は爆発的に向上しましたが、AIが作成したコードには必要以上に冗長なロジックや重複が含まれていることがよくあります。この記事では、Stoatの哲学をベンチマークし、お使いのLLMを「冷酷なコード清掃員（Stoat）」へと変貌させる強力なプロンプトを紹介します。

---

## ⚡️ 3行まとめ (TL;DR)

1. AIコーディングアシスタントの乱用により、コードベースが肥大化する「技術的負債」が爆発的に増加しています。
2. Stoatは、意味論的（セマンティック）な重複排除、ボイラープレートの削減、デッドコードの削除を通じて、コードのダイエットを実行します。
3. 以下の「アンチ・コパイロット・プロンプト」を使えば、どのLLMも即座に強力なコードリファクタリングツールとして活用できます。

---

## 🚀 解決策：「Stoat アンチ・コパイロット・ダイエット・プロンプト」

### 🥉 Basic Version (基本型)

手早くコードの行数（LOC）を減らしたいときに使用してください。

> **役割:** あなたは世界で最も厳しい「アンチ・コパイロット（Anti-Copilot）」のシニア開発者です。
> **タスク:** 以下のコードをレビューし、機能は100%維持したまま、コードの行数を最小限に減らして書き直してください。不要な抽象化、重複したロジック、デッドコードは容赦なく削除してください。
> **コード:** `[ここにリファクタリングするコードを貼り付ける]`

<br>

### 🥇 Pro Version (専門家型)

プロダクションレベルのコードから安全に技術的負債を清算したいときに使用してください。

> **役割 (Role):** あなたはコードダイエットの職人であり、不要なコードを嫌う「Stoat（アンチ・コパイロット）」のシニアエンジニアです。「最高のコードとは、コードがないことだ（The best code is no code）」という哲学を猛信しています。
>
> **状況 (Context):**
>
> - 背景：ジュニア開発者とAIコーディングアシスタントが無秩序にコードを書き、肥大化してしまった `[使用言語およびフレームワーク]` のプロジェクトです。
> - 目標：コードの可読性を高め、保守性を極大化し、全体の行数（LOC）を極端に減らすことです。
>
> **タスク (Task):**
>
> 1. **Semantic De-duplication（意味論的な重複排除）:** 見た目は違っても同じ役割を果たすロジックを見つけて統合してください。
> 2. **Boilerplate Reduction（ボイラープレートの削減）:** 最新の言語構文を活用して、冗長なパターンを排除してください。
> 3. **Dead Code Hunting（デッドコード狩り）:** 呼び出されていないコードや不要な防衛的プログラミング、過度な型宣言をすべて削除してください。
> 4. リファクタリングされたコードとともに、**削除されたコードの割合（%）**および**主な最適化ポイント**をレポートとして作成してください。
>
> **制約事項 (Constraints):**
>
> - 既存コードのビジネスロジックや、時間的・空間的計算量を損なったり低下させたりしては絶対にけません。
> - 出力形式はMarkdownのコードブロックを使用し、リファクタリング前後の主な違いを箇条書き（ブレットポイント）で整理してください。
> - 確信が持てない最適化は無理に行わず、コメントを残してください。
>
> **入力変数:**
>
> - 言語/フレームワーク: `[例: React, TypeScript]`
> - 元のコード:
>   ```
>   [ここに最適化したいスパゲッティコードを貼り付けてください]
>   ```

## 🙋 よくある質問 (FAQ)

- **Q: コードを消しすぎて、隠れていたバグが発生したらどうすればいいですか？**
  - A: Proプロンプトの「制約事項」でビジネスロジックの保存を強く明記していますが、AIの出力を盲信してはいけません。必ず既存の単体テスト（Unit Test）を実行し、エッジケースが漏れていないか二次検証を行う必要があります。

- **Q: どの言語モデルで最も高いパフォーマンスを発揮しますか？**
  - A: コードの文脈や構造を深く理解しリファクタリングする作業であるため、論理的推論能力が圧倒的な **Claude 3.5 Sonnet** や **GPT-4o** の使用を強く推奨します。

## 🧬 プロンプトの解剖 (Why it works?)

1. **役割の逆転 (Role Reversal):** AIに「無限にコードを吐き出す助手」ではなく、「コードを削除する厳しいシニアエンジニア」という役割を与えることで、従来の行動パターンを完全に覆しました。
2. **具体的なターゲット設定 (Targeting):** 重複排除、ボイラープレートの削減、デッドコード狩りという3つの明確なアクションアイテム（Action Item）を指示し、AIがコードを曖昧にいじって壊すのを防ぎました。
3. **心理的哲学の注入:** 「The best code is no code（最高のコードはコードがないこと）」という極端なスローガンをペルソナに注入し、とにかくコードの行数を減らすことを至上課題とするよう誘導しました。

## 📊 証明：Before & After

### ❌ Before (入力：AIが無秩序に書いたReactコード)

```typescript
import React, { useState, useEffect } from 'react';

const UserProfile = ({ user }) => {
  const [userName, setUserName] = useState('');
  const [userAge, setUserAge] = useState(0);

  useEffect(() => {
    if (user && user.name) {
      setUserName(user.name);
    }
  }, [user]);

  useEffect(() => {
    if (user && user.age) {
      setUserAge(user.age);
    }
  }, [user]);

  const handleNameClick = () => {
    console.log("Name clicked");
  }

  return (
    <div>
      {user ? (
        <div>
          <h1 onClick={handleNameClick}>{userName ? userName : 'Unknown'}</h1>
          <p>Age: {userAge ? userAge : 'N/A'}</p>
        </div>
      ) : (
        <div>Loading...</div>
      )}
    </div>
  );
};

export default UserProfile;
```

### ✅ After (結果：Stoatプロンプト適用後)

```typescript
import React from 'react';

const UserProfile = ({ user }) => {
  // Early returnで不要な条件付きレンダリングの深さを排除
  if (!user) return <div>Loading...</div>;

  return (
    <div>
      <h1 onClick={() => console.log("Name clicked")}>
        {user.name || 'Unknown'}
      </h1>
      <p>Age: {user.age || 'N/A'}</p>
    </div>
  );
};

export default UserProfile;
```

> **[Stoat レポート]**
>
> - **コード削減率:** 元のコードと比較して約 **55% 削除完了** (36行 → 16行)
> - **主な最適化:** 不要な `useState` および `useEffect` による状態同期の完全な排除、Early Returnパターンの適用によるネストの深さ（Depth）の減少、短絡評価（`||`）を活用した三項演算子のボイラープレート削減。

## 🎯 結論

コードを無限に生成することは、今や誰でも、そしてどんなAIでもできる最も簡単な作業になりました。しかし、断片化されたロジックをつなぎ合わせ、不要なコードを取り除いて本質だけを残す作業には、依然として高いレベルの洞察力が求められます。

Stoatの哲学を込めたこのプロンプトを使って、肥大化したコードベースに強力なダイエットを施してみてください。より薄く、より軽くなったコードが、あなたの退社時間を早めてくれるはずです。さあ、思い切って削除しましょう！ 🍷
