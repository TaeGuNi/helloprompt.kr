# 複雑なプロンプトの構造化：CREATEフレームワーク

AIモデルが進化するにつれて、私たちはますます複雑なタスクを依頼するようになっています。単純な質問には単純なプロンプトで十分ですが、複雑なビジネスロジックや創造的な作業を指示する場合には、体系的なアプローチが必要です。

この記事では、プロンプトを論理的に構造化し、AIのパフォーマンスを最大化する**CREATEフレームワーク**を紹介します。

## CREATEフレームワークとは？

CREATEは、効果的なプロンプト構成のための6つの重要な要素の頭文字をとったものです。

1.  **C**ontext (文脈): AIに役割と状況を与えます。
2.  **R**equest (依頼): 実行すべき具体的な作業を明示します。
3.  **E**xamples (例示): 望ましい結果の例(Few-shot)を提供します。
4.  **A**djustments (調整): トーン、スタイル、制約条件を設定します。
5.  **T**ype of Output (出力形式): 結果の形式を指定します（表、コード、マークダウンなど）。
6.  **E**xtras (その他): エッジケースの処理や特別な指示を含めます。

---

## 段階別適用ガイド

### 1. Context (文脈設定)

AIにペルソナを与えると、回答の質が変わります。

> 「あなたは10年の経験を持つシニアソフトウェアエンジニアです。レガシーコードを現代的なクリーンコードにリファクタリングする専門知識を持っています。」

### 2. Request (具体的な依頼)

曖昧さを避け、明確な動詞を使用してください。

> 「以下に提供されたPython関数を分析し、可読性を高め、実行速度を最適化する方向でリファクタリングしてください。」

### 3. Examples (例示)

AIは例を通してパターンを学習します。入力と出力のペアを見せてください。

> **入力:**
> `def calc(x,y): return x+y`
>
> **出力:**
>
> ```python
> def add_numbers(a: int, b: int) -> int:
>     """2つの数字の和を返します。"""
>     return a + b
> ```

### 4. Adjustments (詳細調整)

回答のトーン＆マナーを決定します。

> 「説明は簡潔で専門的な用語を使用しつつ、ジュニア開発者でも理解できるように親切な口調を維持してください。」

### 5. Type of Output (出力形式)

結果物をどのように受け取るか指定します。

> 「結果は、まず変更されたコード全体をコードブロックで提示し、その下に変更理由を箇条書きで要約してください。」

### 6. Extras (追加指示)

予期せぬ状況を防ぎます。

> 「もしコードにセキュリティ上の脆弱性が発見された場合は、リファクタリングとは別に警告メッセージを出力してください。」

## 結論

CREATEフレームワークを使用することで、プロンプト作成時間を短縮しながら、結果物の品質を一貫して維持することができます。複雑な問題を解決する必要がある場合、漠然と質問するのではなく、CREATEの構造に合わせて考えを整理してみてください。AIはあなたが設計した分だけ賢く答えてくれるでしょう。
