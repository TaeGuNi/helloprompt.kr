![\"Stop Pasting Context: Build Your First MCP Server in 5 Minutes\"](/images/hooks/mcp-server-tutorial.jpg)

개발자나 데이터 분석가라면 누구나 공감할 끔찍한 일상이 있습니다. 바로 ChatGPT나 Claude의 빈 대화창을 띄워놓고, 사내 데이터베이스 스키마(`schema.sql`)나 수천 줄에 달하는 API 명세서를 드래그해서 복붙하는 끝없는 반복 작업입니다. "이 테이블 구조를 바탕으로 쿼리를 짜줘", "우리 사내 API 명세서가 이건데 어떻게 호출해야 해?"라며 매번 새로운 대화창을 열 때마다 텍스트 뭉치를 수동으로 떠먹여야 합니다. 

컨텍스트 윈도우가 조금만 꼬이거나 토큰 제한에 걸리면, AI는 엉뚱한 테이블을 참조하거나 존재하지 않는 컬럼을 지어냅니다. 심지어 절대 외부로 유출되어서는 안 될 <b>민감한 사내 보안 코드</b>나 <b>고객 데이터</b>를 브라우저 창에 복사해 넣을 때마다 등골이 서늘해집니다. 이 과정에서 우리는 코딩에 집중하는 것이 아니라, AI의 입맛에 맞게 프롬프트를 다듬고 컨텍스트를 조립하는 단순 노동자로 전락하고 맙니다.

이 무의미한 막노동을 언제까지 반복해야 할까요? 하루에 프롬프트 창을 10번만 새로 연다고 가정해도, 텍스트를 찾아 복사하고 붙여넣는 사전 작업에만 최소 30분이 허공으로 날아갑니다. 일주일이면 2시간 반, 한 달이면 하루 온종일을 온전히 텍스트 복붙에 버리는 셈입니다. 단순히 시간이 낭비되는 것을 넘어, 개발자의 창의력과 집중력마저 산산조각이 나버립니다.

더 치명적인 문제는 바로 <span style="color:var(--color-cyber-cyan)">컨텍스트의 파편화(Context Fragmentation)</span>입니다. 어제 붙여넣은 스키마가 오늘 오전에 업데이트되었는데, AI는 여전히 어제의 낡은 스키마를 바탕으로 코드를 짭니다. 결국 잘못된 코드로 인한 에러를 잡느라 수십 분을 허비하고, "차라리 내가 직접 짜고 말지"라며 화를 내게 됩니다. 남들은 AI로 퇴근 시간을 앞당긴다는데, 우리는 AI의 비위를 맞추고 학습시키느라 오히려 야근을 자처하고 있는 역설적인 상황에 빠진 것입니다.

하지만 2026년 현재, 이 모든 고통을 단숨에 끝낼 게임 체인저가 등장했습니다. 바로 Anthropic이 주도하는 <b>Model Context Protocol(MCP)</b>입니다. 이 거대한 기술적 도약이 AI 생태계의 표준으로 자리 잡으면서, 더 이상 AI에게 수동으로 데이터를 떠먹여 줄 필요가 완전히 사라졌습니다. AI가 내 로컬 데이터에 <b>직접 빨대를 꽂아 실시간으로 알아서 읽어오게</b> 만들 수 있기 때문입니다.

MCP는 쉽게 비유하자면 AI 모델 전용 <b>범용 USB 포트</b>와 같습니다. 내 컴퓨터에 서버를 단 한 번만 띄워두고 포트를 연결해 두면, Claude Desktop이나 Ollama 같은 클라이언트가 내 로컬 파일 시스템, 내부 데이터베이스, 심지어 폐쇄망에 있는 사내 API에 직접 접근해 최신 정보를 실시간으로 스트리밍합니다. 

더욱 놀라운 점은 이처럼 강력한 맞춤형 서버를 구축하는 데 단 5분이면 충분하다는 것입니다. 오늘 소개할 프롬프트를 활용하면, 개발자가 직접 복잡한 보일러플레이트 코드를 짜는 대신, AI에게 지시해 고유한 비즈니스 환경에 딱 맞는 완벽한 TypeScript 기반의 MCP 서버 코드를 순식간에 <b>스캐폴딩(Scaffolding)</b>할 수 있습니다. 수십 개의 의존성과 초기 설정을 헤맬 필요 없이, 완벽하게 동작하는 코드를 한 번에 얻을 수 있습니다.

이제 이 프롬프트를 적용한 후의 놀라운 변화를 상상해 보세요. 아무것도 없는 빈 대화창을 열고 <i>"오늘 새로 가입한 유저 수가 몇 명이야?"</i> 혹은 <i>"어제 발생한 결제 오류 로그의 핵심 원인이 뭐야?"</i>라고 자연스럽게 툭 던지기만 하면 끝입니다. 

Claude가 백그라운드에서 조용히 내 로컬 DB나 로그 파일에 직접 쿼리를 날리고, 한 치의 오차도 없는 정확한 최신 데이터를 기반으로 완벽한 답변을 내놓습니다. 데이터 유출 걱정 없이, 내 컴퓨터 안에서 가장 쾌적하고 안전하며 강력한 나만의 맞춤형 AI 인프라가 완성됩니다. 여러분은 이제 단순한 코더(Coder)를 넘어, 나만의 강력한 AI 시스템을 지휘하는 <b>아키텍트(Architect)</b>로 거듭나는 첫걸음을 내딛게 됩니다. 지금 바로 그 치트키를 공개합니다.

## ⚡️ 3줄 요약 (TL;DR)

1. **무의미한 복붙의 종말:** MCP를 활용해 LLM이 내 로컬 데이터(DB, 로그, API)에 직접 접근하고 상호작용하도록 만듭니다.
2. **단 한 줄로 서버 구축:** 번거로운 초기 설정 없이, 프롬프트 하나만으로 완벽하게 구동되는 맞춤형 MCP 서버 코드를 자동 생성합니다.
3. **압도적인 프라이버시 보호:** 민감한 데이터를 외부로 유출할 필요 없이, 가장 안전한 로컬 환경에서 모델의 컨텍스트 윈도우로 직접 스트리밍합니다.

> - 목표: 내 요구사항에 딱 맞는 단일 파일 형태(`index.ts`)의 완전하고 에러 없이 실행 가능한 MCP 서버 코드를 생성해야 합니다.
> 
> **요청 (Task):**
>

>    - **목적:** `[원하는 서버의 목적을 구체적으로 기재(예: 특정 데이터베이스의 오늘 가입자 수를 조회하는 기능)]`
> 2. 작성된 코드는 사용자가 터미널에서 즉시 테스트해 볼 수 있도록 상세한 주석과 함께 복사-붙여넣기 친화적으로 구성하세요.
> 
> **제약사항 (Constraints):**
> 
> - 반드시 공식 `@modelcontextprotocol/sdk` 패키지를 사용해야 합니다.
> - 도구의 입력 파라미터가 복잡할 경우, 스키마 검증을 위해 `zod` 패키지를 활용하세요(권장).
> - 서버 인스턴스 생성 시 `ListToolsRequestSchema` 및 `CallToolRequestSchema` 인터페이스를 완벽하게 구현해야 합니다.
> - 도구 핸들러(Handler) 내부에는 예외 처리(`try/catch`) 로직을 반드시 포함하여 서버 런타임이 다운되지 않도록 방어하세요.
> - **부연 설명 없이 오직 TypeScript 코드 블록 하나만 출력하세요.**
> 
> **주의사항 (Warning):**
> 
> - 실제로 존재하지 않는 가상의 SDK 메서드를 지어내지 마세요. 불확실한 경우 기본 Node.js API를 활용하여 우회해야 합니다.

## 💡 작성자 코멘트 (Insight & How to use)

이 프롬프트가 지니는 진짜 가치는 단순히 AI에게 코드를 짜달라고 부탁하는 막연한 접근을 넘어, **MCP 생태계가 요구하는 엄격한 프로토콜 규격을 AI에게 강제적으로 주입한다**는 데 있습니다. 

제가 처음 커스텀 MCP 서버를 만들어보려 했을 때 겪었던 처참한 실패가 이 프롬프트의 뼈대가 되었습니다. 단순히 "내 사내 DB를 읽어와서 알려주는 MCP 서버를 좀 만들어줘"라고 지시했더니, AI는 겉보기에는 아주 그럴싸해 보이지만 실제로는 아예 동작조차 하지 않는 **유령 SDK 메서드**를 잔뜩 지어냈습니다. MCP 통신의 핵심 라이프사이클인 `ListTools`와 `CallTool` 요청 처리는 까맣게 무시한 채, 그저 흔해 빠진 일반적인 Express.js 웹 서버 코드를 뱉어내기 일쑤였죠. 결국 그 엉망진창인 코드를 고치고 디버깅하느라 반나절이라는 아까운 시간을 허비해야만 했습니다.

이 뼈아픈 실패를 온몸으로 겪고 나서 깨달은 단 하나의 진리는, <span style="color:var(--color-cyber-cyan)">'초안 작성 단계에서 AI에게 아키텍처의 주도권을 절대 넘겨주면 안 된다'</span>는 점이었습니다. 

그래서 저는 프롬프트의 지시사항 내부에 공식 `@modelcontextprotocol/sdk`의 필수 인터페이스인 `ListToolsRequestSchema`와 `CallToolRequestSchema`를 반드시 구현하도록 명시적으로 못 박았습니다. 나아가 도구의 입력 파라미터가 꼬이는 것을 방지하기 위해 런타임 타입 검증 라이브러리인 `zod`의 사용을 강제하는 안전장치를 촘촘하게 박아 넣었습니다. 그 결과, 단 한 번의 프롬프트 실행만으로도 문법적 오류나 타입 에러가 전혀 없는, 즉시 터미널에서 실행 가능한 **완벽한 단일 파일(`index.ts`) 서버 코드**가 탄생하게 되었습니다.

특히 현업 실무에서 이 맞춤형 MCP 서버가 진정한 빛을 발하는 순간은 바로 <b>'압도적인 보안(Privacy)'</b>과 <b>'동적 컨텍스트(Dynamic Context)'</b>가 결합할 때입니다. 기존의 방식에서는 사내망 깊숙한 곳에 갇혀 있는 민감한 고객 로그 데이터나 비공개 API 명세서를 AI와 결합하기 위해 보안 팀의 눈치를 보며 일일이 비식별화 작업을 거쳐야만 했습니다. 

하지만 철저히 로컬 환경에서 구동되는 나만의 커스텀 MCP 서버를 띄워두면 상황은 완전히 달라집니다. 여러분이 다루는 민감한 데이터는 절대 외부 클라우드 서버로 전송되지 않으며, 오직 **로컬 물리 메모리 안에서만 안전하게 처리**됩니다. Claude Desktop 같은 클라이언트는 단지 로컬 호스트(localhost)에 질의(Query)를 던지고 결과값만 안전하게 받아갈 뿐입니다. 

실무에 바로 적용하실 때 핵심 팁은 **변수 제어법**입니다. 프롬프트 내의 `[목적]` 변수를 입력할 때는 모호하게 적지 말고 구체적인 도구의 형태를 명시해야 합니다. 예를 들어, `[특정 데이터베이스의 오늘 가입자 수를 조회하는 기능]`이라고 적기보다는, `[PostgreSQL 데이터베이스의 users 테이블에서 date 컬럼을 기준으로 오늘 가입한 유저 수를 SELECT 하는 쿼리를 실행하는 도구]`라고 명확히 제어하는 것이 좋습니다. 도구가 받을 파라미터와 반환해야 할 데이터 형식을 명확히 할수록, AI는 `zod` 스키마를 더 정교하게 짜냅니다.

저는 실제로 이 프롬프트를 응용하여 사내 Jira 이슈 트래커 및 내부 슬랙(Slack) 알림과 실시간으로 연동되는 강력한 MCP 서버를 직접 구축해 사용 중입니다. 아침에 출근하자마자 <i>"지금 내게 할당된 이번 스프린트 티켓 중 진행률이 50% 미만인 것들만 추려서 요약해 줘"</i>라고 치면, AI가 실시간으로 Jira API를 찔러 최신 데이터를 가져온 뒤 컨텍스트를 구성하여 즉석에서 완벽한 브리핑을 생성합니다. 

여러분도 처음에는 특정 폴더의 텍스트 파일을 읽어오는 단순한 '읽기 전용(Read-only)' 도구로 가볍고 안전하게 시작해 보세요. 서버의 골격이 눈에 익고 데이터가 오고 가는 작동 원리가 체화되면, 점차 데이터베이스에 직접 쿼리를 날리고 사내 메신저에 자동 알림을 보내는 등, 나만의 로컬 인프라를 자유자재로 지휘하는 진정한 생산성의 신세계를 경험하게 될 것입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 코딩 지식이 전혀 없어도 이 프롬프트를 사용할 수 있나요?**
  - A: 네, 물론입니다! `[목적]` 란에 "사내 위키 문서 검색 기능"처럼 자연어만 적어주셔도, AI가 구조적으로 완벽한 코드를 대신 작성해 줍니다. 여러분은 생성된 코드를 복사해 실행 환경에 붙여넣고 스크립트를 구동하기만 하면 됩니다.

- **Q: 생성된 서버 코드는 어떻게 실행하고 LLM 클라이언트에 연결하나요?**
  - A: 코드를 `index.ts` 파일로 저장한 뒤, Claude Desktop의 설정 파일(`claude_desktop_config.json`) 내 `mcpServers` 항목에 해당 스크립트의 실행 명령어(예: `npx tsx index.ts`)와 절대 경로를 추가해 주시면 즉각적으로 연동되어 도구 탭에 나타납니다.

- **Q: 이 프롬프트로 만든 서버는 Claude에서만 작동하나요?**
  - A: 아닙니다. Model Context Protocol은 특정 모델에 종속되지 않는 범용적인 <b>개방형 표준(Open Standard)</b>입니다. 따라서 Ollama, Cursor, Zed 등 MCP 규격을 지원하는 모든 클라이언트 환경에서 수정 없이 완벽하게 호환됩니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **제약조건(Constraints)의 강력한 통제:** `@modelcontextprotocol/sdk`와 `zod` 라이브러리의 사용을 강제하고, 필수 스키마(`ListToolsRequestSchema`, `CallToolRequestSchema`)를 엄격하게 명시했습니다. 이를 통해 AI가 스펙에 어긋나는 유령 코드를 지어내는 할루시네이션(환각) 현상을 원천적으로 차단합니다.
2. **단일 파일 및 코드 전용 출력 통제:** 여러 파일로 파편화된 복잡한 아키텍처 대신 단일 파일(`index.ts`) 구조를 지시하고, 불필요한 부연 설명을 완전히 제거하도록 명령했습니다. 그 결과 코드를 복사하고 테스트하는 데 걸리는 시간(Time-to-Value)이 비약적으로 단축됩니다.
3. **런타임 예외 처리 강제화:** 로컬 환경에서 도구 실행 도중 예상치 못한 에러가 발생하더라도 MCP 서버 프로세스 전체가 다운(Crash)되지 않도록, `try/catch` 방어 로직을 사전에 철저하게 규정했습니다.
