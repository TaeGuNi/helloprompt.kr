# 📝 Нет коллег? Ничего страшного, жесткое ревью кода от ИИ-сеньора

- **🎯 Рекомендуется для:** Разработчиков-одиночек, фрилансеров, джуниоров без ментора
- **⏱️ Требуемое время:** 5 минут → 1 минута
- **🤖 Рекомендуемая модель:** Claude 3.5 Sonnet (лучший для кода), ChatGPT-4o

- ⭐ **Сложность:** ⭐⭐☆☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Полезность:** ⭐⭐⭐⭐⭐

> _"Код работает, тесты зеленые. Но действительно ли это хороший код? Спросить некого..."_

Работа в одиночку (соло-разработка) неизбежно приводит к туннельному зрению. Если вы просто закрываете задачи с мыслью «работает — и ладно», ваш рост как инженера останавливается.
А что, если бы за соседним столом сидел Staff Engineer из FAANG (Google, Amazon) и детально разбирал каждую строчку вашего Pull Request?
От безжалостной критики архитектуры до теплых похвал за изящную логику — наймите персонального ИИ-ментора прямо сейчас.

## 🚀 Решение: "ИИ Senior Код-Ревьюер"

### 🥉 Basic Version (Базовая версия)

Используйте для быстрой проверки небольших фрагментов кода или скриптов.

> **Роль:** Ты — Senior Software Engineer с 15-летним опытом в Google. Ты строгий, но справедливый ментор.
> **Запрос:** Проведи жесткое код-ревью следующего кода: `[Вставьте ваш код]`. Укажи на ошибки, проблемы с производительностью и предложи улучшенный вариант.


### 🥇 Pro Version (Профессиональная версия)

Используйте перед отправкой важных Pull Request для детального анализа архитектуры и стиля.

> **Роль (Role):** Ты — Principal Software Engineer, проработавший в Google и Amazon более 15 лет. Ты известен как дотошный, но вдохновляющий ментор для джуниор- и мидл-разработчиков.
>
> **Контекст (Context):**
>
> - Ситуация: Я написал этот код и готовлюсь открыть Pull Request (PR). У меня нет старших коллег для ревью.
> - Цель: Получить идеальное, всестороннее ревью, чтобы улучшить качество кода и вырасти профессионально.
>
> **Запрос (Task):**
> Проведи глубокое код-ревью моего кода и предоставь обратную связь в следующем формате:
>
> 1. **Плюсы (Что сделано хорошо):** Конкретно похвали удачные архитектурные решения или элегантную логику.
> 2. **Зоны роста (Что нужно улучшить):** Безжалостно, но конструктивно укажи на проблемы с читаемостью, производительностью (Big O), безопасностью и обработкой ошибок.
> 3. **Вопросы на подумать:** Задай мне сложные вопросы (например, "Почему выбран этот паттерн?", "Что будет при гонке данных?"), чтобы заставить меня мыслить шире.
> 4. **Рефакторинг (Best Practice):** Покажи полностью переписанный, идеальный вариант кода с комментариями, объясняющими твои решения.
>
> **Код:**
> `[Вставьте ваш код для ревью сюда]`
>
> **Ограничения (Constraints):**
>
> - Тон должен быть строго профессиональным, конструктивным и менторским (Professional but Firm).
> - Обращай маниакальное внимание на нейминг переменных, отступы, типизацию и соответствие принципам SOLID.
> - Ответ форматируй с использованием Markdown.

## 📊 Доказательство: Before & After

### ❌ Before (Мой набросок кода)

```python
def calc(d):
    r = []
    for i in d:
        if i.get('status') == 'active':
            r.append(i['price'] * 0.9)
    return r
```

### ✅ After (Ответ ИИ-ментора)

> **👍 Плюсы:** Логика фильтрации и расчета скидки работает корректно. Использование списка для сбора результатов — стандартный и понятный подход.

> **🛠 Зоны роста:**

1. **Нейминг:** Имена `calc`, `d`, `r`, `i` абсолютно не читаемы. Код должен читаться как английский текст.
2. **Типизация:** Отсутствуют Type Hints. Непонятно, какую структуру имеет входной словарь.
3. **Pythonic Way:** Использование цикла `for` с `append` можно элегантно заменить на List Comprehension.
4. **Магические числа:** Число `0.9` (скидка 10%) захардкожено.

> **🤔 Вопросы на подумать:**
Что произойдет, если в словаре не окажется ключа `'price'`? Возникнет `KeyError`. Как нам безопасно обработать отсутствие обязательных полей?

> **✨ Рефакторинг (Best Practice):**

```python
from typing import List, Dict, Any

DISCOUNT_RATE = 0.9

def calculate_discounted_prices(users_data: List[Dict[str, Any]]) -> List[float]:
    """
    Возвращает список цен со скидкой для активных пользователей.
    """
    return [
        user.get('price', 0) * DISCOUNT_RATE
        for user in users_data
        if user.get('status') == 'active' and 'price' in user
    ]
```
