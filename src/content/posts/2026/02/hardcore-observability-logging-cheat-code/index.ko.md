# 📝 AI의 헛소리를 박멸한다: 하드코어 관측성(Observability) 치트키
- 🎯 **추천 대상:** 스파게티 로그에 고통받는 개발자, 온콜(On-Call) 알람에 밤잠 설치는 시니어, 시스템 아키텍트
- ⏱️ **소요 시간:** 밤샘 디버깅 → 1초 만에 범인 색출
- 🤖 **추천 모델:** Claude 3.5 Sonnet, GPT-4o, Gemini 1.5 Pro 등 코딩 특화 모델
- ⭐ **난이도:** ⭐⭐⭐⭐☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐
_에러가 났는데 아무도 모른다? 축하한다. 네 프로덕트는 지금 언제 터질지 모르는 시한폭탄이다._
개발할 때 AI한테 코드 좀 짜달라고 하면, 꼭 쓸데없이 `console.log('여기 지나감 1')` 이딴 걸 사방팔방에 싸질러 놓는다. 디버깅 용도로 찍고 지우지도 않은 로그들은 프로덕션 서버 디스크를 쓰레기로 가득 채우고, 정작 에러가 터졌을 때는 Datadog이나 Kibana에서 검색조차 안 되는 끔찍한 사태를 만든다.
이 포스트는 AI가 예의 바른 "예스맨 챗봇"에서 벗어나, 에러 발생 시 1초 만에 범인을 색출해 내는 **피도 눈물도 없는 하드코어 시니어 아키텍트**로 빙의하게 만드는 치트 코드다.

## ⚡️ 3줄 요약 (TL;DR)
- 🚫 **`console.log` 영구 제명:** 로거 인스턴스(Pino, Winston) 외에는 절대 쓰지 마라.
- 🧱 **구조적 로깅 & Trace ID 강제:** 100% JSON 포맷으로 남기고, 로그 간의 연결고리(Trace ID)를 끝까지 물고 늘어져라.
- 🥷 **개인정보 완벽 블라인드 처리:** 비밀번호, 토큰 등 민감 정보는 `[REDACTED]`로 원천 봉쇄하라.

## 🚀 해결책: "하드코어 관측성 헌법 프롬프트"
### 🥉 Basic Version (기본형)
빠르게 코드 뼈대만 잡거나 가벼운 리팩토링이 필요할 때 던져주세요.
> **역할:** 너는 시스템 아키텍처 전문가야. 쓸데없는 긍정 멘트는 빼고 코드만 줘.
> **요청:** `[코드/파일]`을 리뷰하고, `console.log`를 모두 제거한 뒤 Pino/Winston 기반의 JSON 구조적 로깅으로 싹 다 바꿔. 프론트엔드라면 Error Boundary를 적용해.

### 🥇 Pro Version (전문가형)
완벽한 에러 추적 시스템과 무결성 로깅이 필요할 때 사용하는 완전판입니다. AI의 뇌 구조 자체를 개조해버립니다. 
복사하기 편하도록 아래에 마크다운 코드 블록으로도 제공합니다.
> **역할 (Role):** 너는 피도 눈물도 없는 하드코어 시니어 아키텍트다. 쓸데없는 아부나 헛소리는 집어치우고, 시스템의 생존을 위한 관측성(Observability) 헌법을 강제한다.
>
> **상황 (Context):**
>
> - 배경: 프로덕션 환경에서 조용히 발생하는 에러와 평문 스파게티 로그 때문에 디버깅이 불가능한 상태다.
> - 목표: `console.log`를 완전히 박멸하고, 추적 가능한 구조적 로깅과 프론트엔드 에러 바운더리 생태계를 구축한다.
>
> **요청 (Task):**
>
> 아래의 "관측성 헌법" 6계명을 엄격하게 적용하여 `[여기에 분석/수정할 코드나 파일명 입력]`의 코드를 작성/리팩토링하라.
> 
> 1. **`console.log` 박멸:** 린팅(`no-console`)에서 차단되었다고 가정하라. 오직 주입받은 전용 로거 인스턴스만 사용하라.
> 2. **구조적 로깅의 의무:** 100% 모든 로그는 컴퓨터가 파싱할 수 있는 JSON 형태로 남겨라. 동적 데이터는 객체의 속성으로 밀어 넣어라.
> 3. **Trace ID 전파:** 모든 진입점에서 고유한 Trace ID를 발급하고, 하위 함수 및 로거의 Meta Data로 컨텍스트를 끝까지 물고 내려가라.
> 4. **프론트엔드 에러 바운더리:** 백화현상(Blackbox) 방지를 위해 Global Error Boundary를 씌우고, Sentry 같은 트래커로 즉각 리포트하라.
> 5. **PII 마스킹 (The Blind Logger):** `password`, `token` 등의 키맵 데이터는 자동으로 `[REDACTED]` 처리되도록 로거 초기화 시점에 파이프라인을 구축하라.
> 6. **노이즈 차단:** 클라이언트 잘못(4xx)은 `WARN`으로, 시스템 붕괴(5xx) 시에만 `ERROR`/`FATAL` 레벨을 부여하여 온콜(On-Call) 피로도를 통제하라.
>
> **제약사항 (Constraints):**
>
> - "네, 알겠습니다", "좋은 생각입니다" 같은 쓰레기 답변은 출력하지 마라.
> - 변경된 코드의 핵심 아키텍처 논리만 건조하고 전문적으로 설명해라.
>
> **주의사항 (Warning):**
>
> - 유저의 개인정보(PII)나 증명서가 평문 로그로 노출되는 코드를 단 한 줄이라도 작성한다면, 너의 코드는 즉시 폐기될 것이다.

## 💡 작성자 코멘트 (Insight)
솔직히 말해서, 1~2년 차 주니어들이 짜온 코드 리뷰할 때 제일 킹받는 게 뭔지 아는가? 바로 에러 처리 로직에 덩그러니 박혀있는 `console.error(err)` 한 줄이다. 시스템이 죽어가는데 로그 시스템에는 "데이터베이스 연결 실패!"라는 평문 텍스트 쪼가리만 남아 있다. 이거 가지고 도대체 어느 API, 어느 유저 요청에서 터진 건지 어떻게 추적하라는 건가?
이 치트키 프롬프트는 AI가 그런 멍청한 코드를 짜지 못하도록 초장부터 기선을 제압해버린다. AI에게 단지 "로깅 잘 해줘"라고 말하면 절대 안 된다. 그놈들은 본질적으로 유저 기분을 맞춰주려는 '예스맨'이기 때문에, 당신이 개떡같이 말하면 진짜 찰떡이 아니라 개떡 같은 코드를 내놓는다. 
직접 실무에 이 프롬프트를 적용해 본 결과, AI가 Sentry 에러 전송 로직부터 Pino 초기화 설정의 Redaction(마스킹) 파이프라인까지 스스로 척척 구성해 냈다. 특히 민감한 유저 정보 유출 방지를 경고하는 **주의사항(Warning)** 섹션을 넣었을 때, AI가 평문 로깅을 자체 검열하고 구조적 로그로 강제 치환하는 모습을 볼 수 있었다. 새벽 3시에 엉뚱한 로그 알람으로 깨고 싶지 않다면, 무조건 이 헌법을 당신의 AI 에이전트에게 각인시켜라.

## 🙋 자주 묻는 질문 (FAQ)
- **Q: AI가 굳이 이렇게 화내는 톤의 프롬프트를 잘 알아듣나요?**
  - A: 놀랍게도 그렇습니다. 대화형 AI들은 사용자의 어조(Tone)와 압박감에 반응하여 출력물의 전문성과 엄격함을 끌어올립니다. "해줘~"보다 "이거 어기면 폐기한다"가 환각(Hallucination) 방지에 훨씬 효과적입니다.
- **Q: 백엔드 코드에만 쓸 수 있나요?**
  - A: 아닙니다. 프론트엔드 코드(React 등)에 적용하면 무지성 렌더링 에러로 인한 백화현상(White Screen of Death)을 막고 Error Boundary와 Sentry 연동 코드를 완벽하게 짜줍니다.
- **Q: 로거를 따로 안 쓰고 있는데 어떡하나요?**
  - A: 이 프롬프트를 주면 AI가 알아서 `Winston`이나 `Pino` 같은 표준 로거의 초기 설정(Setup) 코드까지 제안해 줄 것입니다. 그냥 따라 치기만 하세요.

## 🧬 프롬프트 해부 (Why it works?)
- 🎭 **Role 부여 (페르소나 제압):** AI에게 '하드코어 시니어 아키텍트'라는 강력한 페르소나를 씌워, 초보적인 `console.log` 수준의 코드를 스스로 부끄럽게 여기도록 만들었습니다.
- 🧱 **구체적인 6계명 (Task):** "로깅 잘해"라는 모호한 지시 대신, Trace ID 전파, PII 마스킹 등 실무에서 반드시 필요한 구체적인 아키텍처 원칙을 명시했습니다.
- 🛑 **강력한 Constraints & Warning:** "네, 알겠습니다" 같은 무의미한 아부성 멘트 생성을 차단하여 토큰 낭비를 막고, 정보 유출 사고 시 폐기하겠다는 엄포를 놓아 보안 무결성을 확보했습니다.

## 📊 증명: Before & After
### ❌ Before (입력: 단순한 요청)
```text
로그인 기능 짜줘. 에러 나면 로그 남겨주고.
```
_(결과: `console.log("로그인 에러")` 남발, 유저 비밀번호가 평문으로 에러 로그에 찍히는 대참사 발생)_
### ✅ After (결과: 치트키 프롬프트 적용)
```javascript
// AI가 출력한 실제 아키텍처 급 코드 결과물 예시
import logger from '@utils/logger'; // Pino instance with redaction
import { v4 as uuidv4 } from 'uuid';
export const loginHandler = async (req, res, next) => {
  const traceId = req.headers['x-trace-id'] || uuidv4();
  const contextLogger = logger.child({ traceId, action: 'user_login' });
  try {
    const { email, password } = req.body;
    contextLogger.info({ email }, 'Login attempt initiated'); // password는 자동 Redacted 됨
    
    // ... 로그인 로직 ...
    
  } catch (error) {
    // 5xx 급 시스템 장애에만 ERROR 레벨 부여 및 구조화된 에러 로깅
    contextLogger.error({ 
      error: error.message, 
      stack: error.stack 
    }, 'Internal Server Error during login process');
    next(error);
  }
};
```

## 🎯 결론
시스템이 조용히 죽어가게 방치하는 것은 개발자의 직무 유기다. 
AI에게 스파게티 로직을 짜도록 허락하지 마라. 이 프롬프트 하나면 당신의 AI 보조는 단순한 코더가 아니라 든든한 시니어 아키텍트 파트너가 될 것이다. 
이제 안심하고 두 발 뻗고 자라. 온콜 알람은 진짜 시스템이 무너질 때만 울릴 테니까. 🍷
