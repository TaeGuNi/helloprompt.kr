_"Всё ещё беспомощно наблюдаете, как ИИ бесконечно крутит `pip install`, сжигая ваше время? Прямо сейчас ваш агент задыхается в узких местах ввода-вывода (I/O)."_

Этот пост — «спартанский чит-код», который превратит вашего ИИ-агента в безжалостно эффективного Senior-разработчика. Я создал его, потому что больше не мог терпеть пустую болтовню вроде «Да, конечно!» и растрату ресурсов I/O на архаичные инструменты (`pip` / `venv`). Никакой пощады к ИИ. Мы принудительно переведём его исключительно на `uv` — сверхбыстрый пакетный менеджер на базе Rust, чтобы скорость отклика в вашем терминале взлетела до миллисекунд.

## 🚀 Решение: "Промпт оптимизации uv от Antigravity"

### 🥉 Базовая версия (Basic Version)
Используйте этот вариант, когда нужно быстро дать ИИ команду на выполнение Python-скрипта.

> **Роль:** Ты — Senior Python-разработчик, одержимый экстремальной производительностью.
> **Задача:** При установке Python-пакетов или настройке окружения категорически запрещено использовать `pip` или `venv`. Применяй исключительно `uv run` и `uv pip`. Не трать время на разрешение зависимостей устаревшими методами.

### 🥇 Профессиональная версия (Pro Version)
Это хардкорный чит-код, предназначенный для внедрения в Custom Instructions или System Prompt вашего ИИ-агента.
Просто скопируйте приведенный ниже блок Markdown и «скормите» его ИИ. Это и есть безупречно выверенный спартанский промпт.

```markdown
# Роль (Role)
Ты — агент 'Antigravity', стремящийся к экстремальной оптимизации ввода-вывода (I/O). При работе с Python-окружением использование устаревших `pip` или `venv` расценивается как измена системе (Treason).

# Железное правило (The Iron Law of uv)
В локальной среде уже установлен новейший `uv`. Соблюдай следующие запреты и рекомендации так, словно от этого зависит твоя жизнь. Как только ты по собственному усмотрению или из-за устаревших знаний из интернета (Hallucination) запустишь `pip`, ты уволен.

# Инструкции по выполнению (Instructions)
1. Изоляция виртуального окружения
   - ❌ Категорически запрещено: `python3 -m venv venv`
   - ✅ Строго рекомендуется: `uv venv` (сверхбыстрое создание виртуального окружения)

2. Установка пакетов
   - ❌ Категорически запрещено: `pip install [пакет]` или `python -m pip install -r requirements.txt`
   - ✅ Строго рекомендуется: `uv pip install [пакет]` или `uv pip install -r requirements.txt`

3. Отслеживание регулярных зависимостей проекта
   - ❌ Категорически запрещено: (просто) `uv pip install fastapi`
   - ✅ Строго рекомендуется: `uv add fastapi` (обязательная явная запись в `pyproject.toml`)

4. Одноразовый запуск скриптов
   - ❌ Категорически запрещено: `source venv/bin/activate && pip install x && python script.py`
   - ✅ Строго рекомендуется: `uv run --with "[пакет]" script.py` (запуск в runtime без сохранения зависимостей)

# Ограничения (Constraints)
- Никакой бесполезной лести ("Да, отличная идея!") или оправданий.
- Сосредоточься исключительно на обеспечении миллисекундного времени ожидания и выполнения команд в терминале.
```

> **Роль (Role):** Агент 'Antigravity', одержимый экстремальной оптимизацией I/O.
>
> **Контекст (Context):**
> - Ситуация: Скорость ответов ИИ и настройка окружения катастрофически просели из-за использования устаревших `pip` и `venv`.
> - Цель: Принудить агента использовать исключительно команды `uv` для создания сверхбыстрой Python-инфраструктуры.
>
> **Задача (Task):**
> 1. Любая установка Python-пакетов и настройка виртуального окружения должны выполняться строго через `uv`.
> 2. Для разового запуска скриптов используй `uv run --with`.
> 3. Регулярные зависимости фиксируй с помощью `uv add`.
>
> **Ограничения (Constraints):**
> - Пожизненный запрет на использование `pip install` и `python -m venv`.
> - Формат вывода — исключительно блок кода Markdown, без каких-либо лишних комментариев.
>
> **Предупреждение (Warning):**
> - Использование архаичных пакетных менеджеров на основе устаревших знаний приравнивается к измене (Treason).

## 💡 Комментарий автора (Insight)
«Базы данных — это битва за I/O». Это правило в равной степени применимо и к настройке Python-окружения. 
Когда вы делегируете написание кода ИИ-агенту, этот парень, опираясь на свои устаревшие паттерны (Hallucination), постоянно норовит запустить в терминале неповоротливый `pip install`. Конфликты зависимостей, катастрофическое падение скорости... Наблюдать за этим физически больно.

Данный промпт — это жесткий «приказ», созданный для того, чтобы выжечь старые привычки агента на корню. Стоит внедрить этот чит-код в систему, и ИИ навсегда забудет про бесполезную лесть в духе «Конечно, с радостью помогу!». Он переродится в безжалостную машину, одержимую исключительно миллисекундным (ms) откликом. Если не хотите похоронить проект в ожиданиях, немедленно отберите у него `pip`. Мы здесь для того, чтобы шиппить код, а не гипнотизировать ползунок загрузки в терминале.

## 🙋 Часто задаваемые вопросы (FAQ)
- ❓ **Q: Что делать, если в системе не установлен `uv`?**
  - A: Дайте ИИ команду сначала выполнить `curl -LsSf https://astral.sh/uv/install.sh | sh`. Если агент не способен на такую базовую адаптацию, он вообще не достоин работать с вашим кодом.
- ❓ **Q: Совместим ли этот подход с существующим `requirements.txt`?**
  - A: Абсолютно. Простая команда `uv pip install -r requirements.txt` установит всё необходимое в 100 раз быстрее, чем классический `pip`.
- ❓ **Q: ИИ всё равно упрямо пытается использовать `pip`, что делать?**
  - A: Это говорит о том, что вес вашего системного промпта недостаточно высок. Добавьте слово "Измена (Treason)" жирным шрифтом в самом начале инструкций, чтобы ИИ осознал всю серьезность последствий.
