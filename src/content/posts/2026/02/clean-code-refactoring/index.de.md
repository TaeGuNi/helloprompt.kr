Spaghetti-Code ist der natürliche Feind eines jeden Entwicklers. 3. **Modernisierung:** Aktualisiert veraltete Syntax auf den allerneuesten Sprachstandard bei gleichzeitig maximaler Performance.

### 🥇 Pro Version (Der Experte)

Nutzen Sie diesen detaillierten Prompt für komplexe Module, die eine tiefgreifende, architektonische Überarbeitung erfordern.

> **Rolle (Role):** Du bist ein `[Clean Code Evangelist und Senior Software-Architekt]`.
>
> **Kontext (Context):**
>
> - Hintergrund: Mein Code erfüllt zwar funktional seinen Zweck, ist jedoch extrem schwer zu lesen (Legacy/Dirty Code) und kaum wartbar.
> - Ziel: Ein vollständiges Refactoring zu einem makellosen, hervorragend strukturierten und wartbaren Code, der Best Practices strikt befolgt.

> **Aufgabe (Task):**
>
> 1. **[Naming]**: Ändere Variablen- und Funktionsnamen so, dass ihre wahre Absicht sofort klar wird (z. B. `a` → `userAge`).

> 3. **[Modernisierung]**: Wende die modernsten Paradigmen und Syntax-Features der jeweiligen Sprache an (z. B. ES6+ für JavaScript, Python 3.10+).
> 4. **[Analyse]**: Erstelle einen klaren "Vorher/Nachher"-Vergleich und begründe deine Designentscheidungen präzise und nachvollziehbar.
>
> **Einschränkungen (Constraints):**
>
> - Die Kernfunktionalität und Business-Logik des Codes dürfen unter **keinen** Umständen verändert werden.
> - Nutze Kommentare ausschließlich dort, wo das "Warum" (nicht das "Was") zwingend erklärt werden muss. Der Code selbst hat selbsterklärend zu sein.
>
> **Zielcode:**

Als Softwareentwickler verbringen wir weitaus mehr Zeit mit dem Lesen von Code als mit dem eigentlichen Schreiben. Diese Best Practices können Sie direkt in Ihre Codebase übernehmen oder als fundierte Diskussionsgrundlage für das nächste Code-Review im Team nutzen. ## 📊 Beweis: Vorher & Nachher

### ❌ Vorher (Eingabe)

```python
def calc(x, y):
    if x > 19:
        print("Erwachsener")
        p = y * 0.9
        return p
    else:
        print("Student")
        return y
```

_(Das Problem: Der Funktionsname `calc` ist völlig nichtssagend, die Variablen `x` und `y` sind kryptisch, und sogenannte Magic Numbers wie `19` oder `0.9` verwirren jeden Leser massiv.)_

### ✅ Nachher (Ergebnis)

```python
def calculate_discounted_price(age: int, price: float) -> float:
    ADULT_AGE_THRESHOLD = 20
    DISCOUNT_RATE = 0.9

    if age >= ADULT_AGE_THRESHOLD:
        print("Sie sind ein Erwachsener.")
        return price * DISCOUNT_RATE

    print("Sie sind ein Student.")
    return price
```

**[Die Verbesserungen auf einen Blick:]**

1. **Sprechende Namen:** Aus `calc` wurde präzise `calculate_discounted_price`, `x` wurde zu `age` und `y` zu `price`.
2. **Magische Zahlen eliminiert:** Konstanten wie `ADULT_AGE_THRESHOLD` machen die zugrunde liegende Geschäftslogik sofort greifbar.
3. **Guard Clauses (Early Return):** Das völlig unnötige `else` wurde entfernt. Das reduziert die Einrückungstiefe und macht den Code deutlich geradliniger.
4. **Type Hints (Python 3):** Parameter und Rückgabewerte sind nun strikt typisiert, was die IDE-Unterstützung und Fehlererkennung massiv verbessert.
