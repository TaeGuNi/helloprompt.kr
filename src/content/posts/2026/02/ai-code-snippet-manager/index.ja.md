# 📝 バラバラのコードスニペット、AIで完璧に整理！（AI Code Snippet Manager）

- **🎯 おすすめの対象:** 1〜3年目のジュニアエンジニア、自分だけのコードWikiを作りたい方、コピペしたコードが溢れかえっている方
- **⏱️ 所要時間:** 30分 → 1分に短縮
- **🤖 おすすめのモデル:** Claude 3.5 Sonnet（コードのリファクタリングと文書化に最適）、ChatGPT (GPT-4o)

- ⭐ **難易度:** ⭐☆☆☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「確かに昔書いたコードなんだけど…なんて名前だったっけ？」検索窓を叩き続けて30分が経過しているなら、今すぐやめましょう。_

エンジニアなら誰しも、デスクトップの隅に「宝箱」のようなテキストファイルを一つは持っているはずです。Stack Overflowでやっと見つけた正規表現、絶妙に書かれたユーティリティ関数、いつか必ず使いそうなNginxの設定ファイルまで。

しかし、いざ急いで探そうとすると、どこに保存したのか、どう使うコードだったのか思い出せず、結局Google検索を最初からやり直す羽目になります。コードスニペットは「検索のしやすさ」と「使用例」が命です。適当に拾ってきた断片的なコードもAIに任せれば、標準的なコメントはもちろん、使用例やタグまで完璧に付与されたライブラリへと生まれ変わります。あなたの乱雑なメモ帳を体系的なナレッジベース（Knowledge Base）へと変貌させるプロンプトをご紹介します。

## ⚡️ 3行まとめ (TL;DR)

1. **検索可能な資産へ変換:** 出処不明のコードの塊に、明確なタイトル、タグ、メタデータを付与します。
2. **自動ドキュメント化:** パラメータと戻り値に対する標準的なコメント（JSDocなど）と、実際に動作するサンプルコードを即座に生成します。
3. **コード品質の改善:** 崩れたフォーマットや曖昧な変数名をAIが修正し、すぐに再利用可能な状態にしてくれます。

## 🚀 解決策：「AIコードスニペットマネージャー」

### 🥉 Basic Version（基本型）

素早くコードのコメントと基本構造だけを整えたい場合に使用してください。

> **役割（Role）:** あなたはシニアソフトウェアエンジニアリングメンターです。
>
> **タスク（Task）:** 以下のコードを分析してフォーマットを整え、どのような役割を果たすコードなのかを1行で要約し、コアとなるコメントを追加してください。
>
> **コード:**
> `[ここにコピーしたコードを貼り付ける]`


### 🥇 Pro Version（専門家型）

自分だけの完璧なコードWikiや、Notion、Obsidianのデータベースに入れる「完成形のMarkdownドキュメント」が必要な場合に使用してください。

> **役割（Role）:**
> あなたは、分かりやすいドキュメント化とクリーンコード（Clean Code）にこだわる、10年目のシニアテクニカルライター兼エンジニアです。
>
> **コンテキスト（Context）:**
>
> - 背景：私はインターネットや過去のプロジェクトから適当にコピーしておいたコードの断片を集め、自分だけのコードナレッジベース（Notion、Obsidian）を構築しています。
> - 目標：このコードを後で検索しやすくし、未来の自分が一瞬で理解して再利用できるように、完璧にドキュメント化されたMarkdownスニペットに変換すること。
>
> **タスク（Task）:**
>
> 提供された`[コード]`を分析し、以下の構造のMarkdownドキュメントを作成してください。
>
> 1. メタデータ：`Title`（明確な目的）、`Language`、`Tags`（検索用キーワードを3〜5個）
> 2. 説明（Description）：このコードがどのような問題を解決するのかを1〜2行で要約してください。
> 3. リファクタリングとコメント（Refactored Code）：インデントとフォーマットを整え、JSDoc（または該当言語の標準docstring）スタイルでパラメータ（`@param`）と戻り値（`@return`）に関する詳細なコメントを追加してください。曖昧な変数名があれば、可読性が高くなるように修正してください。
> 4. 使用例（Usage Example）：このコードを実際のプロジェクトでどのように呼び出して使うのかを示す、簡単な実行サンプルコードを作成してください。
>
> **制約事項（Constraints）:**
>
> - 出力形式は、全体を一つのMarkdownドキュメントとして作成してください。
> - 変数や関数名をむやみに大きく変更せず、オリジナルの意図を最大限維持してください。
>
> **注意事項（Warning）:**
>
> - コードに機密情報（IPアドレス、パスワードなど）が含まれている場合は、先に警告メッセージを出力してください。
> - `[コード]`の部分は私が継続的に変更して入力するため、最初の返答では「整理するコードを入力してください。」とだけ答えてください。

## 🙋 よくある質問 (FAQ)

- **Q: 会社のセキュリティポリシー上、外部のAIにコードを入力してはいけないのですが、どうすればいいですか？**
  - A: APIキー、DBのパスワード、社内サーバーのIPや独自のコアビジネスロジックなどの「機密情報」は、必ず削除するか、ダミーの値（`XXX`、`1234`）でマスキングしてから入力する必要があります。徹底したセキュリティが求められる場合は、ローカルで動作するLlama 3などのオープンソースモデルを活用することをおすすめします。

- **Q: コードのフォーマットや変数名が自分のスタイルと違う形に変更されたらどうすればいいですか？**
  - A: プロンプトの**制約事項（Constraints）**にご自身のコーディング規約を追加してください。（例：_「変数名は常にcamelCaseを維持し、ダブルクォーテーションの代わりにシングルクォーテーションを使用して。」_）

- **Q: すべてのプログラミング言語に対応していますか？**
  - A: はい。Python、JavaScript、Java、C++などの主要言語はもちろん、SQLクエリやBashスクリプト、さらには複雑なDockerfileやKubernetesのYAMLファイルのコメント付けにも完璧に対応します。

  return Math.floor(t / (1000 * 60 * 60 * 24));

### ✅ After（結果）

````markdown
**Title:** 2つの日付間の日数（Days）の差を計算する関数
**Language:** JavaScript
**Tags:** `#Date`, `#Utility`, `#TimeDifference`, `#JavaScript`

**Description:**
2つの日付文字列（またはDateオブジェクト）を入力として受け取り、2つの日付間の正確な日数（Days）の差を整数で返すユーティリティ関数です。

**Refactored Code:**

```javascript
/**
 * 2つの日付間の日数（Days）の差を計算します。
 *
 * @param {string|Date} targetDate - 比較対象の日付
 * @param {string|Date} baseDate - 基準となる日付
 * @returns {number} 2つの日付間の日数の差（targetDateが未来の場合は正の数を返す）
 */
function calculateDaysDifference(targetDate, baseDate) {
  const timeDifference =
    new Date(targetDate).getTime() - new Date(baseDate).getTime();
  return Math.floor(timeDifference / (1000 * 60 * 60 * 24));
}
```
````

**Usage Example:**

```javascript
const today = "2026-02-12";
const dDay = "2026-12-25";

const daysLeft = calculateDaysDifference(dDay, today);
console.log(`クリスマスまであと${daysLeft}日です！`);
// 出力: クリスマスまであと316日です！
```
