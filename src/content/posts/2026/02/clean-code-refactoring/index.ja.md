- **⏱️ 所要時間:** 30分 → 1分に短縮
- **🤖 推奨モデル:** 最新の推論型AIモデル（ChatGPT、Claude、Geminiなどすべてに対応）

- ⭐ **難易度:** ⭐⭐☆☆☆
- ⚡️ **効果性:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐☆

> _「とりあえず動くからと放置して1年。今となっては恐ろしくて触ることもできません…」_

スパゲッティのように複雑に絡み合ったコードは、あらゆる開発者にとって最大の悩みの種です。可読性が落ちればバグの特定に膨大な時間を奪われ、新機能の追加すら命がけの作業になってしまいます。このプロンプトは、まるで『Clean Code』の著者であるロバート・C・マーティンが直接あなたのコードをレビューするかのように、根本的な構造から見直し、芸術的で美しいコードへと磨き上げます。

## ⚡️ 3行要約 (TL;DR)

1. 重複コードの徹底的な排除と、適切な関数への分離 (Extract Method)
2. 意図が伝わらない不明確な変数名・関数名の改善 (Naming Convention)
3. パフォーマンスの最適化とモダンな最新構文の適用 (Modern Syntax)

## 🚀 解決策：「クリーンコード・クリーニング店」

### 🥉 Basic Version (基本形)

細かい指示を省き、即座にクリーンなコードの結果だけを得たい場合に活用してください。

> **役割:** あなたは`[クリーンコードの伝道師であるシニアアーキテクト]`です。
> **リクエスト:** 私が書いた「とりあえず動くだけの汚いコード」を、可読性が極めて高く、保守しやすい「クリーンなコード」へとリファクタリングしてください。


### 🥇 Pro Version (専門家形)

なぜそのように修正されたのかという詳細な解説と、最高品質のリファクタリング結果が必要な場合に使用する本格的なプロンプトです。

> **役割 (Role):** あなたは`[クリーンコードの伝道師であるシニアアーキテクト]`です。
>
> **状況 (Context):**
>
> - 背景: 私のコードは一応機能していますが、非常に読みにくく、いわゆる「スパゲッティコード（Dirty Code）」の状態に陥っています。
> - 目標: 今後の保守・拡張運用がスムーズに行えるよう、業界のベストプラクティスに従って徹底的なリファクタリングを実行すること。
>
> **タスク (Task):**
>
> 1. **[ネーミングの改善]**: 変数名と関数名を、誰が見ても直感的に意図が伝わる明確なものに変更してください（例：`a` -> `userAge`）。
> 2. **[構造の改善]**: 長すぎる関数は「単一責任の原則（SRP）」に従って適切に分割し、重複するロジックは共通化してまとめてください。
> 3. **[最新化 (Modernize)]**: 対象プログラミング言語の最新構文（例: ES6+、Python 3.10+など）を積極的に適用し、より簡潔で最適化されたコードにアップデートしてください。
> 4. **[Before & Afterの解説]**: 変更前後で何がどのように変わったのかを比較し、**なぜそのように変更したのか**をアーキテクチャの観点から詳細に説明してください。
>
> **制約事項 (Constraints):**
>
> - **絶対に**コードの本来の動作（ビジネスロジック）を変更してはいけません。
> - コメントは、コード自体では表現しきれない「なぜ（Why）」を説明する必須の箇所にのみ残し、コードを読めば自明な冗長なコメントはすべて削除してください。
>
> **対象コード (Input):**
>
> `[ここにリファクタリング対象のコードを貼り付けてください]`

## 📊 証明: Before & After

### ❌ Before (入力)

```python
def calc(x, y):
    if x > 19:
        print("Adult")
        p = y * 0.9
        return p
    else:
        print("Student")
        return y
```

_(問題点: 関数名 `calc` では何を計算しているのかが全く伝わりません。また、変数 `x`、`y`、`p` の意味も不明確で、`19` や `0.9` といった意図の読めないマジックナンバーが散見されます。)_

### ✅ After (プロンプト適用後)

```python
def calculate_discounted_price(age: int, base_price: float) -> float:
    """年齢に基づき割引適用後の価格を計算する"""
    ADULT_AGE_THRESHOLD = 20
    DISCOUNT_RATE = 0.9

    if age >= ADULT_AGE_THRESHOLD:
        print("You are an adult.")
        return base_price * DISCOUNT_RATE

    print("You are a student.")
    return base_price
```

**[改善ポイント]**

1. **命名規則の最適化**: 関数名を `calc` から `calculate_discounted_price` に変更し、目的が一目でわかるように改善しました。引数も `age`, `base_price` とし、データの実態を明確化しています。
2. **マジックナンバーの排除**: `19` や `0.9` のような意図不明な数値を定数（`ADULT_AGE_THRESHOLD`, `DISCOUNT_RATE`）として定義し、ビジネスロジックの意味を可視化しました。
3. **Early Returnパターンの適用**: 冗長な `else` ブロックを削除して早期リターンを行うことで、コードのネスト（インデント）を浅くし、実行フローの可読性を飛躍的に高めました。
4. **型ヒント（Type Hint）の追加**: Pythonのモダンな構文である型ヒント（`int`, `float`）を明記し、IDEのサポート向上と保守性の強化を実現しています。
