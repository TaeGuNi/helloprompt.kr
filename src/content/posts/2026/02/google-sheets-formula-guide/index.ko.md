> _"'최종_진짜_마지막.xlsx' 파일 메일에 첨부했습니다... 아, 팀장님이 먼저 수정하셨네요. 다시 취합해서 보내겠습니다."_

제발 그 지긋지긋한 파일 릴레이는 이제 멈추세요. **구글 시트(Google Sheets)**는 단순한 웹용 스프레드시트가 아닙니다. 몇 가지 핵심 함수만 제대로 다룰 줄 안다면, 단순한 문서 도구를 넘어 실시간 협업이 가능한 우리 팀만의 **강력한 미니 데이터베이스(DB)**로 탈바꿈합니다. 특히 `QUERY`, `IMPORTRANGE` 같은 구글 시트 전용 함수에 `AppScript`를 결합하면, 기존 엑셀로는 상상조차 할 수 없었던 완벽한 업무 자동화 마법을 경험할 수 있습니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **QUERY 함수:** 시트 내 방대한 데이터를 SQL 문법(`Select * Where...`)으로 정교하게 검색하고 필터링하는 최강의 무기입니다.
2. **IMPORTRANGE 함수:** 여러 구글 시트 파일에 흩어진 파편화된 데이터를 실시간으로 한곳에 모아 통합 대시보드를 구축합니다.
3. **AppScript 자동화:** 자바스크립트 기반의 코드를 활용하여 특정 조건(예: 재고 부족, 마감 임박) 발생 시 이메일이나 슬랙 알림을 자동으로 발송합니다.

## 🚀 해결책: "Sheets Formula Wizard"

### 🥉 Basic Version (기본 데이터 쿼리)

VLOOKUP이나 단순 필터 기능으로는 도저히 감당할 수 없는 복잡한 다중 조건의 데이터를 실시간으로 추출하고 싶을 때 활용하세요.

> **역할 (Role):** 너는 최고 수준의 데이터 분석 전문가이자 `[구글 시트(Google Sheets) 마스터]`야.
>
> **요청 (Task):** `[Data 시트]`의 `[A:E 범위]`에서, `[C열(부서)이 '마케팅'이고 D열(점수)이 80점 이상]`인 행만 정확히 추출하여 다른 시트에 보여주는 **QUERY 함수**를 작성해 줘. 함수 작성 후, 엑셀 초보자도 원리를 쉽게 이해할 수 있도록 작동 방식을 간단명료하게 설명해 줘.

### 🥇 Pro Version (AppScript 기반 알림 자동화)

데이터에 변화가 생길 때마다 담당자에게 즉시 알림을 보내는 등, 구글 시트를 완벽한 맞춤형 SaaS(서비스형 소프트웨어)처럼 진화시키고 싶을 때 사용하세요.

> **역할 (Role):** 너는 10년 차 업무 자동화 전문가이자 `[구글 앱스스크립트(GAS) 수석 개발자]`야.
>
> **상황 (Context):**
>
> - 배경: 현재 구글 시트를 활용해 `[재고 관리]`를 진행하고 있어.
> - 목표: `[C열(현재 재고)의 값이 10개 미만]`으로 떨어지는 즉시, 담당자(`[admin@company.com]`)에게 `"🚨 재고 부족: [상품명]"`이라는 제목의 이메일을 자동으로 발송하는 시스템을 구축하고 싶어.
>
> **요청 (Task):**
>
> 1. 시트 내 특정 값이 수정될 때 자동으로 실행되도록 `onEdit(e)` 트리거를 활용하는 **AppScript 코드**를 작성해 줘.
> 2. `MailApp.sendEmail` 클래스를 사용하여 이메일 발송 로직을 안전하고 정확하게 구현해.
> 3. `[상품명]` 데이터는 재고가 떨어진 해당 행의 `[A열]`에 위치한다고 가정해.
> 4. 작성된 코드를 스크립트 편집기에 붙여넣고 트리거 권한을 부여하는 세부 설정 방법까지 단계별로 상세히 설명해 줘.
>
> **제약사항 (Constraints):**
>
> - 불필요한 시스템 연산을 방지하기 위해, 오직 변경된 셀이 'C열'일 때만 알림 로직이 작동하도록 이벤트 객체(`e`)를 활용한 조건문을 **반드시** 추가해 줘.
> - 출력 형식은 복사하기 쉽도록 마크다운 코드 블록으로 제공해.
>
> **주의사항 (Warning):**
>
> - 현재 작동하지 않는 구형(Deprecated) API는 절대 사용하지 마. 실무에서 즉시 구동되는 확실하고 검증된 최신 코드만 제공해.

## 💡 작성자 코멘트 (Insight & How to use)

구글 시트가 엑셀을 압도적으로 능가하는 결정적 지점은 바로 **'외부 데이터와의 무한한 연결성'**입니다. `QUERY` 함수가 시트 내부의 데이터를 자유자재로 요리한다면, **`IMPORTXML`**과 **`GOOGLEFINANCE`** 함수는 외부 세계의 방대한 데이터를 실시간으로 쓸어 담습니다.

예를 들어, 빈 셀에 `=GOOGLEFINANCE("NASDAQ:AAPL", "price")` 라고 입력하는 순간 실시간 애플 주가가 연동됩니다. `IMPORTXML`을 활용하면 복잡한 파이썬(Python) 크롤링 코드를 짤 필요 없이, 경쟁사 웹사이트의 가격 정보나 환율을 실시간으로 긁어올 수 있죠. 여기에 Pro 버전 프롬프트로 생성한 AppScript 트리거를 연결하면 어떻게 될까요? "환율이 1,300원을 돌파하면 슬랙으로 즉시 알림 보내기" 같은 나만의 강력한 자동화 봇을 단 10분 만에 완성할 수 있습니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 데이터가 10만 줄이 넘어가니 시트가 너무 버벅거립니다. 해결책이 있나요?**
  - A: 구글 시트는 구조상 약 500만 셀 또는 4만 행 이상이 누적되면 처리 속도가 급격히 저하됩니다. 시트는 데이터를 시각적으로 '보여주는' 대시보드 용도이지, 거대한 정형 데이터를 쌓아두는 DW(Data Warehouse)가 아닙니다. 데이터가 수만 건을 넘어간다면 원본 데이터는 **BigQuery**나 **Airtable** 같은 전문 솔루션에 적재하고, 구글 시트는 필요한 요약 데이터만 가볍게 끌어와 보여주는 아키텍처로 전환해야 합니다.

- **Q: 기존에 쓰던 복잡한 엑셀 파일(.xlsx)을 구글 시트로 가져와도 수식이 깨지지 않나요?**
  - A: `VLOOKUP`, `INDEX`, `MATCH` 같은 범용적인 함수는 100% 완벽하게 호환됩니다. 하지만 엑셀의 핵심인 매크로(VBA)는 구글 시트 환경에서 전혀 작동하지 않습니다. 매크로는 자바스크립트 기반의 **AppScript(GAS)**로 완전히 새롭게 작성해야 합니다. 이때 Pro 프롬프트를 활용해 AI에게 VBA 코드를 GAS로 번역해 달라고 요청하면 작업 시간을 획기적으로 단축할 수 있습니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **SQL 문법의 전략적 활용 유도:** `QUERY` 함수는 데이터베이스의 SQL과 매우 유사한 문법 체계를 가집니다. 프롬프트에 "A열, B열을 가져오고 조건은..." 식으로 구어체로 풀어서 지시하면, AI가 이를 찰떡같이 분석해 `Select * Where` 구문으로 변환한 뒤 완벽하게 작동하는 정교한 수식을 뱉어냅니다.
2. **트리거 및 이벤트 객체의 명확한 통제:** AppScript 자동화에서 발생하는 오류의 90%는 '언제 실행되는가(Trigger)'와 '무엇이 변경되었는가(Event Object)'를 명확히 지정하지 않아서 발생합니다. 프롬프트 내에 `onEdit(e)` 트리거 사용과 변경된 셀(C열)에 대한 제약사항을 엄격하게 통제함으로써, 무한 루프에 빠지거나 엉뚱한 셀을 건드렸을 때 메일이 오발송되는 대참사를 원천 차단했습니다.

## 📊 증명: 통쾌한 결과물 (Before & After)

### ❌ Before (아날로그 수작업의 늪)

원본 데이터가 하나 추가될 때마다 매번 필터를 풀었다가 다시 걸고, 필요한 행만 드래그해서 복사한 뒤, 다른 보고서 시트에 붙여넣기를 끝없이 반복합니다. (휴먼 에러 발생, 심각한 시간 낭비 🐢)

### ✅ After (QUERY 함수로 구현한 자동화)

```plaintext
=QUERY(Data!A:E, "Select A, B Where C = '마케팅' and D >= 80", 1)
```

이제 이 수식 단 한 줄이면 모든 작업이 끝납니다. 원본 `Data` 시트에 새로운 신규 입사자가 추가되거나 팀원들의 평가 점수가 바뀌면, 결과 시트 역시 마우스 클릭 한 번 할 필요 없이 **실시간으로 1초 만에 자동 업데이트**됩니다. (완전 자동화 달성 🚀)

## 🎯 결론 (Epilogue)

엑셀은 여전히 개인의 업무를 돕는 훌륭하고 정밀한 '계산기'입니다. 하지만 구글 시트는 클라우드 위에서 팀원 모두를 연결해 주는 **'유기적인 자동화 시스템'**입니다.

강력한 함수 몇 개와 AI의 도움을 받아 AppScript를 입맛대로 주무를 수 있다면, 굳이 비싼 비용을 지불하며 사내 업무용 프로그램(SaaS)을 도입할 필요가 없습니다. 지금 여러분의 브라우저 탭에 켜져 있는 그 평범한 구글 시트가, 사실은 세상에서 가장 다루기 쉬운 최고의 자동화 도구입니다.

지금 당장 빈 시트를 열고 `=QUERY(` 를 입력해 보세요. 꽉 막혀 있던 여러분의 업무 흐름과 퇴근 시간이 극적으로 달라질 것입니다. 🍷
