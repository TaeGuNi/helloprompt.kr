## 🚀 Решение: "Генератор Shell-скриптов"

### 🥉 Базовая версия (Basic Version)

Используйте этот промпт, когда нужно быстро навести порядок на рабочем столе или в папке загрузок.

> **Роль:** Ты `[системный инженер Linux с 10-летним стажем]`.
> **Задача:** Напиши `[скрипт оболочки, который автоматически отсортирует файлы в папке ~/Downloads по их расширениям, создаст соответствующие вложенные папки (например, Images для jpg, Docs для pdf) и переместит туда файлы]`. Также объясни, как выдать этому скрипту права на выполнение.

### 🥇 Продвинутая версия (Pro Version)

Этот промпт раскрывает свою истинную мощь в реальной production-среде, когда нужно отслеживать логи ошибок сервера и мгновенно выстраивать систему оповещений.

> **Роль (Role):** Ты `[Senior DevOps-инженер]`, отвечающий за облачную инфраструктуру.
>
> **Контекст (Context):**
>
> - Цель: `[/var/log/nginx/access.log]`
> - Задача: `[Если за последнюю минуту зафиксировано 10 и более HTTP-ответов со статусом 500 (Internal Server Error), немедленно отправить экстренное уведомление в корпоративный канал Slack]`
>
> **Задание (Task):**
>
> 1. Составь максимально легкую и быструю команду для парсинга свежих логов, используя только встроенные утилиты Linux, такие как `tail`, `grep` и `awk`.
> 2. Реализуй логику подсчета количества ошибок с помощью условного оператора `if`.
> 3. Добавь код для отправки предупреждающего сообщения через Slack Webhook URL с использованием `curl`. (Оформи как переменную `[WEBHOOK_URL]`).
> 4. Включи команду для добавления написанного скрипта в `crontab`, чтобы он автоматически выполнялся каждую минуту.
>
> **Ограничения (Constraints):**
>
> - Используй только чистые встроенные команды Bash и базовые пакеты, без внешних зависимостей вроде Python.
> - Добавь обработку исключений: при возникновении ошибки скрипт не должен аварийно завершаться, он должен записывать лог ошибки.
>
> **Предупреждения (Warning):**
>
> - В приоритете — минимальное потребление ресурсов и оптимальные цепочки команд (Pipe), чтобы не создавать нагрузку на рабочий сервер.

## 💡 Комментарий автора (Insight)

Истинная сила скриптов оболочки кроется в символе конвейера (`|`). Как только вы поймете концепцию перенаправления вывода одной команды на вход другой (например, `cat file.txt | grep "error" | wc -l`), вы сможете создавать бесконечное множество решений, собирая их как кубики Lego. 
Когда вы просите ИИ написать скрипт, обязательно добавляйте условие: *"используй только встроенные команды (Built-in commands)"*. Так вы получите максимально эффективный код, который работает молниеносно и не требует тяжелых сред выполнения. И еще одно золотое правило: всегда привыкайте сначала использовать команду `echo`, чтобы визуально протестировать (Dry Run), какие файлы или директории будут изменены, прежде чем запускать реальный скрипт.

## 🙋 Часто задаваемые вопросы (FAQ)

- **В: При запуске скрипта появляется ошибка прав доступа (Permission Denied). Как это исправить?**
  - О: Это происходит потому, что у файла скрипта нет прав на выполнение. Введите в терминале команду `chmod +x имя_скрипта.sh`, чтобы выдать права, а затем запустите его снова с помощью `./имя_скрипта.sh`.

- **В: Можно ли использовать Shell-скрипты, написанные с помощью этого промпта, в среде Windows?**
  - О: Да, абсолютно. Если у вас Windows 10 или более новая версия, вы можете установить WSL (Windows Subsystem for Linux) или использовать эмуляторы терминала вроде Git Bash, чтобы без проблем выполнять те же самые команды Linux.

- **В: Не опасно ли запускать скрипты, в которых есть команда `rm -rf`?**
  - О: Это может быть крайне опасно. Чтобы предотвратить катастрофу, рекомендуется добавлять в самое начало скрипта параметры `set -e` (немедленная остановка при ошибке) и `set -u` (остановка при использовании необъявленной переменной).

## 🧬 Разбор промпта (Why it works?)

1. **Принудительное использование стандартных утилит (`tail`, `grep`, `awk`):** Явно потребовав от ИИ использовать ключевые инструменты конвейера Linux, мы предотвратили создание громоздких решений на Python или Node.js, добившись получения максимально быстрого и легкого Shell-скрипта.
2. **Интеграция в реальном времени (Slack Webhook):** Скрипт не просто выводит результат в терминал, но и отправляет данные в корпоративный мессенджер. Это закладывает фундамент для "боевой системы мониторинга", которую можно сразу внедрять в рабочие процессы.
3. **Автоматизация по расписанию (`crontab`):** Мы не остановились только на написании скрипта, но и сразу запросили метод его регулярного запуска. Это позволяет пользователю настроить автоматизированный конвейер, работающий 24/7 без вмешательства человека.
