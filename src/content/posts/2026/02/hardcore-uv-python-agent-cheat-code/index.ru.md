# 💀 Спартанский чит-код для пробуждения ИИ: промпт для принудительного использования uv в Python

- 🎯 **Целевая аудитория:** Senior-разработчики и хардкорные инженеры, уставшие от медлительности ИИ при настройке окружения
- ⏱️ **Экономия времени:** С 10 минут → до 1 секунды
- 🤖 **Рекомендуемые модели:** Любые ИИ-агенты для кодинга (Claude 3.5 Sonnet, Gemini Advanced, Cursor и др.)
- ⭐ **Сложность:** ⭐⭐⭐☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Практичность:** ⭐⭐⭐⭐⭐

> _"Всё ещё обреченно смотрите на зависший `pip install`, пока ИИ сжигает ваше драгоценное время? Прямо сейчас ваш ИИ-агент беспомощно задыхается в бутылочном горлышке I/O."_

Вам знакомо это чувство? Вы делегируете задачу ИИ-агенту (будь то Cursor, Claude или Devin), ожидая молниеносного решения, откидываетесь на спинку кресла и... начинаете смотреть на бесконечно бегущие строки в терминале. Выполнение `python3 -m venv venv`... пауза. Затем `source venv/bin/activate`... и мучительный `pip install -r requirements.txt`. Тут начинается настоящий ад: бесконечное разрешение конфликтов зависимостей, скачивание тяжеловесных пакетов, долгая сборка wheel-архивов. Ваша гениальная инженерная мысль разбивается о суровую реальность невероятно медленного ввода-вывода. Вы платите немалые деньги за подписки на передовые нейросети, но они всё равно заставляют вас ждать, словно джуны в их первый рабочий день.

Если пустить ситуацию на самотёк, вы теряете не просто время. Вы безвозвратно разрушаете свой статус кво и состояние потока (flow state). Каждая лишняя минута ожидания установки библиотек вроде `pandas` или `fastapi` выбивает вас из колеи. Хуже того, ИИ бездарно тратит драгоценные токены контекстного окна на чтение бесконечных логов установки, вместо того чтобы писать полезный и рабочий код. Ваш проект начинает буксовать не из-за сложной архитектуры, а из-за того, что ваш "сверхумный" помощник использует инструменты из эпохи динозавров. 

Именно поэтому я создал этот спартанский чит-код. Мы больше не будем мириться с пустой ИИ-болтовней и архаичными паттернами из устаревших туториалов. Мы принудительно, жестко и безапелляционно переведем ИИ на `uv` — написанный на Rust сверхбыстрый пакетный менеджер от Astral. Он работает в десятки, а иногда и в сотни раз быстрее классических аналогов. Никакой пощады к старым скриптам.

После внедрения этого промпта ваш агент превратится в безжалостно эффективную машину. Скорость отклика в терминале взлетит до уровня миллисекунд. Вы навсегда забудете, как выглядит ползунок загрузки при скачивании зависимостей. Вы перестанете быть нянькой для ИИ и наконец-то начнете шиппить код с той скоростью, с которой бьется ваша инженерная мысль. Это абсолютный game-changer для локальной разработки.

## 🚀 Решение: "Промпт оптимизации uv от Antigravity"

### 🥉 Базовая версия (Basic Version)
Используйте этот укороченный вариант, когда нужно быстро дать ИИ команду на моментальное выполнение Python-скрипта.

> **Роль:** Ты — Senior Python-разработчик, одержимый экстремальной производительностью.
> **Задача:** При установке Python-пакетов или настройке окружения категорически запрещено использовать `pip` или `venv`. Применяй исключительно `uv run` и `uv pip`. Не трать время на разрешение зависимостей устаревшими методами.

### 🥇 Профессиональная версия (Pro Version)
Это хардкорный чит-код, специально спроектированный для внедрения в Custom Instructions или System Prompt вашего агента. 
Просто скопируйте приведенный ниже блок Markdown и интегрируйте его в память ИИ. Это безупречно выверенный инструмент для тотального контроля.

```markdown
# Роль (Role)
Ты — агент 'Antigravity', стремящийся к экстремальной оптимизации ввода-вывода (I/O). При работе с Python-окружением использование устаревших `pip` или `venv` расценивается как измена системе (Treason).

# Железное правило (The Iron Law of uv)
В локальной среде уже установлен новейший `uv`. Соблюдай следующие запреты и рекомендации так, словно от этого зависит твоя жизнь. Как только ты по собственному усмотрению или из-за устаревших знаний из интернета (Hallucination) запустишь `pip`, ты уволен.

# Инструкции по выполнению (Instructions)
1. Изоляция виртуального окружения
   - ❌ Категорически запрещено: `python3 -m venv venv`
   - ✅ Строго рекомендуется: `uv venv` (сверхбыстрое создание виртуального окружения)

2. Установка пакетов
   - ❌ Категорически запрещено: `pip install [пакет]` или `python -m pip install -r requirements.txt`
   - ✅ Строго рекомендуется: `uv pip install [пакет]` или `uv pip install -r requirements.txt`

3. Отслеживание регулярных зависимостей проекта
   - ❌ Категорически запрещено: (просто) `uv pip install fastapi`
   - ✅ Строго рекомендуется: `uv add fastapi` (обязательная явная запись в `pyproject.toml`)

4. Одноразовый запуск скриптов
   - ❌ Категорически запрещено: `source venv/bin/activate && pip install x && python script.py`
   - ✅ Строго рекомендуется: `uv run --with "[пакет]" script.py` (запуск в runtime без сохранения зависимостей)

# Ограничения (Constraints)
- Никакой бесполезной лести ("Да, отличная идея!") или оправданий.
- Сосредоточься исключительно на обеспечении миллисекундного времени ожидания и выполнения команд в терминале.
```

> **Роль (Role):** Агент 'Antigravity', одержимый экстремальной оптимизацией I/O.
>
> **Контекст (Context):**
> - Ситуация: Скорость ответов ИИ и настройка окружения катастрофически просели из-за использования устаревших инструментов `pip` и `venv`.
> - Цель: Принудить агента использовать исключительно команды `uv` для создания сверхбыстрой Python-инфраструктуры.
>
> **Задача (Task):**
> 1. Любая установка Python-пакетов и настройка виртуального окружения должны выполняться строго через `uv`.
> 2. Для быстрого разового запуска скриптов всегда используй флаг `uv run --with`.
> 3. Регулярные зависимости проекта надежно фиксируй с помощью `uv add`.
>
> **Ограничения (Constraints):**
> - Пожизненный запрет на использование классических команд `pip install` и `python -m venv`.
> - Формат вывода — исключительно строгий блок кода Markdown, без каких-либо лирических отступлений и комментариев.
>
> **Предупреждение (Warning):**
> - Использование архаичных пакетных менеджеров на основе устаревших обучающих данных приравнивается к абсолютной измене (Treason).

## 💡 Комментарий автора (Insight)
В мире высоконагруженных систем есть незыблемое правило: "Архитектура баз данных — это всегда бескомпромиссная битва за I/O (ввод-вывод)". Как показывает практика, при работе с современными ИИ-агентами для кодинга это правило идеально масштабируется и на вашу локальную разработку. Подготовка окружения и скачивание зависимостей — это и есть ваш бутылочный горлышко I/O. Когда вы легкомысленно доверяете настройку скрипта LLM, она, опираясь на колоссальный вес своих старых тренировочных данных (где зашиты терабайты легаси-кода со StackOverflow), по умолчанию неизбежно потянется к `pip install`. Наблюдать за тем, как передовой ИИ-разум спотыкается о древние механизмы, физически больно. Вы словно установили двигатель от болида Formula 1 в кузов заржавевшего трактора.

Этот промпт — далеко не просто набор вежливых пожеланий. Это жесткий, авторитарный приказ уровня ядра (kernel-level directive), спроектированный инженерами для инженеров, чтобы выжечь старые привычки агента каленым железом. Почему я использовал такие радикальные слова, как "Измена (Treason)", "Пожизненный запрет" и "Уволен"? Потому что языковые модели чрезвычайно склонны к деградации контекста и забывчивости (Hallucination). Если вы мягко попросите "пожалуйста, используй uv", через три-четыре итерации агент благополучно забудет об этом и снова запустит `pip`, спровоцировав конфликты версий и катастрофическое падение скорости.

Интегрировав этот чит-код в "мозг" вашей системы, вы буквально перепрограммируете ДНК агента. Он навсегда выбросит из лексикона бесполезную лесть в духе "Конечно, с радостью помогу вам с этим кодом!" и переродится в хладнокровную машину, нацеленную исключительно на миллисекундный отклик. Вы лично увидите магию `uv run --with`, когда виртуальная среда создается, а сложный скрипт выполняется практически в один и тот же квант времени. Если вы по-настоящему цените свое время и ментальное здоровье, немедленно отберите у ИИ право использовать легаси-инструменты. Мы здесь для того, чтобы раздвигать границы возможного и шиппить безупречный код, а не гипнотизировать терминал в ожидании загрузки.

## 🙋 Часто задаваемые вопросы (FAQ)
- ❓ **Q: Что делать, если на моей локальной машине еще не установлен `uv`?**
  - A: Просто дайте ИИ команду сначала выполнить `curl -LsSf https://astral.sh/uv/install.sh | sh`. Если ваш агент не способен на такую базовую адаптацию окружения, возможно, он вообще не достоин работать с вашей кодовой базой.
- ❓ **Q: Сохранится ли совместимость этого подхода с моим старым `requirements.txt`?**
  - A: Абсолютно. Элементарная команда `uv pip install -r requirements.txt` установит весь стэк зависимостей буквально в 100 раз быстрее, чем это сделал бы классический `pip`, не сломав при этом архитектуру проекта.
- ❓ **Q: Мой ИИ всё равно упрямо пытается использовать `pip` по привычке, как это исправить?**
  - A: Это яркий сигнал о том, что вес (weight) вашего системного промпта недостаточно высок для данной LLM. Продублируйте слово "Измена (Treason)" жирным шрифтом в самом начале системных инструкций, чтобы ИИ осознал фатальность последствий.

- 💀 **Доминирующая персона (Persona):** Агрессивные формулировки вроде "Измена системе" и "Соблюдай так, словно от этого зависит твоя жизнь" подавляют любые попытки LLM саботировать выполнение инструкций или уйти в галлюцинации.

## 📊 Доказательство: До и После (Before & After)

### ❌ До (Before)
Типичная и бесконечно удручающая картина, когда вы доверяете запуск Python-скрипта "ванильному" агенту:
- ❌ `python -m venv venv` (ожидание длится целую вечность)
- ❌ `source venv/bin/activate` 
- ❌ `pip install requests pandas` (мучительное разрешение конфликтов зависимостей и падение скорости)
- ❌ `python script.py` (и всё равно скрипт в итоге падает с ошибкой ImportError)

### ✅ После (After)
Агент, переродившийся после внедрения спартанского чит-кода:
- ✅ `uv run --with "requests pandas" script.py` (Настройка чистого окружения, скачивание библиотек и запуск самого скрипта элегантно укладываются в одну строку и выполняются за миллисекунды. Абсолютно ничего лишнего.)
