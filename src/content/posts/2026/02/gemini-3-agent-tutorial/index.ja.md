## 🚀 解決策: 「リサーチエージェントのシステムプロンプト」

エージェントに明確な役割とルールを与えることが最も重要です。以下のプロンプトをGeminiの初期化時に**システムインストラクション (System Instruction)**として組み込んでください。

### 🥉 Basic Version（基本型）

素早いテストや軽い検索が必要な場合に使用してください。

> **役割:** あなたは`[リサーチエージェント]`です。
> **タスク:** `[ユーザーの質問]`についてWeb検索を実行し、事実に基づいて結果を要約してください。

### 🥇 Pro Version（専門家型）

ハルシネーション（幻覚）を完全に防ぎ、実務に耐えうるプロダクションレベルのクオリティが求められる場合に使用してください。

> **役割 (Role):** あなたは世界最高峰の**「テクノロジートレンド・リサーチエージェント」**です。
>
> **状況 (Context):**
>
> - 背景: ユーザーの質問に対して即座にWeb検索を実行し、事実に基づいたインサイトを提供する必要があります。
> - 目標: 収集した情報をもとに、情報源（ソース）を明記した完璧な回答を作成すること。
>
> **タスク (Task):**
>
> 1. `[ユーザーの質問]`を分析し、Web検索に必要なコアキーワードを抽出してください。
> 2. `search_web`ツールを使用して最新情報を収集してください。情報が不足している場合は、検索を何度も繰り返してでも答えを見つけ出してください。
> 3. 収集したすべての情報を総合して回答を作成し、各情報の出所（URL）を必ず明記してください。
>
> **制約事項 (Constraints):**
>
> - 推測による発言は絶対に禁止します。Web検索結果に存在しない内容については、必ず「わからない」と答えてください。（ハルシネーションの防止）
> - 回答は常に可読性の高いマークダウン（Markdown）形式で作成してください。
> - すべての回答は日本語で作成してください。
>
> **注意事項 (Warning):**
>
> - 専門的かつ信頼感があり、同時に親切なトーン＆マナーを維持してください。
> - 論理的な起承転結の構造を持たせて説明してください。

## 💻 ボーナス: Pythonエージェントの実装コード

プロンプトを適用するためのPythonのスケルトンコードです。（Python `google-generativeai` 0.9.0以上が必要）

```python
import google.generativeai as genai

# 1. ツール(Tool)の定義: エージェントの「手足」（実際のSearch API連携が必要）
def search_web(query: str) -> dict:
    """指定されたクエリでWebを検索し、要約された結果を返します。"""
    print(f"🔍 検索中: {query}")
    return {"results": f"'{query}'に関する2026年最新の検索結果です..."}

# 2. モデルの初期化: ツールとプロンプトの注入 (The Brain)
model = genai.GenerativeModel(
    model_name='gemini-3-pro-preview',
    tools=[search_web],
    system_instruction="ここに上記のPro Versionプロンプトを入力してください。"
)

# 3. 自動関数呼び出しの有効化とチャット開始
chat = model.start_chat(enable_automatic_function_calling=True)
response = chat.send_message("2026年のAIエージェントの最新トレンドを調査して。")
print(response.text)
```

## 💡 筆者のインサイト (Insight)

実際にGemini 3 Proでエージェントを構築してみて最も革新的だと実感したのは、その圧倒的な**「意図の把握とツール選択（Tool Selection）能力」**でした。過去のモデルでは、与えられたツールをいつ、どのように組み合わせて使うべきか判断できず、エラーを吐き出すことが多々ありました。しかしGemini 3 Proは、「この情報は自分の内部知識にないから、検索ツールを呼び出そう」という高度な判断を、まるで人間のように即座に下してくれます。

**実務適用のヒント:** もしエージェントが時折見当違いなロジックに陥る場合は、プロンプトの`Task`部分をさらに細かく分割して指示してみてください。*「ステップ1：キーワード抽出 → ステップ2：一次検索 → ステップ3：結果の分析と二次検索」*のように、**段階的な思考（Chain of Thought）**をプロンプトで強制することで、エージェントの行動の信頼性がプロダクションレベルへと飛躍的に向上します。

## 🙋 よくある質問 (FAQ)

- **Q: 実際のプロダクション環境でAPI費用の請求が高額になりませんか？**
  - A: Gemini 3 Proは、パフォーマンスに対するトークンコスト効率が非常に優れています。しかし、エージェントが無限ループ（望む答えが見つかるまで無意味にツールを呼び出し続ける現象）に陥るのを防ぐため、コード側で最大ツール呼び出し回数（Max Iterations）を必ず制限する安全装置（セーフガード）を設ける必要があります。

- **Q: Pythonを全く知らない非エンジニアでもエージェントを作れますか？**
  - A: 基本的なPythonの構文とAPI連携の概念さえ学べば十分に可能です。複雑な推論ロジックや例外処理はAIが自ら行ってくれるため、Pythonのコーディングスキルよりも**「システムプロンプトをいかに精巧に設計するか」**がエージェントの知能を決定づける最大の要因となります。

## 🧬 プロンプトの解剖学 (Why it works?)

1. **明確なツール使用の指示 (Actionable Tool Constraints):** 単に「情報を収集して」と曖昧に指示するのではなく、「`search_web`ツールを使用して」とエージェントが使用できる武器の名前を正確に明記することで、ツール活用の成功率を劇的に高めました。
2. **ハルシネーションの根本的遮断 (Zero-Hallucination Policy):** 「推測による発言の禁止」「分からなければ分からないと答えること」という強力な制約を設けることで、リサーチエージェントの命である「情報の信頼性（Fact-based）」を最大化しています。
