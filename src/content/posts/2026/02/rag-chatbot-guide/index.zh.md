# 🧠 构建 RAG（检索增强生成）：用自有数据打造智能聊天机器人

- **🎯 推荐对象：** 厌倦了“一问公司规定就胡说八道”的聊天机器人的负责人、想要用 AI 检索公司内部 Wiki 的开发者
- **⏱️ 所需时间：** 30分钟（管道构建）
- **🤖 推荐模型：** GPT-4o-mini（高性价比）+ OpenAI Embeddings

- ⭐ **难度：** ⭐⭐⭐⭐☆
- ⚡️ **效果：** ⭐⭐⭐⭐⭐
- 🚀 **实用性：** ⭐⭐⭐⭐⭐

> _"GPT 明明很聪明，为什么连我们公司的年假规定都不知道？"_

这是理所当然的，因为它并没有学习过贵公司的规定。但如果每次都要对模型进行微调（Fine-tuning），那将是一笔无法承受的巨额成本和时间开销。

针对这个问题，最完美的标准答案就是 **RAG（Retrieval-Augmented Generation，检索增强生成）**。“检索（Retrieval）相关文档，将其添加（Augmented）到提示词中，然后让 AI 生成（Generation）回答。” 只要理解了这个直观的概念，您就能在短短 30 分钟内，构建出一个完全零幻觉（Hallucination）、专属于贵公司的完美定制化 AI 聊天机器人。

---

## ⚡️ 3行总结 (TL;DR)

1. **嵌入 (Embedding)：** 将文本文档按语义拆分并转化为数字向量，存储到向量数据库（如 Pinecone, ChromaDB 等）中。
2. **检索 (Retrieval)：** 当用户输入问题时，从向量数据库中找出与问题语义最相似的文档。
3. **生成 (Generation)：** 将找出的文档作为上下文（Context）插入到提示词中，并指示 AI：“请基于这些内容进行回答”。

## 💡 作者点评 (Insight)

RAG 管道最终的回答质量，有八成取决于**“文本块（Chunk）大小”**和**“嵌入模型（Embedding Model）的性能”**。

如果把文档拆得太碎，核心上下文（Context）就会断裂，导致 AI 胡言乱语；反之，如果拆得太大，不仅严重浪费 Token，还会降低精准回答的准确度。在实际业务中，使用 `RecursiveCharacterTextSplitter` 时，将 `chunk_size` 设置为 500~1000，`chunk_overlap` 设置为 100~200，是目前普遍认可的最佳优化起点。

此外，如果面对的是包含大量表格或图片的复杂内部文档，仅靠纯文本拆分是不够的，您必须结合 Markdown 解析或类似 LlamaParse 这样的高级文档结构化工具。

## 🙋 常见问题 (FAQ)

- **Q: 中文文档的检索效果好吗？**
  - A: 非常好。OpenAI 最新的 `text-embedding-3-small` 或 `text-embedding-3-large` 模型在多语言（包括中文）嵌入性能上占据绝对优势。如果您追求在特定国内垂直领域的极致性能，也强烈建议您考虑使用 智谱AI (Zhipu AI) 或 零一万物 (01.AI) 等国内领先的嵌入 API 模型。

- **Q: 把公司的机密文档发送给 OpenAI API 安全吗？**
  - A: 根据 OpenAI 的政策，使用其付费 API 时，用户的输入数据绝对不会被用于模型训练。但如果贵公司的安全规定极其严格，您就需要结合能在本地运行的开源大模型（如 Llama 3, Qwen2）和本地嵌入模型（如 BGE-m3），构建一个完全离线、私有化部署（On-Premise）的 RAG 系统。

- **Q: AI 总是编造文档里没有的谎言（幻觉）怎么办？**
  - A: 必须通过强有力的提示词工程（Prompt Engineering）来进行控制。在传入 `RetrievalQA` 链的提示词模板中，明确施加限制：_“你必须且只能基于提供的 [Context] 进行回答。如果在上下文中找不到相关内容，请只回答‘在文档中未找到相关内容’。”_ 这样就能抑制 99% 以上的幻觉。

## 🧬 提示词解剖 (Why it works?)

1. **明确指定具体的库和类：** 并非泛泛而谈的“帮我写代码”，而是精准指定了 `PyPDFLoader`、`RecursiveCharacterTextSplitter`、`ChromaDB` 等实际业务中常用的模块。这从根本上杜绝了 AI 生成陈旧或低效代码的可能性。
2. **强制设定 Chunk Size 和 Overlap 参数：** 提供了“拆分为 1000 字符，重叠 200 字符”这一具体数值作为数据预处理的核心，迫使 AI 立即输出经过优化的代码。
3. **内置防幻觉（Hallucination）提示词：** 在约束条件中，强制加入了控制聊天机器人致命弱点（撒谎）的系统提示词设置。这样您就能获得一份在生产环境级别也可以直接进行安全测试的代码。
