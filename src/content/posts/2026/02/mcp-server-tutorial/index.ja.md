# 📝 コンテキストのコピペはもう終わり：5分で作る初めてのMCPサーバー

- **🎯 おすすめの対象:** プロンプトに毎回DBスキーマやAPIドキュメントをコピペすることに疲れた開発者、ローカルデータの活用者
- **⏱️ 所要時間:** 5分 → 永久的な時間の節約
- **🤖 推奨モデル:** Claude Desktop、OllamaなどMCP対応モデル

- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **実用性:** ⭐⭐⭐⭐⭐

> _「Claudeを開いてDBスキーマをコピペする。10分後に新しいウィンドウを開いてまたコピペする。このうんざりする反復作業、そろそろ終わりにしませんか？」_

2026年現在、**MCP (Model Context Protocol)** はLLMとデータを接続する事実上の標準（デファクトスタンダード）として定着しました。AIモデルのための「USBポート」だとお考えください。毎回手動でコンテキストを読ませる代わりに、サーバーを一度だけ接続しておけば、モデルが自動的にあなたのファイル、データベース、内部APIをリアルタイムで「読み取って」くれます。

すでにGitHubやファイルシステムと連携する優れた汎用MCPサーバーは存在します。しかし、真の威力が発揮されるのは、**「自社独自のビジネスロジック」**をLLMに接続する時です。

本記事では、退屈なボイラープレートコードを直接書く代わりに、AIに指示してわずか5分で完璧なTypeScriptベースのカスタムMCPサーバーをスキャフォールディング（足場組み）するプロンプトをご紹介します。

1. **単純なコピペの終焉:** MCPを通じて、LLMがローカルデータ（DB、ログ、API）に直接アクセスできるようにします。
2. **プロンプトでサーバー構築:** 複雑な初期設定なしに、1つのプロンプトで完璧なMCPサーバーのコードを自動生成します。
3. **圧倒的なプライバシー保護:** データを外部に流出させることなく、ローカル環境で直接モデルのコンテキストウィンドウへストリーミングします。

## 💡 筆者のインサイト (Insight)

このプロンプトの核心は、AIに「コードを書いて」と漠然と頼むのではなく、MCPサーバーが必ず備えるべき**プロトコルの中核仕様（ListToolsおよびCallToolリクエストの処理）**を強制した点にあります。

自分でMCPサーバーを構築して使用する際に得られる最大のメリットは、まさに**「圧倒的なセキュリティ（Privacy）」**です。汎用のウェブ検索ツールを使用すると、検索クエリが外部サーバーを経由しますが、ローカルのPostgreSQLデータベースや内部APIに接続されたカスタムMCPサーバーを使用すれば、データはユーザーの管理下でのみ安全に転送されます。機密性の高い社内ドキュメントをコピーしてチャットウィンドウに貼り付ける必要は全くありません。

最初は、このプロンプトを利用してローカルのログファイルを読み込むといった、単純な「読み取り専用（Read-only）」ツールから作ってみてください。そして動作原理が理解できたら、データベースを更新する機能まで拡張し、自分だけの優秀なAIアシスタントを完成させることができます。

## 🙋 よくある質問 (FAQ)

- **Q: コーディングの知識が全くなくても、このプロンプトを使用できますか？**
  - A: はい、可能です！`[目的]`の部分に「東京の天気を教える機能」のように自然言語で記入すれば、Claudeが自動的に完璧なコードを作成してくれます。生成されたコードをコピーして実行環境に貼り付けるだけです。

- **Q: 生成されたサーバーコードはどのように実行して接続するのですか？**
  - A: コードを`index.ts`として保存した後、Claude Desktopの設定ファイル（`claude_desktop_config.json`）の`mcpServers`の項目に該当スクリプトのパスを追加すれば自動的に連携されます。

- **Q: このプロンプトで作ったサーバーはClaudeでしか使えませんか？**
  - A: いいえ。Model Context Protocolはオープンスタンダードであるため、OllamaやCursorなど、MCPをサポートするすべてのプラットフォームやクライアントで同様に動作します。

## 🧬 プロンプトの解剖学 (Why it works?)

1.  **制約事項 (Constraints) による強力な統制:** `@modelcontextprotocol/sdk`と`zod`の使用を強制し、必須スキーマ（`ListToolsRequestSchema`、`CallToolRequestSchema`）を明示することで、AIが仕様に反するハルシネーション（幻覚）を起こすのを根本から防ぎました。
2.  **単一ファイルおよびコード専用の出力:** 複数のファイルに分割された複雑な構造の代わりに、単一ファイル（`index.ts`）構造を指示し、補足説明を排除することで、ユーザーのテスト速度（Time-to-Value）を最大化しました。
3.  **例外処理の強制化:** ツールの実行中に発生する可能性のあるエラーによってMCPサーバー全体がクラッシュするのを防ぐため、`try/catch`ロジックをあらかじめ規定しました。
