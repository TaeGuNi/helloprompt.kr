## 📊 証明: Before & After

### ❌ Before (入力コード)

```python
def find_duplicate(nums):
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            if nums[i] == nums[j]:
                return nums[i]
    return None
```

### ✅ After (結果)

**分析:**

- **時間複雑度:** O(N^2)。二重`for`ループを使用しています。Nが10,000を超えても非常に遅くなります。
- **空間複雑度:** O(1)。追加のメモリをほとんど使用しません。

**改善されたコード (Hash Set 使用):**

```python
def find_duplicate_optimized(nums):
    seen = set()
    for num in nums:
        if num in seen:
            return num
        seen.add(num)
    return None
```

**改善結果:**

- **時間複雑度:** O(N)。ループを一度だけ回ります。
- **空間複雑度:** O(N)。`seen`集合にデータを保存しなければなりません。(Time-Space Tradeoff)
