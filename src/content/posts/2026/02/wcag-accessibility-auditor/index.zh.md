# ♿️ 我的代码对所有人都友好吗？Web无障碍 (WCAG) 自动审计机器人

- **🎯 推荐对象：** 公共机构/大型企业项目的开发者、希望提升作品集细节的求职者、资深前端开发者
- **⏱️ 预计耗时：** 手动检查无障碍清单 2 小时 → AI 自动诊断 1 分钟
- **🤖 推荐模型：** Claude 3.5 Sonnet（代码分析能力极强）、GPT-4o

- ⭐ **难度：** ⭐☆☆☆☆
- ⚡️ **效果：** ⭐⭐⭐⭐⭐
- 🚀 **实用度：** ⭐⭐⭐⭐☆

> _"即使实现了华丽的动画，但如果有人连登录按钮都无法点击，那么这段代码就是失败的。"_

Web 无障碍（Web Accessibility）不仅是一种“关怀”，更是服务的“基本要求”，甚至正逐渐成为一种法律义务。然而，每次编码时都去死记硬背庞大且复杂的 WCAG（Web Content Accessibility Guidelines）2.2 指南几乎是不可能的。
这个提示词（Prompt）能够扫描您编写的标记（Markup）和组件代码，在 1 分钟内找出在屏幕阅读器和键盘导航环境中可能出现的致命无障碍缺陷，并将其重构为完美的语义化代码。

---

## ⚡️ 3句话总结 (TL;DR)

1. **转换为语义化标记：** 消除滥用的 `<div>` 标签，自动转换为有意义的 HTML5 语义化标签。
2. **完美遵循 WCAG 2.2：** 严格验证对比度、焦点（Focus）移动、WAI-ARIA 的滥用等苛刻的无障碍标准。
3. **实战测试指南：** 提供仅使用屏幕阅读器（VoiceOver, NVDA）和键盘（Tab）即可进行验证的具体测试方法。

## 💡 作者见解 (Insight)

改善 Web 无障碍性的工作往往容易演变成“滥用 WAI-ARIA”。然而，错误地添加 `aria-label` 或 `role` 反而会给屏幕阅读器用户带来致命的困扰。
这个提示词的核心在于强迫 AI 遵循 **“First Rule of ARIA（优先使用原生 HTML 标签）”** 的原则。
在实际工作中，当您制作复杂的自定义选择框（Select Box）或模态框（Modal）时，只要通过这个提示词的测试，就能极其精准地捕捉到意想不到的键盘焦点丢失问题或表单标签遗漏。特别是在与 Claude 3.5 Sonnet 模型配合使用时，它能展现出压倒性的代码质量，简直就像您的身边坐着一位严谨的无障碍专家在为您进行详尽的代码审查。

## 🙋 常见问题 (FAQ)

- **Q: 可以通过界面设计 (UI) 图片来进行无障碍审计吗？**
  - A: 是的，完全可以。您可以在向 GPT-4o 或 Claude 3.5 Sonnet 提供组件截图的同时，附加一句提示：“请检查色彩对比度 (Color Contrast) 是否符合 WCAG AA 标准”。它甚至能帮您找出隐藏在视觉设计中的缺陷。

- **Q: 我正在使用 Tailwind CSS，它会自动帮我修改类名吗？**
  - A: 会的。只需在提示词的背景 (Context) 部分明确指出：“请使用 Tailwind CSS 来应用屏幕阅读器专用 (`sr-only`) 的实用类”，AI 就会输出完全契合您技术栈的代码。

---

## 🧬 提示词解剖 (Why it works?)

1. **提供明确的审查标准：** 并不是简单地说“帮我优化无障碍性”，而是将“WCAG 2.2 Level AA”这一国际标准规范注入到角色设定中，从而完美抑制 AI 的幻觉（产生虚假信息）。
2. **强制执行 First Rule of ARIA：** 防止 AI 盲目滥用 ARIA 属性，引导它最大限度地利用 HTML 原生规范来实现根本的语义化标记。
3. **可操作的 (Actionable) 测试指南：** 不仅仅是提供正确的代码，还要求提供具体的测试场景，让开发者能够亲自使用键盘进行操作和验证，从而使实际工作中的实用性最大化。

## 📊 案例证明：Before & After

### ❌ Before (输入：用 `div` 堆砌出的最糟糕的按钮)

```html
<div class="submit-btn" onclick="sendData()">
  <img src="send-icon.png" />
  <span>发送</span>
</div>
```

### ✅ After (结果：完美适配键盘焦点与屏幕阅读器)

```html
<!-- 对于无意义的装饰性图片，提供空的 alt 或使用 aria-hidden="true" 以便屏幕阅读器忽略它 -->
<!-- 使用 button 标签，不仅支持 onClick，原生也支持键盘的 Enter/Space 操作 -->
<button type="button" class="submit-btn" onClick="sendData()">
  <img src="send-icon.png" alt="" aria-hidden="true" />
  <span>发送</span>
</button>
```
