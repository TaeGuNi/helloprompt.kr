# 📝 엑셀, JSON, CSV... 데이터 포맷 변환 노가다 탈출

- **🎯 추천 대상:** 개발자, 데이터 분석가, 퍼포먼스 마케터
- **⏱️ 소요 시간:** 30분 → 1분 단축
- **🤖 추천 모델:** Claude 3.5 Sonnet (복잡한 구조 변환에 강함), ChatGPT (GPT-4o)

- ⭐ **난이도:** ⭐☆☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"대리님, 이 엑셀 데이터 웹에 올려야 하는데 JSON으로 좀 바꿔주실 수 있나요? 아, 그리고 키(Key) 값은 전부 영문 소문자 스네이크 케이스로 통일 부탁드려요."_

데이터를 다루다 보면 필연적으로 '포맷 변환'이라는 단순 노동과 마주하게 됩니다.
온라인 무료 변환 툴을 쓰자니 회사 보안 규정 때문에 찝찝하고, 파이썬(Python) 스크립트나 정규식을 짜자니 '이거 하나 하려고 코딩까지 해야 하나' 싶어 현타가 오죠.
이제 데이터를 그냥 복사해서 AI에게 던져주세요. 단순 변환은 물론, 데이터 구조 매핑, 필드명 변경, 불필요한 값 제거까지 한 번에 처리해 줍니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **보안 걱정 끝:** 로컬 환경의 AI(또는 엔터프라이즈 AI)를 활용하면 외부 툴 없이 안전하게 변환 가능
2. **복합 변환 수행:** 단순 포맷 변경(CSV → JSON)뿐만 아니라 키(Key)값 매핑과 데이터 타입(Type) 캐스팅까지 동시 처리
3. **즉시 사용 가능한 쿼리:** DB 마이그레이션을 위한 SQL Insert 문이나 파이썬 데이터프레임 코드로도 직접 생성 가능

## 🚀 해결책: "만능 데이터 컨버터"

### 🥉 Basic Version (기본형)

복잡한 조건 없이, 빠르고 단순하게 포맷만 바꿀 때 사용하세요.

> **역할:** 너는 시니어 데이터 엔지니어 역할을 수행해.
> **요청:** 아래 데이터를 `[JSON]` 포맷으로 변환해 줘. 다른 설명은 생략하고 코드블럭으로 최종 결과물만 출력해.
>
> ```text
> [여기에 변환할 데이터를 붙여넣으세요]
> ```

<br>

### 🥇 Pro Version (전문가형)

데이터 클렌징, 키값 매핑, 특정 필드 제외 등 디테일한 가공이 필요할 때 사용하세요.

> **역할 (Role):** 너는 10년 차 데이터 엔지니어이자 ETL 파이프라인 설계 전문가야.
>
> **상황 (Context):**
>
> - 배경: 제공된 데이터를 타겟 시스템(DB, API 등)의 규격에 맞게 변환하고 정제해야 해.
> - 목표: 조건에 완벽하게 부합하는 무결점 데이터를 산출하는 것.
>
> **요청 (Task):**
>
> 1. 제공된 **[소스 데이터]**를 분석하여 **[타겟 포맷]**으로 변환해.
> 2. 변환 과정에서 반드시 아래의 **[변환 규칙]**을 엄격하게 적용해.
> 3. 코드블럭 안에 변환된 최종 데이터만 깔끔하게 출력해.
>
> **입력 변수 (Variables):**
>
> - **[소스 포맷]**: `[CSV]`
> - **[타겟 포맷]**: `[JSON Array]`
> - **[변환 규칙]**:
>   1. 한글 헤더명을 영문 스네이크 케이스(snake_case)로 변경 (예: '고객 이름' -> 'customer_name')
>   2. '나이' 필드는 숫자형(Integer)으로 타입 변환
>   3. '개인정보\_주민번호' 필드는 결과물에서 완전히 제외(Drop)할 것
>
> **제약사항 (Constraints):**
>
> - 데이터를 누락하거나 임의로 값을 지어내지 마.
> - 변환 프로세스에 대한 설명이나 인사이트는 출력하지 말고, 오직 최종 결과물만 코드블럭으로 제시해.
>
> **소스 데이터:**
>
> ```text
> [여기에 원본 데이터를 붙여넣으세요]
> ```

## 💡 작성자 코멘트 (Insight)

이 프롬프트의 진정한 가치는 **'데이터 정제(Data Cleansing)'와 '포맷 변환'을 하나의 파이프라인으로 통합**했다는 점입니다.
실무에서는 단순히 CSV를 JSON으로 바꾸는 것보다, 프론트엔드 API 명세서에 맞게 키(Key) 값을 카멜 케이스(camelCase)로 수정하거나, 날짜 포맷(`YYYY-MM-DD`)을 통일하는 등 부가적인 작업이 훨씬 번거롭습니다.
Pro 버전의 `[변환 규칙]`에 정규식으로도 처리하기 까다로운 예외 상황("빈 값은 null로 처리하고, 직업이 없는 사람은 '무직'으로 통일해")을 자연어로 적어보세요. 놀라운 정확도로 데이터를 가공해 줍니다. 특히 Claude 3.5 Sonnet 모델이 복잡한 JSON 중첩 구조(Nested Object)를 다룰 때 압도적인 성능을 보여줍니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 엑셀 파일(xlsx)을 직접 첨부해도 되나요?**
  - A: Advanced Data Analysis 기능이 있는 ChatGPT(GPT-4o)나 Claude에서는 엑셀 파일을 직접 업로드해도 잘 작동합니다. 단, 회사 보안에 민감한 데이터라면 파일 첨부 대신 직접 내용의 일부만 복사해서 프롬프트에 붙여넣고 마스킹하는 방식을 권장합니다.

- **Q: 변환해야 할 데이터가 수만 줄이 넘어가면 어떻게 하나요?**
  - A: LLM의 컨텍스트 윈도우(Token 한도)를 초과하거나, 출력이 중간에 끊길 수 있습니다. 이럴 때는 프롬프트 방향을 틀어 "이 변환 규칙을 수행하는 파이썬 코드를 작성해 줘"라고 요청하세요. 스크립트를 생성한 뒤, 로컬 PC에서 실행하는 것이 가장 안전하고 확실합니다.

- **Q: 결과물이 출력되다 말고 중간에 잘려 나옵니다.**
  - A: "계속해 줘(Continue)"라고 입력하면 이어서 출력해 줍니다. 애초에 프롬프트 하단 제약사항에 "결과물이 길면 임의로 생략하지 말고 나누어서라도 끝까지 출력해 줘"라는 조건을 추가하는 것도 좋은 팁입니다.

## 🧬 프롬프트 해부 (Why it works?)

1.  **ETL 전문가 페르소나 부여:** AI에게 단순 변환기가 아닌 '데이터 엔지니어' 역할을 부여함으로써, 데이터의 무결성(Integrity)과 타입(Type)을 스스로 신경 쓰게 만듭니다.
2.  **명확한 파라미터 분리:** 입력(소스), 출력(타겟), 그리고 규칙(변환 룰)을 마크다운 문법으로 명확히 구분하여, AI가 지시사항을 혼동하지 않고 순차적으로 처리하도록 구조화했습니다.
3.  **엄격한 출력 통제 (Constraints):** "설명은 생략하고 코드블럭만 출력해"라는 제약을 통해, 사용자가 변환된 데이터를 곧바로 '복사-붙여넣기' 할 수 있도록 UX를 최적화했습니다.
