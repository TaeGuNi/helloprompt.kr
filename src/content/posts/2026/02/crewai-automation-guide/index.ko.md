## 🤖 CrewAI: AI 팀원들에게 업무 지시하는 법 (프로세스 자동화)

- **🎯 추천 대상:** "A 작업 후 B, 그리고 C"와 같은 절차적 업무가 잦은 PM, 콘텐츠 발행을 자동화하고 싶은 마케터 및 개발자
- **⏱️ 소요 시간:** 15분 (파이썬 스크립트 작성 및 테스트 기준)
- **🤖 추천 모델:** GPT-4o (또는 로컬 Ollama 모델)

- ⭐ **난이도:** ⭐⭐⭐☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"AutoGen은 너무 수다스럽고 통제가 안 되나요? 명확한 지휘 체계로 일사불란하게 움직이는 '군대식' AI 팀이 필요하다면 CrewAI가 완벽한 해답입니다."_

에이전트(Agent) 기술이 고도화되면서 여러 AI가 협업하는 시대가 열렸습니다. 하지만 자유로운 토론에 특화된 프레임워크들은 실무에 적용했을 때 엉뚱한 결론으로 빠지거나 무한 루프에 갇히는 치명적인 단점이 있습니다. 현업에서는 **'정해진 순서대로 확실하게 일을 처리하는'** 통제 가능한 파이프라인이 훨씬 유용합니다. CrewAI는 에이전트마다 명확한 역할(Role)과 목표(Goal)를 부여하고, 마치 컨베이어 벨트처럼 순차적으로 업무를 처리하게 만드는 가장 실용적이고 안정적인 프레임워크입니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **채용 (Agent):** 각기 다른 전문성과 백스토리를 가진 AI 직원(리서처, 작가 등)을 세밀하게 정의합니다.
2. **지시 (Task):** 각 직원에게 구체적인 업무와 기대하는 최종 결과물의 형태를 할당합니다.
3. **실행 (Process):** 이들을 하나의 팀(Crew)으로 묶어, 앞사람의 결과물이 뒷사람에게 오차 없이 전달되도록 순차적(Sequential)으로 실행시킵니다.

## 🚀 해결책: "CrewAI Pipeline Prompt"

### 🥉 Basic Version (뉴스레터 팀)

단순한 정보 수집 및 요약 파이프라인의 기초 구조를 잡을 때 유용합니다. 빠르게 코드를 뽑아내어 즉각적으로 테스트해 볼 수 있습니다.

> **역할 (Role):** 너는 파이썬 기반의 CrewAI 스크립트 작성 전문가야.
>
> **요청 (Task):** CrewAI를 사용하여 두 명의 에이전트로 구성된 기본 파이썬 코드를 작성해 줘.
>
> 1.  **Researcher:** '2026년 최신 AI 트렌드'를 웹에서 검색해서 핵심만 요약하는 역할.
> 2.  **Writer:** Researcher의 요약본을 바탕으로 읽기 쉽고 매력적인 블로그 글을 작성하는 역할.
>
> 두 에이전트가 순차적으로 작동하도록 `Sequential` 프로세스를 적용해 코드를 완성해 줘.

### 🥇 Pro Version (주식 분석 리포트)

외부 도구(Tool) 연동과 상세한 백스토리(Backstory)가 포함된 실무용 고급 파이프라인입니다. 에이전트의 손에 무기를 쥐여주는 본격적이고 강력한 자동화 스크립트입니다.

> **역할 (Role):** 너는 실무 적용에 능숙한 시니어 CrewAI 아키텍트야.
>
> **상황 (Context):**
>
> - 배경: 매일 아침 글로벌 증시 동향과 주요 경제 뉴스를 리서치하고 요약하는 작업이 너무 많은 리소스를 차지함.
> - 목표: 외부 API와 검색 도구를 활용하여 '매일 아침 주식 시장 분석 리포트'를 자동으로 생성하는 AI 크루(Crew) 스크립트 구축.
>
> **요청 (Task):**
> 다음 세 명의 에이전트를 포함하는 완벽한 CrewAI 파이썬 코드를 작성해 줘. 각 에이전트의 `Role`, `Goal`, `Backstory`를 매우 구체적으로 설정해야 해.
>
> 1.  **Market Analyst:** `yfinance` 도구를 사용하여 어제 미국 증시(S&P 500, Nasdaq) 주요 지표를 수집하고 분석함.
> 2.  **News Scraper:** `SerperDevTool` (구글 검색 도구)을 사용하여 주요 경제 뉴스 헤드라인 5개를 수집함.
> 3.  **Chief Editor:** 위 두 에이전트의 데이터를 종합하여 직관적인 '3분 브리핑' 리포트를 작성하고, 마크다운 형식의 파일(`[오늘 날짜]_report.md`)로 저장함.
>
> **제약사항 (Constraints):**
>
> - 실행 프로세스는 반드시 `sequential`로 설정할 것.
> - 에이전트가 사용할 도구(Tools)를 임포트하고 할당하는 부분을 명확히 주석으로 설명할 것.
> - `.env` 파일에서 API 키(OPENAI_API_KEY, SERPER_API_KEY)를 로드하는 기본 셋업 코드를 포함할 것.

## 💡 작성자 코멘트 (Insight)

CrewAI의 진정한 파괴력은 에이전트에게 **'도구(Tool)'**를 쥐여줄 때 발휘됩니다. LLM 자체의 지식에만 의존하면 필연적으로 환각(Hallucination)이 발생하지만, `FileReadTool`, `WebsiteSearchTool` 혹은 직접 만든 커스텀 파이썬 함수를 도구로 연결하면 에이전트가 현실 세계의 데이터를 직접 탐색하고 조작할 수 있게 됩니다.

**🔥 실무 팁 (Troubleshooting):**
만약 에이전트가 예상치 못한 엉뚱한 행동을 하거나 작업을 끝맺지 못한다면, 십중팔구 **`Task`의 `expected_output`(기대 결과물)**이 모호하게 설정되었기 때문입니다. "잘 요약해 줘"와 같은 추상적인 지시 대신, "3개의 불릿 포인트로 구성된 500자 이내의 마크다운 텍스트"처럼 극도로 구체적인 가이드라인을 제시해야 파이프라인이 병목 현상 없이 매끄럽게 굴러갑니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 토큰 비용이 너무 많이 나오지 않을까요?**
  - A: 리서치나 단순 요약 같은 앞단(Front)의 에이전트에게는 저렴한 GPT-4o-mini나 Claude-3.5-Haiku를 배정하고, 최종 글을 다듬는 Editor 에이전트에게만 GPT-4o를 할당하는 방식(Model Delegation)을 취하면 전체 구동 비용을 1/10 수준으로 획기적으로 줄일 수 있습니다.

- **Q: 데이터 보안 때문에 외부 API를 쓸 수 없는데 어떡하죠?**
  - A: CrewAI는 로컬 LLM 연동을 완벽하게 지원합니다. Ollama를 통해 사내 서버에 구축된 Llama 3나 Qwen 모델을 `llm=Ollama(model="llama3")` 형태로 연결해 보세요. 데이터 유출 걱정 없이 안전한 사내 전용 AI 팀을 꾸릴 수 있습니다.

- **Q: 작업 중간에 사람이 직접 개입해서 컨펌할 수는 없나요?**
  - A: 물론 가능합니다! 에이전트 설정 시 `human_input=True` 옵션을 부여하면, 해당 에이전트가 작업을 마친 직후 콘솔 창에서 사용자의 피드백을 대기합니다. 이때 "다시 써와" 또는 "진행시켜"와 같이 직접 지시를 내릴 수 있습니다.

## 🧬 프롬프트 해부 (Why it works?)

1.  **Backstory(백스토리)의 힘:** 프롬프트에서 각 에이전트에게 "너는 월스트리트에서 10년 구른 애널리스트야"와 같은 백스토리를 강력하게 부여했습니다. LLM은 이 백스토리를 바탕으로 페르소나를 극대화하며, 결과물의 톤앤매너와 분석의 깊이를 단순한 기계적 요약 수준 이상으로 끌어올립니다.
2.  **명확한 Handoff (업무 인수인계):** `Sequential` 프로세스를 명시함으로써, 첫 번째 에이전트(Analyst)가 생성한 데이터가 두 번째(Scraper)와 세 번째(Editor) 에이전트의 작업 컨텍스트로 자연스럽게 흘러가도록 설계했습니다. 이는 인간 팀원들이 메신저를 통해 파일을 주고받으며 협업하는 과정을 완벽하게 코드로 구현해 낸 것입니다.

## 📊 증명: Before & After

### ❌ Before (인간의 고군분투)

1. 야후 파이낸스에 접속하여 주요 지수를 확인하고 메모 (10분)
2. 구글에서 'US Economy News' 검색 후 관련 기사 5개 정독 (20분)
3. 메모장을 켜서 수집한 내용을 종합하고 리포트 형태로 포맷팅 (30분)
4. **결과:** 매일 아침 1시간씩 귀중한 리소스 낭비. 피곤한 날은 **"아 귀찮아 대충 쓰자."**라며 품질 저하 발생 🐢

### ✅ After (CrewAI 파이프라인 도입)

1. 터미널에서 `python run_crew.py` 엔터.
2. (여유롭게 커피 한 잔 내리고 오면)
3. `report.md` 파일이 완벽한 마크다운 양식으로 자동 생성 완료. ☕️🚀
4. **결과:** 매일 아침 단 1분 소모. 컨디션과 무관하게 항상 일정한 고품질의 리포트 유지 가능.

## 🎯 결론

실력 있는 직원을 채용하는 것은 하늘의 별 따기지만, 유능한 AI 에이전트 팀을 꾸리는 것은 스크립트 몇 줄이면 충분합니다.
블로그 발행, 시장 조사, 코드 리뷰 등 명확한 단계가 존재하는 반복 업무(Workflow)가 있다면 지금 바로 당신만의 **Crew**를 결성해 보세요.

이제 당신은 직접 땀 흘리며 일하는 실무자가 아니라, 든든한 AI 팀원들에게 업무를 지시하고 컨펌하는 **관리자(Manager)**로 거듭나야 합니다. 자동화를 통해 업무 효율을 극대화하고 쿨하게 칼퇴하시길 바랍니다! 🍷
