## ⚡️ 3行摘要 (TL;DR)

1. 从 AWS Cost Explorer 导出“按月/按服务分类的成本” CSV 数据。
2. 将数据与当前架构信息提供给 AI，指示其找出“低效的支出项目”。
3. 立即在实际业务中应用 AI 建议的竞价实例 (Spot Instance)、自动扩展 (Auto Scaling) 以及 Savings Plan 策略。

## 🚀 解决方案："Cloud Cost Cutter" 提示词

### 🥉 Basic Version (基础版：揪出僵尸资源)

当您对资金流失毫无头绪时，使用此提示词可以快速找出闲置的冗余资源。

> **角色：** 你是一位资深云架构师 (Senior Cloud Architect)。
> **请求：** 我查看了 AWS 账单，发现 `[EBS Volume]` 的费用异常偏高。请帮我找出所有未连接 (Unattached) 的卷、过期的快照以及已分配但未使用的 Elastic IP，并提供准确的 AWS CLI 删除命令以及相关注意事项。

<br>

### 🥇 Pro Version (专业版：竞价实例与架构优化)

当您需要结构性的成本削减和自动化的扩展策略时，请使用这个完美的提示词。

> **角色 (Role)：** 你是一位负责企业级基础设施的首席 AWS FinOps 顾问。
>
> **背景 (Context)：**
>
> - **当前架构：** 5 台 EC2 (On-Demand，按需实例，24小时常驻)，1 台 RDS (MySQL)。
> - **流量模式：** 流量仅集中在工作日的工作时间（09:00~18:00），夜间和周末几乎没有流量。
> - **目标：** 在维持当前工作负载稳定性的前提下，将基础设施成本至少降低 50%。
>
> **请求 (Task)：**
>
> 1. **竞价实例 (Spot Instance) 引入策略：** 在当前的工作负载中，提出一个安全利用竞价实例将成本降低 70% 以上的架构方案。（必须包含实例中断时的应对机制）
> 2. **自动扩展 (Auto Scaling) 与调度：** 提供具体的 `ASG (Auto Scaling Group)` 配置值和 `Instance Scheduler` 应用方案，以便在夜间将服务器缩减至最低限度（1台），并在周末彻底关闭开发环境。
> 3. **承诺折扣 (Savings Plan / RI)：** 请用 Markdown 表格形式，对比采用 1 年预付承诺（Compute Savings Plan）与当前按需付费模式的预计节省金额。
>
> **限制条件 (Constraints)：**
>
> - 必须基于 AWS 官方的最佳实践 (Well-Architected Framework) 提供建议。
> - RDS 数据库**绝对禁止**应用竞价实例。
>
> **注意事项 (Warning)：**
>
> - 省略模糊的概念解释，重点提供工程师明天早上就能立刻执行的、极具实操性的 Action Item。

## 💡 作者点评 (Insight)

在云端成本管理中，最常见也是最致命的错误就是**“在周末和夜间依然保持开发/测试服务器（Dev/Stage）24 小时运行”**。仅仅是引入 `Instance Scheduler` 在非工作时间自动关闭这些实例，就能立即砍掉开发服务器 60% 以上的费用。基于此提示词获得的优化思路，您可以向 AI 提出后续请求：**“请帮我编写能够应用此策略的 Terraform 代码（或 CloudFormation 模板）”**，从而一步到位实现基础设施自动化。

## 🙋 常见问题 (FAQ)

- **Q: 听说竞价实例 (Spot Instance) 会突然被强制关闭，这不是很危险吗？**
  - A: 没错。因此，竞价实例只适用于无状态 (Stateless) 的 API 服务器、基于容器的工作负载或异步批处理作业 (Batch Processing)。**绝对不能**将其用于处理核心流量的主服务器或数据库。

- **Q: RDS（数据库）的费用太高了，这部分该如何削减？**
  - A: 与其一直使用昂贵的按需实例，不如考虑引入根据流量弹性扩展的 Aurora Serverless v2，或者通过分离只读副本 (Read Replica) 来分散负载，从而优化架构。您可以要求 AI 帮您计算这种架构转换的成本以及预计的节省金额。

---

## 🧬 提示词剖析 (Why it works?)

1. **提供明确的流量模式：** 提供了“日间集中，夜间/周末闲置”的具体业务背景，引导 AI 得出基于**调度和动态扩展**的优化策略，而不是简单地建议“降低实例规格”。
2. **设定具体的量化目标（降低 50%）：** 与其含糊其辞地说“帮我降本”，不如给出具体的数字目标，迫使 AI 提出采用竞价实例或 Savings Plan 等更具突破性的决策。
3. **风险控制 (Constraints)：** 明确限制“不要对数据库使用竞价实例”，提前阻断了因盲目追求成本削减而导致服务中断的致命幻觉 (Hallucination)。
