## 📝 Chega de Fragmentos de Código Desorganizados: Domine a Arte com a IA! (AI Code Snippet Manager)

- **🎯 Recomendado para:** Desenvolvedores júnior (1 a 3 anos de experiência), profissionais que desejam criar sua própria wiki de código, e qualquer pessoa com excesso de trechos de código copiados e colados.
- **⏱️ Tempo necessário:** 30 minutos → Reduzido para 1 minuto
- **🤖 Modelos recomendados:** Claude 3.5 Sonnet (excelente em refatoração e documentação de código), ChatGPT (GPT-4o)

- ⭐ **Dificuldade:** ⭐☆☆☆☆
- ⚡️ **Eficácia:** ⭐⭐⭐⭐⭐
- 🚀 **Utilidade:** ⭐⭐⭐⭐⭐

> _"Se você perde 30 minutos na barra de pesquisa frustrado pensando 'Tenho certeza de que já escrevi esse código antes... como era mesmo?', é hora de mudar sua estratégia."_

Qualquer desenvolvedor provavelmente tem aquele arquivo de texto salvo no canto da área de trabalho, servindo como um verdadeiro "baú do tesouro". Aquela expressão regular complexa que você suou para encontrar no StackOverflow, uma função utilitária incrivelmente bem escrita ou até mesmo arquivos de configuração do Nginx que você tem certeza de que usará novamente um dia.

No entanto, quando você realmente precisa deles com urgência, não consegue lembrar onde os guardou ou como a lógica funcionava, o que o obriga a pesquisar no Google tudo de novo. A força vital de um trecho de código (Snippet) está na sua "facilidade de pesquisa" e nos "exemplos de uso prático". Ao confiar até mesmo os códigos mais fragmentados e copiados às pressas à IA, eles são transformados em uma biblioteca estruturada com comentários padronizados, exemplos de uso e tags relevantes. Apresentamos o prompt definitivo que transformará seu bloco de notas desorganizado em uma Base de Conhecimento (Knowledge Base) sistemática e altamente profissional.

---

## ⚡️ Resumo em 3 Linhas (TL;DR)

1. **Transformação em Ativos Pesquisáveis:** Atribui títulos claros, tags e metadados a blocos de código de origem desconhecida.
2. **Documentação Automática:** Gera instantaneamente comentários padronizados (como JSDoc) para parâmetros e valores de retorno, além de exemplos práticos de execução.
3. **Melhoria na Qualidade do Código:** A IA corrige formatações quebradas e nomes de variáveis ambíguos, deixando o código pronto para reutilização imediata.

## 🚀 A Solução: "Gerenciador de Snippets de Código com IA"

### 🥉 Versão Basic (Básica)

Use esta versão quando precisar refinar rapidamente a estrutura básica e adicionar comentários essenciais ao seu código.

> **Função (Role):** Você é um mentor sênior de engenharia de software.
>
> **Tarefa (Task):** Analise o código abaixo, corrija a formatação, adicione um resumo conciso de uma linha explicando o seu propósito e inclua os comentários essenciais.
>
> **Código:**
> `[Cole seu código aqui]`

### 🥇 Versão Pro (Profissional)

Ideal para quando você precisa de um "documento Markdown perfeito" para integrar à sua própria wiki de código ou banco de dados no Notion/Obsidian.

> **Função (Role):**
> Você é um redator técnico e desenvolvedor sênior com 10 anos de experiência, obcecado por documentação impecável e Clean Code.
>
> **Contexto (Context):**
>
> - Cenário: Estou construindo minha própria base de conhecimento de código (Notion, Obsidian) reunindo fragmentos de código que extraí da internet ou de projetos anteriores.
> - Objetivo: Transformar esse código em um snippet Markdown perfeitamente documentado para que seja fácil de pesquisar no futuro e para que "meu eu do futuro" possa compreendê-lo e aplicá-lo em questão de segundos.
>
> **Tarefa (Task):**
>
> Analise o `[Código]` fornecido e crie um documento Markdown com a seguinte estrutura:
>
> 1. Metadados: `Title` (propósito claro), `Language`, `Tags` (3 a 5 palavras-chave para otimização de pesquisa).
> 2. Descrição (Description): Resuma o problema que este código resolve em 1 ou 2 linhas claras e objetivas.
> 3. Refatoração e Comentários (Refactored Code): Corrija a indentação e a formatação, e adicione comentários detalhados para parâmetros (`@param`) e valores de retorno (`@return`) utilizando o padrão JSDoc (ou a docstring padrão da linguagem correspondente). Se houver nomes de variáveis ambíguos, renomeie-os para maximizar a legibilidade.
> 4. Exemplo de Uso (Usage Example): Elabore um exemplo prático de execução demonstrando como chamar e utilizar este código em um cenário real.
>
> **Restrições (Constraints):**
>
> - O formato de saída deve ser um único documento em Markdown.
> - Preserve a intenção original do código ao máximo; evite alterar drasticamente os nomes de variáveis ou funções sem extrema necessidade.
>
> **Avisos (Warning):**
>
> - Se você detectar informações sensíveis (como IPs, senhas, tokens de API, etc.) no código, exiba uma mensagem de alerta antes de prosseguir.
> - Como fornecerei diferentes trechos na seção `[Código]` de forma contínua, responda à minha primeira mensagem apenas com: "Por favor, insira o código que deseja organizar."

## 🙋 Perguntas Frequentes (FAQ)

- **P: E se a política de segurança da minha empresa proibir o envio de códigos para IAs externas?**
  - R: Informações confidenciais críticas, como chaves de API, senhas de banco de dados, IPs de servidores internos ou lógicas de negócios proprietárias, devem ser absolutamente removidas ou mascaradas com valores fictícios (ex: `XXX`, `1234`) antes do envio. Se o seu ambiente exige segurança extrema, recomendamos a adoção de modelos open-source executados localmente, como o Llama 3.

- **P: O que faço se a formatação do código ou a nomenclatura das variáveis não seguirem as convenções da minha equipe?**
  - R: Você pode facilmente adicionar as suas próprias diretrizes de código à seção de **Restrições (Constraints)** do prompt. (Exemplo: _"Mantenha rigorosamente os nomes das variáveis em camelCase e utilize aspas simples em vez de aspas duplas."_)

- **P: O prompt suporta todas as linguagens de programação?**
  - R: Sim. Ele opera de forma brilhante com linguagens populares como Python, JavaScript, Java e C++, e se adapta perfeitamente a queries SQL, scripts Bash ou até mesmo na adição de comentários elucidativos em Dockerfiles complexos e arquivos YAML do Kubernetes.

## 🧬 Anatomia do Prompt (Por que funciona?)

1. **Atribuição de Função (Role):** Ao equipar a IA com a persona de um "redator técnico e desenvolvedor sênior", nós a forçamos a transcender a simples correção de sintaxe, abordando a tarefa sob a ótica de criar um "documento" altamente legível para outros desenvolvedores (ou para você mesmo no futuro).
2. **Otimização de Pesquisa (Tags & Title):** Exigir a extração estratégica de metadados facilita imensamente a pesquisa no Notion ou Obsidian, maximizando a taxa de reutilização e a eficiência a longo prazo.
3. **Exigência de Exemplo de Uso:** Ao instruir de forma explícita a geração de um guia de uso prático, eliminamos a carga cognitiva que seria desperdiçada tentando decifrar o funcionamento do código meses depois.

## 📊 Prova: Antes e Depois (Before & After)

### ❌ Antes (Entrada)

```javascript
//   (Calculando diferença de tempo)
function cd(d1, d2) {
  let t = new Date(d1).getTime() - new Date(d2).getTime();
  return Math.floor(t / (1000 * 60 * 60 * 24));
}
```

### ✅ Depois (Resultado)

````markdown
**Title:** Função para Calcular a Diferença de Dias Entre Duas Datas
**Language:** JavaScript
**Tags:** `#Date`, `#Utility`, `#TimeDifference`, `#JavaScript`

**Description:**
Uma função utilitária que recebe duas strings de data (ou objetos Date) e retorna a diferença exata em dias (Days) entre elas como um número inteiro.

**Refactored Code:**

```javascript
/**
 * Calcula a diferença em dias entre duas datas.
 *
 * @param {string|Date} targetDate - A data alvo para comparação
 * @param {string|Date} baseDate - A data base
 * @returns {number} A diferença em dias (retorna um número positivo se a targetDate for no futuro)
 */
function calculateDaysDifference(targetDate, baseDate) {
  const timeDifference =
    new Date(targetDate).getTime() - new Date(baseDate).getTime();
  return Math.floor(timeDifference / (1000 * 60 * 60 * 24));
}
```
````

**Usage Example:**

```javascript
const today = "2026-02-12";
const dDay = "2026-12-25";

const daysLeft = calculateDaysDifference(dDay, today);
console.log(`Faltam ${daysLeft} dias para o Natal!`);
// Saída: Faltam 316 dias para o Natal!
```
