## 📝 My Code is Trash?! Clean Code Refactoring Coach

- **🎯 추천 대상:** 주니어부터 시니어까지 모든 개발자
- **⏱️ 소요 시간:** 30분 → 1분 단축
- **🤖 추천 모델:** 모든 대화형 AI (ChatGPT, Claude, Gemini 등)

- ⭐ **난이도:** ⭐⭐☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐☆

> _"나중에 꼭 리팩토링해야지 다짐해놓고, 1년 뒤 다시 열어본 내 코드에 경악한 적 있으신가요?"_

스파게티 코드는 개발 생산성을 갉아먹는 조용한 암살자입니다. 가독성이 무너진 코드 베이스에는 버그가 교묘하게 숨어들고, 간단한 기능 추가조차 끝없는 지연을 초래합니다. 이 프롬프트는 소프트웨어 장인정신과 '클린 코드' 철학을 완벽하게 체화한 **수석 엔지니어(Principal Engineer)**의 페르소나를 AI에 부여합니다. 여러분의 레거시 코드가 직관적이고 견고하게 빛날 때까지, 타협 없는 코드 리뷰와 무자비한 리팩토링을 경험해 보세요.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **로직 분리 및 단순화:** 악취를 풍기는 중복 코드를 제거하고, 비대해진 함수를 단일 책임 원칙(SRP)에 입각하여 깔끔하게 모듈화합니다.
2. **시맨틱 네이밍:** 암호문처럼 난해한 변수와 함수명을, 의도가 명확히 드러나는 자기 문서화(Self-documenting) 네이밍으로 전면 교체합니다.
3. **모던 신택스 적용:** 레거시 문법을 타겟 언어의 최신 표준(Idiomatic syntax)으로 업그레이드하여 성능과 가독성을 동시에 극대화합니다.

## 🚀 해결책: "클린 코드 리파이너"

### 🥉 Basic Version (기본형)

복잡한 설정 없이, 엉킨 코드의 구조를 즉각적이고 빠르게 정돈하고 싶을 때 활용하세요.

> **역할 (Role):** 너는 `[수석 소프트웨어 아키텍트]`야.
> **요청 (Task):** 다음 코드를 가독성이 높고 유지보수하기 쉬운 클린 코드로 리팩토링해 줘.

### 🥇 Pro Version (전문가형)

심도 있는 코드 리뷰와 함께, 실무에 즉시 투입 가능한 엔터프라이즈급 리팩토링 결과를 원할 때 사용하는 강력한 프롬프트입니다.

> **역할 (Role):** 너는 `[클린 코드 전도사이자 수석 엔지니어(Principal Engineer)]`야.
>
> **상황 (Context):**
>
> - 배경: `[현재 내 코드는 기능적으로는 작동하지만, 강하게 결합되어 있고 가독성이 떨어지는 스파게티 코드야.]`
> - 목표: `[이 코드를 확장 가능하고 견고하며, 다른 개발자가 쉽게 유지보수할 수 있도록 완벽하게 리팩토링하는 것]`
>
> **요청 (Task):**
>
> 1. **시맨틱 네이밍:** 변수와 함수명을 직관적이고 자기 문서화(Self-documenting)가 가능하도록 전면 수정해. (예: `a`를 `userAge`로 변경)
> 2. **구조적 무결성:** 단일 책임 원칙(SRP)에 따라 비대한 함수를 잘게 쪼개고, 중복된 로직을 우아하게 통합해.
> 3. **모더니제이션:** 타겟 언어의 가장 최신 관용적 문법(Idiomatic syntax)을 사용하여 리팩토링해. (예: JavaScript는 ES6+, Python은 3.10+ 기준)
> 4. **코드 리뷰:** 변경된 각 항목에 대해 **왜(Why)** 그렇게 수정했는지 명확히 설명하는 'Before/After 분석' 피드백을 제공해.
>
> **제약사항 (Constraints):**
>
> - 핵심 비즈니스 로직과 기능적 동작은 **반드시** 기존과 100% 동일하게 유지해야 해.
> - 복잡한 비즈니스 로직에만 주석을 추가하고, 코드 자체로 의도가 설명된다면 불필요한 주석은 과감히 제거해.
> - 최종 결과물 코드는 반드시 마크다운 코드 블럭 형식으로 출력해.
>
> **주의사항 (Warning):**
>
> - 확실하게 검증되지 않은 최신 문법이나 존재하지 않는 라이브러리를 지어내지 마. (환각 방지)
>
> **대상 코드:**
>
> `[여기에 리팩토링이 필요한 스파게티 코드를 붙여넣으세요]`

## 💡 작성자 코멘트 (Insight)

이 프롬프트는 끔찍한 레거시 코드를 물려받았거나, 데드라인에 쫓겨 급조한 프로토타입을 프로덕션 레벨로 끌어올려야 할 때 **구세주** 역할을 톡톡히 해냅니다. 이 프롬프트의 진정한 가치는 단순히 예쁜 코드를 얻는 데 있지 않고, **'코드 리뷰(Code Review)'** 지시사항을 통한 **강제적 학습**에 있습니다.

AI에게 Early Return 패턴을 적용한 이유나 매직 넘버(Magic Number)를 상수로 치환한 근거 등, 코드 변경의 **'왜(Why)'**를 집요하게 설명하도록 강제했습니다. 이를 통해 개발자는 맹목적으로 코드를 복제하는 것을 넘어, 시니어 엔지니어의 사고방식을 자연스럽게 체득하게 됩니다. PR(Pull Request)을 올리기 전 이 프롬프트를 통해 셀프 리뷰를 진행해 보시길 강력히 권장합니다. 야근과 피로에 찌든 개발자가 무심코 저지르기 쉬운 치명적인 실수와 코드 스멜(Code Smell)을 즉각적이고 완벽하게 잡아낼 것입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 이 프롬프트를 사용하면 핵심 비즈니스 로직이 망가지거나 앱에 버그가 생기지 않을까요?**
  - A: 프롬프트의 제약사항(Constraints)에 **기존 동작을 완벽히 보존**하도록 강력한 안전장치를 걸어두었습니다. 하지만 AI가 리팩토링한 코드를 프로덕션에 적용하기 전에는, 반드시 유닛 테스트(Unit Test)나 통합 테스트를 실행하여 예상치 못한 사이드 이펙트(Edge-case regression)가 없는지 교차 검증해야 합니다.

- **Q: 실무에서 잘 쓰이지 않는 마이너한 언어나 아주 오래된 레거시 버전의 언어에도 적용할 수 있나요?**
  - A: 네, 완벽하게 작동합니다. 대형 언어 모델(LLM)은 현존하는 거의 모든 프로그래밍 언어의 방대한 저장소를 이미 학습했습니다. 프롬프트의 `[상황]` 변수에 정확한 언어와 버전을 명시해 주기만 하면 됩니다. (예: _"이 코드는 레거시 Perl 5 기반 시스템의 스크립트야."_)

- **Q: 사내 비공개 소스코드를 퍼블릭 AI에 그대로 붙여넣어도 보안상 안전한가요?**
  - A: **절대 주의가 필요합니다.** 회사에서 데이터 학습이 차단된 안전한 엔터프라이즈급 AI 요금제(예: ChatGPT Enterprise)를 사용 중인 것이 아니라면, 민감한 API 키나 자격 증명(Credential), 기업의 핵심 기밀 로직은 퍼블릭 AI 모델에 절대 입력해선 안 됩니다. 필요한 경우 민감 데이터를 더미 값으로 마스킹한 후 사용하세요.

## 🧬 프롬프트 해부 (Why it works?)

1. **명시적 페르소나 (Explicit Persona):** AI의 역할을 단순한 코더가 아닌 '클린 코드 전도사 및 수석 엔지니어'로 승격시켰습니다. 이를 통해 단순히 에러 없이 작동만 하는 코드를 뱉어내는 대신, 가독성과 확립된 소프트웨어 디자인 패턴을 최우선으로 고려하도록 AI의 추론 회로를 최적화했습니다.
2. **명확한 제약 조건 (Clear Constraints):** "불필요한 주석 제거" 및 "기존 로직 보존" 등의 엄격한 가드레일을 설정했습니다. 이는 AI가 의도치 않은 기능을 임의로 추가하는 환각 현상(Hallucination)을 원천 차단하고, 오로지 코드의 구조적 개선과 품질 향상에만 컴퓨팅 리소스를 집중하게 만듭니다.

## 📊 증명: Before & After

### ❌ Before (입력)

```python
def calc(x, y):
    if x > 19:
        print("성인")
        p = y * 0.9
        return p
    else:
        print("학생")
        return y
```

### ✅ After (결과)

```python
def calculate_discounted_price(age: int, base_price: float) -> float:
    ADULT_AGE_THRESHOLD = 20
    DISCOUNT_RATE = 0.9

    if age >= ADULT_AGE_THRESHOLD:
        print("성인입니다.")
        return base_price * DISCOUNT_RATE

    print("학생입니다.")
    return base_price
```

**[개선 포인트 분석]**

1. **시맨틱 네이밍:** `calc`라는 모호한 함수명이 `calculate_discounted_price`로 변경되어, 코드를 읽는 순간 함수의 목적이 단번에 드러납니다.
2. **매직 넘버 제거:** 코드 내부에 하드코딩 되어있던 `19`와 `0.9` 같은 의미 불명의 숫자들이 명확하고 수정하기 쉬운 상수(Constants)로 대체되었습니다.
3. **가드 클로즈(Guard Clauses) 적용:** Early Return 기법을 활용하여 불필요한 `else` 블록을 제거하고, 전체적인 들여쓰기(Indentation) 뎁스를 줄여 가독성을 획기적으로 높였습니다.
