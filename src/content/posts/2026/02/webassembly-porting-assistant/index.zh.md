## 🦀 WebAssembly (Wasm) 移植：突破 Web 性能的极限

- **🎯 推荐对象：** 急需优化前端性能的开发者、希望将繁重后端逻辑（C++/Rust/Go）迁移到浏览器的工程师
- **⏱️ 所需时间：** 过去耗时数日的绑定踩坑 → 缩短至短短 5 分钟
- **🤖 推荐模型：** Claude 3.5 Sonnet（代码转换能力极强）、GPT-4o

- ⭐ **难度：** ⭐⭐⭐⭐☆
- ⚡️ **有效性：** ⭐⭐⭐⭐⭐
- 🚀 **实用度：** ⭐⭐⭐☆☆

> _“为了 10% 的性能提升，我曾熬夜死磕 JS 优化；直到将计算逻辑移植为 Wasm，速度瞬间飙升了 10 倍。但这该死的‘移植’过程，到底怎样才能变得简单点？”_

现代浏览器早已不再是单纯的文档查看器。当涉及视频处理、复杂加密算法或大规模数值计算时，仅靠 JavaScript (JS) 的 V8 引擎必然会遭遇性能瓶颈。WebAssembly (Wasm) 无疑是完美的破局之道，但将现有的 C++ 或 Rust 代码转换为 Wasm，还要处理复杂的 JS 共享内存及绑定（Binding）机制，往往伴随着令人绝望的踩坑体验。

这款提示词不仅能完成基础的代码翻译，更能一次性为您生成防止内存泄漏的安全架构，以及完美契合的 JS 桥接代码，堪称无可挑剔的“Wasm 移植助手”。

## ⚡️ 3句话总结 (TL;DR)

1. **打破语言壁垒：** 能够将 C++、Rust、Go 等传统的遗留代码，瞬间转换为可在 Web 端原生运行的 Wasm 模块。
2. **绑定代码自动化：** 自动生成 JS 与 Wasm 之间的数据类型映射及共享内存（Shared Memory）通信代码。
3. **保障类型安全：** 同步输出完美的 TypeScript 接口定义文件（`.d.ts`），前端团队开箱即用。

## 💡 作者点评 (Insight)

在尝试进行 WebAssembly 移植时，让大多数前端开发者半途而废的往往是这个致命问题：**“到底该怎么把庞大的 JS 数组传给 Wasm，然后再安全地把结果拿回来？”** 处理简单的数字或字符串轻而易举，但当涉及大容量的图像缓冲区或音频流（如 `Float32Array`）时，就必须直接掌控底层内存。

这款提示词的核心精髓在于**任务 2（内存架构设计）**。如果你只是一般性地让 AI 写代码，它通常会吐出“把内存整体深拷贝一遍再传递”的低效代码。但只要像上述提示词那样明确施加“最小化复制开销”和“利用共享内存”的强约束，AI 就会真正调动 `wasm-bindgen` 的强大特性（例如直接访问指针内存），为你写出“真正能扛住生产环境流量”的硬核代码。此外，在 Webpack 或 Vite 构建环境下加载 Wasm 时常会遇到异步时序的坑，通过强制 AI 提供 React 示例，也能将这些痛点一并抹平。

## 🙋 常见问题 (FAQ)

- **Q：我的现有代码库太庞大，无法一次性复制粘贴进去，该怎么办？**
  - A：强烈建议不要把整个项目一股脑扔给 AI。您可以按核心函数或独立模块为单位进行拆分，分批运行提示词。如果模块间耦合度较高，可以在 `[背景 (Context)]` 中先对 AI 简述“该模块的整体架构依赖”，这样 AI 就能结合上下文生成更精准的绑定接口。

- **Q：转换出来的 Rust 代码我到底该怎么编译？**
  - A：使用官方提供的工具链 `wasm-pack` 非常简单。通常 AI 会在生成的代码下方附上标准编译命令（如 `wasm-pack build --target web`）。您只需直接复制并在终端执行，就能瞬间得到一个可以直接在前端项目中引入使用的 `pkg` 文件夹。

- **Q：用 Emscripten 移植 C++ 代码，和用 Rust + wasm-bindgen 相比有什么区别？**
  - A：Emscripten 确实非常适合将庞大复杂的 C/C++ 遗留项目“一键打包”搬到 Web 上，但它生成的代码往往体积臃肿，且与现代 JS 交互的 API 显得非常笨重。相比之下，Rust 搭配 `wasm-bindgen` 能够生成体积极其精简的模块，并且与 TypeScript 生态的契合度拥有压倒性优势，这也是目前前端领域事实上的工程标准。因此，本提示词也强烈引导您使用 Rust 进行重构。

## 🧬 提示词解剖 (Why it works?)

1. **端到端 (E2E) 的落地导向：** 不仅局限于语法层面的翻译，更进一步要求输出在现代前端框架（如 React）中的加载与渲染链路，从而交付出“拿到就能跑的生产级资产”。
2. **严苛的性能约束 (Constraints)：** 引入 Wasm 的唯一理由就是“榨干性能”。通过强行注入“最小化复制”和“避免序列化”等专家级关键字，彻底切断了 AI 试图生成低效代码的退路。
3. **类型安全边界 (Type Safety)：** 通过强制要求生成 `.d.ts` 文件，在 JS 的动态世界与 Wasm 的静态世界之间建立了一道坚固的类型防线，在开发阶段就将潜在的运行时错误彻底扼杀。

### ✅ After (使用 Pro 提示词时的 AI 回答)

```rust
// 利用内存指针直接进行原地修改，避免拷贝 (Zero-copy)
#[wasm_bindgen]
pub fn apply_filter_in_place(data: &mut [u8], width: usize, height: usize) {
    // 直接访问由 JS 的 Uint8Array 指向的 Wasm 线性内存并进行操作
    for i in (0..data.len()).step_by(4) {
        // 直接对红、绿、蓝通道应用滤镜
        let r = data[i];
        let g = data[i + 1];
        let b = data[i + 2];

        data[i] = (r as f32 * 0.393 + g as f32 * 0.769 + b as f32 * 0.189) as u8;
        // ... (省略) 彻底消除了内存分配/释放带来的额外开销
    }
}
```
