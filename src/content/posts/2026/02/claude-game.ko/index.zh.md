游戏开发的范式正在发生翻天覆地的变化。过去，单是实现 2D 物理引擎并管理事件监听器可能就需要耗费数天时间；但现在，得益于 **Claude Code** 这种自主型 AI 编程智能体，游戏开发的准入门槛已被彻底打破。它不仅仅是简单的代码补全，而是能够深刻理解项目架构并自主编写代码。本文将分享我如何利用 Claude Code 在短短 1 小时内完成一款基于浏览器的 2D 高尔夫游戏，并附上逐步指导的实战提示词。

## ⚡️ 3句话总结 (TL;DR)

1. **AI 智能体的进化：** Claude Code 超越了单纯的代码片段生成，它能主动设计从物理运算（速度、摩擦力）到渲染逻辑的完整游戏架构。
2. **压倒性的生产力：** 无需再为繁琐的鼠标事件处理和碰撞数学公式发愁，只需通过提示词下达指令，即可轻松实现“拖拽射击 (Drag-to-shoot)”机制。
3. **高扩展性的模块化：** AI 会从一开始就将 `Physics`、`Input`、`Level` 类分离，编写高度模块化的代码，为后续添加新功能（如多人游戏、新关卡等）打下坚实基础。

## 🚀 解决方案："Claude Code 游戏开发提示词"

### 🥉 Basic Version (基础版)

当你想快速搭建游戏骨架（Boilerplate）时使用的初始设置提示词。

> **角色：** 你是一名`[资深游戏开发者]`。
> **请求：** 请使用 `[HTML5 Canvas]` 制作一款基于浏览器的 2D 高尔夫游戏。需要包含高尔夫球和球洞，并且必须采用鼠标拖拽射击（drag-to-shoot）的操控方式。

\

### 🥇 Pro Version (专业版)

用于构建物理引擎、输入处理、关卡设计完全分离的模块化游戏架构的高阶提示词。

> **角色 (Role)：** 你是一名拥有 10 年经验的 `[资深前端游戏客户端开发者]`，也是我完美的结对编程搭档。
>
> **背景 (Context)：**
>
> - 背景：我打算仅使用原生 Vanilla JS 和 HTML5 Canvas，开发一款轻量级的 2D 俯视视角高尔夫游戏。
> - 目标：编写高可维护性的代码，确保物理引擎、输入事件、关卡状态管理完全模块化。
>
> **请求 (Task)：**
>
> 1. 将文件结构化分离为 `index.html`（Canvas 容器）、`game.js`（主循环）、`physics.js`（向量及摩擦力/碰撞数学运算）。
> 2. 在 `physics.js` 中，精确实现草地摩擦力（Friction）和墙壁碰撞反弹（Bounce）的逻辑。
> 3. 利用鼠标的 `mousedown`、`mousemove`、`mouseup` 事件，添加视觉指示器（Indicator Line）功能，在拖拽球体时显示发射轨迹和力度。
> 4. 创建 `Obstacle` 类，并设计 3 个难度递增的关卡（Level）数组。
>
> **约束条件 (Constraints)：**
>
> - 绝对不能使用任何外部游戏引擎库（如 Phaser.js 等）。
> - 必须为所有核心逻辑添加注释，解释其运行原理。
> - 使用模块模式或 ES6 Class 语法，以面向对象的方式编写代码。

## 💡 作者见解 (Insight)

这套提示词的核心在于，将 AI 视为“架构师”而非单纯的“代码生成器”。如果不从一开始就通过约束条件（Constraints）明确指示“分离文件”、“拆分核心类”，AI 往往会将所有代码像意大利面条一样全部塞进 `index.html` 这一个文件里。特别是通过强制分离物理运算和状态管理模块，我们在后续添加风阻（Wind）或集成 WebSockets 时，代码的修改范围得到了极大的缩减。Claude Code 保持上下文的能力极其出色，因此“先搭骨架，再添血肉”的开发方式是最高效的。

## 🙋 常见问题 (FAQ)

- **Q: 如果我完全不懂游戏物理相关的数学公式，也能开发吗？**
  - A: 完全可以！你只需用自然语言描述物理现象，比如“实现草地的摩擦力”、“碰到墙壁时反弹”，Claude 会自动将向量点积、恢复系数等复杂的数学公式转换为精准的代码。

- **Q: 如何让游戏在移动端浏览器上也支持触摸操作？**
  - A: 只需要在 Pro 版提示词的 Task 部分增加一句话：“不仅要映射鼠标事件，还要映射 `touchstart`、`touchmove`、`touchend` 事件，确保其在移动端环境下也能完美运行。”即可轻松搞定。

## 🧬 提示词解剖 (Why it works?)

1.  **明确的架构指令：** 明确要求将代码拆分为 `game.js` 和 `physics.js` 等独立文件，大幅降低了代码复杂度，强制 AI 专注于各自的业务领域。
2.  **具象化的机制描述：** 不仅仅说“做个高尔夫游戏”，而是非常详细地描述了用户体验（UX），如“在拖拽球体时显示发射轨迹和力度的视觉指示器”，从而将开发意图精准无误地传达给 AI。

// 完全模块化、面向对象的物理引擎结构 (physics.js)
export class PhysicsBody {
  constructor(x, y) {
    this.position = new Vector(x, y);
    this.velocity = new Vector(0, 0);
    this.friction = 0.98; // 草地摩擦系数
  }

  update() {
    // 将摩擦力应用于速度
    this.velocity.multiply(this.friction);

    // 更新位置
    this.position.add(this.velocity);

    // 处理画布边界碰撞 (调用恢复系数相关逻辑)
    this.handleWallCollisions();
  }
}
