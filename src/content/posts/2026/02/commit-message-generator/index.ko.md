## 📝 "커밋 메시지 추천 좀" 고민 끝! Conventional Commits 자동 생성

- **🎯 추천 대상:** 주니어 개발자, 오픈소스 기여자, 협업을 시작하는 팀
- **⏱️ 소요 시간:** 5분 → 10초 단축
- **🤖 추천 모델:** ChatGPT (GPT-4o), Claude 3.5 Sonnet, Gemini 2.5 Pro

- ⭐ **난이도:** ⭐☆☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"코드 짜는 데는 3시간, 커밋 메시지 고민하는 데 30분... 'Update code' 치고 퇴근하고 싶었던 적 없으신가요?"_

`fix: bug`, `update code`, `진짜_최종_수정`... 혹시 아직도 이렇게 무의미한 커밋 로그를 남기고 계신가요?
좋은 커밋 메시지는 프로젝트 히스토리 관리의 핵심이자, 동료와 미래의 나를 위한 최고의 배려입니다. 하지만 막상 코딩을 끝내고 지친 상태에서 영어로, 그것도 엄격한 구조에 맞춰 메시지를 작성하려다 보면 머리에 쥐가 나기 일쑤죠.
이제 더 이상 번역기를 돌리며 작문 스트레스를 받을 필요가 없습니다. 작업한 내용이나 `git diff` 결과만 프롬프트에 던져주세요. 오픈소스 생태계의 글로벌 표준인 **Conventional Commits** 스타일에 맞춰, 그대로 복사해서 붙여넣을 수 있는 완벽한 영문 커밋 메시지를 즉석에서 생성해 드립니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. `git diff` 결과만 붙여넣으면 글로벌 표준인 Conventional Commits 규격으로 즉시 변환됩니다.
2. `feat`, `fix`, `refactor` 등 변경 사항의 성격에 부합하는 정확한 **타입(Type)**을 AI가 스스로 분류합니다.
3. 작업의 의도(Why)와 구체적 내용(What)이 논리적으로 담긴 본문(Body)까지 완벽한 영문으로 작성해 드립니다.

## 🚀 해결책: "글로벌 표준 커밋 메시지 생성기"

### 🥉 Basic Version (기본형)

빠르게 제목 한 줄만 필요할 때 가볍게 사용해 보세요.

> **역할:** 너는 완벽한 Git 히스토리를 추구하는 시니어 개발자야.
> **요청:** 내가 제공하는 변경 내용을 보고, Conventional Commits 표준에 맞춘 영문 커밋 메시지 제목(`type(scope): subject` 형식)을 3가지 제안해 줘.


### 🥇 Pro Version (전문가형)

제목, 본문, 꼬리말까지 완벽하게 구조화된 수준 높은 커밋 메시지가 필요할 때 사용하세요. 아래 프롬프트를 복사하여 AI 창에 그대로 붙여넣으시면 됩니다.

> **역할 (Role):**
> 너는 수많은 오픈소스를 관리하며 완벽한 Git 히스토리에 집착하는 엄격한 메인테이너이자 시니어 엔지니어링 매니저야.
>
> **상황 (Context):**
>
> - 배경: 내가 방금 코드 작성을 마쳤고, 커밋을 남겨야 하는 상황이야.
> - 목표: 동료들이 리뷰하기 쉽고, 나중에 히스토리를 추적하기에 완벽한 커밋 메시지를 작성해야 해.
>
> **요청 (Task):**
> 아래의 `[변경 사항]`을 분석하여 **Conventional Commits v1.0.0** 규칙에 맞는 완벽한 커밋 메시지 초안을 3가지 옵션으로 제시해 줘.
>
> 1. **형식:** `<type>(<scope>): <subject>` (제목 1줄)
> 2. **본문 (Body):** 왜(Why) 변경했고 무엇(What)을 어떻게 해결했는지 불릿 포인트(`-`)로 명확히 요약할 것.
> 3. **꼬리말 (Footer):** (선택 사항) 관련된 이슈 번호가 있다면 `Closes #이슈번호` 형태로 작성할 것.
>
> **제약사항 (Constraints):**
>
> - Type은 반드시 다음 중 하나만 사용할 것: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`, `revert`
> - Subject(제목)는 50자를 넘지 않도록 간결하게 작성하고, 반드시 **영문 명령문(동사 원형)**으로 시작할 것 (예: `Add`, `Fix`, `Update`). 끝에 마침표(`.`)는 찍지 마.
> - 본문(Body)은 제목과 한 줄 띄우고 작성하며, 영문으로 작성해 줘.
> - 각 옵션의 차이점이나 포커스(예: 버그 수정 중심, UI 개선 중심)를 한국어로 짧게 덧붙여 줘.
>
> **입력 (Input):**
>
> - 변경 사항: `[여기에 git diff 결과나 수정한 내용을 한국어로 편하게 입력하세요]`

## 💡 작성자 코멘트 (Insight)

이 프롬프트의 핵심은 개발자들을 가장 괴롭히는 **'영작의 압박'**과 **'엄격한 포맷팅'**이라는 두 가지 허들을 한 번에 제거해 준다는 점입니다. 

단순히 "이거 고쳤어"라고 말하는 것을 넘어, 터미널에서 `git diff` 명령어를 실행한 뒤 출력된 코드 변경 내역을 그대로 복사하여 `[변경 사항]`에 붙여넣어 보세요. AI가 추가되고 삭제된 코드 라인의 맥락을 스스로 분석하여, 작업의 의도까지 짚어내는 소름 돋는 커밋 메시지를 완성해 냅니다. 

**💡 실무 활용 꿀팁:**
터미널 환경을 벗어나 브라우저를 띄우는 것조차 귀찮다면, Cursor나 GitHub Copilot Chat 등 에디터에 내장된 AI의 System Prompt로 이 프롬프트를 등록해 두세요. 코딩을 마친 뒤 **"방금 수정한 내용 커밋 메시지로 써줘"**라는 짧은 명령어 한 마디면, 여러분의 개발 생산성이 비약적으로 상승할 것입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 회사 내부 커밋 규칙(예: Jira 티켓 번호 필수 포함)이 따로 있는데 어떻게 적용하나요?**
  - A: 프롬프트의 **제약사항 (Constraints)** 섹션에 `- 제목 맨 앞에 반드시 '[ABC-123]' 형식의 지라 티켓 번호를 붙여 줘`라는 조건을 단 한 줄만 추가하시면, 팀의 고유한 컨벤션을 완벽하게 준수하는 결과를 얻을 수 있습니다.

- **Q: 제목은 영어로 하되, 커밋 본문(Body)은 한국어로 작성하고 싶은데요?**
  - A: 프롬프트 내용 중 `본문(Body)은 제목과 한 줄 띄우고 작성하며, 영문으로 작성해 줘.`라는 문장을 `본문은 한국어로 명확하게 작성해 줘.`로 살짝 수정해 보세요. 글로벌 표준의 껍데기는 유지하면서, 팀원들과의 소통 효율은 극대화하는 실용적인 포맷이 완성됩니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **표준 스펙 강제 (Conventional Commits v1.0.0):** AI에게 막연하게 "좋은 메시지를 써줘"라고 요청하는 대신, 전 세계적으로 통용되는 명확한 표준 규격을 직접 지시함으로써 출력물의 포맷이 흔들리지 않도록 강력하게 고정했습니다.
2. **명확한 제약 조건 (Constraints):** "명령문으로 시작", "50자 제한", "마침표 금지" 등 깐깐한 시니어 개발자가 코드 리뷰에서 지적할 법한 엄격한 린트(Lint) 규칙들을 프롬프트 레벨에서 선제적으로 차단합니다. 이를 통해 추가적인 수정 비용을 0으로 만듭니다.
3. **선택권 부여 (3가지 옵션):** AI가 단 하나의 정답을 강요하지 않도록 설계했습니다. 성격이 다른 3가지 옵션을 동시에 제시하게 하여, 사용자가 자신이 의도한 작업의 핵심과 가장 잘 맞는 최적의 메시지를 취사선택할 수 있습니다.

**프롬프트 적용 결과:**

**Option 1 (기능 버그 수정 우선순위)**

```text
fix(auth): resolve login failure issue and update button UI

- Fix logic error that prevented users from logging in
- Change the login button color to red for better visibility
- docs: Add installation guide to README.md
```

**Option 2 (UI 및 스타일 중심)**

```text
style(ui): update login button color to red and fix login bug

- Change login button color to enhance UI consistency
- Fix authentication bug causing login failures
- docs: Update README with installation instructions
```
