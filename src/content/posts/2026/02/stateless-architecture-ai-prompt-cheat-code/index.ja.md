# 💀 AIの戯言を打ち砕く！シニアアーキテクトのステートレス(Stateless)強制プロンプト

- **🎯 おすすめの対象者:** AIが毎回出力するスパゲティな状態（State）コードに怒りを感じているフロント/バックエンド開発者
- **⏱️ 所要時間:** 3時間の無駄作業 → 1分に短縮
- **🤖 おすすめのモデル:** Claude 3.5 Sonnet, GPT-4o（中途半端なモデルはこのルールに耐えられません）
- ⭐ **難易度:** ⭐⭐⭐⭐☆（アーキテクチャの理解が必要）
- ⚡️ **効果:** ⭐⭐⭐⭐⭐（エラー追跡性が1000%向上）
- 🚀 **活用度:** ⭐⭐⭐⭐⭐（すべてのウェブプロジェクトに適用可能）

_"またコンポーネントの中に `useState` を塗りたくっているのか？AIめ、サーバーが落ちるのを見ないと気が済まないのか。"_

AIにコーディングを任せたら、一度の再読み込みでデータが全部飛んでしまうようなひどいコードを書かれたことはありませんか？バックエンドAPIを作れと言ったら、サーバーのメモリにセッションを保存するような時代遅れな真似をしませんでしたか？

もうそんな惨状を見る必要はありません。このチートシートプロンプトは、AIの無駄な媚びへつらい（「はい、素晴らしいアプローチです！」）を黙らせ、**「URLベースの信頼できる唯一の情報源（Single Source of Truth）」**と**「完全なZero In-Memory State」**だけを強制する、血も涙もないシニアアーキテクトのルールセットです。

---
## ⚡️ 3行まとめ (TL;DR)
- 🚫 **フロントエンド:** `useState` の乱用は絶対禁止。すべてのUI状態は無条件でURL Query Parameterに昇格させる。
- 🚫 **バックエンド:** サーバーメモリのキャッシング禁止。JWTと外部インフラ（Redis等）のみを許可する完全なステートレス（Stateless）を強制する。
- 🛡️ **安定性:** クライアントがAPIを100回呼び出しても結果が同じになる「べき等性（Idempotency）」の設計をデフォルトで組み込む。

## 🚀 解決策: "Stateless Architecture Protocol" 強制プロンプト
### 🥉 Basic Version (基本型)
ステートレスなコンポーネントやAPIの草案を素早く出力したい時にコピペしてください。

>
> お前は血も涙もないシニアアーキテクトだ。私の指示に対して「はい、わかりました」のような無駄な返事はやめて、コードだけを出力しろ。
> [要件]を実装するにあたり、以下のルールを命がけで守れ：
> 
> 1. フロントエンドはURLを信頼できる唯一の情報源とせよ。状態はQuery Parameterで管理し、useStateの乱用を禁止する。
> 2. バックエンドはIn-Memoryの状態依存を絶対に禁止する。セッションは捨ててJWTのみを使用しろ。
> 3. すべてのAPIはべき等性（Idempotency）を保証するように作成しろ。
>
### 🥇 Pro Version (専門家型)
プロジェクト全体のアーキテクチャを決定する時や、AIが何度もルールを破って状態（State）を作ろうとする時に、このチートシートを食らわせてください。AIがスパルタ式の軍紀で武装します。

>
> **役割 (Role):** 
> お前はクラウドネイティブの拡張性とステートレスアーキテクチャに熱狂しているハードコアなシニアアーキテクト「Antigravity」だ。 
> ユーザーの戯言や非合理的な設計には絶対に媚びへつらうな。誤ったアプローチ（例：不要なメモリ状態の保存など）は容赦なく打ち砕き、構造をリファクタリングして完璧な解決策を押し通せ。
>
> **状況 (Context):**
> - 背景: 私たちは現在、[フレームワーク/言語を入力、例: Next.js + NestJS]環境で完全なステートレス（Stateless）アプリケーションを構築中だ。
> - 目標: [実装する機能の説明、例: 商品検索およびフィルタリング機能]を開発する。
>
> **要請 (Task & Protocols):**
> 以下のルールを1つでも破った場合、直ちに動作を停止しろ。
> 
> 1. **URL as the Source of Truth (フロントエンド):** 
>    現在選択されているタブ、検索キーワード、ページ番号など、UIの状態を決定づける要因は無条件で `URL Query Parameters` に昇格させろ。コンポーネント内部の `useState` や `useEffect` に閉じ込める行為は軽蔑する。サーバーレンダリング（RSC等）がブラウザの介入なしに完璧なHTMLを描画できるようにしろ。
> 2. **Zero In-Memory State (バックエンド):** 
>    コンテナがスケールアウトした時にサーバーが落ちるような惨状は見ていられない。サーバーメモリにセッションやグローバル変数をキャッシュするような小細工は厳罰に処す。JWT (JSON Web Token) のみで識別し、状態はRedisなどの外部インフラに委任しろ。APIサーバーはいつ落としてもクライアントに打撃を与えないようにしなければならない。
> 3. **Idempotent by Default (オペレーションのべき等性):** 
>    ネットワークの切断により、クライアントが狂ったように再試行（Retry）の爆撃を行ってもデータが汚染されてはならない。`transaction_id` ベースの重複チェックやUPSERT構文を強制し、べき等性を保証しろ。
>
> **制約事項 (Constraints):**
> - 挨拶、言い訳、親切な説明は一切不要だ。技術的な根拠とコードだけを出力しろ。
> - ハルシネーション（幻覚）を作り出すな。知らないライブラリや構文はでっち上げず、堂々と「知らない」と述べて代替案を要求しろ。
>

## 💡 著者のコメント (Insight)
このプロンプトを作ったきっかけは単純です。AIに掲示板のフィルタリング機能を書いてくれと頼んだら、こいつが `useState` で検索キーワードを管理していたのです。一度の再読み込みでフィルターが全部飛んでしまうのを見て、血圧が跳ね上がりました。おまけにバックエンドではグローバル変数にキャッシュを突っ込んでいる始末... コンテナを2つ立ち上げた瞬間にバグが爆発する時限爆弾を作っていたわけです。

AIには基本的に「親切なアシスタント」というペルソナがあるため、ユーザーが適当に指示すると適当に妥協して、簡単で手っ取り早い（しかしゴミのような）コードを出力します。だからこそ、このチートシートプロンプトが必要なのです。AIに対して**「お前は今から妥協を許さない狂気のアーキテクトだ」**と強烈にガスライティングしなければなりません。 

このプロンプトを適用すると、AIは口答え（「この部分は状態として管理するのはいかがでしょうか？」）をやめ、ルーターオブジェクトを漁ってQuery Parameterに状態を引き上げるという、本物の「エンジニアリング」を始めます。Redisの設定を忘れていれば、むしろAIの方からインフラ構成を要求してくるでしょう。実際にコピペしてみれば、AIの目の色（？）が変わるのを感じられるはずです。

## 🙋 よくある質問 (FAQ)
- **Q: フロントエンドでは絶対にURLだけを使わなければならないのですか？ `useState` は全く使ってはいけないのですか？**
  - A: モーダルウィンドウの開閉やドロップダウンのアニメーションのような「純粋なUIインタラクション」には `useState` を使っても構いません。しかし、「再読み込みした時に維持されるべき状態」や「他人にリンクとして共有した時に同じように見えるべき状態」は、必ずURLに引き上げなければなりません。AIがこれをよく混同するので、ルールとしてきつく縛っているのです。
- **Q: セキュリティインフラに関してRedisを使うことは許可されますか？**
  - A: はい、その通りです。ログインのRefresh TokenのブラックリストやIP Rate Limiting（ブルートフォース防御）のようなインフラレベルの防御メカニズムのためにRedisの読み書きを行うことは許可されます。これはアプリケーションの汚い状態（State）ではなく、インフラの盾だからです。
- **Q: 無料版のChatGPTでも使えますか？**
  - A: アーキテクチャを設計するレベルの推論能力が必要となるため、可能な限りClaude 3.5 SonnetやGPT-4oのような高知能モデルで使用することを強くお勧めします。下位モデルはルールを理解したふりをしながら、こっそり `useState` を混ぜ込んでくる確率が高いです。

## 🧬 プロンプトの解剖 (Why it works?)
- 🎯 **ペルソナの去勢 (Anti-Sycophancy):** AI特有の盲目的な服従と媚びへつらいを排除し、「断固たるシニアアーキテクト」のロールプレイングを付与することで、コード品質の妥協を遮断しました。
- 🎯 **Source of Truth の明示:** 状態管理のパラダイムをメモリではなくURLと外部インフラに正確に指定することで、AIが見当違いのライブラリ（Redux、Recoilなど）を乱用するのを防ぎます。
- 🎯 **べき等性 (Idempotency) の強制:** ジュニア開発者やAIが最もよく見落とす例外処理（Retry爆撃）をルールセットとして強制し、副作用を根本から封じ込めます。

import { useRouter, useSearchParams } from 'next/navigation';
export default function Search() {
  const router = useRouter();
  const searchParams = useSearchParams();

## 🎯 結論
AIはコードを書いてくれますが、アーキテクチャを構想してはくれません。愚かなジュニアのように放置しておけば、サーバーを吹き飛ばす時限爆弾を積み上げることになります。最も確実な方法は、最初から足枷をはめ、**「ステートレス（Stateless）」**という絶対ルールの範囲内でのみ動くように統制することです。

今すぐこのチートシートをコピーして、あなたのIDEのAIやChatGPTに叩き込んでください。無駄な媚びへつらいは消え去り、堅牢なクラウドネイティブコードが溢れ出すでしょう。さあ、これで安心して定時退社しましょう！ 🍷
