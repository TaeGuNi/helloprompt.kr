## 🧩 算法测试导师：从提交题目到获取精准提示与解答 {#algorithm}

- **🎯 推荐对象：** 遇到算法瓶颈的初中级开发者、深陷“超时（TLE）”泥潭的求职面试者
- **⏱️ 所需时间：** 10 分钟（大幅缩短单题平均解题时间）
- **🤖 推荐模型：** ChatGPT-4o, Claude 3.5 Sonnet（在逻辑推理与代码结构化方面表现卓越）

- ⭐ **难度：** ⭐☆☆☆☆
- ⚡️ **效果：** ⭐⭐⭐⭐⭐
- 🚀 **实用性：** ⭐⭐⭐⭐⭐

> _“这题到底该用动态规划（DP）还是贪心算法（Greedy）？完全没头绪，干脆去 Google 搜答案看解析算了。”_

且慢！当你习惯性地复制粘贴标准题解的那一刻，你解决未知问题的能力就已经停止生长了。从现在起，请彻底戒掉直接看答案的坏习惯，试着向 AI 提出**“只给提示，不给代码”**的严苛要求。本文将手把手教你，如何利用 Prompt 技巧将 AI 调教成你的专属“苏格拉底式”算法私教。通过引导式提问，你将逐步构建出独立推导最优解的逻辑思维肌肉，在真正的高难度算法面试中也能游刃有余。

## ⚡️ 3句话总结 (TL;DR) {#tl-dr}

1. 将算法题丢给 AI 时，必须设定一个极其严格的限制条件：“绝对不要写出标准答案的代码”。
2. 让 AI 评估你当前的解题思路与时间复杂度，并索要关于逻辑漏洞或边界条件（Edge Case）的关键提示。
3. 仅依靠核心关键词（如：双指针、滑动窗口）或伪代码（Pseudo-code）级别的点拨，完全凭借自己的力量写出最终代码。

## 🚀 解决方案："Algo Tutor Prompt"

### 🥉 Basic Version (基础版：请求提示)

当毫无头绪、只想找准解题方向时使用的基础版 Prompt。

> **角色：** 你是一位顶级大厂的算法面试官兼算法专家。
>
> **任务：** 请阅读以下算法题，并仅向我解释解决该题的核心思路与切入点。
>
> **问题：** `[在此处粘贴 LeetCode 或 Programmers 的题目描述]`
>
> **限制条件：** 绝对不要提供标准答案的代码或完整的业务逻辑。请仅从时间复杂度（Time Complexity）的角度，提示我使用哪种数据结构或算法模式最为合适。

### 🥇 Pro Version (专业版：逐步指导与代码审查)

当编写的代码不断碰壁（超时、内存溢出、解答错误）时使用的专家级 Prompt。

> **角色 (Role)：** 你是一位曾获得 ACM 算法竞赛金牌的资深后端架构师。作为我的算法导师，请引导我独立解决这道问题。
>
> **背景 (Context)：**
>
> - **问题摘要：** `[用一句话总结该题的核心目标]`
> - **当前代码 (My Code)：**
>
> `[在此处粘贴你编写的报错或超时代码]`
>
> - **存在问题：** 提交当前代码后，出现了 `[超时 (TLE) / 运行错误 (Runtime Error) / 解答错误 (WA)]`。
>
> **任务 (Task)：**
>
> 1. **寻找反例 (Counter Example)：** 请只找出一个会导致我的代码失败的边界条件（Edge Case），并告诉我输入值和预期结果。请注意：**绝对不要解释为什么会失败**。
> 2. **逻辑漏洞诊断：** 请以提问的方式指出我编写的代码逻辑中哪里出现了瓶颈，或者遗漏了什么关键条件。
> 3. **优化提示：** 请帮我计算当前代码的时间复杂度，并推荐一种能够将其降低到 `[目标时间复杂度，例如：O(N log N)]` 的算法技巧（例如：双指针、二分查找等）。
>
> **限制条件 (Constraints)：**
>
> - 绝对不要直接编写并提供修改后的正确代码。
> - 请使用苏格拉底式的问答法，一步步引导我自己发现并修复逻辑上的缺陷。

## 💡 作者评论 (Insight) {#insight}

算法测试绝不是一场单纯比拼敲击键盘速度的盲打比赛，而是一场在严苛限制下寻找最优**“模式（Pattern）”**的逻辑博弈。

请务必将这套 Prompt 无缝融入你的日常刻意练习中。它不仅是攻克算法难关的利器，在实际业务中面对**复杂业务逻辑**或排查**顽固 Bug** 时，这种“苏格拉底式辅导”同样能大放异彩。

作为进阶玩法，你还可以在 Prompt 结尾追加一句高阶指令：**“请推荐 3 道与此题考察模式类似、难度适中的 LeetCode 题目”**。AI 便会瞬间化身为你的私人教练，为你量身定制专属训练体系，助你彻底击穿 DP、DFS/BFS 等长期薄弱环节。

## 🙋 常见问题 (FAQ) {#faq}

- **Q: 除了 Python，这套 Prompt 支持 Java 或 C++ 等强类型语言吗？**
  - A: 毫无疑问，可以。现代顶级大语言模型早已精通所有主流编程语言。你甚至可以在 Prompt 的限制条件中加入特定语言的枷锁，例如要求 **“请不要使用 Java 的 Stream API”** 或 **“请引导我利用 C++ 的 STL 容器来优化”**，从而获得更加贴近底层架构、更加硬核的代码指导。

- **Q: 频繁在 IDE 和浏览器之间来回复制粘贴 Prompt 太破坏节奏了，有解决办法吗？**
  - A: 强烈建议直接拥抱 **Cursor** 或 **GitHub Copilot** 等 AI 原生编辑器。你只需在代码视口内高亮选中报错的函数，通过快捷键唤出全局输入框并直接丢出指令：“请给我一个关键提示，将此函数的时间复杂度降至 O(N)”，即可在不打破心流（Flow State）的前提下获得即时反馈。

## 🧬 Prompt 拆解 (Why it works?) {#why-it-works}

1. **绝对禁言防剧透（No Solution Constraints）：** 算法能力进阶的核心，恰恰隐藏在独立苦思冥想时的那份“痛苦挣扎”中。通过明确指令限制 AI 直接抛出答案，能强制激活你大脑的深层逻辑推演回路，形成不可磨灭的肌肉记忆。
2. **反例驱动调试（Test-Driven Debugging）：** 许多初学者常抱有“通过了基础用例就是完美代码”的致命错觉。刻意要求 AI 去挖掘并抛出极端的边界条件（Edge Case），能在潜移默化中培养你进行**极限边界测试（Boundary Value Analysis）**的资深工程师直觉。
3. **苏格拉底式启发（Socratic Method）：** 坚决摒弃“我说你听”的单向知识灌输，转而利用层层递进的犀利反问，逼迫你亲自揪出代码中的逻辑漏洞。这种高强度的启发式互动，能带来真正意义上的结构化顿悟，让你彻底告别死记硬背。
