# 构建复杂提示词：CREATE 框架

随着 AI 模型的不断发展，我们向它们分配的任务也越来越复杂。虽然简单的提示词足以应对简单的问题，但在指导 AI 处理复杂的业务逻辑或创造性任务时，需要一种系统化的方法。

本文将介绍 **CREATE 框架**，这是一种逻辑化构建提示词以最大化 AI 性能的方法。

## 什么是 CREATE 框架？

CREATE 代表了构建有效提示词的六个关键要素。

1.  **C**ontext (背景): 为 AI 分配角色和情境。
2.  **R**equest (请求): 明确说明需要执行的具体任务。
3.  **E**xamples (示例): 提供期望输出的示例 (Few-shot)。
4.  **A**djustments (调整): 设定语气、风格和约束条件。
5.  **T**ype of Output (输出类型): 指定结果的格式（表格、代码、Markdown 等）。
6.  **E**xtras (额外事项): 包含边缘情况处理或特殊说明。

---

## 分步应用指南

### 1. Context (背景设置)

赋予 AI 一个角色会显著改变回答的质量。

> “你是一名拥有 10 年经验的高级软件工程师。你擅长将遗留代码重构为现代的整洁代码。”

### 2. Request (具体请求)

避免模棱两可，使用清晰的动词。

> “分析下面提供的 Python 函数，并对其进行重构，以提高可读性并优化执行速度。”

### 3. Examples (提供示例)

AI 通过示例学习模式。展示输入和输出对。

> **输入:**
> `def calc(x,y): return x+y`
>
> **输出:**
>
> ```python
> def add_numbers(a: int, b: int) -> int:
>     """返回两个数字的和。"""
>     return a + b
> ```

### 4. Adjustments (细节调整)

决定回答的语调和风格。

> “解释应简洁并使用专业术语，但要保持亲切的语调，以便初级开发人员也能理解。”

### 5. Type of Output (输出格式)

指定你希望如何接收结果。

> “请首先在代码块中展示修改后的完整代码，然后在下方用项目符号摘要说明修改原因。”

### 6. Extras (额外说明)

防止意外情况。

> “如果在代码中发现安全漏洞，请在重构之外单独输出警告信息。”

---

## 结论

使用 CREATE 框架可以缩短编写提示词的时间，同时保持输出质量的一致性。当面临复杂问题时，不要模糊地提问，而是按照 CREATE 的结构整理思路。你设计的提示词越精细，AI 的回答就越智能。
