# 🎨 Tambo SDK로 제너레이티브 UI(Generative UI) 1분 만에 구축하기

- **🎯 추천 대상:** 프론트엔드 개발자, 풀스택 개발자, AI 서비스 기획자
- **⏱️ 소요 시간:** 3시간 → 1분 단축
- **🤖 추천 모델:** 코딩 특화 AI (Cursor, Windsurf, GitHub Copilot 등)

- ⭐ **난이도:** ⭐⭐⭐☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"챗봇이 의미 없는 텍스트만 줄줄이 뱉어내는 시대는 끝났습니다. 유저의 말 한마디에 완벽하게 작동하는 인터랙티브 UI 컴포넌트가 즉석에서 렌더링되는 마법, 아직도 수동으로 하드코딩하고 계신가요?"_

인공지능(AI)이 텍스트 생성을 넘어 **사용자 인터페이스(UI)** 자체를 실시간으로 조립하고 렌더링하는 '제너레이티브 UI(Generative UI)' 시대가 도래했습니다. 기존에는 Vercel AI SDK의 `ai/rsc` 등을 활용해 복잡한 스트리밍 로직과 보일러플레이트를 직접 짜야 했지만, 이제는 **Tambo SDK**를 통해 React 애플리케이션의 평범한 챗봇을 강력한 인터랙티브 도구로 순식간에 탈바꿈시킬 수 있습니다.

이 포스트에서는 코딩 AI 에이전트(Cursor, Windsurf 등)에게 던져주기만 하면, Tambo SDK 기반의 제너레이티브 UI 컴포넌트 레지스트리와 채팅 뼈대를 단 1분 만에 완성해 주는 스캐폴딩(Scaffolding) 프롬프트를 소개합니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **제너레이티브 UI**는 유저의 대화 맥락과 의도에 맞춰 가장 적절한 React 컴포넌트를 즉석에서 선택하고 렌더링하는 차세대 UX 기술입니다.
2. **Tambo SDK**를 사용하면 복잡한 상태 관리나 스트리밍 로직 없이도 AI가 사용할 'UI 컴포넌트 도구함(Registry)'을 매우 직관적으로 구축할 수 있습니다.
3. 아래의 프롬프트를 코딩 에이전트에 입력하면 UI 컴포넌트 정의부터 Zod 스키마 타이핑, 채팅 페이지 연동까지 완벽하게 자동 셋업됩니다.

## 🚀 해결책: "Tambo SDK 제너레이티브 UI 스캐폴더"

### 🥉 Basic Version (기본형)

빠르게 프로젝트 뼈대만 잡고 싶을 때 사용하세요.

> **역할:** 너는 시니어 프론트엔드 개발자야.
> **요청:** 내 Next.js (App Router) 프로젝트에 `@tambo/sdk`와 `ai` 패키지를 설치하고, 제너레이티브 UI를 지원하는 챗봇 페이지를 만들어줘. 장소 정보를 보여주는 `PlaceCard` 컴포넌트를 만들고 Tambo 레지스트리에 등록하는 코드까지 한 번에 작성해 줘.

<br>

### 🥇 Pro Version (전문가형)

실무에 즉시 투입 가능한 완벽한 타입 안정성과 폴더 구조가 필요할 때 사용하세요.

> **역할 (Role):** 너는 제너레이티브 UI(Generative UI) 아키텍처와 Vercel AI SDK에 정통한 시니어 프론트엔드 엔지니어(React/Next.js App Router)야.
>
> **상황 (Context):**
>
> - 배경: 현재 일반적인 텍스트 기반 챗봇을 운영 중인데, 사용자 경험(UX)을 극대화하기 위해 대화 문맥에 맞는 동적 UI(카드 리스트, 예약 폼, 차트 등)를 실시간으로 렌더링하려고 해.
> - 목표: `Tambo SDK`를 도입하여, 사용자의 여행 계획 요청("제주도 여행 일정 짜줘")에 반응해 인터랙티브한 `PlaceCard` UI를 렌더링하는 채팅 페이지 구조를 셋업하는 것.
>
> **요청 (Task):**
> 다음 4가지 스텝의 코드를 완벽한 TypeScript 기반으로 작성해 줘.
>
> 1. **패키지 설치 명령어:** `pnpm`을 사용한 `@tambo/sdk`, `ai`, `zod` 라이브러리 설치.
> 2. **UI 컴포넌트 (`components/PlaceCard.tsx`):** 장소 이름(name), 설명(description), 평점(rating)을 받아 렌더링하는 심미적인 여행지 카드 컴포넌트 제작. (Tailwind CSS 활용)
> 3. **Tambo 레지스트리 (`lib/tambo.ts`):** `createTambo`를 사용하여 `PlaceCard`를 도구(Tool)로 등록하고, `zod`를 이용해 AI가 파싱할 엄격한 데이터 스키마(Schema)를 정의할 것.
> 4. **채팅 페이지 (`app/chat/page.tsx`):** `TamboProvider`와 `GenerativeUI` 래퍼를 활용하여 AI 메시지를 렌더링하고 사용자 입력을 처리하는 클라이언트 컴포넌트(`"use client"`) 구축.
>
> **제약사항 (Constraints):**
>
> - 모든 코드는 즉시 복사-붙여넣기하여 실행 가능해야 해. (Placeholder 코드 절대 금지)
> - 불필요한 부연 설명이나 서론은 생략하고, 직관적인 코드블럭 위주로 답변해 줘.
>
> **주의사항 (Warning):**
>
> - Server Component와 Client Component의 경계를 명확히 구분해서 작성해. 에러가 나면 안 돼.

## 💡 작성자 코멘트 (Insight)

Tambo SDK의 진정한 강력함은 **'Zod 스키마 기반의 정교한 데이터 타이핑'**에 있습니다. 프론트엔드 개발자는 AI가 컴포넌트에 주입할 Props를 완벽하게 예측하고 강제할 수 있기 때문에, 런타임 렌더링 에러에 대한 두려움 없이 UI/UX 디자인에만 온전히 집중할 수 있습니다.

위 프롬프트를 응용하여 `ChartCard`, `PaymentForm` 등 여러분의 비즈니스 도메인에 특화된 컴포넌트들을 레지스트리에 계속 추가해 보세요. 시스템이 알아서 문맥을 파악하고 가장 적절한 인터페이스를 유저에게 던져줄 것입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: Vercel AI SDK의 `generateId`나 `streamUI`를 직접 쓰는 것과 무엇이 다른가요?**
  - A: Vercel AI SDK의 기본 기능만으로도 제너레이티브 UI 구현은 가능하지만, 도구 호출(Tool Calling)과 컴포넌트 매핑 처리를 위한 보일러플레이트 코드가 상당히 많이 요구됩니다. Tambo SDK는 이 복잡한 과정을 '레지스트리(Registry)' 패턴으로 추상화하여 개발 생산성을 압도적으로 높여줍니다.

- **Q: 기존 React/Next.js 프로젝트에 붙이기 어렵지 않나요?**
  - A: 전혀 어렵지 않습니다. `TamboProvider`로 앱을 감싸고, AI 응답 텍스트를 `GenerativeUI` 컴포넌트에 넘겨주기만 하면 내부적으로 컴포넌트 파싱과 렌더링을 자동으로 처리합니다.

## 🧬 프롬프트 해부 (Why it works?)

1.  **정확한 스택과 환경 지정:** Next.js App Router, TypeScript, pnpm 등 개발 환경을 명확히 하여 코딩 에이전트의 환각(Hallucination)과 호환성 충돌을 원천 차단했습니다.
2.  **논리적인 구조화 (Chain-of-Thought):** 패키지 설치부터 컴포넌트 구현, 레지스트리 등록, 최종 뷰 연동까지 프론트엔드 개발의 실제 흐름(Workflow)과 동일한 순서로 지시하여 완벽하게 작동하는 코드를 유도했습니다.
3.  **Zod 스키마 강제:** 제너레이티브 UI의 핵심인 데이터 무결성을 보장하기 위해 `zod` 라이브러리 활용을 명시적으로 요구하여 Props 에러를 예방했습니다.
