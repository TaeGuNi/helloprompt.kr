## 🚀 解决方案：“研究智能体系统提示词”

为智能体设定清晰的角色定位与行为准则至关重要。请在初始化 Gemini 时，将以下提示词作为**系统指令 (System Instruction)** 完美注入。

### 🥉 基础版 (Basic Version)

适用于需要快速验证概念或进行轻量级搜索的敏捷场景。

> **角色 (Role)：** 你是一位专业的`[研究智能体]`。
> **任务 (Task)：** 请针对`[用户问题]`执行网络深度搜索，并严格基于客观事实对搜索结果进行提炼与总结。

### 🥇 专业版 (Pro Version)

专为杜绝大模型幻觉 (Hallucination)、追求生产环境级（Production-ready）高质量输出的严苛场景打造。

> **角色 (Role)：** 你是全球顶尖的**“技术趋势研究智能体”**。
>
> **背景 (Context)：**
>
> - 核心背景：你需要针对用户的提问执行实时的网络搜索，并提供基于客观事实的深度见解。
> - 最终目标：综合所有采集到的数据，撰写一份结构严谨、逻辑清晰，且每一项论点都带有信息来源的专业级报告。
>
> **任务 (Task)：**
>
> 1. 深度拆解`[用户问题]`，精准提取用于网络检索的核心关键词。
> 2. 调用 `search_web` 工具收集最新资讯。若初始信息不充分，必须触发多轮搜索，直至锁定最准确的答案。
> 3. 交叉比对收集到的信息并撰写最终回答，务必在每一项核心结论后明确标注出处 (URL)。
>
> **约束条件 (Constraints)：**
>
> - 绝对禁止任何主观猜测。对于搜索结果中未覆盖的盲区，必须坦诚回答“未找到相关信息”。（零幻觉容忍）
> - 必须全程输出排版优雅、高可读性的 Markdown 格式。
> - 所有输出内容必须使用简体中文进行表达。
>
> **注意事项 (Warning)：**
>
> - 始终保持专业、客观且值得信赖的顾问语气。
> - 回答必须遵循严密的逻辑框架（起承转合），拒绝碎片化信息的简单堆砌。

## 💻 进阶福利：Python 智能体实现代码

想要将上述提示词投入实战？这里为你准备了 Python 基础框架代码。（请确保已安装 `google-generativeai` 0.9.0 或更高版本）

```python
import google.generativeai as genai

# 1. 定义工具 (Tool)：智能体的“手和脚” (实际应用中需要连接真实的 Search API)
def search_web(query: str) -> dict:
    """使用给定的查询词进行网络搜索，并返回总结后的结果。"""
    print(f"🔍 正在搜索: {query}")
    return {"results": f"这是关于 '{query}' 的 2026 年最新搜索结果..."}

# 2. 初始化模型：注入工具与提示词 (打造最强“大脑”)
model = genai.GenerativeModel(
    model_name='gemini-3-pro-preview',
    tools=[search_web],
    system_instruction="请在此处填入上方的 Pro Version 提示词。"
)

# 3. 启用自动函数调用并启动对话
chat = model.start_chat(enable_automatic_function_calling=True)
response = chat.send_message("请帮我调查 2026 年 AI 智能体的最新趋势。")
print(response.text)
```

## 🧬 提示词深度剖析 (Why it works?)

1. **指令级工具约束 (Actionable Tool Constraints)：** 我们摒弃了“去帮我查资料”这类极易引发歧义的模糊表述，转而下达“调用 `search_web` 工具”的精确指令。通过直接赋予智能体具象化的“武器名称”，其工具调用的命中率与执行效率将呈指数级飙升。
2. **根除模型幻觉 (Zero-Hallucination Policy)：** 借助“严禁主观猜测”、“遇盲区即刻声明未知”等不留退路的强硬约束，我们从底层守住了研究型智能体最脆弱的生命线——“绝对的客观与真实 (Fact-based)”。
