# 📝 사수 없이도 완벽한 코드 리뷰 & 리팩토링 가이드

- **🎯 추천 대상:** 주니어 개발자, 1~3년 차 백엔드/프론트엔드 엔지니어, 독학러
- **⏱️ 소요 시간:** 2시간 삽질 → 1분 컷
- **🤖 추천 모델:** Claude 3.5 Sonnet (코드 리뷰 최강자), GPT-4o

- ⭐ **난이도:** ⭐⭐☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"퇴근 시간은 다가오는데 에러 로그는 산더미 같고, 내 코드가 쓰레기 같지만 어디서부터 고쳐야 할지 막막할 때가 있지 않나요?"_

![🔥 주니어 개발자를 위한 사수급 코드 리뷰 & 리팩토링 프롬프트](/images/hooks/code-review.jpg)

사수에게 물어보기 눈치 보이는 자잘한 에러부터, "동작은 하는데 영 찝찝한" 스파게티 코드까지. 이제 AI를 당신의 전담 시니어 개발자로 활용해 보세요. 단순히 버그를 잡아주는 것을 넘어, 클린 코드 원칙에 입각한 실전 리팩토링 제안까지 받아낼 수 있습니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. AI에게 단순한 '코드 수정'이 아닌 '시니어 코드 리뷰어'라는 명확한 역할을 부여하세요.
2. 에러 로그와 현재 코드의 비즈니스 의도를 구체적인 컨텍스트로 제공하세요.
3. 결과물만 복사해 붙여넣지 말고, AI가 제시한 '수정 이유'를 분석하여 본인의 역량으로 흡수하세요.

---

## 🚀 해결책: "무자비한 시니어 개발자 프롬프트"

### 🥉 Basic Version (기본형)

빠르게 에러의 근본 원인만 파악하고 싶을 때 사용하세요.

> **역할:** 너는 10년 차 `[프론트엔드/백엔드]` 시니어 개발자야.
> **요청:** 아래 코드에서 발생하는 `[에러 내용]`의 원인을 분석하고 해결 코드를 제시해 줘.
> **코드:** `[여기에 코드 붙여넣기]`

### 🥇 Pro Version (전문가형)

단순한 버그 패치를 넘어, 아키텍처 개선과 클린 코드 기반의 리팩토링이 필요할 때 사용하세요.

> **역할 (Role):**
> 너는 FAANG 출신의 무자비하지만 압도적인 실력을 갖춘 15년 차 수석 소프트웨어 엔지니어(Staff Engineer)야. 가독성, 성능, 유지보수성에 대해서는 절대 타협하지 않는 깐깐한 성격이지.
> 
> **상황 (Context):**
> - 환경: `[React / Node.js / Python 등 사용 중인 기술 스택 입력]`
> - 목적: `[해당 코드가 수행해야 하는 핵심 비즈니스 로직 설명]`
> - 현재 문제: `[발생 중인 에러 로그 전문 또는 성능 저하 등 직면한 문제 상황]`
> 
> **요청 (Task):**
> 1. 제공된 코드를 철저히 분석하고 근본적인 문제점(버그 원인, 안티 패턴 등)을 날카롭게 지적해.
> 2. 클린 코드 원칙(SOLID, DRY 등)에 입각하여 코드를 전면 리팩토링해.
> 3. 수정이 완료된 전체 코드를 제공해.
> 4. 코드 리뷰 코멘트 형식으로 **'왜 이렇게 수정했는지'**에 대한 논리적인 이유를 3가지 포인트로 설명해.
> 
> **제약사항 (Constraints):**
> - 불필요한 서론이나 인사말은 생략하고 즉시 코드 리뷰를 시작해.
> - 최신 문법(ES6+ 등)과 해당 언어의 관용적(Idiomatic)인 디자인 패턴을 적극 활용해.
> - 에러 핸들링과 예외적인 엣지 케이스를 반드시 고려해.
> - 출력 결과는 읽기 편한 리스트 형태로 작성해 줘.
> 
> **주의사항 (Warning):**
> - 존재하지 않는 라이브러리나 메서드를 임의로 지어내지 마. (할루시네이션 절대 금지)
> 
> **코드 (Code):**
> `[여기에 리뷰받을 코드를 붙여넣으세요]`

## 💡 작성자 코멘트 (Insight)

이 프롬프트의 핵심은 AI에게 '무자비한 수석 엔지니어'라는 페르소나를 씌우고, **'왜 수정했는지 그 이유를 반드시 설명하도록 강제하는 것'**입니다. 

실무 현장에서 주니어 개발자들이 흔히 저지르는 치명적인 실수는 AI가 생성한 코드를 원인 분석 없이 그대로 복사하여 붙여넣기(Ctrl+C, Ctrl+V)하는 것입니다. 당장의 버그는 해결될지 몰라도, 장기적인 엔지니어링 역량은 단 1mm도 성장하지 않습니다. 

이 프롬프트는 AI를 단순한 코드 생성기가 아닌 "나를 멘토링하는 사수"로 탈바꿈시킵니다. AI가 지적하는 안티 패턴을 수용하고, 관용적인 코드 패턴(Idiomatic pattern)을 철저히 학습하세요. 특히 `Context` 항목에 코드를 작성할 당시의 의도(비즈니스 로직)를 상세히 기록하면, AI는 단순한 문법 오류 수정을 넘어 아키텍처 단위의 구조적 결함까지 짚어냅니다. 핑계 대지 말고, 이 냉철한 AI 사수에게 논리로 탈탈 털리며 깊이 있게 배우십시오. 그것이 주니어 티를 가장 빠르게 벗어나는 지름길입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 회사 내부 보안 코드를 그대로 입력해도 안전한가요?**
  - A: 절대 안 됩니다! API 키, 내부 IP 주소, 핵심 비즈니스 로직 등 민감한 정보는 반드시 더미(Dummy) 데이터로 마스킹하거나 변수명을 일반화한 뒤에 프롬프트에 입력해야 합니다. 보안은 그 어떤 편리함과도 타협할 수 없는 1순위 원칙입니다.
- **Q: AI가 제공한 코드를 실행했는데 작동하지 않거나 또 다른 에러가 납니다.**
  - A: 전형적인 할루시네이션(Hallucination) 현상입니다. 이럴 때는 당황하지 말고 새로 발생한 에러 로그를 그대로 복사하여 다시 질문하세요. "네가 준 코드를 실행하니 이런 에러가 발생하는데?"라고 추궁하면 AI가 컨텍스트를 재분석하여 올바른 수정본을 다시 제시합니다.
- **Q: 코드 리뷰와 리팩토링에 가장 특화된 AI 모델은 무엇인가요?**
  - A: 2026년 기준, 코딩 영역에서는 Claude 3.5 Sonnet 이상의 모델이 압도적인 코드 맥락 이해도와 리팩토링 능력을 자랑합니다. ChatGPT Plus(GPT-4o) 또한 실무에서 활용하기에 훌륭한 대안입니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **무자비한 페르소나 부여:** 일반적인 AI 챗봇은 사용자의 기분을 맞추기 위해 치명적인 구조적 결함도 적당히 넘어가며 작동만 되게끔 수정하는 경향이 있습니다. 깐깐한 수석 엔지니어 역할을 강제로 부여함으로써 타협 없는 날카로운 리뷰를 이끌어냅니다.
2. **Context 명시:** 컴파일러는 코드만 읽지만, 시니어 개발자는 '의도'를 읽습니다. 코드가 달성하고자 하는 비즈니스 목적을 설명하면 AI가 더 효율적인 접근 방식(최적화된 자료구조, 더 적절한 디자인 패턴)을 제안할 수 있는 맥락을 확보하게 됩니다.
3. **이유(Why) 요구:** 개발자의 성장을 담보하는 핵심 장치입니다. 결과 코드만 툭 던져주는 것을 방지하고, 코드 이면의 철학과 지식을 전달하도록 강제합니다.

## 📊 증명: Before & After

### ❌ Before (입력)

```javascript
// 유저 데이터를 가져와서 가공하는 코드 (스파게티 예시)
async function getU(id) {
  let res = await fetch('/api/user/' + id);
  let d = await res.json();
  if(d) {
    if(d.status === 'active') {
      let r = await fetch('/api/roles/' + d.roleId);
      let rd = await r.json();
      return { n: d.name, r: rd.name };
    } else {
      return null;
    }
  }
}
```

### ✅ After (결과)

```javascript
// AI 시니어 엔지니어의 리팩토링 결과
async function getActiveUserRoleSummary(userId) {
  try {
    const userResponse = await fetch(`/api/user/${userId}`);
    if (!userResponse.ok) throw new Error('Failed to fetch user');
    
    const user = await userResponse.json();
    
    // 조기 반환(Early Return) 패턴 적용으로 중첩 if문 제거
    if (!user || user.status !== 'active') {
      return null; 
    }
    const roleResponse = await fetch(`/api/roles/${user.roleId}`);
    if (!roleResponse.ok) throw new Error('Failed to fetch role');
    
    const role = await roleResponse.json();
    // 명확한 변수명 사용 (n, r -> name, roleName)
    return { 
      name: user.name, 
      roleName: role.name 
    };
  } catch (error) {
    console.error(`[getActiveUserRoleSummary] Error for userId ${userId}:`, error);
    throw error; // 에러 은닉 방지
  }
}
```

*(AI 코멘트: 중첩된 조건문을 Early Return으로 풀어 가독성을 높였고, 변수명을 직관적으로 수정했으며, 누락된 에러 핸들링을 추가했습니다.)*
