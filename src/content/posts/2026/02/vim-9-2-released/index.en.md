Highly recommended when you need a deep dive beyond a simple summary, including the impact on your existing projects (Breaking Changes) and specific migration strategies.

> **Role:** You are a veteran DevOps engineer with 20 years of experience and a top-tier technical document master.
>
> **Context:**
>
> - Background: Vim 9.2 (or `[Specific Technology]`) has just had a major release, and I need to review whether to apply it to my current production environment.
> - Goal: To perfectly understand the core of the release notes and pre-analyze any critical impacts on the existing environment.
>
> **Task:**
>
> 1. Select the 5 most important updates based on the content of the `[Original Release Notes]`.
> 2. Analyze the **risk factors (Breaking Changes)** that this update might have on the existing environment (version compatibility, etc.) with specific evidence.
> 3. Provide detailed **example code and application guides** so that newly added useful features can be used immediately in practice.
> 4. Evaluate the practical benefits of this update from the perspective of a `[My Main Language/Environment]` developer.
>
> **Constraints:**
>
> - Maximize readability by appropriately mixing Markdown bullet points and a structured format.
> - If technical terms appear, add a brief and clear explanation in parentheses.
> - Thoroughly exclude any speculative information that is not certain, and answer based only on the provided original release note data.
>
> **Warning:**
>
> - Never arbitrarily make up content not present in the original text (Hallucinations are strictly forbidden).

The true destructive power of this prompt is that it doesn't stop at the one-dimensional task of mechanically translating massive English text. Don't just enter a vague term like 'Developer'. In this variable field, try entering your specific job role and the detailed stack you are currently handling as specifically as possible, such as **'Linux server administrator handling high-volume traffic'**, **'React-based frontend developer sensitive to performance optimization'**, or **'Java backend engineer maintaining financial legacy systems'**.

Furthermore, this prompt strictly controls the AI's behavior through constraints. The command **"Thoroughly exclude any speculative information that is not certain"** fundamentally blocks hallucinations, a chronic problem with AI. This is because making up technical flaws or imaginary features not present in the original release notes can actually become a deadly poison in practice.
