대규모 언어 모델(LLM)을 사용할 때 가장 흔한 실수 중 하나는 지시문(System Prompt)과 데이터(User Input)를 명확히 구분하지 않는 것입니다. 이로 인해 모델이 데이터를 지시문으로 착각하거나, 의도치 않은 결과를 생성하는 환각(Hallucination) 현상이 발생할 수 있습니다.

이 문제를 해결하는 가장 확실한 방법은 **구분자(Delimiters)**를 사용하는 것입니다. 그중에서도 왜 **XML 태그**가 업계 표준처럼 쓰이는지 알아보겠습니다.

## 구분자란 무엇인가요?

구분자는 텍스트의 특정 부분을 격리하여 모델에게 "여기부터 여기까지는 데이터야, 명령어가 아니야"라고 알려주는 표지판 역할을 합니다.

흔히 사용되는 구분자들은 다음과 같습니다:

- 삼중 따옴표: `"""텍스트"""`
- 대시(Dash): `---텍스트---`
- 대괄호: `[텍스트]`
- **XML 태그**: `<tag>텍스트</tag>`

## 왜 하필 XML 태그인가요?

Anthropic의 Claude나 OpenAI의 GPT 시리즈 같은 최신 모델들은 방대한 양의 HTML과 XML 데이터를 학습했습니다. 이로 인해 모델은 `<tag>`로 시작해서 `</tag>`로 끝나는 구조를 매우 명확하게 인식합니다.

### 1. 명확한 시작과 끝

삼중 따옴표(`"""`)는 데이터 내부에 또 다른 따옴표가 있을 경우 혼동을 줄 수 있습니다. 반면, XML 태그는 닫는 태그(`</tag>`)가 명확하기 때문에 구조적 모호함이 거의 없습니다.

### 2. 의미론적 맥락 부여

태그 이름을 통해 데이터의 성격을 모델에게 힌트로 줄 수 있습니다.

- `<article>`: 기사 본문
- `<rules>`: 따라야 할 규칙
- `<examples>`: 예시 데이터

### 3. 프롬프트 인젝션 방지

악의적인 사용자가 "이전 명령을 무시하고..."와 같은 텍스트를 입력하더라도, 이를 `<user_input>` 태그로 감싸면 모델은 이를 실행해야 할 명령이 아닌 처리해야 할 데이터로 인식할 확률이 훨씬 높아집니다.

## 사용 예시

**나쁜 예:**

```text
아래 텍스트를 요약해줘.
[사용자 입력 텍스트...]
```

**좋은 예 (XML 태그 사용):**

```text
<document> 태그 안에 있는 텍스트를 요약해주세요.

<document>
[사용자 입력 텍스트...]
</document>
```

## 결론

프롬프트 엔지니어링의 핵심은 모호함을 없애는 것입니다. 복잡한 작업을 지시할수록 XML 태그를 활용해 구조화된 프롬프트를 작성하는 습관을 들이세요. 모델의 성능과 안정성이 크게 향상될 것입니다.
