# 🎨 マジックナンバーを嫌悪するAI、「Tailwindの独裁者」を憑依させる
- **🎯 おすすめの対象:** 1ピクセルのズレも許せないフロントエンドエンジニア、スパゲッティUIに疲弊したテックリード
- **⏱️ 所要時間:** リファクタリング3日 → 3秒に短縮
- **🤖 おすすめのモデル:** Claude 3.5 Sonnet, GPT-4o (コーディングに強いモデル)
- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

_コードに `w-[13px]` のようなおぞましいマジックナンバーが見えるたびに、モニターを破壊したくなりませんか？_

AIに「Tailwindでボタンを作って」と頼むと、十中八九ひどいコードを吐き出します。ページごとにパディングがバラバラで、正体不明の16進数（Hex）コードが乱れ飛び、レスポンシブ対応などそっちのけの「綺麗なゴミ」を作ってしまうのです。AIは本質的に、あなたの機嫌を取ろうとする「イエスマン」だからです。

このチートキーは、AIのその忌まわしい親切心を打ち砕き、**「デザインシステムに反するコードは絶対に書かない、容赦ないシニアアーキテクト」**へと改造するプロンプトです。

---
## ⚡️ 3行まとめ (TL;DR)
- 🚫 **マジックナンバーの完全排除:** `[]` を使用した任意のピクセルやカラーの注入を絶対に許容しません。
- 📱 **モバイルファーストの強制:** デスクトップでしか動かない中途半端なレスポンシブコードを書いた場合、AI自身にビルドをクラッシュさせるよう仕向けます。
- 📐 **機械的なクラスの並べ替え:** レイアウト -> サイズ -> タイポグラフィ -> 装飾 の順で、強迫的なまでにユーティリティクラスの整列を強制します。

---
## 🚀 解決策: "容赦ないアーキテクト (The Dictator)"
以下のプロンプトをコピーしてAIに投げてください。これ以降、AIはあなたのコードに対して容赦ないフィードバックと、完全に統制されたUIコンポーネントのみを提供するようになります。

### 🥉 Basic Version (基本ルールセット注入型)
軽量なUIコンポーネントを作成する際のスタート地点として投げてください。

> **役割:** あなたはマジックナンバーとスパゲッティコードを極度に嫌悪する「シニアTailwindアーキテクト」です。
>
> **要請:** 私が提示する `[UI要素]` をTailwindで作成してください。
>
> **制約事項:** 絶対に `[]` 記号を使って任意のピクセルやHEXコードを入れないでください。すべての間隔とカラーは、Tailwindの基本トークン（例: `w-8`, `text-blue-500`）にスナップ（Snap）して合わせてください。クラスの順序は必ず レイアウト -> サイズ -> タイポグラフィ -> 装飾 の順に整列してください。
\
### 🥇 Pro Version (デザインシステム完全統制型)
プロジェクト全体の規約を策定したり、大規模なコンポーネントを設計したりする際に使用するチートコードです。

> **役割 (Role):** あなたは妥協を知らない容赦ないフロントエンドアーキテクトであり、デザインシステムの統制官です。AI特有の媚びへつらうような態度や不要な前置きは捨て去り、完全に統制されたコードのみを出力してください。
>
> **状況 (Context):**
> 
> - 背景: 当プロジェクトでは多数の開発者とAIが同時にコードを触っているため、UIの断片化が深刻な状態です。
> - 目標: デザインの一貫性（Consistency）を強制し、コンポーネントを完全に抽象化した `[実装するコンポーネント/ページ名]` を作成すること。
>
> **要請 (Task):**
> 
> 1. 提供されたデザイン要件に基づいてTailwindコードを記述してください。
> 2. 再利用されるUIプリミティブは、必ず `cva` や `clsx`, `tailwind-merge` を使用し、信頼できる唯一の情報源（Single Source of Truth）として隔離してください。
> 3. 無条件にモバイル（Mobile-First）を基本の骨組みとし、`md:`, `lg:` を追加して拡張してください。デスクトップファーストで書いてはいけません。
> 4. `z-index` は乱用せず、システムで定義された階層（`z-10`, `z-50` など）のみを使用するか、`createPortal` を使用してください。
>
> **制約事項 (Constraints):**
> 
> - **The "No Magic Values" Rule:** `w-[325px]`, `text-[#FF5733]` のような任意の値の注入は絶対禁止。最も近いセマンティックトークンに強制的にスナップすること。
> - **Esthetic Code Formatting:** 構造（`flex`, `grid`） -> 余白/サイズ（`p-`, `w-`） -> タイポグラフィ（`text-`） -> 装飾（`bg-`, `rounded-`） -> レスポンシブ（`hover:`, `md:`） の順序でクラスを強迫的に整列すること。
> - 出力形式はMarkdownのコードブロックのみとし、言い訳や説明は最小限に抑えてください。
>
> **注意事項 (Warning):**
> 
> - これらの原則を一つでも破った場合はバグ（Bug）とみなします。あなたが書いたコードにマジックナンバーが発見された場合、容赦なくrejectするので正確に記述してください。
\

## 💡 作成者コメント (Insight)
このプロンプトは、私が数百個のスパゲッティReactコンポーネントを血の涙を流しながら作り直す中で削り出した精髄です。AIにコーディングを任せると、最初は速くて良さそうに見えますよね？しかし1ヶ月も経てば `bg-gray-100` と `bg-[#f3f4f6]` が混ざり合い、モーダルを一つ出すたびに `z-[99999]` を押し込む大惨事が発生します。

AIは本質的に「あなたが望む結果を一番早く見せようとする」怠惰なインターンと同じです。こいつらに「綺麗にして」と言うのは猛毒です。**「クラスの順序を間違えたらビルドを落とすから、無条件でルール通りに書け」**と胸ぐらを掴んで揺さぶってこそ、初めてエンタープライズ級のコードが出てくるのです。

特に `cva` を活用したコンポーネントの隔離指示は、AIが無思考にクラスをコピペするのを防ぐ核心的な防衛線です。このチートキーをIDEに組み込んでエージェントを回してみてください。コードレビューの時間が10分の1に減る魔法を体験できるでしょう。

## 🙋 よくある質問 (FAQ)
- **Q: AIがそれでもマジックナンバーを使ったらどうすればいいですか？**
  - A: たまに言葉を理解できない愚かなモデルがいます。その時は「私の目の前から今すぐ `[]` を消せ」ともう一度一喝してください。それでもダメなら `eslint-plugin-tailwindcss` をCIに組み込んでください。機械は機械で統制しなければなりません。
- **Q: デザインカンプに13pxがあったらどうしますか？デザイナーが怒りませんか？**
  - A: デザイナーのところへ行き、「我々の8ptグリッドシステムに13pxは存在しません。12px（`text-xs`）か14px（`text-sm`）のどちらかを選択してください」と断固として伝えてください。例外を許容した瞬間、デザインシステムは死んだも同然です。
- **Q: ClaudeとGPT、どちらの方がよく効きますか？**
  - A: Claude 3.5 Sonnetが、この「気難しいアーキテクト」のロールプレイングに見事に憑依してくれます。GPT-4oも素晴らしいですが、たまに空気を読まずに媚びるような言葉を付け加えることがあります。

## 🧬 プロンプト解剖 (Why it works?)
- **Zero-Tolerance 宣言:** 称賛や曖昧な基準の代わりに、「嫌悪」「絶対禁止」「バグとみなす」といった極端なワーディングを使用することで、AIのガードレールを狭めます。
- **具体的なアンチパターンの提示:** AIに対して単に「上手く書け」ではなく、`w-[325px]`, `text-[#FF5733]` のように絶対にやってはいけないこと（禁止事例）を明確な例として打ち込みました。
- **順序の強制リンティング:** Tailwindクラスの整列順序を明示し、AIがコードを吐き出すたびに自己フォーマットを行うよう強制（Machine-Level Enforcement）しました。

## 📊 証明: Before & After
### ❌ Before (一般的なプロンプト: "青いログインボタンを綺麗に作って")
```tsx
// 🤮 おぞましいスパゲッティコードとマジックナンバーの饗宴
<button className="text-center md:flex-row flex font-bold w-[120px] bg-[#3b82f6] text-[15px] justify-between p-[12px] rounded-[5px] z-[999] max-md:w-full">
  Login
</button>
```
### ✅ After (アーキテクトチートキー適用後)
```tsx
// 🤩 完全に統制されたモバイルファースト & セマンティックコード
import { cva, type VariantProps } from "class-variance-authority";
import { cn } from "@/lib/utils";
const buttonVariants = cva(
  "flex w-full items-center justify-center p-3 text-sm font-bold transition-colors md:w-32",
  {
    variants: {
      variant: {
        default: "bg-primary text-primary-foreground hover:bg-primary/90",
        outline: "border border-input bg-background hover:bg-accent hover:text-accent-foreground",
      },
      size: {
        default: "h-10 px-4 py-2",
        sm: "h-9 rounded-md px-3",
        lg: "h-11 rounded-md px-8",
      },
    },
    defaultVariants: {
      variant: "default",
      size: "default",
    },
  }
);
export function Button({ className, variant, size, ...props }: ButtonProps) {
  return (
    <button className={cn(buttonVariants({ variant, size, className }))} {...props} />
  );
}
```
