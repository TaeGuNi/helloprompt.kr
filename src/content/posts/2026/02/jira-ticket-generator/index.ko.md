## 🎫 Jira 티켓 자동 생성: 기획서만 넣으면 개발 티켓 10개가 뚝딱

- **🎯 추천 대상:** 하루 종일 "티켓 만들어주세요" 소리 듣는 PM/PO, 백로그 정리가 귀찮은 스크럼 마스터
- **⏱️ 소요 시간:** 1분 (기획서 입력 → CSV 변환)
- **🤖 추천 모델:** Claude 3.5 Sonnet (문서 구조화 및 논리적 분할 능력 탁월)

- ⭐ **난이도:** ⭐☆☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"기획서는 다 썼고... 이제 지라(Jira)에 하나씩 옮겨 적어야지. 하아..."_

완성된 기획서를 열어두고 지라에 끊임없이 복사하고 붙여넣는 단순 노동, 대체 언제까지 사람이 직접 해야 할까요? 개발자와의 명확한 소통을 위해 기획안을 티켓으로 잘게 쪼개는 작업은 필수적입니다. 하지만 그 과정 자체는 100% 자동화가 가능한 **기계적인 업무**에 불과합니다.

이제 AI에게 기획서를 통째로 던져주고, **"개발자가 즉시 작업에 착수할 수 있는 태스크(Task) 단위로 쪼개줘"**라고 지시해 보세요. 하루 종일 여러분의 진을 빼놓던 백로그 생성 작업이 단 1분 만에 끝납니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. 완성된 기획서(PRD)의 전체 텍스트를 그대로 복사합니다.
2. AI에게 "이 내용을 Epic, Story, Task로 분류하고 Jira Import용 CSV 포맷으로 만들어 줘"라고 지시합니다.
3. 생성된 텍스트를 파일로 저장한 뒤, 지라의 'CSV 가져오기' 기능으로 업로드하여 수십 개의 티켓을 일괄 생성합니다.

## 🚀 해결책: "Jira Backlog Generator"

### 🥉 Basic Version (기본형)

단순한 할 일 목록(To-Do List)을 빠르게 뽑아내어 전체적인 작업 볼륨을 가늠하고 싶을 때 사용해 보세요.

> **역할 (Role):** 너는 `[시니어 기획자 겸 스크럼 마스터]`야.
> 
> **기획서 (Context):** `[여기에 기획안 전체 내용을 붙여넣으세요]`
> 
> **요청 (Task):** 이 기획 내용을 개발자가 즉시 작업할 수 있는 **상세 업무(Task)** 단위로 잘게 쪼개어 체크리스트로 만들어 줘. 이때 프론트엔드(FE)와 백엔드(BE) 작업 범위를 명확하게 구분해 줘.

### 🥇 Pro Version (전문가형)

지라에 즉시 업로드할 수 있는 완벽한 구조의 CSV 데이터를 추출할 때 사용하는 실전용 프롬프트입니다. 까다로운 AC(인수 조건)까지 AI가 꼼꼼하게 작성해 줍니다.

> **역할 (Role):** 너는 애자일(Agile) 프로세스에 능통한 시니어 스크럼 마스터(Scrum Master)이자 테크니컬 PM이야.
>
> **상황 (Context):**
>
> - 배경: 새로운 기능에 대한 기획안(PRD) 작성이 완료되었으며, 이를 바탕으로 스프린트 플래닝을 진행해야 해.
> - 목표: 기획 내용을 Jira에 즉시 등록할 수 있는 완벽한 구조의 CSV 파일 데이터로 변환하는 것.
>
> **요청 (Task):**
>
> 1. 아래 제공된 기획서를 분석하여 개발에 필요한 모든 티켓을 **Jira CSV 포맷**으로 작성해 줘.
> 2. 다음 컬럼을 반드시 순서대로 포함해야 해: `Issue Type`, `Summary`, `Description`, `Priority`, `Assignee`
> 3. `Assignee` 컬럼은 빈칸으로 남겨둬.
> 4. `Description`에는 개발자가 직접 테스트할 수 있도록 **Given-When-Then 형식의 인수 조건(Acceptance Criteria, AC)**을 반드시 포함해 줘.
>
> **기획서 (PRD):**
> `[여기에 기획안 전체 내용을 붙여넣으세요]`
>
> **제약사항 (Constraints):**
>
> - 계층 구조는 반드시 **1. Epic -> 2. Story -> 3. Sub-task(FE/BE 구분)** 순서로 명확하게 떨어져야 해.
> - 출력 형식은 마크다운 표(Table)가 아닌, 복사해서 엑셀이나 메모장에 붙여넣기 좋은 **쉼표로 구분된 텍스트(CSV 원시 데이터)** 형태로만 출력해 줘. 결과물은 코드 블록 안에 담아 줘.
>
> **주의사항 (Warning):**
>
> - 기획서에 없는 기능이나 정책을 임의로 지어내지 마. 내용이 불확실하거나 누락된 부분은 Description에 **"[확인 필요]"**라고 명확히 명시해.

## 💡 작성자 코멘트 (Insight)

실무에서 개발자들이 진정으로 환호하는 기획 문서는 화려한 디자인이 입혀진 화면 정의서가 아닙니다. 바로 **"AC(Acceptance Criteria, 인수 조건)가 명확하게 정의된 티켓"**입니다. 이 프롬프트가 지닌 강력함의 핵심은 단순히 텍스트를 작업 단위로 분할하는 것을 넘어, AI가 티켓의 `Description` 영역에 **Given-When-Then** 형식의 행동 주도(BDD) 인수 조건을 작성하도록 강력하게 **강제(Constraint)**했다는 점에 있습니다.

실제 현업 스쿼드에서 이 자동화 방식을 도입해 본 결과, 스프린트 플래닝 시간마다 반복되던 *"이 버튼 누르면 구체적으로 어떻게 동작하나요?"* 혹은 *"네트워크 오류 시 예외 처리는 어떻게 하죠?"*와 같은 핑퐁 질문이 획기적으로 줄어들었습니다. 지라 티켓의 퀄리티가 곧 전체 스프린트의 개발 속도를 결정짓습니다. AI를 통해 기계적인 문서 변환 작업은 단축하고, PM으로서 본연의 업무인 '정책의 빈틈을 메우는 일'에 집중해 보세요.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: AI가 만들어준 결과를 어떻게 지라(Jira) 시스템에 업로드하나요?**
  - A: AI가 코드 블록으로 출력한 CSV 원시 데이터를 그대로 복사하여 메모장이나 텍스트 편집기에 붙여넣은 뒤, `jira_tickets.csv`와 같은 이름으로 확장자를 지정해 저장하세요. 이후 지라(Jira) 상단 메뉴에서 **[이슈(Issues)] → [CSV에서 이슈 가져오기(Import issues from CSV)]** 기능을 선택하여 해당 파일을 업로드하기만 하면 수십 개의 티켓이 계층 구조에 맞게 일괄 생성됩니다.

- **Q: 기획서 내용이 너무 방대해서 AI의 컨텍스트 토큰 제한에 걸리면 어떻게 하나요?**
  - A: 문서 전체를 한 번에 밀어 넣기보다는, 에픽(Epic)이나 핵심 기능 단위(예: '로그인/회원가입 시스템', '결제 모듈 파트')로 쪼개어 여러 번에 걸쳐 프롬프트를 실행하는 방식을 권장합니다. 이렇게 청크(Chunk)를 나누어 입력하면 훨씬 더 디테일하고 누락 없는 정확한 티켓이 생성됩니다.

- **Q: 커스텀 필드인 태그(Labels)나 스토리 포인트(Story Points)도 한 번에 자동으로 넣을 수 있나요?**
  - A: 물론입니다! 프롬프트의 '요청 (Task)' 영역에 *"Labels 컬럼을 추가하고, 프론트엔드 작업에는 'frontend', 백엔드에는 'backend' 태그를 각각 달아줘. 또한 Story Points 컬럼을 추가해 예상 구현 난이도에 따라 피보나치수열(1, 2, 3, 5, 8) 중 하나를 배정해 줘"*라고 구체적인 추가 지시를 내리면 완벽하게 반영된 결과물을 얻을 수 있습니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **계층 구조(Hierarchy)의 엄격한 강제:** **Epic → Story → Sub-task**로 이어지는 애자일(Agile) 프레임워크의 표준 구조를 명시했습니다. 이를 통해 AI가 맥락 없이 작업을 파편화하거나 엉뚱한 단위로 티켓을 남발하는 현상을 원천적으로 차단했습니다.
2. **AC(인수 조건) 작성 자동화:** 개발 산출물의 품질을 좌우하는 BDD(Behavior-Driven Development) 기반의 **Given-When-Then** 문법을 강제했습니다. 이로써 단순한 '할 일 목록'을 넘어, QA 테스트 케이스로도 즉시 활용할 수 있는 실용성 높은 티켓이 완성됩니다.
3. **데이터 포맷 최적화(CSV 지향):** 시각적으로만 보기 좋은 마크다운 표(Table) 대신, 시스템 친화적인 **CSV 원시 데이터 포맷**을 명시적으로 요구했습니다. 사용자가 결과를 복사해 파일로 저장한 뒤 즉시 지라에 임포트(Import) 할 수 있도록 실무 UX를 완벽하게 고려한 설계입니다.

프롬프트 적용 후, 애자일 원칙에 기반한 완벽한 계층 구조와 테스트 가능한 인수 조건(AC)이 포함된 데이터가 단 1분 만에 도출됩니다.

```csv
Sub-task,[BE] 장바구니 추가 API 구현 (POST /cart),"**Given** 유효한 세션이 있을 때, **When** POST /cart 요청이 들어오면, **Then** DB에 데이터를 저장하고 200 OK를 반환해야 한다.",High,
```

## 🎯 결론

유능한 기획자나 PM의 진짜 실력은 '기획서 문서의 두께'에서 나오는 것이 아닙니다. 바로 **'개발자가 막힘없이 코딩에 집중할 수 있도록 작업을 명확하게 쪼개주는 능력'**에 있습니다. 시간만 축내던 번거롭고 반복적인 문서 구조화 작업은 이제 똑똑한 AI 비서에게 전면 위임하세요.

잘 작성된 기획서만 프롬프트에 던져주면, 수십 개의 티켓이 담긴 백로그가 완벽하게 정리되어 나옵니다. 이제 끝없는 지라 티켓 생성 스트레스에서 벗어나, 가장 먼저 노트북을 덮고 여유로운 칼퇴를 즐기시길 바랍니다! 🍷
