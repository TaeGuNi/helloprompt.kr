# 📝 Освоение извлечения структурированных данных с помощью Google LangExtract

- **🎯 Рекомендуется для:** Data-инженеров, AI-разработчиков, Backend-специалистов
- **⏱️ Экономия времени:** С часов отладки Regex → до 5 минут на написание схемы
- **🤖 Рекомендуемая модель:** Gemini 1.5 Pro / Flash

- ⭐ **Сложность:** ⭐⭐⭐☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Применимость:** ⭐⭐⭐⭐⭐

> _"Вы всё ещё пытаетесь вытащить структурированные данные из LLM с помощью хрупких регулярок и умоляющих промптов вроде «пожалуйста, верни только валидный JSON»?"_

Хотя большие языковые модели (LLM) великолепно справляются с генерацией текстов или кода, они часто спотыкаются на задаче, критически важной для реального бизнеса: **надежном извлечении структурированных данных**. Заставить ИИ стабильно возвращать точный JSON, не добавляя лишнего текста (галлюцинаций) и не нарушая схему, всегда было настоящей головной болью.

Но теперь правила игры изменились. Google представила **LangExtract** — мощную Python-библиотеку, которая решает проблему парсинга на корню. В этом руководстве мы разберем, как LangExtract превращает хаотичные неструктурированные тексты в строгие объекты данных, экономя ваши нервы и часы на отладку.

---

## ⚡️ 3 кратких вывода (TL;DR)

1. **Строгая типизация с Pydantic:** Вы определяете схему, а ИИ возвращает данные ровно в этом формате. Никаких сюрпризов.
2. **Встроенная самокоррекция (Self-Correction):** Если модель ошибается в формате, LangExtract автоматически отправляет ошибку обратно для исправления "под капотом".
3. **Нативная интеграция с Gemini:** Инструмент идеально оптимизирован для молниеносной работы с Function Calling в моделях Gemini 1.5.

## 🚀 Решение: "Промпт для LangExtract (Code-First)"

### 🥉 Basic Version (Базовая версия)

Идеально подходит для быстрых задач, где схема данных минимальна и прямолинейна.

> **Роль:** Ты — эксперт по извлечению данных.
>
> **Задача:** Извлеки информацию о `[основной сущности]` из предоставленного текста и верни данные строго в соответствии с переданной JSON-схемой.

<br>

### 🥇 Pro Version (Продвинутая версия)

Когда входящие данные запутаны, содержат много вложенностей, неформальную речь или пропущенные значения, используйте этот детализированный подход.

> **Роль (Role):** Ты — Senior Data Engineer. Твоя задача — извлечь структурированную информацию из `[Тип документа: e-mail, чек, юридический договор]`.
>
> **Контекст (Context):**
>
> - Входящие данные часто содержат опечатки, сленг или неформальную речь.
> - Нам нужно преобразовать их в кристально чистый формат для записи в реляционную базу данных.
>
> **Задача (Task):**
>
> 1. Проанализируй текст и извлеки следующие сущности: `[список полей, например: тема встречи, участники, время начала]`.
> 2. Если какое-либо поле отсутствует в тексте, явно верни `null`. Категорически запрещено придумывать или домысливать данные.
> 3. Любые упоминания времени и дат преобразуй в строгий формат `ISO 8601`.
>
> **Ограничения (Constraints):**
>
> - Формат вывода: Строго JSON, соответствующий предоставленной схеме Pydantic.
> - Никаких вступительных или заключительных слов (например, "Вот ваш JSON:"), только сырой валидный JSON.
>
> **Предупреждение (Warning):**
>
> - Если информация противоречива или не поддается однозначному извлечению, оставь поле пустым. Точность важнее полноты.

## 💡 Авторский инсайт (Insight)

Раньше я тратил дни на написание многоэтажных промптов и "костылей" на Python, просто чтобы заставить RAG-систему правильно понимать параметры поискового запроса пользователя. Если модель возвращала `{'name': 'John'}` вместо `{'Name': 'John'}`, весь пайплайн мог рухнуть в production.

LangExtract меняет всё, внедряя подход **Code-First**. Вместо того чтобы уговаривать модель на естественном языке, мы просто скармливаем ей класс `Pydantic`. Но настоящая киллер-фича здесь — это **механизм самокоррекции (Self-Correction)**. Если ИИ вдруг выдал невалидный JSON, LangExtract ловит ошибку валидации на уровне Python, отправляет её обратно модели и говорит: "Ты ошибся в типе поля `start_time`, исправь". Это делает пайплайны извлечения невероятно надежными и отказоустойчивыми.

## 🙋 Часто задаваемые вопросы (FAQ)

- **Q: Работает ли LangExtract с моделями не от Google (например, OpenAI GPT-4o)?**
  - A: LangExtract изначально разработан и глубоко оптимизирован под инфраструктуру Vertex AI и Function Calling моделей Gemini. Если ваш стек построен на OpenAI, лучше использовать связку LangChain + Pydantic (с методом `with_structured_output`).

- **Q: Что делать, если исходный текст слишком большой (например, контракт на 100 страниц)?**
  - A: У Gemini 1.5 Pro гигантское контекстное окно (до 2 млн токенов). Поэтому LangExtract легко "проглотит" весь документ за один раз без необходимости вручную строить сложные пайплайны разбиения текста на чанки (chunking).

- **Q: Это платное удовольствие?**
  - A: Сама библиотека LangExtract — это open-source инструмент. Вы платите только за токены API, используемые при обращении к моделям Gemini. При этом у Gemini 1.5 Flash есть весьма щедрый бесплатный уровень (Free Tier), которого с головой хватит для тестирования и пет-проектов.

## 🧬 Анатомия промпта (Why it works?)

1.  **Native Function Calling:** LangExtract не пытается парсить текст обычными регулярками. Он использует способность современных LLM нативно вызывать функции (Tools), заставляя модель возвращать аргументы строго по заданным типам.
2.  **Жесткая схема Pydantic:** Использование Pydantic в качестве контракта гарантирует, что поле, ожидающее тип `datetime`, никогда не получит в ответ невнятную строку вроде "завтра после обеда". Валидация происходит на уровне кода.

## 📊 Доказательство: Before & After

### ❌ Before (Традиционный подход)

```text
Prompt:
Извлеки данные из письма: "Завтра в 14:00 встречаемся по маркетингу. Будут Иван и Мария."
ВЕРНИ ТОЛЬКО JSON: {"topic": "", "participants": [], "time": ""}

Result:
Вот ваш JSON:
{
  "topic": "маркетинг",
  "participants": ["Иван", "Мария"],
  "time": "завтра 14:00" // Ошибка: неверный формат даты! Код сломается.
}
```

### ✅ After (Использование LangExtract)

```python
from google.langextract import DataExtractor
from pydantic import BaseModel
from datetime import datetime

# 1. Определяем строгую схему
class Meeting(BaseModel):
    topic: str
    participants: list[str]
    time: datetime

# 2. Инициализируем экстрактор
extractor = DataExtractor(model="gemini-1.5-flash")

# 3. Извлекаем данные (ИИ сам приведет время к datetime)
result = extractor.extract(
    Meeting,
    "Завтра в 14:00 встречаемся по маркетингу. Будут Иван и Мария."
)

print(result)
# Гарантированный результат в виде типизированного объекта Python:
# Meeting(topic='маркетинг', participants=['Иван', 'Мария'], time=datetime.datetime(2026, 2, 22, 14, 0))
```
