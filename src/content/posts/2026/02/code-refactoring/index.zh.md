## 📝 我的代码是垃圾？！整洁代码重构教练

- **🎯 推荐受众:** 开发者、初中级程序员、代码审查员
- **⏱️ 所需时间:** 5分钟 → 缩短至 1分钟
- **🤖 推荐模型:** 所有对话型 AI (ChatGPT, Claude, Gemini 等)

- ⭐ **难度:** ⭐⭐☆☆☆
- ⚡️ **有效性:** ⭐⭐⭐⭐⭐
- 🚀 **实用性:** ⭐⭐⭐⭐☆

> _“先让它跑起来，以后再重构。”——当你说出这句话时，那段代码就注定沦为一座无人敢碰的遗迹。_

“意大利面条式代码”（Spaghetti Code）是所有开发者的噩梦。糟糕的代码可读性不仅会让 Bug 隐匿无形、难以排查，更会让后续的新功能开发举步维艰。本篇提示词将直接召唤罗伯特·C·马丁（《整洁代码》作者）“附体”，帮你彻底清洗那些惨不忍睹的混乱逻辑，让你的代码脱胎换骨，重新变得清爽、优雅且易于维护。

---

## ⚡️ 3句话总结 (TL;DR)

1. **消除重复与函数拆分 (Extract Method):** 将臃肿复杂的逻辑拆解为遵循单一职责原则的精简函数。
2. **重塑命名规范 (Naming Convention):** 彻底告别含义不明的 `a`, `b`, `calc`，换上直观且高度表意的变量与函数名。
3. **现代化语法升级 (Modern Syntax):** 自动应用最新的语言特性，全面提升代码的执行效率与优雅度。

## 🚀 解决方案："整洁代码重构教练"

### 🥉 Basic Version (基础版)

当你时间紧迫，只需要快速拿到重构结果时使用。

> **角色:** 你是 `[整洁代码布道者兼首席架构师]`。
> **请求:** 请把以下仅能勉强运行的凌乱代码，重构为“易读且易于维护”的整洁代码，并详细解释修改原因。

### 🥇 Pro Version (专家版)

当你需要深入理解重构细节、全面提升代码整体质量时使用。

> **角色 (Role):** 你是一位精通《整洁代码 (Clean Code)》核心理念的资深布道者兼首席架构师。
>
> **情况 (Context):**
>
> - **背景**：我的代码目前虽然勉强能跑，但由于种种历史原因，它已经沦为一团难以阅读和扩展的“脏代码”（Dirty Code）。
> - **目标**：请帮我对其进行深度重构，使其重新具备极高的可维护性与可读性。
>
> **请求 (Task):**
>
> 1. **[命名优化]**：将所有变量名和函数名重构为直观、高度表意且符合命名规范的名称（例如：将 `a` 修改为 `userAge`）。
> 2. **[结构重组]**：将过于庞大、承担多重职责的函数按功能进行拆解（严格遵循单一职责原则），并提取、合并其中重复的逻辑。
> 3. **[现代化改造]**：在确保向下兼容的前提下，尽可能应用该编程语言最新的语法特性（如 ES6+、Python 3.10+ 等）。
> 4. **[修改对比]**：请详细列出修改前后的差异对比（Before & After），并逐条解释为什么要进行这样的修改，以及这种修改具体带来了哪些好处。
>
> **约束事项 (Constraints):**
>
> - **绝对原则**：重构后的代码行为（核心业务逻辑）必须与原代码保持 **100% 一致**。
> - **注释精简**：仅在处理极其复杂的业务逻辑时，才允许添加必要的注释。如果代码本身已经足够清晰（Self-documenting），请务必删除所有多余的废话注释。
>

>

## 🙋 常见问题 (FAQ)

- **Q: 它能处理公司内部极其复杂的业务逻辑吗？**
  - A: 完全可以。但为了防止 AI 产生“幻觉（Hallucination）”或意外篡改核心业务逻辑，强烈建议你在提示词的“情况 (Context)”板块中，简明扼要地补充该段代码的真实业务意图（例如：“这段代码主要用于计算电商平台 VIP 用户的折后最终价”）。

- **Q: 这样做会不会有泄露公司机密代码的风险？**
  - A: 如果代码中涉及敏感用户数据、API Key 或核心商业机密，**请绝对不要将其直接粘贴到公共 AI 模型中**。建议在重构前，先将关键变量名替换为脱敏的 `dummy_data`，或者直接在公司内部署的企业级私有化大模型中使用此提示词。

## 📊 证明: Before & After

### ❌ Before (输入)

```python
def calc(x, y):
    if x > 19:
        print("") # 打印成人
        p = y * 0.9
        return p
    else:
        print("") # 打印学生
        return y
```

### ✅ After (结果)

```python
def calculate_discounted_price(age, price):
    ADULT_AGE_THRESHOLD = 20
    DISCOUNT_RATE = 0.9

    if age >= ADULT_AGE_THRESHOLD:
        print("成人")
        return price * DISCOUNT_RATE

    print("学生")
    return price
```

**[修改解析]**

1. **函数与变量重命名**: 将含糊不清的 `calc` 重命名为 `calculate_discounted_price`，瞬间点明了函数的真实意图；同时将 `x`, `y` 替换为语义化的 `age`, `price`，让代码做到一目了然。
2. **消除魔术数字 (Magic Numbers)**: 将代码中硬编码的 `19` 和 `0.9` 提取为具名常量 `ADULT_AGE_THRESHOLD` 和 `DISCOUNT_RATE`，这不仅极大地提升了代码可读性，更为后期的维护与修改铺平了道路。
3. **结构优化 (Early Return)**: 果断移除了冗余的 `else` 分支，采用“提前返回（Early Return）”设计模式，有效降低了代码的缩进层级，使逻辑分支更加扁平化。
