## 📝 Unsloth: LLMファインチューニングコードを1分で完成させるプロンプト

- **🎯 おすすめの対象者:** AIエンジニア、ジュニア開発者、AI研究者
- **⏱️ 所要時間:** 3時間（VRAMデバッグ等） → 1分に短縮
- **🤖 推奨モデル:** Claude 3.5 Sonnet, GPT-4o

- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「LLMのファインチューニング中、幾度となく発生するOOM（Out of Memory）エラーや、果てしなく続く学習時間に、始める前から疲弊していませんか？」_

大規模言語モデル（LLM）を自社独自のデータセットで学習させることは非常に魅力的ですが、現実には「膨大なVRAM要件」と「絶望的な学習時間」という高い壁に阻まれがちです。特に16GBのVRAM（T4、RTX 3060/4060など）しか搭載していない環境では、7Bサイズのモデルですら学習を完遂するのは至難の業です。

このような慢性的なリソース不足を一気に解決してくれる救世主が**Unsloth**です。従来比で**学習速度を2倍以上に引き上げ、メモリ使用量を最大60%も削減**してくれるこの驚異的なライブラリを活用し、ご自身のハードウェア環境に最適なファインチューニングコードを一瞬で生成できる魔法のプロンプトをご紹介します。

## ⚡️ 3行まとめ (TL;DR)

1. 複雑なUnslothの最適化設定（LoRA、4bit量子化など）を、AIが一発で正確にコード化してくれます。
2. お使いのGPU環境とデータセットの構造をプロンプトに入力するだけで、エラーなく即実行可能なColab / Jupyter用スクリプトが手に入ります。
3. 16GBの限られたVRAM環境であっても、Llama-3-8Bなどの最新モデルを安全かつ正常にファインチューニングできます。

## 🚀 解決策: "Unslothファインチューニング・スクリプトジェネレーター"

### 🥉 Basic Version (基本型)

素早く、基本的な構造のUnslothコードの骨組みが必要な場合にご活用ください。

> **役割:** あなたはLLMの最適化およびファインチューニングの専門家です。
>
> **タスク:** `[Llama-3-8B]`モデルを`[Alpaca]`データセットでファインチューニングするための、UnslothベースのPythonスクリプトを作成してください。16GBのVRAMを搭載したGPU環境で実行する予定です。

### 🥇 Pro Version (専門家型)

自身のデータセットやハードウェアのスペックに合わせて、微細なチューニングが施された完璧な実務用コードが必要な場合は、こちらを使用してください。

> **役割 (Role):** あなたはPyTorchとHugging Faceエコシステムに精通しており、特に「Unsloth」ライブラリを活用したモデルの軽量化と学習最適化（SFT）において圧倒的な知見を持つ、シニアAIエンジニアです。
>
> **状況 (Context):**
>
> - 目標: 限られたGPUリソース（16GB VRAM）の環境下で、LLMのファインチューニングを確実に成功させること。
> - 環境: Google Colab（無料のT4インスタンス）またはローカルのJupyter環境。
>
> **タスク (Task):**
>
> 以下の変数（`[ ]`）情報に基づいて、コピー＆ペーストですぐに実行可能なUnslothベースのファインチューニング用フルスクリプトを作成してください。
>
> 1. 環境構築、およびUnslothや依存パッケージのインストールコマンドを網羅すること。
> 2. `FastLanguageModel`を活用した4bit量子化モデルのロード処理を記述すること。
> 3. 最適化されたLoRAアダプタの設定を行うこと（`r`、`target_modules`、`gradient_checkpointing = "unsloth"`は必須条件）。
> 4. `SFTTrainer`を使用した学習設定（batch_sizeやlearning_rateなど、メモリエラーが発生しない安全な値に厳密にチューニングすること）。
> 5. 学習完了後、GGUF形式およびLoRAアダプタとしてモデルを保存するコードを含めること。
>
> **変数 (Variables):**
>
> - ベースモデル: `[unsloth/llama-3-8b-bnb-4bit]`
> - データセット構造: `[Hugging Faceのyahma/alpaca-cleaned形式に準拠。instruction、input、outputカラムが存在する]`
> - 最大シーケンス長: `[2048]`
> - エポックまたはステップ: `[max_steps = 60]`
>
> **制約事項 (Constraints):**
>
> - Pythonのコードブロック1つにきれいにまとめて出力すること。
> - コードの主要な各ステップには、意図を説明する詳細なコメント（日本語）を記述すること。
> - OOM（メモリ不足）を引き起こす可能性のある危険なパラメータ設定（例: batch_sizeの無理な拡大など）は絶対に避けること。

## 💡 筆者コメント (Insight)

Unslothは、手動で極限まで最適化されたTritonカーネルを使用しており、「ファインチューニング界の革命」とまで呼ばれています。しかし、初めてこのライブラリに触れる開発者は、`FastLanguageModel`の初期化処理や、`SFTTrainer`のハイパーパラメータ（特に`gradient_accumulation_steps`など）の微調整に苦戦し、結局頻繁にOOMエラーに遭遇してしまいます。

このプロンプトの真髄は、AIが**「あなたのGPUメモリの限界を計算した最適なハイパーパラメータ」**を自動的に設定してくれる点にあります。特に`gradient_checkpointing = "unsloth"`という特殊なオプションを書き忘れてエラーになるケースが非常に多いのですが、Proプロンプトではこの部分を強制的に指定することで、失敗の確率を劇的に下げました。実務環境では、データセットの構造を自社のデータに合わせて変数として入力するだけで即座に利用できるため、開発の生産性が文字通り飛躍的に向上します。

## 🙋 よくある質問 (FAQ)

- **Q: Unslothはどのようなモデルをサポートしていますか？**
  - A: Llama、Mistral、Gemma、Qwenなど、現在人気のあるオープンソースアーキテクチャのほとんどをサポートしています。プロンプト内の「ベースモデル」変数を変更するだけで、即座に対応可能です。
- **Q: Colabの無料版（T4）でも本当に動作するのでしょうか？**
  - A: はい、完璧に動作します。Unslothの4bit量子化と独自のメモリ最適化技術のおかげで、無料のT4インスタンス1台であっても、8Bパラメータ規模のモデルまでは全く問題なく学習させることができます。
- **Q: 日本語の独自データセットでファインチューニングするにはどうすればよいですか？**
  - A: プロンプトの「データセット構造」変数に、日本語データセットの正確なカラム名（例：`instruction`、`output`の代わりに`質問`、`回答`など）を指定すれば、AIが前処理マッピング（`formatting_prompts_func`）のコードをあなたのデータセットに最適化して修正してくれます。

## 🧬 プロンプトの解剖学 (Why it works?)

1. **専門家のペルソナとライブラリの特定:** 「Unsloth最適化の専門家」として役割を固定することで、一般的で非効率なHugging Faceのコードではなく、Unsloth専用の最適化構文（`FastLanguageModel`など）を優先して使用するように強制しました。
2. **明確なパイプラインの指示:** パッケージのインストール ➔ モデルのロード ➔ データの前処理 ➔ 学習 ➔ 保存という、MLOpsにおけるパイプラインの順序を明確に指定し、途中で必須のロジックが絡まったり欠落したりするリスクを防ぎます。
3. **制約事項による安定性の確保:** 「OOM発生リスクの回避」を明示的に指示することで、AIが保守的で安全なバッチサイズとaccumulation stepsを先制して提案するように誘導しました。

## 📊 検証: Before & After

### ❌ Before (一般的なファインチューニングコードをリクエストした場合)

```python
# 一般的なHugging Face Transformersベースで記述されており、
# 16GB環境で実行すると「CUDA Out of Memory」エラーが発生する危険性が非常に高い
from transformers import AutoModelForCausalLM
model = AutoModelForCausalLM.from_pretrained("unsloth/llama-3-8b")
# ... (メモリ最適化オプションが欠落)
```

### ✅ After (このプロンプトを適用した場合の出力の一部)

```python
# Unslothの最適化パッケージを活用したエラーのない完璧なSFTコード
from unsloth import FastLanguageModel
import torch

# 1. 4bit量子化およびメモリ最適化モデルのロード
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name = "unsloth/llama-3-8b-bnb-4bit",
    max_seq_length = 2048,
    dtype = None,
    load_in_4bit = True, # 重要なメモリ節約手法
)

# 2. パラメータ効率化のためのLoRAアダプタの適用
model = FastLanguageModel.get_peft_model(
    model,
    r = 16,
    target_modules = ["q_proj", "k_proj", "v_proj", "o_proj",
                      "gate_proj", "up_proj", "down_proj"],
    use_gradient_checkpointing = "unsloth", # 長いコンテキストとVRAMの最適化に必須
    # ... (省略)
)
```
