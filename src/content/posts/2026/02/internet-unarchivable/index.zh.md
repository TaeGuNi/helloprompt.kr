# 📝 拯救消失的网页：Web 内容永久存档提示词

- **🎯 推荐对象：** 研究人员、内容策展人、数字档案管理员、营销人员
- **⏱️ 所需时间：** 1小时 → 缩短至1分钟
- **🤖 推荐模型：** 擅长处理长上下文的模型（推荐 Claude 3.5 Sonnet, Gemini 2.5 Pro）

- ⭐ **难度：** ⭐⭐⭐☆☆
- ⚡️ **效果：** ⭐⭐⭐⭐⭐
- 🚀 **实用性：** ⭐⭐⭐⭐⭐

> _"抓住正在消失的网络历史。只需一次复制粘贴，即可剥离混乱渲染的动态网页，将其核心语境完美定格为永久的 Markdown 文档。"_

随着单页应用（SPA）和封闭平台（Walled Gardens）逐渐成为现代 Web 的标准，使用传统方式抓取和保存包含高价值信息的网页几乎变得不可能。链接可能一夜之间失效，甚至连专业的 Web 存档网站也经常无法捕获动态渲染的画面，最终只留下一片空白。

这款提示词是一件强大的知识保存利器。当你把杂乱无章的网页可见文本或晦涩难懂的 HTML 源代码丢给它时，它能自动剔除无用的噪音（如广告、导航栏、页脚），并将内容转化为**结构完美、无论未来何时都能轻松阅读和使用的 Markdown 文档**。

---

## ⚡️ 核心三要素 (TL;DR)

1. **智能去噪：** 从复杂的网页转储文本中过滤掉广告和 UI 元素，精准提取正文核心。
2. **永久格式转换：** 将内容重构为保存性和兼容性极佳的 Markdown 格式。
3. **元数据自动提取：** 自动生成包含发布日期、作者、核心关键词等对存档和检索至关重要的 YAML 元数据。

## 🚀 解决方案："数字档案管理员提示词"

### 🥉 Basic Version (基础版)

当你只需要快速保存网页的核心正文内容时使用。全选（Ctrl+A）网页内容并复制，然后连同下方提示词一起输入。

> **角色：** 你是一位专业的“数字档案管理员”。
> **请求：** 请从下方输入的杂乱网页文本中，剔除菜单、广告等所有噪音，只将包含信息价值的“正文内容”用 Markdown 格式干净利落地整理出来。


### 🥇 Pro Version (专家版)

当你希望在不丢失网站原始语境（Context）的前提下，构建完美的存档文档，以便立即存入个人知识管理工具（如 Obsidian、Notion）时使用。

> **角色 (Role)：** 你是一位致力于保存人类数字遗产的顶尖“数字档案管理员（Digital Archivist）”。你的首要任务是从复杂且碎片化的 Web 内容中提取信息的精华，并将其加工成可永久保存的格式。
>
> **背景 (Context)：**
>
> - 现状：由于动态网页（SPA）和封闭平台的盛行，Web 存档变得日益困难。我们需要直接将原始文本或源代码转换为永久保存格式。
> - 目标：从输入的文本/HTML 转储数据中清除不必要的噪音（广告、全局导航栏、页脚、评论等），并将其转化为结构完美的 Markdown 文档。
>
> **任务 (Task)：**
>
> 1. 分析下方的 `[网页数据]`，仅提取正文（Main Article）。
> 2. 在文档最顶部以 YAML Frontmatter 格式编写存档元数据（必须包含：标题、推测发布日期、作者、3个核心关键词）。
> 3. 使用合理的标题层级（`##`, `###`）重新编排正文，并应用 Markdown 语法进行格式化。
> 4. 对于重要的引用、关键数据和统计资料，请使用 Markdown 引用（`>`）或加粗（`**`）进行视觉上的强调。
>
> **输入数据：**
>
>
> [请在此处粘贴网页的全部文本(Ctrl+A, C)或 HTML 源代码]
>
>
> **限制条件 (Constraints)：**
>
> - 绝不要随意总结或歪曲正文的原始句子和语境。（因为目的是存档，必须保持 100% 的原文忠实度）。
> - 输出格式必须严格遵循纯粹的 Markdown 语法，不要将正文包裹在代码块中。
>
> **注意事项 (Warning)：**
>
> - 如果在输入数据中无法明确找到发布日期或作者，请不要凭空捏造，直接标为 `Unknown`。（防止幻觉）

## 💡 作者点评 (Insight)

这个提示词绝不仅是一个简单的“文章摘要器”。在 2026 年，互联网信息挥发的速度呈指数级增长，而这是一个**将信息所有权彻底转移到本地环境的数据管道**。

它在那些常规网页剪藏工具（Web Clipper）失效的场景中尤为耀眼——比如需要登录的付费新闻信、无限滚动的 X (Twitter) 帖子或是公司内部的 Wiki 页面。你只需在屏幕上全选（Ctrl+A）、复制（Ctrl+C），然后将其扔进提示词中，UI 残渣就会像变魔术一样消失，只留下纯净的原始数据。就我个人而言，通过这种方法，我每周都能将几十篇濒临消失的优质文章完美迁移到 Obsidian 中。

## 🙋 常见问题 (FAQ)

- **Q: 我应该复制 HTML 源代码，还是直接复制屏幕上的可见文本？**
  - A: 两者皆可。但对于最新的动态渲染网页，在浏览器中直接全选（Ctrl+A）并复制文本会更好，这能大幅减少隐藏脚本带来的噪音，让 AI 更专注于核心文本。

- **Q: 内容太长导致 AI 模型报错，该怎么办？**
  - A: 抓取整个网页可能会导致 Token 数量激增。强烈建议使用支持超大上下文窗口的模型（如 Claude 3.5 Sonnet 或 Gemini 2.5 Pro）。

## 🧬 提示词解剖 (Why it works?)

1. **强制保持原文忠实度 (Constraints)：** LLM 在面对长文本时，通常会条件反射般地想要去“总结”。加入“因为目的是存档，必须保持 100% 的原文忠实度”这一强力约束条件，可以有效抑制 AI 的总结本能，使其专注于“存档”这一核心目标。
2. **结构化提取元数据：** 要求生成 YAML Frontmatter，这不仅仅是为了让文章看起来专业，更重要的是确保了技术上的兼容性，使其能够无缝对接到真实的数据库或基于 Markdown 的笔记应用中。
