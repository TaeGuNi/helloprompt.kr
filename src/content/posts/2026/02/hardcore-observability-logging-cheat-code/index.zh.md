## ⚡️ 3 句话总结 (TL;DR)

- 🚫 **永久封杀 `console.log`：** 全面弃用原生 console，强制接管并注入专用的工业级日志实例 (如 Pino, Winston)。
- 🧱 **强制结构化日志与 Trace ID：** 100% 以 JSON 格式输出日志，并利用 Trace ID 死死咬住跨函数的请求上下文。
- 🥷 **完美屏蔽敏感隐私 (PII)：** 必须在源头拦截密码、Token 等致命信息，强制将其脱敏替换为 `[REDACTED]`。

## 💡 作者点评 (Insight)

说实话，在 Review 那些初级开发者的代码时，你知道最让人抓狂的场景是什么吗？就是在一个致命的异常处理分支里，孤零零地躺着一行 `console.error(err)`。线上业务都已经全面熔断了，监控大盘里却只留下了一句“数据库连接失败！”的干瘪废话。就凭这种破烂线索，你要怎么去追踪是哪个接口、哪个无辜用户的请求炸了？

这份提示词的本质，就是在代码生成的**最前端防线**彻底震慑住 AI。千万不要只对 AI 说一句轻飘飘的“请帮我把日志写规范些”。大模型的骨子里都是试图讨好人类的“Yes Man”，如果你提需求时含糊其辞，它们生成的错误追踪代码绝对会是一团灾难。

在我们的核心业务流中深度应用这套提示词后，我发现大模型已经能够极其熟练地自行搭建从 Sentry 错误上报到 Pino 脱敏 (Redaction) 管道的完整链路。特别是在引入了针对用户隐私泄露风险的 **警告 (Warning)** 机制后，AI 甚至学会了主动进行自我代码审查，将所有可能泄露的明文日志强行拦截并替换为高标准的结构化日志。如果你不想在凌晨 3 点被毫无营养的 On-Call 警报吵醒，那就必须将这部“可观测性宪法”死死地刻进你的 AI Agent 脑子里。

## 🙋 常见问题 (FAQ)

- **Q: AI 真的能听懂这种带有情绪压力的强硬指令吗？**
  - A: 令人惊讶的是，确实能。现代大语言模型会根据用户的语气 (Tone) 和施加的上下文压力，动态调整输出策略并大幅提升专业严谨度。相比于客气的“请帮我做…”，“违反此条则代码作废”的压迫感在抑制 AI 幻觉 (Hallucination) 方面有奇效。
- **Q: 这套终极指令只能用于后端 Node.js/Python 代码吗？**
  - A: 绝对不是。当你将其应用于前端代码 (如 React、Vue) 时，它能极为有效地阻止因无脑渲染错误而导致的白屏死机 (White Screen of Death)，并主动为你封装出完美的全局 Error Boundary 与 Sentry 追踪接入层。
- **Q: 如果我们当前的项目甚至连专门的日志库都没接入怎么办？**
  - A: 完全不必担心。只要你在上下文中喂给它这套提示词，AI 就会主动“越俎代庖”，为你生成基于 `Winston` 或 `Pino` 这类工业级标准日志库的初始配置 (Setup) 模板代码。你只需直接复制粘贴并安装依赖即可。

### ❌ Before (输入：极其粗糙的请求)
```text
帮我写个登录功能，报错的话记得打日志。
```
_(结果：大模型在错误分支里疯狂滥用 `console.log("登录失败")`，极易引发用户密码被明文打印进公共日志的史诗级安全惨剧)_

### ✅ After (结果：应用硬核终极指令)
```javascript
// AI 输出的真实架构师级别代码示例
import logger from '@utils/logger'; // 带有脱敏管道的 Pino 实例
import { v4 as uuidv4 } from 'uuid';

export const loginHandler = async (req, res, next) => {
  const traceId = req.headers['x-trace-id'] || uuidv4();
  const contextLogger = logger.child({ traceId, action: 'user_login' });

  try {
    const { email, password } = req.body;
    contextLogger.info({ email }, 'Login attempt initiated'); // password 会被自动 Redacted

    // ... 登录核心逻辑 ...

  } catch (error) {
    // 仅在 5xx 级别的系统故障时赋予 ERROR 级别，并输出结构化错误日志
    contextLogger.error({ 
      error: error.message, 
      stack: error.stack 
    }, 'Internal Server Error during login process');
    next(error);
  }
};
```
