# 📝 Excel、JSON、CSV... データフォーマット変換単純作業脱出

- **🎯 推奨対象:** エンジニア、データアナリスト、事務職全般
- **⏱️ 所要時間:** 30分 → 1分に短縮
- **🤖 推奨モデル:** すべての対話型AI (ChatGPT, Claude, Geminiなど)

- ⭐ **難易度:** ⭐⭐☆☆☆
- ⚡️ **効果性:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐☆

> _「このExcelファイルのデータをJSONに変換して、APIで送ってください。」_

開発やデータ分析の現場では、データフォーマットの変換が日常茶飯事です。
オンラインの変換ツールを使おうにも機密データだからアップロードできないし、その都度スクリプトを書くのも本末転倒。
そんな時は、データをコピーして貼り付けるだけで大丈夫です。AIがあなたの指示通りに瞬時に変換してくれます。

---

## ⚡️ 3行要約 (TL;DR)

1. **自由自在なフォーマット変換:** JSON ↔ CSV ↔ XML など、あらゆる形式に対応。
2. **高度なデータ加工:** 不要なフィールドの削除や、データ構造の変更（マッピング）も同時に処理。
3. **実用的な出力:** そのまま使えるSQLのINSERT文生成など、活用度の高い変換が可能。

## 💡 作成者コメント (Insight)

このプロンプトの真の価値は、単なるフォーマット変換にとどまらず「データクレンジング」も同時に行える点にあります。例えば、CSVデータをJSON配列に変換する過程で、「日付の表記を『YYYY-MM-DD』に統一して」「空欄（Null）の項目は除外して」といった指示を追加するだけで、ETLパイプラインの一部をAIに代替させることができます。機密情報を扱う場合は、顧客名やメールアドレスなどをダミーデータに置換してからAIに渡す（マスキング処理）工夫をすると、セキュリティ面でも安心して活用できます。

## 🙋 よくある質問 (FAQ)

- **Q: 変換するデータ量に制限はありますか？**
  - A: AIモデルのコンテキストウィンドウ（一度に処理できるトークン数）に依存します。例えば、大容量モデルであれば数千行のCSVも処理可能ですが、エラーを防ぐため数百行ずつ分割して処理することをおすすめします。

- **Q: 階層が深い複雑なJSONやXMLでも正しく変換されますか？**
  - A: はい、可能です。ただし、フラットな形式（CSVなど）に変換する場合、AIが独自の判断で階層を結合・省略することがあります。「ネストされたデータはアンダースコア(\_)で繋いで展開して」のように、展開ルールを具体的に指示するとより正確な結果が得られます。

- **Q: 社外秘のデータを変換したいのですが、セキュリティは大丈夫ですか？**
  - A: API経由での利用は通常学習データとして使用されませんが、無料版のウェブブラウザのチャットインターフェースを使用する場合は学習データとして活用される可能性があります。機密データを扱う場合は、個人情報が含まれていないか事前に確認するか、エンタープライズ版のAIを使用してください。

## 🧬 プロンプト解剖 (Why it works?)

1. **専門家のペルソナ設定:** 「データエンジニア・ETL専門家」という役割を与えることで、AIがデータ構造の整合性やエッジケースに対する注意力を高めます。
2. **変換ルールの明確化:** 単に「変換して」と頼むのではなく、「変換元」「変換先」「マッピングルール」を構造化して渡すことで、意図しないデータの欠損やフォーマットの崩れ（ハルシネーション）を防ぎます。
3. **余計な出力の抑制:** 「結果のみをコードブロックで出力して」と指示することで、AIの無駄な挨拶や説明を省き、すぐに結果をコピペして実務に投入できるように最適化されています。

## 📊 証明: Before & After

### ❌ Before (入力 CSV)

```csv
名前,年齢,職業
ホン・ギルドン,30,開発者
キム・チョルス,25,デザイナー
```

### ✅ After (結果 - JSON + フィールド名変更)

**リクエスト:** CSVをJSONに変換し、フィールド名を英語に変更してください。

```json
[
  {
    "name": "ホン・ギルドン",
    "age": 30,
    "job": "開発者"
  },
  {
    "name": "キム・チョルス",
    "age": 25,
    "job": "デザイナー"
  }
]
```

### ✅ After (結果 - SQL Insert文生成)

**リクエスト:** データをMySQLの `users` テーブルに追加するためのINSERT文に変換してください。

```sql
INSERT INTO users (name, age, job) VALUES
('ホン・ギルドン', 30, '開発者'),
('キム・チョルス', 25, 'デザイナー');
```
