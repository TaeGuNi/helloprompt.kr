# ⚖️ 콘텐츠 품질 평가 모델 (Quality Evaluation Model)

Hello Prompt 블로그의 모든 글은 발행 전 이 모델을 통해 엄격하게 심사받아야 합니다.
**총점 500점 만점**이며, 5개 항목 중 **단 하나라도 95점 미만**일 경우 해당 글은 즉시 **반려(Reject) 및 재작성**해야 합니다.

---

## 1. 주제 적합성 (Relevance) [100점]

> _"우리 블로그의 정체성(Identity)과 맞는가?"_

- **평가 기준:**
  - (40점) **타겟 일치:** AI 초보자~중급자(직장인/개발자)가 읽기에 난이도가 적절한가?
  - (30점) **목적 부합:** "실용적인 프롬프트 가이드"라는 블로그 목적에 부합하는가? (단순 뉴스나 잡담 금지)
  - (30점) **톤앤매너:** 친절하고, 명확하고, 권위 있는 어조를 유지하는가?

- **탈락 예시 (Fail):** "AI의 역사", "단순 뉴스 요약", "너무 전문적인 논문 리뷰"

## 2. 유용성 (Utility) [100점]

> _"당장 써먹을 수 있는가? 시간/돈을 아껴주는가?"_

- **평가 기준:**
  - (40점) **즉시 실행 가능성:** 본문의 프롬프트를 복사(Copy)해서 붙여넣기(Paste)만 하면 작동하는가?
  - (30점) **명확한 효용:** 이 글을 읽기 전과 후, 독자의 생산성이 확실히 달라지는가?
  - (30점) **결과 검증:** 결과 예시(Output)가 독자의 기대를 충족할 만큼 고품질인가?

- **탈락 예시 (Fail):** 개념만 설명하고 프롬프트가 없는 글, 실행해봐도 결과가 별로인 글.

## 3. 독창성 (Originality) [100점]

> _"뻔한 소리(Cliché)가 아닌가? 인간의 인사이트가 있는가?"_

- **평가 기준:**
  - (40점) **차별화:** 시중에 널린 "ChatGPT 사용법"과 다른, 우리만의 팁이나 관점이 있는가?
  - (30점) **구조적 독창성:** 프롬프트 설계(Role, Context, Constraints)가 정교하고 창의적인가?
  - (30점) **인사이트:** AI가 쓴 글 티가 나지 않고, 작성자의 경험/노하우가 녹아있는가?

- **탈락 예시 (Fail):** "ChatGPT에게 인사하는 법", 위키피디아 수준의 일반론.

## 4. 희소성 (Scarcity) [100점]

> _"다른 곳에서는 쉽게 찾을 수 없는 정보인가?"_

- **평가 기준:**
  - (50점) **정보의 깊이:** 구글 검색 1페이지에 나오는 글들보다 더 깊이 있고 구체적인가?
  - (50점) **니치(Niche) 타겟팅:** 특정 상황(예: "개발자의 레거시 코드 리팩토링", "마케터의 카피라이팅")을 뾰족하게 겨냥했는가?

- **탈락 예시 (Fail):** 누구나 다 아는 상식, 이미 유튜브/블로그에 도배된 흔한 팁.

## 5. 진실성 (Factuality) [100점]

> _"거짓(Hallucination)은 없는가? 논리적으로 완벽한가?"_

- **평가 기준:**
  - (50점) **팩트 체크:** 언급된 AI 모델명, 기능, 제한사항이 최신 정보와 일치하는가?
  - (50점) **논리적 무결성:** 프롬프트의 논리 구조(인과관계)에 오류가 없는가? 그럴싸해 보이는 거짓말이 없는가?

- **탈락 예시 (Fail):** 없는 기능을 있다고 설명함, 실행되지 않는 코드 포함.

---

## 🚨 심사 프로세스 (Workflow)

1. **초안 작성 (Draft):** 에이전트가 글 작성
2. **자가 점검 (Self-Audit):** 위 5개 기준으로 점수 매기기
3. **판정 (Decision):**
   - `PASS`: 모든 항목 95점 이상 → 발행
   - `FAIL`: 하나라도 95점 미만 → **원인 분석 후 즉시 재작성**

---

## 📜 문서(Documentation) 품질 기준 [100점]

블로그 글뿐만 아니라, 프로젝트 관리 문서(`README.md` 등)도 다음 기준으로 평가합니다.

1.  **최신성 (Up-to-date) [30점]:** 코드가 변경되었을 때 문서도 즉시 업데이트되었는가? (코드와 문서의 동기화)
2.  **완전성 (Completeness) [30점]:** 누락된 정보(예: 예외 처리, 환경 설정) 없이 모든 케이스를 다루는가?
3.  **가독성 (Readability) [20점]:** 줄글이 아닌, 구조화된 목차/표/다이어그램을 사용했는가?
4.  **접근성 (Accessibility) [20점]:** 신입 개발자나 비전문가도 이해할 수 있는 쉬운 용어를 사용했는가?
