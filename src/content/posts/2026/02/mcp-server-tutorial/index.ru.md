В реалиях 2026 года **MCP (Model Context Protocol)** стал де-факто стандартом подключения LLM к внешним источникам. Воспринимайте его как универсальный «USB-порт» для искусственного интеллекта. Вместо того чтобы каждый раз скармливать контекст вручную, достаточно один раз поднять сервер — и модель начнет самостоятельно, в реальном времени, «читать» ваши файлы, базы данных и внутренние API.

Сегодня уже есть отличные готовые MCP-серверы для интеграции с GitHub или локальной файловой системой. Но настоящая магия происходит тогда, когда вы подключаете к LLM **уникальную бизнес-логику вашей компании**.

В этой статье мы не будем писать шаблонный код с нуля. Мы дадим вам готовый промпт, который заставит ИИ написать идеальный кастомный MCP-сервер на TypeScript всего за 5 минут.

## 🚀 Решение: "Генератор MCP-сервера"

### 🥉 Basic Version (Базовая версия)

Используйте, когда нужно быстро получить базовый каркас MCP-сервера.

> **Роль:** Ты Senior TypeScript разработчик.
> **Задача:** Напиши простой код MCP-сервера с использованием `@modelcontextprotocol/sdk`, который выполняет следующую задачу: `[укажите цель, например: показывает погоду в определенном городе]`.

### 🥇 Pro Version (Продвинутая версия)

Используйте, когда нужен надежный и масштабируемый код сервера, готовый к применению в рабочей среде.

> **Роль (Role):** Ты Senior TypeScript инженер, глубоко разбирающийся в экосистеме Model Context Protocol (MCP).
>
> **Контекст (Context):**
>
> - Ситуация: Мне нужен кастомный MCP-сервер для интеграции локальных данных или внутреннего API с LLM.
> - Цель: Сгенерировать полный, исполняемый код MCP-сервера в виде одного файла (`index.ts`), идеально подходящий под мои требования.
>
> **Задача (Task):**
>
> 1. Напиши код MCP-сервера, отвечающий следующим требованиям:
>    - **Назначение:** `[Подробно опишите назначение сервера. Например: чтение последних 50 строк определенного лог-файла]`
> 2. Код должен быть снабжен комментариями и оптимизирован для формата "скопировал-вставил", чтобы я мог сразу запустить его в терминале.
>
> **Ограничения (Constraints):**
>
> - Обязательно используй официальный пакет `@modelcontextprotocol/sdk`.
> - Если входные параметры инструмента (tool) сложные, используй библиотеку `zod` для валидации схемы (рекомендуется).
> - При создании сервера должны быть полностью реализованы интерфейсы `ListToolsRequestSchema` и `CallToolRequestSchema`.
> - Внутри обработчика инструмента (Handler) обязательно используй конструкцию `try/catch` для перехвата исключений, чтобы сервер не упал при ошибке.
> - **Выведи только один блок кода на TypeScript без каких-либо дополнительных объяснений.**
>
> **Предупреждения (Warning):**
>
> - Не выдумывай несуществующие методы SDK. Если не уверен, используй стандартные API Node.js в качестве обходного пути.

## 💡 Комментарий автора (Insight)

Главная сила этого промпта — не в абстрактной просьбе «написать код», а в жестком требовании соблюдать **ключевые спецификации протокола (обработка запросов ListTools и CallTool)**, без которых не обходится ни один валидный MCP-сервер.

Важнейшее преимущество собственного MCP-сервера кроется в **абсолютной безопасности и приватности данных (Privacy)**. Когда вы используете стандартные инструменты веб-поиска, ваши запросы неизбежно проходят через сторонние серверы. Но с кастомным сервером, подключенным напрямую к вашей PostgreSQL или корпоративному API, данные передаются по защищенному каналу и остаются под вашим полным контролем. Вам больше никогда не придется отправлять конфиденциальные внутренние документы в окно облачного чата.

Для начала попробуйте создать с помощью этого промпта простой инструмент формата read-only (например, для чтения локальных логов). Как только вы поймете механику, вы сможете легко расширить функционал до записи данных в базу, превратив модель в мощного и автономного ИИ-ассистента.

## 🙋 Часто задаваемые вопросы (FAQ)

- **Вопрос: Подойдет ли этот промпт, если я вообще не умею программировать?**
  - Ответ: Абсолютно! Просто опишите свою задачу в блоке `[Назначение]` простыми словами. Например: «Функция, которая выдает погоду в Москве», — и Claude сам напишет корректный код. Вам останется лишь скопировать его и запустить.

- **Вопрос: Как запустить и подключить сгенерированный сервер?**
  - Ответ: Сохраните полученный код в файл `index.ts`, а затем пропишите путь к этому скрипту в секции `mcpServers` конфигурационного файла Claude Desktop (`claude_desktop_config.json`). Интеграция подхватится автоматически.

- **Вопрос: Сервер, созданный этим промптом, работает только с Claude?**
  - Ответ: Нет. Model Context Protocol — это открытый стандарт. Ваш сервер будет точно так же работать с Ollama, Cursor и любыми другими ИИ-клиентами, поддерживающими MCP.

## 🧬 Анатомия промпта (Why it works?)

1. **Строгий контроль через ограничения (Constraints):** Принудительное использование `@modelcontextprotocol/sdk` и `zod`, а также явное требование реализовать схемы `ListToolsRequestSchema` и `CallToolRequestSchema`, на корню пресекают любые галлюцинации модели и отклонения от спецификации.
2. **Один файл и ничего лишнего:** Условие поместить весь код в один файл (`index.ts`) и строгий запрет на текстовые пояснения сводят к минимуму время от запроса до работающего результата (Time-to-Value).
3. **Отказоустойчивость:** Заранее прописанная логика `try/catch` страхует от непредвиденных ошибок при вызове инструментов и не дает MCP-серверу упасть в процессе работы.
