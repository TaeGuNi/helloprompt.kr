## 📝 Tambo SDKで始めるジェネレーティブUI (Generative UI) 完全ガイド

- **🎯 おすすめの読者:** フロントエンドエンジニア、React開発者、プロダクトマネージャー
- **⏱️ 所要時間:** 60分 → 10分（Tambo SDK導入による短縮）
- **🤖 推奨モデル:** Claude 3.5 Sonnet, GPT-4o (関数呼び出し/Function Callingに優れたモデル)

- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果性:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「チャットボットがテキストしか返さない時代は終わりました。AIがユーザーの意図を汲み取り、最適なUIをリアルタイムで生成する体験を提供できていますか？」_

AIが単なるテキストや画像を生成するだけでなく、**ユーザーインターフェース（UI）**そのものをリアルタイムで構築する時代が到来しました。これが**「ジェネレーティブUI (Generative UI)」**です。

本記事では、React環境でこのジェネレーティブUIを最もスマートに実装できる強力なツール**「Tambo SDK」**を活用し、AIアシスタントにリッチなコンポーネントを描画させるためのシステムプロンプトと具体的な実装手法を解説します。

---

## ⚡️ 3行まとめ (TL;DR)

1. **Generative UIの台頭:** AIはテキスト応答の枠を超え、文脈に最適なReactコンポーネント（カードやグラフ等）を直接レンダリング可能に。
2. **Tambo SDKによる圧倒的時短:** 複雑なFunction Callingのパースや状態管理をSDKが巻き取り、開発体験（DX）を劇的に向上。
3. **プロンプト設計の重要性:** AIに「いつ・どのUIコンポーネントを提示すべきか」を的確に指示するプロンプトが、UXの質を決定づける。

---

## 🚀 解決策: ジェネレーティブUI実装プロンプト

ジェネレーティブUIを成功に導く鍵は、単なるコードの実装にとどまりません。AIに対して**「どのような状況で特定のコンポーネント（ツール）を呼び出すべきか」**を正確に定義する、システムプロンプトの緻密な設計にあります。

### 🥉 Basic Version (基本システムプロンプト)

まずは、UIコンポーネントを呼び出すための最もシンプルなベースラインとなる指示です。

> **役割 (Role):** あなたはユーザーの要望に合わせて最適なUIコンポーネントを提供するAIアシスタントです。
> **指示 (Instruction):** ユーザーが特定の場所や観光地について尋ねた場合は、必ず `showPlace` ツール（UIコンポーネント）を呼び出して応答してください。

### 🥇 Pro Version (実践向けシステムプロンプト)

実際のプロダクト環境でも堅牢に機能するよう、制約条件とフローを明確に定義したプロンプト設計です。

> **役割 (Role):**
> あなたは旅行計画をサポートする、専門的なUX/UIエージェントです。
>
> **状況 (Context):**
>
> - 背景: ユーザーはReactで構築されたチャットインターフェースを通じてあなたと対話しています。
> - 目標: 単なるテキストの羅列を避け、視覚的に直感的なUIコンポーネント（カードやボタンなど）を駆使してユーザー体験（UX）を最大化すること。
>
> **要求 (Task):**
>
> 1. ユーザーの意図を深く分析し、利用可能なツール（UIコンポーネント）が文脈に適しているか判断してください。
> 2. 場所やホテルの情報を提供する際は、長々としたテキストで説明するのではなく、必ず `showPlace` ツールを実行してください。
> 3. ツールを呼び出す際は、`name` (場所の名前), `description` (魅力的な説明), `rating` (1-5の評価) のパラメーターを正確に生成してください。
>
> **制約事項 (Constraints):**
>
> - `showPlace` ツールを連続して3つ以上呼び出さないこと（画面の専有を防ぐため）。
> - UIコンポーネント内で表現される情報を、通常のテキストメッセージで重複して出力しないこと。
>
> **注意事項 (Warning):**
>
> - 存在しない架空の場所や、根拠のない評価を生成しないこと（ハルシネーションの厳格な防止）。

#### 💻 実装ステップ: Tambo SDKとAIの連携

Tambo SDKを利用する場合、`pnpm install @tambo/sdk ai` でパッケージを導入した後、以下のようにコンポーネントとZodスキーマを登録するだけで、極めてスムーズにジェネレーティブUIを実装できます。

```tsx
// lib/tambo.ts
import { createTambo } from "@tambo/sdk";
import { PlaceCard } from "../components/PlaceCard";
import { z } from "zod";

// AIが呼び出す「UIツール」の定義
export const tambo = createTambo({
  components: {
    showPlace: {
      component: PlaceCard,
      description: "特定の場所や観光地の情報をリッチなカード形式で表示します。",
      schema: z.object({
        name: z.string().describe("場所の正式名称"),
        description: z.string().describe("ユーザーの興味を惹く短い説明"),
        rating: z.number().min(1).max(5).describe("5段階評価"),
      }),
    },
  },
});
```

## 💡 作成者コメント (Insight)

従来のVercel AI SDK単体でもジェネレーティブUI（`ai/rsc` 等）の実装は可能でしたが、ストリームのパース処理やクライアントコンポーネントへの受け渡しなど、ボイラープレートコードが肥大化しやすいという課題がありました。

Tambo SDKの最大の魅力は、**「コンポーネント」と「Zodスキーマ」をマッピングするだけで、ツール呼び出しからUIレンダリングまでの複雑なパイプラインをSDK側が完全に自動化してくれる点**にあります。これは、B2B SaaSのダッシュボードや複雑なデータ入力を伴うエージェントUIにおいて、開発工数を圧倒的に削減する強力な武器となります。実務上のコツとしては、スキーマの `describe` に記述する説明文（実質的なプロンプトの一部）をどれだけ具体的に書き込めるかが、AIの挙動を安定させる決定的な要因となります。

## 🙋 よくある質問 (FAQ)

- **Q: Tambo SDKはReact以外のフレームワーク（VueやSvelteなど）でも利用可能ですか？**
  - A: 現状、Tambo SDKはReactエコシステム（Next.jsなど）に特化して最適化されています。他のフレームワークを採用している場合は、Vercel AI SDKのコア機能を活用して独自に実装を組む必要があります。

- **Q: AIが意図しない不適切なタイミングでUIコンポーネントを描画してしまいます。**
  - A: プロンプトの **制約事項 (Constraints)** をさらに強固にするか、Tamboレジストリ内で定義した各コンポーネントの `description` をより厳格な発火条件（例：「ユーザーから"予約したい"という明確な意思表示があった場合のみ使用する」など）に書き換えることで、精度を大幅に改善できます。

- **Q: 独自のUIライブラリ（shadcn/uiなど）と組み合わせて使用することは可能ですか？**
  - A: はい、完全に互換性があります。Tamboはあくまで連携を担うレイヤーであるため、コンポーネントの中身自体は `shadcn/ui` や `Tailwind CSS` などで自由にスタイリングしたものをそのまま適用できます。

## 🧬 プロンプト解剖 (Why it works?)

1. **RoleとContextの明確化:** AIに対して単なるチャットボットではなく「専門的なUX/UIエージェント」であるという明確なペルソナを付与することで、単純なテキスト出力よりもツール呼び出し（UI生成）を積極的に優先するよう誘導しています。
2. **情報の冗長性排除（制約）:** 「UIコンポーネントで表現される情報はテキストで重複させない」と厳格に指示することで、画面上に同じ情報が二重に表示されてしまうUXの著しい劣化を未然に防いでいます。
3. **Zodスキーマへのプロンプト埋め込み:** コード内に記述された `.describe('...')` が実質的なマイクロプロンプトとして機能し、AIがコンポーネントのレンダリングに必要なPropsを極めて正確に推論・生成するための強力なガイドとなっています。

## 📊 証明: Before & After

### ❌ Before (従来のチャットインターフェース)

従来のチャットボットでは、情報をすべてテキストで羅列するため視認性が低く、ユーザーを次のアクションに誘導しきれない課題がありました。

```text
ユーザー: 済州島のおすすめホテルを教えて。

AI: 済州島のおすすめホテルは以下の通りです。
1. シラホテル済州 (評価: 4.8) - 豪華なリゾート体験ができます。
2. ロッテホテル済州 (評価: 4.7) - 家族連れに最適なプールがあります。
（※テキストが長く、視認性が低く、直接予約アクションに繋がらない）
```

### ✅ After (Tambo SDKによるジェネレーティブUI)

```text
ユーザー: 済州島のおすすめホテルを教えて。

AI: 素晴らしい選択ですね。済州島で人気のリゾートホテルを2つピックアップしました。

[ 🏨 シラホテル済州の画像付きカードUI ]
⭐ 4.8 | 豪華なリゾート体験ができます。
[ 予約ページへ進む (ボタン) ]

[ 🏨 ロッテホテル済州の画像付きカードUI ]
⭐ 4.7 | 家族連れに最適なプールがあります。
[ 予約ページへ進む (ボタン) ]
```

※ AIは短い挨拶のみを出力し、実際のホテル情報はTambo SDKを通じてリッチなReactコンポーネント（PlaceCard）として即座に画面上にレンダリングされます。

## 🎯 結論

ジェネレーティブUIは、ユーザーとAIの対話をより直感的でアクション指向な体験へと根本から進化させます。Tambo SDKと精緻に練り上げられたシステムプロンプトを組み合わせることで、フロントエンド開発者は複雑なインフラ構築や状態管理に頭を悩ませることなく、次世代のUX設計そのものにリソースを集中できるようになります。

まずはシンプルなカードコンポーネント一つから、あなたのアプリケーションに「AIが自律的に生成するUI」を組み込み、その圧倒的な体験の違いを実感してみてください！ 🍷
