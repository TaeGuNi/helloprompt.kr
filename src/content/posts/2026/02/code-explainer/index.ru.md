Я использую этот промпт почти каждый день, особенно когда приходится погружаться в проекты на языках, с которыми я редко работаю (например, когда нужно быстро подправить Bash-скрипт или разобраться в чужом регулярном выражении). А если указать "Я мидл-разработчик, переходящий с Java на Python", он будет объяснять код, проводя аналогии с Java. Это колоссально экономит время на гугление незнакомых конструкций!

## 🙋 Часто задаваемые вопросы (FAQ)

- **Q: Можно ли использовать этот промпт для анализа целых файлов (сотни строк)?**
  - A: Да, современные модели (особенно Claude 3.5 Sonnet и Gemini 2.5 Pro) отлично справляются с большими файлами. Однако для лучшего результата рекомендую анализировать код логическими блоками по 50-100 строк (например, по одной сложной функции за раз).

- **Q: Справляется ли ИИ с непопулярными языками программирования?**
  - A: Вполне неплохо, но чем популярнее язык (Python, JS, Java, C++), тем качественнее будет объяснение синтаксических нюансов. Для редких языков (например, Haskell или старых версий Fortran) ИИ может делать предположения, поэтому всегда проверяйте логику.

- **Q: Что делать, если код использует внутренние библиотеки компании?**
  - A: ИИ не знает о ваших внутренних библиотеках. В таком случае добавьте в блок `Ситуация (Context)` пару строк с объяснением, что делают эти проприетарные функции, иначе ИИ начнет фантазировать (галлюцинировать) об их назначении.

## 🧬 Анатомия промпта (Why it works?)

1. **Задание уровня аудитории:** Указание вашего текущего грейда заставляет ИИ калибровать сложность языка. Он не будет объяснять Junior'у, что такое цикл `for`, но детально разберет, почему здесь применен генератор.
2. **Требование контекстных комментариев:** Установка "объяснять _почему_, а не _что_" предотвращает появление бесполезных комментариев вроде `i++ // увеличивает i на 1`.
3. **Изоляция сложных концепций:** Блок "Разбор синтаксиса" выносит тяжелую теорию за пределы кода, сохраняя листинг чистым и читаемым.

## 📊 Доказательство: До и После

### ❌ До (Непонятный фрагмент на Python)

```python
result = [x * 2 for x in range(10) if x % 2 == 0]
```

### ✅ После (Результат работы ИИ)

**Обзор:**
Этот код создает новый список, в котором содержатся только четные числа от 0 до 8, причем каждое из них умножено на 2.

**Построчный анализ:**

```python
# Мы проходим по числам от 0 до 9 (включительно) с помощью range(10)
# Условие 'if x % 2 == 0' отфильтровывает только четные числа
# Выражение 'x * 2' удваивает каждое прошедшее фильтр число
# Результат автоматически собирается в новый список 'result'
result = [x * 2 for x in range(10) if x % 2 == 0]
```

**Разбор синтаксиса:**

- **List Comprehension (Генератор списков):** Это особая фишка Python, позволяющая создавать новые списки в одну короткую строку, заменяя собой классический цикл `for` с вложенным условием `if` и методом `.append()`.
- **Оператор `%` (Остаток от деления):** Проверка `x % 2 == 0` — это стандартный паттерн в программировании для определения четности числа (если делится на 2 без остатка, значит четное).
