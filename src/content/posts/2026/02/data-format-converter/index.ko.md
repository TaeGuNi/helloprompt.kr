# 📝 엑셀, JSON, CSV... 데이터 포맷 변환 노가다 탈출

- **🎯 추천 대상:** 개발자, 데이터 분석가, 퍼포먼스 마케터
- **⏱️ 소요 시간:** 30분 → 1분 단축
- **🤖 추천 모델:** Claude 3.5 Sonnet (복잡한 구조 변환에 탁월함), ChatGPT (GPT-4o)

- ⭐ **난이도:** ⭐☆☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"대리님, 이 엑셀 데이터 웹에 올려야 하는데 JSON으로 좀 바꿔주실 수 있나요? 아, 그리고 키(Key) 값은 전부 영문 소문자 스네이크 케이스로 통일 부탁드려요."_

![\"엑셀, JSON, CSV... 데이터 포맷 변환 노가다 탈출\"](/images/hooks/data-format-converter.jpg)

데이터를 다루다 보면 필연적으로 '포맷 변환'이라는 단순 반복 노동과 마주하게 됩니다.
온라인 무료 변환 툴을 쓰자니 회사 보안 규정 때문에 찝찝하고, 파이썬(Python) 스크립트나 정규식을 직접 짜자니 '고작 이거 하나 하려고 코딩까지 해야 하나' 싶어 깊은 현타가 오곤 하죠.
이제 복잡하게 생각하지 말고 데이터를 그대로 복사해서 AI에게 던져주세요. 단순한 포맷 변환은 물론, 데이터 구조 매핑, 필드명 변경, 불필요한 값 제거까지 단 한 번의 프롬프트로 완벽하게 처리해 줍니다.

## ⚡️ 3줄 요약 (TL;DR)

1. **보안 걱정 끝:** 로컬 환경의 AI(또는 엔터프라이즈 AI)를 활용하면 외부 툴 유출 리스크 없이 안전하게 데이터 변환이 가능합니다.
2. **복합 변환 수행:** 단순 포맷 변경(CSV → JSON)을 넘어, 키(Key)값 매핑과 데이터 타입(Type) 캐스팅까지 동시에 처리합니다.
3. **즉시 사용 가능한 쿼리:** DB 마이그레이션을 위한 SQL Insert 문이나 파이썬 데이터프레임 코드로도 다이렉트 생성이 가능합니다.

## 💡 작성자 코멘트 (Insight)

이 프롬프트의 진정한 가치는 **'데이터 정제(Data Cleansing)'와 '포맷 변환'을 단일 파이프라인으로 통합**했다는 점에 있습니다.
실무 현장에서는 단순히 CSV를 JSON으로 바꾸는 것보다, 프론트엔드 API 명세서 규격에 맞게 키(Key) 값을 카멜 케이스(camelCase)로 일괄 수정하거나, 중구난방인 날짜 포맷(`YYYY-MM-DD`)을 통일하는 등의 전처리 작업이 훨씬 번거롭고 시간이 오래 걸립니다.
Pro 버전의 `[변환 규칙]` 항목에 정규식으로도 구현하기 까다로운 엣지 케이스("빈 값은 null로 처리하고, 직업 란이 비어있는 사람은 '무직'으로 통일해")를 자연어로 툭 던져보세요. 놀라운 정확도로 데이터를 입맛에 맞게 가공해 줍니다. 특히 Claude 3.5 Sonnet 모델은 깊고 복잡한 JSON 중첩 구조(Nested Object)를 파싱하고 재구성할 때 압도적인 성능을 자랑합니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 엑셀 파일(xlsx)을 직접 첨부해서 변환해도 되나요?**
  - A: Advanced Data Analysis(데이터 분석) 기능이 탑재된 ChatGPT(GPT-4o)나 Claude에서는 엑셀 파일을 직접 업로드해도 무리 없이 작동합니다. 단, 회사 보안에 민감한 데이터라면 파일 첨부 기능 대신 직접 내용의 일부만 복사해서 프롬프트에 붙여넣고 중요 정보는 마스킹 처리하는 방식을 강력히 권장합니다.

- **Q: 변환해야 할 데이터가 수만 줄이 넘어가면 어떻게 처리하나요?**
  - A: 데이터가 너무 방대할 경우 LLM의 컨텍스트 윈도우(Token 한도)를 초과하거나, 출력이 중간에 끊기는 현상이 발생할 수 있습니다. 이럴 때는 프롬프트의 목적을 틀어 "이 변환 규칙을 완벽하게 수행하는 파이썬(Python) 코드를 작성해 줘"라고 요청하세요. 생성된 스크립트를 로컬 PC에서 직접 실행하는 것이 대용량 데이터 처리에는 가장 안전하고 확실한 방법입니다.

- **Q: 결과물이 다 출력되지 않고 중간에 뚝 끊겨서 나옵니다.**
  - A: 대화창에 "계속해 줘(Continue)"라고 입력하면 끊긴 부분부터 이어서 출력해 줍니다. 애초에 프롬프트 하단 제약사항(Constraints) 부분에 "결과물이 길더라도 임의로 생략하지 말고, 나누어서라도 끝까지 출력해 줘"라는 조건을 명시해 두는 것도 좋은 실무 팁입니다.

## 🧬 프롬프트 해부 (Why it works?)

1.  **ETL 전문가 페르소나 부여:** AI에게 단순한 '포맷 변환기'가 아닌 '시니어 데이터 엔지니어' 역할을 부여함으로써, 스스로 데이터의 무결성(Integrity)과 데이터 타입(Type)을 꼼꼼하게 검증하고 챙기도록 유도했습니다.
2.  **명확한 파라미터 분리 구조:** 입력(소스), 출력(타겟), 그리고 제어 규칙(변환 룰)을 마크다운 문법으로 명확히 블록화하여, AI가 복잡한 지시사항을 혼동하지 않고 순차적이고 논리적으로 처리하도록 설계했습니다.
3.  **엄격한 출력 통제 (Constraints):** "설명은 생략하고 코드블럭만 출력해"라는 강력한 제약을 걸어, 사용자가 변환이 완료된 데이터를 곧바로 '복사-붙여넣기' 하여 업무에 적용할 수 있도록 UX(사용자 경험)를 극대화했습니다.

## 📊 증명: Before & After

단순한 지시어와 엔지니어링 된 프롬프트의 결과물 차이는 극명합니다. 제공된 프롬프트를 활용하면 훨씬 더 구체적이고 구조화된 답변을 얻을 수 있으며, 소모적인 반복 작업에 들어가는 귀중한 시간을 극적으로 단축할 수 있습니다.

### ❌ Before (입력 데이터 - CSV 형태의 지저분한 텍스트)

```csv
고객 이름,나이,가입 일자,직업,개인정보_주민번호
홍길동,삼십,2023/01/05,백엔드 개발자,900101-1234567
김철수,25,23-11-12,,990203-2345678
```

### ✅ After (결과 - 정제된 타겟 JSON Array)

**요청한 규칙:** 한글 헤더명을 영문 스네이크 케이스로 변경, 나이는 숫자형(Integer)으로 치환, 가입일자는 `YYYY-MM-DD` 표준 포맷으로 통일, 직업이 비어있으면 '미상'으로 처리, 민감한 주민번호 필드 삭제.

```json
[
  {
    "customer_name": "홍길동",
    "age": 30,
    "join_date": "2023-01-05",
    "job": "백엔드 개발자"
  },
  {
    "customer_name": "김철수",
    "age": 25,
    "join_date": "2023-11-12",
    "job": "미상"
  }
]
```
