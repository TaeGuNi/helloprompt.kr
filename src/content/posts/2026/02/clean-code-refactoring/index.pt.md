# 📝 Meu Código é Lixo! Treinador de Refatoração Clean Code

- **🎯 Recomendado para:** Desenvolvedores de todos os níveis, Analistas de Sistemas, Engenheiros de Software
- **⏱️ Tempo necessário:** 5 minutos → Reduzido para menos de 1 minuto
- **🤖 Modelo recomendado:** Qualquer modelo de IA conversacional (ChatGPT, Claude, Gemini, etc.)

- ⭐ **Dificuldade:** ⭐⭐☆☆☆
- ⚡️ **Eficácia:** ⭐⭐⭐⭐⭐
- 🚀 **Utilidade:** ⭐⭐⭐⭐☆

> _"Eu disse que consertaria mais tarde, mas um ano se passou. Agora tenho até medo de encostar nesse código."_

Código espaguete é o maior pesadelo de qualquer desenvolvedor. Quando a legibilidade é baixa, encontrar bugs e adicionar novas funcionalidades torna-se um verdadeiro tormento. Este prompt atua como o próprio Robert C. Martin (autor de _Clean Code_), limpando e polindo seu código até que ele brilhe com excelência.

---

## ⚡️ Resumo em 3 linhas (TL;DR)

1. **Eliminação de Redundâncias:** Remove códigos duplicados e separa responsabilidades (Extract Method).
2. **Nomenclatura Semântica:** Melhora nomes obscuros de variáveis e funções, adotando convenções claras e intuitivas.
3. **Otimização e Modernização:** Refatora a lógica para melhorar o desempenho e aplica as sintaxes mais modernas disponíveis para a sua linguagem.

### 🥇 Versão Pro (Expert Version)

Use esta versão quando a qualidade dos detalhes e a arquitetura forem cruciais.

> **Papel (Role):** Você é um `[Evangelista de Clean Code e Arquiteto de Software Sênior]`.
>
> **Contexto (Context):**
>
> - Cenário: Meu código atual funciona, mas está muito confuso, bagunçado e difícil de entender (Código Sujo / Spaghetti Code).
> - Objetivo: Refatorá-lo seguindo as melhores práticas para que se torne um código altamente legível e fácil de manter a longo prazo.
>
> **Tarefa (Task):**
>
> 1. **[Nomenclatura Semântica]**: Renomeie variáveis e funções para que sejam intuitivas e representem claramente seus propósitos (ex: de `a` para `userAge`).
> 2. **[Melhoria Estrutural]**: Divida funções longas e complexas com base no Princípio de Responsabilidade Única (SRP) e unifique lógicas duplicadas (DRY).
> 3. **[Modernização]**: Sempre que possível, aplique a sintaxe e os recursos mais recentes da linguagem (ES6+, Python 3.10+, Java 21+, etc.).
> 4. **[Análise de Impacto Antes & Depois]**: Apresente as mudanças detalhadamente e justifique as razões arquiteturais e lógicas de cada alteração.
>
> **Restrições (Constraints):**
>
> - O comportamento final do código (sua lógica de negócios) NUNCA deve ser alterado.
> - Adicione comentários apenas quando for estritamente necessário para explicar o "porquê" de algo complexo. Remova comentários inúteis que explicam "o que" o código faz (o próprio código deve ser autoexplicativo).
>
> **Código Alvo:**
>
> `[Cole o seu código desorganizado aqui]`

## 💡 Comentário do Autor (Insight)

A principal barreira para aplicar _Clean Code_ no dia a dia não é a falta de conhecimento, mas a **pressão do tempo**. Em ambientes de entrega contínua, muitas vezes o "feito" supera o "perfeito". Este prompt atua como seu par de programação sênior, assumindo a carga cognitiva de organizar a estrutura sem quebrar a lógica do negócio.

Uma excelente forma de utilizá-lo é durante a preparação para um _Code Review_ (Pull Request). Antes de submeter suas alterações, passe os arquivos modificados por este prompt. Ele não só melhorará a qualidade da sua entrega, como também ensinará padrões mais eficientes da linguagem que você talvez não estivesse utilizando, acelerando o seu crescimento profissional.

## 🙋 Perguntas Frequentes (FAQ)

- **Q: O prompt pode acidentalmente quebrar a lógica do meu programa?**
  - **A:** Teoricamente não, pois definimos explicitamente nas restrições que o comportamento deve permanecer intacto. No entanto, é **fundamental** ter testes unitários ou realizar validações manuais após a refatoração. A IA pode ocasionalmente introduzir bugs sutis ao interpretar incorretamente uma lógica que não estava devidamente documentada.

- **Q: Funciona bem com qualquer linguagem de programação?**
  - **A:** Sim! O prompt é agnóstico em relação à linguagem. Quer você esteja programando em JavaScript, Python, Go, Rust ou C#, os princípios de _Clean Code_ (como SRP, DRY e nomenclaturas claras) são universais. A IA adaptará as sugestões sintáticas à linguagem específica do seu código original.

- **Q: Devo colar o arquivo inteiro ou apenas a função problemática?**
  - **A:** Para obter os melhores resultados, cole apenas a função, a classe ou o bloco de código que você deseja melhorar (geralmente até 200-300 linhas). Se você fornecer um arquivo imenso, a IA pode cortar a resposta ou perder o foco nos detalhes importantes.

## 🧬 Anatomia do Prompt (Why it works?)

1. **Atribuição de Papel e Autoridade:** Ao definir a IA como "Evangelista de Clean Code e Arquiteto de Software Sênior", calibramos seu padrão de respostas para fornecer análises profundas e baseadas em princípios sólidos da engenharia de software (como os ensinamentos de _Uncle Bob_).
2. **Restrições Protetoras:** A regra rigorosa de "nunca alterar a operação do código" atua como uma barreira contra alucinações. O modelo é forçado a focar apenas na estrutura, evitando a reinvenção desnecessária da lógica de negócios.
3. **Foco na Modernização:** A instrução específica para aplicar as sintaxes mais recentes força o modelo a entregar uma refatoração que não apenas melhora a legibilidade, mas também o desempenho e a idiomaticidade do código, removendo padrões obsoletos.

## 📊 Prova: Antes e Depois

### ❌ Antes (Entrada)

```python
def calc(x, y):
    if x > 19:
        print("Adulto")
        p = y * 0.9
        return p
    else:
        print("Estudante")
        return y
```

(O nome da função `calc` não reflete o que ela faz. Os parâmetros `x` e `y` e a variável local `p` são completamente obscuros, exigindo adivinhação do leitor.)

### ✅ Depois (Resultado)

```python
def calculate_discounted_price(age, price):
    ADULT_AGE_THRESHOLD = 20
    DISCOUNT_RATE = 0.9

    if age >= ADULT_AGE_THRESHOLD:
        print("Você é um adulto.")
        return price * DISCOUNT_RATE

    print("Você é um estudante.")
    return price
```

**[Pontos de Melhoria Observados]**

1. **Nomenclatura Clara:** A função `calc` foi renomeada para `calculate_discounted_price`, revelando imediatamente o seu propósito. Os parâmetros também foram esclarecidos (`age` e `price`).
2. **Remoção de Números Mágicos (Magic Numbers):** Os valores numéricos soltos (`19` e `0.9`) foram isolados em constantes nomeadas (`ADULT_AGE_THRESHOLD` e `DISCOUNT_RATE`), adicionando contexto e facilitando futuras atualizações.
3. **Fim do `Else` Desnecessário:** A profundidade de indentação foi reduzida e a complexidade ciclomática diminuiu consideravelmente através do uso do padrão de retorno antecipado (_Early Return_).
