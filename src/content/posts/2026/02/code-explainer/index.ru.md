## 🙋 Часто задаваемые вопросы (FAQ)

- **Q: Можно ли использовать этот промпт для анализа целых файлов (на сотни строк)?**
  - A: Да, современные модели (особенно Claude 3.5 Sonnet и Gemini 2.5 Pro) отлично справляются с объемными файлами. Однако для получения действительно глубокого результата я настоятельно рекомендую анализировать код логическими блоками по 50–100 строк (например, скармливать по одной сложной функции за раз).

- **Q: Справляется ли ИИ с редкими или непопулярными языками программирования?**
  - A: Вполне неплохо, но нужно понимать: чем популярнее язык (Python, JS, Java, C++), тем качественнее и точнее будет объяснение синтаксических нюансов. Для редких языков (например, Haskell или старых версий Fortran) ИИ может делать логические предположения, поэтому всегда критически оценивайте результат.

- **Q: Что делать, если код использует внутренние проприетарные библиотеки компании?**
  - A: ИИ не имеет доступа к вашей закрытой кодовой базе и не знает о внутренних библиотеках. В таком случае просто добавьте в блок `Ситуация (Context)` пару строк с кратким объяснением того, что делают эти кастомные функции. Иначе модель начнет фантазировать (галлюцинировать) об их истинном назначении.

## 🧬 Анатомия промпта (Why it works?)

1. **Задание уровня аудитории:** Указание вашего текущего грейда заставляет ИИ калибровать сложность ответа. Он не станет объяснять Junior-специалисту, как работает базовый цикл `for`, но детально разберет, почему в конкретном случае применен генератор.
2. **Требование контекстных комментариев:** Строгая установка «объяснять **почему**, а не **что**» на корню пресекает появление бесполезных комментариев в духе `i++ // увеличивает i на 1`.
3. **Изоляция сложных концепций:** Блок «Разбор синтаксиса» выносит тяжелую сухую теорию за пределы самого кода, сохраняя итоговый листинг чистым, лаконичным и читаемым.

## 📊 Доказательство: До и После

### ❌ Before (Непонятный фрагмент на Python)

```python
result = [x * 2 for x in range(10) if x % 2 == 0]
```

### ✅ After (Результат работы ИИ)

**Обзор:**
Этот код создает новый список, в котором содержатся только четные числа от 0 до 8, причем каждое из них умножено на 2.

**Построчный анализ:**

```python
# Мы проходим по числам от 0 до 9 (включительно) с помощью range(10)
# Условие 'if x % 2 == 0' отфильтровывает только четные числа
# Выражение 'x * 2' удваивает каждое прошедшее фильтр число
# Результат автоматически собирается в новый список 'result'
result = [x * 2 for x in range(10) if x % 2 == 0]
```

**Разбор синтаксиса:**

- **List Comprehension (Генератор списков):** Это особая фишка Python, позволяющая создавать новые списки в одну короткую строку, заменяя собой классический цикл `for` с вложенным условием `if` и методом `.append()`.
- **Оператор `%` (Остаток от деления):** Проверка `x % 2 == 0` — это стандартный паттерн в программировании для определения четности числа (если делится на 2 без остатка, значит четное).
