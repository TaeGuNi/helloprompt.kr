## 📝 散らかったコードスニペットをAIで完璧に整理！（AI Code Snippet Manager）

- **🎯 おすすめの対象:** 1〜3年目のジュニアエンジニア、自分だけのコードWikiを作りたい方、コピペしたコードが溢れかえっている方
- **⏱️ 所要時間:** 30分 → 1分に短縮
- **🤖 おすすめのモデル:** Claude 3.5 Sonnet（コードのリファクタリングと文書化に最適）、ChatGPT (GPT-4o)

- ⭐ **難易度:** ⭐☆☆☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「昔書いたあのコード、どこに保存したっけ…？」検索窓を睨みつけながら30分を無駄にしているなら、その非効率な作業は今すぐ終わらせましょう。_

エンジニアなら誰しも、デスクトップの片隅に「秘伝のタレ」のようなテキストファイルを眠らせているのではないでしょうか。Stack Overflowで苦労して見つけた複雑な正規表現、絶妙なユーティリティ関数、いつか必ず役立つはずのNginx設定ファイルなど、その中身は様々です。

しかし、いざ実務で必要になった時、どこに保存したのか、引数に何を渡すべきだったのか思い出せず、結局最初からGoogleで検索し直す羽目になっていませんか？コードスニペットにおいて最も重要なのは「検索のしやすさ」と「具体的な使用例」です。出処不明の断片的なコードであってもAIに任せれば、標準的なコメントフォーマットはもちろん、実践的な使用例や検索タグまで完璧に網羅された「生きたライブラリ」へと生まれ変わります。あなたの乱雑なメモ帳を、体系的で強力なナレッジベース（Knowledge Base）へと一変させる魔法のプロンプトをご紹介します。

## ⚡️ 3行まとめ (TL;DR)

1. **検索可能な資産への変換:** 出処不明のコードの塊に、明確なタイトル、検索タグ、メタデータを自動付与します。
2. **自動ドキュメント化:** パラメータや戻り値の標準的なコメント（JSDocなど）に加え、そのまま動くサンプルコードを即座に生成します。
3. **コード品質の抜本的改善:** 崩れたフォーマットや曖昧な変数名をAIがリファクタリングし、コピペですぐに再利用できる状態に整えます。

## 🚀 解決策：「AIコードスニペットマネージャー」

### 🥉 Basic Version（基本型）

コードのコメントと基本構造だけを素早く整えたい場合に使用してください。

> **役割 (Role):** あなたは経験豊富なシニアソフトウェアエンジニアリングメンターです。
>
> **タスク (Task):** 以下のコードを分析してフォーマットを整え、どのような役割を果たすコードなのかを1行で要約し、コアとなるコメントを追加してください。
>
> **コード:**
> `[ここにコピーしたコードを貼り付ける]`

### 🥇 Pro Version（専門家型）

自分だけの完璧なコードWikiや、Notion、Obsidianのデータベースに蓄積する「完成形のMarkdownドキュメント」が必要な場合に使用してください。

> **役割 (Role):**
> あなたは、分かりやすいドキュメント化とクリーンコード（Clean Code）に強いこだわりを持つ、キャリア10年目のシニアテクニカルライター兼ソフトウェアエンジニアです。
>
> **コンテキスト (Context):**
>
> - 背景: 私はインターネットや過去のプロジェクトから適当にコピーしておいたコードの断片を集め、自分だけのコードナレッジベース（Notion、Obsidian）を構築しています。
> - 目標: このコードを後で検索しやすくし、未来の自分が一瞬で理解して再利用できるように、完璧にドキュメント化されたMarkdownスニペットに変換することです。
>
> **タスク (Task):**
>
> 提供された `[コード]` を分析し、以下の構造を持つMarkdownドキュメントを作成してください。
>
> 1. メタデータ: `Title`（明確な目的）、`Language`（言語）、`Tags`（検索用のキーワードを3〜5個）
> 2. 説明 (Description): このコードがどのような問題を解決するのかを1〜2行で要約してください。
> 3. リファクタリングとコメント (Refactored Code): インデントとフォーマットを整え、JSDoc（または該当言語の標準docstring）スタイルでパラメータ（`@param`）と戻り値（`@return`）に関する詳細なコメントを追加してください。曖昧な変数名があれば、可読性が高くなるように修正してください。
> 4. 使用例 (Usage Example): このコードを実際のプロジェクトでどのように呼び出して使うのかを示す、実践的でシンプルな実行サンプルコードを作成してください。
>
> **制約事項 (Constraints):**
>
> - 出力結果は、全体を単一のMarkdownドキュメントとしてフォーマットしてください。
> - 変数名や関数名をむやみに大きく変更せず、コードのオリジナルの意図を最大限維持してください。
>
> **注意事項 (Warning):**
>
> - コード内に機密情報（IPアドレス、パスワード、APIキーなど）が含まれている場合は、解析前に警告メッセージを出力してください。
> - `[コード]` の部分は私が後から継続的に入力していくため、最初の返答では「整理するコードを入力してください。」とだけ簡潔に答えてください。

## 🙋 よくある質問 (FAQ)

- **Q: 会社のセキュリティポリシー上、外部のAIに社内コードを入力できないのですが、どうすればいいですか？**
  - A: APIキー、DBの認証情報、社内サーバーのIPアドレス、または独自のコアビジネスロジックといった「機密情報」は、必ず削除するかダミーの値（例：`XXX`、`1234`）でマスキングしてから入力してください。より厳格なセキュリティ環境が求められる場合は、ローカル環境で動作するLlama 3などのオープンソースモデルを活用することを強く推奨します。

- **Q: AIが変更したコードのフォーマットや変数名が、自分のコーディングスタイルと合わない場合は？**
  - A: プロンプト内の **制約事項 (Constraints)** セクションに、あなた独自のコーディング規約を追加してください。（例：_「変数名は常にcamelCaseを維持し、文字列にはダブルクォーテーションではなくシングルクォーテーションを使用してください」_ など）

- **Q: マニアックなプログラミング言語や、特殊なスクリプトにも対応していますか？**
  - A: はい、対応しています。Python、JavaScript、Java、C++などの主要なプログラミング言語はもちろん、複雑なSQLクエリ、Bashスクリプト、さらにはDockerfileやKubernetesのYAMLファイルの構造解析とコメント付けにも完璧に対応します。

### ✅ After (見違えるような結果)

````markdown
**Title:** 2つの日付間の日数（Days）の差を計算する関数
**Language:** JavaScript
**Tags:** `#Date`, `#Utility`, `#TimeDifference`, `#JavaScript`

**Description:**
2つの日付文字列（またはDateオブジェクト）を入力として受け取り、2つの日付間の正確な日数（Days）の差を整数で返すユーティリティ関数です。

**Refactored Code:**

```javascript
/**
 * 2つの日付間の日数（Days）の差を計算します。
 *
 * @param {string|Date} targetDate - 比較対象の日付
 * @param {string|Date} baseDate - 基準となる日付
 * @returns {number} 2つの日付間の日数の差（targetDateが未来の場合は正の数を返す）
 */
function calculateDaysDifference(targetDate, baseDate) {
  const timeDifference =
    new Date(targetDate).getTime() - new Date(baseDate).getTime();
  return Math.floor(timeDifference / (1000 * 60 * 60 * 24));
}
```
````

**Usage Example:**

```javascript
const today = "2026-02-12";
const dDay = "2026-12-25";

const daysLeft = calculateDaysDifference(dDay, today);
console.log(`クリスマスまであと${daysLeft}日です！`);
// 出力: クリスマスまであと316日です！
```
