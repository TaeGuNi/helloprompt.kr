# Структурирование сложных промптов: Фреймворк CREATE

По мере развития моделей ИИ мы поручаем им все более сложные задачи. Хотя для простых вопросов достаточно простых промптов, инструктаж ИИ по сложной бизнес-логике или творческим задачам требует систематического подхода.

Эта статья представляет **Фреймворк CREATE** — метод логического структурирования промптов для максимизации производительности ИИ.

## Что такое Фреймворк CREATE?

CREATE расшифровывается как шесть ключевых элементов, необходимых для эффективного составления промптов.

1.  **C**ontext (Контекст): Назначение роли и ситуации для ИИ.
2.  **R**equest (Запрос): Четкое формулирование конкретной задачи.
3.  **E**xamples (Примеры): Предоставление примеров (Few-shot) желаемого результата.
4.  **A**djustments (Настройки): Установка тона, стиля и ограничений.
5.  **T**ype of Output (Тип вывода): Указание формата результата (таблица, код, markdown и т.д.).
6.  **E**xtras (Дополнительно): Обработка крайних случаев или специальные инструкции.

---

## Пошаговое руководство

### 1. Context (Контекст)

Придание ИИ персоны значительно меняет качество ответа.

> "Вы — старший инженер-программист с 10-летним опытом. Вы специализируетесь на рефакторинге устаревшего кода в современный чистый код."

### 2. Request (Запрос)

Избегайте двусмысленности и используйте четкие глаголы.

> "Проанализируйте приведенную ниже функцию Python и выполните ее рефакторинг для улучшения читаемости и оптимизации скорости выполнения."

### 3. Examples (Примеры)

ИИ обучается шаблонам на примерах. Покажите пары входных и выходных данных.

> **Вход:**
> `def calc(x,y): return x+y`
>
> **Выход:**
>
> ```python
> def add_numbers(a: int, b: int) -> int:
>     """Возвращает сумму двух чисел."""
>     return a + b
> ```

### 4. Adjustments (Настройки)

Определите тон и манеру ответа.

> "Делайте объяснения краткими и используйте профессиональную терминологию, но сохраняйте дружелюбный тон, доступный для младших разработчиков."

### 5. Type of Output (Тип вывода)

Укажите, как вы хотите получить результат.

> "Представьте результат, сначала показав полный измененный код в блоке кода, а затем краткое резюме изменений в виде маркированного списка."

### 6. Extras (Дополнительно)

Предотвращение неожиданных ситуаций.

> "Если в коде будет обнаружена уязвимость безопасности, пожалуйста, выведите отдельное предупреждающее сообщение в дополнение к рефакторингу."

## Заключение

Использование фреймворка CREATE позволяет сократить время написания промптов, сохраняя при этом постоянное качество вывода. Сталкиваясь со сложными проблемами, вместо того чтобы спрашивать расплывчато, организуйте свои мысли в соответствии со структурой CREATE. ИИ ответит настолько разумно, насколько вы его спроектируете.
