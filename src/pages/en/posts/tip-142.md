### âœ… After (Prompt Result)

**[Cause Analysis]**
It happened because you tried to render the screen before receiving data from the API. The variable `userList` is still in an `undefined` state because data hasn't come in yet, but you used the `.map()` function on it, so an error occurred. It's an asynchronous processing issue.

**[Solution]**

```text
// Before fix: userList.map(...)
// After fix: Use optional chaining (?.)
userList?.map(user => ...)
```

I added the `?.` operator to execute map only when there is data.

**[Prevention]**
When writing API communication code, it's good to always assume 'data might come late' or 'might fail' and insert code like `if (!data) return loading...` first.
