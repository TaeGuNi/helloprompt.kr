## 💡 作者点评 (Insight)

在使用 AI 辅助编程时，最重要的一点是**“不要仅仅将 AI 视为自动补全工具，而是将其作为讨论结构、设计架构的资深搭档”**。
不要妄想一开始就打造出一款完美无瑕的庞大游戏，而是应该遵循：核心物理机制（摩擦力、碰撞） -> 输入处理（拖拽击球反馈） -> 内容扩展（添加关卡设计）的**逐步构建 (Step-by-step)** 流程，这才是提高成功率的秘诀。

特别是 Claude Code，它在保持上下文并重构代码方面的能力极强。在实现过程中，如果你提出结构性优化的要求，例如“请将管理障碍物的逻辑抽离成一个单独的类进行重构”，你将获得一份极易维护的高质量代码。

## 🧬 提示词剖析 (Why it works?)

1. **强制模块化 (Modularity)：** 从一开始就强制要求分离文件，搭建好了架构骨架。这样即使代码量增加，也不会变成意大利面条式代码，便于后续的维护和扩展。
2. **明确视觉反馈 (Visual Indicator)：** 不仅仅局限于实现操作，还明确指出了需要“显示力度和方向的辅助线”，极大地提升了游戏的操控感 (UX) 和直观性。
3. **兼顾扩展性 (Level Design)：** 引导 AI 采用“数组形式的关卡数据配置”，而非硬编码单一关卡，从而建立了一个数据驱动的结构，方便开发者未来轻松地无限扩展关卡。

### ✅ After (应用专业版提示词后生成的物理逻辑示例)

```javascript
// Generated by Claude Code
update() {
  this.velocity.x *= 0.98; // 摩擦力：模拟草地阻力，使球速缓慢衰减
  this.velocity.y *= 0.98;

  this.x += this.velocity.x;
  this.y += this.velocity.y;

  // 画布墙壁碰撞处理（自然反弹）
  if (this.x < 0 || this.x > canvas.width) {
    this.velocity.x *= -1; // 翻转 X 轴方向
  }
  if (this.y < 0 || this.y > canvas.height) {
    this.velocity.y *= -1; // 翻转 Y 轴方向
  }
}
```

_结果：Claude Code 准确运用了阻尼 (Damping) 数学概念（随时间推移降低速度），完美还原了“高尔夫球场草地的摩擦力”。_
