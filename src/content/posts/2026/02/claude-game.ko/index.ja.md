# ⛳️ Claude Codeでゴルフゲームを作成：AIゲーム開発の新時代

- **🎯 おすすめの対象:** ゲーム開発の初心者、フロントエンド開発者、AIコーディングアシスタントの活用法を知りたい実務者
- **⏱️ 所要時間:** 1時間 → 10分に短縮（プロトタイプ基準）
- **🤖 おすすめのモデル:** Claude 3.7 Sonnet (Claude Code環境)

- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「複雑な物理エンジンやCanvasレンダリングを、もうゼロからコーディングする必要はありません。AIエージェントが、あなたの完璧なシニア開発パートナーになります。」_

ゲーム開発のパラダイムが完全に変わろうとしています。過去には、2D物理エンジンを実装し、イベントリスナーを管理するだけでも数日を要していましたが、現在では**Claude Code**のような自律型AIコーディングエージェントのおかげで、そのハードルは事実上なくなりました。単なるコードの自動補完を超え、プロジェクトのアーキテクチャを理解して自らコードを記述するClaude Codeを活用し、わずか1時間でブラウザベースの2Dゴルフゲームを完成させる実践プロセスを、段階的なプロンプトとともに共有します。

---

## ⚡️ 3行まとめ (TL;DR)

1. **AIエージェントの進化:** Claude Codeは単なるコードスニペットの生成にとどまらず、物理演算（速度、摩擦）からレンダリングのロジックまで、ゲームアーキテクチャ全体を主導的に設計します。
2. **圧倒的な生産性:** 煩雑なマウスイベントの処理や衝突計算の数式に悩む必要はなく、プロンプトの指示だけで「ドラッグ＆シュート（Drag-to-shoot）」のメカニズムを瞬時に実装できます。
3. **拡張性の高いモジュール化:** AIが最初から `Physics`, `Input`, `Level` クラスを分離してモジュール化されたコードを作成するため、今後の機能追加（マルチプレイ、新レベルなど）が非常に容易になります。

## 🚀 解決策：「Claude Code ゲーム開発プロンプト」

### 🥉 Basic Version（基本型）

素早くゲームの骨組み（ボイラープレート）を作りたい時に使用する初期設定のプロンプトです。

> **役割:** あなたは `[シニアゲーム開発者]` です。
> **タスク:** `[HTML5 Canvas]` を使用して、ブラウザベースの2Dゴルフゲームを作成してください。ボールとホールがあり、マウスでドラッグして打つ（drag-to-shoot）方式にしてください。

\

### 🥇 Pro Version（専門家型）

物理エンジン、入力処理、レベルデザインまで完全に分離された、モジュール型のゲームアーキテクチャを構築する際に使用する高度なプロンプトです。

> **役割 (Role):** あなたは10年の経験を持つ `[シニアフロントエンド・ゲームクライアント開発者]` であり、完璧なペアプログラミングのパートナーです。
>
> **状況 (Context):**
>
> - 背景: 純粋なVanilla JSとHTML5 Canvasのみを使用して、軽量な2Dトップビューのゴルフゲームを開発しようとしています。
> - 目標: 物理エンジン、入力イベント、レベルの状態管理が完全にモジュール化され、保守性の高いコードを記述することです。
>
> **タスク (Task):**
>
> 1. `index.html`（Canvasコンテナ）、`game.js`（メインループ）、`physics.js`（ベクトルおよび摩擦・衝突の数学演算）にファイルを分離して構造化してください。
> 2. `physics.js`には、芝生の摩擦（Friction）や壁に衝突した際の跳ね返り（Bounce）のロジックを精巧に実装してください。
> 3. マウスの `mousedown`, `mousemove`, `mouseup` イベントを活用し、ボールを引いた時の軌道とパワーを示す視覚的なインジケーター（Indicator Line）機能を追加してください。
> 4. `Obstacle` クラスを作成し、難易度が徐々に上がる3つの異なるレベル（Level）配列を構成してください。
>
> **制約事項 (Constraints):**
>
> - 外部のゲームエンジンライブラリ（Phaser.jsなど）は絶対に使用しないでください。
> - すべての主要なロジックにはコメントを追加し、動作原理を説明してください。
> - モジュールパターンまたはES6のClass構文を使用し、オブジェクト指向のコードを記述してください。

## 💡 筆者のコメント (Insight)

このプロンプトの核心は、AIを単なる「コード生成器」ではなく「アーキテクト」として扱う点にあります。最初から「ファイルを分離して」「クラスを分けて」と明確に指示（Constraints）しない限り、AIはすべてのコードを `index.html` 1つのファイルに詰め込み、スパゲッティコードにしてしまう傾向があります。特に、物理演算と状態管理を別々のモジュールに分離するよう強制することで、後から風（Wind）の抵抗を追加したり、WebSocketsを連携させたりする際の修正範囲を劇的に減らすことができました。Claude Codeはコンテキストを維持する能力に優れているため、まずは骨組みを作り、そこに肉付けをしていくアプローチが最も効率的です。

## 🙋 よくある質問 (FAQ)

- **Q: ゲームの物理や数学の知識が全くなくても可能ですか？**
  - A: はい、完全に可能です。「芝生の摩擦力を実装して」「壁にぶつかったら跳ね返るようにして」といった自然言語で物理現象を描写するだけで、ベクトルの内積や反発係数などの複雑な数学の公式はClaudeが自動でコードに変換してくれます。

- **Q: モバイルブラウザ環境でタッチ操作にも対応させるにはどうすればいいですか？**
  - A: Pro Versionプロンプトのタスク（Task）部分に、「`マウスイベントだけでなく、touchstart, touchmove, touchend イベントもマッピングして、モバイル環境でも同様に動作するようにしてください`」という一文を追加するだけで、きれいに解決します。

## 🧬 プロンプトの解剖 (Why it works?)

1.  **明確なアーキテクチャの指示:** ファイルを `game.js`, `physics.js` などに分割するように明示することで、コードの複雑さを抑え、AIが各ドメインに集中できるようにしました。
2.  **具体的なメカニズムの描写:** 単に「ゴルフゲーム」とするのではなく、「マウスで引いた時の軌道とパワーを示す視覚的なインジケーター」とユーザーエクスペリエンス（UX）を非常に詳細に描写し、開発の意図を正確に伝えました。

```javascript
// 完全にモジュール化され、オブジェクト指向で設計された物理エンジン (physics.js)
export class PhysicsBody {
  constructor(x, y) {
    this.position = new Vector(x, y);
    this.velocity = new Vector(0, 0);
    this.friction = 0.98; // 芝生の摩擦係数
  }

  update() {
    // 速度に摩擦力を適用
    this.velocity.multiply(this.friction);

    // 位置の更新
    this.position.add(this.velocity);

    // Canvasの境界での衝突処理（反発係数を適用するロジックの呼び出し）
    this.handleWallCollisions();
  }
}
```
