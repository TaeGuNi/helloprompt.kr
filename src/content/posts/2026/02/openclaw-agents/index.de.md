Das Framework für professionelles Systemdesign und kompromisslose Sicherheitskontrollen in Multi-Agenten-Architekturen.

> **Kontext (Context):**
>
> - Hintergrund: Das `[Projektziel]` muss zwingend erreicht werden. Dies erfordert komplexe Entscheidungsbäume und eine nahtlose Multi-Agenten-Kollaboration.

## 💡 Experten-Kommentar (Insight)

Dieser Prompt ist exakt darauf kalibriert, die mächtigsten Features von OpenClaw 2026 systematisch zu orchestrieren: **Multi-Agenten-Kollaboration** und eine **sichere Ausführungsumgebung**. Meine Praxiserfahrung zeigt schonungslos: Wenn Rollenverteilung und Zugriffsrechte zwischen den Agenten nicht von der ersten Sekunde an glasklar definiert sind, verheddert sich das System unweigerlich in Endlosschleifen oder triggert blindlings die falschen APIs.

Setzen Sie für das `[Projektziel]` beispielsweise "Automatisierte Fehlerbehebung und PR-Erstellung" ein. Sie erhalten auf Anhieb den belastbaren Blueprint einer Architektur, in der ein Code-Analyse-Agent, ein Lösungs-Entwickler und ein Test-Runner wie ein Schweizer Uhrwerk ineinandergreifen. Dieser Prompt ist ein massiver Hebel, der Ihnen im Systemdesign und beim Debugging Dutzende von Stunden ersparen wird.

## 🙋 Häufig gestellte Fragen (FAQ)

- **F: Kann ich diesen Prompt auch in älteren Versionen von OpenClaw verwenden?**
  - A: Grundsätzlich ja. Beachten Sie jedoch, dass die mit der 2026er Edition eingeführten nativen Features für das **Advanced Memory Management** in Legacy-Versionen unter Umständen ins Leere laufen. Passen Sie die Architektur im Prompt in diesem Fall manuell an Ihre spezifische Laufzeitumgebung an.

- **F: Was passiert, wenn ein Agent "rogue" geht und versucht, nicht autorisierte Werkzeuge aufzurufen?**
  - A: Fügen Sie den Einschränkungen (Constraints) einfach diesen rigorosen Riegel hinzu: "Jeder Agent darf unter keinen Umständen auf Werkzeuge zugreifen, die nicht explizit zugewiesen wurden, und muss vor jeder kritischen Ausführung zwingend einen Human-in-the-Loop-Approval abwarten."

## 📊 Der Beweis: Before & After

### ❌ Before (Einfache Anweisung)

```text
Erstelle einen Agenten zur Datenanalyse mit OpenClaw.
```

_(Ergebnis: Ein einzelner, völlig überlasteter Agent versucht alle Aufgaben auf einmal zu übernehmen, kollabiert am Kontext-Limit oder liefert einen generischen Prompt voller kritischer Sicherheitslücken.)_

### ✅ After (Mit diesem Prompt)

```text
[Hauptagent: Datenanalyse-Master]
- Rolle: Koordination der entiren Analyse-Pipeline und Ableitung finaler strategischer Erkenntnisse.
- Berechtigungen: Nur-Lese-Zugriff auf die Produktionsdatenbank, Aufgabenzuweisung an Sub-Agenten.

[Sub-Agent 1: Daten-Kollektor]
- Rolle: Echtzeit-Crawling von Rohdaten aus spezifischen externen APIs.
- Sicherheitseinschränkung: Darf ausschließlich im Vorfeld freigegebene APIs (Strict Whitelist) aufrufen (Strenge Werkzeugkontrolle).

[Kommunikationsfluss]
1. Kollektor ruft Daten ab und lädt sie validiert in den Kurzzeitspeicher.
2. Master analysiert die Daten im Kurzzeitspeicher und persistiert ausschließlich die wichtigsten, verifizierten Erkenntnisse im Langzeitspeicher...
```

_(Ergebnis: Eine extrem robuste, hermetisch gesicherte Multi-Agenten-Architektur mit glasklar getrennten Verantwortlichkeiten und strikt limitierten Berechtigungen.)_
