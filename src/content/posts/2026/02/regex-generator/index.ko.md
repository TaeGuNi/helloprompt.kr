## 📝 Regex(정규표현식) 생성기: 암호문 같은 정규식, 말로 짜자

- **🎯 추천 대상:** `^([a-z0-9_.-]+)@([\da-z.-]+)\.([a-z.]{2,6})$` 이 코드만 봐도 머리가 아픈 분, 당장 입력 폼 유효성 검사를 구현해야 하는 프론트엔드 및 백엔드 개발자
- **⏱️ 소요 시간:** 10분 → 10초 단축
- **🤖 추천 모델:** ChatGPT-4o, Claude 3.5 Sonnet (복잡한 패턴 인식 및 코드 생성 최강)

- ⭐ **난이도:** ⭐☆☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"비밀번호는 영문, 숫자, 특수문자 포함 8자 이상... 이걸 정규식으로 어떻게 짜더라?"_

정규표현식(Regex)은 아무리 경험이 풍부한 시니어 개발자라 할지라도 실무에서 마주할 때마다 구글 창을 열게 만드는 까다로운 영역입니다. 난해하고 복잡한 기호들의 나열을 억지로 외우며 시간을 낭비하는 것은 매우 비효율적입니다. 이제는 **자연어(Natural Language)**로 가볍게 설명해 보세요. "이메일 형식 찾아줘"라는 아주 단순한 문장 하나만으로도, AI는 복잡한 기호들을 완벽하게 조합하여 무결점의 정규표현식 코드를 즉각적으로 생성해 냅니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. 추출하고자 하는 데이터나 찾고 싶은 문자열 패턴을 AI에게 자연어로 명확히 지시합니다.
2. AI가 반환한 타겟 언어(JS, Python 등) 맞춤형 정규식 코드와 캡처 그룹(Capture Group)에 대한 상세한 해설을 확인합니다.
3. [Regex101](https://regex101.com)과 같은 검증 사이트에서 실제 예시 데이터를 활용해 엣지 케이스를 테스트한 뒤 실무 코드에 적용합니다.

## 🚀 해결책: "Regex Wizard"

### 🥉 Basic Version (기본형)

이메일, 휴대폰 번호와 같이 실무에서 빈번하게 요구되는 기본적인 패턴을 신속하게 검증할 때 활용하세요.

> **역할 (Role):** 너는 `[시니어 개발자]`야.
> **요청 (Task):** `[한국 휴대폰 번호(010-XXXX-XXXX) 형식을 검증하는 JavaScript 정규표현식]`을 작성해 줘. 단, 하이픈(-)의 존재 유무와 관계없이 검증 가능해야 해.

### 🥇 Pro Version (전문가형)

방대한 서버 로그 파일이나 비정형 텍스트 데이터에서 특정 핵심 데이터만을 정교하게 발라내야 하는 복잡한 상황에 최적화된 프롬프트입니다.

> **역할 (Role):** 너는 10년 차 시니어 백엔드 개발자이자 정규표현식(Regex) 마스터야.
>
> **상황 (Context):**
>
> - 배경: 방대한 시스템 로그 파일에서 특정 사용자의 요청 데이터만 정제하여 데이터베이스에 적재해야 해.
> - 목표: 가공되지 않은 로그 텍스트에서 날짜, 유저 ID, IP 주소 데이터만 한 치의 오차 없이 추출(Capture)하는 정규식 작성.
>
> **요청 (Task):**
>
> 1. 아래 제공된 `[입력 데이터]`를 정밀하게 분석하여, `[추출 목표]`에 해당하는 핵심 정보만을 캡처 그룹으로 묶어내는 Python 정규표현식을 작성해 줘.
> 2. 작성된 정규식의 각 문법적 요소와 캡처 그룹이 수행하는 역할을 주니어 개발자도 쉽게 이해할 수 있도록 단계별로 해설해 줘.
> 3. 잠재적으로 발생할 수 있는 예외 케이스(Edge Case)를 예측하고, 이를 방어하기 위한 기술적 조언을 추가해 줘.
>
> **입력 데이터 (Input):**
> `[2026-02-09 14:00:01] INFO [User:12345] Login Success (IP: 192.168.0.1)`
>
> **추출 목표 (Goal):**
>
> 1. 날짜 및 시간 (`2026-02-09 14:00:01`)
> 2. 유저 ID (`12345`)
> 3. IP 주소 (`192.168.0.1`)
>
> **제약사항 (Constraints):**
>
> - 결과물은 반드시 마크다운 코드블록 형태로 출력해 줘.
> - Python의 `re` 모듈 환경에서 수정 없이 즉시 실행 가능한 완벽한 코드여야 해.
>
> **주의사항 (Warning):**
>
> - 의도하지 않은 공백이나 잉여 문자가 캡처 그룹에 혼입되지 않도록 가장 타이트하게 정규식을 구성해 줘.

## 💡 작성자 코멘트 (Insight)

정규표현식을 다룰 때 마주하는 가장 치명적인 적은 단연코 **'예외 케이스(Edge Case)'**입니다.
예를 들어, 단순히 "숫자만 찾아줘"라고 AI에게 지시할 경우 `123`과 같은 기본 정수는 완벽하게 매칭하지만, `-123`(음수)이나 `123.45`(소수점)와 같은 변형된 데이터는 누락하는 치명적인 오류를 범하게 됩니다. 따라서 프롬프트를 설계할 때부터 **"음수나 소수점이 포함될 가능성도 고려해야 하나요?"**라고 AI가 스스로 엣지 케이스를 점검하도록 유도하거나, 발생 가능한 다양한 포맷의 예시 데이터를 프롬프트에 명시적으로 제공하여 사전 테스트를 거치는 것이 실무 환경에서 예상치 못한 버그를 원천 차단하는 핵심 노하우입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 프로그래밍 언어마다 정규표현식 문법에 차이가 있나요?**
  - A: 네, 미묘하지만 무시할 수 없는 확실한 문법적 차이가 존재합니다. JavaScript, Python, Java, Go 등 각 언어 생태계마다 채택하고 있는 정규식 엔진(PCRE, RE2 등)의 스펙이 다르기 때문입니다. 따라서 AI에게 프롬프트를 작성할 때는 반드시 "JavaScript 환경에서 사용할 정규식을 작성해 줘"와 같이 타겟 언어를 명확히 지정해야만 문법 충돌 없는 완벽히 호환되는 코드를 확보할 수 있습니다.

- **Q: AI가 생성해 준 정규식을 검증 없이 바로 프로덕션 코드에 적용해도 안전한가요?**
  - A: 절대 권장하지 않습니다. AI가 도출한 정규식은 특정 패턴에서는 완벽하게 동작하더라도 보이지 않는 논리적 허점을 내포하고 있을 위험이 있습니다. 코드를 실무에 반영하기 전, 반드시 [Regex101](https://regex101.com)과 같은 전문 정규식 테스트 샌드박스에 코드를 이식하여 다양한 예외 테스트 문자열(Edge cases)을 직접 교차 검증(Cross-check)하는 안전망 구축 과정이 필수적입니다.

## 🧬 프롬프트 해부 (Why it works?)

1.  **예시 데이터의 명시적 제공 (Few-shot Prompting):** 정규표현식은 적용되는 텍스트의 문맥(Context)에 따라 그 패턴이 극적으로 변합니다. 실무에서 다루는 실제 로그 형태나 비정형 텍스트 샘플을 프롬프트 내에 직접 주입하면, AI의 패턴 추론 능력과 결과물의 정확도를 100%에 가깝게 극대화할 수 있습니다.
2.  **캡처 그룹(Capture Group)의 전략적 활용:** 단순히 "일치하는 것을 찾아줘(Match)"라는 1차원적 지시를 넘어, "특정 데이터 영역만 정확히 추출해 줘(Extract)"라고 목적을 구체화했습니다. 이를 통해 AI가 정규식의 괄호 `()` 문법을 적극 활용하여 프로그래머가 후속 로직에서 가공하기 쉬운 형태로 데이터를 발라내도록 영리하게 유도한 것입니다.

## 📊 증명: Before & After

### ❌ Before (직접 작성한 정규식)

```text
\d+
```

_(단순히 숫자 정보만 추출하려는 안일한 접근으로 인해, 로그 데이터 내에 존재하는 날짜와 시간 정보까지 무분별하게 매칭되어 버리면서 정작 필요한 데이터는 파싱할 수 없게 된 참담한 상태 😱)_

### ✅ After (AI가 생성한 정규식 및 설명)

**패턴:**

```python
^\[(.*?)\] INFO \[User:(\d+)\] .* \(IP: ([\d\.]+)\)$
```

**설명:**

- `(.*?)`: 대괄호(`[]`) 내부에 위치한 타임스탬프 데이터를 Non-greedy 방식(가장 짧게 일치)으로 매칭하여 안전하게 캡처합니다. (날짜 및 시간 데이터)
- `(\d+)`: `User:` 접두어 직후에 등장하는 고유한 숫자 ID만을 정확히 타겟팅하여 캡처합니다. (유저 ID 데이터)
- `([\d\.]+)`: 숫자와 점(`.`)으로만 구성된 IP 주소 고유의 형태를 인식하여 해당 구간만을 독립적으로 캡처합니다. (IP 주소 데이터)

_(개발자가 의도한 핵심 데이터 요소들만 완벽하게 캡처 그룹(Capture Group)으로 분리해 낸 압도적인 성공 사례 🎯)_
