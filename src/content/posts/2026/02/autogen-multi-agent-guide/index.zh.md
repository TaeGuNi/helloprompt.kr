# 🤖 AutoGen 入门：让 AI 智能体互相协作对话

- **🎯 推荐对象：** 感到单打独斗力不从心的独立开发者、希望构建自动化工作流的极客 (Tech Geek)
- **⏱️ 所需时间：** 20分钟（环境配置与 Demo 运行）
- **🤖 推荐模型：** GPT-4o, Claude 3.5 Sonnet（智能体之间的逻辑对话质量至关重要）

- ⭐ **难度：** ⭐⭐⭐⭐☆
- ⚡️ **有效性：** ⭐⭐⭐⭐⭐
- 🚀 **实用度：** ⭐⭐⭐⭐☆

> _"把所有任务都交给一个 AI，是不是经常触碰能力天花板？让负责策划的 AI、负责编码的 AI 和负责代码审查的 AI，为你组建一支完美的虚拟开发团队吧。"_

由微软 (Microsoft) 开发的 **AutoGen** 是一款强大的多智能体（Multi-Agent）框架，它能将这种想象化为现实。只需创建多个 AI 角色（Persona）并将它们置于同一个工作空间中，它们就会自主展开对话、互相提供反馈，并最终共同完成项目。这标志着真正意义上的**“自主协作 (Autonomous Collaboration)”**时代已经到来。

---

## ⚡️ 3行核心总结 (TL;DR)

1. **角色分工的魔法：** 将复杂的单一提示词拆分为策划、开发、审查等不同角色的智能体，将输出质量最大化。
2. **代码自主执行 (Code Execution)：** AI 编写的代码会在沙盒/容器环境中直接运行，AI 会根据报错信息进行自主调试 (Debugging)。
3. **无限循环控制：** 通过设置人类管理者 (Human-in-the-loop) 的介入时机，有效防止 Token 浪费并精准把控项目方向。

## 🚀 解决方案："多智能体团队构建提示词"

### 🥉 Basic Version (基础版：双人组合 - 开发者 & 执行者)

适用于生成最基础的 1:1 智能体交互代码的提示词，快速体验 AutoGen 的魅力。

> **角色：** 你是一位 `[资深 Python 开发者]`。
>
> **任务：** 请使用 AutoGen 框架，编写一个让两个智能体协同工作的 Python 脚本。
>
> 1. **AssistantAgent (Coder)：** 负责编写 Python 代码。
> 2. **UserProxyAgent (Executor)：** 负责在本地环境中执行 Coder 编写的代码，并将运行结果反馈给 Coder。
>
> **目标：** 让这两个智能体合作完成“找出 1 到 100 之间的所有素数 (Prime Number)，并将其保存为 txt 文件”的任务。

\

### 🥇 Pro Version (专业版：构建虚拟 IT 公司 - PM, Dev, QA)

适用于构建高度还原真实业务场景的 3 人组群聊 (Group Chat) 协作系统，输出质量更高、细节更丰富。

> **角色 (Role)：** 你是一位 `[来自微软的 AutoGen 框架架构师]`。
>
> **背景 (Context)：**
>
> - 背景：我平时工作繁忙，没有足够的时间独自开发独立项目，因此希望利用 AI 智能体组建一支虚拟开发团队。
> - 目标：自动化完成一个基于 Python 的简易“贪吃蛇网页游戏 (Snake Game)”，涵盖从需求策划到测试的完整生命周期。
>
> **任务 (Task)：**
>
> 请利用 AutoGen 的 `GroupChat` 和 `GroupChatManager`，编写一段完整的 Python 代码，实现以下 3 个智能体的协同工作：
>
> 1. **PM (Product Manager)：** 负责制定游戏的核心规则和 UI/UX 策划，并下发开发指令。
> 2. **Coder (Developer)：** 根据 PM 的策划文档，编写 HTML/CSS/JS 代码。
> 3. **Reviewer (QA/Critic)：** 负责审查 Coder 提交的代码，找出潜在 Bug 或逻辑漏洞，并给出具体的修改建议。
>
> **约束条件 (Constraints)：**
>
> - 工作流顺序：必须严格按照“PM 提出策划方案 ➔ Coder 编写代码 ➔ Reviewer 提供反馈 ➔ Coder 修改代码”的循环进行。
> - 交互限制：将最大对话轮数 (`max_round`) 限制为 10 次。
> - 代码要求：提供的 Python 脚本必须完整且可直接运行（无报错）。请为每个智能体设定极为具体且符合其身份的 `system_message`。
>
> **注意事项 (Warning)：**
>
> - 不要捏造不存在的 AutoGen API，如果遇到不确定的参数，请直接说明。(防止幻觉)

## 💡 作者见解 (Insight)

AutoGen 真正的破坏力在于它将 LLM 的文本生成能力与**“代码执行 (Code Execution)”**功能完美结合。在过去，聊天机器人写出代码后，我们需要手动复制粘贴到 IDE 中；如果运行报错，还要再把错误日志复制回去问 AI。这个过程既繁琐又耗时。

但是，只要将 `UserProxyAgent` 与 Docker 容器连接，AI 就能自主运行代码，并通过分析报错日志来重写逻辑。也就是说，你只需要坐在屏幕前，像主管一样看着这群 AI 激烈地讨论、不知疲倦地 Debug，最后直接验收完美的最终成果即可。这种“放手不管”的开发体验，是传统编码方式无法比拟的。

## 🙋 常见问题 (FAQ)

- **Q: 除了 OpenAI API，可以接入开源的本地模型（比如 Llama 3）吗？**
  - A: 当然可以，且完美兼容。你可以使用 LM Studio 或 Ollama 启动本地服务，然后在 AutoGen 的 `llm_config` 中将 `base_url` 指向本地 API 地址。这样就能彻底告别 Token 焦虑，零成本管理你的“智能体军团”。

- **Q: 如果智能体们在对话中一直达不成共识，陷入死循环（无限 Loop）怎么办？**
  - A: 这是多智能体系统中最常见的痛点。你可以通过严格限制 `max_consecutive_auto_reply`（最大连续回复次数）来强行熔断；或者将 `human_input_mode` 设置为 `"TERMINATE"`，并约定当对话中出现特定关键词（如“任务完成”、“代码已验收”）时，立即终止自主对话，将控制权交还给人类管理者。

- **Q: 它和 CrewAI 框架有什么区别？**
  - A: AutoGen 的核心优势在于智能体之间**“自由度极高的对话 (Conversation)”**以及强大的代码执行能力，非常适合复杂的逻辑推演和编程开发。而 CrewAI 则更侧重于沿着预设的流水线和**“流程 (Process)”**顺序执行任务。简而言之：需要创造力或写代码，选 AutoGen；需要做流程化的市场调研或文案撰写，选 CrewAI。

## 🧬 提示词解剖 (Why it works?)

1. **分散认知负荷 (Cognitive Load Distribution)：** 如果你把产品策划、代码编写和逻辑审查的约束条件全部塞进一个提示词里，AI 极易出现“遗忘中间指令 (Lost in the middle)”的现象。通过拆分角色并赋予独立的 `system_message`，每个 AI 只需要专注自己的“本职工作”，输出质量自然呈指数级飙升。
2. **引入批判性思维机制 (Critic-in-the-loop)：** 将开发者 (Coder) 和审查者 (Reviewer) 剥离开来，本质上是模拟了真实互联网公司的代码审查 (Code Review) 文化。LLM 通常很难客观地挑出自己生成的代码中的毛病；引入独立的 QA 智能体，能够强制提升代码的健壮性和逻辑闭环。
