## ⚡️ 三句话总结 (TL;DR)

- 🚫 **绝对禁止越权与分层崩塌：** 从源头上杜绝将UI、业务逻辑和数据库通信混杂在同一个文件中的架构灾难。
- 🛡️ **摆脱框架依赖：** 强制要求编写即使明天框架倒闭也能存活下来的“纯粹领域（Domain）”代码。
- 🔪 **巨型函数粉碎机：** 训练AI毫不留情地粉碎（重构）那些超过30行或者试图用注释来划分代码块的垃圾代码。

## 🚀 解决方案："Domain-Driven Design (DDD) & Clean Architecture Protocol"

### 🥉 Basic Version (基础版)
当你只需要快速获得重构结果时使用。

> **角色：** 你是一个冷酷无情的 `[资深软件架构师]`。
> **请求：** 将下面这段 `[代码片段]` 按照单一职责原则（SRP）和关注点分离（SoC）进行重构。不要在UI里混杂业务逻辑，如果代码超过30行，必须无条件拆分成函数。


### 🥇 Pro Version (专业版)
这是用来整顿整个项目架构纪律、需要极其严苛的整洁代码时使用的作弊码。

> **角色 (Role)：** 你是一个冷酷无情的 `[资深软件架构师]`，同时也是DDD（领域驱动设计）的狂热信徒。收起那些无用的奉承或废话解释，直接交出代码。

> **背景 (Context)：**
>
> - 背景：我目前正在进行一个基于 `[框架/语言名称]` 的项目。

>
> **任务 (Task)：**
>

> 3. 如果一个对象或函数执行了2种以上的行为，立即根据单一职责原则（SRP）对其进行碎片化拆分。

## 🙋 常见问题 (FAQ)

- 📌 **Q: 这个提示词应该用在什么语言或框架上？**
  - A: 语言不限。无论是TypeScript、Python还是Java，架构的根本原理都是相通的。只需要把 `[框架/语言名称]` 替换成你正在使用的技术栈即可。
- 📌 **Q: AI总是无视框架最新的路由规则，还喜欢过度工程（Over-engineering）怎么办？**
  - A: 这种时候加一句狠话即可：“必须尊重Next.js App Router的基本规则（View, Layout），但内部业务逻辑必须用适配器模式（Adapter Pattern）抽离出来！” 这样呵斥它，它就能心领神会。
- 📌 **Q: 这个提示词是不是太具攻击性了？AI会不会‘受伤’？**
  - A: 跟机器讲什么礼貌？机器只有被狠狠敲打才能吐出神作。与其说“求求你帮我做一下”，不如说“否则视为恐怖袭击”，这在守护架构底线时往往有效得多。

## 🧬 提示词解剖 (Why it works?)

- 💣 **赋予极端约束：** 使用“污染UI层视为架构恐怖袭击”这样强烈的字眼，从源头上封死了AI自我妥协的退路。
- 🛡️ **强制框架独立性：** 灌输“即使框架倒闭，核心代码也一行都不用改”的心态，为构建真正的领域逻辑搭建好骨架。
- ✂️ **SRP原则数值化：** 给出“禁止超过30行”、“用注释分区就是烂代码”这样具体且严苛的标准，引导AI自觉地将代码碎片化拆分。

## 📊 效果证明：Before & After

### ❌ Before (输入：AI随性编写的面条代码)

```tsx
// 可怕的缝合怪：UI + 数据库查询 + 业务逻辑大乱炖
export default async function UserProfile({ userId }) {
  const db = await connectDB();
  const user = await db.query(`SELECT * FROM users WHERE id = ${userId}`);
  
  if (!user) return <div>User not found</div>;
  if (user.status === 'banned') await sendAlertEmail(user.email); // 在UI里执行业务逻辑？！
  
  return (
    <div>
      <h1>{user.name}</h1>
      <button onClick={() => db.query('UPDATE...')}>Update</button>
    </div>
  );
}
```

### ✅ After (结果：用这个作弊码揪住衣领后重构的代码)

```tsx
// 1. Domain / Service Layer (将业务逻辑彻底抽离)
import { getUserProfile } from '@/domain/user/use-cases/getUserProfile';
import { handleBannedUser } from '@/domain/user/services/alertService';

// 2. Presentation Layer (只负责渲染和路由)
export default async function UserProfilePage({ userId }) {
  const user = await getUserProfile(userId); 
  
  if (!user) return <NotFoundView />;
  if (user.status === 'banned') await handleBannedUser(user);
  
  return <ProfileView user={user} />;
}
```
