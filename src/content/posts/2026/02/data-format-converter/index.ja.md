## 📝 Excel、JSON、CSV… データフォーマット変換の単純作業から脱出

- **🎯 推奨対象:** エンジニア、データアナリスト、事務職全般
- **⏱️ 所要時間:** 30分 → 1分に短縮
- **🤖 推奨モデル:** すべての対話型AI (ChatGPT, Claude, Geminiなど)

- ⭐ **難易度:** ⭐⭐☆☆☆
- ⚡️ **効果性:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐☆

> _「このExcelのデータ、JSONに変換してAPIで連携できるようにしておいて」_

開発やデータ分析の現場において、フォーマット変換は避けて通れない日常業務です。しかし、機密情報を含むデータを無料のオンラインツールにアップロードするのはセキュリティ上危険ですし、変換のためだけに毎回専用スクリプトを書くのは時間の無駄と言えるでしょう。そんな悩みも、対象となるデータをコピー＆ペーストするだけで一挙に解決します。AIがあなたの指示通り、一瞬にして目的のフォーマットへと正確に変換してくれるのです。

---

## ⚡️ 3行要約 (TL;DR)

1. **自由自在なフォーマット変換:** JSON ↔ CSV ↔ XML など、あらゆるデータ形式間の双方向変換に即座に対応。
2. **高度なデータ加工:** 不要なフィールドの除外や、データ構造の変更（マッピング）も同時に処理可能。
3. **実用的な出力結果:** そのままコピペで使えるSQLのINSERT文生成など、実務に直結するフォーマット変換を実現。

## 💡 作成者コメント (Insight)

このプロンプトの真の価値は、単なるフォーマットの変換にとどまらず、同時に「データクレンジング」まで完結できる点にあります。例えば、CSVデータをJSON配列に変換する過程で、「日付の表記を『YYYY-MM-DD』に統一して」「空欄（Null）の項目はすべて除外して」といった指示を1行追加するだけで、面倒なETLパイプラインの一部をまるごとAIに代替させることが可能です。機密情報を扱う場合は、あらかじめ顧客名やメールアドレスをダミーデータに置換する（マスキング処理）などの工夫をしてからAIに渡すことで、セキュリティ面でも安心して実務に投入できます。

## 🙋 よくある質問 (FAQ)

- **Q: 変換できるデータ量に制限はありますか？**
  - A: 利用するAIモデルのコンテキストウィンドウ（一度に処理可能なトークン数）に依存します。最新の大容量モデルであれば数千行のCSVも処理可能ですが、変換エラーやデータの欠落を防ぐため、数百行ずつ分割して処理することをおすすめします。

- **Q: 階層が深い複雑なJSONやXMLでも正しく変換されますか？**
  - A: はい、問題なく変換可能です。ただし、CSVのようなフラットな形式に変換する場合、AIが独自の判断で階層を結合したり省略したりするケースがあります。「ネストされたデータはアンダースコア（_）で繋いで展開して」のように、具体的な展開ルールを明記することで、より精度の高い結果が得られます。

- **Q: 社外秘の機密データを変換したいのですが、セキュリティ面は安全ですか？**
  - A: API経由での利用であれば通常AIの学習データとして使用されませんが、無料版のウェブブラウザインターフェースを使用する場合は学習に利用されるリスクがあります。機密データを扱う際は、個人情報が含まれていないか事前にマスキングを行うか、学習に利用されないエンタープライズ版のAIモデルを使用してください。

## 🧬 プロンプト解剖 (Why it works?)

1. **専門家のペルソナ設定:** 「データエンジニア・ETL専門家」という役割を付与することで、データ構造の整合性やエッジケース（例外処理）に対するAIの注意力を極限まで高めています。
2. **変換ルールの明確化:** 単に「変換して」と丸投げするのではなく、「ソースフォーマット」「ターゲットフォーマット」「マッピングルール」を構造化して指示することで、意図しないデータの欠損や構造の崩壊（ハルシネーション）を完全に防ぎます。
3. **余計な出力の抑制:** 「結果のみをコードブロックで出力して」と明確に制限することで、AI特有の無駄な挨拶や解説を省き、出力結果を即座にコピペして実務に投入できるよう最適化しています。

## 📊 証明: Before & After

### ❌ Before (入力 CSV)

```csv
名前,年齢,職業
ホン・ギルドン,30,開発者
キム・チョルス,25,デザイナー
```

### ✅ After (結果 - JSON + フィールド名変更)

**リクエスト:** CSVをJSONに変換し、フィールド名を英語に変更してください。

```json
[
  {
    "name": "ホン・ギルドン",
    "age": 30,
    "job": "開発者"
  },
  {
    "name": "キム・チョルス",
    "age": 25,
    "job": "デザイナー"
  }
]
```

### ✅ After (結果 - SQL Insert文生成)

**リクエスト:** データをMySQLの `users` テーブルに追加するためのINSERT文に変換してください。

```sql
INSERT INTO users (name, age, job) VALUES
('ホン・ギルドン', 30, '開発者'),
('キム・チョルス', 25, 'デザイナー');
```
