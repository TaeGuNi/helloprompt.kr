## 📝 Refatorar Código Sujo para Código Limpo em 1 Segundo

- **🎯 Recomendado para:** Desenvolvedores, Engenheiros de Software, Tech Leads
- **⏱️ Tempo necessário:** 5 minutos → Reduzido para 1 segundo
- **🤖 Modelo recomendado:** Qualquer modelo de IA conversacional de ponta (ChatGPT, Claude, Gemini, etc.)

- ⭐ **Dificuldade:** ⭐⭐☆☆☆
- ⚡️ **Eficácia:** ⭐⭐⭐⭐⭐
- 🚀 **Utilidade:** ⭐⭐⭐⭐☆

> _"Não consigo nem entender o código que escrevi ontem... Como vou desembaraçar esse espaguete sem quebrar o sistema inteiro?"_

Todos nós já passamos por isso: criar nomes de variáveis incompreensíveis na pressa de bater uma meta ou escrever funções monolíticas que ultrapassam as 100 linhas.
Mas e se você tivesse um assistente sênior ao seu lado, pronto para refatorar aquele código que dá até vergonha de mostrar no Code Review? Alguém capaz de transformá-lo em uma obra-prima digna de elogios, seguindo rigorosamente as práticas de **Clean Code**?
Com apenas um prompt, você garante legibilidade impecável, alta performance e facilidade de manutenção em um piscar de olhos.

---

## ⚡️ Resumo em 3 linhas (TL;DR)

1. **Nomenclatura Intuitiva:** Renomeia variáveis e funções para revelar suas intenções de forma cristalina.
2. **Separação de Responsabilidades:** Elimina código duplicado e fragmenta funções excessivamente complexas.
3. **Modernização Absoluta:** Aplica as sintaxes mais recentes e otimiza a performance geral da aplicação.

## 🚀 A Solução: "Refatorador de Clean Code"

### 🥉 Versão Básica (Basic Version)

Use esta versão quando precisar apenas de um resultado rápido e direto ao ponto.

> **Papel:** Você é um Engenheiro de Software Sênior com 20 anos de experiência e um verdadeiro mestre em 'Clean Code'.
> **Solicitação:** Transforme o código espaguete a seguir em um código impecavelmente limpo, de fácil leitura e manutenção.

### 🥇 Versão Pro (Pro Version)

Use esta versão quando precisar de um refinamento profundo e um controle cirúrgico sobre a refatoração.

> **Papel (Role):** Você é um Engenheiro de Software Sênior com 20 anos de experiência, especialista em arquitetura de software e nos princípios de 'Clean Code' (SOLID, DRY, KISS).
>
> **Situação (Context):**
>
> - Cenário: O código que escrevi funciona perfeitamente, mas a legibilidade é péssima e a manutenção virou um pesadelo. Os nomes das variáveis são confusos e há muita lógica de negócios duplicada.
> - Objetivo: Refatorar o código para que qualquer desenvolvedor da equipe consiga entendê-lo instantaneamente, mantendo exatamente o mesmo comportamento da versão original.
>
> **Tarefa (Task):**
>
> Refatore o código fornecido abaixo seguindo rigorosamente os seguintes critérios:
>
> 1. **Nomenclatura:** Renomeie variáveis e funções para que suas intenções fiquem evidentes. Evite qualquer tipo de abreviação obscura.
> 2. **Separação de Responsabilidades (SRP):** Desmembre funções gigantescas em blocos menores, garantindo que cada função execute apenas uma única tarefa.
> 3. **Remoção de Duplicações:** Extraia blocos de lógicas repetidas para funções utilitárias isoladas.
> 4. **Comentários Estratégicos:** Adicione comentários _exclusivamente_ onde for essencial explicar o 'Porquê' de uma decisão arquitetural. O 'O quê' deve ser autoexplicativo pelo próprio código.
> 5. **Sintaxe Moderna:** Aplique as melhores práticas e as sintaxes mais modernas da linguagem utilizada.
>
> **[Insira o seu código sujo aqui]**
>
> **Restrições (Constraints):**
>
> - NUNCA altere a lógica de negócios ou o comportamento final do código existente.
> - Retorne a resposta utilizando blocos de código em Markdown.
> - Ao final, apresente uma lista resumindo as principais mudanças realizadas (Antes vs. Depois).
>
> **Aviso (Warning):**
>
> - Não adicione bibliotecas externas, a menos que elas já estejam sendo utilizadas no código original.

## 💡 Comentário do Autor (Insight)

Esta técnica de refatoração com IA é um verdadeiro "salva-vidas" antes de abrir aquele temido Pull Request (PR). Muitos desenvolvedores têm receio de usar IA para refatorar por medo de que a lógica de negócios seja quebrada de forma silenciosa. É exatamente por isso que a instrução **"NUNCA altere a lógica de negócios"** e a exigência de um resumo das alterações são tão cruciais na Versão Pro.

**Dica de ouro:** Ao colar o seu código, inclua também os testes unitários da função (se existirem). A IA usará os testes como garantia de que a refatoração não quebrou absolutamente nada. E, caso não existam testes, você pode até mesmo pedir para a IA gerá-los na mesma interação!

## 🙋 Perguntas Frequentes (FAQ)

- **Q: A IA pode quebrar o meu código em produção?**
  - A: Sempre existe um pequeno risco. A IA é excelente para sugerir melhorias estruturais, mas você **precisa** rodar seus testes unitários após a refatoração. Nunca copie, cole e envie direto para produção sem antes testar e validar o funcionamento no seu ambiente local.

- **Q: Posso refatorar arquivos inteiros de uma só vez?**
  - A: Depende do tamanho do arquivo e do limite de tokens (Context Window) da IA escolhida. O cenário ideal é refatorar função por função, ou em pequenos módulos (de até 300 a 500 linhas), para garantir que a IA preste atenção a todos os detalhes e não "esqueça" partes vitais do seu código.

- **Q: Isso funciona com qualquer linguagem de programação?**
  - A: Com certeza! Este prompt é totalmente agnóstico em relação à linguagem. Seja JavaScript, Python, Go, Java ou Rust, a IA identificará a linguagem fornecida e aplicará as convenções específicas e sintaxes modernas adequadas a ela.

## 📊 Prova: Antes e Depois

### ❌ Antes (Entrada)

```javascript
function calc(d) {
  let t = 0;
  for (let i = 0; i < d.length; i++) {
    if (d[i].price > 100) {
      t += d[i].price * 0.9;
    } else {
      t += d[i].price;
    }
  }
  return t;
}
```

### ✅ Depois (Resultado)

```javascript
/**
 * Calcula o valor total do carrinho de compras.
 * Aplica um desconto de 10% para itens com valor acima do limite.
 */
function calculateTotalPrice(items) {
  const DISCOUNT_THRESHOLD = 100;
  const DISCOUNT_RATE = 0.9;

  return items.reduce((total, item) => {
    const isEligibleForDiscount = item.price > DISCOUNT_THRESHOLD;
    const finalPrice = isEligibleForDiscount
      ? item.price * DISCOUNT_RATE
      : item.price;

    return total + finalPrice;
  }, 0);
}
```

_Mudanças realizadas:_

- _Nome da função: `calc` → `calculateTotalPrice`_
- _Nome da variável: `d` → `items`_
- _Extração de "Magic Numbers" para constantes claras (`DISCOUNT_THRESHOLD`, `DISCOUNT_RATE`)._
- _Substituição do loop `for` imperativo pelo método declarativo `reduce`._
- _Extração da lógica condicional para a variável `isEligibleForDiscount` visando maior legibilidade._
