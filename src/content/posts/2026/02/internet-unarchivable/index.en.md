## 🚀 This is How Real Experts Write

### 🥉 Basic Version

Use this when you want to quickly and safely preserve only the core body of a web page as text. Select all (`Ctrl+A`) and copy (`Ctrl+C`) the entire web page, then input it with the prompt below.

> **Role:** You are a professional 'Digital Archivist'.
>
> **Request:** From the complex webpage text entered below, exclude all noise like menus or advertisements, and organize only the 'body content' containing information into a clean Markdown format.

### 🥇 Pro Version

Use this when you want to build a flawless archive document that can be immediately stored in personal knowledge management tools like Obsidian or Notion without damaging the original context of the website at all.

> **Role:** You are a top-tier 'Digital Archivist' dedicated to preserving humanity's digital heritage. Your primary mission is to extract the essence of information from complex and fragmented web content and process it into a form suitable for permanent preservation.
>
> **Context:**
>
> - Background: Web archiving is becoming increasingly difficult due to dynamic web pages (SPA) and walled garden platforms, requiring direct conversion of original text or source code into a permanent preservation format.
> - Goal: Completely remove unnecessary noise (ads, GNB, footer, comments, etc.) from the input text/HTML dump data and convert it into a perfectly structured Markdown document.
>
> **Task:**
>
> 1. Deeply analyze the `[Webpage Data]` below to accurately extract only the main article.
> 2. Create archiving metadata at the very top of the document in YAML Frontmatter format (Required items: title, estimated date of publication, author, 3 core keywords).
> 3. Rearrange the extracted body into a logical heading structure (`##`, `###`) and format it with high readability using Markdown syntax.
> 4. Make important quotes, key data, and statistical figures visually stand out using Markdown blockquote (`>`) or bold (`**`) syntax.
>
> **Input Data:**
>
> `[Paste the entire webpage text (Ctrl+A, C) or HTML source code here]`
>
> **Constraints:**
>
> - Do not arbitrarily summarize or distort the original sentences or subtle nuances of the main text. (Originality must be maintained at 100% as the ultimate goal is archival preservation.)
> - The output format must strictly follow pure Markdown syntax without any code block wrappers.
>
> **Warning:**
>
> - If the publication date or author cannot be clearly found in the input data, do not make them up and instead mark them as `Unknown`. (To prevent hallucinations.)

## 🙋 Frequently Asked Questions (FAQ)

- **Q: Should I copy and paste the HTML source code, or the text visible on the screen?**
  - A: Surprisingly, both methods work great. However, for complex dynamically rendered pages—a recent web trend—selecting all text on the browser screen (`Ctrl+A`) and copying it is much more advantageous and stable for minimizing invisible script noise and intuitively delivering only the core content to the AI.

- **Q: The pasted content is too long, and the AI model outputs a context limit error. How do I fix this?**
  - A: Blindly copying an entire webpage can cause the number of tokens the AI needs to process to expand rapidly. Therefore, I strongly recommend using the latest models (e.g., Claude 3.5 Sonnet, Gemini 2.5 Pro) that boast overwhelming input limits and specialize in handling long contexts.

## 🧬 Prompt Anatomy (Why it works?)

1. **Forced Originality (Constraints):** When Large Language Models (LLM) face a vast amount of text, they exhibit a fatal bias toward 'summarizing' that content as a reflex. By placing a strong constraint in the prompt that "originality must be maintained at 100% as the purpose is archival preservation," I have precisely designed it to thoroughly suppress the AI's uncontrollable summarizing instinct and focus solely on the original purpose of preserving the source data.
2. **Structured Metadata Extraction:** I forced the generation of YAML Frontmatter at the very top of the result. Through this, I have secured perfect technical compatibility that goes beyond just making a text easy for humans to read, allowing it to be immediately integrated and searched in personal databases or Markdown-based knowledge management applications (PKM).
