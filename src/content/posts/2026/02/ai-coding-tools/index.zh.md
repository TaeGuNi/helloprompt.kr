## 🚀 解决方案："架构师级代码重构提示词"

请立刻停止输入“帮我优化这个函数”这种低效指令。在AI编程工具（如 Cursor 的 Composer 或 Copilot Chat）中喂入以下专业提示词，让AI瞬间化身为您的高薪架构搭档。

### 🥉 Basic Version (基础版)

当您只想快速重构单一函数、提升代码可读性、统一规范并补充注释时，这套轻量级指令最为顺手。

> **角色：** 你是一名极其严谨的资深`[后端 / 前端]`开发者。
> 
> **任务：** 请重构以下`[函数名]`的代码。你必须重点提升其可读性，对变量名进行极其准确的语义化重命名，并为核心业务逻辑补充简明扼要的注释说明。

### 🥇 Pro Version (专业版)

当您面对历史包袱沉重、错综复杂的“意大利面条式”祖传代码时，此提示词将展现出核弹级的重构威力。请务必将整个文件或相关的文件树作为上下文（Context）提供给AI，随后执行以下指令。

> **角色 (Role)：** 你是一名拥有10年超大型分布式系统设计经验的顶级软件架构师。
> 
> **背景 (Context)：**
> 
> - 现状：目前提供的`[文件名或模块名]`代码耦合度极高，后期维护成本巨大，已经处于典型的“反模式 (Anti-pattern)”状态。
> - 目标：我需要你严格遵循`[SOLID原则，尤其是单一职责原则(SRP)与依赖倒置原则(DIP)]`对其进行彻底的模块化外科手术，使其具备极高可测试性与优异的易扩展性。
> 
> **任务 (Task)：**
> 
> 1. 深度诊断：请一针见血地指出当前代码中存在的3个最致命的设计缺陷或反模式，并冷酷地剖析其未来可能引发的灾难性危害。
> 2. 结构化重构：请分步输出解决上述痛点后的重构代码。在输出代码前，必须以架构师的口吻，深度解释本次重构所做的设计权衡 (Trade-off)。
> 3. 规范文档：请为拆解出的核心类与接口，撰写严格符合`[JSDoc / TypeDoc]`标准的工业级 API 文档。
> 
> **约束条件 (Constraints)：**
> 
> - 性能底线：重构后的代码在时间复杂度 (Big O) 和内存消耗上，绝对不允许劣于当前版本。
> - 依赖限制：必须绝对保持`[package.json / pom.xml 等依赖配置文件]`中现有的第三方库及其版本，严禁私自引入任何全新的依赖包。
> - 输出格式：所有的解释说明部分，必须使用 Markdown 代码块与 Checklist 格式进行专业排版。
> 
> **注意 (Warning)：**
> 
> - 杜绝幻觉：你输出的代码必须保证复制后可直接编译运行。在生成最终代码前，请在沙盒思维中进行极其严格的语法与类型推导校验。如果遇到缺失的上下文或模糊的变量，绝不允许凭空捏造，必须向我提问索要完整信息。

## 💡 作者见解 (Insight)

在极为高压的研发排期中，我强烈建议您将 **Cursor** 的“Composer”功能 (Cmd+I / Ctrl+I) 彻底焊死在您的日常工作流里。它最具颠覆性的一点，在于将AI的认知视野从局促的“单文件维度”暴力拉升到了宏观的“全局代码库维度”。当您抛出上述的 Pro 版提示词时，它能精准梳理错综复杂的跨文件依赖，并一键无缝应用多文件联动变更。这种丝滑入骨的体验，简直就像您用白菜价雇佣了一位随叫随到、永不疲倦的高级工程师，在旁边陪您通宵进行结对编程 (Pair Programming)。

在这里，我要特别敲黑板强调提示词中的**“先指出反模式并解释危害”**这一步。许多新手觉得这纯属多此一举，但这恰恰是强行激活大语言模型（LLM）底层**“思维链 (Chain-of-Thought)”**的顶级高阶技巧。它以强硬姿态逼迫AI在盲目生成代码之前，先建立起系统性的全局思考框架，这能让最终产出的代码质量产生断层式的质变。

当然，技术选型永远是一门关于权衡的艺术。如果您正身处金融、医疗或政务等对数据合规与代码泄露有着“零容忍”红线的强监管行业，那么选择提供企业级 SLA 保证、白纸黑字承诺“绝不使用用户代码训练模型”的 **GitHub Copilot Enterprise**，才是您作为技术负责人最为稳妥、成熟的商业底牌。

## 🙋 常见问题解答 (FAQ)

- **Q: 我是一名在校大学生或求职者，囊中羞涩，到底该选哪个？**
  - A: 闭眼选 **Codeium**。它对个人开发者展现出了令人难以置信的慷慨，不仅提供几乎无限制的免费代码补全与 Chat 功能，其极速的响应与强悍的上下文感知能力，应对日常学习和个人开源项目绝对绰绰有余。

- **Q: 面对如此复杂的重构，底层的 AI 模型到底该怎么切换？**
  - A: 在应对深度的代码逻辑推理与庞大规模的重构场景时，**Claude 3.5 Sonnet** 目前展现出了统治级别的领先优势。如果您是 Cursor 用户，请务必在设置中将默认模型强制锁死为 Claude 3.5 Sonnet，以解锁最极致的架构级代码生成体验。

- **Q: 直接将 AI 生成的代码丢进生产环境，会有版权暴雷的风险吗？**
  - A: 如果您使用的是正规企业版工具，合同中通常会附带知识产权赔偿条款作为兜底“免死金牌”。但如果您白嫖的是免费版或个人版，直接大段照搬生成的代码，确实有极大概率与某些开源协议（如 GPL）发生致命冲突。记住，AI 永远只是副驾驶，**极其严苛的 Code Review (代码审查)** 依然是每位开发者绝对不可推卸的底线责任。

## 🧬 提示词解剖 (Why it works?)

1. **高阶角色降维打击 (10年经验架构师)：** 明确的顶级身份设定，能瞬间将AI从底层的“CRUD码农”思维中拔出来，迫使它站在系统架构的俯视视角审视代码，从而给出极具前瞻性与高可维护性的降维解法。
2. **铁腕划定边界 (约束条件与依赖限制)：** 通过极其严苛的性能红线与第三方库引入禁令，完美掐死AI喜欢“过度设计”或乱加不熟悉技术栈的臭毛病，确保重构方案在工程落地上的绝对安全性。
3. **强行锁死思维链 (前置诊断任务)：** 逼迫AI先当“诊断医生”再当“主刀大夫”。这种“先切准脉再下刀”的流程设计，极大地粉碎了模型生成代码幻觉的概率，让逻辑的严密性无懈可击。

function processUserData(user) {
  if (user.age < 18) {
    console.log("Too young");
    return false;
  }
  let discount = 0;
  if (user.type === "premium") {
    discount = 0.2;
  } else if (user.type === "basic") {

  saveToDb(user, finalPrice);
  return true;

### ✅ After (AI 重构结果)

一键注入 Pro 版提示词后，AI 完美剥离了年龄校验与价格计算逻辑，彻底剿灭了所有硬编码常量，并乖乖奉上了极其规范的 API 文档。

```javascript
/**
 * 用户等级折扣配置表
 * @constant {Object}
 */
const DISCOUNT_RATES = {
  PREMIUM: 0.2,
  BASIC: 0.05,
  DEFAULT: 0
};

const MINIMUM_AGE = 18;
const BASE_PRICE = 100;

/**
 * 校验用户是否符合年龄要求
 * @param {Object} user - 用户实体
 * @returns {boolean}
 */
const isUserEligible = (user) => user.age >= MINIMUM_AGE;

/**
 * 计算用户最终享有价格
 * @param {string} userType - 用户类型
 * @returns {number} 最终结算价格
 */
const calculateFinalPrice = (userType) => {
  const normalizedType = userType?.toUpperCase();
  const discountRate = DISCOUNT_RATES[normalizedType] ?? DISCOUNT_RATES.DEFAULT;
  return BASE_PRICE * (1 - discountRate);
};

/**
 * 处理用户核心业务流程
 * @param {Object} user - 用户实体
 * @param {Function} saveStrategy - 数据持久化注入接口
 * @returns {boolean} 流程是否成功完成
 */
function processUserData(user, saveStrategy = saveToDb) {
  if (!isUserEligible(user)) {
    console.warn("[Validation] User does not meet the minimum age requirement.");
    return false;
  }

  const finalPrice = calculateFinalPrice(user.type);
  
  // 采用依赖注入，提升代码的高可测试性与解耦
  saveStrategy(user, finalPrice);
  
  return true;
}
```
