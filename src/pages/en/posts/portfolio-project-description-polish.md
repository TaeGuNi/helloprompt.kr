>
> **Context:** I am writing project description to put in my portfolio. I want to refine it so **'Problem Solving Ability'** stands out, not simple function listing.
>
> **Input Material (My Project Draft):**
> [Write project description roughly here. e.g.: Made shopping mall app. Put payment function. Used React.]
>
> **Task:**
>
> 1. Rewrite content richly according to structure below.
>    - **Project Background:** Why this project started (User's pain/problem)
>    - **Key Challenges:** Hardest point during development/planning process
>    - **My Solution:** How solved that difficulty technically/logically
>    - **Results:** What achievement resulted (Quantitative/Qualitative)
> 2. Mix technical terms appropriately to increase reliability.
>
> **Constraints:**
>
> - Remove obvious words like "Implemented various functions".
> - Mention where and why specific Tech Stack was used.
