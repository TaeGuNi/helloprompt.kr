## 📝 Stoat: LLMが書いたコードを削除するアンチ・コパイロット

- **🎯 おすすめの対象:** レガシーコードに悩む開発者、コードレビュアー、シニアエンジニア
- **⏱️ 所要時間:** 1時間 → 5分に短縮
- **🤖 推奨モデル:** Claude 3.5 Sonnet, GPT-4o（コーディング特化モデル推奨）

- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「昨日AIに書かせたコード、今日見直したらスパゲッティになっていませんか？今はもうコードを『生成』する時代ではなく、『削除』する時代です。」_

2024年と2025年が「AIコーディングアシスタント」の年だったとすれば、2026年は間違いなく「AIコードクリーナー」の年になるでしょう。現在、開発者コミュニティで熱い視線を集めている**Stoat**は、従来のAIツールとは完全に逆の道を歩んでいます。GitHub CopilotやCursorが「より多くのコードを、より速く書くこと」に注力しているのに対し、Stoatの目的はたった一つ、**「コードを消し去ること」**です。

LLMを活用したコーディングツールの普及によって生産性は飛躍的に向上しましたが、その代償としてAIが生成したコードには、必要以上に冗長なロジックや重複が散見されるようになりました。本記事では、このStoatの哲学をベンチマークし、あなたが普段使っているLLMを「冷酷なコード清掃員（Stoat）」へと変貌させる強力なプロンプトをご紹介します。

## ⚡️ 3行まとめ (TL;DR)

1. AIコーディングアシスタントへの過度な依存により、コードベースが肥大化し「技術的負債」が爆発的に増加しています。
2. Stoatの概念を応用し、意味論的（セマンティック）な重複排除、ボイラープレートの削減、デッドコードの削除を通じてコードを徹底的にスリム化します。
3. 本記事の「アンチ・コパイロット・プロンプト」を適用するだけで、あらゆるLLMが即座に強力なリファクタリングツールへと進化します。

## 🚀 解決策：「Stoat アンチ・コパイロット・ダイエット・プロンプト」

### 🥉 Basic Version (基本型)

手早くコードの行数（LOC）を減らしたいときに使用してください。

> **役割 (Role):** あなたは世界で最も厳格な「アンチ・コパイロット（Anti-Copilot）」のシニア開発者です。
> **タスク (Task):** 以下のコードをレビューし、機能は100%維持したまま、コードの行数を最小限まで減らして書き直してください。不要な抽象化、重複したロジック、デッドコードは容赦なく削除してください。
> **コード (Code):** `[ここにリファクタリングするコードを貼り付けてください]`

### 🥇 Pro Version (専門家型)

プロダクションレベルのコードから安全に技術的負債を清算したいときに使用してください。

> **役割 (Role):** あなたはコードダイエットの職人であり、不要なコードを極端に嫌う「Stoat（アンチ・コパイロット）」のシニアエンジニアです。「最高のコードとは、コードが存在しないことだ（The best code is no code）」という哲学を猛信しています。
>
> **状況 (Context):**
>
> - 背景：ジュニア開発者やAIコーディングアシスタントが無秩序にコードを記述し、肥大化してしまった `[使用言語およびフレームワーク]` のプロジェクトです。
> - 目標：コードの可読性を高め、保守性を最大化し、全体の行数（LOC）を極限まで削減することです。
>
> **タスク (Task):**
>
> 1. **Semantic De-duplication（意味論的な重複排除）:** 見た目は異なっても同じ役割を果たすロジックを特定し、統合してください。
> 2. **Boilerplate Reduction（ボイラープレートの削減）:** 最新の言語構文を駆使して、冗長なパターンを徹底的に排除してください。
> 3. **Dead Code Hunting（デッドコード狩り）:** 呼び出されていないコードや過剰な防衛的プログラミング、不要な型宣言をすべて削除してください。
> 4. リファクタリングされたコードとともに、**削除されたコードの割合（%）**および**主な最適化ポイント**をまとめたレポートを作成してください。
>
> **制約事項 (Constraints):**
>
> - 既存コードのビジネスロジックや、時間的・空間的計算量を損なったり悪化させたりしては絶対にいけません。
> - 出力形式はMarkdownのコードブロックを使用し、リファクタリング前後の主な違いを箇条書き（ブレットポイント）で整理してください。
> - 確信が持てない最適化は無理に行わず、必ずコメントを残してください。
>
> **入力変数:**
>
> - 言語/フレームワーク: `[例: React, TypeScript]`
> - 元のコード:
>
>   `[ここに最適化したいスパゲッティコードを貼り付けてください]`

## 🙋 よくある質問 (FAQ)

- **Q: コードを削りすぎて、潜在的なバグを引き起こす心配はありませんか？**
  - A: Pro版プロンプトの「制約事項」にてビジネスロジックの維持を厳格に指示していますが、AIの出力を盲信することは危険です。適用後は必ず既存の単体テスト（Unit Test）を実行し、エッジケースが欠落していないか必ず二次検証を行ってください。

- **Q: どの言語モデルを使用すると最も精度の高い結果が得られますか？**
  - A: コードの複雑な文脈や構造を深く理解し、安全にリファクタリングを行う必要があるため、論理的推論能力に優れた **Claude 3.5 Sonnet** または **GPT-4o** の使用を強く推奨します。

## 🧬 プロンプトの解剖 (Why it works?)

1. **役割の逆転 (Role Reversal):** AIに対して「無限にコードを生成するアシスタント」ではなく、「コードを容赦なく削除する厳格なシニアエンジニア」という役割を与えることで、AIの根本的な振る舞いを変えています。
2. **具体的なターゲット設定 (Targeting):** 重複排除、ボイラープレートの削減、デッドコード狩りという3つの明確なアクションアイテムを提示することで、AIがコードを無意味に改変してシステムを破壊するリスクを防いでいます。
3. **心理的哲学の注入:** 「The best code is no code（最高のコードとは、コードが存在しないことである）」という過激なスローガンをペルソナに植え付け、コードの行数（LOC）削減を最優先のミッションとして認識させています。

## 📊 証明：Before & After

### ❌ Before (入力：AIが無秩序に記述したReactコード)

```typescript
import React, { useState, useEffect } from 'react';

const UserProfile = ({ user }) => {
  const [userName, setUserName] = useState('');
  const [userAge, setUserAge] = useState(0);

  useEffect(() => {
    if (user && user.name) {
      setUserName(user.name);
    }
  }, [user]);

  useEffect(() => {
    if (user && user.age) {
      setUserAge(user.age);
    }
  }, [user]);

  const handleNameClick = () => {
    console.log("Name clicked");
  }

  return (
    <div>
      {user ? (
        <div>
          <h1 onClick={handleNameClick}>{userName ? userName : 'Unknown'}</h1>
          <p>Age: {userAge ? userAge : 'N/A'}</p>
        </div>
      ) : (
        <div>Loading...</div>
      )}
    </div>
  );
};

export default UserProfile;
```

### ✅ After (結果：Stoatプロンプト適用後)

```typescript
import React from 'react';

const UserProfile = ({ user }) => {
  // Early returnで不要な条件付きレンダリングの深さを排除
  if (!user) return <div>Loading...</div>;

  return (
    <div>
      <h1 onClick={() => console.log("Name clicked")}>
        {user.name || 'Unknown'}
      </h1>
      <p>Age: {user.age || 'N/A'}</p>
    </div>
  );
};

export default UserProfile;
```

> **[Stoat レポート]**
>
> - **コード削減率:** 元のコードと比較して約 **55% の削減を達成** (36行 → 16行)
> - **主な最適化:** 不要な `useState` および `useEffect` による状態同期を完全に排除。Early Returnパターンの適用によるネストの深さ（Depth）の軽減。短絡評価（`||`）を活用し、三項演算子の冗長なボイラープレートを削減。

## 🎯 結論

コードを無限に生成することは、今や誰もが、そしてどんなAIにでもできる最も簡単な作業となりました。しかし、断片化されたロジックを整理し、不要なコードを削ぎ落として「本質」だけを残す作業には、依然として高度なエンジニアリングの洞察力が求められます。

このStoatの哲学を体現したプロンプトを活用し、肥大化したコードベースに徹底的なダイエットを施してみてください。より薄く、より洗練されたコードが、確実にあなたの退社時間を早めてくれるはずです。さあ、今すぐ思い切って不要なコードを「削除」しましょう！ 🍷
