В 2026 году экосистема Rust стала мощнее, чем когда-либо. Асинхронное программирование (Async Programming) наконец достигло стадии полной стабильности, а возможности оптимизации компилятора шагнули далеко вперед. Этот промпт поможет вам мгновенно отрефакторить устаревший код (legacy) в соответствии с самыми современными, безопасными и идиоматичными стандартами Rust 2026.

## 💡 Комментарий автора (Insight)

Этот промпт выходит далеко за рамки простой конвертации синтаксиса — он меняет саму парадигму проектирования на «идиоматичную» (Idiomatic) структуру, которую предпочитает компилятор Rust 2026. Строго ограничивая контекст асинхронного рантайма, мы не даем ИИ писать чересчур креативный (но нерабочий) код. Во время миграции высоконагруженного сервера в реальном проекте этот промпт помог мне всего за 3 минуты разрешить десятки ошибок компиляции, связанных с `async trait`. Это самый надежный способ одновременно добиться оптимизации производительности и гарантировать безопасность.

## 📊 Доказательство: До и После (Before & After)

### ❌ До (Ввод)

```rust
use std::future::Future;
use std::pin::Pin;

// Устаревший паттерн асинхронных трейтов (создает накладные расходы из-за использования Box)
trait DataFetcher {
    fn fetch_data(&self) -> Pin<Box<dyn Future<Output = Result<String, std::io::Error>> + Send + '_>>;
}

struct ApiClient;

impl DataFetcher for ApiClient {
    fn fetch_data(&self) -> Pin<Box<dyn Future<Output = Result<String, std::io::Error>> + Send + '_>> {
        Box::pin(async move {
            Ok("Data from 2023".to_string())
        })
    }
}
```

### ✅ После (Результат)

```rust
// Новейший стандарт асинхронных трейтов Rust 2026 (Native Async Trait)
trait DataFetcher {
    async fn fetch_data(&self) -> Result<String, std::io::Error>;
}

struct ApiClient;

impl DataFetcher for ApiClient {
    // Устранены накладные расходы на аллокацию Box::pin, максимизирована оптимизация компилятора
    async fn fetch_data(&self) -> Result<String, std::io::Error> {
        Ok("Data optimized for 2026".to_string())
    }
}
```
