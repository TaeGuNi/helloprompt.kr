## 📝 엑셀 함수 몰라도 OK: 개떡 같은 데이터 찰떡같이 정제하기

- **🎯 추천 대상:** CRM 데이터를 다루는 퍼포먼스 마케터, 고객 명단을 관리하는 영업 관리자, 영수증 및 전표를 처리하는 백오피스 담당자
- **⏱️ 소요 시간:** 2시간 → 5분 단축
- **🤖 추천 모델:** ChatGPT (Advanced Data Analysis 모드 권장), Claude 3.5 Sonnet

- ⭐ **난이도:** ⭐⭐☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐☆

> _"서울시 강남구, 강남구 삼성동, Seoul Gangnam... 제각각 입력된 주소록 데이터의 줄을 맞추느라 오늘도 모니터 앞에서 퇴근 시간을 넘기고 계시진 않나요?"_

흔히 데이터 분석 업무의 8할은 '노가다'라 불리는 전처리(Data Pre-processing)라고 하죠. 하지만 이제 우리가 데이터 과학자처럼 파이썬(Python)의 판다스(Pandas) 라이브러리를 능숙하게 다룰 필요는 없습니다. AI에게 우리가 원하는 **'결과물의 예시'** 단 몇 개만 툭 던져주면, 수천 줄의 엉망진창인 데이터 속에서도 스스로 완벽한 규칙을 찾아내어 칼같이 정렬해 주기 때문입니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **패턴 인식(Pattern Recognition):** AI는 정규표현식(Regex)의 마스터입니다. 복잡한 엑셀 수식을 고민하며 밤을 새울 필요 없이, 원하는 결과물의 형태만 명확히 보여주세요.
2. **퓨샷 러닝(Few-Shot Learning):** "딱 이렇게만 바꿔줘"라며 2~3개의 정답 예시를 제시하면, AI가 나머지 수만 건의 로우 데이터(Raw Data)를 동일한 패턴으로 자동 변환합니다.
3. **포맷 통일의 자유:** CSV, 마크다운 표 등 내가 실무에서 당장 복사해 붙여넣기 가장 편한 출력 형식을 지정하여 업무 효율을 극대화할 수 있습니다.

## 🚀 해결책: "데이터 클리너(Data Cleaner) 프롬프트"

### 🥉 Basic Version (기본형)

복잡한 조건 없이 빠르게 특정 텍스트만 추출하거나 단순한 형태 변환이 필요할 때 가볍게 사용해 보세요.

> **역할 (Role):** 너는 `[데이터 분석가]`야.
> **요청 (Task):** 아래 목록에서 휴대폰 번호만 정확히 추출해서 `010-xxxx-xxxx` 형식으로 통일해 줘.
>
> `[여기에 데이터 목록 붙여넣기]`

### 🥇 Pro Version (전문가형)

오탈자가 난무하고 형식이 제각각인 텍스트 데이터를, 당장 시스템에 업로드할 수 있는 완벽하게 구조화된 표 형태로 정제해야 할 때 사용하는 강력한 프롬프트입니다.

> **역할 (Role):** 너는 10년 차 시니어 데이터 엔지니어(Data Engineer)야.
>
> **상황 (Context):**
>
> - 배경: 고객들이 수기로 입력하여 오타가 많고 형식이 제각각인 `[주소 및 연락처]` 로우 데이터(Raw Data)가 있어.
> - 목표: 이 데이터를 당장 CRM 시스템에 업로드할 수 있도록 무결한 표준 포맷으로 정제(Normalization)하는 것.
>
> **요청 (Task):**
>
> 1. 아래 `[Input Data]`를 꼼꼼히 분석해 줘.
> 2. 지정된 `[변환 규칙]`에 맞춰 데이터를 가공하고, `[예시]`와 동일한 형태의 포맷으로 출력해 줘.
>
> **변환 규칙 (Rules):**
>
> - **주소:** '시/도' 및 '구/군' 단위까지 대한민국 표준 행정구역 명칭으로 통일할 것 (예: '서울' -> '서울특별시', '강남' -> '강남구').
> - **전화번호:** 모든 특수문자와 공백을 제거한 후, 반드시 `010-0000-0000` 형식으로 통일할 것. 번호가 누락된 경우 'NULL'로 표기.
> - **이름:** 이름 사이의 모든 공백을 제거할 것 (예: '홍 길 동' -> '홍길동').
>
> **예시 (Few-Shot Examples):**
>
> - 입력: "홍 길 동 / 010 1234 5678 / 서울 강남 삼성동"
> - 출력: | 홍길동 | 010-1234-5678 | 서울특별시 강남구 삼성동 |
> - 입력: "김철수 / 010-9876-4321 / 경기 성남 분당"
> - 출력: | 김철수 | 010-9876-4321 | 경기도 성남시 분당구 |
>
> **제약사항 (Constraints):**
>
> - 결과물은 반드시 CSV 형식(콤마로 구분)의 텍스트 코드블록으로만 출력해 줘. 인사말이나 부연 설명은 일절 금지.
> - 규칙을 적용해도 도저히 해석이 불가능한 쓰레기 데이터는 해당 행 맨 끝에 'Error' 태그를 추가해 줘. (환각 방지)
>
> **[Input Data]:**
> `[여기에 정제가 필요한 로우 데이터를 붙여넣으세요]`

## 💡 작성자 코멘트 (Insight)

이 프롬프트가 실무에서 압도적인 성능을 뿜어내는 핵심 비밀은 바로 **'퓨샷 프롬프팅(Few-Shot Prompting)'** 기법에 있습니다. AI에게 복잡한 변환 규칙을 구구절절 코딩하듯 설명하는 것보다, "A는 무조건 B로 바꿔"라는 명확한 정답 예시(Shot)를 2~3개만 툭 던져주는 것이 훨씬 직관적이며 치명적인 오류를 방지합니다.

특히 ChatGPT 유료 버전(Plus) 사용자라면 **Advanced Data Analysis** 기능을 반드시 적극적으로 활용하셔야 합니다. 엑셀(`.xlsx`)이나 `.csv` 원본 파일을 채팅창에 통째로 업로드한 뒤 위 프롬프트를 입력해 보세요. ChatGPT가 백그라운드에서 스스로 파이썬(Python) 코드를 작성하고 실행하여, 수만 건의 더러운 데이터도 단 몇 초 만에 흠잡을 데 없이 완벽하게 정제해 냅니다. 엑셀에서 끝없는 필터를 걸어가며 밤새워 수작업하던 시대는 이제 영원히 끝났습니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 회사의 민감한 고객 데이터를 그대로 업로드해도 안전한가요?**
  - A: **절대 안 됩니다.** 고객의 실명이나 실제 전화번호와 같은 민감한 개인정보는 반드시 엑셀 단계에서 사전 마스킹 처리(예: 홍\*동, 010-1234-\*\*\*\*)를 거쳐 비식별화된 가명 데이터로 변환한 후 업로드해야 합니다. 기업 차원의 안전한 데이터 처리가 필수적이라면, 외부 서버로 데이터가 전송되지 않는 사내망 전용 로컬(Local) LLM의 도입을 보안팀과 진지하게 논의해 보세요.

- **Q: 정제해야 할 데이터가 수만 건이라 프롬프트 입력창에 한 번에 다 들어가지 않습니다.**
  - A: 일반적인 AI 챗봇의 텍스트 입력창은 한 번에 처리할 수 있는 토큰(Token) 수에 물리적인 제한이 있습니다. 데이터를 50~100건씩 분할하여 일괄 처리하거나, 가장 추천하는 방식인 **파일 첨부 기능**(ChatGPT Plus의 Advanced Data Analysis, Claude Pro 등)을 활용하여 문서 전체를 한 번에 업로드하고 처리하는 방식을 권장합니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **페르소나 부여 (Role):** AI를 단순한 챗봇이 아닌 **'10년 차 시니어 데이터 엔지니어'**라는 날카로운 전문가 페르소나로 셋팅하여, 데이터의 무결성 유지와 복잡한 엣지 케이스(예외 상황) 처리에 대한 AI의 집중력을 비약적으로 끌어올렸습니다.
2. **명확한 퓨샷 예시 (Few-Shot Examples):** 두루뭉술하고 추상적인 지시를 철저히 배제했습니다. 입력값과 완벽한 기댓값을 1:1로 정확히 매칭하여 보여줌으로써, AI가 사용자의 의도에서 벗어나 엉뚱한 결과물을 낼 확률을 제로에 가깝게 통제합니다.
3. **예외 처리 안전망 (Error Handling):** 도저히 해석이 불가능한 쓰레기 데이터에 억지로 규칙을 끼워 맞춰 거짓 정보를 지어내는 치명적인 환각(Hallucination) 현상을 원천 차단했습니다. **'Error'** 태그를 강제하여, 실무자가 문제가 되는 행만 빠르게 찾아내어 후속 검수(Human-in-the-loop)를 진행할 수 있도록 안전망을 구축했습니다.
4. **출력 형식 강제 (Constraints):** 오직 순수한 CSV 포맷으로만 답변하도록 제약 조건을 걸었습니다. 사용자가 불필요한 인사말 없이 결과물만 그대로 복사하여 엑셀의 **'텍스트 나누기'** 기능으로 즉시 실무에 적용할 수 있도록, 불필요한 작업 동선을 극단적으로 최소화했습니다.
