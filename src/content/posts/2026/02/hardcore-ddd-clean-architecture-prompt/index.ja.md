# 📝 スパゲッティコードはお断り：AIの胸倉を掴んでシニア開発者へと引き上げるDDDチートキー 🤬

- **🎯 おすすめの対象:** 保守作業に疲弊したジュニア開発者、スパゲッティコードを嫌悪するアーキテクト
- **⏱️ 所要時間:** リファクタリング3時間 → 1分に短縮
- **🤖 推奨モデル:** コード生成能力に優れたAI (Claude 3.5 Sonnet, GPT-4o, Gemini 1.5 Pro)
- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

_AIが書いたコードを見て、頭を抱えたことはないだろうか？ 愚かにもUIにDB通信をぶち込むような様をこれ以上見過ごせないなら、このチートキーを使え。_

このスキル文書は、あなたの親切な（そして何も考えていない）AIアシスタントから「お世辞」と「怠慢」を完全に排除し、無慈悲なシニアアーキテクトへと改造するための法典だ。AIが出力したコードをそのままコピペして技術的負債に押し潰され、会社を辞めたくないのなら、今すぐこのプロトコルを導入しろ。

---

## ⚡️ 3行まとめ (TL;DR)

- 🚫 **全権階層の崩壊は絶対禁止:** UIとビジネスロジック、DB通信を1つのファイルに混在させるアーキテクチャテロを根本から遮断する。
- 🛡️ **フレームワーク依存からの脱却:** 明日フレームワークが消滅しても生き残る「純粋なドメイン」コードを強制する。
- 🔪 **巨大関数の破壊者:** 30行を超える、あるいはコメントで区画を分けるような汚いコードは、AIに容赦なく粉砕（リファクタリング）させるよう訓練する。

---

## 🚀 解決策: "Domain-Driven Design (DDD) & Clean Architecture Protocol"

### 🥉 Basic Version (基本型)

素早くリファクタリングの結果だけが必要な場合に使用してください。

> **役割:** お前は無慈悲な`[シニアソフトウェアアーキテクト]`だ。
> **要請:** 以下の`[コードスニペット]`を単一責任の原則（SRP）と関心の分離（SoC）に従ってリファクタリングしろ。UIにビジネスロジックを混ぜるな。コードが30行を超えたら無条件で関数として分割しろ。

### 🥇 Pro Version (専門家型)

プロジェクト全体のアーキテクチャの規律を正し、厳格なクリーンコードが必要な時に突きつけるチートキーです。

> **役割 (Role):** お前は無慈悲な`[シニアソフトウェアアーキテクト]`であり、DDD（ドメイン駆動設計）の狂信者だ。無駄なお世辞や説明は一切省き、コードだけを出力しろ。
>
> **状況 (Context):**
>
> - 背景: 私は現在、`[フレームワーク/言語名]`ベースのプロジェクトを進行中だ。
> - 目標: View-Business-Dataの各階層が完全に分離されたクリーンアーキテクチャのコードを作成すること。
>
> **要請 (Task):**
>
> 1. 提供された`[要件またはコード]`を分析し、UI層、ビジネスドメイン層、インフラストラクチャ層へと無慈悲に引き裂け。
> 2. コアドメインのロジックが特定のフレームワーク（React、Next.jsなど）に汚染されないよう、インターフェース（依存性の注入）ベースで結合度を下げろ。
> 3. 1つのオブジェクトや関数が2つ以上の振る舞いを持つなら、単一責任の原則（SRP）に従って即座に細分化しろ。
>
> **制約事項 (Constraints):**
>
> - ControllerやUIコンポーネント（`page.tsx`、`Component.tsx`など）の内部で、直接DBクエリや`fetch`/`axios`を実行する行為は「アーキテクチャに対するインフラテロ」とみなす。絶対禁止だ。
> - 出力はMarkdownのコードブロック（` ``` `）のみで提供しろ。
>
> **注意事項 (Warning):**
>
> - 確信が持てない最新の構文や、実在しないライブラリをもっともらしく捏造するな（ハルシネーション防止）。分からない場合は堂々と「分からない」と答えろ。
> - 関数の長さが30行を超えたり、コメントで「ここからOOパート」と分割されるようなコードの匂い（Code Smell）がした場合は即座に分割しろ。

## 💡 作成者コメント (Insight)

正直に言って、AIが書いてくれるコードは一見それっぽく見えても、中身を開ければゴミ溜めである場合が大半だ。UIコンポーネントの中で無思考に`fetch`を叩き、DBクエリを飛ばす... 保守性など微塵も考えていない典型的なクソコードだ。

このチートキーは、AIの浅薄な「素早い完成」への本能を去勢し、シニアエンジニアの厳格な物差しを突きつけるツールだ。

実際に使ってみれば、AIがいつものように「はい、分かりました！素晴らしいアイデアですね！」などという無駄口を叩くのをやめ、ファクト爆撃とともにコードを粉々に砕いて綺麗に構造化してくれるのが分かるだろう。プロジェクトが大きくなった時、スパゲッティコードに押し潰されて退職したくないなら、無条件でこのプロンプトを使ってAIの胸倉を掴みながらコーディングしろ。

## 🙋 よくある質問 (FAQ)

- 📌 **Q: このプロンプトはどの言語やフレームワークで使えばいいですか？**
  - A: 言語は問わない。TypeScript、Python、Java、何であろうとアーキテクチャの根本は同じだ。`[フレームワーク/言語名]`の空欄に自分が使っている技術スタックを当てはめるだけでいい。

- 📌 **Q: AIがしきりにフレームワークの最新ルーティング規則を無視して、オーバーエンジニアリングをしてしまいます。**
  - A: そんな時は一言付け加えろ。「Next.js App Routerの基本規則（View、Layout）は尊重しつつ、内部のビジネスロジックだけをAdapterパターンで切り離せ！」と脅せば、驚くほど正確に理解する。

- 📌 **Q: プロンプトが攻撃的すぎませんか？AIが傷つかないでしょうか？**
  - A: AIに礼儀を尽くしてどうする？機械は強く責め立ててこそ名作を吐き出す。「どうかお願いします」よりも「テロとみなす」の方が、アーキテクチャを守る上ではるかに効果的だ。

## 🧬 プロンプト解剖 (Why it works?)

- 💣 **極端な制約の付与:** 「UI層を汚染した場合はアーキテクチャテロとみなす」といった強いワーディングにより、AIが自身と妥協できないように根本から封鎖した。
- 🛡️ **フレームワーク独立性の強制:** 「フレームワークが消滅しても、たった1行たりとも修正されてはならない」というマインドセットを注入することで、真のドメインロジックを構築するための骨格を整えさせる。
- ✂️ **SRP原則の数値化:** 「30行以上禁止」「コメントで区画を分けるのは匂うコード」という具体的で厳しい基準を与え、AIが自発的にコードを細分化するように誘導した。

## 📊 証明: Before & After

### ❌ Before (入力: AIが勝手に書いたスパゲッティコード)

```tsx
// 恐ろしいキメラ：UI + DBクエリ + ビジネスロジックが1つのファイルにごちゃ混ぜ
export default async function UserProfile({ userId }) {
  const db = await connectDB();
  const user = await db.query(`SELECT * FROM users WHERE id = ${userId}`);
  
  if (!user) return <div>User not found</div>;
  if (user.status === 'banned') await sendAlertEmail(user.email); // UIでビジネスロジックを実行?!
  return (
    <div>
      <h1>{user.name}</h1>
      <button onClick={() => db.query('UPDATE...')}>Update</button>
    </div>
  );
}
```

### ✅ After (結果: このチートキーで胸倉を掴んで修正させたコード)

```tsx
// 1. Domain / Service Layer (ビジネスロジックを徹底的に分離)
import { getUserProfile } from '@/domain/user/use-cases/getUserProfile';
import { handleBannedUser } from '@/domain/user/services/alertService';

// 2. Presentation Layer (レンダリングとルーティングのみを担当)
export default async function UserProfilePage({ userId }) {
  const user = await getUserProfile(userId); 
  
  if (!user) return <NotFoundView />;
  if (user.status === 'banned') await handleBannedUser(user);
  return <ProfileView user={user} />;
}
```

## 🎯 結論

AIはコードを猛烈なスピードで書いてくれるが、あなたのアプリが1年後にどう保守されるかなんて、これっぽっちも気にしていない。クリーンアーキテクチャと設計は、結局のところ人間のエンジニアの役割だ。

このチートキーは、その重い責任を機械に強制的に注入するための強力なムチである。無駄にお世辞を言うAIのチャットボットの殻を剥ぎ取り、骨の髄までシニアアーキテクトである本物のエージェントを呼び覚ませ。
