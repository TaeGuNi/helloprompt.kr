- **🤖 Рекомендуемые модели:** Claude 3.5 Sonnet (лучший для код-ревью), GPT-4o

## ⚡️ Краткая выжимка (TL;DR)

1. Не просите ИИ банально «починить» баг — сразу назначайте ему роль «строгого Senior-ревьюера».
2. Задавайте максимально четкий контекст: прикладывайте логи ошибок и подробно описывайте заложенную бизнес-логику.
3. Откажитесь от слепого копипаста — вникайте в причины и аргументацию, которую ИИ предоставляет для каждого предложенного изменения.

> **Роль:** Ты — Senior `[Frontend/Backend]` разработчик с 10-летним опытом коммерческой разработки.
> **Задача:** Найди первопричину `[описание ошибки]`, возникающей в представленном ниже коде, и предложи исправленный вариант.
> **Код:** `[Вставьте сюда ваш проблемный код]`

### 🥇 Продвинутая версия (Pro Version)

> **Роль (Role):**
> Ты — безжалостный, но невероятно компетентный Staff Software Engineer с 15-летним опытом работы в компаниях уровня FAANG. Ты абсолютно бескомпромиссен во всём, что касается читаемости, производительности и долгосрочной поддерживаемости кода.

> **Задача (Task):**
> 1. Проведи глубокий анализ предоставленного кода и выяви фундаментальные проблемы (корневые причины багов, архитектурные антипаттерны и т. д.).

## 💡 Комментарий автора (Insight)

Ключевая особенность данного промпта кроется в создании суровой персоны «безжалостного техлида» и в бескомпромиссном требовании аргументировать каждую строчку измененного кода.

Самая разрушительная ошибка начинающих специалистов — это слепой копипаст (Ctrl+C, Ctrl+V) сгенерированных решений. Да, локальный баг исчезнет в ту же секунду, но ваша инженерная экспертиза не сдвинется ни на миллиметр.

Этот промпт дает ИИ совершенно иную директиву: не «просто сделай за меня», а «научи меня думать правильно». Вдумчиво анализируйте подсвеченные антипаттерны и визуально закрепляйте в памяти корректные идиоматические конструкции (idiomatic patterns). Если в блоке `Context` вы детально раскроете изначальный бизнес-замысел, нейросеть не просто поправит синтаксис, но и обнажит скрытые архитектурные уязвимости. Не пытайтесь защищать свой неидеальный код — позвольте виртуальному Staff-инженеру разобрать его на винтики. Именно осознанная работа над подобными ошибками является самым коротким путем к уверенному уровню Middle и Senior.

## 🙋 Часто задаваемые вопросы (FAQ)

- **Q: Насколько безопасно передавать ИИ проприетарный код компании в исходном виде?**
  - A: Это категорически недопустимо! Любые API-ключи, внутренние IP-адреса, доступы к базам данных и чувствительную бизнес-логику необходимо строго маскировать фиктивными (Dummy) значениями, а нейминг критичных переменных — анонимизировать. В вопросах информационной безопасности компромиссов быть не может.

- **Q: Что делать, если сгенерированный нейросетью код абсолютно неработоспособен и вызывает краш?**
  - A: Вероятнее всего, вы столкнулись с классической ИИ-галлюцинацией (Hallucination). Сохраняйте спокойствие: просто скопируйте полный трейсбек ошибки и скормите его обратно в чат. Достаточно короткой фразы: «Предложенный тобой код падает со следующей ошибкой: [лог]», и модель самостоятельно проведет отладку.

- **Q: Какая языковая модель на сегодняшний день демонстрирует наилучшие результаты в код-ревью?**
  - A: По состоянию на 2026 год, в задачах software engineering безоговорочным лидером остается Claude 3.5 Sonnet (и ее старшие версии). Она феноменально удерживает сложный контекст и предлагает поистине элегантные архитектурные решения. ChatGPT Plus (на базе GPT-4o) также выступает как крайне мощная и надежная альтернатива.

## 🧬 Разбор промпта (Why it works?)

1. **Инъекция «безжалостной» персоны:** Базовые настройки ИИ излишне «вежливы» — модель склонна закрывать глаза на жуткие структурные костыли, если скрипт в принципе способен выполниться. Роль бескомпромиссного Staff-инженера форсирует проведение ревью по строжайшим стандартам топовых технологических компаний.
2. **Приоритизация контекста (Context):** Машина парсит синтаксис, но Senior-инженер считывает «намерения». Предоставляя ИИ развернутое описание изначальной бизнес-задачи, вы открываете ему возможность предложить принципиально иной, алгоритмически более совершенный подход (оптимальные структуры данных, паттерны проектирования).
3. **Форсирование аргументации (Why):** Это ключевой катализатор вашего профессионального роста. Мы искусственно лишаем ИИ возможности просто «выплюнуть» готовый сниппет — он вынужден детально защищать свои архитектурные решения, тем самым напрямую передавая вам свою экспертизу.

## 📊 Доказательство: До и После (Before & After)

### ❌ Before (Ввод)

```javascript
// Код для получения и обработки данных пользователя (Пример спагетти-кода)
async function getU(id) {
  let res = await fetch('/api/user/' + id);
  let d = await res.json();
  if(d) {
    if(d.status === 'active') {
      let r = await fetch('/api/roles/' + d.roleId);
      let rd = await r.json();
      return { n: d.name, r: rd.name };
    } else {
      return null;
    }
  }
}
```

### ✅ After (Результат)

```javascript
// Результат рефакторинга от ИИ Senior-инженера
async function getActiveUserRoleSummary(userId) {
  try {
    const userResponse = await fetch(`/api/user/${userId}`);
    if (!userResponse.ok) throw new Error('Failed to fetch user');
    
    const user = await userResponse.json();
    
    // Применение паттерна раннего возврата (Early Return) избавляет от вложенных if
    if (!user || user.status !== 'active') {
      return null; 
    }
    const roleResponse = await fetch(`/api/roles/${user.roleId}`);
    if (!roleResponse.ok) throw new Error('Failed to fetch role');
    
    const role = await roleResponse.json();
    // Использование понятных имен переменных (n, r -> name, roleName)
    return { 
      name: user.name, 
      roleName: role.name 
    };
  } catch (error) {
    console.error(`[getActiveUserRoleSummary] Error for userId ${userId}:`, error);
    throw error; // Предотвращение скрытия ошибок
  }
}
```

*(Комментарий ИИ: Глубокая вложенность условных операторов была устранена за счет паттерна Early Return, что кардинально повысило читаемость. Идентификаторы переменных приведены к семантически понятному виду, а также интегрирован надежный блок перехвата и обработки исключений.)*
