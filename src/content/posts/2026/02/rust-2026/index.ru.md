В 2026 году экосистема Rust стала мощнее, чем когда-либо. Асинхронное программирование (Async Programming) наконец-то достигло абсолютной стабильности, а возможности оптимизации компилятора вышли на совершенно новый уровень. Однако ручная миграция существующих проектов на стандарты 2026 года — это по-прежнему мучительно долгая и ресурсоёмкая задача. Этот промпт поможет вам мгновенно отрефакторить устаревший код (legacy) в полном соответствии с самыми современными, безопасными и идиоматичными паттернами Rust 2026 года.

## 🚀 Решение: "Мастер миграции Rust 2026"

### 🥉 Базовая версия (Basic Version)

Используйте этот вариант, когда нужно быстро подсветить точки роста в вашем текущем коде.

> **Роль:** Ты — `[Core-контрибьютор Rust 2026]`.
> **Задача:** Оптимизируй этот `[устаревший код на Rust]` в строгом соответствии с новейшими стандартами асинхронного программирования 2026 года.


### 🥇 Продвинутая версия (Pro Version)

Применяйте этот промпт для безупречного production-ready рефакторинга и получения глубокой аналитики по изменённому коду.

> **Роль (Role):** Ты — `[Core-контрибьютор Rust 2026 и Senior-архитектор]` с более чем 10-летним опытом системного программирования.
>
> **Контекст (Context):**
> 
> - Фон: Наша команда осуществляет масштабную миграцию `[асинхронного кода на старых версиях Rust]`, чтобы полностью адаптировать его под новейший компилятор и реалии асинхронной экосистемы 2026 года.
> - Цель: Превратить старую кодовую базу в `[безопасный и идиоматичный код Rust 2026]`, который выжимает максимум производительности за счёт современных фич компилятора.
>
> **Задача (Task):**
> 
> 1. Проведи глубокий анализ предоставленного кода и выполни его рефакторинг по стандартам асинхронного программирования Rust 2026.
> 2. Подробно объясни в комментариях к коду, какие именно новые механизмы оптимизации компилятора были задействованы.
> 3. Составь экспертную оценку ожидаемого прироста производительности и гарантий безопасности памяти для изменённых участков.
>
> **Вводные данные (Variables):**
> 
> - `[асинхронный код на старых версиях Rust]`: (Вставьте сюда устаревший код для рефакторинга)
>
> **Ограничения (Constraints):**
> 
> - Структурируй ответ, используя маркдаун-списки и чистые блоки кода.
> - Применяй исключительно те фичи, которые официально стабилизированы (Stable) к 2026 году. (Использование экспериментальных функций из Nightly-сборок строго запрещено).
> - Итоговый код не должен вызывать предупреждений при проверке `cargo clippy`.
>
> **Предупреждение (Warning):**
> 
> - Не выдумывай несуществующий синтаксис. Если сомневаешься в наличии новой фичи, используй надёжный, проверенный Stable-синтаксис. (Абсолютная защита от галлюцинаций ИИ).

## 💡 Комментарий автора (Insight)

Этот промпт выходит далеко за рамки примитивной замены синтаксиса. Он трансформирует саму парадигму проектирования, приводя её к той самой идиоматичной (Idiomatic) структуре, которую так "любит" компилятор Rust 2026. Строго ограничивая контекст выполнения только стабильными фичами, мы отсекаем попытки ИИ написать чересчур "креативный", но абсолютно нерабочий код. В ходе недавней миграции высоконагруженного сервера в реальном проекте этот подход позволил мне буквально за 3 минуты распутать клубок из десятков ошибок компиляции, связанных с `async trait`. На сегодняшний день это самый надёжный инструмент для одновременного буста производительности и железобетонной гарантии безопасности.

## 📊 Доказательство: До и После (Before & After)

### ❌ До (Ввод)

```rust
use std::future::Future;
use std::pin::Pin;

// Устаревший паттерн асинхронных трейтов (создает накладные расходы из-за использования Box)
trait DataFetcher {
    fn fetch_data(&self) -> Pin<Box<dyn Future<Output = Result<String, std::io::Error>> + Send + '_>>;
}

struct ApiClient;

impl DataFetcher for ApiClient {
    fn fetch_data(&self) -> Pin<Box<dyn Future<Output = Result<String, std::io::Error>> + Send + '_>> {
        Box::pin(async move {
            Ok("Data from 2023".to_string())
        })
    }
}
```

### ✅ После (Результат)

```rust
// Новейший стандарт асинхронных трейтов Rust 2026 (Native Async Trait)
trait DataFetcher {
    async fn fetch_data(&self) -> Result<String, std::io::Error>;
}

struct ApiClient;

impl DataFetcher for ApiClient {
    // Устранены накладные расходы на аллокацию Box::pin, максимизирована оптимизация компилятора
    async fn fetch_data(&self) -> Result<String, std::io::Error> {
        Ok("Data optimized for 2026".to_string())
    }
}
```
