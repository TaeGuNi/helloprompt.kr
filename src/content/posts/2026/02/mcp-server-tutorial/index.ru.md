В реалиях 2026 года **MCP (Model Context Protocol)** стал де-факто стандартом для подключения LLM к внешним данным. Считайте, что это «USB-порт» для моделей ИИ. Вместо того чтобы каждый раз скармливать контекст вручную, достаточно один раз подключить сервер, и модель сама будет в реальном времени «читать» ваши файлы, базы данных и внутренние API.

Уже существуют отличные универсальные MCP-серверы для интеграции с GitHub или файловой системой. Но настоящая магия начинается, когда вы подключаете к LLM **уникальную бизнес-логику вашей компании**.

Сегодня мы не будем писать скучный шаблонный код вручную. Мы покажем промпт, который заставит ИИ всего за 5 минут сгенерировать идеальный кастомный MCP-сервер на TypeScript.

> **Роль:** Ты Senior TypeScript разработчик.
> **Задача:** Напиши простой код MCP-сервера с использованием `@modelcontextprotocol/sdk`, который выполняет следующую задачу: `[укажите цель, например: показывает погоду в определенном городе]`.

<br>

### 🥇 Pro Version (Продвинутая версия)

Используйте, когда нужен надежный и масштабируемый код сервера, готовый к применению в рабочей среде.

> **Роль (Role):** Ты Senior TypeScript инженер, глубоко разбирающийся в экосистеме Model Context Protocol (MCP).
>
> **Контекст (Context):**
>
> - Ситуация: Мне нужен кастомный MCP-сервер для интеграции локальных данных или внутреннего API с LLM.
> - Цель: Сгенерировать полный, исполняемый код MCP-сервера в виде одного файла (`index.ts`), идеально подходящий под мои требования.
>
> **Задача (Task):**
>
> 1. Напиши код MCP-сервера, отвечающий следующим требованиям:
>    - **Назначение:** `[Подробно опишите назначение сервера. Например: чтение последних 50 строк определенного лог-файла]`
> 2. Код должен быть снабжен комментариями и оптимизирован для формата "скопировал-вставил", чтобы я мог сразу запустить его в терминале.
>
> **Ограничения (Constraints):**
>
> - Обязательно используй официальный пакет `@modelcontextprotocol/sdk`.
> - Если входные параметры инструмента (tool) сложные, используй библиотеку `zod` для валидации схемы (рекомендуется).
> - При создании сервера должны быть полностью реализованы интерфейсы `ListToolsRequestSchema` и `CallToolRequestSchema`.
> - Внутри обработчика инструмента (Handler) обязательно используй конструкцию `try/catch` для обработки исключений, чтобы сервер не упал при ошибке.
> - **Выведи только один блок кода на TypeScript без каких-либо дополнительных объяснений.**
>
> **Предупреждения (Warning):**
>
> - Не выдумывай несуществующие методы SDK. Если не уверен, используй стандартные API Node.js в качестве обходного пути.

## 💡 Комментарий автора (Insight)

Главная сила этого промпта заключается не в абстрактной просьбе "напиши код", а в жестком требовании соблюдать **ключевые спецификации протокола (обработка запросов ListTools и CallTool)**, которыми должен обладать любой MCP-сервер.

Самое большое преимущество создания собственного MCP-сервера — это **абсолютная безопасность и приватность (Privacy)**. При использовании общедоступных инструментов веб-поиска ваши запросы проходят через внешние серверы. Однако с кастомным MCP-сервером, подключенным к локальной базе данных PostgreSQL или внутреннему API, данные передаются безопасно и остаются полностью под вашим контролем. Вам больше никогда не придется копировать конфиденциальные внутренние документы в окно чата.

Начните с создания простого инструмента "только для чтения", например, для просмотра локальных лог-файлов с помощью этого промпта. Как только вы поймете принцип его работы, вы сможете расширить функционал до изменения данных в базе, создав тем самым своего идеального ИИ-ассистента.

## 🙋 Часто задаваемые вопросы (FAQ)

- **Вопрос: Могу ли я использовать этот промпт, если вообще не умею программировать?**
  - Ответ: Да, конечно! Просто опишите вашу задачу в блоке `[Назначение]` простыми словами, например: "Функция, которая сообщает погоду в Москве", и Claude сам напишет идеальный код. Вам останется только скопировать его и запустить.

- **Вопрос: Как запустить и подключить сгенерированный код сервера?**
  - Ответ: Сохраните код в файл `index.ts`, а затем добавьте путь к этому скрипту в секцию `mcpServers` конфигурационного файла Claude Desktop (`claude_desktop_config.json`). Интеграция произойдет автоматически.

- **Вопрос: Сервер, созданный с помощью этого промпта, работает только в Claude?**
  - Ответ: Нет. Model Context Protocol — это открытый стандарт. Сервер будет одинаково хорошо работать с Ollama, Cursor и любыми другими платформами и клиентами, поддерживающими MCP.

## 🧬 Анатомия промпта (Why it works?)

1.  **Строгий контроль через ограничения (Constraints):** Принудительное использование `@modelcontextprotocol/sdk` и `zod`, а также явное указание обязательных схем (`ListToolsRequestSchema`, `CallToolRequestSchema`) на корню пресекает галлюцинации ИИ и отклонения от спецификации.
2.  **Один файл и вывод только кода:** Требование поместить всё в один файл (`index.ts`) вместо сложной многофайловой структуры и запрет на лишние пояснения максимально сокращают время до получения результата (Time-to-Value).
3.  **Принудительная обработка исключений:** Заранее прописанная логика `try/catch` предотвращает падение всего MCP-сервера из-за непредвиденных ошибок во время выполнения инструмента.
