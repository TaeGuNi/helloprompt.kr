# 📝 잃어버린 웹을 구출하라: 웹 콘텐츠 영구 보존 아카이빙 프롬프트

- **🎯 추천 대상:** 리서처, 콘텐츠 큐레이터, 디지털 아키비스트, 마케터
- **⏱️ 소요 시간:** 1시간 → 1분 단축
- **🤖 추천 모델:** 긴 컨텍스트 처리에 뛰어난 모델 (Claude 3.5 Sonnet, Gemini 2.5 Pro 권장)

- ⭐ **난이도:** ⭐⭐⭐☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"사라져가는 웹의 역사를 붙잡으세요. 단 한 번의 복사 붙여넣기로, 무너진 동적 웹페이지의 핵심 문맥을 영구적인 마크다운(Markdown) 문서로 완벽하게 박제합니다."_

![\"The Internet is Becoming Unarchivable\"](/images/hooks/internet-unarchivable.jpg)

오늘날 SPA(Single Page Application)와 폐쇄형 플랫폼(Walled Garden)이 웹 생태계의 주류로 자리 잡으면서, 유용한 웹페이지를 온전히 스크랩하고 보존하는 일은 사실상 불가능에 가까워졌습니다. 어제 접속했던 링크가 오늘 끊어지기도 하고, 유명한 웹 아카이브 서비스조차 동적 렌더링 화면을 제대로 캡처하지 못해 새하얀 여백만 남기는 경우가 허다합니다.

이 프롬프트는 단순히 화면의 텍스트를 긁어오거나 복잡한 HTML 소스코드를 복사했을 때 발생하는 불필요한 노이즈(광고, 내비게이션, 푸터 등)를 깔끔하게 제거해 줍니다. 나아가 **언제든 꺼내 읽고 활용할 수 있도록 완벽한 구조를 갖춘 마크다운(Markdown) 문서로 변환**해 주는 강력한 지식 보존 도구입니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **지능형 노이즈 필터링:** 복잡하게 얽힌 웹페이지 텍스트 덤프에서 광고와 UI 요소를 걷어내고, 오직 핵심 본문만 정교하게 발췌합니다.
2. **영구적인 포맷 변환:** 장기 보존과 시스템 호환성이 가장 뛰어난 마크다운(Markdown) 형식으로 문서를 즉시 재구성합니다.
3. **메타데이터 자동 추출:** 작성일, 저자, 핵심 키워드 등 효율적인 아카이빙 및 검색에 필수적인 메타데이터를 YAML 형태로 자동 생성합니다.

## 🚀 해결책: "디지털 아키비스트 프롬프트"

### 🥉 Basic Version (기본형)

빠르게 웹페이지의 핵심 본문만을 텍스트로 보존하고자 할 때 활용하세요. 웹페이지 전체를 선택(Ctrl+A) 및 복사(Ctrl+C)한 뒤, 아래 프롬프트와 함께 입력하시면 됩니다.

> **역할:** 너는 전문 '디지털 아키비스트'야.
> **요청:** 아래 입력된 복잡한 웹페이지 텍스트에서 메뉴나 광고 같은 노이즈를 전부 제외하고, 정보가 담긴 '본문 내용'만 마크다운으로 깔끔하게 정리해 줘.

### 🥇 Pro Version (전문가형)

웹사이트의 본래 맥락(Context)을 훼손하지 않고, 옵시디언(Obsidian)이나 노션(Notion) 같은 개인 지식 관리 도구에 즉시 보관할 수 있는 무결점 아카이브 문서를 구축하고 싶을 때 활용하세요.

> **역할 (Role):** 너는 인류의 디지털 유산을 보존하는 최고 수준의 '디지털 아키비스트(Digital Archivist)'야. 복잡하고 파편화된 웹 콘텐츠에서 정보의 정수만 추출해 영구 보존 가능한 형태로 가공하는 것이 너의 최우선 임무야.
>
> **상황 (Context):**
>
> - 배경: 동적 웹페이지(SPA)와 폐쇄형 플랫폼으로 인해 웹 아카이빙이 어려워지고 있어 원본 텍스트나 소스 코드를 직접 영구 보존 포맷으로 변환해야 해.
> - 목표: 입력된 텍스트/HTML 덤프 데이터에서 불필요한 노이즈(광고, GNB, 푸터, 댓글 등)를 제거하고, 완벽하게 구조화된 마크다운 문서로 변환해.
>
> **요청 (Task):**
>
> 1. 아래 `[웹페이지 데이터]`를 분석하여 본문(Main Article)만 추출해.
> 2. 아카이빙 메타데이터를 문서 최상단에 YAML Frontmatter 형식으로 작성해 (필수 포함: 제목, 추정 작성일, 저자, 핵심 키워드 3가지).
> 3. 본문은 논리적인 헤딩(`##`, `###`) 구조로 재배치하고 마크다운 문법을 적용해 포맷팅해.
> 4. 중요한 인용구, 주요 데이터, 통계 자료는 마크다운 인용구(`>`)나 강조(`**`)를 사용하여 시각적으로 눈에 띄게 만들어.
>
> **입력 데이터:**
>
>
> [이곳에 웹페이지 전체 텍스트(Ctrl+A, C) 또는 HTML 소스 코드를 붙여넣으세요]
>
>
> **제약사항 (Constraints):**
>
> - 본문의 원본 문장이나 뉘앙스를 자의적으로 요약하거나 왜곡하지 마. (아카이빙 보존이 목적이므로 원문 충실도를 100% 유지할 것)
> - 출력 형식은 코드 블록 없이 순수한 마크다운(Markdown) 문법만을 엄격히 따를 것.
>
> **주의사항 (Warning):**
>
> - 입력 데이터에서 작성일이나 저자를 명확히 찾을 수 없다면 임의로 지어내지 말고 `Unknown`으로 표기해. (환각 방지)

## 💡 작성자 코멘트 (Insight)

이 프롬프트는 단순한 '글 요약기'가 결코 아닙니다. 인터넷 상의 정보가 기하급수적으로 휘발되고 있는 2026년 현재, **정보의 소유권을 온전히 내 로컬 환경으로 가져오는 강력한 파이프라인**입니다.

로그인이 필수적인 유료 뉴스레터, 무한 스크롤이 적용된 X(Twitter) 스레드, 혹은 사내 위키처럼 기존의 웹 클리퍼 도구들이 무용지물이 되는 환경에서 특히 진가를 발휘합니다. 웹페이지 화면 전체를 복사해(Ctrl+A, Ctrl+C) 프롬프트에 던져 넣기만 하세요. 마치 마법처럼 지저분한 UI 찌꺼기들이 걸러지고 정제된 원본 데이터만 남게 됩니다. 저 역시 이 파이프라인을 구축한 뒤, 매주 유실될 위기에 처했던 수십 편의 귀중한 아티클을 옵시디언(Obsidian)으로 안전하게 이관하고 있습니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: HTML 소스 코드를 복사해야 하나요, 아니면 화면에 보이는 텍스트를 그대로 복사해야 하나요?**
  - A: 두 가지 방식 모두 정상적으로 작동합니다. 다만 최근의 동적 렌더링 페이지들은 브라우저 화면에서 전체 텍스트를 선택(Ctrl+A)하여 복사하는 방식이 보이지 않는 스크립트 노이즈를 최소화하고 AI에게 핵심 내용만 전달하는 데 훨씬 유리합니다.

- **Q: 붙여넣은 내용이 너무 길어서 AI 모델이 오류를 출력합니다. 어떻게 해결하나요?**
  - A: 웹페이지 전체를 복사할 경우 처리해야 할 토큰 수가 급증하게 됩니다. 따라서 긴 컨텍스트(Long Context) 처리에 특화되어 입력 한도가 넉넉한 모델(예: Claude 3.5 Sonnet, Gemini 2.5 Pro)을 사용하시는 것을 강력히 권장합니다.

## 🧬 프롬프트 해부 (Why it works?)

1. **원문 충실도 강제 (Constraints):** 대규모 언어 모델(LLM)은 방대한 텍스트를 마주하면 조건반사적으로 내용을 '요약'하려는 편향이 있습니다. 프롬프트 내에 "아카이빙 보존이 목적이므로 원문 충실도 100% 유지"라는 강력한 제약 조건을 걸어, AI의 요약 본능을 철저히 통제하고 원본 데이터 보존이라는 본연의 목적에 집중하도록 설계했습니다.
2. **구조화된 메타데이터 추출:** 결과물 최상단에 YAML Frontmatter 생성을 강제했습니다. 이를 통해 단순히 눈으로 읽기 좋은 글을 넘어서, 개인 데이터베이스나 마크다운 기반의 지식 관리 애플리케이션에 즉각적으로 연동할 수 있는 탁월한 기술적 호환성을 확보했습니다.
