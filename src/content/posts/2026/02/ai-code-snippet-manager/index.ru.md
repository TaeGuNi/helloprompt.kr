# 📝 Идеальный порядок в хаосе кода с помощью ИИ! (AI Code Snippet Manager)

- **🎯 Целевая аудитория:** Junior-разработчики (1-3 года опыта), желающие создать собственную базу знаний, и все, кто устал от бесконечного копипаста.
- **⏱️ Затраты времени:** Сокращение с 30 минут → до 1 минуты.
- **🤖 Рекомендуемые модели:** Claude 3.5 Sonnet (превосходен для рефакторинга и документирования кода), ChatGPT (GPT-4o).

- ⭐ **Сложность:** ⭐☆☆☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Полезность:** ⭐⭐⭐⭐⭐

> _«Вроде бы я уже писал этот код... Как же он назывался?» Если вы опять тратите полчаса на поиск по своим же файлам — пора остановиться._

У каждого разработчика где-то на рабочем столе припрятан текстовый файл, похожий на «пещеру сокровищ». Там лежат регулярные выражения, с трудом найденные на StackOverflow, гениальные служебные функции и файлы конфигурации Nginx, которые «обязательно когда-нибудь пригодятся».

Но когда они нужны здесь и сейчас, вспомнить, где они лежат и как именно работают, бывает невозможно, поэтому приходится гуглить всё с нуля. Главное в сниппетах (фрагментах кода) — это легкость поиска и наличие понятных примеров использования. Доверьте ИИ свои разрозненные куски кода, и они мгновенно превратятся в структурированную библиотеку со стандартными комментариями, тегами и примерами из реальной практики. Представляем промпт, который превратит ваш хаотичный блокнот в систематизированную базу знаний (Knowledge Base).

## ⚡️ Краткая выжимка (TL;DR)

1. **Превращение в доступный актив:** Присвоение непонятным кускам кода четких заголовков, тегов и метаданных.
2. **Автоматическая документация:** Мгновенная генерация стандартизированных комментариев (например, JSDoc) для параметров и возвращаемых значений, а также рабочих примеров.
3. **Улучшение качества кода:** ИИ исправляет нарушенное форматирование и неоднозначные названия переменных, делая код сразу готовым к повторному использованию.

## 🚀 Решение: «ИИ-менеджер сниппетов кода»

### 🥉 Базовая версия (Basic Version)

Используйте, когда нужно быстро привести в порядок структуру и комментарии.

> **Роль:** Ты — Senior Software Engineering Mentor.
>
> **Задача:** Проанализируй предоставленный код, исправь форматирование, напиши краткое резюме из одного предложения о том, что он делает, и добавь ключевые комментарии.
>
> **Код:**
> `[Вставьте сюда скопированный код]`

\

### 🥇 Продвинутая версия (Pro Version)

Используйте для создания «идеального Markdown-документа», готового для добавления в вашу личную базу знаний, Notion или Obsidian.

> **Роль (Role):**
> Ты — Senior Technical Writer и разработчик с 10-летним стажем, который одержим понятной документацией и чистым кодом (Clean Code).
>
> **Контекст (Context):**
>
> - База: Я собираю разрозненные фрагменты кода из интернета и прошлых проектов, чтобы создать собственную базу знаний (в Notion или Obsidian).
> - Цель: Превратить этот код в идеально задокументированный Markdown-сниппет, чтобы его было легко найти в поиске, и чтобы я в будущем мог понять и использовать его за одну секунду.
>
> **Задача (Task):**
>
> Проанализируй предоставленный `[Код]` и создай Markdown-документ следующей структуры:
>
> 1. Метаданные (Metadata): `Title` (четкая цель), `Language`, `Tags` (3-5 ключевых слов для поиска).
> 2. Описание (Description): Резюме из 1-2 предложений о том, какую проблему решает данный код.
> 3. Рефакторинг и комментарии (Refactored Code): Исправь отступы и форматирование. Добавь подробные комментарии для параметров (`@param`) и возвращаемых значений (`@return`) в стиле JSDoc (или стандартных докстрингов выбранного языка). Если есть переменные с неоднозначными названиями, переименуй их для улучшения читаемости.
> 4. Пример использования (Usage Example): Напиши простой рабочий пример того, как этот код можно вызвать и применить в реальном проекте.
>
> **Ограничения (Constraints):**
>
> - Вывод должен быть оформлен в виде единого Markdown-документа.
> - Не меняй кардинально названия функций и переменных без необходимости, постарайся максимально сохранить первоначальную логику и задумку.
>
> **Предупреждения (Warning):**
>
> - Если в коде присутствуют конфиденциальные данные (IP-адреса, пароли и т. д.), сначала выведи предупреждающее сообщение.
> - `[Код]` я буду вводить каждый раз заново, поэтому на первое сообщение просто ответь: "Пожалуйста, введите код для обработки."

Инвестируйте всего 1 минуту ради «себя в будущем», и потом вы сэкономите долгие часы.

## 🙋 Часто задаваемые вопросы (FAQ)

- **В: Можно ли использовать этот промпт, если политика безопасности компании запрещает передавать код внешним ИИ-сервисам?**
  - О: Обязательно удаляйте или маскируйте (например, `XXX`, `1234`) любые конфиденциальные данные: API-ключи, пароли БД, внутренние IP-адреса серверов и проприетарную бизнес-логику. Если требуется строгая безопасность, рекомендуется использовать open-source модели, такие как Llama 3, развернутые локально.

- **В: Что делать, если ИИ изменит форматирование или стиль именования переменных вопреки моим предпочтениям?**
  - О: Вы можете добавить ваши стандарты кодирования в блок **Ограничения (Constraints)** промпта. (Например: _«Всегда используй camelCase для названий переменных и одинарные кавычки вместо двойных»_).

- **В: Поддерживаются ли все языки программирования?**
  - О: Да. Промпт отлично работает не только с популярными языками, такими как Python, JavaScript, Java, C++, но и с SQL-запросами, Bash-скриптами и даже сложными файлами конфигурации Dockerfile или Kubernetes YAML.

```javascript
function cd(d1, d2) {
  let t = new Date(d1).getTime() - new Date(d2).getTime();
  return Math.floor(t / (1000 * 60 * 60 * 24));
}
```

### ✅ После (Результат)

````markdown
**Title:** Функция вычисления разницы в днях между двумя датами
**Language:** JavaScript
**Tags:** `#Date`, `#Utility`, `#TimeDifference`, `#JavaScript`

**Description:**
Служебная функция, которая принимает две строки дат (или объекты Date) и возвращает точную разницу в днях в виде целого числа.

**Refactored Code:**

```javascript
/**
 * Вычисляет разницу в днях между двумя датами.
 *
 * @param {string|Date} targetDate - Целевая дата для сравнения
 * @param {string|Date} baseDate - Базовая дата
 * @returns {number} Разница в днях (возвращает положительное число, если targetDate находится в будущем)
 */
function calculateDaysDifference(targetDate, baseDate) {
  const timeDifference =
    new Date(targetDate).getTime() - new Date(baseDate).getTime();
  return Math.floor(timeDifference / (1000 * 60 * 60 * 24));
}
```
````

**Usage Example:**

```javascript
const today = "2026-02-12";
const dDay = "2026-12-25";

const daysLeft = calculateDaysDifference(dDay, today);
console.log(`До Рождества осталось ${daysLeft} дней!`);
// Вывод: До Рождества осталось 316 дней!
```
