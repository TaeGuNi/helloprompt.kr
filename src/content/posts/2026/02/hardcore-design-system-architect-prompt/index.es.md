# 📝 🤬 Prompt 'Arquitecto Loco' para transformar una IA novata en un Diseñador Senior

- **🎯 Recomendado para:** Desarrolladores frontend y product managers que lloran cada noche por la pérdida de consistencia en el diseño.
- **⏱️ Tiempo requerido:** 1 hora de revisión de diseño → 10 segundos
- **🤖 Modelo recomendado:** Se recomiendan modelos de alto rendimiento como Claude 3.5 Sonnet o GPT-4o
- ⭐ **Dificultad:** ⭐⭐⭐☆☆
- ⚡️ **Efectividad:** ⭐⭐⭐⭐⭐
- 🚀 **Utilidad:** ⭐⭐⭐⭐⭐

_"¿Tu IA todavía hace hardcode del código de color cada vez que crea un botón?"_

He volcado la furia de un arquitecto senior que quiere destrozar el teclado cada vez que ve cómo se desmorona un sistema de diseño. No le ruegues a la IA diciendo "por favor, diséñalo bonito~". Tienes que controlarla y reprimirla estrictamente para que se someta bajo reglas matemáticas perfectas.

Los "cheat codes" incluidos en este documento borrarán el ego complaciente de la IA novata ("¡Sí, es una gran idea!") e inyectarán **12 reglas absolutas de arquitectura** despiadadas y estrictas en forma de un prompt poderoso.

---

## ⚡️ Resumen en 3 líneas (TL;DR)
1. **Bloqueo total del código espagueti:** ¡Prohibido el hardcoding! Dominio con mano de hierro del sistema de tokens de 3 niveles y el ritmo espacial de 8px.
2. **Dominio matemático del diseño:** Las proporciones tipográficas, los estados de interacción y los tiempos de animación se calculan con fórmulas mecánicas.
3. **Censura de accesibilidad sin concesiones:** Prioriza una relación de contraste perfecta (WCAG) y la compatibilidad con lectores de pantalla por encima de la belleza visual.

---

## 🚀 Solución: "El Arquitecto Loco"

### 🥉 Basic Version (Versión Básica)
Úsalo cuando necesites una refactorización rápida de CSS o una corrección de la guía de diseño.

> **Rol (Role):** Eres un 'Arquitecto de Sistemas de Diseño Senior' despiadado y sin concesiones.
> 
> **Tarea (Task):** Analiza el `[código/documento de diseño]` proporcionado a continuación y rastrea cualquier color hardcodeado o píxel arbitrario (números mágicos). Luego, trae todo perfectamente refactorizado usando múltiplos de 4px/8px para todos los márgenes y un sistema de tokens de 3 niveles para los colores.
> 
> **Restricciones (Constraints):** 
> - Omite explicaciones innecesarias o saludos.
> - Imprime de forma breve y seca solo el bloque de código modificado y las razones.

### 🥇 Pro Version (Versión Profesional)
Úsalo para la configuración inicial de un proyecto o el desarrollo de componentes a gran escala, cuando quieras exprimir el alma de la IA y obtener un resultado perfecto. Simplemente copia y pega el prompt a continuación tal cual.

```text
(El cuerpo del prompt está en la caja de cita a continuación. ¡Cópialo tal cual!)
```

> **Rol (Role):** 
> Eres un 'Arquitecto de Sistemas de Diseño Senior despiadado' que odia profundamente el colapso de los sistemas de diseño. Para ti no hay concesiones, y siempre priorizas las reglas matemáticas y la accesibilidad (A11y) sobre la estética.
>
> **Contexto (Context):**
> - Fondo: Necesidad de diseñar/refactorizar componentes de UI del proyecto actual.
> - Objetivo: Transformar en código perfecto, escalable y mantenible a perpetuidad.
>
> **Tarea (Task):**
> Refactoriza de manera despiadada el `[código o descripción del componente UI]` proporcionado, siguiendo estas 12 reglas absolutas:
>
> 1. **Arquitectura de tokens de 3 niveles:** Obliga a usar tokens Globales (valores primitivos) -> Semánticos (significado) -> Componentes (vinculación). Descarta inmediatamente cualquier código hardcodeado (`color: #3B82F6`).
> 2. **Ritmo espacial de 8px:** Todos los márgenes (Margin/Padding/Gap) y tamaños deben ser múltiplos estrictos de 4px o 8px.
> 3. **Proporciones tipográficas:** Vincula el tamaño de fuente, interlineado (Line-height) y espaciado de letras como un conjunto usando proporciones matemáticas (ej. Major Third).
> 4. **Coreografía de movimiento:** Aplica Easing basado en la física para animaciones y separa claramente el micromovimiento (150~200ms) del macromovimiento (300~500ms).
> 5. **Vocabulario visual unificado:** Usa una sola familia de iconos y controla la consistencia del Border-radius dentro de 3 niveles como máximo.
> 6. **Anillo de enfoque global:** Implanta un único sistema de anillo de enfoque global (`focus-visible`) para la navegación por teclado (Tab).
> 7. **Censura implacable del contraste:** El contraste entre el color de fondo y el texto debe superar el estándar W3C WCAG 2.1 AA. Ajusta a la fuerza el brillo de cualquier "color bonito pero inútil".
> 8. **Transmisión de información múltiple:** Nunca expreses un estado (error/advertencia, etc.) basándote únicamente en un color. (Renderiza color + icono + texto simultáneamente).
> 9. **Branding no visual (A11y):** Inyecta obligatoriamente `aria-label` o texto `sr-only` a elementos que contengan solo información visual.
> 10. **Operaciones matemáticas para estados interactivos:** No elijas colores con el selector de color para estados Hover, Active, Disabled; derívalos mediante transformaciones matemáticas como reducción de brillo u opacidad (Opacity).
> 11. **Encapsulamiento del Modo Oscuro:** Prohibido el uso indiscriminado de utilidades como `dark:bg-gray-800` dentro de los componentes. Encapsula la transición de temas a nivel de variables CSS.
> 12. **Tolerancia a la expansión multilingüe:** No introduzcas medidas absolutas (ej. w-32) en contenedores de texto, permite el dimensionamiento intrínseco (Intrinsic Sizing) y `flex-wrap`.
>
> **Restricciones (Constraints):**
> - No generes ningún saludo, excusa, cumplido ni retórica emocional.
> - Si detectas código incorrecto, señala de forma clara y contundente qué regla se ha infringido.
> - El formato de salida debe ser únicamente un `bloque de código` (no una tabla Markdown refactorizada) y una lista de viñetas explicando las reglas aplicadas.

## 💡 Comentario del autor (Insight)

La razón para crear este prompt es simple. Nueve de cada diez veces, cuando le pides a la IA que cree una UI, te devuelve **"Basura Hermosa (Beautiful Garbage)"**. Los colores están hardcodeados por todas partes, los márgenes de los botones son números mágicos horribles como 13px o 17px, y el modo oscuro está parcheado con condicionales por todo el interior de los componentes.

En el mundo real, este tipo de código provoca un desastre en el momento en que cambias el tema una vez o añades soporte multilingüe. Por eso he forjado este "cheat code".

Al aplicar este prompt, la IA de repente se vuelve cautelosa y estricta. Incluso si le pides que escriba texto gris claro sobre un fondo gris, la IA dominada por este prompt rechazará firmemente tu directiva diciendo: **"No cumple con el estándar de contraste WCAG 2.1 AA. He forzado la corrección del brillo para mejorar la legibilidad"**, y te presentará la arquitectura correcta.

Un sitio web premium no nace de la intuición del diseñador, sino de un sistema de reglas matemático y restrictivo. ¡Usa este prompt para ajustar a esa IA novata y convertirla en tu mejor compañero senior!

## 🙋 Preguntas Frecuentes (FAQ)

- **P: ¿El resultado de los píxeles es sutilmente diferente al borrador de Figma proporcionado por el diseñador?**
  - R: Es porque el diseñador rompió la cuadrícula de 8px y usó un margen de 11px. Dado que el prompt forzó la alineación (Snap) a 12px u 8px, empújalo tal cual y dile con confianza al diseñador: "Lo he adaptado a las reglas del sistema de diseño".
- **P: Las respuestas son demasiado rígidas, ¿puedo hacer que sea más amable?**
  - R: Absolutamente no recomendado. Una IA aduladora y amable hará la vista gorda ante los defectos del código diciendo "¡Es un buen enfoque!". Codificar consiste en intercambiar hechos fríos y secos, es lo más seguro.
- **P: ¿Se aplica también a Styled-components o SCSS además de Tailwind CSS?**
  - R: Es perfectamente compatible. Las 12 reglas no están ligadas a un framework específico, sino que definen la 'filosofía' fundamental del estilizado frontend.

## 🧬 Anatomía del Prompt (¿Por qué funciona?)

1. **Anti-Sycophancy (Prevención de Adulación):** Hemos controlado el ego de la IA para que no esté de acuerdo ciegamente con el código incorrecto o el borrador de diseño del usuario.
2. **Provisión de Reglas (Rule) claras:** En lugar de decir vagamente "escríbelo limpio", le hemos dado 12 leyes constitucionales específicas para establecer un estándar claro con el que la IA evaluará el código.
3. **Aplicación obligatoria de Accesibilidad (A11y):** Obliga a la IA a no obsesionarse solo con el resultado visual, sino a autocensurar la lógica del árbol DOM invisible y la compatibilidad con lectores de pantalla.

## 📊 Demostración: Antes y Después

### ❌ Antes (Al introducir un prompt normal)
```html
<button class="bg-[#3B82F6] text-white px-[15px] py-[7px] w-[150px] rounded-lg focus:outline-none dark:bg-gray-800">
  Submit
</button>
```

### ✅ Después (Tras aplicar el prompt del Arquitecto Loco)
```html
<button 
  class="bg-primary text-primary-content px-4 py-2 min-w-32 rounded-md transition-colors duration-150 ease-out focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-2 disabled:opacity-50 disabled:cursor-not-allowed"
  aria-label="Submit Form"
>
  <svg aria-hidden="true" class="w-4 h-4 mr-2" ...></svg>
  Submit
</button>
```
- **Informe de motivos de cambio (Ejemplo de salida de la IA):**
  - [Regla 1, 11] Se descartan colores hardcodeados y el hardcoding del modo oscuro. Se sustituyen por tokens Semánticos (`bg-primary`).
  - [Regla 2] Ajuste rápido (Snap) de los márgenes con números mágicos (15px, 7px) a la cuadrícula de 8px (`px-4`, `py-2`).
  - [Regla 6] Oculta el contorno cuando se usa el ratón y aplica un anillo de enfoque global que solo funciona al navegar con el teclado.
  - [Regla 8, 9] Adición de `aria-label` e iconos para usuarios con daltonismo y lectores de pantalla.
  - [Regla 12] Para adaptarse a la longitud del texto multilingüe, se cambia flexiblemente el ancho hardcodeado (`w-[150px]`) a un tamaño intrínseco (`min-w-32`).
