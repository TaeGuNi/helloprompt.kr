## 📊 証明：痛快な結果（Before & After）

実際の現場でボトルネックとなっているPandasパイプラインコードを**Gemini 3 Pro**に投入した際に測定した、実際のベンチマーク結果です。単純なエラー修正を超えて、どのようにアーキテクチャ自体が最適化されるかを確認してください。

### ❌ Before（私たちが経験していた苦痛）

数十万件のデータを扱うパイプラインで`iterrows`を使用する、典型的な遅いレガシーコードです。$O(n^2)$の退屈なループ演算のせいで、システム全体の足を引っ張っていました。

```python
import pandas as pd
import numpy as np

# 数十万件のデータをiterrowsで巡回 (非常に遅い)
def calculate_discount_legacy(df):
    results = []
    for index, row in df.iterrows():
        if row['member_grade'] == 'VIP':
            results.append(row['price'] * 0.8)
        elif row['member_grade'] == 'GOLD':
            results.append(row['price'] * 0.9)
        else:
            results.append(row['price'])
    df['final_price'] = results
    return df
```

### ✅ After（完璧に変身した結果）

![Gemini 3 Pro：実戦コーディングベンチマーク](/images/hooks/gemini-3-pro-review.jpg)

Gemini 3 Proはループを完全に排除し、Cレベルの速度で動作する**条件付きベクトル化（Vectorization）演算**を提案しました。驚くべきことに、実行時間は4.2秒から0.08秒へと、**実に52倍向上**しました。

```python
import pandas as pd
import numpy as np

# numpy.selectを活用した条件付きベクトル化演算 (Cレベルの速度)
def calculate_discount_optimized(df):
    conditions = [
        (df['member_grade'] == 'VIP'),
        (df['member_grade'] == 'GOLD')
    ]
    choices = [df['price'] * 0.8, df['price'] * 0.9]

    # ループなしでメモリブロック単位で一度に演算処理
    df['final_price'] = np.select(conditions, choices, default=df['price'])
    return df
```
