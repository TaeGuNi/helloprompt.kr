**[Root Cause Analysis]**
It happened because you tried to render the screen before receiving data from the API. The variable `userList` is in an `undefined` state because data hasn't arrived yet, but you used the `.map()` function on it, causing an error. It's an asynchronous processing issue.

**[Solution]**

```text
// Before fix: userList.map(...)
// After fix: Use optional chaining (?.)
userList?.map(user => ...)
```

I attached the `?.` operator to execute map only when data exists.

**[Prevention]**
When writing API communication code, it is good to always assume that 'data might arrive late' or 'it might fail' and insert code like `if (!data) return loading...` first.
