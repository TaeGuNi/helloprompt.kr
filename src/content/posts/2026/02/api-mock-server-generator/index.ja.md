# 📝 バックエンドを待つ必要なし！AIモックサーバー生成器 (API Mock Gen)

- **🎯 おすすめの対象:** フロントエンドエンジニア、フルスタックエンジニア、プロダクトマネージャー
- **⏱️ 所要時間:** 1日 → 1分に短縮
- **🤖 おすすめのモデル:** ChatGPT (GPT-4o), Claude 3.5 Sonnet, Gemini 2.5 Flash

- ⭐ **難易度:** ⭐⭐☆☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「企画も決まり画面も実装したのに、データを取得するAPIが未完成で、ダミーデータをハードコーディングしていませんか？」_

フロントエンドエンジニアの永遠の悩み、「バックエンドAPIはまだですか？」🐢
APIが完成するまで待ち、仮のコードを書き、後で実際のAPIと連携する際に再びコードを修正するという非効率な反復作業は、もう終わりにしましょう。API仕様書（Swagger、Notionのドキュメント、テキストの説明など）さえあれば、AIがわずか1分で完璧に動作するモックサーバー（Mock Server）を作成してくれます。

---

## ⚡️ 3行まとめ (TL;DR)

1. **ボトルネックの解消:** バックエンドの開発完了を待たずに、フロントエンドのロジックとUIを完璧に実装できます。
2. **本番さながらのテスト環境:** ネットワーク遅延（Delay）やエラーステータスコードをシミュレーションし、ローディングスピナーやエラー処理のUIを検証します。
3. **コピペで即実行:** 複雑な設定は一切不要。`node server.js`の1行ですぐに実行できるコードを取得できます。

### 🥇 Pro Version (専門家型)

ランダムな遅延、エラーケース、大量のダミーデータなど、より本番に近いディテールが必要な場合に使用してください。

> **役割 (Role):** あなたは`[シニアバックエンドエンジニア]`であり、`[テスト環境構築の専門家]`です。
>
> **状況 (Context):**
>
> - 背景: `[バックエンドAPIの開発が遅れており、フロントエンドのロジックテストがストップしている状況]`
> - 目標: `[API仕様書をもとに、実際の挙動に近い完璧なモックサーバーを構築すること]`
>
> **タスク (Task):**
>
> 以下のAPI仕様書をもとに、実行可能なモックサーバーの全体コードを作成してください。
>
> **技術スタック:**
>
> - 言語: `[TypeScript / JavaScript]`
> - フレームワーク: `[Express.js / Hono / Next.js Route Handler などから選択]`
>
> **API仕様書:**
>
> 1. GET /users: ユーザー一覧を返す (ページネーション対応、id, name, email フィールドを含む)
> 2. GET /users/:id: 特定ユーザーの詳細情報
> 3. POST /users: 新規ユーザー登録 (バリデーション失敗時の 400 エラーレスポンス例を含む)
> 4. DELETE /users/:id: ユーザー削除 (権限なしの 403 エラーレスポンス例を含む)
>
> **必須要件:**
>
> 1. **ダミーデータ:** `faker.js`などを使用するか、現実的なハードコーディングデータを10〜20件程度生成してください。
> 2. **遅延時間のシミュレーション:** 実際のネットワークのように500ms〜1500ms程度のランダムな遅延を追加してください（ローディングスピナーのテスト用）。
> 3. **エラーケースの注入:** 約5〜10%の確率で 500 Internal Server Error を返すようにロジックを構成してください（エラー処理UIのテスト用）。
> 4. **コード作成:** コピーしてすぐに実行（`node server.js` または `ts-node server.ts`）できるように、ポート3000で動作する1つのファイルにまとめてください。
>
> **制約事項 (Constraints):**
>
> - 出力形式はマークダウンのコードブロックで囲んで提供してください。
> - 追加のパッケージインストールが必要な場合は、コマンド（`npm install ...`）をコード上部のコメントに明記してください。
>
> **注意事項 (Warning):**
>
> - フロントエンドエンジニアがサーバー設定で悩まないよう、CORS設定（`cors`パッケージまたはヘッダー）を必ず含めてください。

## 💡 筆者コメント (Insight)

このプロンプトを使用する際の最大のコツは、**「ランダム遅延 (Random Delay)」**と**「エラーケース (Random Error)」**を必ず含めるようAIに要求することです。
ローカル環境のモックサーバーは応答速度が0.01秒単位とあまりにも速すぎるため、気をつけないとスケルトンUI (Skeleton UI) やローディングスピナーのレンダリング状態を見落としてしまいます。また、5%の確率で500エラーを発生させるよう設定しておけば、エッジケースでのエラー用トースト (Toast) ポップアップやフォールバックUI (Fallback UI) が正常に機能するかどうかを、事前に完璧にテストできます。

バックエンドエンジニアから「APIできました！」と連絡が来たときに、「はい、連携とエラーテストはすでに完了しています」とクールに返答する快感をぜひ味わってみてください。😎

## 🙋 よくある質問 (FAQ)

- **Q: SwaggerのJSONファイルがありますが、これでも作成できますか？**
  - A: はい！プロンプトの「API仕様書」部分にSwagger (OpenAPI) のJSONをテキストとしてそのまま貼り付ければ、より正確な型とレスポンススキーマを持つモックサーバーを作成してくれます。

- **Q: POSTやPUTで追加・修正したデータは保持されますか？**
  - A: 基本的にはメモリ（配列）上に保存されるため、サーバーを再起動すると初期化されます。もしデータの永続化が必要な場合は、プロンプトに*「lowdbやjson-serverを使用してローカルファイル (db.json) にデータを保存して」*と追加でリクエストしてみてください。

- **Q: Next.jsを使っていますが、Express以外でも可能ですか？**
  - A: もちろんです！「技術スタック」部分に`Next.js API Routes (App RouterのRoute Handler)`と指定すれば、Next.jsプロジェクト内にそのまま組み込んで使えるコードを出力してくれます。

## 🧬 プロンプトの解剖 (Why it works?)

1. **Role（役割）とContext（状況）の付与:** AIに単なるコーダーではなく「テスト環境構築の専門家」というペルソナを与えることで、フロントエンドエンジニアが実際に直面する不便さ（CORSエラー、ローディング遅延の欠如など）を先回りして解決させます。
2. **現実的な制約条件の追加:** 無意味な「test1」「test2」といったデータではなく、`faker.js`や現実的なダミーデータを要求することで、UIの視覚的な完成度を本番プロダクトと同等レベルで確認できます。
3. **エッジケースのシミュレーション:** エラー応答の確率やランダムなネットワーク遅延時間を明示的に要求し、単なる成功（200 OK）ケースだけでなく、エラーハンドリングまで強制的に検証できる環境を構築しました。

```javascript
]);

// ローディング状態やエラーステータスをテストするには、コードを都度コメントアウトして手動で書き換える必要がある
```

### ✅ After (結果: AIが作成したモックサーバー)

```javascript
// npm install express cors @faker-js/faker
const express = require("express");
const cors = require("cors");
const { faker } = require("@faker-js/faker");

const app = express();
app.use(cors());
app.use(express.json());

// 現実的なダミーデータを15件自動生成
let users = Array.from({ length: 15 }).map(() => ({
  id: faker.string.uuid(),
  name: faker.person.fullName(),
  email: faker.internet.email(),
  createdAt: faker.date.recent(),
}));

// 本番想定: ランダム遅延ミドルウェア (500ms 〜 1500ms の遅延)
const randomDelay = (req, res, next) => {
  const delay = Math.floor(Math.random() * 1000) + 500;
  setTimeout(next, delay);
};

// 本番想定: ランダムエラー発生ミドルウェア (5%の確率で 500 エラー)
const randomError = (req, res, next) => {
  if (Math.random() < 0.05) {
    return res.status(500).json({ error: "Internal Server Error Simulation" });
  }
  next();
};

app.use(randomDelay);
app.use(randomError);

app.get("/users", (req, res) => {
  res.json({ data: users, total: users.length });
});

app.listen(3000, () => {
  console.log("🚀 Mock Server is running on http://localhost:3000");
});
```
