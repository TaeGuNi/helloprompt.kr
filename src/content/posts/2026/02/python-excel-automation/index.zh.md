## 🙋 常见问题解答 (FAQ)

- **Q：如果这些 Excel 文件都加了不同的密码怎么办？**
  - A：你可以借助 Python 的 `msoffcrypto-tool` 库来解锁并读取受密码保护的文件。你可以直接在提示词里对 AI 说：“_这些文件可能被加密了，可能的密码列表是 ['1234', '5678']。请在代码中加入密码穷举解密逻辑，解锁后再进行汇总。_”

- **Q：如果需要汇总的各个 Excel 文件，其列的顺序或者表头名字有些许差异，代码会报错吗？**
  - A：是的，如果结构不一致，简单的合并会导致数据错位。这时候你需要给 AI 更具体的预处理指令。比如：“_A 门店的表格数据是从第 3 行开始的，而 B 门店的表头名字有点不一样。请在代码中加入逻辑：自动寻找包含‘营收’二字的列，并将它们统一映射为一个标准列名再进行合并。_”

- **Q：几十万行甚至上百万行的超大 Excel 文件也能搞定吗？**
  - A：Excel 软件本身在处理超过 100 万行的数据时往往会卡死甚至崩溃，但 Python 的 `pandas` 库处理几百万行数据不过是眨眼间的事（只要你的电脑内存足够）。如果你遇到了内存溢出（OOM）报错，只需对 AI 说：“_请优化代码，不要一次性将所有数据读入内存，而是采用 chunk（分块）的方式分批读取并追加保存。_”

## 🧬 提示词原理解析 (Why it works?)

1.  **明确限定文件名规则（如 `store_*.xlsx`）：** 通过在提示词中指定文件名的正则模式，可以精准引导 AI 使用 `glob` 或 `pathlib` 模块进行筛选。这能有效防止代码“误伤”文件夹里混杂的其他无关 Excel 文件或图片，避免汇总出一堆垃圾数据。
2.  **提前规划数据类型转换（预处理要求）：** 在 Excel 里看起来是数字，但在 Python 读取时经常会被识别为带有格式的字符串（String），导致求和计算时疯狂报错。在提示词阶段就明确要求剥离逗号和货币符号并转换为整型（Integer），可以帮你省去 90% 的 Debug 时间。
3.  **强制加入异常处理 (`try-except`)：** 在处理几十上百个文件时，墨菲定律一定会生效——总有一两个文件损坏或者格式完全乱套。通过在约束条件中强制要求加入异常处理，可以确保脚本不会半路夭折，而是默默跳过“坏苹果”并跑完全程，大大提升了自动化机器人的鲁棒性（Robustness）。
