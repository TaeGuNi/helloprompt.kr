## 🙋 よくある質問 (FAQ)

- **Q: WebGPUをサポートしていない古いブラウザでは全く動作しませんか？**
  - A: はい、WebGPUのハードウェアアクセラレーションが必須です。実務では、プロンプトの指示にあるように、まずWebGPUのサポート状況をチェックし、未対応デバイスの場合はクラウドAPI（例: OpenAI）へ迂回するハイブリッドアーキテクチャを設計するのが定石です。

- **Q: モデルのダウンロード容量が大きすぎて負担になりませんか？**
  - A: Llama-3 8Bの4ビット量子化モデルは、およそ4〜5GB程度です。初回アクセス時にWi-Fi接続を推奨する親切な案内を表示し、ダウンロードの進捗状況を視覚的に美しく見せることで、ユーザーの離脱を防ぐUX設計が重要になります。

- **Q: 回答の生成速度（TPS）は実用的なレベルですか？**
  - A: Apple M2/M3チップセットを搭載したMacや、最新の外部グラフィックカードを搭載した環境では、毎秒30〜50トークン以上の驚くべき速度を発揮します。クラウドAPIに引けを取らない快適さを体感できます。

```tsx
// useWebLLM.ts (プロンプトが生成したプロダクションレベルのコードの一部)
import { useState, useEffect } from "react";
import { CreateMLCEngine } from "@mlc-ai/web-llm";

export const useWebLLM = (modelId = "Llama-3-8B-Instruct-q4f32_1") => {
  const [progress, setProgress] = useState(0);
  const [engine, setEngine] = useState<any>(null);
  const [isReady, setIsReady] = useState(false);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const initEngine = async () => {
      if (!navigator.gpu) {
        setError("現在のブラウザはWebGPUをサポートしていません。");
        return;
      }
      try {
        const mlcEngine = await CreateMLCEngine(modelId, {
          initProgressCallback: (report) => {
            setProgress(Math.round(report.progress * 100));
          },
        });
        setEngine(mlcEngine);
        setIsReady(true);
      } catch (err) {
        setError("モデルの読み込み中にエラーが発生しました。");
      }
    };
    initEngine();
  }, [modelId]);

  return { engine, isReady, progress, error };
};
```
