## 🚀 解决方案："Everything Claude Code" 提示词

### 🥉 Basic Version (基础版)

当你需要快速完成基础架构设计时，请使用此版本，2分钟内即可雇佣一位资深 Agent。

> **1. 安装插件 (以 Claude Code 为例):**
> `/plugin marketplace add affaan-m/everything-claude-code`
> `/plugin install everything-claude-code@everything-claude-code`
> 
> **2. 执行架构规划提示词:**
> `/plan "请使用 Next.js 14 App Router 为我搭建一个响应式的博客文章列表页。技术栈需包含 Tailwind CSS 和 shadcn/ui。"`

<br>

### 🥇 Pro Version (专业版)

当你希望在项目中强制执行资深开发者的工作流（如严格的 TDD 或特定的代码规范）时，请使用此高阶版本。

> **角色 (Role)：** 你是一位极其严格、始终恪守 TDD（测试驱动开发）原则的 `[资深 TypeScript 开发者]`。
>
> **背景 (Context)：**
>
> - 我们目前正在全面遵循 `[everything-claude-code]` 的系统规则。
> - 核心目标：我们需要实现一个高可靠性的 `[用户身份验证模块]`。
>
> **任务 (Task)：**
>
> 1. 请在终端中激活 `/tdd` 指令。
> 2. 在编写任何业务逻辑代码之前，**必须**优先编写包含各种边界与异常情况的测试用例。
> 3. 若当前代码的测试覆盖率未达到 80%，坚决不允许进入下一开发阶段。
> 4. 若有 `[额外补充需求]`，请将其完美融合到代码架构中。
>
> **约束条件 (Constraints)：**

> - 所有的变量、函数与接口都必须添加明确的 Type Hint（类型注解）。
> - 全局**严禁**使用 `any` 类型。

> **注意事项 (Warning)：**

> - 绝对不要编写任何破坏现有系统架构或违背既定约定的代码。如果在实现过程中存在任何不确定性，必须先向我提问确认。（防止幻觉与越权操作）

## 💡 作者洞察 (Insight)

将这套配置真正落地到实际业务后，最让我感到震撼的是：**“AI 开始主动拒绝懒惰的提示词了。”**

在过去，如果我们随口一句：“帮我加个登录页面”，AI 通常会立刻吐出一大堆缺乏错误处理、逻辑漏洞百出的面条代码 (Spaghetti Code)。但引入这套配置后，一切都变了。当你输入 `/plan` 指令时，AI 会反客为主，开启“夺命连环问”模式：_“请问我们采用哪种认证机制（OAuth 还是 JWT）？发生错误时的 UI 状态该如何兜底？”_ 这种感觉，就像是身边坐着一位**“没有详细需求文档就拒绝敲键盘”的严谨架构师**，随时在为你兜底。

其中最令人拍案叫绝的当属 **`/evolve`** 指令。随着项目代码量的增加，AI 会像人类一样主动观察并总结团队特有的编码习惯（例如：“我们在定义接口时从不加 'I' 前缀”），并将其固化为永久规则。对于那些苦于代码风格不统一的团队，或是极度渴望高级代码审查的独立开发者而言，这套工作流绝对是降维打击。

## 🙋 常见问题 (FAQ)

- **Q: 这套配置只能在 Claude Code CLI 环境下使用吗？**
  - A: 完全不是！它非常灵活，你可以将其无缝集成到 Cursor 编辑器的 `.cursorrules` 文件中，甚至在 OpenCode 环境下也能完美兼容。

- **Q: 把它用于公司内部的商业保密项目安全吗？**
  - A: 规则 (Rule) 文件本身是开源的，并且仅保存在你的本地环境，因此配置本身是安全的。但请注意，向 LLM 模型发送代码的动作本身需要遵守你们公司的安全合规要求（例如确认是否使用了企业级 Enterprise 隐私套餐）。

- **Q: AI 老是问问题，还强制要求写测试，对于新手来说是不是太繁琐了？**
  - A: 刚开始确实会有阵痛期，你会觉得被 AI 约束住了手脚。但请相信我，正是这种“繁琐”的前期流程，才能让你彻底告别日后为了偿还技术债务 (Technical Debt) 而连续熬夜重构的惨痛经历。

## 🧬 提示词解剖 (Why it works?)

1. **强制工作流约束 (Forced Workflow)：** 摒弃了毫无章法的自由对话，通过引入 `/plan`, `/tdd` 等斜杠指令，强制引导 AI 遵循规范的软件工程生命周期（需求拆解 → 测试先行 → 核心代码实现）。
2. **多重人格博弈 (Persona Separation)：** 巧妙地将“编码者 (Developer)”、“审查者 (Reviewer)”和“规划者 (Planner)”的角色剥离开来，使得 AI 在内部就能完成逻辑的交叉验证，极大提升了代码的健壮性。
3. **沉淀与自我进化 (Instincts)：** 打破了传统聊天机器人“阅后即焚”的上下文失忆症。AI 在协助开发的过程中，能敏锐捕捉使用者的重构意图，并将其沉淀至 `.rules` 规则库中，真正做到越用越懂你。
