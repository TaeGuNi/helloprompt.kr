## 🚀 Решение: Генератор 2D-игр от Claude Code

### 🥉 Базовая версия (Basic Version)

Используйте этот промпт для быстрого создания каркаса проекта и тестирования фундаментальных механик.

> **Роль:** Ты — Senior Game Client Developer (Ведущий разработчик клиентских игр).
> 
> **Задача:** Создай браузерную 2D-игру в гольф, используя HTML5 Canvas. Игра должна включать мяч, лунку (Hole) и механику удара с помощью мыши (drag-to-shoot).

### 🥇 Продвинутая версия (Pro Version)

Этот вариант идеально подходит для создания проекта, близкого к production-ready: от проработки физического движка до модульности и дизайна уровней.

> **Роль (Role):** Ты — Senior Web Game Developer с 10-летним опытом и мой напарник по парному ИИ-программированию.
>
> **Контекст (Context):**
>
> - Ситуация: Я хочу разработать легковесную 2D мини-игру, используя исключительно HTML5 Canvas и Vanilla JavaScript, без привлечения тяжелых движков или сторонних библиотек.
> - Цель: Создать 2D-игру в гольф, где игрок должен забить мяч в лунку (hole-in-one), интуитивно регулируя траекторию и силу выстрела с помощью перетаскивания (drag-and-drop).
>
> **Задача (Task):**
>
> 1. Полностью модулируй архитектуру проекта, разделив его на `index.html` (контейнер Canvas), `game.js` (игровой цикл и управление состояниями) и `physics.js` (векторная математика и физика).
> 2. С высокой точностью реализуй базовый физический движок: добавь `[силу трения (симуляция сопротивления травы)]` и `[отскок при столкновении со стенами]` на границах холста.
> 3. Внедри визуальный индикатор (направляющую линию), который в реальном времени показывает силу и вектор будущего выстрела при натяжении.
> 4. Настрой `[массив конфигурации уровней]`, состоящий минимум из 3 этапов с постепенно возрастающей сложностью за счет добавления препятствий.
>
> **Ограничения (Constraints):**
>
> - Сгенерированный код должен быть полностью готов к запуску. Не допускай сокращений и пропусков (никаких "остальной код здесь").
> - Оставь подробные комментарии к ключевым логическим узлам (затухание скорости, обработчики событий и т.д.) для упрощения будущих доработок.
>
> **Предупреждение (Warning):**
>
> - Вместо излишне сложной и тяжелой симуляции реальной физики, отдай приоритет аркадному, интуитивно понятному стилю управления.

## 💡 Комментарий автора (Insight)

Главный секрет успешного кодинга с искусственным интеллектом — **«воспринимать ИИ не как продвинутый автокомплит, а как Senior-партнера, с которым вы обсуждаете архитектуру и проектируете систему»**.
Успех кроется не в попытке сгенерировать идеальную игру-блокбастер за один запрос, а в **поэтапном наращивании функционала (Step-by-step)**: сначала базовая физика (трение, столкновения), затем обработка ввода (визуальный отклик drag & shoot) и лишь потом — расширение контента (дизайн уровней).

Claude Code обладает феноменальной способностью удерживать контекст и проводить безопасный рефакторинг. Если в процессе работы вы попросите: «Вынеси логику управления препятствиями в отдельный класс для рефакторинга», вы получите чистый, легко масштабируемый и поддерживаемый код.

## 🙋 Часто задаваемые вопросы (FAQ)

- **В: Может ли новичок, ничего не смыслящий в геймдеве и векторной математике, создать игру с этим промптом?**
  - О: Абсолютно! Промпт берет на себя всю работу по проектированию архитектуры и сложным математическим расчетам. Вам достаточно скопировать предоставленный код, разложить его по нужным файлам и просто открыть в браузере.

- **В: Придется ли мне вручную править код, чтобы добавлять препятствия или новые карты?**
  - О: Нет, глубоко копаться в логике не придется. Достаточно попросить Claude на естественном языке: «Добавь после 3-го уровня 4-й этап с вращающейся мельницей по центру», и он, поняв структуру массива `Level`, мгновенно сгенерирует нужное обновление.

- **В: Смогу ли я получить такой же качественный результат, если буду использовать ChatGPT вместо Claude Code?**
  - О: Да, флагманские модели (например, GPT-4o) также выдают превосходный код. Однако веб-версия ChatGPT лишена функционала автоматического создания и перезаписи файлов, который есть у терминального Claude Code или Cursor. Вам придется самостоятельно копировать ответы ИИ и распределять их по файлам `game.js` и `physics.js` согласно инструкциям.

## 🧬 Разбор промпта (Why it works?)

1. **Принудительная модульность (Modularity):** Жесткое требование разделить проект на файлы с самого начала задает правильный архитектурный каркас. Это спасает от спагетти-кода на этапе масштабирования и делает поддержку проекта комфортной.
2. **Акцент на визуальном отклике (Visual Indicator):** Запрос не ограничивается базовым управлением. Явное требование отрисовать «линию вектора и силы» кардинально улучшает UX и делает игровой процесс интуитивным.
3. **Закладка на масштабирование (Level Design):** Указание использовать конфигурационный массив для уровней, а не хардкодить единственный экран, формирует data-driven подход. В будущем вы сможете добавлять новые карты бесконечно и без боли.

```text
Мяч проваливается сквозь стены и навсегда исчезает за пределами экрана. Обработчики кликов конфликтуют, заставляя шар бесконечно ускоряться при каждом нажатии. Из-за отсутствия концепции силы трения мяч катится вечно и никогда не останавливается.
```

### ✅ После (Пример физической логики, сгенерированной Продвинутой версией)

```javascript
// Сгенерировано Claude Code
update() {
  this.velocity.x *= 0.98; // Трение: Симуляция сопротивления травы для плавного замедления
  this.velocity.y *= 0.98;

  this.x += this.velocity.x;
  this.y += this.velocity.y;

  // Обработка столкновений со стенами холста (естественный отскок)
  if (this.x < 0 || this.x > canvas.width) {
    this.velocity.x *= -1; // Смена направления
  }
  if (this.y < 0 || this.y > canvas.height) {
    this.velocity.y *= -1;
  }
}
```

_Результат: Claude Code безупречно применил математическую концепцию демпфирования (затухания) для плавного гашения скорости, идеально сымитировав сопротивление реального газона на поле для гольфа._
