## 📝 Claude Codeの性能低下論争？その真相と「怠惰」を克服するプロンプト

- **🎯 おすすめの対象:** 開発者、プログラマー、AIコーディングアシスタントのユーザー
- **⏱️ 所要時間:** 1時間のデバッグ → 1分に短縮
- **🤖 おすすめのモデル:** Claude 3.5 Sonnet, Claude 3 Opus

- ⭐ **難易度:** ⭐⭐☆☆☆
- ⚡️ **効果性:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「Claude Codeが最近、コードの大部分を省略して『// 残りのコードはここに入ります』とだけ出力するようになり、イライラしていませんか？」_

最近、RedditやX（旧Twitter）などの開発者コミュニティにおいて、「Claude Codeのパフォーマンスが落ちた」「以前よりも怠惰（Lazy）になった」という不満が頻発しています。とりわけ、長いコードを生成する際に全容を記述せず、一部を省略（プレースホルダー化）してしまう現象が、その最大の要因として指摘されています。

しかし、これはモデル自体の推論能力が低下したというよりも、**出力トークンの制限や、効率化を目的とした安全装置（Safety Alignment）が過敏に機能している結果**である可能性が高いと言えます。本記事では、この「怠惰なAI」の目を覚まさせ、最初から最後まで完璧なコードを出力させるための、具体的なプロンプトエンジニアリングの手法を解説します。

---

## ⚡️ 3行まとめ (TL;DR)

1. Claude Codeの性能低下論争の核心は、コードの「省略（プレースホルダー化）」と「怠惰さ」にある。
2. モデルの推論能力そのものが低下したわけではなく、効率性を重視したデフォルト設定が主な原因。
3. 適切な「役割の付与」と「出力に対する制約」をプロンプトに組み込むことで、この問題は100%解決可能。

## 🚀 解決策: "フルコード強制（No-Placeholder）プロンプト"

### 🥉 Basic Version (基本型)

迅速に完全なコードだけを取得したい場合に活用してください。

> **役割:** あなたはシニアソフトウェアエンジニアです。
> **要求:** 以下の`[要件]`を満たすコードを作成してください。コードを記述する際は、いかなる省略やプレースホルダー（例：`// 既存のコード`）も使用せず、そのまま実行可能な完全なコード（Full Code）を最初から最後まで出力してください。

### 🥇 Pro Version (専門家型)

大規模なリファクタリングや複雑なロジックを実装する際、ディテールと品質を維持したい場合に活用してください。

> **役割 (Role):** あなたはシリコンバレーで活躍するトップクラスのシニア開発者であり、一切の妥協を許さない完璧主義者です。
>
> **状況 (Context):**
>
> - 背景: 現在、既存のコードベースを改善し、新機能を統合する作業を行っています。AIがコードの一部を省略すると、コンパイルエラーに直結し、生産性が著しく低下してしまいます。
> - 目標: 省略された箇所が一切ない、コピーしてそのまま本番環境へデプロイ可能な「完全なコードスニペット」を取得すること。
>
> **要求 (Task):**
>
> 1. `[実装する機能や修正点]`に関して、バグのないクリーンなコードを作成してください。
> 2. すべての関数、クラス、メソッドの内部ロジックを最後まで完全に実装してください。
> 3. `[変数]` 部分はユーザーの入力を待つプレースホルダーとして残さず、具体的な要件に合わせて実装してください。
>
> **制約事項 (Constraints):**
>
> - プレースホルダー（`// ここにコードを追加`, `...` など）の使用を**厳格に禁止**します。
> - 既存のコードを修正する場合でも、変更されていない部分を含めてファイル全体のコードを出力してください。
> - 出力形式には必ずMarkdownのコードブロックを使用し、プログラミング言語を明示してください。
>
> **注意事項 (Warning):**
>
> - 出力トークンの上限に達した場合は、コードが途中で途切れてしまっても構いません。私が「続けて」と入力したら、途切れた部分から正確に再開してください。決して怠惰になってコードを要約してはいけません。（不確実なコードを生成するハルシネーションも防止すること）

## 💡 作成者コメント (Insight)

AIコーディングエージェントを実務に導入した際、最も時間を無駄にしてしまうのは「AIが生成した不完全なコードを、人間が手動でつなぎ合わせる瞬間」です。Anthropicのモデルは非常に優秀ですが、サーバーの負荷を抑えるために、意図的に回答を短く済ませようとする傾向が見受けられます。

上記のPro Versionプロンプトを、IDEのカスタムプロンプト（CursorのRules機能やGitHub CopilotのCustom Instructionsなど）にデフォルト設定として組み込んでおけば、毎回「最後まで全部書いて」と指示する手間を省くことができます。実際にこの設定を適用して以来、コードの統合にかかる時間が劇的に短縮されました。AIを単なる「アシスタント」ではなく、即戦力となる「完成されたワーカー」として扱うための必須テクニックです。

## 🙋 よくある質問 (FAQ)

- **Q: このプロンプトを使うと、APIのコスト（トークン使用量）が増加しませんか？**
  - A: はい、コード全体を出力するため出力トークン（Output Tokens）が増加し、コストはわずかに上昇します。しかし、不完全なコードを手直しするための人間の人件費や時間を考慮すれば、圧倒的に高い費用対効果が得られます。

- **Q: ChatGPT（GPT-4o）でもこのプロンプトは有効ですか？**
  - A: もちろんです。GPT-4oでも同様に、コードを省略しようとする「怠惰現象」が見られることがあります。このプロンプトは、あらゆるLLMに対して「完全な出力」を強制する普遍的な効果を発揮します。

- **Q: 長いコードを出力している途中で生成が止まってしまったらどうすればいいですか？**
  - A: 「止まった部分の最後の行から、新しくコードブロックを開いて続きを書いてください」と入力すれば、シームレスに後続のコードを取得することができます。

## 🧬 プロンプト解剖 (Why it works?)

1. **Roleの付与:** AIに「完璧主義のシニア開発者」というペルソナを与えることで、出力されるコードの専門性と品質を底上げしています。
2. **Context（状況）の共有:** 「なぜコードを省略されると困るのか（コンパイルエラーの発生、生産性の低下）」を論理的に説明し、AIが指示に従うための動機付けを強化しています。
3. **Constraints（制約）の徹底:** 単に「全部書いて」と指示するのではなく、AIが多用しがちなパターン（例：`// ここにコードを追加`）を具体的に挙げて禁止（Negative Prompting）し、AIの逃げ道を塞いでいます。
4. **Warning（注意事項）による防御:** 出力トークン制限に対するAIの「恐れ（要約して短く終わらせようとする傾向）」を取り除き、「途切れてもいいから詳細に書き切る」という明確な許可を与えています。

### ✅ After (Pro Version プロンプト適用後)

```javascript
// 最初から最後まで実装された完全なコード
function calculateComplexROI(data) {
  const parsedData = parseData(data);
  let totalRevenue = 0;
  let totalCost = 0;

  for (const item of parsedData) {
    if (item.type === "revenue") {
      totalRevenue += item.amount * item.multiplier;
    } else if (item.type === "cost") {
      totalCost += item.amount;
    }
  }

  const roi = ((totalRevenue - totalCost) / totalCost) * 100;
  const result = {
    netProfit: totalRevenue - totalCost,
    roiPercentage: parseFloat(roi.toFixed(2)),
    isProfitable: roi > 0,
  };

  return result;
}
```

## 🎯 結論

Claude Codeや最新のAIモデルにおける「性能低下」の多くは、AI自身の能力不足ではなく、**私たちが入力するプロンプトが十分に厳格ではないこと**に起因しています。

AIに対して期待する成果の基準（Standard）を明確に定義し、妥協のない指示を与えることで、再び最高水準のコーディングパートナーとして活躍させることができます。今すぐこのプロンプトをご自身の開発ツールに組み込み、日々の業務を効率化して退社時間を早めましょう！ 🍷
