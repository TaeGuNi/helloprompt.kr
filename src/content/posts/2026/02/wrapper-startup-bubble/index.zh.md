# 📝 AI Wrapper 创业公司诊断及护城河 (Moat) 构建提示词

- **🎯 推荐对象：** 准创业者、初创公司 CEO、产品经理 (PO)、风险投资 (VC) 审查员
- **⏱️ 所需时间：** 商业模式诊断 1 小时 → 缩短至 3 分钟
- **🤖 推荐模型：** 所有对话型 AI (ChatGPT, Claude, Gemini 等)

- ⭐ **难度:** ⭐⭐⭐☆☆
- ⚡️ **效果:** ⭐⭐⭐⭐⭐
- 🚀 **实用性:** ⭐⭐⭐⭐☆

> _“如果明天 OpenAI 免费推出了和我们一模一样的功能，我们的产品会立刻倒闭吗？”_

最近 AI 创业热潮席卷全球，但仔细观察就会发现，许多公司仅仅是将 OpenAI 或 Anthropic 的 API 进行简单包装的“套壳 (Wrapper)”服务。一旦科技巨头 (Big Tech) 将这些功能内置，它们将瞬间失去竞争力，面临破产危机。缺乏真正的创新和可防御的“技术护城河 (Moat)”，仅靠营销驱动的商业模式注定无法长久。

今天为大家介绍一款“直击痛点”的诊断提示词。它将帮你检验你的商业模式是否只是虚有其表，并协助你挖掘出在巨头围剿下依然能生存的独特价值主张 (UVP) 和技术护城河。

---

## ⚡️ 三句话总结 (TL;DR)

1. 仅依赖 API 调用的“套壳 (Wrapper)”模式，极易在科技巨头的一次功能更新中土崩瓦解。
2. 本提示词旨在深度剖析当前商业模式的脆弱性，并为你诊断可防御的技术或数据护城河。
3. 通过获取客观且犀利的反馈，你可以跳出同质化 AI 服务的泥潭，获得向不可替代产品转型 (Pivot) 的灵感。

---

## 🚀 解决方案：“初创公司护城河 (Moat) 鉴定器”

### 🥉 Basic Version (基础版)

当你需要快速诊断当前项目的风险程度时使用。

> **角色：** 你是硅谷一位冷酷且极其敏锐的一线风险投资 (VC) 审查员。
> **任务：** 请诊断我正在策划的 AI 服务 `[服务创意/产品理念]` 是否只是一个简单的 API 套壳产品 (Wrapper)。如果 OpenAI 亲自下场推出这项功能，请为我提供 3 个能让该服务存活下来的防御策略。

<br>

### 🥇 Pro Version (专家版)

当你需要深度分析商业模式 (BM)，并制定具体的护城河构建策略时使用。

> **角色 (Role)：** 你是 Y Combinator 合伙人级别的 AI 商业战略家，也是一位眼光毒辣的风险投资人。
>
> **背景 (Context)：**
>
> - 现状：我们目前正在运营（或策划）一款面向 `[目标客户群体]` 的服务，核心功能是 `[核心功能及服务说明]`。
> - 底层技术：主要调用 `[正在使用的 AI 模型，例如：GPT-4o, Claude 3.5 Sonnet]` 的 API。
> - 目标：希望突破单纯“套壳 (Wrapper)”的局限，构建属于我们自己独一无二的技术或数据护城河 (Moat)。
>
> **任务 (Task)：**
>
> 1. **生存概率诊断：** 当科技巨头 (如 Google, OpenAI 等) 推出原生替代功能时，请冷酷地评估我们商业模式将受到的冲击，并以百分比 (%) 形式给出存活概率。
> 2. **脆弱性剖析：** 请一针见血地指出我们的服务可能被视为简单套壳的 3 个致命弱点。
> 3. **护城河构建策略：** 请从以下三个维度，为我们提供切实可行的防御策略建议：
>    - 数据护城河（能够独家收集/处理的专有数据）
>    - 工作流深度整合（如何无缝且深度地嵌入客户现有的业务流程）
>    - 网络效应或社区生态
> 4. 你的回答必须完全结合并深挖 `[核心功能及服务说明]` 的具体业务场景。
>
> **约束条件 (Constraints)：**
>
> - 为保证移动端阅读体验，输出格式必须严格使用 Markdown 列表 (Bullet Point)。禁止使用表格。
> - 严禁说任何客套话或无意义的赞美，必须优先提供直击要害、极其现实且冷酷的反馈。
>
> **警告 (Warning)：**
>
> - 切勿提出不切实际的技术幻想，或初创公司当前根本无力承担的烧钱策略（例如：从零开始训练底座模型）。

## 💡 作者洞察 (Writer's Insight)

这个提示词是我亲自评估无数 AI 产品企划书时，用来“一针见血”指出致命伤的专属诊断器。
回顾 GPT API 刚发布时，市面上如雨后春笋般涌现了无数 PDF 总结工具。但随着 ChatGPT 原生支持了文档分析功能，这些产品大多瞬间销声匿迹。你要明白：**AI 是一个出色的“引擎”，但你的产品绝不能仅仅只是这个引擎本身。**

在运行这个提示词后，请特别留意 AI 给出的关于 **“工作流深度整合 (Workflow Integration)”** 的建议。你的产品能否与用户每天都在用的 Slack、Jira，甚至是企业内部的遗留系统 (Legacy System) 紧密绑定？这种转换成本才是早期 AI 创业公司所能拥有的最强大、最现实的护城河。

## 🙋 常见问题 (FAQ)

- **Q：AI 给出的评价太负面了，看完了很受打击怎么办？**
  - A：这是因为我们在提示词的约束条件中强制要求了“冷酷的反馈”。但请相信我，在企划阶段被 AI “毒打”一顿，总好过产品上线后被真实市场无情抛弃。深呼吸，然后针对诊断结果继续追问 AI：“基于以上弱点，请为我提供 3 个可以弥补缺陷的转型 (Pivot) 灵感”。

- **Q：我的产品是面向普通用户的 B2C App，不是 B2B SaaS，也能用这个提示词吗？**
  - A：完全可以。如果是 B2C 产品，你只需在“背景 (Context)”部分，额外补充你们目前掌握的“获客渠道”或已有的“社区基础”。AI 模型会据此为你推演出非常精彩的、以“网络效应”为核心的护城河策略。

## 🧬 提示词解剖 (Why it works?)

1. **赋予顶级专家角色：** 设定“Y Combinator 合伙人”的人设，强制 AI 摒弃客套话，从真实的融资与生死存亡视角给出极其犀利的反馈。
2. **提供明确的诊断框架：** 直接指定“数据护城河、工作流整合、网络效应”这 3 个经过商业验证的评估维度，防止 AI 给出假大空的泛泛而谈。
3. **设置现实约束条件：** 预先排除了诸如“训练自有底座模型”这类初创团队根本无力执行的建议，将回答的实用性拉满，同时有效减少了 AI 的幻觉 (Hallucination)。

## 📊 效果验证：Before & After

### ❌ Before (普通提问)

```text
我们正在做一个调用 ChatGPT API 来自动生成博客文章的服务。你觉得这个主意怎么样？
```

### ✅ After (使用 Pro 版提示词的结果)

```text
[冷酷的生存概率诊断]
- 生存概率：不足 15%
- 致命原因：市面上同质化产品泛滥，且 Notion AI 或 ChatGPT 官方应用本身就能完美胜任博客撰写。仅靠套壳文本生成，巨头稍微优化一下 UI 交互，你们就会流失所有客户。

[3 个致命的套壳弱点]
1. 毫无不可替代性：用户完全没有非用你们产品不可的理由，随时可以切换到竞品或官方 API。
2. 零技术壁垒：一个计算机专业的大学生，利用周末两天就能手搓出一个一模一样的 Demo。
3. 利润空间被无限压缩：API 调用成本是刚性的，但产品缺乏差异化必然导致无底线的价格战，最终商业模式根本无法跑通。

[护城河构建策略建议]
- 数据护城河：不要做泛泛的“自动写文章”。去专注收集客户企业内部“历史高转化率文章的行文风格与受众数据”，做到 100% 完美复刻特定品牌的专属 Tone & Manner（品牌调性）。这是巨头通用模型做不到的。
- 工作流深度整合：绝对不要让功能停留在“生成文本”这一步。必须向下游延伸，开发一键自动发布至 WordPress/微信公众号的插件，并集成 SEO 数据追踪面板，直接垄断客户的内容分发生态管线。
```

## 🎯 结语

请不要畏惧从一个简单的 API “套壳产品 (Wrapper)”起步。你要知道，当今许多伟大的 AI 服务，它们最初的原型也仅仅是个 Wrapper。

真正决定生死的，是你能否以最快的速度，在这个“壳”里注入**“属于你们自己的骨骼与血液（独家数据资产与深度的用户工作流）”**。希望这款提示词能成为你的得力参谋，助你构建起一座即便面对科技巨头狂轰滥炸，依然固若金汤的专属商业城堡！ 🚀
