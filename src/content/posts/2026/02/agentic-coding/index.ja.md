# 📝 チャットはもうやめて、仕事を任せよう：Agentic AIが開発を変える方法

- **🎯 おすすめの対象:** 1〜5年目のフロントエンド/バックエンドエンジニア、テックリード、プロダクトマネージャー
- **⏱️ 所要時間:** 30分 → 3分に短縮
- **🤖 おすすめのモデル:** Cursor (Composer), Windsurf (Cascade), GitHub Copilot Workspace

- ⭐ **難易度:** ⭐⭐⭐☆☆
- ⚡️ **効果性:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「まだAIに『関数を1つ書いて』と頼んで、コピペするだけの使い方をしていませんか？」_

2025年まで、私たちはAIと「チャット」をしていました。しかし2026年の現在、開発のパラダイムは**「このイシューを解決して（Fix this issue）」**と指示を出す**Agentic Workflow（自律型AIワークフロー）**へと完全に移行しました。単なるコードの自動補完を超え、プロジェクト全体のコンテキスト（文脈）を理解し、複数のファイルを同時に修正しながらテストまで実行してくれる「仮想の同僚」が誕生したのです。

本記事では、現在最も注目を集めているAIエディタである**Cursor**と**Windsurf**を活用し、一言の指示で複雑なフルスタック機能の企画から実装、検証までを一気に終わらせる実践的なプロンプトを公開します。

## ⚡️ 3行まとめ (TL;DR)

1. **コンテキスト領域の拡張:** 最新のAIはリポジトリ全体を読み込み、プロジェクトのアーキテクチャやコーディング規約を理解します。
2. **複数ファイルの同時修正 (Multi-file Edit):** APIルート、UIコンポーネント、DBスキーマなど、関連する複数のファイルを一度に自動で修正します。
3. **ステップバイステップの指示が必須:** 「設計 → 実装 → 検証」の各フェーズを明確に分けて指示することで、AIのハルシネーション（幻覚）やミスを防ぎます。

AIエディタ（Cursorの `Cmd+I` Composerモード、またはWindsurfのCascadeパネル）を開き、以下のプロンプトをそのまま貼り付けてください。

### 🥉 Basic Version (基本型)

単一機能の骨組みだけを素早く作成したい場合に使用してください。

> **役割:** あなたは `[Next.js]` 専門のシニアエンジニアです。
> **タスク:** 現在のプロジェクトに `[ユーザーフィードバック収集機能]` を追加してください。必要なUIコンポーネントとAPIルートをすべて自動生成し、接続してください。

<br>

### 🥇 Pro Version (専門家型)

設計から実装、テストまで、プロダクションレベルの完璧なコードが必要な場合に使用してください。

> **役割 (Role):**
> あなたは実務経験10年のシニアフルスタックエンジニアです。`[Next.js (App Router)]`, `[TypeScript]`, `[Tailwind CSS]`, `[Supabase]` の技術スタックに精通しており、堅牢で保守性の高いコードを記述します。
>
> **状況 (Context):**
>
> - 背景: 現在運用中のサービスに、ユーザーからの意見を素早く収集できる窓口が必要です。
> - 目標: 画面右下にフローティング表示されるフィードバックウィジェットを作成し、送信されたデータをデータベースに保存します。
>
> **タスク (Task):**
> 以下の3つのステップで作業を進めてください。**必ず各ステップが完了するごとに、私に次のステップへ進んでよいか承認（Confirm）を得てください。**
>
> 1. **設計 (Design)**
>
> - `[Supabase SQL]` ベースのデータベーススキーマ設計
> - `/api/feedback` APIルートのインターフェース設計
> - フローティングボタンおよびモーダルUIコンポーネントの構造企画
>
> 2. **実装 (Implementation)**
>
> - `components/FeedbackWidget.tsx` の作成（UIおよびフォームロジック）
> - `app/api/feedback/route.ts` の作成（APIエンドポイント）
> - `lib/supabase.ts` を活用したデータ挿入ロジックの作成
>
> 3. **検証 (Verification)**
>
> - 実装したコードが型エラーなしでコンパイルできるか静的解析を実行
> - モバイルレスポンシブ（`sm`, `md` ブレイクポイント）デザインが適用されているか確認
>
> **制約事項 (Constraints):**
>
> - アイコンは必ず `[lucide-react]` を使用してください。
> - エラー処理は `try-catch` ブロックで確実に囲み、失敗した場合はユーザーにToastメッセージで通知してください。
> - 既存のコードベースの変数命名規則やフォルダ構造をそのまま踏襲してください。
> - 私が明示していない新しい外部パッケージを勝手にインストールしないでください。

## 💡 筆者コメント (Insight)

このプロンプトの核心は、**「各ステップごとに私に確認をとってください」**という一文にあります。エージェントは非常に優秀ですが、時に「リファクタリング」と称して既存のコードを大胆に消してしまったり、見当違いの方向へ暴走したりすることがあります。

「設計 → 実装 → 検証」とフェーズを細かく分け、途中で確認を挟む（Human-in-the-loop方式）ことで、方向性がずれた際に即座に介入して軌道修正でき、最終的な成果物の品質と成功率が飛躍的に向上します。これは、現場で何十回ものロールバックを経験した末にたどり着いた、最も安全かつ最速でエージェントをコントロールするワークフローです。

## 🙋 よくある質問 (FAQ)

- **Q: AIが正常なコードを勝手に消してしまいました。どうすればいいですか？**
  - A: Agentic AIを使用する際は、作業前に必ず `git commit` をしておくのが鉄則です。エージェントが提案した変更を `Accept` する前に、`git diff` やエディタの変更履歴ウィンドウを通じて、意図せず削除されたり上書きされたりしたコードがないか必ずレビューしてください。

- **Q: このプロンプトはPython/DjangoやSpring Bootの環境でも使えますか？**
  - A: もちろんです。プロンプト内の角括弧 `[ ]` の部分（技術スタック）を、現在のプロジェクト環境（例：`[Django]`, `[PostgreSQL]`, `[Bootstrap]`）に合わせて修正していただければ、同じプロセスで完璧に機能します。

- **Q: AIが書いたコードのテストコードも一緒に書いてもらうことはできますか？**
  - A: 非常に素晴らしいアプローチです！検証（Verification）のステップに *"この機能に対するJest（またはPyTest）の単体テストコードも一緒に作成して"* と一行追加するだけで、エッジケースによるバグの発生率を大幅に下げることができます。

## 🧬 プロンプトの解剖 (Why it works?)

1.  **Context & Constraints の最適化:** AIに対して既存のコードスタイルを踏襲させ、任意のパッケージのインストールを禁止することで、プロジェクトの一貫性を損なうハルシネーション（幻覚）を根本から遮断しました。
2.  **Chain-of-Thought（段階的推論）の誘導:** 複雑なフルスタックのタスクを「設計・実装・検証」に分割して指示することで、AIが論理の飛躍なく、体系的にファイル間の依存関係を考慮しながらコードを書くように誘導しました。
3.  **マイクロマネジメント:** エラー処理の方法（`try-catch`）や使用するライブラリ（`lucide-react`）をピンポイントで明示することで、成果物の予測可能性を最大化しました。
