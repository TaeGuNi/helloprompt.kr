> _"留住正在消逝的网络历史。只需一次复制粘贴，即可剥离动态网页中杂乱的渲染元素，将其核心语境完美定格为永久的 Markdown 文档。"_

随着单页应用（SPA）和“围墙花园”（Walled Gardens）等封闭式平台逐渐成为现代 Web 的常态，依靠传统方式抓取并保存高价值网页信息已变得举步维艰。不仅链接可能在转瞬之间失效（Link Rot），甚至连专业的 Web 存档网站也经常在抓取动态渲染页面时败下阵来，最终留下的往往只有一片空白的报错画面。

这款提示词是一件强大的数字资产保存利器。无论你输入的是满屏杂乱无章的网页可见文本，还是晦涩难懂的 HTML 源码，它都能自动精准地剔除一切无用的噪音（如横幅广告、全局导航栏、页脚信息），并将核心内容重构为**结构完美、历久弥新且随时可供调用的 Markdown 文档**。

---

## ⚡️ 3句话要点总结 (TL;DR)

1. **智能去噪处理：** 从复杂的网页数据转储中过滤掉广告和多余的 UI 元素，精准淬炼出文章的正文核心。
2. **永久格式转换：** 将碎片化的内容重构为极具长期保存价值与高兼容性的 Markdown 格式。
3. **元数据自动提取：** 自动生成包含发布日期、作者、核心关键词等对于后续存档与检索至关重要的 YAML Frontmatter 元数据。

## 🚀 解决方案：“数字档案管理员”提示词

### 🥉 Basic Version (基础版)

当你仅需快速留存网页的核心正文时，请使用此版本。直接全选（Ctrl+A）并复制网页的全部可见内容，连同下方的提示词一并发送给 AI。

> **角色：** 你是一位专业的“数字档案管理员”。
> **请求：** 请从下方输入的杂乱网页文本中，剔除菜单、广告等一切干扰噪音，只将包含信息价值的“正文内容”用 Markdown 格式干净利落地提取并整理出来。

### 🥇 Pro Version (专家版)

当你希望在不丢失网站原始语境（Context）的前提下，构建完美的结构化存档文档，以便无缝导入个人知识管理系统（如 Obsidian、Notion）时，请务必使用此版本。

> **角色 (Role)：** 你是一位致力于保存人类数字遗产的顶尖“数字档案管理员（Digital Archivist）”。你的首要任务是从复杂且碎片化的 Web 内容中淬炼信息精华，并将其加工成可永久保存的格式。
>
> **背景 (Context)：**
>
> - 现状：由于动态网页（SPA）和封闭平台的盛行，Web 存档变得日益困难。我们需要直接将原始文本或源代码转换为永久保存格式。
> - 目标：从输入的文本/HTML 数据转储中清除不必要的噪音（广告、全局导航栏、页脚、评论等），并将其转化为结构完美的 Markdown 文档。
>
> **任务 (Task)：**

> 3. 使用合理的标题层级（`##`, `###`）重新编排正文，并应用 Markdown 语法进行格式化。
> 4. 对于重要的引用、关键数据和统计资料，请使用 Markdown 引用（`>`）或加粗（`**`）进行视觉上的强调。

> **输入数据：**
>
>
> [请在此处粘贴网页的全部文本(Ctrl+A, C)或 HTML 源代码]

>
> **限制条件 (Constraints)：**
>
> - 严禁擅自总结或曲解正文的原始语句与语境。（因为目的是存档，必须保持 100% 的原文忠实度）。
> - 输出格式必须严格遵循纯粹的 Markdown 语法，不要将正文包裹在代码块中。

## 💡 作者点评 (Insight)

这个提示词绝不仅是一个简单的“文章摘要器”。在 2026 年，互联网信息消亡的速度正呈指数级增长，而这套提示词，正是一条**将信息所有权彻底夺回并转移至本地环境的专属数据管道**。

它在那些常规网页剪藏工具（Web Clipper）无能为力的场景下尤显神威——比如需要强制登录的付费新闻信、无限滚动的 X (Twitter) 帖子，或是存在访问限制的公司内部 Wiki 页面。你只需在屏幕上无脑全选（Ctrl+A）、复制（Ctrl+C），然后将其全盘扔进提示词中，那些繁杂的 UI 残渣便会如变魔术般瞬间消散，只留下纯净无比的原始知识数据。就我个人而言，借助这套工作流，我每周都能将数十篇濒临失效的优质长文完美迁移到 Obsidian 的永久金库中。

## 🙋 常见问题 (FAQ)

- **Q: 我应该复制 HTML 源代码，还是直接复制屏幕上的可见文本？**
  - A: 两者均可。但对于采用最新动态渲染技术的网页，在浏览器中直接全选（Ctrl+A）并复制可见文本往往效果更好。这样能大幅剔除底层的隐藏脚本和追踪器带来的干扰噪音，让 AI 将算力完全集中于解析核心文本。

- **Q: 网页内容太长导致 AI 模型报错（超出 Token 限制），该怎么办？**
  - A: 直接抓取超长网页确实容易导致 Token 消耗激增并触发上下文窗口限制。强烈建议使用支持超大上下文窗口的旗舰级模型（如 Claude 3.5 Sonnet 或 Gemini 2.5 Pro）来执行此任务。

## 🧬 提示词解剖 (Why it works?)

1. **强制保持原文忠实度 (Constraints)：** LLM 在面对长篇文本时，往往会本能地触发“摘要归纳”机制。我们在提示词中加入“因为目的是存档，必须保持 100% 的原文忠实度”这一强力约束条件，能够有效抑制 AI 的总结冲动，迫使其将所有注意力聚焦于“完整存档”这一核心目标。
2. **结构化提取元数据：** 要求强制生成 YAML Frontmatter，这绝不仅仅是为了让文档在视觉上显得更专业，更重要的是确保了极高的技术层面的兼容性。它使得生成的内容能够无缝对接到各种真实的数据库系统或基于 Markdown 的现代化笔记应用中，便于后期的检索与管理。
