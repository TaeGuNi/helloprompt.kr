# ⌨️ Vim 단축키 마스터: 마우스 없이 코딩하는 법

- **🎯 추천 대상:** 코딩하다가 마우스 잡으러 손 뻗는 0.5초가 아까운 괴짜 개발자, 터미널에서 `vi` 켰다가 못 빠져나와서 컴퓨터 껐다 켠 사람
- **⏱️ 소요 시간:** 평생 (하지만 10분이면 기본은 함)
- **🤖 추천 모델:** ChatGPT-4o (단축키 조합 설명)

- ⭐ **난이도:** ⭐⭐⭐⭐⭐ (진입 장벽 높음)
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐ (익숙해지면 빛의 속도)
- 🚀 **활용도:** ⭐⭐⭐☆☆

> _"어떻게 나가요? `:q`? `:wq`? 아, 살려줘..."_

Vim은 에디터가 아니라 **'언어'**입니다. `d`(삭제) `2`(두 번) `w`(단어) -> `d2w` (단어 두 개 삭제). 이 문법을 깨우치는 순간, 당신은 텍스트를 편집하는 게 아니라 **지휘**하게 됩니다.

---

## ⚡️ 3줄 요약 (TL;DR) {#tl-dr}

1.  **Normal Mode:** 명령을 내리는 상태. (ESC 누르면 됨)
2.  **Insert Mode:** 글을 쓰는 상태. (`i` 누르면 됨)
3.  **Visual Mode:** 드래그하는 상태. (`v` 누르면 됨)

## 🚀 해결책: "Vim Tutor Prompt"

### 🥉 Basic Version (생존 키트)

일단 파일 수정하고 저장하고 끄기.

> **요청:** "리눅스 서버에서 설정 파일을 고쳐야 해. `vi`로 파일을 열고, '특정 단어 검색'해서 '수정'하고 '저장 후 종료'하는 키만 딱 알려줘."

<br>

### 🥇 Pro Version (매크로 & 레지스터)

반복 작업 자동화.

> **역할 (Role):** 너는 Vim만 20년 쓴 고인물 개발자야.
>
> **상황 (Context):**
>
> ```json
> { "id": 1, "name": "a" },
> { "id": 2, "name": "b" },
> ... (100줄)
> ```
>
> 이 JSON 데이터에서 `id` 값을 전부 100씩 더해주고 싶어. (1 -> 101, 2 -> 102...)
>
> **요청 (Task):**
>
> 1.  **매크로(Macro):** `q` 키를 써서 한 줄을 수정하는 매크로를 녹화하고, 나머지 99줄에 적용(`@`)하는 순서를 알려줘.
> 2.  **레지스터(Register):** 복사한 내용을 숫자 레지스터(`"1p`)나 이름 레지스터(`"ap`)에 따로 저장하는 법을 알려줘.
> 3.  **플러그인:** VS Code에서 Vim을 쓸 때 꼭 깔아야 할 플러그인 설정(`settings.json`)을 추천해줘.

Vim을 억지로 쓰진 마세요. 하지만 **VS Code에 Vim 확장을 깔아서** 쓰는 건 강력 추천합니다.
이동은 `hjkl`로 하고, 다중 커서나 리팩토링은 VS Code 기능을 쓰는 **하이브리드** 방식이 현대 개발자의 국룰입니다.

## 🙋 자주 묻는 질문 (FAQ) {#faq}

- **Q: `hjkl`이 왜 방향키죠?**
  - A: 옛날 키보드엔 화살표 키가 없었거든요. (진짜임) 오른손이 기본 자리(Home Row)에서 벗어나지 않게 하기 위함입니다.

- **Q: 한글 입력은요?**
  - A: Vim의 최대 적은 '한영 전환'입니다. 입력 모드에서 나올 때 자동으로 영문으로 바꿔주는 `ime-auto-switch` 같은 플러그인이 필수입니다.

## 🧬 프롬프트 해부 (Why it works?) {#why-it-works}

1.  **반복 작업 예시:** "100줄 수정"이라는 구체적 상황을 주면, AI가 단순 단축키 나열이 아니라 **'매크로(녹화 및 재생)'**라는 Vim의 꽃을 설명해 줍니다.
2.  **환경 설정(VS Code):** 순수 Vim은 너무 하드코어합니다. 대중적인 VS Code 환경을 전제로 팁을 달라고 하면 훨씬 실용적인 답변이 나옵니다.

## 📊 증명: Before & After

### ❌ Before (마우스)

마우스 잡고 드래그 -> 키보드로 수정 -> 다시 마우스 잡고 다음 줄 클릭 -> (손목 터널 증후군 🏥)

### ✅ After (Vim)

`qa` (녹화 시작) -> `^cw` (수정) -> `j` (아래로) -> `q` (녹화 끝) -> `99@a` (99번 반복)
