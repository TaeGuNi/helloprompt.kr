- ⭐ **难度：** ⭐☆☆☆☆
- ⚡️ **效果：** ⭐⭐⭐⭐⭐
- 🚀 **实用度：** ⭐⭐⭐⭐⭐

> _“‘这代码我以前肯定写过……叫什么来着？’如果你在搜索框前愣了半小时还没找到，是时候停下来换个思路了。”_

作为开发者，几乎每个人的电脑里都有一个塞满“宝藏”的备忘录。好不容易从 Stack Overflow 扒下来的正则表达式、某个写得极具巧思的工具函数，还有那些总觉得“以后肯定用得上”的 Nginx 配置文件。

然而，到了真正急需的时候，你却怎么也想不起它们被丢在了哪个角落；就算侥幸找到，也早就忘了当时是怎么调用的，最后只好从头再 Google 一遍。事实上，代码片段（Snippet）的核心生命力就在于**“易于检索”**与**“开箱即用”**。

哪怕是随手复制的零散代码，只要交给 AI，它就能瞬间摇身一变，成为自带标准注释、使用示例和分类标签的完美资产。今天，我将向你分享一个超强提示词，帮你把杂乱无章的记事本，彻底蜕变成系统、专业的个人知识库（Knowledge Base）。

## ⚡️ 3句话总结 (TL;DR)

1. **打造高检索率资产：** 为来源不明的代码块自动提取标题、标签与元数据，让全局搜索轻而易举。
2. **文档全自动化生成：** 瞬间补全包含参数和返回值的标准注释（如 JSDoc），并附带立即可运行的示例代码。
3. **重构提升代码质量：** AI 自动修正凌乱的缩进与语义不明的变量名，让代码随时保持最高级别的复用状态。

### 🥉 Basic Version (基础版)

> **代码：**
> `[在此处粘贴你复制的代码]`

### 🥇 Pro Version (专业版)

当你准备打造完美的个人代码 Wiki，或者希望将代码无缝接入 Notion、Obsidian 数据库时，请使用此专业版提示词，它将为你生成极其规范的“完整 Markdown 文档”。

> **角色 (Role)：**
> 你是一位拥有 10 年经验的资深技术作家兼开发者，对清晰的文档和整洁的代码（Clean Code）有着极其严苛的追求。
>
> **背景 (Context)：**
> 
> - 现状：我正在整理从网上或过往项目中随手复制的零散代码，准备构建个人的代码知识库（Notion / Obsidian）。
> - 目标：将我提供的代码转化为一份结构完美、高度规范的 Markdown 文档。它必须极易检索，并确保未来的我哪怕只看一眼，也能在 1 秒内领悟并直接投入使用。
>
> **任务 (Task)：**
> 
> 请深入分析我提供的 `[代码]`，并严格按照以下结构生成 Markdown 文档：
> 
> 1. 元数据 (Metadata)：包含 `Title`（明确的功能目的）、`Language` 以及 `Tags`（3~5个高频搜索关键字）。
> 2. 描述 (Description)：用 1~2 句话一针见血地总结这段代码的核心作用与解决的痛点。
> 3. 重构与注释 (Refactored Code)：修正缩进与排版，严格使用 JSDoc（或目标语言的标准 Docstring）为参数（`@param`）和返回值（`@return`）补充详尽注释。若存在表意不明的变量名，请在不改变原意的前提下进行语义化重构。
> 4. 使用示例 (Usage Example)：编写一段简短、可独立运行的示例代码，直观展示在实际业务中应如何调用该函数。
>
> **约束条件 (Constraints)：**
> 
> - 输出格式必须且只能是一份完整的 Markdown 文档。
> - 必须忠实于原代码的核心逻辑，严禁对算法思路或主函数名进行大范围的随意篡改。
>
> **注意事项 (Warning)：**
> 
> - 若代码中包含任何敏感信息（如 IP 地址、数据库密码、API Key 等），请务必在文档最顶端输出高亮警告。
> - 接下来我会不断向你发送新的 `[代码]`，在你的首次回复中，只需回答一句：“请输入您需要整理的代码。”即可。

## 🙋 常见问题 (FAQ)

- **Q: 公司的安全政策严格，禁止将业务代码上传给外部 AI 怎么办？**
  - A: 遇到涉及 API 密钥、数据库密码、内网 IP 或核心业务算法的“敏感代码”时，请务必在发送前将其剔除，或替换为 `XXX`、`1234` 等虚拟值进行脱敏。若你的开发环境具有极高的保密级别，强烈建议使用本地部署的开源大语言模型（如 Llama 3 或 Qwen）来完成整理工作。

- **Q: 如果 AI 重构后的代码风格（如命名规范）不符合我的个人习惯怎么办？**
  - A: 你只需在提示词的 **约束条件 (Constraints)** 区块中，硬性注入你的个人代码规范即可。例如补充说明：_“所有变量必须严格遵守 camelCase 命名法，且字符串统一使用单引号而非双引号。”_

- **Q: 这套提示词兼容所有编程语言吗？冷门语言也可以吗？**
  - A: 绝对没问题。它不仅能完美驾驭 Python、JavaScript、Java、C++ 等主流语言，还能游刃有余地解析 SQL 复杂查询、晦涩的 Bash 脚本，甚至是结构繁复的 Dockerfile 或 Kubernetes YAML 配置文件。

```javascript
// 时间差求取
function cd(d1, d2) {
  let t = new Date(d1).getTime() - new Date(d2).getTime();
  return Math.floor(t / (1000 * 60 * 60 * 24));
}
```

### ✅ After (AI 处理结果)

````markdown
**Title:** 计算两个日期之间的天数 (Days) 差值
**Language:** JavaScript
**Tags:** `#Date`, `#Utility`, `#TimeDifference`, `#JavaScript`

**Description:**
这是一个实用工具函数，接收两个日期字符串（或 Date 对象）作为输入，并以整数形式返回这两个日期之间确切的天数差值。

**Refactored Code:**

```javascript
/**
 * 计算两个日期之间的天数 (Days) 差值。
 *
 * @param {string|Date} targetDate - 需要对比的目标日期
 * @param {string|Date} baseDate - 作为对比基准的日期
 * @returns {number} 两个日期之间的天数差（若 targetDate 在 baseDate 之后，则返回正数）
 */
function calculateDaysDifference(targetDate, baseDate) {
  const timeDifference =
    new Date(targetDate).getTime() - new Date(baseDate).getTime();
  return Math.floor(timeDifference / (1000 * 60 * 60 * 24));
}
```

**Usage Example:**

```javascript
const today = "2026-02-12";
const dDay = "2026-12-25";

const daysLeft = calculateDaysDifference(dDay, today);
console.log(`距离圣诞节还有 ${daysLeft} 天！`);
// 输出: 距离圣诞节还有 316 天！
```
````
