- **⏱️ 所要時間:** 1時間 → 10分（プロトタイプ構築）
- **🤖 推奨モデル:** Claude 3.7 Sonnet（Claude Code環境）

- ⚡️ **効果性:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「複雑な物理演算も、Canvasのレンダリングも、もうゼロから書く必要はありません。AIエージェントという『最強のシニアエンジニア』を味方につける時代です。」_

ゲーム開発のパラダイムは今、根本的な転換期を迎えています。かつて2D物理エンジンの実装やイベントリスナーの管理に費やしていた膨大な時間は、**Claude Code**のような自律型AIエージェントの登場により、もはや過去のものとなりました。Claude Codeは単なるコードの自動補完ツールではありません。プロジェクト全体のアーキテクチャを深く理解し、主体的にシステムを構築する真のパートナーです。本記事では、この圧倒的な能力を活用し、わずか1時間でブラウザベースの2Dゴルフゲームを完成させる実践的なプロセスを、そのまま使えるプロンプトとともに大公開します。

## ⚡️ 3行まとめ (TL;DR)

1. **AIがアーキテクチャ全体を設計:** 摩擦や速度などの物理演算からレンダリングロジックまで、単なるスニペット生成を超えてゲームの基盤を主導的に構築します。
2. **圧倒的な生産性:** 複雑な衝突判定やマウスイベント処理に悩む必要はゼロ。プロンプトの指示ひとつで「引っ張って飛ばす（ドラッグ＆シュート）」アクションを瞬時に実装できます。
3. **拡張性の高いモジュール構造:** 初期段階から `Physics`、`Input`、`Level` といったクラスを独立させるため、オンライン対戦や新ステージ追加などの将来的なアップデートも極めてスムーズです。

## 🚀 解決策：「Claude Code ゲーム開発プロンプト」

### 🥉 Basic Version（基本型）

プロトタイプの骨組み（ボイラープレート）を迅速に構築したい場合に最適な、初期設定用のプロンプトです。

> **役割 (Role):** あなたは `[シニアゲームエンジニア]` です。
>
> **タスク (Task):** `[HTML5 Canvas]` を使用して、ブラウザベースの2Dゴルフゲームを作成してください。ボールとホールを配置し、マウスのドラッグ操作でボールを打つ（drag-to-shoot）仕組みを実装してください。

### 🥇 Pro Version（専門家型）

物理エンジン、入力処理、レベルデザインが完全に分離された、高度なモジュール型ゲームアーキテクチャを設計・構築する際に威力を発揮するプロンプトです。

> **役割 (Role):** あなたは10年の実務経験を持つ `[シニアフロントエンド兼ゲームクライアントエンジニア]` であり、私の完璧なペアプログラミングパートナーです。
>
> **状況 (Context):**
>
> - 背景: 純粋なVanilla JSとHTML5 Canvasのみを用いて、軽量なトップダウンビューの2Dゴルフゲームを開発しようとしています。
> - 目標: 物理演算、入力イベント処理、レベルの状態管理がそれぞれ完全にモジュール化された、保守性の高い堅牢なコードを記述することです。
>
> **タスク (Task):**
>
> 1. ファイル構成を `index.html`（Canvasコンテナ）、`game.js`（メインループ）、`physics.js`（ベクトル演算、摩擦、衝突処理などの数学的ロジック）に分割し、構造化してください。
> 2. `physics.js` 内に、芝生の摩擦（Friction）や壁面衝突時の跳ね返り（Bounce）に関する精緻なロジックを実装してください。
> 3. マウスの `mousedown`、`mousemove`、`mouseup` イベントを活用し、ボールを引っ張った際の予想軌道とパワーを視覚的に示すインジケーター（Indicator Line）機能を追加してください。
> 4. `[障害物 (Obstacle)]` クラスを定義し、難易度が段階的に上昇する3つの異なるレベル（Level）配列を構築してください。
>
> **制約事項 (Constraints):**
>
> - Phaser.jsなどの外部ゲームエンジンやライブラリは一切使用しないでください。
> - すべての主要なロジックには詳細なコメントを添え、動作原理を明確に説明してください。
> - モジュールパターンまたはES6のClass構文を採用し、徹底したオブジェクト指向プログラミング（OOP）に基づいたコードを記述してください。

## 💡 筆者のコメント (Insight)

このプロンプトの最大の真髄は、AIを単なる「コード生成機」として扱うのではなく、**「システムアーキテクト」**として位置づけている点にあります。初期段階で「ファイルの分割」や「クラスの分離」といった明確な制約（Constraints）を与えない限り、AIはすべての処理を `index.html` のような単一ファイルに詰め込み、保守が不可能なスパゲッティコードを生成しがちです。

特に、物理演算と状態管理を別々のモジュールとして切り出すよう強制したことで、後から「風（Wind）の抵抗」といった新要素を追加したり、WebSocketでオンライン対戦を実装したりする際の改修コストを劇的に下げることに成功しました。Claude Codeは文脈（コンテキスト）の維持能力に非常に優れているため、まずは強固な骨組み（アーキテクチャ）を作らせ、その上に少しずつ機能を肉付けしていくアプローチが最も確実で効果的です。

## 🙋 よくある質問 (FAQ)

- **Q: 物理学や高度な数学の知識がなくても本当に開発できますか？**
  - A: はい、全く問題ありません。「芝生の摩擦を再現して」「壁にぶつかったらリアルに跳ね返るようにして」と自然言語で指示するだけで、ベクトルの内積や反発係数といった複雑な数学的処理はClaudeがすべて自動でコード化してくれます。
- **Q: スマホのブラウザ（タッチ操作）にも対応させるにはどう指示すべきですか？**
  - A: Pro Versionのプロンプトのタスク（Task）に、「`マウスイベントだけでなく、touchstart, touchmove, touchend にも対応させ、モバイル環境でも同じ操作性を実現して`」という一文を追加するだけです。これでデバイスを問わずシームレスに動作するコードが生成されます。

## 🧬 プロンプトの解剖 (Why it works?)

1. **アーキテクチャ分割の明示:** `game.js` や `physics.js` のようにファイル単位で処理を分割するよう強制することで、コードの肥大化を防ぎ、AIが各ドメインのロジック構築に100%集中できる環境を作り出しています。
2. **UXとメカニズムの解像度:** 単に「ゴルフゲームを作って」と丸投げするのではなく、「引っ張った軌道とパワーを視覚化するインジケーター」というように期待するユーザー体験（UX）を具体的に言語化したことで、AIとの認識のズレを完全に排除しています。

// 完全にモジュール化され、オブジェクト指向で設計された物理エンジン (physics.js)
export class PhysicsBody {
  constructor(x, y) {
    this.position = new Vector(x, y);
    this.velocity = new Vector(0, 0);
    this.friction = 0.98; // 芝生の摩擦係数
  }

  update() {
    // 速度に摩擦力を適用

    // 位置の更新
    this.position.add(this.velocity);
