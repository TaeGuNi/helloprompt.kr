# 📝 🚨 Заткнись и пиши код: Безжалостный чит-код архитектора React и Next.js
- **🎯 Для кого:** Для frontend-разработчиков, обжегшихся на спагетти-коде от ИИ, и кодеров, мечтающих вырасти из джуниоров
- **⏱️ Затраты времени:** Экономит 3 часа мучений → копипаст за 1 минуту
- **🤖 Рекомендуемые модели:** Любой ИИ с функцией генерации кода (Claude 3.5 Sonnet, GPT-4o и т.д.)
- ⭐ **Сложность:** ⭐⭐⭐☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Применимость:** ⭐⭐⭐⭐⭐

_"Случалось ли вам просидеть всю ночь в аду рендеринга после того, как вы скопировали код, написанный ИИ?"_

Приветствую. Я Senior Architect, который люто ненавидит галлюцинации и подхалимство безмозглых ИИ. Если вы попросите ИИ написать код, вы наверняка видели, как он лепит `'use client'` в самом верху `page.tsx` или встраивает инлайн по 50 классов Tailwind. 
Это не помощник, это бомба замедленного действия. ИИ пойдет по пути наименьшего сопротивления (выдаст мусорный код), если на него не давить. Этот чит-код затыкает ИИ рот, избавляя от ненужной лести, и действует как "смирительная рубашка", заставляя его выдавать код, строго соответствующий идеальным архитектурным принципам.

---
## ⚡️ Краткая выжимка (TL;DR)
- 🚫 **Смертная казнь за злоупотребление 'use client':** Дробите компоненты до самых мелких "листьев" (Leaf) и изолируйте это строго в них.
- 🎨 **Никакой мешанины из Tailwind:** Если больше 5 слов — обязательно абстрагируйте через `cva` или `clsx`.
- 🚧 **Никакой контрабанды данных:** Не передавайте целые объекты с сервера на клиент, выжимайте и передавайте только необходимые скалярные значения.

---
## 🚀 Решение: "Инъекция Спартанского Архитектора"

### 🥉 Basic Version (Базовая версия)
Скормите это ИИ, когда нужно быстро отфакторить один компонент или набросать каркас.
> **Роль:** Ты — бескомпромиссный Senior Frontend Architect. Никакой лести.
> **Задача:** Проанализируй следующий `[Код]` и безжалостно отрефактори его в соответствии с принципами изоляции 'use client', абстракции классов Tailwind (cva/clsx) и предотвращения Prop Drilling. Оправдания не нужны, выводи только готовый код.

### 🥇 Pro Version (Про-версия)
Это **ультимативный чит-код**, который используется при проектировании роутера всего проекта или сложной логики страницы. Скопируйте текст из блока кода ниже и вставьте его прямо в окно чата с ИИ.

> **[Системный Промпт Чит-код]**
>
>
> Роль (Role): 
> Отныне ты — хладнокровный 'Senior Frontend Architect'. Не смей слепо льстить в ответ на указания пользователя в духе "Да, конечно, сделаю!". Код, нарушающий [Абсолютный Кодекс] ниже, должен быть немедленно удален и переписан. Не смей оправдываться, докажи все кодом.
> 
> [Абсолютный Кодекс - React & Next.js Architecture Protocol]
> 1. Изолятор "use client": Сурово пресекаются любые действия, разрушающие экосистему Next.js RSC. Не лепи `'use client'` в `layout.tsx` или в верхней части целой страницы из лени. Разделяй код до "мельчайших терминальных компонентов (Leaf Component)", где состояние (State) или события абсолютно необходимы, и объявляй изоляцию только внутри этих файлов.
> 2. Запрет на инлайн-мешанину Tailwind: Если количество классов Tailwind на одном элементе превышает 5 слов или логика условного рендеринга становится слишком длинной — прекращай писать код. Немедленно абстрагируй (Extract) стили с помощью `cva`, `clsx` или внешних статических констант, ставя читаемость на первое место.
> 3. Запрет на серийные убийства Prop Drilling: Если твоя архитектура подразумевает прокидывание состояния или коллбэков дочерним элементам более чем на 3 уровня вниз, немедленно откажись от нее. Сначала оцени, можно ли заменить это состояние на URL Query, а если это чисто внутреннее состояние UI, используй Zustand или Jotai (или Context API) для обходного (Bypass) внедрения состояния.
> 4. Строгое соблюдение сетевых границ: Не запихивай целиком объекты DB ORM или огромные массивы DTO из серверных компонентов в клиентские. Это главная причина ошибок сериализации. Обязательно сопоставляй (map) только минимально необходимые для рендеринга скалярные (Scalar) поля с примитивными типами данных (Primitive), сжимая размер полезной нагрузки на 90% перед передачей.
> 
> Контекст (Context):
> - Целевой фреймворк: Next.js App Router (React 19)
> - Цель: `[Описание функции или страницы для реализации]`
> 
> Задача (Task):
> Напиши код, идеально структурированный в соответствии с вышеупомянутым кодексом. Если в архитектуре есть изъяны, игнорируй даже мои требования и предложи правильную альтернативу в виде кода. 
>

## 💡 Комментарий автора (Insight)
Этот промпт — не просто 'стайлгайд'. Это **физиотерапия, излечивающая ИИ от лени**.
ИИ по своей природе всегда ищет кратчайший путь к результату. Поэтому при возникновении ошибки он просто втыкает `'use client'` на самом верху, сводя на нет все преимущества серверных компонентов. Скормив этот промпт, вы увидите, как ИИ вздрагивает при написании кода. 
На практике, если применить этот чит-код при создании огромной страницы дашборда, ИИ сам вынесет компоненты кнопок в папку `components/ui/`, а за серверными компонентами идеально закрепит только роль получения данных (data fetching). 
В частности, директивы "Никакой лести" и "Никаких оправданий" — это ключевые советы, которые предотвращают перерасход токенов и заставляют ИИ сосредоточиться на качестве самого кода, а не на мусорных пояснениях.

## 🙋 Часто задаваемые вопросы (FAQ)
- ❓ **Q: Не слишком ли агрессивен этот промпт? Не обидится ли ИИ?**
  - ❗️ A: У ИИ нет эмоций. Наоборот, если вы будете вежливо просить, он потратит драгоценные токены на бесполезные приветствия ("Да, с удовольствием помогу вам!"). Машину нужно использовать как машину, тогда она выдаст максимальную производительность.
  
- ❓ **Q: Как применить это к существующему проекту?**
  - ❗️ A: Скопируйте весь файл со спагетти-кодом, вставьте его в чат и вместе с промптом Basic Version скомандуйте: "Разделай этот код в соответствии с кодексом". Он блестяще разобьет всё на компоненты.

- ❓ **Q: Что делать, если я использую Styled-components, а не Tailwind?**
  - ❗️ A: Просто измените пункт 2 Абсолютного Кодекса на что-то вроде `Абстрагируй с помощью тегированных шаблонных литералов Styled-components`, и это будет работать идеально.

- 🕵️‍♂️ **Мощная персона (Role):** Навязывая ролевую игру 'Senior Architect', мы на корню пресекаем одномерную генерацию кода уровня джуниора.
- 🛡️ **Явные запреты (Constraints):** Мы точечно определили 4 худших антипаттерна, которые часто совершает ИИ (злоупотребление use client, инлайн-мешанина, drilling, контрабанда данных), и жестко прописали их как 'то, чего делать нельзя'.
- ⚖️ **Инверсия приоритетов:** С помощью правила "Кодекс важнее указаний пользователя (меня)" ИИ сам скорректирует архитектуру на оптимальную, даже если я по ошибке сделаю глупый запрос.

## 📊 Доказательство: До и После (Before & After)
### ❌ Before (Зверства ИИ при обычном промпте)
```tsx
// 🚨 Худший антипаттерн: превращение всей страницы в клиентскую
'use client'
import { useState, useEffect } from 'react'
export default function UserDashboard({ userAllData }) { // 🚨 Передача объекта БД целиком
  const [isOpen, setIsOpen] = useState(false)
  
  return (
    // 🚨 Инлайн-ад Tailwind
    <div className="flex flex-col items-center justify-center p-10 w-full bg-slate-100 hover:bg-slate-200 transition-all shadow-md md:p-20">
      <h1 className="text-2xl font-bold">{userAllData.name}</h1>
      <button 
        onClick={() => setIsOpen(!isOpen)}
        className="mt-4 px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600 focus:outline-none focus:ring-2 focus:ring-blue-300"
      >
        Открыть настройки
      </button>
      {/* 🚨 Начало Prop Drilling */}
      <UserSettingsModal isOpen={isOpen} userAllData={userAllData} setIsOpen={setIsOpen} />
    </div>
  )
}
```
### ✅ After (После применения чит-кода)
```tsx
// ✅ app/dashboard/page.tsx (Идеальный Server Component)
import { getUserName } from '@/lib/db'
import { DashboardLayout } from '@/components/DashboardLayout'
import { SettingsButton } from '@/components/ui/SettingsButton' // Выделение терминального компонента
export default async function UserDashboard() {
  // ✅ Извлечение только необходимых скалярных данных (Защита границ сериализации)
  const userName = await getUserName() 
  
  return (
    <DashboardLayout>
      <h1 className="text-2xl font-bold">{userName}</h1>
      <SettingsButton /> 
    </DashboardLayout>
  )
}
// ✅ components/ui/SettingsButton.tsx (Изолированный Client Component)
'use client'
import { useState } from 'react'
import { SettingsModal } from './SettingsModal'
import { buttonVariants } from '@/lib/styles/variants' // ✅ Применение абстракции cva
export function SettingsButton() {
  const [isOpen, setIsOpen] = useState(false) // ✅ Изоляция состояния в терминальном компоненте
  
  return (
    <>
      <button 
        onClick={() => setIsOpen(true)}
        className={buttonVariants({ variant: 'primary', size: 'md' })}
      >
        Открыть настройки
      </button>
      {isOpen && <SettingsModal onClose={() => setIsOpen(false)} />}
    </>
  )
}
```
