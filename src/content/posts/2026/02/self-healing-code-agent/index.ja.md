## 📝 自己修復コード(Self-Healing Code)：自らバグを修正するAIエージェントの構築

- **🎯 おすすめの対象:** 開発者、データエンジニア、AIエージェントの導入を検討しているリーダー
- **⏱️ 所要時間:** 無限のデバッグループ → 自動化により0分へ
- **🤖 推奨モデル:** GPT-4o, Claude 3.5 Sonnet, Gemini 2.5 Pro（コーディング特化モデル）

> _「AIが書いたコードでエラーが出たとき、未だにエラーメッセージをコピーしてAIに貼り付ける作業を繰り返していませんか？」_

AIにPythonスクリプトを生成させて実行したら `SyntaxError` が発生する。そのエラーをコピーしてAIに貼り付け、修正を依頼して初めてコードが動く。

しかし、ここで一つ根本的な問いを投げかけたいと思います。**なぜ「人間」が、AIとターミナルの間で単なるコピー＆ペーストの仲介役を担っているのでしょうか？**

2026年現在、最先端のエンジニアたちは単にAIへコード作成を依頼するだけの段階をとうに過ぎています。彼らは**「自己修復システム（Self-Healing Systems）」**を構築しているのです。本記事では、「作成 → 実行 → 修正」という退屈なデバッグループを完全に自動化し、受動的なコード生成ツールを「自律的に問題を解決するエージェント」へと変貌させる画期的な手法を解説します。

---

## ⚡️ 3行要約 (TL;DR)

1. AIが一度で完璧なコードを書くという「One-Shot」の幻想から抜け出しましょう。
2. 実行エラー（`stderr`）をキャプチャし、AIにフィードバックとして打ち返す「反省（Reflection）ループ」の構築が鍵となります。
3. Pythonの `subprocess` モジュールを活用すれば、わずか50行のコードで強力な自己修復エージェントを作成できます。

## 💻 Pythonでのループ自動化 (Implementation)

LangChainなどの重量級AIフレームワークを導入する必要はありません。Pythonの標準ライブラリである `subprocess` を使うだけで、このリフレクションループを見事にオーケストレーションできます。

```python
import subprocess
# OpenAIや他のLLMクライアントが設定されていることを前提とします。
# from openai import OpenAI
# client = OpenAI()

def generate_fix(prompt):
    # LLMを呼び出して修正されたコードを取得する関数（GPT-4、Claude、Llama 3など）
    # response = client.chat.completions.create(...)
    # return response.choices[0].message.content
    pass

def run_and_heal(script_path, max_retries=3):
    for attempt in range(max_retries):
        # 1. コード実行: エラーを安全にキャプチャするため、別プロセスで実行
        process = subprocess.run(
            ['python', script_path],
            capture_output=True, text=True
        )

        # 成功: returncode 0はエラーがないことを意味する
        if process.returncode == 0:
            print(f"✅ 成功!（試行回数: {attempt + 1}）")
            return process.stdout

        # 失敗: stderrからエラーメッセージを抽出
        error_msg = process.stderr
        print(f"❌ 失敗（試行回数: {attempt + 1}）: {error_msg.strip().splitlines()[-1]}")

        # 2. 自己修復 (Agentic Step)
        with open(script_path, 'r') as f:
            broken_code = f.read()

        healing_prompt = f"""
        このエラーを基にPythonコードを修正してください。

        [元のコード]
        {broken_code}

        [エラーメッセージ]
        {error_msg}

        説明やマークダウンの装飾は省き、「修正されたコード」のみを返してください。
        """

        fixed_code = generate_fix(healing_prompt)

        # 修正されたコードで元のスクリプトを上書き
        with open(script_path, 'w') as f:
            f.write(fixed_code)

        print(f"🩹 コードの修復が完了しました。再実行します...")

    print("💀 最大再試行回数を超過したため、修復に失敗しました。")
    return None
```

## 💡 作成者コメント (Insight)

この「自己修復（Self-Healing）」パターンは、私たちのAI活用パラダイムを根本から変革するポテンシャルを秘めています。

以前、Webスクレイピングの自動化ツールを開発した際、AIが存在しないCSSセレクタをハルシネーション（幻覚）として出力し、スクリプトが途中で停止してしまうという致命的な問題に直面しました。その時、単なる構文エラーの修正にとどまらず、「抽出したリストが空（Empty）であればエラーと見なして例外を投げる」という**検証ループ（Validation Loop）**を意図的に組み込みました。

すると驚くべき現象が起きました。エージェントが正常にデータを抽出できるまで、**自ら別のCSSセレクタを次々と試し、コードを継続的に修正**し始めたのです。これはもはや単なるバグ修正の枠を超え、AIが与えられた環境（未知のWebページのDOM構造）に対して自律的に**適応（Adaptation）**していくプロセスそのものでした。AIにコードを生成させること以上に重要なのは、生成されたコードの動作を「検証」し、その結果に対してAI自身に「責任」を持たせるループを設計することなのです。

## 🙋 よくある質問 (FAQ)

- **Q: コードを上書きすることで、スクリプトが完全に壊れて回復不能になったらどうしますか？**
  - A: 実務環境においては、 `script_path` を直接上書きするのではなく、 `script_v1.py` 、 `script_v2.py` のようにイミュータブルなバージョン管理を行うか、Gitのコミットプロセスを自動化していつでも安全にロールバック（切り戻し）できるフェイルセーフを設けることを強く推奨します。

- **Q: AIが誤った修正を繰り返し、無限ループに陥る危険性はありませんか？**
  - A: だからこそ `max_retries` （最大再試行回数）のハードリミット設定が不可欠なのです。通常、3〜5回の試行で解決できないエラーは、プロンプト自体のコンテキスト不足か、人間のアーキテクトによる介入が必要な構造的欠陥である可能性が極めて高いと判断すべきです。

- **Q: 自己修復ループにはどのLLMモデルを採用するのが最適ですか？**
  - A: 高度なコード解析と論理的な推論能力が要求されるため、GPT-4o、Claude 3.5 Sonnet、あるいはコーディングタスクに特化してファインチューニングされた最新のローカルモデルを強力にお勧めします。

## 🧬 プロンプトの解剖 (Why it works?)

1. **エラー原因の分析を強制（Chain-of-Thought）:** いきなりコードの修正作業に入らせるのではなく、まずはエラーログの分析という先行タスクを与えることで、AIに問題の根本原因を深く推論・理解させるよう誘導しました。
2. **出力の厳格な制御（Systematic Output）:** 「一切の説明を省き、コードのみを出力せよ」という強力な制約を課すことで、Python側で修正コードをパースしてファイルに書き込む際、不要な自然言語テキストの混入による二次的な `SyntaxError` の発生を未然に防いでいます。

## 📊 証明：Before & After

### ❌ Before（従来の方法）

```text
1. ユーザーがAIにコード作成を依頼
2. コード実行後、SyntaxErrorなどのバグが発生
3. ユーザーが手動でエラーメッセージ (Traceback) をコピー
4. AIにデバッグを依頼し、コードの再作成を待つ
5. コードをコピーして再実行（無限ループ）
```

### ✅ After（自己修復システムの適用）

```text
1. ユーザーがAIに目標を指示してコードを作成
2. システムが自動実行し、エラー (stderr) をキャプチャ
3. AIエージェントが自らエラーを分析し、再記述 (Self-Healing)
4. 完璧に動作する最終的な成果物のみをユーザーに提供
```
