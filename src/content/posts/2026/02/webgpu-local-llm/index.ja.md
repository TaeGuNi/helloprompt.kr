## 🙋 よくある質問 (FAQ)

- **Q: WebGPUに非対応の古いブラウザでは全く動作しないのでしょうか？**
  - A: はい、WebGPUによるハードウェアアクセラレーションが必須条件となります。実務環境においては、プロンプトの制約事項にもある通り、まずデバイスのWebGPUサポート状況を判定し、非対応の場合は自動的にクラウドAPI（OpenAIなど）へフォールバックする「ハイブリッドアーキテクチャ」を設計するのがベストプラクティスです。

- **Q: モデルのダウンロード容量が大きすぎると、ユーザーの負担になりませんか？**
  - A: Llama-3 8Bの4ビット量子化モデルの場合、およそ4〜5GBのデータ通信が発生します。そのため、初回アクセス時にはWi-Fi環境での接続を推奨するガイドを表示し、ダウンロードの進捗を視覚的かつ洗練されたプログレスバーで提示することで、ユーザーの離脱を最小限に抑える緻密なUX設計が鍵となります。

- **Q: テキストの生成速度（TPS：Tokens Per Second）は実用に耐えうるレベルですか？**
  - A: Apple M2/M3チップセットを搭載したMacや、最新のディスクリートGPU環境であれば、毎秒30〜50トークンという驚異的な推論速度を記録します。これは商用のクラウドAPIと全く遜色のない、極めて快適なレスポンス体験です。

```tsx
// useWebLLM.ts (プロンプトが生成したプロダクションレベルのコードの一部)
import { useState, useEffect } from "react";
import { CreateMLCEngine } from "@mlc-ai/web-llm";

export const useWebLLM = (modelId = "Llama-3-8B-Instruct-q4f32_1") => {
  const [progress, setProgress] = useState(0);
  const [engine, setEngine] = useState<any>(null);
  const [isReady, setIsReady] = useState(false);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const initEngine = async () => {
      if (!navigator.gpu) {
        setError("現在のブラウザはWebGPUをサポートしていません。");
        return;
      }
      try {
        const mlcEngine = await CreateMLCEngine(modelId, {
          initProgressCallback: (report) => {
            setProgress(Math.round(report.progress * 100));
          },
        });
        setEngine(mlcEngine);
        setIsReady(true);
      } catch (err) {
        setError("モデルの読み込み中にエラーが発生しました。");
      }
    };
    initEngine();
  }, [modelId]);

  return { engine, isReady, progress, error };
};
```
