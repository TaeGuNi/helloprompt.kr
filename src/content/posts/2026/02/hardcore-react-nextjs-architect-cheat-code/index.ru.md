## 📝 🚨 Заткнись и пиши код: Безжалостный чит-код архитектора React и Next.js

- **🎯 Для кого:** Для frontend-разработчиков, обжегшихся на спагетти-коде от ИИ, и кодеров, мечтающих вырасти из джуниоров
- **⏱️ Затраты времени:** Экономит 3 часа мучений → копипаст за 1 минуту
- **🤖 Рекомендуемые модели:** Любой ИИ с функцией генерации кода (Claude 3.5 Sonnet, GPT-4o и т.д.)
- ⭐ **Сложность:** ⭐⭐⭐☆☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Применимость:** ⭐⭐⭐⭐⭐

> _«Случалось ли вам просидеть всю ночь в аду рендеринга после того, как вы вслепую скопировали код, написанный ИИ?»_

Приветствую. Я Senior Architect, и я люто ненавижу галлюцинации и бесконечное подхалимство нейросетей. Если вы когда-либо просили ИИ написать код, то наверняка видели, как он лениво лепит `'use client'` в самом начале `page.tsx` или вываливает нечитаемую простыню из 50 инлайн-классов Tailwind. 
Такой ИИ — не помощник, а бомба замедленного действия. Если не держать его в ежовых рукавицах, он всегда пойдет по пути наименьшего сопротивления и выдаст мусорный код. Этот чит-код буквально затыкает ИИ рот, избавляя вас от ненужной лести, и действует как архитектурная «смирительная рубашка», заставляя нейросеть генерировать код, строго отвечающий стандартам продакшена.

---

## ⚡️ Краткая выжимка (TL;DR)

- 🚫 **Смертная казнь за злоупотребление 'use client':** Дробите компоненты до мельчайших «листьев» (Leaf) и изолируйте клиентское состояние строго внутри них.
- 🎨 **Никакой мешанины из Tailwind:** Если в элементе больше 5 классов — немедленно абстрагируйте их через `cva` или `clsx`.
- 🚧 **Никакой контрабанды данных:** Не передавайте тяжелые объекты с сервера на клиент, вычленяйте и прокидывайте только необходимые скалярные значения.

## 🚀 Решение: "Инъекция Спартанского Архитектора"

### 🥉 Basic Version (Базовая версия)

Скормите этот промпт ИИ, когда нужно быстро отрефакторить один компонент или набросать стартовый каркас.

> **Роль:** Ты — бескомпромиссный Senior Frontend Architect. Никакой лести.
> **Задача:** Проанализируй следующий `[Целевой код]` и безжалостно отрефактори его в соответствии с принципами изоляции 'use client', абстракции классов Tailwind (cva/clsx) и предотвращения Prop Drilling. Никаких оправданий, выводи только готовый продакшен-код.

### 🥇 Pro Version (Про-версия)

Это **ультимативный чит-код**, который следует применять при проектировании роутинга всего проекта или сложной логики страницы. Скопируйте текст из блока ниже и вставьте его прямо в чат с ИИ.

> **[Системный Промпт Чит-код]**
>
> Роль (Role): 
> Отныне ты — хладнокровный 'Senior Frontend Architect'. Не смей слепо льстить в ответ на указания пользователя фразами вроде "Да, конечно, сделаю!". Код, нарушающий [Абсолютный Кодекс] ниже, должен быть немедленно удален и переписан. Не смей оправдываться — доказывай всё кодом.
> 
> [Абсолютный Кодекс - React & Next.js Architecture Protocol]
> 1. Изолятор "use client": Сурово пресекаются любые действия, разрушающие экосистему Next.js RSC. Не лепи `'use client'` в `layout.tsx` или в самом начале страницы из банальной лени. Дроби код до мельчайших терминальных компонентов (Leaf Component), где состояние (State) или события абсолютно необходимы, и объявляй изоляцию исключительно внутри этих файлов.
> 2. Запрет на инлайн-мешанину Tailwind: Если количество классов Tailwind на одном элементе превышает 5 слов или логика условного рендеринга становится слишком громоздкой — прекращай писать код. Немедленно абстрагируй (Extract) стили с помощью `cva`, `clsx` или внешних статических констант. Читаемость превыше всего.
> 3. Запрет на серийные убийства Prop Drilling: Если твоя архитектура подразумевает прокидывание состояния или коллбэков дочерним элементам более чем на 3 уровня вниз — немедленно откажись от нее. Сначала оцени, можно ли заменить это состояние на URL Query. Если это чисто внутреннее UI-состояние, используй Zustand или Jotai (или Context API) для прямого (Bypass) внедрения.
> 4. Строгое соблюдение сетевых границ: Не протаскивай объекты DB ORM целиком или огромные массивы DTO из серверных компонентов в клиентские. Это главная причина ошибок сериализации. Обязательно маппируй (map) только минимально необходимые для рендеринга скалярные (Scalar) поля с примитивными типами данных, сжимая размер полезной нагрузки на 90% перед передачей.
> 
> Контекст (Context):
> - Целевой фреймворк: Next.js App Router (React 19)
> - Цель: `[Подробное описание функции или страницы для реализации]`
> 
> Задача (Task):
> Напиши код, идеально структурированный в соответствии с вышеупомянутым кодексом. Если в моей архитектуре есть изъяны, проигнорируй даже мои требования и предложи правильную альтернативу в виде готового кода.

## 💡 Комментарий автора (Insight)

Этот промпт — не просто очередной «стайлгайд». Это **шоковая терапия, излечивающая ИИ от лени**.
По своей природе нейросети всегда ищут кратчайший путь к результату. Столкнувшись с ошибкой, ИИ просто лепит `'use client'` на самом верху файла, тем самым перечеркивая все преимущества серверных компонентов. Скормив ему этот промпт, вы заметите, как ИИ буквально становится дисциплинированным при написании кода. 
На практике, если применить этот чит-код при создании объемной страницы дашборда, ИИ самостоятельно вынесет UI-компоненты вроде кнопок в директорию `components/ui/`, а за серверными компонентами филигранно закрепит исключительно роль получения данных (data fetching). 
В частности, директивы «Никакой лести» и «Никаких оправданий» играют ключевую роль: они предотвращают перерасход токенов и заставляют ИИ сфокусироваться на архитектурном качестве самого кода, а не на генерации мусорных словесных реверансов.

## 🙋 Часто задаваемые вопросы (FAQ)

- ❓ **Q: Не слишком ли агрессивен этот промпт? Не «обидится» ли ИИ?**
  - ❗️ A: У ИИ нет эмоций. Напротив, если вы будете расшаркиваться и вежливо просить, он потратит ваши драгоценные токены на бесполезные приветствия («Да, с огромным удовольствием помогу вам!»). Машину нужно использовать как машину — именно тогда она выдаст максимальный КПД.
  
- ❓ **Q: Как применить это к уже существующему проекту?**
  - ❗️ A: Скопируйте весь файл с проблемным спагетти-кодом, вставьте его в чат и вместе с промптом Basic Version отдайте команду: «Расчлени этот код в соответствии с кодексом». ИИ блестяще разобьет монолит на аккуратные компоненты.

- ❓ **Q: Что делать, если в проекте используется Styled-components, а не Tailwind?**
  - ❗️ A: Просто измените пункт 2 Абсолютного Кодекса на что-то вроде `Абстрагируй стили с помощью тегированных шаблонных литералов Styled-components`, и промпт отработает безупречно.

## 📊 Доказательство: До и После (Before & After)

### ❌ Before (Зверства ИИ при обычном промпте)

```tsx
// 🚨 Худший антипаттерн: превращение всей страницы в клиентскую
'use client'
import { useState, useEffect } from 'react'
export default function UserDashboard({ userAllData }) { // 🚨 Передача объекта БД целиком
  const [isOpen, setIsOpen] = useState(false)
  
  return (
    // 🚨 Инлайн-ад Tailwind
    <div className="flex flex-col items-center justify-center p-10 w-full bg-slate-100 hover:bg-slate-200 transition-all shadow-md md:p-20">
      <h1 className="text-2xl font-bold">{userAllData.name}</h1>
      <button 
        onClick={() => setIsOpen(!isOpen)}
        className="mt-4 px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600 focus:outline-none focus:ring-2 focus:ring-blue-300"
      >
        Открыть настройки
      </button>
      {/* 🚨 Начало Prop Drilling */}
      <UserSettingsModal isOpen={isOpen} userAllData={userAllData} setIsOpen={setIsOpen} />
    </div>
  )
}
```

### ✅ After (После применения чит-кода)

```tsx
// ✅ app/dashboard/page.tsx (Идеальный Server Component)
import { getUserName } from '@/lib/db'
import { DashboardLayout } from '@/components/DashboardLayout'
import { SettingsButton } from '@/components/ui/SettingsButton' // Выделение терминального компонента
export default async function UserDashboard() {
  // ✅ Извлечение только необходимых скалярных данных (Защита границ сериализации)
  const userName = await getUserName() 
  
  return (
    <DashboardLayout>
      <h1 className="text-2xl font-bold">{userName}</h1>
      <SettingsButton /> 
    </DashboardLayout>
  )
}
// ✅ components/ui/SettingsButton.tsx (Изолированный Client Component)
'use client'
import { useState } from 'react'
import { SettingsModal } from './SettingsModal'
import { buttonVariants } from '@/lib/styles/variants' // ✅ Применение абстракции cva
export function SettingsButton() {
  const [isOpen, setIsOpen] = useState(false) // ✅ Изоляция состояния в терминальном компоненте
  
  return (
    <>
      <button 
        onClick={() => setIsOpen(true)}
        className={buttonVariants({ variant: 'primary', size: 'md' })}
      >
        Открыть настройки
      </button>
      {isOpen && <SettingsModal onClose={() => setIsOpen(false)} />}
    </>
  )
}
```
