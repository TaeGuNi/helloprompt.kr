![\"Stop Pasting Context: Build Your First MCP Server in 5 Minutes\"](/images/hooks/mcp-server-tutorial.jpg)

2026년 현재, **MCP(Model Context Protocol)**는 LLM과 데이터를 잇는 사실상의 표준(De-facto standard)으로 자리 잡았습니다. 이는 쉽게 말해 AI 모델 전용 'USB 포트'입니다. 매번 수동으로 컨텍스트를 떠먹여 주는 수고를 덜고, 서버를 한 번만 연결해 두면 모델이 알아서 내 로컬 파일과 데이터베이스, 내부 API를 실시간으로 읽어옵니다.

물론 깃허브(GitHub)나 파일 시스템(Filesystem)을 연동하는 훌륭한 범용 MCP 서버는 이미 존재합니다. 하지만 이 기술의 진가는 **'우리 회사만의 고유한 비즈니스 로직'**을 LLM에 직접 연결할 때 비로소 발휘됩니다.

오늘은 복잡한 보일러플레이트 코드를 직접 짜는 대신, AI에게 지시해 단 5분 만에 완벽한 TypeScript 기반의 맞춤형 MCP 서버를 스캐폴딩(Scaffolding)하는 프롬프트를 공개합니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **무의미한 복붙의 종말:** MCP를 활용해 LLM이 내 로컬 데이터(DB, 로그, API)에 직접 접근하고 상호작용하게 만듭니다.
2. **프롬프트 단 한 줄로 서버 구축:** 번거로운 초기 설정 없이 프롬프트 하나만으로 완벽하게 구동되는 MCP 서버 코드를 자동 생성합니다.
3. **압도적인 프라이버시 보호:** 민감한 데이터를 외부로 유출할 필요 없이, 로컬 환경에서 모델의 컨텍스트 윈도우로 안전하게 스트리밍합니다.

## 🚀 해결책: "MCP Server Scaffolder"

### 🥉 Basic Version (기본형)

기본적인 MCP 서버의 골격만 빠르게 뽑아내고 싶을 때 활용하세요.

> **역할:** 너는 시니어 TypeScript 개발자야.
> **요청:** `@modelcontextprotocol/sdk`를 사용하여 `[원하는 목적, 예: 특정 도시의 날씨를 알려주는]` 역할을 하는 간단한 MCP 서버 코드를 작성해 줘.


### 🥇 Pro Version (전문가형)

실무에 즉시 투입할 수 있을 만큼 견고하고 확장 가능한 서버 코드가 필요할 때 활용하세요.

> **역할 (Role):** 너는 Model Context Protocol(MCP) 생태계에 정통한 시니어 TypeScript 엔지니어입니다.
>
> **상황 (Context):**
>
> - 배경: 로컬 데이터나 내부 API를 LLM과 연동하기 위한 커스텀 MCP 서버가 필요합니다.
> - 목표: 내 요구사항에 딱 맞는 단일 파일 형태(`index.ts`)의 완전하고 실행 가능한 MCP 서버 코드를 생성해야 합니다.
>
> **요청 (Task):**
>
> 1. 다음 요구사항을 충족하는 MCP 서버 코드를 작성해 주세요.
>    - **목적:** `[여기에 원하는 서버의 목적을 구체적으로 적어주세요. 예: 특정 로그 파일의 마지막 50줄을 읽어오는 기능]`
> 2. 작성된 코드는 사용자가 터미널에서 즉시 실행해 볼 수 있도록 주석과 함께 복사-붙여넣기 친화적으로 구성하세요.
>
> **제약사항 (Constraints):**
>
> - 반드시 공식 `@modelcontextprotocol/sdk`를 사용해야 합니다.
> - 도구의 입력 파라미터가 복잡할 경우, 스키마 검증을 위해 `zod` 패키지를 활용하세요 (권장).
> - 서버 생성 시 `ListToolsRequestSchema` 및 `CallToolRequestSchema` 인터페이스를 완벽하게 구현해야 합니다.
> - 도구 핸들러(Handler) 내부에 예외 처리(`try/catch`) 로직을 반드시 포함하여 서버가 다운되지 않도록 하세요.
> - **부연 설명 없이 오직 TypeScript 코드 블럭 하나만 출력하세요.**
>
> **주의사항 (Warning):**
>
> - 실제로 존재하지 않는 SDK 메서드를 지어내지 마세요. 불확실한 경우 기본 Node.js API를 활용하여 우회하세요.

## 💡 작성자 코멘트 (Insight)

이 프롬프트의 핵심은 AI에게 막연히 "코드를 짜줘"라고 부탁하는 데 있지 않습니다. MCP 서버가 반드시 갖춰야 할 **프로토콜의 핵심 규격(ListTools 및 CallTool 요청 처리)**을 명시적으로 강제하여 오류를 차단한 것이 비결입니다.

직접 맞춤형 MCP 서버를 구축했을 때 얻을 수 있는 가장 큰 무기는 바로 **'압도적인 보안(Privacy)'**입니다. 범용 웹 검색 툴은 쿼리가 외부 서버를 거쳐야 하지만, 로컬 PostgreSQL 데이터베이스나 내부 API에 연결된 커스텀 MCP 서버를 활용하면 모든 데이터가 오직 여러분의 철저한 통제하에 머뭅니다. 보안이 생명인 내부 문서를 복사해 외부 챗창에 붙여넣는 위험한 행동을 완전히 근절할 수 있습니다.

처음에는 이 프롬프트를 활용해 로컬 로그 파일을 읽어오는 단순한 '읽기 전용(Read-only)' 도구부터 가볍게 시작해 보세요. 작동 원리가 손에 익으면, 이후 데이터베이스에 직접 개입하고 수정하는 기능까지 확장하여 나만의 강력하고 정교한 AI 비서를 완성할 수 있습니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 코딩 지식이 전혀 없어도 이 프롬프트를 사용할 수 있나요?**
  - A: 네, 물론입니다! `[목적]` 란에 "서울의 날씨를 알려주는 기능"처럼 자연어만 적어주셔도, Claude가 구조적으로 완벽한 코드를 대신 작성해 줍니다. 여러분은 생성된 코드를 복사해 실행 환경에 붙여넣기만 하면 됩니다.

- **Q: 생성된 서버 코드는 어떻게 실행하고 연결하나요?**
  - A: 코드를 `index.ts` 파일로 저장한 뒤, Claude Desktop의 설정 파일(`claude_desktop_config.json`) 내 `mcpServers` 항목에 해당 스크립트의 실행 경로만 추가해 주시면 즉각적으로 연동됩니다.

- **Q: 이 프롬프트로 만든 서버는 Claude에서만 작동하나요?**
  - A: 아닙니다. Model Context Protocol은 범용적인 개방형 표준(Open Standard)입니다. 따라서 Ollama, Cursor 등 MCP를 지원하는 모든 클라이언트와 플랫폼에서 완벽하게 호환됩니다.

## 🧬 프롬프트 해부 (Why it works?)

1.  **제약조건(Constraints)의 강력한 통제:** `@modelcontextprotocol/sdk`와 `zod`의 사용을 강제하고, 필수 스키마(`ListToolsRequestSchema`, `CallToolRequestSchema`)를 엄격하게 명시했습니다. 이를 통해 AI가 스펙에 어긋나는 코드를 지어내는 할루시네이션(환각) 현상을 원천 차단했습니다.
2.  **단일 파일 및 코드 전용 출력:** 여러 파일로 파편화된 복잡한 아키텍처 대신 단일 파일(`index.ts`) 구조를 지시하고 불필요한 부연 설명을 제거했습니다. 그 결과 코드를 복사하고 테스트하는 데 걸리는 시간(Time-to-Value)을 극대화했습니다.
3.  **예외 처리 강제화:** 도구 실행 도중 예상치 못한 에러가 발생하더라도 MCP 서버 전체가 다운(Crash)되지 않도록, `try/catch` 방어 로직을 미리 철저하게 규정했습니다.
