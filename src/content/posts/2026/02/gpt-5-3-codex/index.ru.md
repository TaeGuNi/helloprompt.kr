### 🥇 Pro Version (Продвинутая версия)

Идеально подходит для масштабного рефакторинга кросс-файловых зависимостей с использованием всего огромного окна в 200k токенов.

> **Роль (Role):** Ты — Lead Архитектор ПО и эксперт по оптимизации кода.
>
> **Контекст (Context):**
>
> - Фон: Я использую GPT-5.3 Codex Spark в своей IDE. У нас есть legacy-код, который необходимо кардинально обновить и обезопасить.
> - Цель: Глубокий рефакторинг с максимально точным учетом всех связей между компонентами целого проекта.
>
> **Задача (Task):**
>
> 1. Проанализируй предоставленный `[Основной файл]` и его зависимости из `[Связанные модули]`.
> 2. Полностью перепиши код, применяя актуальные паттерны проектирования.
> 3. Внедри строгую типизацию и продвинутую обработку ошибок.
> 4. Оптимизируй производительность алгоритма везде, где это технически возможно.
>
> **Ограничения (Constraints):**
>
> - Выведи результат исключительно в виде чистого Markdown-блока с кодом.
> - Не используй устаревшие (deprecated) библиотеки и методы.
>
> **Внимание (Warning):**
>
> - Опирайся строго на предоставленный контекст. Если какой-то класс или интерфейс не определен в контексте, категорически не выдумывай его реализацию (никаких галлюцинаций!), а просто оставь комментарий `// TODO: Реализовать [Название]`.

## 🧬 Разбор промпта (Why it works?)

1. **Использование сильных сторон модели:** Промпт напрямую обращается к уникальной способности Codex Spark обрабатывать массивный контекст (`[Связанные модули]`).
2. **Constraints (Ограничения):** Жесткое требование "выводить только блок с кодом" кардинально экономит время разработчика на чтение лишнего текста и позволяет моментально скопировать результат.
3. **Warning (Предотвращение галлюцинаций):** Команда "не выдумывать реализацию недостающих классов" критически важна для безопасной работы с большими проектами, где ИИ часто склонен фантазировать о несуществующих методах.

### ❌ Before (Устаревший и нестабильный подход)

```javascript
// Старый код без типов и с классическим callback-адом
function fetchUserData(userId, callback) {
  database.query(
    "SELECT * FROM users WHERE id = '" + userId + "'",
    function (err, result) {
      if (err) {
        callback(err);
      } else {
        callback(null, result[0]);
      }
    },
  );
}
```

### ✅ After (Результат рефакторинга через Codex Spark)

```typescript
// Современный, безопасный от инъекций и строго типизированный код
interface UserProfile {
  id: string;
  username: string;
  email: string;
  createdAt: Date;
}

export async function fetchUserData(
  userId: string,
): Promise<UserProfile | null> {
  try {
    const query =
      "SELECT id, username, email, created_at as createdAt FROM users WHERE id = $1";
    const { rows } = await database.query<UserProfile>(query, [userId]);

    return rows[0] ?? null;
  } catch (error) {
    console.error(
      `[DB Error] Failed to fetch profile for user ${userId}:`,
      error,
    );
    throw new DatabaseOperationError("Не удалось получить данные пользователя");
  }
}
```
