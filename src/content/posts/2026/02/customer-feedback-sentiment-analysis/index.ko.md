# 📝 구구절절한 고객 피드백, 긍정/부정 분석하기

- **🎯 추천 대상:** CS 담당자, 프로덕트 매니저(PM), 마케터, 1~3년 차 기획자
- **⏱️ 소요 시간:** 2시간 → 1분 단축
- **🤖 추천 모델:** ChatGPT (GPT-4o), Claude 3.5 Sonnet (긴 텍스트 처리에 유리)

- ⭐ **난이도:** ⭐⭐☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"수백, 수천 개의 고객 리뷰... 이걸 언제 다 읽고 엑셀에 분류해서 보고서 쓰실 건가요?"_

고객의 소리(VOC)에서 인사이트를 뽑아내는 것은 서비스 개선의 핵심입니다. 하지만 날것의 방대한 텍스트 데이터를 일일이 읽고 '긍정, 부정, 중립'으로 분류하는 작업은 엄청난 시간과 감정 노동을 요구합니다. 이제 AI에게 이 지루한 작업을 맡기고, 당신은 '개선 전략'을 세우는 데 집중하세요.

---

## ⚡️ 3줄 요약 (TL;DR)

1. 구구절절한 고객 피드백을 **긍정/중립/부정으로 자동 분류**하고 핵심 키워드를 추출합니다.
2. 엑셀이나 CSV 데이터를 그대로 복사해서 붙여넣기만 하면 **보고서용 요약본**이 완성됩니다.
3. 부정적인 리뷰에서 **가장 시급하게 해결해야 할 페인 포인트(Pain Point)**를 즉시 파악할 수 있습니다.

---

## 🚀 해결책: "VOC 감정 분석기"

### 🥉 Basic Version (기본형)

빠르게 대략적인 여론의 흐름만 파악하고 싶을 때 사용하세요.

> **역할:** 너는 10년 차 CS 품질 관리 및 데이터 분석 전문가야.
> **요청:** 아래 나열된 고객 리뷰 데이터를 읽고, 전체적인 감정(긍정/부정/중립) 비율과 가장 많이 언급된 불만 키워드 3가지만 빠르게 요약해 줘.
>
> `[여기에 리뷰 텍스트 복사 붙여넣기]`

<br>

### 🥇 Pro Version (전문가형)

주간/월간 VOC 보고서 작성이나 프로덕트 개선 회의를 위한 디테일한 인사이트가 필요할 때 사용하세요.

> **역할 (Role):** 너는 IT 서비스의 CS 데이터를 분석하고 프로덕트 개선 방향을 도출하는 '10년 차 수석 데이터 애널리스트'야.
>
> **상황 (Context):**
>
> - 배경: 최근 한 달간 우리 서비스에 접수된 고객 피드백(앱스토어 리뷰, 고객센터 문의 등)의 원본 데이터야.
> - 목표: 이 방대한 비정형 데이터를 정량적/정성적으로 분석하여, 경영진과 개발팀이 즉각적으로 액션을 취할 수 있는 VOC 보고서를 작성하는 것.
> - 데이터: `[여기에 엑셀이나 텍스트 형태의 리뷰 데이터 100~500개 붙여넣기]`
>
> **요청 (Task):**
>
> 1. **감정 비율 분석:** 전체 리뷰를 분석하여 긍정, 중립, 부정의 비율(%)을 수치화해 줘.
> 2. **부정 리뷰 심층 분석 (Pain Points):** 부정적인 피드백에서 가장 많이 언급된 불만 사항 Top 3를 선정하고, 각각의 구체적인 고객 사례(원문 인용)와 발생 원인을 추정해 줘.
> 3. **긍정 리뷰 심층 분석 (Aha Moments):** 고객들이 가장 만족해하는 우리 서비스의 핵심 강점 Top 3를 요약해 줘.
> 4. **우선순위 액션 플랜:** 위 분석을 바탕으로, 프로덕트 팀이 다음 스프린트에서 당장 개선해야 할 최우선 과제 2가지를 제안해 줘.
>
> **제약사항 (Constraints):**
>
> - 개인적인 추측이나 과장은 배제하고, 오직 제공된 '데이터'에만 근거하여 객관적인 톤앤매너로 작성할 것.
> - 시각적으로 가독성을 높이기 위해 마크다운 불릿 포인트(-)를 적극적으로 활용할 것.
> - 키워드를 제시할 때는 반드시 (언급 횟수 또는 빈도수)를 함께 표기할 것.

## 💡 작성자 코멘트 (Insight)

이 프롬프트는 엑셀에 쌓인 VOC 데이터를 복사(Ctrl+C)해서 챗창에 붙여넣기(Ctrl+V)할 때 가장 빛을 발합니다. 특히 Claude 3.5 Sonnet이나 ChatGPT의 'Advanced Data Analysis' 기능을 활용해 엑셀(.xlsx)이나 CSV 파일을 직접 업로드한 뒤 이 프롬프트를 입력하면, 수만 건의 데이터도 단 몇 분 만에 완벽한 분석 보고서로 탈바꿈합니다.

단순히 "분석해 줘"라고 하는 대신, **"액션 플랜을 도출해 줘"**라는 태스크를 부여했기 때문에 실무자가 결과물을 그대로 복사해서 보고서의 결론 부분에 활용하기 매우 좋습니다. 만약 특정 기능 출시 이후의 반응을 보고 싶다면, 상황(Context)에 *"최근 다크 모드 업데이트 이후의 반응에 특히 주목해 줘"*라고 한 줄만 추가해 보세요.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 한 번에 얼마나 많은 리뷰를 넣을 수 있나요?**
  - A: 무료 버전(GPT-3.5 등)은 한 번에 50~100개 정도가 한계지만, 유료 모델(GPT-4o, Claude 3.5 Sonnet)을 사용하거나 문서 파일을 직접 업로드하면 수천 개의 리뷰도 거뜬히 분석할 수 있습니다.
- **Q: 엑셀이나 CSV 파일을 직접 올려도 이 프롬프트가 작동하나요?**
  - A: 네, 완벽하게 작동합니다! 파일을 업로드한 후, 프롬프트 내용 중 `[여기에 엑셀이나... 데이터 붙여넣기]` 부분을 **"업로드한 파일 데이터를 바탕으로"**라고 수정해서 지시하면 훨씬 빠르고 깔끔하게 처리됩니다.

- **Q: 분석 결과가 조금 두루뭉술한 것 같아요.**
  - A: 제공한 리뷰 데이터의 양이 너무 적거나, 원문 내용이 짧은 경우 그럴 수 있습니다. 그럴 때는 Task에 *"불만 사항의 원인을 UI/UX 측면과 서버 오류 측면으로 나누어서 세밀하게 분석해 줘"*처럼 분석 기준을 더 구체적으로 쪼개주면 훨씬 날카로운 결과가 나옵니다.

## 🧬 프롬프트 해부 (Why it works?)

1.  **Role (수석 데이터 애널리스트):** AI에게 단순 번역기나 요약기가 아닌, '인사이트를 도출하는 전문가'라는 페르소나를 부여하여 결과물의 깊이를 더했습니다.
2.  **Task 분할 (감정 비율 → 심층 분석 → 액션 플랜):** 단순 요약을 넘어 현업에서 필요한 보고서의 흐름(현황 파악 → 원인 분석 → 해결책 제시)을 강제하여 즉시 실무에 적용할 수 있도록 설계했습니다.
3.  **Constraints (객관성 유지, 원문 인용):** AI가 지어내는 할루시네이션(Hallucination)을 막기 위해 철저히 제공된 데이터에만 근거할 것과, 고객의 '원문'을 직접 인용하라는 제약을 걸어 보고서의 신뢰도를 높였습니다.

## 📊 증명: Before & After

### ❌ Before (일반적인 지시)

```text
아래 리뷰 50개 읽고 분석 좀 해줘. 사람들이 뭘 좋아하고 뭘 싫어해?
(리뷰 텍스트 나열)
```

> **결과:** "고객들은 대체로 긍정적인 반응을 보이고 있습니다. 앱이 빠르다는 칭찬이 많지만, 종종 튕긴다는 의견도 있습니다. 개선이 필요합니다." (보고서에 쓰기에는 너무 성의 없고 구체적인 수치가 없음)

### ✅ After (Pro Version 적용 결과)

```text
## 📊 VOC 감정 분석 및 액션 플랜 보고서

**1. 종합 감정 비율**
- 긍정(Positive): 68%
- 부정(Negative): 22%
- 중립(Neutral): 10%

**2. 🚨 부정 리뷰 심층 분석 (Pain Points)**
- **1위: 결제 오류 (15회)**
  - 원문 인용: "카드 결제창에서 계속 무한 로딩이 걸려요."
  - 원인 추정: PG사 연동 모듈 타임아웃 오류
- **2위: 속도 저하 (8회)**
  - 원문 인용: "이미지 많은 페이지 넘길 때 너무 버벅거립니다."
  - 원인 추정: 대용량 이미지 캐싱 처리 누락

**3. ✨ 긍정 리뷰 심층 분석 (Aha Moments)**
- **1위: 직관적인 UI (24회)**
  - 원문 인용: "메뉴 찾기가 너무 쉬워서 부모님도 잘 쓰십니다."
- **2위: 빠른 고객 응대 (12회)**
  - 원문 인용: "1:1 문의 남겼는데 10분 만에 해결해주셨어요."

**4. 🚀 우선순위 액션 플랜 (Next Sprint)**
- **결제 모듈 안정화 (최우선):** 무한 로딩 이슈 해결을 위한 PG사 API 타임아웃 로그 점검 및 예외 처리 로직 추가
- **이미지 로딩 최적화:** 메인 피드의 대용량 이미지 Lazy Loading 적용 및 리사이징 서버 도입 검토
```
