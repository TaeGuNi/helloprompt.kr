### 🥇 Pro Version (전문가형)

멀티 에이전트 기반의 정교한 아키텍처 설계와 엄격한 보안 통제가 요구되는 실무 환경에서 활용하세요.

> **역할 (Role):** 너는 OpenClaw 2026 프레임워크 설계에 정통한 수석 AI 아키텍트야.
>
> **상황 (Context):**
>
> - **배경:** 고도의 의사결정과 유기적인 멀티 에이전트 협업이 필수적인 `[프로젝트 목표]`를 성공적으로 완수해야 해.
> - **목표:** OpenClaw 환경에서 도구 실행의 안정성을 보장하고 메모리 컨텍스트를 최적화하는, 완벽한 무결성의 자율 에이전트 아키텍처를 설계하는 것.
>
> **요청 (Task):**
>
> 1. `[메인 에이전트 역할]`을 총괄로 설정하고, 이를 보조하는 서브 에이전트들의 명확한 R&R(역할과 책임) 및 협업 워크플로우를 정의해 줘.
> 2. 각 에이전트가 호출할 `[필요한 도구(API, 검색 등)]`의 접근 권한 수준과 안전한 실행 환경(Secure Tool Execution)을 위한 제약 조건을 엄격하게 명시해 줘.
> 3. 단기(Short-term) 및 장기(Long-term) 메모리 관리 전략이 정교하게 반영된 에이전트별 핵심 시스템 프롬프트를 도출해 줘.
>
> **제약사항 (Constraints):**
>
> - 모든 출력은 가독성 높은 마크다운 형식을 준수하며, 에이전트 간의 데이터 통신 흐름은 단계별 불릿 포인트(List)로 구조화할 것.
> - OpenClaw 2026의 최신 보안 가이드라인인 **'권한 최소화(Least Privilege)'** 및 **'도구 실행 전 검증(Pre-execution Validation)'** 원칙을 반드시 적용할 것.
>
> **주의사항 (Warning):**
>
> - 검증되지 않았거나 존재하지 않는 가상의 도구(API) 스펙을 임의로 생성하지 마. 불확실한 요소는 반드시 "확인 필요"로 명시할 것. (환각 응답 절대 금지)

이 프롬프트는 OpenClaw 2026의 가장 강력한 무기인 **'멀티 에이전트 협업 구조'**와 **'안전한 도구 실행 환경'**을 뼈대부터 탄탄하게 설계하는 데 특화되어 있습니다. 실무에서 자율 에이전트 시스템을 직접 구축해 보면, 초기 단계에서 에이전트 간의 역할 분담과 도구 접근 권한을 엄격하게 통제하지 않을 경우 시스템이 무한 루프에 빠지거나 치명적인 API를 오작동시키는 문제에 빈번하게 직면합니다.

이 프롬프트의 `[프로젝트 목표]` 변수에 **'자동화된 버그 수정 및 PR 생성'**과 같은 실무 과제를 입력해 보세요. 코드를 딥다이브하여 분석하는 에이전트, 최적의 수정안을 제안하는 에이전트, 그리고 회귀 테스트를 검증하는 에이전트가 톱니바퀴처럼 완벽하게 맞물려 돌아가는 아키텍처 초안을 즉각적으로 얻을 수 있습니다. 초기 설계에 소모되는 수십 시간의 시행착오를 단숨에 단축해 주는 압도적인 무기입니다.

  - A: 네, 기본적인 아키텍처 설계용으로 충분히 활용 가능합니다. 다만, 2026 버전에 새롭게 도입된 **'고도화된 메모리 관리(Advanced Memory Management)'** 기능은 하위 버전에서 100% 호환되지 않을 수 있으므로, 구버전 환경에 맞춰 해당 조건을 유연하게 튜닝하여 적용하시길 권장합니다.

- **Q: 에이전트가 통제 범위를 벗어나 불필요한 도구를 남용하거나 과도한 API 호출을 발생시키면 어떻게 방어하나요?**
  - A: 프롬프트의 제약사항(Constraints) 블록에 다음 보안 지침을 강력하게 추가해 보세요. _"각 에이전트는 사전에 인가된 도구 외에는 어떠한 리소스에도 접근할 수 없으며, 상태 변경을 유발하는 도구 실행 전에는 반드시 **사용자 개입(Human-in-the-loop)을 통한 명시적 승인**을 대기해야 한다."_

## 🧬 프롬프트 해부 (Why it works?)

1.  **압도적인 페르소나 부여 (Role):** AI에게 '수석 AI 아키텍트'라는 최상위 전문가 페르소나를 부여함으로써, 일반적인 챗봇 수준을 뛰어넘는 고도화된 시스템 설계 인사이트를 강제로 끌어냈습니다.
2.  **핵심 기능으로의 포커싱 (Context):** 단순 자동화가 아닌, OpenClaw 2026의 정수인 **'복잡한 의사결정'**과 **'멀티 에이전트 유기적 협업'**에 AI의 사고 과정을 집중시켰습니다.
3.  **보안 프레임워크 강제 (Constraints):** '권한 최소화' 및 '도구 실행 전 검증'이라는 엄격한 제약 조건을 주입하여, 자율 에이전트가 초래할 수 있는 원치 않는 데이터 삭제나 권한 오남용 등 치명적인 보안 사고를 설계 단계부터 원천 차단했습니다.
