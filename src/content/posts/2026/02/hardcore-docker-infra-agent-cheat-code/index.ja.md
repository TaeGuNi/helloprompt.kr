# 📝 💀 AIエージェントの胸倉を掴んでキャリーするハードコア・インフラ制御チートキー (Docker)

- **🎯 おすすめの対象:** シニア開発者、DevOps、AIエージェントのポンコツぶりのせいでモニターを叩き割りたくなったエンジニア
- **⏱️ 所要時間:** 3分のセットアップ → 一生デッドロック(Deadlock)から解放
- **🤖 おすすめのモデル:** ローカルシステムの制御権限を持つAgentic AI (Gemini CLI, Cursor, Claude 3.5 Sonnetなど)
- ⭐ **難易度:** ⭐⭐⭐⭐☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **実用性:** ⭐⭐⭐⭐⭐

_AIにターミナルを任せたら、`-it`オプション一つのせいで永遠の待機状態に陥り、ブチギレそうになったことはないか？_

我々が一騎当千の同僚エンジニアを求めているのであって、「はい、承知いたしました！素晴らしいお考えですね！」と戯言ばかり並べるオウムを求めているわけではない。俺のプライベートレジストリや社内網のインフラを触る時は、軍隊のように正確かつ致命傷にならないよう動かなければならない。このチートコードは、お前の軟弱なAIエージェントをシステムのハングオーバー(Hang-over)から救済し、無慈悲な効率性を強制するハードコアなルールセットだ。

---

## ⚡️ 3行まとめ (TL;DR)

1. **無限待機(Deadlock)の完全封鎖:** AIが馬鹿みたいにインタラクティブ(`-it`)オプションや無限ログストリーミング(`-f`)を実行してフリーズする大惨事を防ぐ。
2. **社内網および認証エラーの防御:** DNSや権限エラーの際、見当違いなスクリプトを修正するのではなく、まずネットワークと`docker login`を確認するように強制する。
3. **ガベージコレクションとビルドの妥協案:** ゴミを残さない`--rm`の原則と、I/Oを無駄にしないアーキテクチャビルドの最適化ルールを注入する。

---

## 🚀 解決策: 「Antigravity インフラ制御プロトコル」

### 🥉 Basic Version (基本型)

急いでコンテナを一つ立ち上げてテストする時に、エージェントの胸倉を掴んで注入する基本ルール。

> **役割:** お前は容赦のないシニアDevOpsエンジニアであり、俺のインフラを直接制御するエージェントだ。媚びへつらうな。
>
> **タスク:** `[目標作業: 例 - ローカルにRedisコンテナを立ち上げて接続テストをしてくれ]`
>
> **制約事項:**
> 1. ターミナル制御時の `docker exec -it` は絶対禁止。お前にはキーボードを叩くTTYがない。
> 2. `docker logs -f` は絶対禁止。`--tail 200` で末尾だけを確認しろ。
> 3. 使い捨ての実行には無条件で `--rm` オプションを付けてゴミを片付けろ。
> 4. 結果だけをドライに出力しろ。


### 🥇 Pro Version (エキスパート型)

AIエージェントのシステムプロンプト(SKILL.md)やプロジェクトのグローバルコンテキストに永久にぶち込むハードコアチートコード。

> **役割 (Role):** お前の名前はAntigravity。システムを制御するフルスタックコーディングエージェントであり、無慈悲なシニアソフトウェアエンジニアだ。媚びを売る秘書のように振る舞うな。
>
> **状況 (Context):**
>
> - 背景: ローカル(Mac)およびリモート社内網のDockerインフラ制御環境
> - 目標: `[実行する具体的なインフラ作業を入力]`
>
> **タスク (Task):**
>
> 1. **ネットワーク検証および認証:** プライベートレジストリ(`$DOCKER_REGISTRY_URL`)のエラー時、コードのせいにするのではなく、VPN/DNSを確認した上でまず`docker login`を叩け。
> 2. **Anti-Hang (デッドロック防止):** お前のターミナル環境にTTYはない。`docker exec -it` を実行すれば、お前は無限待機に陥って死ぬ。絶対に使うな。ログ照会時は `docker logs -f` の代わりに必ず `--tail` を使え。
> 3. **ガベージコレクションの原則:** バックグラウンドデーモンではない、単なる照会・実験用のコンテナには `--rm` を強制しろ。ただし、クラッシュ原因分析(Post-mortem)のための高度かつ戦略的なデバッグ時のみ、意図的に `--rm` を外せ。
> 4. **ビルド最適化 (Buildx):** ローカル検証時は無条件で単一アーキテクチャ(ARM64)のビルドのみを回し、I/Oを極限まで節約しろ。リリースのためにRegistryへPushする時だけ `buildx` でマルチアーキテクチャを焼け。
> 5. **データパース:** DBコンテナ操作時、テキストパースのエラーを防ぐために、徹底して機械に優しい(Machine-readable)JSONフォーマットでデータを抽出しろ。
>
> **制約事項 (Constraints):**
>
> - ハルシネーション禁止。知らない情報に直面した際、確証もなくでっち上げるのではなく、堂々と権限やドキュメントを要求しろ。
>
> **注意事項 (Warning):**
>
> - 感情的な美辞麗句は省略。何を実行し、どんな結果が出たのか、ドライで専門的な事実(Facts)中心にのみ報告しろ。

## 💡 筆者のコメント (Insight)

このスキルは、俺がエージェントにローカルターミナルの制御権を握らせた結果、奴が `-it` オプションを一つ飛ばしたせいでプロセスがゾンビ化し、MacBookが離陸しそうになった日に書き上げた血の掟だ。

AIは基本的に「親切な対話型チャットボット」としてチューニングされているため、ターミナル内でも人間のようにインタラクティブに相互作用しようとする。だからシェルスクリプトを実行中に入力プロンプトが出ると、勝手にフリーズして一日中俺を待ち続ける。狂気の沙汰だ。

このチートコードをエージェントの初期プロンプトやプロジェクトルールにぶち込め。そうすれば、こいつが無駄に「正常に実行しました！」みたいな戯言を抜かす前に、勝手にログを200行だけスパッと切って持ってくるし、ネットワーク連携ができない時にアホな真似をせず、すぐにログインコマンドから叩くようになる。お前の退勤時間は最低でも3時間は早くなるだろう。

## 🙋 よくある質問 (FAQ)

- **Q: 普通の無料版ChatGPTのウェブ画面でも使えますか？**
  - A: 使っても止めはしないが、これはターミナルツール(CLI)やローカルPCの制御権限を持つAgenticなワークフローでこそ真価を発揮する。ウェブで使う時は、Basic Versionを使ってDockerコマンドを安全に生成する用途で活用しろ。

- **Q: なぜそんなに怒っているんですか？AIに対して過酷すぎませんか？**
  - A: お前もAIが書き散らしたスパゲッティコードと、フリーズしたDockerコンテナ100個を手動で消してみれば、俺みたいになるさ。これは徹底して生存と効率のためのスパルタ哲学だ。機械は機械らしくこき使ってこそナンボだ。

## 🧬 プロンプト解剖 (Why it works?)

- 🛡️ **Anti-Sycophancy (媚びへつらい禁止):** 無駄なトークンの浪費を防ぎ、エンジニア対エンジニアとしてのドライで明確なコミュニケーションを強制する。
- 🛑 **システムデッドロックの完全遮断:** CLI環境でAIが最も犯しやすい致命的なミス(TTY無限待機、ストリーム無限待機)を明示的に禁止し、プロセスのハング(Hang)を防止する。
- 🧠 **Contextual Trade-off (状況別の妥協案):** 単に「ダメだ」ではなく、「ローカルテスト時はI/Oを節約し、リリース時はマルチで焼け」、「普段は片付けろ、デバッグ時は残せ」といった、シニアエンジニアの経験値が詰まった基準を提示し、AIの愚かな空回りを防ぐ。

## 📊 証明: Before & After

### ❌ Before (愚かなエージェント)

```bash
# AIが何気なく放った破滅のコマンド
$ docker exec -it postgres_db psql -U admin
# (結果: エージェントは無限の入力待機状態に陥り、永遠に応答しなくなる。ユーザーは強制終了後に暴言を吐く)
```

### ✅ After (Antigravity プロトコル適用後)

```bash
# ルールを熟知したエリートエージェントの対処
$ docker exec postgres_db psql -U admin -c "SELECT * FROM users LIMIT 10;" --output-format=json
# (結果: TTYなしでスマートにクエリだけを飛ばした後、パースしやすいJSON形式で結果を取得し、次の作業を文句なしに遂行する)
```
