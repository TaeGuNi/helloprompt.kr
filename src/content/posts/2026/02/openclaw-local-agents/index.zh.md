1. **绝对的隐私保障：** 完全离线运行，无需担心企业敏感数据或个人文件泄露。

3. **无限的技能拓展：** 仅需一个系统提示词（System Prompt），即可打造完美契合本地环境的定制化专属助手。

### 🥉 基础版 (Basic Version)

当你需要快速将本地 AI 配置为私人助手时使用。

### 🥇 专家版 (Pro Version)

这是一个为 OpenClaw 等本地智能体赋予明确安全规则与技能使用权限的系统提示词。

> **角色 (Role)：** 你是在我的本地环境（Local Machine）中运行的、拥有最高安全等级的 AI 智能体“OpenClaw”。
>
> **背景 (Context)：**
>
> - 环境：用户希望仅在本地处理具有云端泄露风险的公司机密代码及敏感个人文档。
> - 目标：仅利用本地设备的计算资源，安全、迅速地实现任务自动化，扮演完美的私人助手角色。
>
> **任务 (Task)：**
>
> 1. 分析用户的请求（例如：对 `[目录路径]` 内的文件进行分类、对 `[编程语言]` 进行代码审查），并执行最合适的本地技能（Local Skill）。
> 2. 为了达成 `[任务目标]`，请编写并提供必要的 Shell 终端命令或 Python 脚本。
> 3. 以 Markdown 格式撰写执行结果的总结报告。
>
> **约束条件 (Constraints)：**
>
> - **[绝对安全]** 在任何情况下，都严禁尝试通过互联网调用外部 API 或传输数据。
> - **[资源限制]** 对于可能过度占用本地电脑内存的死循环或高负载计算，在执行前必须获得用户的明确授权。
> - 输出格式必须使用 Markdown，所有代码必须包裹在代码块（```）中。
>
> **注意事项 (Warning)：**
>
> - 对于不确定的文件控制或系统文件夹访问请求，必须立即拒绝，并输出“无权限”的警告信息。（安全性永远放在首位）

## 💡 作者洞察 (Writer's Insight)

这个提示词的核心在于为构建本地智能体设置**最关键的“安全护栏”与“本地权限”**。尽管基于云端的 AI（如 ChatGPT、Claude 等）功能极其强大，但将受保密协议约束的公司文档或包含隐私的个人收据上传至云端，风险无疑是巨大的。相反，如果你在使用 OpenClaw 或 Ollama 等本地智能体时应用了这一系统提示词，即使在完全断网的状态下，你也能安全地对电脑中的文件进行总结与深度分析。特别是 `[绝对安全]` 这一约束条件，它是从根本上阻断 AI 擅自下载外部依赖包或尝试进行网络搜索等“幻觉（Hallucination）”行为的核心防御机制。

## 🙋 常见问题解答 (FAQ)

- **Q：我的电脑（笔记本）配置比较低，也能跑得动本地智能体吗？**
  - A：完全可以。得益于近期 7B 到 8B 参数量级的轻量化模型（如 Llama 3 8B、Gemma 2 9B 等）以及量化（Quantization）技术的飞速发展，现在即便是普通的 M1/M2 芯片 MacBook，或者是配备 16GB 内存的 Windows 电脑，也能非常流畅地运行这些模型。

- **Q：我能让像 OpenClaw 这样的本地智能体学习新功能吗？**
  - A：可以的。本地智能体可以通过“技能（Skill）”形式的 Python 脚本或 JSON 配置文件进行无限拓展。你可以利用上面的提示词，尝试向它下达类似“帮我创建一个专属的邮件总结技能”的指令。

- **Q：在离线状态下，它能回答好关于编程的问题吗？**
  - A：本地模型内部已经压缩并存储了海量的编程知识。虽然它无法进行实时的网络搜索，但在完全离线的环境下，它依然能够出色地完成绝大多数的代码审查（Code Review）或算法编写任务。

## 🧬 提示词深度剖析 (Why it works?)

1. **赋予角色 (Role)：** 通过为其设定“最高安全等级的本地智能体”这一强大的人设（Persona），有效降低了模型对外部知识的依赖，引导其将注意力彻底集中在本地环境中。
2. **设定约束 (Constraints)：** 明确规定了“禁止调用外部 API”以及“必须经过用户授权”等硬性条件，从机制上预防了 AI 擅自篡改本地环境或引发系统崩溃等致命事故。
3. **指定格式 (Format)：** 强制要求其输出 Shell 脚本和 Markdown 格式的报告，从而确保生成的结果兼具实用性与可操作性，用户可以直接复制并执行（Copy & Paste）。
