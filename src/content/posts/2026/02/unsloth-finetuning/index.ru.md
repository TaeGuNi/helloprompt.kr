## 💡 Комментарий автора (Insight)

Unsloth по праву называют "революцией в файн-тюнинге" благодаря использованию оптимизированных вручную ядер Triton. Однако новички часто сталкиваются с ошибками OOM из-за неправильной инициализации `FastLanguageModel` или неверных настроек `SFTTrainer` (особенно `gradient_accumulation_steps`).

С помощью этого промпта AI самостоятельно подберет **оптимальные гиперпараметры с учетом лимитов вашей памяти GPU**. Очень часто разработчики забывают опцию `gradient_checkpointing = "unsloth"`, что приводит к сбоям. В Pro-версии промпта мы жестко требуем её использования, тем самым радикально снижая риск ошибки. На практике вам остается лишь адаптировать переменную структуры датасета под корпоративные данные, и скрипт готов к бою — это колоссально повышает продуктивность.

## 🧬 Разбор промпта (Why it works?)

1. **Экспертная персона и специфика библиотеки:** Задав роль "Эксперта по оптимизации Unsloth", мы заставляем AI использовать специализированный синтаксис библиотеки (например, `FastLanguageModel`), игнорируя стандартный, но менее эффективный код HuggingFace.
2. **Четкий конвейер (Pipeline):** Строгая последовательность действий — установка пакетов ➔ загрузка модели ➔ препроцессинг ➔ обучение ➔ сохранение — гарантирует, что ни один критически важный этап не будет пропущен.
3. **Безопасность через ограничения:** Указание "избегать риска OOM" превентивно заставляет AI предлагать консервативные и безопасные значения для batch size и accumulation steps.

```python
from transformers import AutoModelForCausalLM
model = AutoModelForCausalLM.from_pretrained("unsloth/llama-3-8b")
# ... (Опции оптимизации памяти отсутствуют)
```

### ✅ После (Результат применения нашего промпта)

```python
# Безупречный код SFT без ошибок, использующий пакет оптимизации Unsloth.
from unsloth import FastLanguageModel
import torch

# 1. Загрузка модели с 4-битным квантованием и оптимизацией памяти
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name = "unsloth/llama-3-8b-bnb-4bit",
    max_seq_length = 2048,
    dtype = None,
    load_in_4bit = True, # Ключевой метод экономии памяти
)

# 2. Подключение LoRA адаптера для эффективности параметров
model = FastLanguageModel.get_peft_model(
    model,
    r = 16,
    target_modules = ["q_proj", "k_proj", "v_proj", "o_proj",
                      "gate_proj", "up_proj", "down_proj"],
    use_gradient_checkpointing = "unsloth", # Обязательно для длинного контекста и экономии VRAM
    # ... (пропущено)
)
```
