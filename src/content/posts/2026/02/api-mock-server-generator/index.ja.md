## ⚡️ 3行まとめ (TL;DR)

1. **ボトルネックの完全排除:** バックエンドの完成を待つことなく、フロントエンドのUIとロジックの実装を完遂できます。
2. **本番さながらのテスト環境:** ネットワーク遅延（Delay）やエラーステータスをシミュレーションし、ローディングやエラーハンドリングを正確に検証します。
3. **コピペで即起動:** 複雑な環境構築は一切不要。`node server.js`を叩くだけで動作するコードを生成します。

## 🚀 解決策: 「APIモックサーバー生成プロンプト」

### 🥉 Basic Version (基本型)

とりあえず今すぐ、手っ取り早くモックAPIを動かしたい場合にご活用ください。

> **役割 (Role):** あなたは優秀な`[シニアバックエンドエンジニア]`です。
> **タスク (Task):** 私が提供する`[API仕様書]`をもとに、`[Express.js]`環境においてコピペですぐに実行できるモックサーバーの完全なコードを作成してください。

### 🥇 Pro Version (専門家型)

ランダムな遅延や意図的なエラーの発生、大量のダミーデータなど、本番環境の挙動を忠実に再現したい場合に最適なプロンプトです。

> **役割 (Role):** あなたは`[シニアバックエンドエンジニア]`であり、`[テスト環境構築のスペシャリスト]`です。
>
> **状況 (Context):**
>
> - 背景: `[バックエンドAPIの実装が遅延しており、フロントエンドのロジック検証が完全にストップしている状況]`
> - 目標: `[API仕様書をもとに、本番環境の挙動を忠実に再現するモックサーバーを構築すること]`
>
> **タスク (Task):**
>
> 以下のAPI仕様書をもとに、即座に実行可能なモックサーバーの全体コードを作成してください。
>
> **技術スタック:**
>
> - 言語: `[TypeScript / JavaScript]`
> - フレームワーク: `[Express.js / Hono / Next.js Route Handler などから選択]`
>
> **API仕様書:**
>
> 1. GET /users: ユーザー一覧を返す (ページネーション対応、id, name, email フィールドを含む)
> 2. GET /users/:id: 特定ユーザーの詳細情報
> 3. POST /users: 新規ユーザー登録 (バリデーション失敗時の 400 エラーレスポンス例を含む)
> 4. DELETE /users/:id: ユーザー削除 (権限エラー時の 403 レスポンス例を含む)
>
> **必須要件:**
>
> 1. **ダミーデータ:** `faker.js`等を利用するか、極めて現実的なハードコーディングデータを10〜20件自動生成してください。
> 2. **遅延時間のシミュレーション:** 実際のネットワーク環境を想定し、500ms〜1500msのランダムな遅延を追加してください（ローディングUI検証用）。
> 3. **エラーケースの注入:** 約5〜10%の確率で「500 Internal Server Error」を意図的に発生させるロジックを組み込んでください（エラーハンドリングUI検証用）。
> 4. **コード構造:** コピーして即実行（`node server.js` または `ts-node server.ts`）できるよう、ポート3000で動作する単一ファイルにすべてまとめてください。
>
> **制約事項 (Constraints):**
>
> - 出力形式はマークダウンのコードブロックを使用してください。
> - 追加パッケージのインストールが必要な場合は、実行コマンド（例: `npm install ...`）をコード最上部のコメント欄に明記してください。
>
> **注意事項 (Warning):**
>
> - フロントエンド側でCORSエラーに悩まされないよう、CORS設定（`cors`パッケージやヘッダーの設定）を必ず実装に含めてください。

## 💡 筆者コメント (Insight)

このプロンプトを最大限に活かす秘訣は、AIに対して**「ランダム遅延 (Random Delay)」**と**「意図的なエラー (Random Error)」**の実装を明確に要求することです。

ローカルで稼働するモックサーバーは応答速度が速すぎるため、意識しないとスケルトンUI (Skeleton UI) やローディングスピナーが正しくレンダリングされているかを見落としがちです。また、5%の確率で500エラーをランダム発生させる設定にしておけば、エッジケースにおけるエラートースト (Toast) やフォールバックUI (Fallback UI) が正常に機能するかどうかを、事前に完璧に検証できます。

バックエンドチームから「APIの準備ができました！」と連絡が来た際、「もう結合テストもエラー検証も全て終わっていますよ」と余裕たっぷりに返答する圧倒的な快感を、ぜひ味わってみてください。😎

## 🙋 よくある質問 (FAQ)

- **Q: SwaggerのJSONファイルがあるのですが、これをもとに生成できますか？**
  - A: はい、可能です！プロンプト内の「API仕様書」の項目にSwagger (OpenAPI) のJSONテキストをそのまま貼り付けるだけで、より厳密な型定義とレスポンススキーマを備えた高精度なモックサーバーを生成してくれます。

- **Q: POSTやPUTリクエストで追加・修正したデータは保存されますか？**
  - A: 基本的にはメモリ（配列）上で管理されるため、サーバーを再起動するとデータは初期化されます。もしデータの永続化が必要な場合は、プロンプトに*「lowdbやjson-serverを利用して、ローカルファイル (db.json) にデータを保存する仕組みにして」*と追加指示を出してみてください。

- **Q: Next.jsを使用しているのですが、Express以外の環境でも動作しますか？**
  - A: もちろんです！「技術スタック」の項目で`Next.js API Routes (App RouterのRoute Handler)`を指定すれば、Next.jsプロジェクトの内部にそのまま組み込んで使用できるAPIコードを的確に出力してくれます。

## 🧬 プロンプトの解剖 (Why it works?)

1. **Role (役割) とContext (状況) の明確化:** AIに単なるプログラマーではなく「テスト環境構築の専門家」というペルソナを与えることで、フロントエンドエンジニアが直面しがちな罠（CORSエラーやネットワーク遅延の欠如など）をAIが先回りして解決するよう誘導しています。
2. **極めて現実的なデータ生成:** 「test1」のような無意味なダミーデータではなく、`faker.js`などを活用したリアリティのあるデータを要求することで、本番環境と同等レベルの視覚的なUI検証を可能にしています。
3. **エッジケースの強制的シミュレーション:** エラー発生率やランダムな遅延時間を明示的に指示することで、単なる成功時（200 OK）の挙動だけでなく、見落としがちなエラーハンドリングUIまでも強制的にテストできる環境を作り出しています。

## 📊 検証: Before & After

### ❌ Before (従来の手法: ハードコーディング)

```javascript
// フロントエンドのコンポーネント内に意味のないデータを直接記述
const [users, setUsers] = useState([
  { id: 1, name: "test1", email: "test@test.com" },
  { id: 2, name: "test2", email: "test2@test.com" },
]);

// ローディング状態やエラーステータスをテストするには、コードを都度コメントアウトして手動で書き換える必要がある
```

### ✅ After (結果: AIが作成したモックサーバー)

```javascript
// npm install express cors @faker-js/faker
const express = require("express");
const cors = require("cors");
const { faker } = require("@faker-js/faker");

const app = express();
app.use(cors());
app.use(express.json());

// 現実的なダミーデータを15件自動生成
let users = Array.from({ length: 15 }).map(() => ({
  id: faker.string.uuid(),
  name: faker.person.fullName(),
  email: faker.internet.email(),
  createdAt: faker.date.recent(),
}));

// 本番想定: ランダム遅延ミドルウェア (500ms 〜 1500ms の遅延)
const randomDelay = (req, res, next) => {
  const delay = Math.floor(Math.random() * 1000) + 500;
  setTimeout(next, delay);
};

// 本番想定: ランダムエラー発生ミドルウェア (5%の確率で 500 エラー)
const randomError = (req, res, next) => {
  if (Math.random() < 0.05) {
    return res.status(500).json({ error: "Internal Server Error Simulation" });
  }
  next();
};

app.use(randomDelay);
app.use(randomError);

app.get("/users", (req, res) => {
  res.json({ data: users, total: users.length });
});

app.listen(3000, () => {
  console.log("🚀 Mock Server is running on http://localhost:3000");
});
```
