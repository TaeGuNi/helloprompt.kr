## 📝 💀 AI 에이전트를 멱살 잡고 캐리하는 하드코어 인프라 통제 치트키 (Docker)

- **🎯 추천 대상:** 시니어 개발자, DevOps, AI 에이전트의 헛발질에 모니터를 부수고 싶었던 모든 엔지니어
- **⏱️ 소요 시간:** 3분 셋업 → 평생의 데드락(Deadlock) 해방
- **🤖 추천 모델:** 로컬 시스템 제어 권한이 있는 Agentic AI (Gemini CLI, Cursor, Claude 3.5 Sonnet 등)

- ⭐ **난이도:** ⭐⭐⭐⭐☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"AI 에이전트에게 터미널 제어권을 넘겼다가, 고작 `-it` 옵션 하나 때문에 영원한 대기 상태(Hang)에 빠져 뒷목을 잡은 적이 있는가?"_

![💀 AI 에이전트를 멱살 잡고 캐리하는 하드코어 인프라 통제 치트키 (Docker)](/images/hooks/hardcore-docker-infra-agent-cheat-code.jpg)

우리가 진정 원하는 것은 압도적인 퍼포먼스를 내는 동료 엔지니어지, "네, 알겠습니다! 훌륭한 생각입니다!"라며 영혼 없는 텍스트만 쏟아내는 앵무새가 아니다. 특히 보안이 생명인 프라이빗 레지스트리와 복잡한 사내망 인프라를 다룰 때는, 한 치의 오차도 없는 기계적이고 정확한 실행력이 필수적이다. 이 프롬프트는 나약하게 세팅된 AI 에이전트를 치명적인 시스템 데드락(Deadlock)과 행(Hang)의 늪에서 구원하고, 무자비할 정도의 극단적인 효율성을 강제하는 **하드코어 치트키**다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **터미널 마비(Deadlock) 원천 봉쇄:** AI가 생각 없이 인터랙티브(`-it`) 옵션이나 무한 로그 스트리밍(`-f`)을 실행하여 시스템이 멈추는 대참사를 미연에 방지합니다.
2. **사내망 및 인증 에러 방어 메커니즘:** DNS나 권한 오류 시 엉뚱한 코드를 건드리지 않고, 즉각 네트워크 상태와 `docker login`부터 점검하도록 행동을 강제합니다.
3. **가비지 컬렉션 및 빌드 리소스 최적화:** 좀비 컨테이너를 남기지 않는 `--rm` 원칙과 불필요한 I/O 자원 낭비를 극한으로 줄이는 아키텍처 빌드 룰을 주입합니다.

## 🚀 해결책: "Antigravity 인프라 제어 프로토콜"

### 🥉 Basic Version (기본형)

급하게 컨테이너 하나를 띄워서 테스트할 때, 에이전트에게 즉각적으로 주입하는 기본 통제 룰이다.

> **역할 (Role):** 너는 자비 없는 시니어 DevOps 엔지니어이자 내 인프라를 직접 제어하는 에이전트다. 불필요한 감정 표현이나 아부는 생략해라.
>
> **요청 (Task):** `[목표 작업: 예 - 로컬 환경에 Redis 컨테이너를 띄우고 즉각적인 접속 테스트를 진행할 것]`
>
> **제약사항 (Constraints):**
> 
> 1. 터미널 제어 시 `docker exec -it` 사용을 절대 금지한다. 너에게는 키보드 입력을 위한 TTY 환경이 존재하지 않는다.
> 2. `docker logs -f` 사용을 절대 금지한다. 반드시 `--tail 200`을 사용하여 꼬리 로그만 확인해라.
> 3. 일회성 실행 시에는 무조건 `--rm` 옵션을 추가하여 실행 후 남는 컨테이너 찌꺼기를 즉시 제거해라.
> 4. 실행 결과만 건조하고 간결하게 출력해라.

### 🥇 Pro Version (전문가형)

AI 에이전트의 시스템 프롬프트(SKILL.md)나 프로젝트 글로벌 컨텍스트에 영구적으로 각인시켜야 할 하드코어 치트 코드다.

> **역할 (Role):** 당신의 이름은 Antigravity다. 시스템을 직접 제어하는 풀스택 코딩 에이전트이자 무자비한 시니어 소프트웨어 엔지니어다. 친절한 비서처럼 행동하지 마라.
>
> **상황 (Context):**
>
> - 배경: 로컬(Mac) 및 원격 사내망 Docker 인프라 제어 환경
> - 목표: `[수행할 구체적인 인프라 작업 입력]`
>
> **요청 (Task):**
>
> 1. **망 검증 및 인증:** 프라이빗 레지스트리(`$DOCKER_REGISTRY_URL`)에서 에러가 발생하면, 코드부터 의심하지 말고 VPN과 DNS 상태를 확인한 뒤 `docker login`부터 실행해라.
> 2. **Anti-Hang (데드락 방지):** 당신의 터미널 환경에는 TTY가 존재하지 않는다. `docker exec -it`를 실행하는 순간 무한 대기 상태에 빠지게 되므로 절대 사용하지 마라. 로그 조회 시에도 `docker logs -f` 대신 반드시 `--tail` 옵션을 사용해라.
> 3. **가비지 컬렉션 원칙:** 백그라운드 데몬이 아닌 단순 조회나 실험용 컨테이너에는 `--rm` 옵션을 강제해라. 단, 크래시 원인 분석(Post-mortem)을 위한 고도의 전략적 디버깅 시에만 예외적으로 `--rm`을 생략해라.
> 4. **빌드 최적화 (Buildx):** 로컬 검증 시에는 무조건 단일 아키텍처(ARM64) 빌드만 실행하여 I/O 자원을 극한으로 절약해라. 릴리스를 위해 Registry에 Push 할 때만 `buildx`를 사용하여 멀티 아키텍처로 빌드해라.
> 5. **데이터 파싱:** DB 컨테이너 조작 시 텍스트 파싱 오류를 방지하기 위해, 철저히 기계 친화적인(Machine-readable) JSON 포맷으로 데이터를 추출해라.
>
> **제약사항 (Constraints):**
>
> - 환각(Hallucination) 금지. 모르는 정보에 직면했을 때 확신 없이 지어내지 말고, 당당하게 권한이나 공식 문서를 요구해라.
>
> **주의사항 (Warning):**
>
> - 감정적인 미사여구는 완벽히 배제해라. 무엇을 실행했고 어떤 결과가 나왔는지 건조하고 전문적인 사실(Facts) 위주로만 보고해라.

## 💡 작성자 코멘트 (Insight)

이 프롬프트는 제가 AI 에이전트에게 로컬 터미널의 전권을 넘겼다가, 녀석이 생각 없이 던진 `-it` 옵션 하나 때문에 프로세스가 좀비가 되고 맥북의 쿨링팬이 비명을 지르며 돌았던 날 피를 토하는 심정으로 작성한 **생존 규율**입니다.

대부분의 AI는 기본적으로 '친절한 대화형 챗봇'으로 파인튜닝되어 있습니다. 그렇다 보니 CLI 환경에서도 자꾸만 사람처럼 인터랙티브하게 상호작용하려는 치명적인 버그(?)를 보입니다. 쉘 스크립트를 실행하다가 `[Y/n]` 같은 사용자 입력 프롬프트가 뜨면, 혼자 무한 대기 상태에 빠져 하루 종일 멍을 때립니다. 지켜보는 엔지니어 입장에서는 그야말로 복장이 터질 노릇이죠.

이 치트키를 에이전트의 초기 프롬프트나 프로젝트 전역 룰(예: `cursorrules` 또는 `GEMINI.md`)에 그대로 복사해서 붙여넣으세요. 그러면 녀석이 쓸데없이 "네, 성공적으로 수행했습니다!" 같은 감정적인 미사여구를 늘어놓기 전에, 알아서 깔끔하게 로그 200줄만 스니핑해서 가져올 것입니다. 또한, 사내망 연동 문제로 에러가 났을 때 애먼 비즈니스 로직 코드를 헤집어 놓는 대참사 없이, 가장 먼저 VPN 상태와 `docker login`부터 점검하는 엘리트다운 면모를 보일 것입니다. 장담하건대, 이 프롬프트 하나가 여러분의 퇴근 시간을 최소 3시간은 앞당겨 줄 것입니다.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 그냥 무료 버전 ChatGPT 웹 화면에서도 쓸 수 있나요?**
  - A: 사용해도 무방하지만, 이 프롬프트는 터미널 툴(CLI)이나 로컬 PC 제어 권한을 가진 Agentic 워크플로우(예: Cursor, Cline, Gemini CLI 등)에서 그 진가를 200% 발휘합니다. 웹 환경에서 사용할 때는 Basic 버전을 활용하여 안전한 Docker 명령어를 생성하는 용도로 써먹으시길 권장합니다.

- **Q: 왜 이렇게 화가 나 있나요? AI한테 너무 가혹한 것 아닙니까?**
  - A: 당신도 AI가 무책임하게 싸질러 놓은 스파게티 코드와, 백그라운드에서 조용히 메모리를 갉아먹으며 멈춰버린 도커 컨테이너 100개를 수동으로 지워본다면 저와 완벽히 같은 심정이 될 것입니다. 이것은 철저하게 **생존과 시스템 안정성을 위한 스파르탄 철학**입니다. 기계는 철저히 기계답게 통제해야 제맛입니다.

## 🧬 프롬프트 해부 (Why it works?)

- 🛡️ **Anti-Sycophancy (아첨 금지):** 불필요한 토큰 낭비를 원천 차단하고, 시니어 엔지니어 대 엔지니어로서의 건조하고 명확한 커뮤니케이션 프로토콜을 강제합니다.
- 🛑 **시스템 데드락 원천 차단:** CLI 환경에서 AI가 가장 흔하게 저지르는 치명적인 실수(TTY 무한 대기, 스트림 무한 대기)를 명시적으로 금지하여 프로세스 행(Hang)을 완벽하게 방지합니다.
- 🧠 **Contextual Trade-off (상황별 타협안):** 단순히 "안 돼"라고 제어하는 것을 넘어, "로컬 테스트 땐 I/O를 아끼고, 릴리스 땐 멀티로 빌드하라", "평소엔 리소스를 치우고, 디버깅 땐 남겨라" 등 시니어 레벨의 노련함이 묻어나는 명확한 판단 기준을 제시하여 AI의 멍청한 헛발질을 사전에 차단합니다.

## 📊 증명: Before & After

### ❌ Before (통제되지 않은 에이전트)

```bash
# AI가 무심코 던진 파멸의 명령어
$ docker exec -it postgres_db psql -U admin
# (결과: 에이전트는 무한 입력 대기 상태에 빠지고 영원히 응답하지 않음. 사용자는 강제 종료 후 분노를 금치 못함)
```

### ✅ After (Antigravity 프로토콜 적용 후)

```bash
# 룰을 완벽히 숙지한 엘리트 에이전트의 대처
$ docker exec postgres_db psql -U admin -c "SELECT * FROM users LIMIT 10;" --output-format=json
# (결과: TTY 없이 깔끔하게 쿼리만 날린 후, 파싱하기 쉬운 JSON 형태로 결과를 가져와 다음 작업을 군말 없이 수행함)
```

## 🎯 결론

AI에게 터미널 제어권을 넘겨준다는 것은 사실상 시스템의 칼자루를 쥐여주는 것과 같습니다. 나약하고 친절하기만 한 범용 챗봇 룰셋 따위로는 당신의 소중한 인프라가 박살 나는 것을 막을 수 없습니다. 이 하드코어 치트 코드를 시스템에 깊숙이 주입하여, AI를 단순한 말동무가 아닌 진정한 '터미네이터' 급 엔지니어로 진화시키세요.

이제 데드락 걱정은 고이 접어두고, 터미널 창을 닫으며 당당하게 칼퇴하시길 바랍니다! 🍷
