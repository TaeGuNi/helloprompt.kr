# ✍️ 작가 가이드 (Writer's Guide)

> **"Great Prompts change the way we work."**

Hello Prompt의 작가(Contributor)가 된 것을 환영합니다.
우리는 단순한 '정보'가 아닌, **'독자의 시간을 아껴주는 도구'**를 만듭니다.
이 문서는 당신이 훌륭한 프롬프트 엔지니어이자 작가로 성장할 수 있도록 돕는 나침반입니다.

---

## 🧭 우리의 철학 (Philosophy)

글을 쓰기 전, 다음 세 가지를 마음에 새겨주세요.

1.  **User Obsession (독자 집착):** 독자는 바쁩니다. 서론으로 시간을 낭비하지 마세요. 가장 빠른 해결책을 먼저 제시하세요.
2.  **Prompt is Product (프롬프트는 제품이다):** 우리가 제공하는 프롬프트는 '소프트웨어'와 같습니다. 버그(오작동)가 없어야 하며, 누구나 쉽게 사용할 수 있어야 합니다.
3.  **Global Standard (글로벌 표준):** 당신의 글은 전 세계 9개국 언어로 번역됩니다. 보편적인 가치를 담되, 현지화가 용이하도록 명확하게 작성하세요.

---

## 🔄 콘텐츠 생산 파이프라인 (Workflow)

1.  **Mining (발굴):** "사람들이 검색하지만, 좋은 답이 없는 질문"을 찾습니다.
2.  **Engineering (설계):** 문제를 해결하는 최적의 프롬프트를 설계하고 테스트합니다.
3.  **Drafting (집필):** `POST_TEMPLATE.md`에 맞춰 글을 작성합니다.
4.  **Auditing (심사):** `QUALITY_MODEL.md` 기준으로 500점 만점 테스트를 수행합니다.
5.  **Localization (번역):** 한국어 원본 확정 후, 9개국어로 확장합니다.

## 1. 주제 선정 (Mining)

> _"좋은 주제는 '구체적인 고통'에서 나옵니다."_

- **Good:** "엑셀 피벗 테이블에서 날짜 형식이 깨질 때 해결법" (구체적, 고통이 명확함)
- **Bad:** "엑셀 잘하는 법" (너무 광범위함), "AI의 미래" (실용성 없음)
- **Tool:** Brave Search, Google Trends, 실무자 인터뷰

## 2. 프롬프트 엔지니어링 (The Core Craft)

우리 블로그의 핵심 경쟁력은 **'검증된 고품질 프롬프트'**입니다.
대충 쓴 프롬프트는 절대 통과될 수 없습니다. 다음 **RTFC 프레임워크**를 따르세요.

### 🏗️ RTFC Framework

1.  **R (Role - 역할):** AI에게 페르소나를 부여하세요.
    - _"너는 10년 차 시니어 카피라이터야."_
2.  **T (Task - 태스크):** 해야 할 일을 동사로 명확히 정의하세요.
    - _"이 제품의 상세 페이지를 보고, 구매 욕구를 자극하는 인스타그램 광고 문구 3개를 작성해."_
3.  **F (Format - 형식):** 결과물의 형태를 지정하세요.
    - _"결과는 마크다운 표(Table) 형식으로 출력해."_
4.  **C (Context & Constraints - 맥락 및 제약):**
    - _"타겟 고객은 2030 직장인 여성이고, 톤앤매너는 발랄하게 해줘. 단, 이모지는 과하게 쓰지 마."_

### 🧪 테스트 필수

작성한 프롬프트는 반드시 **최소 3개 모델(GPT-4o, Claude 3.5 Sonnet, Gemini Pro)**에서 테스트해봐야 합니다.

## 3. 글쓰기 가이드 (Drafting)

### 구조 (Structure)

- **Hook (도입부):** 독자의 문제 상황을 공감하고(Pain point), 해결책을 예고하세요.
- **Solution (해결책):** 프롬프트를 Basic과 Pro 버전으로 나누어 제공하세요.
- **Proof (증명):** Before & After 예시를 통해 효과를 시각적으로 증명하세요.

### 톤앤매너 (Tone & Manner) : 친절한 사수 (Friendly Expert)

우리는 '교수님'이 아니라, 옆자리에 앉은 '일 잘하는 선배'입니다.

| ❌ 이렇게 쓰지 마세요                                                | ✅ 이렇게 쓰세요                                                 |
| :------------------------------------------------------------------- | :--------------------------------------------------------------- |
| "본 프롬프트는 LLM의 할루시네이션을 최소화하기 위해 설계되었습니다." | "이 프롬프트는 AI가 거짓말을 하지 않도록 안전장치를 걸어뒀어요." |
| "작성해라." / "작성하세요." (너무 딱딱하거나 밋밋함)                 | "작성해 보세요!" / "확인해 볼까요?" (권유형, 청유형)             |
| "존나 좋음" (비속어 금지)                                            | "정말 강력합니다." / "놀라운 효과가 있습니다."                   |

### 시각화 (Formatting)

- **중요한 문장:** **볼드체**로 강조합니다.
- **핵심 키워드:** `인라인 코드`로 감싸서 눈에 띄게 합니다.
- **가독성:** 문단은 3~4줄을 넘기지 않도록 끊어주세요.

## 4. 품질 심사 (Auditing)

글을 다 썼다면, `docs/QUALITY_MODEL.md`를 펴고 냉정하게 자가 점검을 하세요.

- [ ] **적합성:** 우리 블로그에 어울리는 주제인가?
- [ ] **유용성:** 당장 복사해서 쓸 수 있는가?
- [ ] **독창성:** 뻔한 내용은 아닌가? 나만의 인사이트가 있는가?
- [ ] **희소성:** 구글링해도 잘 안 나오는 꿀팁인가?
- [ ] **진실성:** 거짓 정보나 과장은 없는가?

> **Tip:** 하나라도 '아니오'가 있다면 발행 버튼을 누르지 마세요.

---

## 5. 다국어 현지화 (Localization)

우리는 번역기(Translation)가 아닌, **현지화(Localization)**를 지향합니다.

- **문화적 적응 (Cultural Adaptation):**
  - 한국의 '추석' → 미국의 'Thanksgiving', 중국의 '춘절'
  - '카카오톡' → 'WhatsApp' 또는 'Line'
- **예시 데이터 변경:** 이름, 지명, 통화 단위(KRW → USD/EUR)를 현지에 맞게 수정하세요.

---

## 🚀 배포 전 최종 체크리스트 (Final Checklist)

1. **파일명:** 영문 소문자 `kebab-case` 인가? (예: `excel-pivot-table.md`)
2. **프론트매터:** 날짜(ISO 8601), 태그, 설명이 정확한가?
3. **이미지:** `src/assets/images/` 경로에 있고, 용량이 최적화되었는가?
4. **테스트:** `pnpm check` 및 `pnpm lint`를 통과했는가?
