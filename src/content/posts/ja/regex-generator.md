_「メールアドレスの有効性チェック一つするために `^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$` これを解読していますか？」_

正規表現 (Regular Expression) は開発者にとって必須ですが、最も扱いたくない領域の一つです。書くたびに文法を調べなければならず、人が書いたものは宇宙語のようです。もう苦しまずにAIに「パターン」だけ話してください。

---

## ⚡️ 3行要約 (TL;DR)

1. 複雑な正規表現の文法を覚えたり検索するのに苦労しないでください。
2. 望むパターン（電話番号、メールなど）を言うだけでAIが正規式を書いてくれ、解釈までしてくれます。
3. テストコードまで一緒にリクエストして、正規式の正確性をすぐに検証できます。

> - 010, 011, 016 などで開始可能。

## 🧬 プロンプト解剖 (Why it works?)

---

## 📊 証明: Before & After

### ❌ Before (入力)

```text
日本の携帯電話番号の正規式書いて。ハイフンありなし両方。010以外に011もOK。
```

<br>

### ✅ After (結果)

```javascript
// JavaScript 例

const regex = /^01([0|1|6|7|8|9])-?([0-9]{3,4})-?([0-9]{4})$/;

// テスト
const phone1 = "010-1234-5678"; // マッチ O
const phone2 = "01012345678"; // マッチ O
const phone3 = "02-123-4567"; // マッチ X

console.log(regex.test(phone1)); // true
```

### 🔍 解説

- `^`: 文字列の始まり
- `01([0|1|6|7|8|9])`: 01で始まり、その後ろに0,1,6,7,8,9のいずれかが来る（識別番号）
- `-?`: ハイフン(-)が0個または1個ある（選択的）
- `([0-9]{3,4})`: 数字0-9が3個または4個来る（真ん中の桁）
- `([0-9]{4})`: 数字0-9が4個来る（終わりの桁）
- `$`: 文字列の終わり

## 🎯 結論

正規表現の文法書を開かないでください。あなたが望む「パターン」が何なのかだけ明確に知っていれば良いのです。
