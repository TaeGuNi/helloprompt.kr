> _"Заметили, что Claude начал лениться, пропускать важные куски кода или выдавать раздражающее '// ...rest of the code...'? Вы не одиноки, но мы знаем, как вернуть ему былую гениальность."_

В последнее время сообщества на Reddit и X (Twitter) переполнены жалобами: некогда блестящий Claude Code стал давать обрывочные ответы и халтурить на сложных задачах. Действительно ли модель «отупела», или это агрессивная оптимизация ресурсов со стороны Anthropic? Независимо от причин, мы, как инженеры, не можем зависеть от капризов нейросетей. В этой статье мы разберем суть проблемы и дадим вам «анти-лень» промпт, который заставит Claude работать на 100%.

## ⚡️ 3 тезиса коротко (TL;DR)

1. **Жалобы обоснованы:** Разработчики массово сталкиваются с «ленивым кодом» и плейсхолдерами из-за внутренних оптимизаций провайдера.
2. **Контекст решает всё:** Отсутствие жестких рамок в запросе позволяет модели экономить токены на вашем коде.
3. **Строгие ограничения (Constraints) — ваш лучший друг:** Добавление явных запретов в промпт полностью устраняет проблему усеченного кода.

## 🚀 Решение: Промпт "Анти-Лень" для Claude

### 🥉 Базовая версия (Basic Version)

Используйте для небольших правок и быстрых скриптов, когда нужен немедленный результат без сокращений.

> **Роль:** Ты `[Senior-разработчик]`.
> **Запрос:** Напиши полный код для `[описание задачи]`, не пропуская ни одной строки. Тебе категорически запрещено использовать комментарии-заглушки вида "// остальной код здесь".

\

### 🥇 Продвинутая версия (Pro Version)

Используйте для сложных архитектурных изменений, создания новых компонентов или рефакторинга крупных файлов.

> **Роль (Role):** Ты `[Ведущий Software Engineer]`, специализирующийся на `[ваша технология, например: React и Node.js]`.
>
> **Контекст (Context):**
>
> - Ситуация: Мне нужно получить полностью рабочий и детализированный код без каких-либо сокращений.
> - Цель: Написать `[абсолютно полный, готовый к продакшену файл/модуль]`.
>
> **Запрос (Task):**
>
> 1. Полностью реализуй `[название модуля или функционала]`.
> 2. Включи всю необходимую бизнес-логику, обработку краевых случаев (edge cases) и безопасную типизацию.
> 3. Детально пропиши функцию `[ключевая часть логики, которую нельзя упустить]`.
>
> **Ограничения (Constraints):**
>
> - Ответ должен содержать исключительно код в формате Markdown.
> - КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО использовать плейсхолдеры, такие как `// TODO`, `// ...existing code...` или `// реализация здесь`.
> - Код должен быть полностью готов к копированию и вставке (Copy-Paste Ready).
>
> **Предупреждение (Warning):**
>
> - Если код физически не помещается в лимит токенов одного ответа, остановись на логически завершенном месте и спроси "Продолжить генерацию?", но ни в коем случае не сокращай саму логику в угоду лимитам.

## 💡 Авторский комментарий (Insight)

Феномен деградации моделей (AI Slop или «лень ИИ») — это почти всегда результат скрытого снижения лимитов на генерацию (max_tokens) или агрессивного системного промптинга со стороны разработчиков ИИ для экономии серверных мощностей. Как инженеры, мы должны понимать: нейросеть выберет путь наименьшего сопротивления, если мы ей это позволим.

В моей практике добавление блока **Ограничения (Constraints)** работает как своеобразный джейлбрейк для продуктивности. Claude гораздо лучше реагирует на жесткие запретительные формулировки («КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО»), чем на вежливые просьбы («пожалуйста, напиши код полностью»). Этот подход экономит часы ручного сшивания кусков кода.

## 🙋 Часто задаваемые вопросы (FAQ)

- **Q: Можно ли использовать этот промпт в API Claude, а не в веб-интерфейсе?**
  - A: Да, и это лучший способ! Если вы работаете через API или используете Cursor/Copilot, добавьте блок с ограничениями прямо в `System Prompt`. Это сделает поведение Claude стабильным на протяжении всего сеанса.

- **Q: Поможет ли это с ChatGPT (GPT-4o)?**
  - A: Абсолютно. Модели OpenAI также склонны к "лени" в часы пиковых нагрузок. Структура промпта универсальна и отлично работает с любыми современными LLM.

## 🧬 Анатомия промпта (Why it works?)

1. **Управление поведением через запреты (Constraints):** Прямой запрет на использование плейсхолдеров закрывает лазейку для экономии токенов.
2. **Задание высоких стандартов (Role):** Роль «Ведущего инженера» заставляет модель подтягивать качество архитектуры и не пренебрегать деталями.
3. **Легальный выход из ситуации (Warning):** Разрешая модели спросить "Продолжить?", мы избегаем молчаливого обрыва кода на полуслове, что позволяет получить весь скрипт частями, но без потерь в логике.

### ✅ После (С использованием нашего промпта)

````text
Пользователь: [Продвинутая версия промпта]

Claude:
```javascript
import express from 'express';
import bcrypt from 'bcrypt';
import { body, validationResult } from 'express-validator';
import { User } from '../models/User.js'; // Предполагаемая модель БД

const router = express.Router();

router.post('/register',
  [
    body('email').isEmail().withMessage('Неверный формат email'),
    body('password').isLength({ min: 8 }).withMessage('Пароль должен содержать минимум 8 символов'),
    body('username').notEmpty().withMessage('Имя пользователя обязательно')
  ],
  async (req, res) => {
    try {
      const errors = validationResult(req);
      if (!errors.isEmpty()) {
        return res.status(400).json({ errors: errors.array() });
      }

      const { email, password, username } = req.body;

      const existingUser = await User.findOne({ email });
      if (existingUser) {
        return res.status(409).json({ message: 'Пользователь с таким email уже существует' });
      }

      const saltRounds = 12;
      const hashedPassword = await bcrypt.hash(password, saltRounds);

      const newUser = new User({
        email,
        username,
        password: hashedPassword
      });

      await newUser.save();

      // Исключаем хэш пароля из ответа
      const userResponse = { id: newUser._id, email: newUser.email, username: newUser.username };

      return res.status(201).json({
        message: 'Регистрация прошла успешно',
        user: userResponse
      });

    } catch (error) {
      console.error('Ошибка при регистрации:', error);
      return res.status(500).json({ message: 'Внутренняя ошибка сервера при регистрации пользователя' });
    }
  }
);

export default router;
````
