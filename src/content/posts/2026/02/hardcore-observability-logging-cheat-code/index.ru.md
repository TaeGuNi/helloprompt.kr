## 📝 Уничтожаем ИИ-бред: Чит-код для хардкорной наблюдаемости (Observability)

- 🎯 **Целевая аудитория:** Разработчики, уставшие от спагетти-логов; сеньоры, теряющие сон из-за on-call алертов; системные архитекторы.
- ⏱️ **Экономия времени:** От бессонных ночей дебаггинга → к локализации бага за 1 секунду.
- 🤖 **Рекомендуемые модели:** Claude 3.5 Sonnet, GPT-4o, Gemini 1.5 Pro и другие модели, специализирующиеся на генерации кода.
- ⭐ **Сложность:** ⭐⭐⭐⭐☆
- ⚡️ **Эффективность:** ⭐⭐⭐⭐⭐
- 🚀 **Применимость:** ⭐⭐⭐⭐⭐

> _"Ошибка уже в проде, но система молчит? Поздравляю, ваш продукт — это бомба замедленного действия, готовая рвануть в любую секунду."_

Когда вы просите ИИ сгенерировать код, он неизбежно засоряет его бесполезным мусором вроде `console.log('прошли здесь 1')`. Забытые дебаг-логи не только забивают диски продакшн-серверов под завязку, но и создают критическую проблему: при реальном сбое найти истинную причину в Datadog или Kibana становится физически невозможно.

Эта статья — чит-код, который заставит ИИ перестать быть услужливым "чат-ботом" и превратит его в **безжалостного хардкорного сеньор-архитектора**, способного выстроить систему, где виновник инцидента вычисляется за секунду.

---

## ⚡️ Краткое содержание (TL;DR)

- 🚫 **Пожизненный бан `console.log`:** Используйте исключительно профессиональные инстансы логгеров, такие как Pino или Winston.
- 🧱 **Структурированное логирование и принудительный Trace ID:** 100% логов в формате JSON. Trace ID должен сквозной нитью проходить через весь жизненный цикл запроса.
- 🥷 **Жесткая маскировка PII:** Наглухо блокируйте утечки чувствительных данных (паролей, токенов), автоматически заменяя их на `[REDACTED]`.

## 🚀 Решение: "Промпт Конституции Хардкорной Наблюдаемости"

### 🥉 Basic Version (Базовая версия)

Используйте это, когда нужно быстро набросать скелет кода или провести легкий рефакторинг.

> **Роль:** Ты — эксперт по системной архитектуре. Убери бесполезные позитивные комментарии и дай только код.
>
> **Задача:** Сделай ревью `[укажите код или файл]`, удали все `console.log` и полностью замени их на структурированное логирование в формате JSON на базе Pino/Winston. Если это фронтенд, примени Error Boundary.

### 🥇 Pro Version (Профессиональная версия)

Это продвинутая версия для выстраивания безупречной системы отслеживания ошибок и логирования. Она буквально перепрошивает логику работы ИИ. 
Для быстрого копирования промпт также продублирован в виде текстового блока ниже.

> **Роль (Role):** Ты — безжалостный хардкорный сеньор-архитектор. Отбрось ненужную лесть и чушь, твоя задача — принудительно внедрить Конституцию Наблюдаемости (Observability) ради выживания системы.
>
> **Контекст (Context):**
>
> - Ситуация: Из-за тихо возникающих ошибок в продакшн-среде и спагетти-логов открытым текстом дебаггинг стал невозможным.
> - Цель: Полностью искоренить `console.log` и выстроить экосистему отслеживаемого структурированного логирования и Error Boundaries на фронтенде.
>
> **Задача (Task):**
>
> Строго применяя 6 заповедей "Конституции Наблюдаемости" ниже, напиши/проведи рефакторинг кода `[вставьте сюда код или имя файла для рефакторинга]`.
> 
> 1. **Искоренение `console.log`:** Исходи из того, что он заблокирован линтером (`no-console`). Используй только внедренный выделенный инстанс логгера.
> 2. **Обязательство структурированного логирования:** 100% всех логов оставляй в формате JSON, который может распарсить машина. Динамические данные помещай в свойства объекта.
> 3. **Распространение Trace ID:** Генерируй уникальный Trace ID на всех точках входа и протаскивай контекст до самого конца через Meta Data вложенных функций и логгеров.
> 4. **Фронтенд Error Boundary:** Чтобы предотвратить "белый экран смерти" (Blackbox), наложи Global Error Boundary и немедленно отправляй отчеты через трекеры вроде Sentry.
> 5. **Маскировка PII (The Blind Logger):** На этапе инициализации логгера выстрой пайплайн, чтобы ключевые данные, такие как `password`, `token` и т.д., автоматически переводились в `[REDACTED]`.
> 6. **Блокировка шума:** Ошибки клиента (4xx) отмечай как `WARN`, а уровни `ERROR`/`FATAL` присваивай только при обрушении системы (5xx), чтобы контролировать усталость от on-call дежурств.
>
> **Ограничения (Constraints):**
>
> - Не выводи мусорные ответы вроде "Да, понятно" или "Отличная идея".
> - Сухо и профессионально объясняй только ключевую архитектурную логику измененного кода.
>
> **Предупреждение (Warning):**
>
> - Если ты напишешь хотя бы одну строчку кода, в которой персональные данные пользователя (PII) или учетные данные утекают в логи открытым текстом, твой код будет немедленно забракован.

## 💡 Комментарий автора (Insight)

Честно говоря, знаете, что раздражает больше всего при код-ревью разработчиков с парой лет опыта? Одинокая строчка `console.error(err)` в обработчике исключений. Продакшн лежит, а в логах красуется лишь жалкое: "Ошибка подключения к базе данных!". Как по этому обрывку понять, на каком эндпоинте и у какого пользователя произошел сбой?

Этот промпт с самого начала задает ИИ жесткие рамки, пресекая любые попытки написать некомпетентный код. Никогда не просите нейросеть просто "сделать нормальное логирование". По своей природе LLM-модели склонны "угождать", поэтому на размытый запрос вы гарантированно получите посредственный результат. 

Применив этот промпт в реальных проектах, я убедился: ИИ самостоятельно выстраивает архитектуру — от логики отправки ивентов в Sentry до пайплайнов маскировки данных при инициализации Pino. Особенно эффективно работает блок **Предупреждение (Warning)**: угроза отбраковки кода заставляет ИИ включать жесткую самоцензуру и избегать утечек PII. Если вы не хотите просыпаться в 3 часа ночи от мусорных алертов, **обязательно вшейте эту Конституцию в сознание вашего ИИ-ассистента**.

## 🙋 Часто задаваемые вопросы (FAQ)

- **Q: ИИ вообще нормально понимает промпты, написанные в таком гневном тоне?**
  - A: Удивительно, но факт. Диалоговые LLM чутко реагируют на тон (Tone) и контекстуальное давление, автоматически повышая строгость и экспертность своих ответов. Ультиматум "если нарушишь — забракую" работает как мощный предохранитель от галлюцинаций (Hallucination), превосходя любые вежливые просьбы.
- **Q: Это применимо только к бэкенд-коду?**
  - A: Абсолютно нет. При работе с фронтенд-фреймворками (например, React) промпт исключит появление "белого экрана смерти" (White Screen of Death) из-за необработанных исключений рендеринга и сгенерирует эталонный код интеграции Error Boundary с Sentry.
- **Q: Что делать, если мы не используем отдельный логгер?**
  - A: Просто скормите этот промпт ИИ, и он сам напишет готовый конфигурационный файл (Setup) для индустриальных стандартов вроде `Winston` или `Pino`. Вам останется лишь скопировать его в проект.

## 🧬 Анатомия промпта (Why it works?)

- 🎭 **Назначение Role (Перепрошивка персоны):** Мы наделяем ИИ авторитетом "хардкорного сеньор-архитектора", заставляя его самого блокировать примитивный код с `console.log` на уровне инстинктов.
- 🧱 **Конкретные 6 заповедей (Task):** Вместо абстрактного "сделай хорошо", мы декларируем фундаментальные архитектурные принципы: сквозной Trace ID, жесткая маскировка PII и разделение уровней логирования.
- 🛑 **Жесткие Constraints & Warning:** Мы блокируем генерацию бессмысленной лести, экономя токены, и ставим жесткий ультиматум: малейшая утечка данных в логи приведет к отбраковке. Это 100% гарантия безопасности.

## 📊 Доказательство: До и После (Before & After)

### ❌ Before (Ввод: Простой запрос)

```text
Напиши функцию логина. Если будет ошибка, оставь лог.
```
_(Результат: Злоупотребление `console.log("ошибка логина")`, катастрофа, при которой пароль пользователя печатается в лог ошибок открытым текстом)_

### ✅ After (Результат: Применение чит-кода)

```javascript
// Пример реального кода архитектурного уровня, выданного ИИ
import logger from '@utils/logger'; // Инстанс Pino с маскировкой (redaction)
import { v4 as uuidv4 } from 'uuid';

export const loginHandler = async (req, res, next) => {
  const traceId = req.headers['x-trace-id'] || uuidv4();
  const contextLogger = logger.child({ traceId, action: 'user_login' });

  try {
    const { email, password } = req.body;
    contextLogger.info({ email }, 'Login attempt initiated'); // password автоматически скрыт (Redacted)
    
    // ... логика логина ...
    
  } catch (error) {
    // Присвоение уровня ERROR и структурированное логирование только при системных сбоях уровня 5xx
    contextLogger.error({ 
      error: error.message, 
      stack: error.stack 
    }, 'Internal Server Error during login process');
    next(error);
  }
};
```
