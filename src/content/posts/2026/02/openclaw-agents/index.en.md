## 💡 Writer's Comment (Insight & How to use)

The reason this prompt delivers such overwhelming performance is that it perfectly applies the essence of prompt engineering: **'Persona Constraint'** and **'System Framework Enforcement'** simultaneously. This master prompt is thoroughly specialized in building a rock-solid foundation for **'Multi-Agent Collaboration'** and **'Secure Tool Execution'**—the most powerful weapons of OpenClaw 2026.

When you try to build an autonomous agent system from scratch in the field, you frequently face issues where the system falls into infinite loops or malfunctions with critical APIs if you don't strictly control role distribution and tool access at the initial stage. In particular, the process of defining the `[Main Agent Role]` and the `[Clear R&R of Sub-Agents]` is the most critical variable determining the success or failure of the entire architecture. There is a world of difference between telling an AI to "analyze code" and telling it, "You are a static analysis agent that verifies syntax errors, and your output must be passed to the performance optimization agent in JSON format." This prompt forces the LLM to rigorously establish these **'role granularities and communication protocols.'**

Furthermore, the **'Least Privilege'** and **'Pre-execution Validation'** principles specified in the prompt are essential safeguards that must be followed when safely introducing autonomous agents in enterprise environments. When an agent uses sensitive or destructive tools—such as modifying external databases or calling payment APIs—the system designed by this prompt thoroughly validates a predefined Whitelist and automatically builds a defense pipeline that requires explicit Human-in-the-loop approval when necessary.

One tip for practical application: I strongly recommend inputting very specific and complex tasks into the `[Project Goal]` variable, such as **'Building an automated bug fix and PR (Pull Request) generation pipeline,'** rather than just "Do market research." Then, the AI will immediately draft an architecture where a Leader agent dives deep into code analysis, a Solution agent proposes the best fixes and logic, and a QA agent verifies regression tests—all working together like the gears of a precision Swiss watch.

In addition, it derives sophisticated strategies for separating short-term memory (current task context) and long-term memory (past solutions and user settings), allowing you to maintain consistent high performance while minimizing API token waste. This is a truly destructive weapon that can slash dozens of hours—or even weeks—of painful trial and error that a developer would otherwise spend on initial system design. By specifically mentioning internal private APIs or SaaS integration tools like Slack or Jira in the `[Required Tools]` variable for your domain, you can secure robust system specifications that are much more realistic and ready for immediate deployment.
