# 🗣️ "어... 어제 뭐 했더라?" 스탠드업 공포증 해결

- **🎯 추천 대상:** 애자일(Agile) 조직 개발자, 기획자, 디자이너
- **⏱️ 소요 시간:** 10분 고민 → 1분 컷
- **🤖 추천 모델:** ChatGPT, Claude, Gemini (모두 잘함)

> _"매일 아침 10시, 줌(Zoom) 회의실 입장 전 심장이 두근거리나요?"_

데일리 스탠드업(Daily Standup)은 짧게 공유하는 자리인데, 막상 말하려면 정리가 안 돼서 횡설수설하게 되죠. 내가 한 일을 돋보이게 하면서도 간결하게 공유하는 템플릿을 소개합니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. **자동 요약:** 메모장에 적어둔 키워드만 던져주면 문장으로 변환.
2. **포맷 통일:** Yesterday / Today / Blockers 표준 형식 준수.
3. **가시성 확보:** 나의 성과와 도움이 필요한 부분을 명확하게 전달.

---

## 🚀 해결책: "Standup Speaker"

### 🥉 Basic Version (기본형)

슬랙(Slack)이나 팀즈(Teams)에 텍스트로 공유할 때.

> **역할:** 너는 스크럼 마스터야.
> **입력:**
> - 어제: 로그인 API 수정, DB 스키마 변경, 팀 회식
> - 오늘: 회원가입 페이지 연동, 디자인 팀 미팅
> - 이슈: AWS 서버 접속 불안정
> **요청:** 위 내용을 깔끔한 스탠드업 양식으로 정리해줘.

<br>

### 🥇 Pro Version (전문가형)

말로 발표해야 하거나, 더 프로페셔널해 보이고 싶을 때.

> **역할 (Role):** 너는 애자일 팀의 유능한 시니어 개발자야.
>
> **상황 (Context):**
>
> - 나는 어제 정말 바빴지만, 겉으로는 별로 한 게 없어 보일까 봐 걱정이야.
> - '삽질'했던 과정은 빼고 '해결한 결과' 위주로 말하고 싶어.
>
> **입력 데이터 (My Notes):**
>
> - 어제: 결제 모듈 버그 잡느라 4시간 씀. 원인은 외부 API 타임아웃이었음. 결국 재시도 로직 넣어서 해결함.
> - 오늘: 어제 해결한 거 배포하고 모니터링 할 예정. 오후엔 기획팀이랑 요건 리뷰 있음.
> - 방해 요소: 지금 테스트 서버가 너무 느려서 테스트가 자꾸 밀림. 인프라 팀 확인 필요.
>
> **요청 (Task):**
>
> 1. 위 내용을 바탕으로 **발표용 대본(Script)**을 작성해줘.
> 2. "어제 ~하느라 고생했다"는 징징거림 대신, "외부 API 타임아웃 문제를 재시도 로직으로 해결했다"는 **성취(Achievement)** 중심으로 바꿔줘.
> 3. Blockers(방해 요소)는 정중하지만 단호하게 인프라 팀의 지원을 요청하는 뉘앙스로 써줘.
>
> **출력 형식:**
>
> - **[Yesterday]**: (핵심 성과 위주)
> - **[Today]**: (구체적 계획)
> - **[Blockers]**: (도움 요청)

---

## 💡 작성자 코멘트 (Insight)

스탠드업은 '나 어제 야근했어요'를 자랑하는 자리가 아니라, '팀이 알아야 할 정보'를 공유하는 자리입니다. AI를 통해 사족을 쳐내고 핵심만 남기면, 동료들이 당신의 말을 더 경청하게 될 것입니다.
