## 🚀 解决方案："Claude Code 游戏开发提示词"

### 🥉 Basic Version (基础版)

当你需要快速拉起一个游戏原型骨架（Boilerplate）时使用的初始指令。

> **角色：** 你是一名`[资深游戏开发者]`。
> **请求：** 请使用 `[HTML5 Canvas]` 开发一款基于浏览器的 2D 高尔夫游戏原型。场景中需包含高尔夫球和球洞，且核心交互必须采用鼠标拖拽射击（drag-to-shoot）机制。

### 🥇 Pro Version (专业版)

用于构建物理引擎、输入处理与关卡状态完全解耦的高阶工程化提示词。

> **角色 (Role)：** 你是一名拥有 10 年开发经验的`[资深前端游戏客户端工程师]`，也是我最完美的结对编程搭档。
>
> **背景 (Context)：**
>
> - 背景：我计划仅使用原生 Vanilla JS 和 HTML5 Canvas，开发一款轻量级的 2D 俯视视角高尔夫游戏。
> - 目标：产出具备极高可维护性的工程代码，确保物理引擎、输入事件流与关卡状态管理实现彻底的模块化解耦。
>
> **请求 (Task)：**
>
> 1. 将项目结构化拆分为 `index.html`（Canvas 容器）、`game.js`（主循环调度）、`physics.js`（向量运算及摩擦力/碰撞物理逻辑）。
> 2. 在 `physics.js` 中，精确实现草地摩擦力（Friction）的衰减效果以及墙壁碰撞的反弹（Bounce）逻辑。
> 3. 监听鼠标的 `mousedown`、`mousemove`、`mouseup` 事件，增加视觉辅助指示器（Indicator Line），在玩家拖拽球体时实时绘制出预期发射轨迹与力度反馈。
> 4. 封装 `Obstacle` 障碍物类，并利用数组设计 3 个难度逐步递增的独立关卡（Level）。
>
> **约束条件 (Constraints)：**
>
> - 绝对禁止引入任何第三方的游戏引擎库（如 Phaser.js 等），必须保持零依赖。
> - 必须为所有核心运算逻辑添加详细的注释，解析其背后的数学原理。
> - 严格遵循 ES6 Class 语法和面向对象编程（OOP）思想编写各业务模块。
