- **🤖 推荐模型：** Claude 3.5 Sonnet（代码结构分析能力极强）、GPT-4o

- ⭐ **难度：** ⭐☆☆☆☆
- ⚡️ **效果：** ⭐⭐⭐⭐⭐

> _"即使实现了再华丽的动画效果，但如果有人连登录按钮都无法点击，那么这段代码就是彻底失败的。"_

Web 无障碍（Web Accessibility）早已不再是单纯的“弱势群体关怀”，它是现代互联网服务的“基础生命线”，甚至在全球多地已成为一项强制性的法律义务。然而，要求开发者在每次高强度编码时，都去死记硬背庞大且极其晦涩的 WCAG（Web Content Accessibility Guidelines）2.2 指南，几乎是不可能完成的任务。

这款提示词（Prompt）能够深度扫描您编写的标记（Markup）和组件代码，在短短 1 分钟内，精准揪出在屏幕阅读器读取或键盘导航环境中可能触发的致命无障碍缺陷，并直接将其重构为完美符合语义化标准的优雅代码。

---

## ⚡️ 3句话总结 (TL;DR)

1. **转换为语义化标记：** 彻底终结滥用 `<div>` 标签的恶习，自动将其转换为具有明确功能意义的 HTML5 原生语义化标签。
2. **完美遵循 WCAG 2.2：** 严格验证色彩对比度、焦点（Focus）移动轨迹、WAI-ARIA 属性滥用等极其苛刻的国际无障碍标准。
3. **实战验证指南：** 提供仅依靠屏幕阅读器（如 VoiceOver、NVDA）和纯键盘（Tab 键）即可进行闭环验证的详尽测试方案。

## 💡 作者见解 (Insight)

在改善 Web 无障碍性的实战中，很多开发者极其容易陷入“盲目滥用 WAI-ARIA”的死亡陷阱。事实上，错误或过度地添加 `aria-label` 或是 `role` 属性，不但起不到帮助作用，反而会给屏幕阅读器用户带来更致命的认知混乱。

这款提示词的真正杀手锏在于：它在指令层面上强迫 AI 必须死守 **“First Rule of ARIA（优先使用原生 HTML 标签）”** 的核心底线。
在实际业务开发中，当您在构建极其复杂的自定义选择框（Select Box）或模态框（Modal）时，只要让您的代码跑一遍这个提示词的测试，就能以令人惊叹的精准度捕捉到那些肉眼难以察觉的键盘焦点丢失问题或是表单标签的遗漏。特别是当它与 Claude 3.5 Sonnet 模型强强联手时，所展现出的压倒性重构质量，真的会让你产生一种“身边就坐着一位严谨苛刻的无障碍架构师在手把手为你 Review 代码”的震撼错觉。

## 🙋 常见问题 (FAQ)

- **Q: 我可以直接上传 UI 设计图来进行无障碍审计吗？**
  - A: 当然可以，而且非常推荐！您可以在向 GPT-4o 或 Claude 3.5 Sonnet 发送组件截图的同时，附加一句极其明确的指令：“请严格检查视觉色彩对比度 (Color Contrast) 是否完全符合 WCAG AA 标准”。它甚至能帮您敏锐地揪出那些隐藏在华丽视觉设计背后的无障碍深坑。

- **Q: 我的项目深度绑定了 Tailwind CSS，它能自动帮我输出对应的实用类名吗？**
  - A: 毫无疑问。您只需在提示词的“背景 (Context)”部分精准补充一句：“请使用 Tailwind CSS 来实现屏幕阅读器专属的隐藏类（如 `sr-only`）”，AI 就会为您输出与您当前技术栈完美契合的、可直接复制粘贴的生产级代码。

## 🧬 提示词解剖 (Why it works?)

1. **注入硬核审查标准：** 我们并没有敷衍地说一句“帮我优化无障碍性”，而是将“WCAG 2.2 Level AA”这一国际公认的硬核指标直接注入到 AI 的角色设定中。这不仅大幅拉升了专业度，更完美抑制了 AI 胡编乱造的幻觉。
2. **强制捍卫 First Rule of ARIA：** 从根源上切断了 AI 盲目堆砌 ARIA 属性的退路，引导它将 HTML 原生规范的潜力压榨到极致，从而交付最纯粹、最根本的语义化标记。
3. **可落地执行 (Actionable) 的测试指南：** 授人以鱼不如授人以渔。它不仅直接交付正确的代码，更倒逼 AI 提供极其具体的键盘与屏幕阅读器测试场景，让开发者能够亲自上手操作和验证，将实际工作中的实用价值拉满。

## 📊 案例证明：Before & After

### ❌ Before (输入：用 `div` 堆砌出的最糟糕的按钮)

```html
<div class="submit-btn" onclick="sendData()">
  <img src="send-icon.png" />
  <span>发送</span>
</div>
```

### ✅ After (结果：完美适配键盘焦点与屏幕阅读器)

```html


<button type="button" class="submit-btn" onClick="sendData()">
  <img src="send-icon.png" alt="" aria-hidden="true" />
  <span>发送</span>
</button>
```

## 🎯 结语
