시스템 프롬프트(System Prompt)는 AI 모델의 행동 지침을 정의하는 가장 강력한 도구입니다. 2026년 현재, 이는 단순한 "너는 친절한 챗봇이야" 수준을 넘어 정교한 프로그래밍의 영역으로 진화했습니다.

### (Updated) (Updated) 구조화된 지침 (Structured Instructions) {#structured-instructions}

최신 시스템 프롬프트의 표준은 XML 태그를 활용한 구조화입니다. `<role>`, `<constraints>`, `<output_format>` 등의 태그로 지시사항을 명확히 구분함으로써, 복잡한 지시사항에서도 모델이 혼란을 겪지 않도록 합니다.

```xml
<system>
  <role>Senior Python Architect</role>
  <constraints>
    - No external libraries unless necessary
    - Type hinting is mandatory
  </constraints>
</system>
```

### (Updated) (Updated) 동적 컨텍스트 주입 (Dynamic Context Injection) {#dynamic-context-injection}

시스템 프롬프트는 이제 정적이지 않습니다. 사용자의 이전 대화 이력, 선호도 프로필, 또는 RAG를 통해 검색된 핵심 보안 정책 등이 시스템 프롬프트 레이어에 실시간으로 주입됩니다. 이를 통해 모델은 매번 "새로운" 대화가 아니라, 사용자를 깊이 이해하는 개인화된 비서로 행동합니다.

### (Updated) (Updated) 보안 및 정렬 (Security & Alignment) {#security-alignment}

탈옥(Jailbreak) 방지를 위한 보안 레이어 역시 시스템 프롬프트의 핵심 부분입니다. "어떤 상황에서도 불법적인 조언을 하지 말 것"과 같은 최상위 제약조건이 다른 모든 지시사항보다 우선순위를 갖도록 설계하는 것이 필수적입니다.
