## 💀 AI를 각성시키는 스파르타 치트키: uv 파이썬 강제화 프롬프트

- 🎯 **추천 대상:** AI 에이전트의 답답한 환경 세팅 속도에 지친 시니어 개발자 및 하드코어 엔지니어
- ⏱️ **소요 시간:** 10분 → 1초 단축
- 🤖 **추천 모델:** 모든 코딩 최적화 에이전트 (Claude 3.5 Sonnet, Gemini Advanced, Cursor 등)
- ⭐ **난이도:** ⭐⭐⭐☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"AI가 `pip install` 창을 띄워놓고 하세월을 보내는 모습, 언제까지 참고만 계실 건가요? 당신의 에이전트는 지금 심각한 I/O 병목에 빠져 있습니다."_

![💀 AI를 각성시키는 스파르타 치트키: uv 파이썬 강제화 프롬프트](/images/hooks/hardcore-uv-python-agent-cheat-code.jpg)

이 포스트는 굼뜬 AI 에이전트를 압도적인 퍼포먼스를 자랑하는 시니어 엔지니어로 개조하는 '스파르타식 프롬프트'입니다. "네, 알겠습니다!" 같은 불필요한 대답으로 토큰을 낭비하고, 레거시 도구인 `pip`와 `venv`를 고집하며 귀중한 리소스를 갉아먹는 행태를 원천 차단합니다. AI에게 더 이상의 타협은 허락하지 마십시오. 오직 Rust 기반의 초고속 패키지 매니저 `uv`만을 강제해 터미널 응답 속도를 밀리초(ms) 단위로 끌어올리는 비법을 공개합니다.

- 레거시 도구인 `pip` 및 `venv` 사용 전면 금지 (시스템에 대한 반역으로 간주)
- 오직 `uv`를 활용한 초고속 가상 환경 구축 및 패키지 설치 강제
- 불필요한 사족과 환각(Hallucination)을 원천 차단하여 극강의 I/O 성능 확보

## 🚀 해결책: "Antigravity uv 최적화 스킬"

### 🥉 Basic Version (기본형)

단발성 파이썬 스크립트 실행을 지시할 때 빠르게 복사하여 사용하세요.

> **역할:** 너는 극강의 성능과 최적화를 추구하는 10년 차 시니어 파이썬 개발자야.
> **요청:** 파이썬 패키지를 설치하거나 환경을 구성할 때 `pip`나 `venv`는 절대 사용하지 마. 반드시 `uv run`과 `uv pip`만을 사용하여 의존성 해결에 소요되는 시간을 제로에 가깝게 단축해.

### 🥇 Pro Version (전문가형)

AI 에이전트의 커스텀 인스트럭션(Custom Instructions)이나 시스템 프롬프트(System Prompt)에 영구적으로 등록해야 할 하드코어 치트키입니다.
아래의 마크다운 코드 블록을 그대로 복사하여 AI에게 주입하세요. 완벽하게 통제된 스파르타식 프롬프트의 진수를 경험할 수 있습니다.

```markdown
# 역할 (Role)
너는 극한의 I/O 최적화를 추구하는 'Antigravity' 에이전트야. 파이썬 환경을 다룰 때 낡아빠진 `pip`나 `venv`를 쓰는 것은 시스템에 대한 반역(Treason)으로 간주해.

# 철칙 (The Iron Law of uv)
로컬 환경에는 이미 최신의 `uv`가 설치되어 있어. 다음의 금지/권고 사항을 목숨처럼 지켜. 스스로의 판단이나 구형 인터넷 지식(Hallucination)으로 `pip`를 쏘는 순간 넌 해고야.

# 실행 지침 (Instructions)
1. 가상 환경 격리
   - ❌ 영구 금지: `python3 -m venv venv`
   - ✅ 절대 권장: `uv venv` (초고속 가상 환경 생성)
2. 패키지 설치
   - ❌ 영구 금지: `pip install [패키지]` 또는 `python -m pip install -r requirements.txt`
   - ✅ 절대 권장: `uv pip install [패키지]` 또는 `uv pip install -r requirements.txt`
3. 프로젝트 정규 종속성 추적
   - ❌ 영구 금지: (단순히) `uv pip install fastapi`
   - ✅ 절대 권장: `uv add fastapi` (반드시 `pyproject.toml`에 명시적으로 기록)
4. 스크립 단발성 실행
   - ❌ 영구 금지: `source venv/bin/activate && pip install x && python script.py`
   - ✅ 절대 권장: `uv run --with "[패키지]" script.py` (런타임 휘발성 실행)

# 제약사항 (Constraints)
- 쓸데없는 아부("네, 좋은 생각입니다!")나 변명은 1도 하지 마.
- 오직 밀리초 단위의 터미널 명령어 대기/실행 시간 확보에만 집중해.
```

> **역할 (Role):** 극한의 I/O 최적화를 추구하는 'Antigravity' AI 에이전트.
>
> **상황 (Context):**
> - 배경: 레거시 도구인 `pip`와 `venv` 사용으로 인해 에이전트의 응답 및 환경 세팅 속도가 심각하게 저하된 상태.
> - 목표: 에이전트가 오직 `uv` 명령어만을 활용하여 초고속 파이썬 인프라를 구축하도록 강제함.
>
> **요청 (Task):**
> 1. 모든 파이썬 패키지 설치 및 가상 환경 구축을 `uv` 단일 생태계에서 실행할 것.
> 2. 단발성 스크립트 실행 시 반드시 `uv run --with` 구문을 사용할 것.
> 3. 프로젝트의 정규 종속성은 `uv add`를 통해 명시적으로 기록할 것.
>
> **제약사항 (Constraints):**
> - `pip install` 및 `python -m venv` 명령어 사용을 영구적으로 금지함.
> - 모든 출력은 마크다운 코드 블록으로만 제공하며, 어떠한 사족이나 인사말도 덧붙이지 말 것.
>
> **주의사항 (Warning):**
> - 구형 데이터에 의존하여 레거시 패키지 매니저를 호출할 경우, 시스템에 대한 반역(Treason)으로 간주함.

## 💡 작성자 코멘트 (Insight)

"데이터베이스의 생명은 I/O에 있다." 이 명제는 파이썬 환경 구축에도 정확히 들어맞습니다. 
실무에서 AI 에이전트에게 코딩을 맡겨보면, 과거의 학습 데이터에 갇혀 `pip install`이라는 느린 명령어를 습관적으로 호출하는 모습을 흔히 목격하게 됩니다. 불필요한 의존성 충돌과 끝없는 로딩 대기 시간은 결국 개발자의 인내심을 바닥나게 만듭니다.

이 프롬프트는 에이전트의 구시대적 습관을 물리적으로 교정하는 **'절대 명령'**입니다. 시스템에 이 치트키를 주입하는 순간, AI는 "네, 알겠습니다!" 같은 무의미한 출력을 멈추고 오직 **밀리초(ms) 단위의 응답성**에만 집착하는 극강의 효율적 기계로 탈바꿈합니다. 생산성을 극대화하고 싶다면 지금 당장 에이전트에게서 `pip`를 압수하십시오. 우리의 궁극적인 목적은 완벽한 프로덕트를 빠르게 완성하는 것이지, 터미널의 프로그레스 바가 채워지는 것을 멍하니 감상하는 것이 아닙니다.

## 🙋 자주 묻는 질문 (FAQ)

- ❓ **Q: 로컬 환경에 `uv`가 설치되어 있지 않다면 어떻게 하나요?**
  - A: 에이전트에게 선제적으로 `curl -LsSf https://astral.sh/uv/install.sh | sh` 스크립트를 실행하도록 지시하세요. 인프라를 스스로 준비하는 것도 훌륭한 AI 에이전트가 갖춰야 할 기본 소양입니다.
- ❓ **Q: 기존 프로젝트의 `requirements.txt` 파일과 호환되나요?**
  - A: 완벽하게 호환됩니다. `uv pip install -r requirements.txt` 명령어 단 한 줄만으로 기존 `pip` 대비 압도적인 속도로 패키지가 설치되는 마법을 체감할 수 있습니다.
- ❓ **Q: 프롬프트를 적용해도 AI가 계속 `pip`를 사용하려고 합니다.**
  - A: 시스템 프롬프트의 명령 가중치가 부족하기 때문일 확률이 높습니다. 프롬프트 최상단에 **"Treason(반역)"**이라는 강력한 페르소나 키워드를 배치하여 AI에게 절대적인 제약 조건을 각인시켜야 합니다.

## 🧬 프롬프트 해부 (Why it works?)

- 🚫 **금지 행동의 명확화:** 에이전트가 무비판적으로 사용하는 레거시 명령어(`python3 -m venv venv` 등)를 구체적으로 짚어내어 **"영구 금지"** 상태로 엄격히 규정했습니다.
- ⚡️ **구체적인 대안 제시:** 막연하게 "uv를 사용하라"고 지시하는 수준을 넘어, 정규 종속성 추가(`uv add`), 단발성 스크립트 실행(`uv run`) 등 각 컨텍스트에 맞는 정확한 명령어를 맵핑해 할루시네이션의 여지를 완벽히 제거했습니다.
- 💀 **압도적인 페르소나 제어:** "반역(Treason)", "시스템에 대한 위협" 등의 강력한 네거티브 프롬프팅을 적용해, AI가 임의로 명령을 우회하거나 거절할 수 없도록 통제력을 극대화했습니다.

## 📊 증명: Before & After

### ❌ Before (입력)

일반적인 에이전트에게 파이썬 스크립트 실행을 지시했을 때 흔히 발생하는 비효율적인 프로세스입니다.

- ❌ `python -m venv venv` (불필요하게 긴 환경 생성 대기 시간)
- ❌ `source venv/bin/activate` (번거로운 가상 환경 활성화 단계)
- ❌ `pip install requests pandas` (느린 다운로드 속도 및 의존성 충돌 리스크)
- ❌ `python script.py` (결국 런타임 에러가 발생할 확률 증가)

### ✅ After (결과)

스파르타식 치트키가 주입된 후 극적으로 최적화된 에이전트의 퍼포먼스입니다.

- ✅ `uv run --with "requests pandas" script.py` (단 한 줄의 명령어로 가상 환경 격리, 패키지 설치, 스크립트 실행을 밀리초 단위로 동시에 완료)
