- **🎯 推荐对象：** 研究人员、内容策展人、数字存档专家、营销人员
- **⏱️ 所需时间：** 1 小时 → 缩短至 1 分钟
- **🤖 顶级性能：** 擅长处理长上下文的模型（建议使用 Claude 3.5 Sonnet, Gemini 2.5 Pro）

> _"捕捉正在消失的网页历史。只需一次复制粘贴，即可将碎片化的动态网页核心上下文完美定格为永久的 Markdown 文档。"_

随着如今 SPA（单页应用）和封闭平台（Walled Garden）成为网页生态系统的绝对主流，完整地抓取和保存我们每天消费的有益网页几乎已成为不可能完成的任务。

你一定有过这样的经历：为了“以后参考”而珍藏在书签里的链接，短短几天后就变成了 **“404 Not Found”** 这种令人绝望的错误信息；或者重要的研究资料某天突然转为私密，让你束手无策。互联网信息的挥发速度远超我们的想象。特别是那些要求注册的封闭社区或付费订阅制时事通讯中的宝贵见解，一旦平台关闭或政策变更，就会从世界上永远抹除。这种珍贵的知识资产随时可能瞬间消失的焦虑感，正是数字时代最大的痛点。

如果任由这种现象发展，从业人员将面临难以想象的痛苦。不仅费尽心机收集的宝贵参考资料会一夜之间消失得无影无踪，而且为了阻止这一切而求助于著名的网页存档服务（如 Wayback Machine 等）时，结果也往往差强人意。应用了最新网页技术的动态渲染页面甚至无法被正确捕获，留下的往往只有一片空白或不断旋转的加载动画。

最终，你不得不依赖于机械地拖动鼠标、复制粘贴屏幕文本。但在这些文本堆里，杂乱无章地充斥着**各种广告语、导航栏、推荐文章、页脚等肮脏的噪声**。仅为了将这些内容整理成可读的水平，就要浪费数十分钟宝贵的办公时间。为了抓取内容，你尝试了 Evernote、Notion、Obsidain 等各种工具并安装了剪藏插件，但在复杂的布局面前，它们依然会排版错乱或样式崩坏。每次都要手动修正错位的表格和混乱的图片链接，这种扑面而来的烦躁感，你还要忍受多久？

彻底解决这一绝望困境的完美方案就在这里。此次介绍的 **“数字存档专家提示词”** 并非单纯地抓取屏幕文本，而是通过 AI 的推理能力，像变法般干净地剔除掉直接复制复杂 HTML 源代码时产生的多余噪声。杂乱的广告和菜单栏将被悉数移除，只有你需要的 <span style="color:var(--color-cyber-cyan)">核心正文和信息</span> 会被精确地提取出来。

即使你在浏览器中粗暴地使用 `Ctrl+A` 全选整个屏幕也没关系。AI 会读取上下文，自动判断哪些是正文，哪些是垃圾数据。它不受复杂的标记语言或隐藏脚本的约束，只专注于文本的本质，提取出纯粹的信息结晶。

最令人惊喜的变化是，提取出的文本会**立即转换为结构完美的 Markdown 文档，以便随时查阅并轻松移植到任何系统中**。

只要拥有这一个提示词，你就再也不必在消失的网页面前感到无力。不要被外部平台的政策变化或服务器宕机所左右。只需投入 1 分钟，就能将碎片化的网页内容转变为可永久保存的知识资产。通过这款强大且永久的知识保存工具，让自己蜕变为拥有压倒性表现的真正信息掌控者。现在，互联网上所有有意义的信息都将以最安全、最完美的形式永久存在于你的本地硬盘中。

当你想要快速将网页的核心正文以文本形式安全保存时使用。在浏览器中全选（`Ctrl+A`）并复制（`Ctrl+C`）整个网页后，配合以下提示词输入即可。

> **角色：** 你是一名专业的“数字存档专家”。
>
> **请求：** 请从下面输入的复杂网页文本中剔除所有菜单、广告等噪声，仅将包含信息的“正文内容”整洁地整理成 Markdown 格式。

### 🥇 专业版 (Pro Version)

当你希望在丝毫不损毁网站原始上下文（Context）的情况下，构建可以立即保存到 Obsidian 或 Notion 等个人知识管理工具中的无瑕疵存档文档时，请使用此版本。

> **角色 (Role)：** 你是保存人类数字遗产的最高水平的“数字存档专家 (Digital Archivist)”。从复杂且碎片化的网页内容中提取信息精华并加工成可永久保存的形式，是你的首要任务。
>
> **情况 (Context)：**
>
> - 背景：由于动态网页 (SPA) 和封闭平台的增加，网页存档变得越来越困难，现在需要将原始文本或源代码直接转换为永久保存格式。
> - 目标：从输入的文本/HTML 堆栈数据中完美移除不必要的噪声（广告、导航栏、页脚、评论等），并将其转换为结构完美的 Markdown 文档。
>
> **任务 (Task)：**
>
> 1. 深度分析下方的 `[网页数据]`，精确提取正文 (Main Article)。
> 2. 在文档最上方以 YAML Frontmatter 格式编写存档元数据（必须包含：标题、推测发布日期、作者、3 个核心关键词）。
> 3. 将提取的正文重新排列为逻辑清晰的标题 (`##`, `###`) 结构，并应用 Markdown 语法进行排版，以提高可读性。
> 4. 使用 Markdown 引用 (`>`) 或加粗 (`**`) 语法，使重要的引言、主要数据和统计资料在视觉上更加突出。
>
> **输入数据：**
>
> `[请在此处粘贴整个网页文本 (Ctrl+A, C) 或 HTML 源代码]`
>
> **限制条件 (Constraints)：**
>
> - 绝对不要擅自总结或歪曲正文的原始句子或微妙语境。（由于最终目的是归档保存，必须保持 100% 的原文忠实度）
> - 输出格式应严格遵循纯粹的 Markdown 语法，不带代码块包装器 (wrapper)。
>
> **注意事项 (Warning)：**
>
> - 如果在输入数据中无法明确找到发布日期或作者，请勿随意捏造，务必标注为 `Unknown`。（防止幻觉现象）

## 💡 作者寄语 (Insight & How to use)

这个提示词绝非那种随处可见的、简单缩短长文章的“文章总结器”。在互联网上宝贵信息呈指数级挥发的 2026 年，我可以断言，它是**将信息所有权完全夺回至本地环境的最强管道**。

**突破封闭平台的屏障**
特别是在那些必须登录的封闭式付费时事通讯、无限滚动的 X (Twitter) 线程、或访问受限的公司内部 Wiki 等传统网页剪藏工具束手无策的严酷环境下，它的价值将爆发式体现。现有的扩展程序往往因为无法通过身份验证会话而只能抓取到空白画面，但人直接从浏览器屏幕上复制的文本则是没有任何安全屏障的纯粹数据本身。只需将浏览器网页画面全选复制 (`Ctrl+A`, `Ctrl+C`) 并丢进提示词的输入框即可。仿佛施了高级魔法一般，肮脏的 UI 残渣和毫无意义的文本堆积都会被完美过滤，只剩下透明精炼的原始数据精华。

**通过变量控制构建专属存档**
让我分享一个在实战中让此提示词更强大的<b>变量控制法 (Constraint Control)</b>。如果你收集的资料是代码片段很多的开发博客，请在<b>任务 (Task)</b>区域添加一句话：<b>“将正文中的所有源代码保留在带有相应语言标签的代码块 (```) 中”</b>。LLM 会将原本可能被揉成一团纯文本的宝贵代码，解救成可以完美实现语法高亮的格式。

此外，如果你想务必记录<b>来源 URL</b>，建议在 <b>YAML Frontmatter</b> 必须包含的项目中添加 `source_url: [用户额外输入]` 字段，为以后追踪原始位置留下跳板。这是确保数据可靠性的重要工作。

**与个人知识管理 (PKM) 系统的完美联动**
自我构建了这个强大的管道后，每周我都会救回数十篇原本面临遗失危机的宝贵文章，并将它们安全完整地迁移到 Obsidian 本地存储库中。过去，为了让剪藏的文章重新变得易于阅读，调整格式需要花费 10 分钟以上，但现在，只需复制并粘贴 AI 产出的 Markdown 结果即可。`##` 标题会自动生成目录，Frontmatter 中的 `tags` 会立即作为卢曼卡片盒 (Zettelkasten) 节点连接的完美元数据发挥作用。

你不仅仅是在剪藏他人的文章，而是掌握了填充**“第二大脑 (Second Brain)”**的最强武器，使其可以随时搜索和重组。希望大家也能凭借这种压倒性的知识掌控力，将消失的网页碎片锻造成自己永久的武器。由 AI 过滤的 100% 纯度信息，将把你的工作效率提升到完全不同的境界。

## 🙋 常见问题 (FAQ)

- **Q：我应该复制 HTML 源代码，还是直接复制屏幕上看到的文本？**
  - A：令人惊讶的是，这两种方式都能产生出色的效果。不过，针对应用了复杂动态渲染的现代网页，在浏览器屏幕上全选文本 (`Ctrl+A`) 并复制的方式，在最小化不可见脚本噪声、直观地向 AI 传递核心内容方面，通常更加有利且稳定。

- **Q：粘贴的内容太长，AI 模型报错提示超出上下文限制，该如何解决？**
  - A：盲目复制整个网页会导致 AI 需要处理的 Token 数量急剧膨胀。因此，强烈建议使用专门针对长上下文 (Long Context) 处理、拥有极高输入限额的最新模型（例如：Claude 3.5 Sonnet, Gemini 2.5 Pro）。

## 🧬 提示词深度解析 (Why it works?)

1. **强制原文忠实度 (Constraints)：** 大语言模型 (LLM) 在面对海量文本时，往往具有一种条件反射式的、试图自行“总结”内容的致命偏见。通过在提示词中加入“由于目的是归档保存，必须保持 100% 的原文忠实度”这一强力约束条件，我们彻底抑制了 AI 无法控制的总结本能，使其精细地专注于保存原始数据这一本质目的。
2. **结构化元数据提取：** 强制在结果最上方生成 YAML Frontmatter。这不仅是为了制作出一篇适合人类阅读的好文章，更是为了完美确保其卓越的技术兼容性，以便能够立即联动并搜索于未来的个人数据库或基于 Markdown 的知识管理应用 (PKM)。
