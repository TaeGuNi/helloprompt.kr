## 📝 正規表現(Regex)の悪夢、もうAIに丸投げしてください

- **🎯 推奨対象:** すべてのエンジニア、データアナリスト、マーケター
- **⏱️ 所要時間:** 30分 → 1分（大幅短縮）
- **🤖 推奨モデル:** すべての対話型AI (ChatGPT, Claude, Geminiなど)

- ⭐ **難易度:** ⭐⭐☆☆☆
- ⚡️ **効果性:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐

> _「メールアドレスのバリデーションのために `^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$` のような暗号を、まだ自力で解読していますか？」_

正規表現（Regular Expression）はエンジニアやデータアナリストにとって避けては通れない必須スキルですが、同時に最も時間を奪われ、頭を悩ませる厄介な領域でもあります。必要になるたびに構文を検索し、他人が書いたコードはまるで解読不能な宇宙語のように見える……そんな経験は誰にでもあるはずです。

もう、自力で暗号解読に挑むのはやめにしましょう。これからの時代、複雑な正規表現を丸暗記する必要はありません。あなたが欲しい「データのパターン」をAIに自然言語で伝えるだけで、エッジケースまで考慮された完璧な正規表現が瞬時に完成するのです。

---

## ⚡️ 3行要約 (TL;DR)

1. **構文の暗記は一切不要:** 複雑な正規表現のルールを覚えたり、毎回リファレンスを検索したりする無駄な時間を完全に排除します。
2. **自然言語からコードへ直結:** 抽出したいパターン（電話番号、メールアドレスなど）を言葉で説明するだけで、AIが正確な正規表現を生成し、その意味まで丁寧に解説してくれます。
3. **テストコードも同時生成:** 正規表現の正確性を即座に検証できるよう、エッジケース（例外的な入力）を含めたテストコードもセットでリクエストします。

## 🚀 解決策：「Regex（正規表現）ジェネレーター」

### 🥉 Basic Version (基本形)

細かい条件がなく、とにかく手っ取り早く正規表現のコードだけが必要な場合に使用してください。

> **役割 (Role):** あなたは`[シニアエンジニア]`です。
> 
> **リクエスト (Task):** 以下の条件を満たす正規表現を作成し、その意味を解説してください。
> 
> **条件 (Context):** `[日本の携帯電話番号（ハイフン有無両方対応）]`

### 🥇 Pro Version (専門家形)

より複雑な条件が絡む場合や、絶対にバグを許せない実務レベルのクオリティが求められる場面で活用してください。

> **役割 (Role):** あなたは正規表現のスペシャリストであり、`[シニアエンジニア]`です。
>
> **状況 (Context):**
>
> - 背景: `[Python / JavaScript]`で入力データのバリデーション（検証）処理を実装しています。
> - 目標: エッジケースを網羅した、完璧かつ安全な正規表現を作成すること。
>
> **リクエスト (Task):**
>
> 1. 以下の要求事項を満たす最適な正規表現を作成してください。
> 2. 生成した正規表現の各記号がどのような意味を持つのか、初心者にもわかるように詳細に解説（解釈）してください。
> 3. この正規表現を即座に検証できるテストコード（マッチする成功ケース / マッチしない失敗ケースの両方）を記述してください。
>
> **要求事項 (Constraints/Variables):**
>
> - 抽出・検証したいパターン: `[日本の携帯電話番号]`
> - 具体的な条件:
>   - `[070, 080, 090のいずれかで始まること]`
>   - `[ハイフン（-）はあってもなくても許容すること]`
>   - `[桁数が正確であること（例: 090-1234-5678 または 09012345678）]`
>
> **注意点 (Warning):**
>
> - ReDoS（正規表現DoS攻撃）の脆弱性を引き起こさないよう、パフォーマンスを考慮した安全なパターンを提示してください。
> - 不確かな構文は使用せず、確実に動作するもののみを提供してください。

## 💡 筆者コメント (Insight)

実務において、正規表現のわずかなミスは致命的なシステム障害（例えば、特定の入力でサーバーがフリーズするReDoS攻撃など）を引き起こす危険性を秘めています。

そのため、AIに単に「正規表現を書いて」と丸投げするのではなく、**「テストケース」と「各記号の意味（解説）」を必ずセットで要求すること**が極めて重要です。このプロンプトを活用すれば、生成されたコードが本当に意図通りに動くのかをテストコードを通して即座に検証できるため、コードレビューにかかる時間を大幅に削減できます。安全性を担保しながら、開発スピードを劇的に引き上げる強力なテクニックです。

  - A: 基本的には高精度ですが、盲信は絶対に禁物です。必ずプロンプトで生成させた「テストケース」を手元で実行し、ビジネス要件を完全に満たしているか確認してから本番環境に適用してください。

- **Q: 特定のプログラミング言語（例えばJavaやPHP）に特化した正規表現を出力させることは可能ですか？**

## 🧬 プロンプト解剖 (Why it works?)

1. **Role（役割）の付与:** AIに「シニアエンジニア」や「正規表現のスペシャリスト」という明確なペルソナを与えることで、初心者向けの浅い回答を排除し、実務レベルに耐えうる堅牢なコードを引き出しています。
2. **Task（タスクの細分化）:** 単なるコード生成で終わらせず、「解説」と「テストコードの作成」を義務付けています。これによりブラックボックス化を防ぎ、人間のエンジニアが最終確認しやすい出力形式を強制しています。
3. **Warning（セキュリティの担保）:** 「ReDoS脆弱性を考慮する」という一文を添えることで、AIが不用意に処理の重いパターン（例: `(a+)+` のようなネストされた量指定子）を出力するリスクを未然に防いでいます。

## 📊 証明: Before & After

### ❌ Before (入力のみの雑な指示)

```text
日本の携帯電話番号の正規表現を書いて。ハイフンありなし両方対応で。
```

_(AIの回答は正規表現のコードだけを唐突に出力することが多く、検証も解説もないため実務で使うには不安が残ります)_

### ✅ After (Pro Versionの出力結果)

```javascript
// JavaScriptでの実装とテスト例

// 070, 080, 090で始まり、ハイフン有無を許容する正規表現
const phoneRegex = /^(070|080|090)-?\d{4}-?\d{4}$/;

// マッチするテストケース (Trueになるべき)
console.log(phoneRegex.test("090-1234-5678")); // true (ハイフンあり)
console.log(phoneRegex.test("08012345678")); // true (ハイフンなし)
console.log(phoneRegex.test("070-9876-5432")); // true (070始まり)

// マッチしないテストケース (Falseになるべき)
console.log(phoneRegex.test("03-1234-5678")); // false (固定電話)
console.log(phoneRegex.test("090-123-45678")); // false (桁数異常)
console.log(phoneRegex.test("0901234567")); // false (桁不足)
```

**🔍 AIによる詳細解説（抜粋）**

- `^`: 文字列の先頭を表します。
- `(070|080|090)`: 070, 080, 090 のいずれかのプレフィックスに一致します。
- `-?`: ハイフン `-` が0回、または1回出現することを許容します。
- `\d{4}`: 数字 (`0-9`) が正確に4桁続くことを意味します。
- `$`: 文字列の末尾を表し、余分な文字が含まれるのを防ぎます。
