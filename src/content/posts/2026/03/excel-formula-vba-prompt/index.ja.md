# 📝 Excelの数式とVBAを1秒で作成するAIプロンプト

- **🎯 おすすめの対象:** Excel関数の地獄に陥っているジュニア層、反復的なデータクレンジングに疲弊しているマーケター、マクロを組む時間がない実務担当者
- **⏱️ 所要時間:** ググるのに30分 → プロンプトをコピペして1分
- **🤖 おすすめのモデル:** ChatGPT (GPT-4o), Claude 3.5 Sonnet
- ⭐ **難易度:** ⭐☆☆☆☆
- ⚡️ **効果:** ⭐⭐⭐⭐⭐
- 🚀 **活用度:** ⭐⭐⭐⭐⭐
_Googleで「Excel VLOOKUP 複数条件」と検索して、広告だらけのブログをさまよう時間はもう終わりです。_
データ整理のために残業していませんか？Excelの数式エラーと格闘したり、動かないVBAコードを見てため息をつくのは、現場ではあまりにもよくあることです。もう関数の構文を完璧に暗記したり、オブジェクト指向プログラミングを勉強したりする必要はありません。AIに**「手持ちのデータ構造」**と**「求める最終結果」**を正確に指示するだけで、そのままコピペして使える完璧な数式とマクロコードを作成してくれます。

- 複雑な複数条件の検索や、反復的なデータ前処理（VBA）作業において、圧倒的な時間短縮効果を発揮します。

## 🚀 解決策: "Excel/VBAマスタープロンプト"
### 🥉 Basic Version（基本型）
手早く簡単な数式やショートカットが必要なときに使用してください。
> **役割:** あなたは20年目の`[Excel/VBAの最高専門家]`です。
> **リクエスト:** 私が持っているデータは`[A列: 名前、B列: 部署、C列: 売上]`です。`[営業部の売上合計]`を求める、最も効率的でエラーのない数式を作成してください。
### 🥇 Pro Version（専門家型）
複雑な複数条件、データ前処理の自動化、またはVBAマクロコードが必要なときに使用してください。このプロンプトはエラーを最小限に抑え、正確なセル参照を生成します。
> **役割 (Role):** あなたはフォーチュン500企業のデータアナリストであり、`[ExcelおよびVBAの最高専門家]`です。
> 
> **状況 (Context):**
> - 背景: 私は毎週`[数百行の生データをクレンジングしてレポート形式に変換]`する作業を行っています。
> - 目標: `[手作業でのコピペ作業をなくし、ボタン一つで自動化するか、完璧な数式を適用]`することが目標です。
> 
> **データ構造 (Data Structure):**
> - シート名: `[RawData]`
> - A列: `[日付 (YYYY-MM-DD)]`
> - B列: `[製品コード (例: PRD-1023)]`
> - C列: `[販売数量 (数値)]`
> - D列: `[担当者名]`
> 
> **リクエスト (Task):**
> 1. 上記のデータを基に、`[特定の日付範囲内で、製品コード別の販売数量の合計を求めるVBAマクロコード]`を作成してください。
> 2. `[動作の仕組みと、コードをExcelモジュールに適用する方法]`を非常に短く明確に説明してください。
> 3. もし関数（数式）で解決する方がVBAよりも効率的で速い場合は、関数のアプローチを優先して提案してください。
> 
> **制約事項 (Constraints):**
> - 出力形式は箇条書き（List）とMarkdownのコードブロックのみにしてください。
> - 不要な前置きや挨拶は絶対に省略し、すぐに本題（コード/数式）から出力してください。
> - VBAコードを作成する際は、各行に日本語で詳細なコメントを必ず付けてください。
> 
> **注意事項 (Warning):**
> - Excelのバージョンによる競合が発生する可能性のある最新の関数（例：XLOOKUP）を使用する場合は、旧バージョン（VLOOKUP+MATCH）の代替案も短く言及してください。不確実なオブジェクトやメソッドは捏造せず、Excelの基本関数のみを使用してください。

## 💡 作成者のコメント (Insight)
このプロンプトの核心は、**「データ構造（Data Structure）」をAIに明確にマッピング（Mapping）してあげること**です。
多くの実務担当者はAIに対して、「Aチームの売上合計を求める関数を教えて」と大雑把に質問します。AIはAチームがどの列（Column）にあるのか、売上が数値なのかテキスト混じりの値なのか知る由もないため、一般的なテンプレートだけを返し、結局ユーザーがセル番地（A2:A100など）を一つ一つ修正する手間が発生します。
しかし、上記のProプロンプトのようにA列、B列にどのようなデータがあるのかを宣言してあげれば、AIが`SUMIFS(RawData!C:C, RawData!A:A, ...)`のようにすぐにコピペして使える実践用のコードを吐き出します。 
実務は実践です。関数の構造を解読したりマクロの構文を勉強したりして、貴重な1時間を無駄にしないでください。あなたの本当の価値はExcelの構文を暗記することにあるのではなく、算出されたデータを解釈してビジネス上の決定を下すことにあります。タイピングはAIに外注し、本質に集中してください。

## 🙋 よくある質問 (FAQ)
- **Q: Googleスプレッドシートでも同じように動作しますか？**
  - A: はい、可能です。ただし、リクエスト（Task）部分に「Googleスプレッドシート用の数式/Apps Scriptで作成して」と明記しないと、関数の互換性の問題が生じる可能性があります。
- **Q: AIが提示した数式を入れたらエラー値（#N/A、#VALUE!）が出た場合はどうすればいいですか？**
  - A: エラーが出た該当のセルをコピーして、AIに「この数式を使ったら`#N/A`エラーが出た。IFERRORで空白処理するコードも追加して」とフィードバックすれば、10秒で完璧な修正版を返してくれます。
- **Q: 無料版のChatGPTでもできますか？**
  - A: はい、簡単なExcelの数式作成であれば無料版でも十分です。ただし、数十行を超える複雑なVBAコードを作成する際は、文脈をしっかりと維持するPro版の方がはるかに精巧です。

- **データ構造のマッピング (Data Structure):** データの見た目を具体的に教えることで、ユーザーが手動でセル参照範囲（$A$1など）を修正する手間を根本から遮断しました。
- **制約条件 (Constraints):** 現場でコピペするのにも忙しいのに、AIが「こんにちは！Excel作業をお手伝いしますね〜」などと長々と喋る前置きの時間をなくしました。
- **最適化のテクニック (Task 3番):** 軽い作業に重いVBAを使わないように、数式の方が速ければ数式を提案せよというロジックを入れ、最も効率的な結果を誘導しました。

## 📊 証明: Before & After
### ❌ Before（入力）
```text
A列に日付、B列に製品コード、C列に数量があるんだけど、PRD-1023の数量合計を求めるマクロお願い。
```
(結果：変数名がめちゃくちゃだったり、セル範囲が固定されていてデータが増えるとエラーになるコードを吐いたり、不要な前置きが長かったりする)
### ✅ After（結果）
```text
1. Excelの数式 (VBAよりも効率的で速い)
=SUMIFS(RawData!C:C, RawData!B:B, "PRD-1023")

2. VBAマクロコードの適用が必ず必要な場合
Sub CalculateProductSales()
    Dim ws As Worksheet
    Dim lastRow As Long
    Dim totalSales As Double
    Dim i As Long
    
    ' シートの設定
    Set ws = ThisWorkbook.Sheets("RawData")
    
    ' データの最終行を取得 (データ追加時に自動対応)
    lastRow = ws.Cells(ws.Rows.Count, "A").End(xlUp).Row
    totalSales = 0
    
    ' 条件検索および合算
    For i = 2 To lastRow ' 1行目はヘッダーなので2行目から開始
        If ws.Cells(i, 2).Value = "PRD-1023" Then
            totalSales = totalSales + ws.Cells(i, 3).Value
        End If
    Next i
    
    ' 結果の出力
    MsgBox "PRD-1023の総販売数量は " & totalSales & "個です。", vbInformation, "結果"
End Sub
```
