「とりあえず動くゴミ（Working Garbage）」を量産していませんか？

納期に追われて急いで書いたコードや、ロジックが複雑に絡み合ったスパゲッティコードは、後々のメンテナンスにおいて深刻な技術的負債となります。数ヶ月後、機能追加やバグ修正のために自分のコードを見返したとき、「一体誰がこんな読みにくいコードを書いたんだ？」と絶望し、履歴を調べて自分の名前が表示されたときの気まずさを経験したことはないでしょうか。

しかし、いざリファクタリングを行おうとしても、現実には膨大な時間と精神力を消耗します。どこから手をつければいいのか分からず、既存のロジックを壊してしまう恐怖と戦いながらコードを修正するのは、骨の折れる作業です。結局、「動いているから今は触らないでおこう」と見て見ぬふりをしてしまうのが、多くの開発者の現実でしょう。

そんな限界を感じている時こそ、AIリファクタリング・エージェントの出番です。あなたの書いた乱雑で読みにくいコードを、SOLID原則に基づいた美しく保守性の高い「クリーンコード」へと一瞬で変身させる、魔法のようなプロンプトをご紹介します。まるで隣に優秀なシニアエンジニアが座り、優しくコードレビューをしてくれているかのような体験を、今日からあなたの開発フローに取り入れてみましょう。

## ⚡️ 3行まとめ (TL;DR)

1. **AIを「シニアエンジニア」に任命する:** 単に「コードを直して」と指示するのではなく、具体的な役割と基準（SOLID原則など）を与えることがクリーンコードへの近道です。
2. **変更の理由を必ず明記させる:** コードのどこを、なぜ修正したのかを論理的に説明させることで、あなた自身のコーディングスキルの向上にも繋がります。
3. **テストコードも一緒に生成:** リファクタリング後は、既存の機能が壊れていないことを保証するための単体テスト（ユニットテスト）が不可欠です。

## 💡 作成者のコメント (Insight)

このプロンプトの最大の価値は、単に綺麗なコードを得ることではなく、**「なぜそのように修正したのか」というシニアエンジニアの高度な思考プロセスを学べる点**にあります。特にジュニア開発者の場合、Pro Versionで出力される「問題点のリストアップ」と「変更の意図」を熟読することで、自分自身のコーディングの癖や、これまで見落としていたアンチパターンに気付くことができます。

実務で活用する際は、ファイル全体を一度に投げるのではなく、関数やクラス単位など、ある程度小さく分割してプロンプトに入力することをおすすめします。これにより、AIのハルシネーション（もっともらしい嘘の出力）を効果的に防ぎ、より精度が高く実用的なリファクタリング結果を得られます。

## 🙋 よくある質問 (FAQ)

- **Q: 会社の本番コードをそのまま入力しても大丈夫ですか？**
  - A: セキュリティや機密保持の観点から、社外秘の情報（APIキーや独自のパスワード、機密性の高いビジネスロジック、個人情報など）は絶対に除外（マスキング）してから入力してください。また、社内のAI利用ガイドラインやセキュリティポリシーに必ず従ってください。

- **Q: リファクタリング後、バグが発生することはありますか？**
  - A: はい、AIも完璧ではないため、特定のエッジケースで挙動が変わってしまうリスクはゼロではありません。だからこそ、プロンプトで「ユニットテストの作成」を強く指示し、変更後に必ずテストを実行して、期待通りの動作をしているかご自身の目で確認することが極めて重要です。

## 🧬 プロンプトの解剖学 (Why it works?)

1.  **Code Smellsの指摘:** 単にコードを修正させるだけでなく、まず「どこが悪いのか」を言語化して分析させることで、AIの推論精度が飛躍的に向上し（Chain-of-Thought効果）、ユーザー自身のアーキテクチャ学習にも大きく貢献します。
2.  **ビジネスロジック保持の制約:** 「振る舞いを変えない」という強力な制約（Constraints）を設けることで、リファクタリングの本来の定義を厳格に守らせ、予期せぬ機能の改変やバグの混入を未然に防ぎます。

## 📊 証明: Before & After

### ❌ Before (入力 - JavaScriptの例)

```javascript
function calc(a, b, c) {
  let res = 0;
  if (c === "add") {
    res = a + b;
  } else if (c === "sub") {
    res = a - b;
  } else if (c === "mul") {
    res = a * b;
  }
  return res;
}
```

### ✅ After (結果)

```javascript
/**
 * 2つの数値に対して指定された演算を実行します。
 * @param {number} a - 最初の数値
 * @param {number} b - 2番目の数値
 * @param {string} operation - 演算の種類 ('add', 'sub', 'mul')
 * @returns {number} 演算結果
 * @throws {Error} サポートされていない演算子が指定された場合
 */
function calculate(a, b, operation) {
  const operations = {
    add: (x, y) => x + y,
    sub: (x, y) => x - y,
    mul: (x, y) => x * y,
  };

  const selectedOperation = operations[operation];

  if (!selectedOperation) {
    throw new Error(`サポートされていない演算子です: ${operation}`);
  }

  return selectedOperation(a, b);
}
```

_(※ AIによる解説: マジックナンバーの排除、オブジェクト・リテラルを活用した条件分岐の簡略化、エラーハンドリングの追加を行いました。)_
