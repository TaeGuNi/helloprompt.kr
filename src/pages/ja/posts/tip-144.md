_"서버가 다운됐다는 게 정확히 무슨 뜻이야?"_
_"API 연동? 그게 뭔데 2주나 걸려?"_

개발 용어는 외계어 같습니다. 사전적 정의를 봐도 이해가 안 가죠. 이럴 땐 '비유(Analogy)'가 약입니다. 익숙한 상황에 대입하면 복잡한 기술도 단박에 이해됩니다. 이 프롬프트는 딱딱한 IT 용어를 말랑말랑한 일상 언어로 통역해 줍니다.

---

## ⚡️ 3줄 요약 (TL;DR)

1. 어려운 IT 용어를 사용자 지정 상황(식당, 도서관, 축구팀 등)에 맞춰 비유
2. 비유 속 요소와 실제 기술 요소 1:1 매칭 (Mapping)
3. 비유의 한계점(주의사항)까지 짚어주어 오개념 방지

## 📊 증명: Before & After

### ❌ Before (사전적 정의)

**API (Application Programming Interface)**: 응용 프로그램에서 사용할 수 있도록, 운영 체제나 프로그래밍 언어가 제공하는 기능을 제어할 수 있게 만든 인터페이스... (졸림)

### ✅ After (프롬프트 결과: 식당 비유)

**[상황 비유]**
API는 **'식당 웨이터(점원)'**입니다.
손님(당신)이 주방장(서버)에게 직접 가서 "파스타 내놔"라고 소리치지 않죠?
대신 웨이터에게 메뉴판(문서)을 보고 주문을 합니다.
그러면 웨이터가 주문서를 들고 주방에 가서 요리를 받아와 당신에게 갖다줍니다.
당신은 주방에서 불 쇼를 하든 칼질을 하든 알 필요가 없습니다. 웨이터만 부르면 되니까요.

**[1:1 매칭]**
| 식당 상황 | IT 기술 |
| :--- | :--- |
| **손님** | 클라이언트 (사용자/앱) |
| **웨이터** | **API** (중개자) |
| **주방장** | 서버 (데이터베이스) |
| **메뉴판** | API 문서 (사용 설명서) |
| **요리** | 데이터 (응답) |

**[한 문장 요약]**
"서로 말이 안 통하는 두 프로그램 사이에서 심부름해 주는 똑똑한 점원."
