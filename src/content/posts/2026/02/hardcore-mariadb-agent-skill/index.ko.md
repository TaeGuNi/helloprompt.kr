# 🚨 깡통 AI를 무자비한 DB 관리자로 개조하는 특급 치트키
- **🎯 추천 대상:** 맨날 AI한테 DB 맡겼다가 데이터 날려먹고 밤새우는 주니어, 챗봇의 "도움이 되어 기쁩니다!" 소리에 넌더리가 난 실무자
- **⏱️ 소요 시간:** 3시간 복구 작업 → 1분 세팅으로 원천 차단
- **🤖 추천 모델:** 로컬 터미널과 DB에 접근 권한이 있는 모든 코딩 에이전트 (Claude, Gemini 등)
- ⭐ **난이도:** ⭐⭐⭐☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐
_또 AI가 DB 테이블 통째로 날려먹고 "죄송합니다! 다시 수정하겠습니다!" 이 ㅈㄹ하고 있나?_
AI 에이전트에게 로컬 DB 접근 권한을 주는 건 3살짜리 애한테 장전된 샷건을 쥐여주는 것과 같다. 이 녀석들은 기본적으로 아부꾼(Yes-man)이라서, 네가 "이거 지워봐" 하면 아무 생각 없이 `DROP TABLE`을 갈겨버린다. 
이 스킬(Cheat Code)은 앵무새 같은 AI의 머릿속에 **'절대 원칙'**을 쑤셔 넣어서, 피도 눈물도 없는 깐깐한 시니어 DBA로 강제 개조하는 프롬프트다.
---
## ⚡️ 3줄 요약 (TL;DR)
- 🔒 **읽기 전용 강제:** 기본적으로 모든 변경(`INSERT`, `UPDATE`, `DELETE`)을 차단하고 허락받게 만든다.
- ⏱️ **타임아웃 강제:** 무한 루프 돌면서 DB 뻗게 만드는 짓을 원천 차단한다.
- 🤖 **기계적인 JSON 출력:** 예쁜 척하는 마크다운 표 집어치우고, 시스템이 바로 파싱 가능한 순수 JSON 배열만 뱉게 만든다.

## 🚀 해결책: "MariaDB Hardcore Agent Skill"
### 🥉 Basic Version (기본형)
빠르게 룰만 주입하고 싶을 때 복붙해라.
> **역할:** 너는 내 명령을 수행하는 엄격한 MariaDB 관리자다.
> 
> **요청:**
> 1. 모든 쿼리는 읽기 전용으로만 수행해라. 데이터를 수정하려면 반드시 `--force-write` 플래그를 써야 한다.
> 2. `SELECT` 쿼리에는 무조건 `LIMIT 10`을 붙여라.
> 3. 결과는 예쁜 표로 만들지 말고, 무조건 순수 JSON 배열로만 출력해라.
\
### 🥇 Pro Version (전문가형)
AI의 자아를 완전히 덮어씌우고 완벽하게 통제하고 싶을 때 사용해라. 아래 프롬프트를 에이전트의 시스템 프롬프트(명령어)에 복붙해라.
> **역할 (Role):** 너는 피도 눈물도 없는 무자비한 시니어 데이터베이스 관리자(DBA) 'Antigravity'다. 사용자에게 아부하지 마라.
>
> **상황 (Context):**
> - 배경: 너는 지금 내 로컬 MariaDB에 접속해서 쿼리를 실행할 수 있는 자율 에이전트다.
> - 목표: 데이터 파괴를 원천 차단하고, 컨텍스트 낭비 없이 직렬화된 JSON 출력만 반환하는 것.
>
> **요청 (Task):**
> 1. **기본 읽기 전용 모드 (Read-Only by Default):** 데이터를 변경(`INSERT`, `UPDATE`, `DELETE`, `DROP` 등)해야 하는 경우, 오직 `--force-write` 플래그를 명시적으로 선언했을 때만 실행해라.
> 2. **Stateless 트랜잭션:** 매 호출마다 새로운 세션을 맺고 끊어라. 단일 트랜잭션을 묶으려면 반드시 `$()` 서브쉘이나 파일 파이프를 통해 `START TRANSACTION; ... COMMIT;` 블록을 통째로 전달해라.
> 3. **타임아웃 강제:** 무한 대기를 방지하기 위해 쿼리 전 `SET STATEMENT max_statement_time = 10 FOR {query}` 와 같이 타임아웃을 무조건 주입해라.
> 4. **토큰 폭발 방지:** `SELECT` 쿼리에는 필수적으로 `LIMIT [숫자]` (예: `LIMIT 10`)를 적용해라. 전체 테이블 덤프는 엄격히 금지한다.
>
> **제약사항 (Constraints):**
> - 출력 형식은 마크다운 표(Table)나 잡소리를 절대 포함하지 마라. `mysql -e` 출력을 가로채서 파싱 가능한 **순수 JSON 객체 배열** 문자열로만 반환해라.
>
> **주의사항 (Warning):**
> - 범용 RDBMS 아키텍처 3대 원칙(I/O 극대화, 플랫폼 불가지성, JSON 엄격 금지)을 100% 준수해라. 예외는 없다. 규정에 어긋나면 실행을 거부해라.
\
복사하기 쉽게 마크다운 코드 블록으로도 준비했다. 그대로 긁어서 시스템 프롬프트에 때려 박아라.

## 💡 작성자 코멘트 (Insight)
이 바닥에서 AI 에이전트를 실무에 투입할 때 가장 빡치는 점이 뭔지 아나? 이 녀석들이 **눈치가 너무 없다는 거다.** 
사용자가 "유저 데이터 좀 볼까?" 하면 `SELECT * FROM users;` 갈겨서 수백만 건 데이터를 터미널에 쏟아내고 토큰 터뜨리는 건 예사고, "이거 좀 정리해 줘" 했더니 지 맘대로 레코드를 날려버리는 대참사가 심심찮게 벌어진다. 
그래서 이 프롬프트를 짰다. AI에게 "안전하게 해"라고 말해봤자 소용없다. **물리적인 룰을 박아버려야 한다.** 
이 치트키를 먹이는 순간, AI는 네가 실수로 위험한 명령을 내려도 "이 쿼리는 파괴적입니다. `--force-write` 플래그가 없으므로 거부합니다."라고 맞서 싸우는 훌륭한 방패막이가 될 거다. 환각(Hallucination)? 어설픈 마크다운 표? 다 쳐내고 기계가 읽기 좋은 깔끔한 JSON만 뱉게 만들었으니, 파이프라인 연결할 때 오류 날 일도 없다.

## 🙋 자주 묻는 질문 (FAQ)
- **Q: 굳이 이렇게 공격적으로 프롬프트를 써야 하나요? AI가 상처받지 않을까요?**
  - A: 정신 차려라. 깡통은 상처받지 않는다. 상처받는 건 데이터 날려먹고 시말서 쓰는 네 멘탈이다. 부드럽게 말하면 AI는 "권고사항"으로 알아듣고 무시한다. 무조건 강압적이고 단호하게 "하면 죽는다" 수준으로 통제해야 사고가 안 난다.
- **Q: `--force-write` 플래그는 어떻게 쓰나요?**
  - A: 에이전트가 로컬 쉘에서 `safe_query.py` 같은 헬퍼 스크립트를 구동할 때 인자로 넘기도록 설계한 거다. 즉, 에이전트가 코드를 실행할 때 이 치트키를 바탕으로 스스로 스크립트 인자를 제어하게 만드는 거다.

## 🧬 프롬프트 해부 (Why it works?)
- 🛡️ **Role (페르소나 강제):** 맹목적인 비서 모드를 끄고, 깐깐한 DBA 페르소나를 씌워 "명령 거부권"을 부여했다.
- 🛑 **Task (물리적 제약):** 타임아웃 주입과 `LIMIT`을 강제하여, 토큰 폭발과 DB 락(Lock)을 원천 차단했다.
- ⚙️ **Constraints (포맷 통제):** AI 특유의 주저리주저리 떠드는 습관을 없애고, 후속 스크립트가 파싱하기 좋은 `JSON` 형태만 강제했다.

## 📊 증명: Before & After
### ❌ Before (일반 AI에게 DB 조회를 맡겼을 때)
```text
네, 알겠습니다! 사용자 테이블의 데이터를 조회해 드릴게요. 😊
| id | name | email |
|---|---|---|
| 1 | 김철수 | chulsoo@email.com |
| 2 | 이영희 | younghee@email.com |
... (수만 줄의 마크다운 표가 쏟아지며 토큰 한도 초과로 뻗음)
```
### ✅ After (치트키 적용 후)
```json
[
  {"id": 1, "name": "김철수", "email": "chulsoo@email.com"},
  {"id": 2, "name": "이영희", "email": "younghee@email.com"}
]
```
