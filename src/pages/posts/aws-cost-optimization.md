# 💸 AWS 비용 50% 절감: EC2, RDS 요금 폭탄 막는 체크리스트

- **🎯 추천 대상:** 월말마다 AWS 청구서 보고 기절하는 스타트업 대표, "서버비 왜 이렇게 많이 나와요?" 소리 듣는 개발자
- **⏱️ 소요 시간:** 5분 (Cost Explorer 분석)
- **🤖 추천 모델:** ChatGPT-4o (비용 데이터 분석)

- ⭐ **난이도:** ⭐⭐☆☆☆
- ⚡️ **효과성:** ⭐⭐⭐⭐⭐
- 🚀 **활용도:** ⭐⭐⭐⭐⭐

> _"t3.medium 인스턴스 10개... 이거 다 쓰고 있는 거 맞아?"_

클라우드는 쓴 만큼 냅니다. 하지만 **안 쓰는데 켜놔서** 내는 돈이 절반입니다. 복잡한 AWS Cost Explorer 볼 필요 없습니다. AI에게 청구서를 던지고 **"어디서 돈이 새는지 찾아줘"**라고 하세요.

---

## ⚡️ 3줄 요약 (TL;DR)

1.  AWS Cost Explorer에서 '월별 서비스별 비용' CSV를 받는다.
2.  AI에게 붙여넣고 **"비효율적인 지출 찾아줘"**라고 한다.
3.  추천받은 대로 RI(예약 인스턴스)나 Savings Plan을 구매한다.

---

## 🚀 해결책: "Cloud Cost Cutter"

### 🥉 Basic Version (좀비 리소스 찾기)

사용하지 않는 자원 찾기.

> **요청:** "AWS 청구 내역을 보니 `EBS Volume` 비용이 높게 나와. 연결되지 않은(Unattached) 볼륨이나 스냅샷을 찾는 AWS CLI 명령어를 알려줘."

<br>

### 🥇 Pro Version (스팟 인스턴스 & 아키텍처 최적화)

구조적인 비용 절감.

> **역할 (Role):** 너는 AWS 비용 최적화(FinOps) 컨설턴트야.
>
> **상황 (Context):**
>
> - **현재 아키텍처:** EC2(On-Demand) x 5대 (상시 가동), RDS(MySQL) 1대.
> - **트래픽 패턴:** 낮 9~6시에만 트래픽이 몰리고 밤엔 거의 없음.
>
> **요청 (Task):**
>
> 1. **Spot Instance 도입:** 현재 워크로드에서 스팟 인스턴스를 활용해 비용을 70% 줄일 수 있는 부분을 제안해.
> 2. **Auto Scaling 전략:** 밤에는 서버를 1대만 남기고 다 끄는(Scale-in) 구체적인 `ASG` 설정값을 줘.
> 3. **Savings Plan:** 1년 약정(RI)을 걸었을 때와 비교해서 얼마나 아낄 수 있는지 표로 계산해줘.

## 💡 작성자 코멘트 (Insight)

가장 흔한 실수는 **"개발 서버(Dev/Stage)를 24시간 켜두는 것"**입니다.
`Instance Scheduler`를 써서 평일 밤과 주말에는 자동으로 꺼지게만 해도 비용의 40%가 날아갑니다. AI에게 **"Instance Scheduler 설정 CloudFormation 코드 짜줘"**라고 하세요.

## 🙋 자주 묻는 질문 (FAQ)

- **Q: 스팟 인스턴스는 갑자기 꺼지잖아요?**
  - A: 네, 그래서 Stateless한 웹 서버나 배치 작업에만 써야 합니다. DB에는 절대 쓰지 마세요.

- **Q: RDS 비용은 어떻게 줄이나요?**
  - A: 오로라(Aurora) 서버리스 v2를 쓰거나, 읽기 전용 복제본(Read Replica)을 적극 활용하세요.

---

## 🧬 프롬프트 해부 (Why it works?)

1.  **트래픽 패턴 명시:** "밤엔 트래픽 없음"이라는 힌트를 주면, AI가 **오토 스케일링**이나 **스케줄링** 같은 시간 기반 최적화 전략을 내놓습니다.
2.  **구체적 목표(70% 절감):** 스팟 인스턴스 도입 시 기대 효과를 수치로 요구하여, 도입 근거를 마련합니다.
