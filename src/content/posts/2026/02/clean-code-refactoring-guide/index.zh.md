## ⚡️ 3 行总结 (TL;DR)

1. **智能重命名：** 瞬间赋予变量与函数精准的语义化名称，大幅提升代码直观性。
2. **职责拆分：** 彻底消除冗余逻辑，精准拆分庞大函数，严格践行单一职责原则（SRP）。
3. **拥抱现代语法：** 无缝引入最新的语法特性与性能优化的最佳实践。

## 🙋 常见问题 (FAQ)

- **Q: AI 重构后的代码，真的可以直接复制粘贴到生产环境中使用吗？**
  - A: 建议在部署前**务必先运行现有的单元测试**！尽管我们在提示词中下达了“绝对禁止更改业务逻辑”的铁律，但大语言模型偶尔仍可能遗漏某些极其冷门的边缘边界条件（Edge Cases）。将其视为一份高质量的“重构草案”，再辅以人工的最终校验，才是最稳妥的最佳实践。

- **Q: 如果我的代码文件长达数千行，AI 能够一次性完美处理完吗？**
  - A: 面对超长代码，受限于大语言模型（LLM）的上下文窗口（Context Window）限制，强制一次性输入极易导致输出被无情截断或重构质量断崖式下降。正确的做法是：以功能模块或单个类/函数为单位，**分批次**交由 AI 进行逐一重构。

## 📊 证明: Before & After

### ❌ Before (输入)

```javascript
function calc(d) {
  let t = 0;
  for (let i = 0; i < d.length; i++) {
    if (d[i].price > 100) {
      t += d[i].price * 0.9;
    } else {
      t += d[i].price;
    }
  }
  return t;
}
```

### ✅ After (结果)

```javascript
/**
 * 计算购物车内商品的总价。
 * 针对单价超过 100 的商品，自动应用 10% 的折扣。
 */
function calculateTotalPrice(items) {
  const DISCOUNT_THRESHOLD = 100;
  const DISCOUNT_RATE = 0.9;

  return items.reduce((total, item) => {
    const finalPrice =
      item.price > DISCOUNT_THRESHOLD ? item.price * DISCOUNT_RATE : item.price;

    return total + finalPrice;
  }, 0);
}
```

_核心变更：将难以理解的模糊命名 `calc` 重构为语义清晰的 `calculateTotalPrice`，提取魔术数字（100, 0.9）为具名常量，并使用现代化的 `reduce` 高阶函数优雅地替代了传统的 `for` 循环。_
